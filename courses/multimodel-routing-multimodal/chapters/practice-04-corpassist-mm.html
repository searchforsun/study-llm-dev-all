flowchart TD
    Upload[附件上传] --> Classify{格式判断}

    Route -->|复杂| Plus[qwen-plus]
    Turbo & Plus --> Answer[生成回答]

Python：完整附件问答实现

python
# corpassist_attachment_qa.py — 完整管线
import os, tempfile, base64
from openai import OpenAI
import pytesseract
from pdf2image import convert_from_path

client = OpenAI(base_url="https://dashscope.aliyuncs.com/compatible-mode/v1")

class AttachmentQAPipeline:
    def __init__(self):
        self.chunks = []

    def process(self, file_path: str):
        ext = os.path.splitext(file_path)[1].lower()
        if ext == ".pdf":
            self._process_pdf(file_path)
        elif ext in (".jpg", ".png", ".jpeg", ".webp"):
            self._process_image(file_path)
        elif ext in (".mp3", ".wav", ".m4a", ".ogg"):
            self._process_audio(file_path)
        else:
            raise ValueError(f"不支持的格式: {ext}")

    def _process_pdf(self, path: str):
        images = convert_from_path(path, dpi=200)
        for i, img in enumerate(images):
            text = pytesseract.image_to_string(img, lang="chi_sim+eng")
            if text.strip():
                self.chunks.append({"source": f"p{i}", "text": text.strip(), "type": "text"})

    def _process_image(self, path: str):
        with open(path, "rb") as f:
            b64 = base64.b64encode(f.read()).decode()
        resp = client.chat.completions.create(
            model="qwen-vl-plus",
            messages=[{"role": "user", "content": [
                {"type": "image_url", "image_url": {"url": f"data:image/jpeg;base64,{b64}"}},
                {"type": "text", "text": "请详细描述这张图片的内容、文字和布局"}
            ]}]
        )
        self.chunks.append({
            "source": "img", "text": resp.choices[0].message.content, "type": "vision"
        })

    def _process_audio(self, path: str):
        with open(path, "rb") as f:
            transcript = client.audio.transcriptions.create(
                model="whisper-1", file=f, response_format="text",
            )
        self.chunks.append({"source": "audio", "text": transcript, "type": "audio"})

    def ask(self, question: str) -> str:
        context = "\n".join([c["text"][:500] for c in self.chunks[:5]])
        # 简单路由：短问答用turbo，复杂用plus
        model = "qwen-turbo" if len(question) < 30 else "qwen-plus"
        resp = client.chat.completions.create(
            model=model,
            messages=[
                {"role": "system", "content": "基于附件内容回答。如果找不到答案请诚实说明。"},
                {"role": "user", "content": f"附件内容:\n{context}\n\n问题: {question}"}
            ],
        )
        return resp.choices[0].message.content

端到端评测与上线门禁

mm metrics定义附件问答的关键指标。Golden Set确保每次改动都经过回归验证。

指标说明目标值
Recall@5检索前5命中正确答案≥ 85%
Answer Accuracy生成回答准确率≥ 90%
PDF处理延迟从上传到向量化完成≤ 30s/10页
不支持的格式非支持格式的降级率100%友好提示
Golden Set通过率回归测试通过比例≥ 90%

Python：Golden Set评测

python
# golden_eval.py — Golden Set评测
from typing import List, Dict

class GoldenEvaluator:
    def __init__(self, qa_pairs: List[Dict]):
        self.qa_pairs = qa_pairs

    def evaluate(self, pipeline: AttachmentQAPipeline) -> Dict:
        passed = 0
        results = []
        for item in self.qa_pairs:
            pipeline.process(item["file"])
            answer = pipeline.ask(item["question"])
            is_correct = item["expected"] in answer
            if is_correct:
                passed += 1
            results.append({
                "question": item["question"],
                "expected": item["expected"],
                "got": answer[:100],
                "pass": is_correct,
            })
        pass_rate = passed / len(self.qa_pairs)
        return {"pass_rate": pass_rate, "passed": passed, "total": len(self.qa_pairs), "results": results}

不推荐
  • 所有附件类型共用同一处理逻辑
  • 处理结果不缓存，每次重复解析
  • 无Golden Set回归测试
  • 不支持格式直接抛异常
推荐
  • 按格式路由到专用处理管线
  • 解析结果缓存到向量库
  • Golden Set + Review Gate门禁
  • 不支持格式返回友好提示+支持列表

常见问题（FAQ）

附件处理结果如何缓存？
文件上传后计算hash，已处理过的文件直接返回缓存。缓存存储在向量库或Redis。文件更新时重新处理并更新缓存。建议：文件hash为主键，处理结果TTL设7天。
多格式附件混合使用（如PDF+音频）怎么办？
支持批量上传，每个附件独立处理。问答时检索所有附件的chunk。注意上下文窗口限制，用Rerank筛选最相关的Top-5 chunk注入。
Golden Set需要多少条？
开始至少20条（覆盖PDF/图片/音频各5-7条）。后续每新增一个格式扩展Golden Set。上线门禁通过率≥90%。

本章结论

  • 格式路由：PDF→OCR+VL，图片→VL，音频→ASR，按类型分管线处理。
  • 检索+生成：解析后向量化存储，问答时检索+LLM生成。
  • 端到端评测：Recall@5 + Answer Accuracy + 延迟 + Golden Set通过率。
  • 缓存策略：文件hash去重，避免重复处理。
  • Review Gate：Golden Set通过率≥90%作为上线门禁。
