为什么要学本章

DashScope可能因过载返回429，或因区域故障不可用。model degrade让S5自动降级到备用模型或区域。region failover在杭州Region故障时切到上海Region。unavailable fallback需处理429、超时和完全不可用三种场景。

学完你能
  • 实现多级降级链（模型→区域→缓存）
  • 配置区域容灾切换
  • 实现熔断保护防止雪崩
  • 处理429限流和超时重试
本章先记住3件事
  • 降级三阶梯：主模型→备模型→备区域→缓存→友好提示。
  • 429要退避：被限流时用指数退避重试，而非立即重试。
  • 熔断防雪崩：连续失败超过阈值后熔断，给下游恢复时间。

多级降级链

降级链从主模型开始，逐级fallback到更便宜的模型、备份区域和缓存结果。

四级降级链
flowchart TD
    Req[请求] --> L1[L1: qwen-plus 主区域]
    L1 -->|成功| Done[返回]
    L1 -->|429/超时| L2[L2: qwen-turbo 主区域]
    L2 -->|成功| Done
    L2 -->|失败| L3[L3: qwen-turbo 备区域]
    L3 -->|成功| Done
    L3 -->|失败| L4[L4: 返回缓存结果]
    L4 -->|无缓存| Error[友好提示]

四级降级：主模型→备模型→备区域→缓存/提示。

Python：降级链实现

python
# degrade_chain.py — 多级降级链
from openai import OpenAI, APIStatusError, APITimeoutError
import time

class DegradeChain:
    def __init__(self):
        self.primary = OpenAI(base_url="https://dashscope.aliyuncs.com/compatible-mode/v1")
        self.backup_region = OpenAI(base_url="https://dashscope-shanghai.aliyuncs.com/compatible-mode/v1")
        self.cache = {}

    def infer(self, prompt: str) -> str:
        last_error = None

        # L1: 主模型 qwen-plus 主区域
        for attempt in range(2):
            try:
                resp = self.primary.chat.completions.create(
                    model="qwen-plus", messages=[{"role": "user", "content": prompt}],
                    timeout=30,
                )
                return resp.choices[0].message.content
            except APIStatusError as e:
                last_error = e
                if e.status_code == 429:
                    time.sleep(2 ** attempt)
                else:
                    break
            except APITimeoutError as e:
                last_error = e
                break

        # L2: 降级到 qwen-turbo 主区域
        try:
            resp = self.primary.chat.completions.create(
                model="qwen-turbo", messages=[{"role": "user", "content": prompt}],
                timeout=15,
            )
            return f"[降级turbo] {resp.choices[0].message.content}"
        except Exception as e:
            last_error = e

        # L3: 切换到备区域
        try:
            resp = self.backup_region.chat.completions.create(
                model="qwen-turbo", messages=[{"role": "user", "content": prompt}],
                timeout=15,
            )
            return f"[降级备区域] {resp.choices[0].message.content}"
        except Exception as e:
            last_error = e

        # L4: 缓存或提示
        cached = self.cache.get(prompt[:50])
        if cached:
            return f"[缓存结果] {cached}"

        return "服务暂时不可用，请稍后重试"

熔断保护与429处理

熔断器在连续失败率超过阈值时断开，停止对下游的调用，给服务恢复时间。

Java：Resilience4j熔断

java
// Resilience4j熔断配置
@Configuration
public class CircuitBreakerConfig {

    @Bean
    public CircuitBreaker dashScopeCB() {
        return CircuitBreaker.of("dashscope",
            io.github.resilience4j.circuitbreaker.CircuitBreakerConfig.custom()
                .failureRateThreshold(50)          // 50%失败率触发熔断
                .waitDurationInOpenState(Duration.ofSeconds(30))  // 30s后尝试恢复
                .slidingWindowSize(10)             // 统计最近10次调用
                .build()
        );
    }

    @Bean
    public TimeLimiter timeLimiter() {
        return TimeLimiter.of(Duration.ofSeconds(30));
    }
}

// DegradeService.java — 降级服务
@Service
public class DegradeService {
    private final CircuitBreaker cb;
    private final TimeLimiter tl;

    public String callWithDegrade(String prompt) {
        return Try.ofSupplier(
            cb.decorateSupplier(() ->
                tl.executeFutureSupplier(
                    () -> CompletableFuture.supplyAsync(() -> callLLM(prompt))
                )
            )
        ).recover(e -> "服务降级中，请稍后重试").get();
    }
}

Python：指数退避重试

python
# retry_with_backoff.py — 指数退避
import random, time

def retry_with_backoff(fn, max_retries: int = 3, base_delay: float = 1.0):
    for attempt in range(max_retries):
        try:
            return fn()
        except APIStatusError as e:
            if e.status_code == 429:
                delay = base_delay * (2 ** attempt) + random.uniform(0, 0.5)
                time.sleep(delay)
            else:
                raise
        except APITimeoutError:
            delay = base_delay * (2 ** attempt)
            time.sleep(delay)
    raise Exception("全部重试失败")
不推荐
  • 429时立即重试（加重限流）
  • 无熔断，持续调用已故障的服务
  • 所有模型走同一条链路
  • 无区域容灾
推荐
  • 指数退避重试429
  • 熔断器保护（50%失败率断开30s）
  • 四级降级链（模型→区域→缓存）
  • 多区域部署+自动切换
CorpAssist · S5容灾实战 CorpAssist S5依赖DashScope杭州Region。某次大促流量突增触发429限流。降级链自动：1) 429→指数退避重试2次→仍429；2) 降级到qwen-turbo（成本更低，更快）；3) 切换到上海Region备用Endpoint；4) 缓存命中返回缓存。整个过程用户感知为「响应稍慢」，无错误提示。

常见问题（FAQ）

降级链中降级到更便宜的模型会不会影响质量？
降级场景下「可用性优先于质量」。qwen-turbo处理FAQ/简单问答质量足够。如果是对质量敏感的创作场景，降级策略应返回缓存或提示稍后重试。
熔断的阈值怎么设？
推荐：50%失败率+10次调用窗口+30s半开。先保守（50%），观察后调整。太低会频繁熔断影响可用性，太高则保护作用不足。
区域容灾需要两地部署吗？
DashScope多Region天然支持。只需要配置两个base_url，一个为主Region，一个为备Region。备Region只做fallback，日常流量不分配。

本章结论

  • 四级降级：主模型→备模型→备区域→缓存，逐级保障可用性。
  • 429退避：指数退避重试+随机扰动，避免加重限流。
  • 熔断保护：50%失败率触发熔断，30s后尝试半开恢复。
  • 区域容灾：主备两Region，备Region仅降级时使用。
  • 缓存兜底：降级到末级时返回缓存结果或友好提示。
