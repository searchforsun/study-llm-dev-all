评测指标

Routing eval 的核心指标包括 quality metric、cost metric 和 router KPI：

类别指标计算方式S5 目标
质量 Golden Set 通过率 模型回答是否满足断言 ≥ 95%
质量 用户采纳率 用户是否使用了生成结果 ≥ 80%
成本 平均单次调用成本 总花费 / 请求数 ≤ ¥0.005
成本 各模型调用占比 turbo/plus/vl 的比例 turbo ≥ 60%
延迟 P95 首 token 延迟 从发起到首 token ≤ 1s
python 
# routing_eval.py — 路由策略离线评测
import json, time, yaml
from openai import OpenAI

client = OpenAI(
    base_url="https://dashscope.aliyuncs.com/compatible-mode/v1",
    api_key="YOUR_KEY",
)

# 黄金数据集
GOLDEN_SET = [
    {"prompt": "公司的年假政策是什么", "intent": "faq", "assert": "年假"},
    {"prompt": "写一个产品宣传语", "intent": "creation", "assert": None},
    {"prompt": "这张图里有什么（附图URL）", "intent": "vision", "assert": None},
]

def eval_routing(strategy_name: str, routing_rules: dict):
    """评测一个路由策略"""
    results = {"total": 0, "passed": 0, "cost": 0.0, "models": {}}

    for item in GOLDEN_SET:
        # 按路由规则选择模型
        rule = next(
            r for r in routing_rules["routing"]["rules"]
            if r["intent"] == item["intent"]
        )
        model = rule["model"]
        results["models"][model] = results["models"].get(model, 0) + 1

        t0 = time.time()
        resp = client.chat.completions.create(
            model=model,
            messages=[{"role": "user", "content": item["prompt"]}],
        )
        content = resp.choices[0].message.content
        latency = time.time() - t0

        results["total"] += 1
        # 质量检查
        if item["assert"]:
            if item["assert"] in content:
                results["passed"] += 1
        else:
            results["passed"] += 1 if len(content) > 20 else 0

        # 成本估算（示例价格）
        input_tokens = resp.usage.prompt_tokens
        output_tokens = resp.usage.completion_tokens
        results["cost"] += (input_tokens * 0.0003 + output_tokens * 0.0006) / 1000

    print(f"\n=== 策略: {strategy_name} ===")
    print(f"通过率: {results['passed']}/{results['total']} ({results['passed']/results['total']*100:.0f}%)")
    print(f"总成本: ¥{results['cost']:.4f}")
    print(f"模型分布: {results['models']}")
    return results

with open("routing-rules.yaml") as f:
    rules = yaml.safe_load(f)

eval_routing("v1", rules)

A/B 测试

AB test route 在生产环境中对比两个路由策略：

python 
# ab_test.py — 在线 A/B 对比
import random, time, statistics

class ABRouter:
    """A/B 路由测试器"""
    def __init__(self, strategy_a: dict, strategy_b: dict, a_weight: float = 0.5):
        self.strategies = {"A": strategy_a, "B": strategy_b}
        self.a_weight = a_weight
        self.stats = {"A": {"count": 0, "latencies": [], "costs": []},
                      "B": {"count": 0, "latencies": [], "costs": []}}

    def route(self, intent: str, prompt: str):
        group = "A" if random.random() < self.a_weight else "B"
        rules = self.strategies[group]["routing"]["rules"]
        rule = next(r for r in rules if r["intent"] == intent)

        t0 = time.time()
        resp = client.chat.completions.create(
            model=rule["model"],
            messages=[{"role": "user", "content": prompt}],
        )
        latency = time.time() - t0

        self.stats[group]["count"] += 1
        self.stats[group]["latencies"].append(latency)
        return resp.choices[0].message.content, group

    def report(self):
        for group, s in self.stats.items():
            avg_latency = statistics.mean(s["latencies"]) if s["latencies"] else 0
            print(f"{group}: {s['count']} 请求, 平均延迟 {avg_latency*1000:.0f}ms")

# 使用
ab = ABRouter(strategy_a=rules, strategy_b=rules)
for _ in range(100):
    ab.route("faq", "公司的年假政策")
ab.report()

Win rate 衡量新策略是否优于旧策略：在相同 prompt 上，新策略输出被用户采纳的比例高于旧策略。

先想 10 秒：在线 A/B 测试时，同一用户可能同时体验 A/B 策略，有问题吗？

有。同一用户在不同请求中体验到不一致的模型质量，可能导致困惑。建议按用户 ID hash 固定分组，确保同一用户始终在同一组。

回归检测

Route regression 在 CI 中自动运行：路由策略变更时，自动在 Golden Set 上跑评测，通过率低于阈值则阻止合并。

yaml 
# .github/workflows/routing-eval.yml
name: 路由策略回归测试
on:
  pull_request:
    paths:
      - 'routing-rules.yaml'
jobs:
  eval:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - run: pip install openai pyyaml
      - run: python routing_eval.py
      - name: 检查通过率
        run: |
          echo "Golden Set 通过率: $(python -c 'from routing_eval import *; print(eval_routing(...))')"

本章结论

  • 指标：quality metric + cost metric + 延迟 = 路由三要素。
  • A/B：AB test route 按用户分组对比。
  • 回归：route regression 作为 CI 门禁。
