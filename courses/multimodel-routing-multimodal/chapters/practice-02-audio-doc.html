音频ASR与分片处理

Whisper是开源语音识别方案。长音频需分片处理，单次处理的音频时长建议不超过10分钟。

音频处理管线
flowchart LR
    Audio[录音文件] --> Split[分片10min]
    Split --> ASR[Whisper/DashScope ASR]
    ASR --> Text[转写文本]
    Text --> Diarize[说话人分离]
    Diarize --> Summary[会议摘要]
    Summary --> VectorDB[(向量库)]

长音频分片转写后做说话人分离和摘要，最终存入向量库。

Python：音频转文字

python
# audio_asr.py — 语音转文字
from openai import OpenAI
from pydub import AudioSegment

client = OpenAI(base_url="https://dashscope.aliyuncs.com/compatible-mode/v1")

def transcribe_audio(audio_path: str) -> str:
    """DashScope语音识别"""
    with open(audio_path, "rb") as f:
        transcript = client.audio.transcriptions.create(
            model="whisper-1", file=f, response_format="text",
        )
    return transcript

def process_long_audio(audio_path: str, chunk_minutes: int = 10):
    """长音频分片处理"""
    audio = AudioSegment.from_file(audio_path)
    chunk_ms = chunk_minutes * 60 * 1000
    segments = []
    for i, start in enumerate(range(0, len(audio), chunk_ms)):
        chunk = audio[start:start + chunk_ms]
        chunk_path = f"/tmp/audio_chunk_{i}.mp3"
        chunk.export(chunk_path, format="mp3")
        text = transcribe_audio(chunk_path)
        segments.append({"segment": i + 1, "text": text, "start_sec": start // 1000})
    return segments

def summarize_meeting(segments: list) -> str:
    """会议摘要生成"""
    full_text = "\n".join([s["text"] for s in segments])
    resp = client.chat.completions.create(
        model="qwen-plus",
        messages=[{"role": "user", "content": f"请总结以下会议内容：\n{full_text}"}]
    )
    return resp.choices[0].message.content

视频帧采样与摘要

视频处理通过提取关键帧，用VL模型理解每一帧的内容，最后汇总为视频摘要。

Python：视频帧采样

python
# video_summary.py — 视频摘要
import cv2, base64, os
from openai import OpenAI

client = OpenAI(base_url="https://dashscope.aliyuncs.com/compatible-mode/v1")

def extract_frames(video_path: str, interval_sec: int = 10, max_frames: int = 10):
    """提取视频关键帧"""
    cap = cv2.VideoCapture(video_path)
    fps = cap.get(cv2.CAP_PROP_FPS)
    frames = []
    count = 0
    while len(frames) < max_frames:
        ret, frame = cap.read()
        if not ret:
            break
        if count % int(fps * interval_sec) == 0:
            path = f"/tmp/frame_{count}.jpg"
            cv2.imwrite(path, frame)
            frames.append(path)
        count += 1
    cap.release()
    return frames

def summarize_video(video_path: str) -> str:
    frames = extract_frames(video_path)
    descriptions = []
    for i, fp in enumerate(frames[:10]):
        with open(fp, "rb") as f:
            b64 = base64.b64encode(f.read()).decode()
        resp = client.chat.completions.create(
            model="qwen-vl-plus",
            messages=[{"role": "user", "content": [
                {"type": "image_url", "image_url": {"url": f"data:image/jpg;base64,{b64}"}},
                {"type": "text", "text": f"描述第{i+1}帧的画面内容"}
            ]}]
        )
        descriptions.append(resp.choices[0].message.content)

    summary = client.chat.completions.create(
        model="qwen-plus",
        messages=[{"role": "user", "content": f"汇总以下帧描述为视频摘要:\n{descriptions}"}]
    )
    return summary.choices[0].message.content

模态边界与降级

Modality boundary定义系统当前能处理的模态范围，超出范围时明确降级。

模态处理方式边界条件不支持时的行为
文本直接输入LLM所有文本格式-
图片VL模型理解≤20MB, JPG/PNG提示格式不支持
音频ASR转文字≤1h, 16KHz采样率提示超长需分段
视频帧采样+VL≤10min, ≤10帧提示仅支持10分钟内
3D模型不支持-明确返回不支持
不推荐
  • 不支持的文件格式直接返回500
  • 长音频不分片，超时失败
  • 视频全部帧处理，耗时过长
  • 不告诉用户系统支持什么格式
推荐
  • 不支持格式返回友好提示+支持列表
  • 长音频分片+并行ASR处理
  • 视频关键帧采样（间隔10s, 最多10帧）
  • 上传前在前端做格式校验

常见问题（FAQ）

Whisper和DashScope ASR哪个好？
Whisper开源免费，支持本地部署，中文准确率约90%。DashScope ASR准确率约95%，有说话人分离能力。推荐低成本场景用Whisper，生产场景用DashScope ASR。
视频帧采样间隔怎么确定？
取决于视频类型。产品介绍视频（内容变化慢）间隔可设30s；培训视频（内容密集）间隔10s。自适应方案：检测帧间差异，变化超过阈值才采样。
音频分片后如何保证上下文连贯？
分片时保留前后各30s重叠（overlap），ASR转写后拼接时去重。说话人分离后按说话人分段更合理。

本章结论

  • ASR转文字：音频先转文本再进入标准管线，长音频分片+说话人分离。
  • 视频帧采样：间隔采样+VL理解+汇总摘要，控制帧数在10帧内。
  • 模态边界：明确定义支持和不支持的模态，不支持时友好降级。
  • 文档扩展：表格/流程图等特殊格式需专用解析器。
  • 分片策略：长音频/视频必须分片，overlap保证上下文连贯。
