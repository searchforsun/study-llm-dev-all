Spring Router

CorpAssist Java 轨通过 spring router 在请求入口解析意图，再委托对应 ChatClient 调用 DashScope 兼容网关。核心不是「写 if-else 选模型」，而是把 路由策略 章节的 route rule 编译成可测试的 Bean。

组件职责S5 场景
RoutePolicyLoader 加载 routing.yaml，校验 model 在能力矩阵内 启动期 + 配置热刷新
IntentClassifier 规则/轻量模型判定 intent faq / vision_qa / content_draft
ChatClientRouter 按 intent 选 ChatClient，失败走 Fallback 附件问答超时切备用 VL
多 ChatModel Bean

为 turbo / plus / vl-plus 各注册 ChatModel，Router 用 @Qualifier 注入，避免单模型硬编码。

ChatClient 门面

业务 Service 只依赖 Router 返回的 ChatClient，不感知底层模型名。

Advisor 链

路由前后可挂成本计数、审计日志 Advisor，与 NS3 指标衔接。

version: "2026.05"
policy:
  default_intent: chat
  rules:
    - intent: faq
      model: qwen-turbo
      temperature: 0.2
    - intent: vision_qa
      model: qwen-vl-plus
      temperature: 0.1
    - intent: content_draft
      model: qwen-plus
      temperature: 0.7
  fallback:
    faq: [qwen-turbo, qwen-plus]
    vision_qa: [qwen-vl-plus, qwen-plus]
  cost_tier:
    qwen-turbo: low
    qwen-vl-plus: medium
    qwen-plus: high

Spring Router 核心实现

java
@Service
public class ChatClientRouter {

  private final RoutePolicy policy;
  private final Map<String, ChatClient> clientsByModel;
  private final IntentClassifier classifier;

  public ChatResponse route(RouteRequest req) {
    String intent = classifier.classify(req);
    RouteRule rule = policy.ruleFor(intent);
    List<String> chain = policy.fallbackChain(intent);

    for (String model : chain) {
      try {
        ChatClient client = clientsByModel.get(model);
        return client.prompt()
            .options(ChatOptions.builder()
                .model(model)
                .temperature(rule.temperature())
                .build())
            .user(req.userMessage())
            .call()
            .chatResponse();
      } catch (Exception ex) {
        log.warn("model {} failed, try next in fallback", model, ex);
      }
    }
    throw new RouteExhaustedException(intent);
  }
}

Python 路由服务

Python RouteService：读取同一 YAML

python
from dataclasses import dataclass
from pathlib import Path
import yaml
from openai import OpenAI

@dataclass
class RouteDecision:
    intent: str
    model: str
    fallback_chain: list[str]

class RouteService:
    def __init__(self, config_path: Path, client: OpenAI):
        self._policy = yaml.safe_load(config_path.read_text(encoding="utf-8"))
        self._client = client

    def decide(self, intent: str) -> RouteDecision:
        rules = {r["intent"]: r for r in self._policy["policy"]["rules"]}
        rule = rules.get(intent, rules[self._policy["policy"]["default_intent"]])
        chain = self._policy["policy"]["fallback"].get(intent, [rule["model"]])
        return RouteDecision(intent=intent, model=rule["model"], fallback_chain=chain)

    def complete(self, intent: str, messages: list[dict]) -> dict:
        decision = self.decide(intent)
        for model in decision.fallback_chain:
            try:
                resp = self._client.chat.completions.create(
                    model=model,
                    messages=messages,
                    temperature=next(
                        r["temperature"] for r in self._policy["policy"]["rules"]
                        if r["model"] == model
                    ),
                )
                return {"model": model, "content": resp.choices[0].message.content}
            except Exception:
                continue
        raise RuntimeError(f"fallback exhausted for intent={intent}")

关键术语速览

spring router · python router · dual stack route · shared config · chat client router · route service · config sync · unified policy

自检：Python 批处理选了 qwen-max，Java 在线仍走 turbo，算双栈成功吗？

不算。双栈成功指同一 intent 映射同一 primary model；若批处理需更高质量，应新增 intent（如 content_draft_premium）写入 YAML，而非两栈各写各的。

NS3 成本路由会在哪一层介入？
advanced 章在 Router 前增加 cost-aware 前置分类；本章只需保证 Router 读 YAML 的 cost_tier 字段并暴露指标，便于 NS3 仪表盘聚合。
DashScope VL 是否走同一 OpenAI 兼容客户端？
是。Spring AI 与 Python OpenAI SDK 均配置 base_url=https://dashscope.aliyuncs.com/compatible-mode/v1；VL 差异在 message 多模态字段，路由层仍按 intent 选 qwen-vl-plus。
