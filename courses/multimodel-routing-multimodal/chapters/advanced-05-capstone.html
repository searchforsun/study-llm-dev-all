为什么要学本章

前四章把成本、降级、治理、面试架构讲清了，但请求从哪进、路由在哪一层执行决定能否上线。CorpAssist NS3 · S5 的生产路径是：api gateway 鉴权限流 → BFF 聚合 → route middleware 选模型 → DashScope。本章作为 gateway capstone，与 LLM 应用后端llm-application-backend 网关章对接，完成 Spring + Python 双栈的 unified entry。

本章先记住 3 件事
  • 统一入口：外部只认 Gateway URL，不暴露各模型厂商 endpoint。
  • 先鉴权后路由：无 tenant 上下文，budget 与 compliance 无法执行。
  • 中间层可 Sidecar：route-service 可嵌 BFF 或独立 Python 服务，策略仍单源。

统一入口

unified entry 对外暴露 /v1/mm/chat、/v1/mm/vision 等路径，内部由 Spring Cloud Gateway 按 predicate 转发到 BFF 或 Python route-service。S5 编辑器、企业门户、开放 API 共用同一入口，便于 rate limit route 与 WAF。

              args:
                redis-rate-limiter.replenishRate: 50

            - name: JwtAuth

与 LLM 应用后端 · Gateway 同构；本课补上 Route Middleware 盒。

不推荐
  • 前端直连 DashScope API Key
  • Gateway 内嵌 model 选择 if-else
  • 批处理与在线 API 不同域名且无统一 audit
推荐
  • Gateway 只做入口治理，路由在 Middleware
  • 统一 trace_id 贯穿 GW → BFF → Router → LLM
  • 策略 YAML 与 llm-application-backend 配置中心同源
CorpAssist · S5 开放 API · 租户限流 品牌 A 日配额 1 万次；Gateway rate limit route 按 X-Tenant-Id 令牌桶；触顶返回 429，BFF 不再进入 Router，保护 spend cap 与模型配额。
先想 10 秒：Route Middleware 放在 BFF 内还是独立服务？

在线路径可 BFF 内嵌 Spring Router（低延迟）；Python 批处理走独立 route-service。两者读同一 policy Git 仓库，通过 config sync 保持一致 — 见 basics-03。

BFF 调用 Route Middleware（Spring）

java
@RestController
@RequestMapping("/v1/mm")
public class MultimodalChatController {

  private final ChatClientRouter router;

  @PostMapping("/chat")
  public ChatResponse chat(
      @RequestHeader("X-Tenant-Id") String tenantId,
      @RequestHeader("X-Request-Id") String requestId,
      @RequestBody ChatRequest body) {
    RouteContext ctx = RouteContext.builder()
        .tenantId(tenantId)
        .requestId(requestId)
        .campaignId(body.campaignId())
        .build();
    return router.route(body.toRouteRequest(ctx));
  }
}

常见误区
  • Gateway 做路由 — Gateway 不应懂 qwen-turbo vs vl-plus；职责混淆难维护。
  • BFF 直连厂商 SDK — 绕过 Middleware，成本/降级/审计全丢。
  • 忽略 batch 路径 — S5 批量活动走 Python，须同 audit schema。

判断练习

  1. 团队在 Gateway GlobalFilter 里写「若 prompt 含图片则 model=vl-max」。请指出架构问题与推荐分层。

    参考答案

    Gateway 不应承担 route middleware 业务路由；且无法复用 Python 批处理。应：Gateway 鉴权限流 → BFF → Middleware 读 routing.yaml 决策。

  2. Gateway 429 与 Router spend cap 触顶，用户看到的体验应如何区分？

    参考答案

    Gateway 429：请求未进 BFF，提示「调用过于频繁」。spend cap：已进入 Router，返回 degrade text「今日生成额度已满，草稿已保存」。HTTP 码与文案应不同，audit 字段分别 rate_limited / budget_exceeded。
