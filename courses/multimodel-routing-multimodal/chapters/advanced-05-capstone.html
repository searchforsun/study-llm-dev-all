Java：Spring Cloud Gateway配置

java
// GatewayConfig.java — 网关配置
@Configuration
public class GatewayConfig {

    @Bean
    public RouteLocator s5Routes(RouteLocatorBuilder builder) {
        return builder.routes()
            // S5 Chat 路由
            .route("s5-chat", r -> r.path("/api/s5/chat/**")
                .filters(f -> f
                    .circuitBreaker(config -> config
                        .setFallbackUri("forward:/fallback/s5")
                        .setName("s5CB"))
                    .requestRateLimiter(config -> config
                        .setRateLimiter(redisRateLimiter()))
                    .addRequestHeader("X-Route-Version", "v2.4"))
                .uri("lb://s5-router"))

            // 多模态路由
            .route("s5-vision", r -> r.path("/api/s5/vision/**")
                .filters(f -> f
                    .circuitBreaker(config -> config
                        .setFallbackUri("forward:/fallback/vision")
                        .setName("visionCB")))
                .uri("lb://s5-vision-service"))

            // 附件上传路由
            .route("s5-upload", r -> r.path("/api/s5/attachment/**")
                .uri("lb://s5-attachment-service"))
            .build();
    }

    @Bean
    public RedisRateLimiter redisRateLimiter() {
        return new RedisRateLimiter(100, 200);  // 100 req/s, 200 burst
    }
}

// FallbackController.java — 降级处理
@RestController
public class FallbackController {
    @GetMapping("/fallback/s5")
    public String s5Fallback() {
        return "{\"error\":\"S5服务暂时不可用\",\"fallback\":true}";
    }
}

Python：BFF路由服务

python
# bff_router.py — BFF路由服务
from fastapi import FastAPI, Header, HTTPException
from typing import Optional

app = FastAPI()

INTENT_MODEL_MAP = {
    "faq": "qwen-turbo",
    "creation": "qwen-plus",
    "vision": "qwen-vl-plus",
    "audio": "qwen-audio-turbo",
}

@app.post("/api/bff/route")
async def bff_route(
    intent: str,
    user_id: str = Header(None),
    x_route_version: str = Header("v2.4"),
):
    model = INTENT_MODEL_MAP.get(intent)
    if not model:
        raise HTTPException(400, f"未知意图: {intent}")

    return {
        "model": model,
        "version": x_route_version,
        "endpoint": f"https://dashscope.aliyuncs.com/compatible-mode/v1",
    }

网关层特征与BFF模式

BFF integration将路由决策放在BFF层。前端只传意图标识（如intent=faq），BFF根据策略决定后端模型。

功能网关层实现BFF层实现
鉴权JWT/OAuth2统一验证透传user_id到业务
限流令牌桶/Redis Rate Limiter业务级限流（按用户）
路由按路径分发到服务按意图分发到模型
熔断服务级Circuit Breaker模型级超时控制
审计请求日志调用审计（含成本）
不推荐
  • 前端直接调模型API（暴露Key）
  • 每个服务自己实现限流和熔断
  • 路由策略散落在各业务代码中
  • 无统一入口，难以做审计
推荐
  • 网关统一入口+鉴权+限流
  • 网关层熔断+降级配置
  • BFF层负责意图→模型路由
  • 审计日志统一收集
CorpAssist · S5网关集成 CorpAssist S5通过Spring Cloud Gateway统一入口。前端请求先过网关鉴权，令牌桶限流（100req/s）。网关根据Path分发到S5服务：/api/s5/chat→S5 Router，/api/s5/vision→VL Service。BFF层根据intent参数决定用turbo/plus/vl模型。网关层配置熔断器，S5服务不可用时自动降级。

常见问题（FAQ）

网关层做路由和BFF层做路由有什么区别？
网关层路由是按服务/路径分发（如/chat→S5、/vision→VL）。BFF层路由是按业务意图分发（如faq→qwen-turbo、creation→qwen-plus）。推荐双层：网关负责服务级分发，BFF负责模型级分发。
网关层的限流和熔断怎么和业务联动？
网关层限流保护后端不被突发流量打爆。熔断器在后端响应慢或连续错误时断开。联动：网关熔断后返回降级页面，BFF层收到熔断信号后自动切换模型策略。
网关集成后如何做灰度发布？
网关支持权重路由：10%流量走新版路由策略，90%走旧版。通过Spring Cloud Gateway的Weight路由谓词实现。灰度期间监控错误率和延迟，确认稳定后逐步提升权重。

本章结论

  • 网关统一入口：Spring Cloud Gateway作为统一入口，集中管理鉴权/限流/路由。
  • 三阶流程：先鉴权→再限流→最后路由，每阶职责清晰。
  • 双层路由：网关层按服务分发，BFF层按意图分发。
  • 熔断降级：网关层Resilience4j熔断器，后端不可用时自动降级。
  • 灰度发布：网关权重路由支持路由策略的灰度发布。
