百炼多模态API调用

DashScope多模态API兼容OpenAI格式，Python和Java都可直接接入。图片支持base64和URL两种方式。

Python和Java都支持多模态API调用，图片传输方式影响性能和延迟。

Python：多模态API三种模式

python
# dashscope_mm.py — 通义多模态调用
from openai import OpenAI
import base64

client = OpenAI(
    base_url="https://dashscope.aliyuncs.com/compatible-mode/v1",
    api_key="YOUR_DASHSCOPE_KEY",
)

# 模式1: base64内联（适合小图）
def image_qa_base64(image_path: str, question: str) -> str:
    with open(image_path, "rb") as f:
        b64 = base64.b64encode(f.read()).decode()
    resp = client.chat.completions.create(
        model="qwen-vl-plus",
        messages=[{"role": "user", "content": [
            {"type": "image_url", "image_url": {"url": f"data:image/jpeg;base64,{b64}"}},
            {"type": "text", "text": question}
        ]}],
        max_tokens=512,
    )
    return resp.choices[0].message.content

# 模式2: URL引用（适合大图/网络图）
def image_qa_url(image_url: str, question: str) -> str:
    resp = client.chat.completions.create(
        model="qwen-vl-plus",
        messages=[{"role": "user", "content": [
            {"type": "image_url", "image_url": {"url": image_url}},
            {"type": "text", "text": question}
        ]}],
    )
    return resp.choices[0].message.content

# 模式3: 多轮多模态对话
def multi_turn_mm():
    messages = [
        {"role": "user", "content": [
            {"type": "image_url", "image_url": {"url": "https://example.com/img1.jpg"}},
            {"type": "text", "text": "这张图展示的是什么？"}
        ]},
        {"role": "assistant", "content": "这是一张产品功能对比图"},
        {"role": "user", "content": [{"type": "text", "text": "图中的主要区别是什么？"}]}
    ]
    resp = client.chat.completions.create(
        model="qwen-vl-plus", messages=messages,
    )
    return resp.choices[0].message.content

Spring AI多模态扩展

Spring AI通过OpenAiChatModel兼容DashScope。多模态消息使用Media对象传递图片。

Java：Spring AI多模态配置

java
// DashScopeMMConfig.java — 多模态配置
@Configuration
public class DashScopeMMConfig {

    @Bean
    public ChatClient dashScopeVLClient() {
        var model = OpenAiChatModel.builder()
            .baseUrl("https://dashscope.aliyuncs.com/compatible-mode/v1")
            .apiKey(System.getenv("DASHSCOPE_API_KEY"))
            .defaultOptions(OpenAiChatOptions.builder()
                .model("qwen-vl-plus")
                .maxTokens(1024)
                .temperature(0.3)
                .build())
            .build();
        return ChatClient.builder(model).build();
    }
}

// MMService.java — 多模态理解服务
@Service
public class MMService {
    private final ChatClient vlClient;

    public MMService(ChatClient dashScopeVLClient) {
        this.vlClient = dashScopeVLClient;
    }

    public String analyzeImage(String imageUrl, String question) {
        var media = new Media(MimeTypeUtils.parseMimeType("image/jpeg"),
                              URI.create(imageUrl));
        var userMessage = new UserMessage(question, List.of(media));
        return vlClient.prompt(new Prompt(userMessage))
            .call().content();
    }

    public String analyzeImageBase64(byte[] imageBytes, String question) {
        String b64 = Base64.getEncoder().encodeToString(imageBytes);
        String dataUrl = "data:image/jpeg;base64," + b64;
        return analyzeImage(dataUrl, question);
    }
}
不推荐
  • 用纯文本API调多模态模型，图片丢失
  • base64编码超大图片（>10MB）导致请求超时
  • 多轮对话中图片只传一次，后续轮次丢失
  • 不设置max_tokens导致响应截断
推荐
  • 使用多模态消息格式（content数组）
  • 大图用URL引用而非base64
  • 多轮对话每轮都传上下文
  • 合理设置max_tokens(512-1024)

常见问题（FAQ）

DashScope和OpenAI的多模态API有什么差异？
DashScope兼容OpenAI格式，message结构和参数基本一致。差异：1) 模型名不同（qwen-vl系列 vs gpt-4o）；2) 最大图片尺寸DashScope为20MB；3) DashScope多模态支持音频输入。推荐用OpenAI SDK设置不同base_url即可切换。
图片base64内联和URL引用哪种好？
base64：适合小图（<1MB），无需额外存储，请求自包含。URL：适合大图，减少请求体大小，可复用缓存。推荐：上传到OSS后用URL引用，兼具性能和可缓存性。
多轮多模态对话如何保持上下文？
每轮消息中都要包含图片的引用（URL或base64）。如果上下文太长，可将上一轮VL理解结果摘要后注入文本，避免重复传图占用大量tokens。

本章结论

  • 兼容OpenAI：DashScope多模态API与OpenAI格式兼容，Python/Java SDK通用。
  • 消息格式：多模态content是数组，含text和image_url两种类型。
  • 图片传输：base64小图、URL大图，建议上传OSS后用URL引用。
  • Spring AI扩展：通过Media对象传递图片，OpenAiChatModel可直接对接DashScope。
  • 模型选择：plus通用、max高精度、audio-turbo语音，按场景选型。
