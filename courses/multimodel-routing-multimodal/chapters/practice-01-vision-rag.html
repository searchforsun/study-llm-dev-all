PDF图文解析管线

PDF文档可能包含扫描件（纯图片）和电子版（可选文字）。图文RAG的处理管线先做版面分析，区分文本块和图片块，各自走不同处理路径。

图文RAG管线
flowchart TD
    PDF[PDF文档] --> Layout[版面分析]
    Layout --> TextBlock[文本块]
    Layout --> ImageBlock[图片块]
    TextBlock --> OCR[OCR文字提取]
    ImageBlock --> VL[VL模型生成描述]
    OCR --> Chunk[按策略分块]
    VL --> Chunk
    Chunk --> Embed[向量化]
    Embed --> VectorDB[(Milvus向量库)]
    Query[用户提问] --> Retrieve[混合检索]
    VectorDB --> Retrieve
    Retrieve --> Rerank[BGE重排序]
    Rerank --> Answer[LLM生成回答]

vision rag管线：图文分拆后各自处理，最终统一向量化检索。

Python：PDF图文解析

python
# vision_rag.py — 图文解析与向量化
import pytesseract
from pdf2image import convert_from_path
from openai import OpenAI
import base64, io

client = OpenAI(base_url="https://dashscope.aliyuncs.com/compatible-mode/v1")

class VisionRAGPipeline:
    def __init__(self):
        self.chunks = []

    def parse_pdf(self, pdf_path: str, dpi: int = 200):
        images = convert_from_path(pdf_path, dpi=dpi)
        for page_num, img in enumerate(images):
            # OCR提取文本
            text = pytesseract.image_to_string(img, lang="chi_sim+eng")
            if text.strip():
                self.chunks.append({
                    "type": "text", "content": text.strip(),
                    "page": page_num, "source": "ocr"
                })
            # VL模型描述图片布局
            buf = io.BytesIO()
            img.save(buf, format="PNG")
            b64 = base64.b64encode(buf.getvalue()).decode()
            resp = client.chat.completions.create(
                model="qwen-vl-plus",
                messages=[{"role": "user", "content": [
                    {"type": "image_url", "image_url": {"url": f"data:image/png;base64,{b64}"}},
                    {"type": "text", "text": "详细描述此页面的布局、图片、图表和文字结构"}
                ]}]
            )
            desc = resp.choices[0].message.content
            self.chunks.append({
                "type": "image_desc", "content": desc,
                "page": page_num, "source": "vl"
            })
        return self.chunks

    def embed_all(self):
        """将所有chunk向量化"""
        vectors = []
        for c in self.chunks:
            resp = client.embeddings.create(
                model="text-embedding-v3", input=c["content"][:512]
            )
            c["vector"] = resp.data[0].embedding
            vectors.append(c)
        return vectors

// VisionRAGService.java — Spring AI图文RAG
@Service
public class VisionRAGService {
    private final VectorStore vectorStore;
    private final VisionService visionService;

    public List searchWithImage(String textQuery, String imageUrl) {
        // 文本查询向量化
        String imageDesc = visionService.describeImage(imageUrl);
        String combined = textQuery + " " + imageDesc;

        return vectorStore.similaritySearch(
            SearchRequest.query(combined).withTopK(5));
    }
}

图文分块策略与向量索引

图文分块的关键在于保持图文关联性。同一页面的文本和图片描述需要关联存储。

分块策略适用场景chunk结构
逐页分块品牌手册、产品目录每页一个chunk，含文本+图片描述
段落分块技术文档、白皮书按段落切分，图片附在所属段落
表格独立分块报价单、数据表整表为一个chunk，保留行列结构
不推荐
  • PDF中图片完全忽略，只索引文字
  • OCR结果不做清洗，噪音直接入库
  • 文本和图片描述用不同向量模型
  • 不分块，整篇文档一个向量
推荐
  • 图片用VL生成描述后一并向量化
  • OCR结果做拼写校正和去噪
  • 文本和图片描述用同一Embedding模型
  • 按页/段落分块，图文关联存储

常见问题（FAQ）

OCR识别错误如何影响检索？
OCR错误导致关键词丢失，召回率下降。建议：1) 用高精度OCR引擎（如阿里云文档智能）；2) OCR后做拼写校正；3) 向量检索对拼写错误有一定容忍度，但关键字检索影响大。
VL模型描述图片的延迟多高？
qwen-vl-plus单张图片约2-5秒。50页PDF含20张图片约40-100秒。建议：1) 后台异步处理；2) 跳过小尺寸图标（<50x50px）；3) 对相似图片缓存描述。
图文向量能不能用同一个模型？
可以。图文都用text-embedding-v3向量化（图片先转文字描述）。统一向量空间保证跨模态检索质量。如果用多模态Embedding模型（如CLIP），可以直接向量化图片，但中文场景text-embedding-v3效果更好。

本章结论

  • 图文分拆处理：OCR提取文本 + VL模型描述图片，各自优化。
  • 统一向量化：文本和图片描述用同一Embedding模型，同一向量空间。
  • 版面分析：先做layout parse再分块，表格/图片/文字各自最佳策略。
  • OCR误差处理：拼写校正 + 向量容忍度，双层保障。
  • 异步处理：PDF解析耗时长，需异步执行并缓存结果。
