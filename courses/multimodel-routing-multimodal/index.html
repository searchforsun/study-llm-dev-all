价格与上下文

价格分层 (price tier) 决定了路由优先级。最贵的模型留给出价最高的场景。

模型输入价格 (¥/1k tokens)输出价格 (¥/1k tokens)上下文窗口S5 路由优先级
qwen-turbo 0.0003 0.0006 128K 1 (优先)
qwen-plus 0.0008 0.002 128K 2 (升档)
qwen-vl-plus 0.005 0.01 32K 3 (仅视觉)
qwen-max 0.02 0.06 128K 4 (仅复杂推理)

上下文窗口 (context window) 决定了模型能「看到」多少输入。S5 场景中：FAQ 问答通常 < 4K tokens；附件 PDF 理解可能需要 32K+ tokens；长文档摘要需要 128K 窗口。

不推荐
  • 所有请求一律 qwen-plus（成本失控）
  • 不看上下文窗口，超长文档被截断
  • 不区分文本和 VL 模型，图片传给纯文本模型
  • 路由策略无版本管理，改策略全靠口述
推荐
  • 按意图分 tier，turbo 优先、plus 升档
  • 用 token 计数预估窗口需求
  • 上传附件时自动判断是否为图片
  • model catalog 用 YAML 维护并 Git 版本管理

Python 能力矩阵查询

python 
# 从 YAML 加载模型目录，按场景查询最优模型
import yaml
from pathlib import Path

class ModelCatalog:
    def __init__(self, path: str = "model-catalog.yaml"):
        with open(path) as f:
            self.models = yaml.safe_load(f)["models"]

    def best_for_scenario(self, scenario: str, needs_vision: bool = False):
        """按场景选择最适合的模型"""
        candidates = [m for m in self.models if scenario in m["suitable_for"]]
        if needs_vision:
            candidates = [m for m in candidates if "vision" in m["capabilities"]]
        # 按 tier 排序（economy 优先）
        tier_order = {"economy": 0, "balanced": 1, "premium": 2}
        candidates.sort(key=lambda m: tier_order.get(m["tier"], 99))
        return candidates[0] if candidates else None

    def estimate_cost(self, model_id: str, input_tokens: int, output_tokens: int) -> float:
        """估算调用成本"""
        model = next(m for m in self.models if m["id"] == model_id)
        return (input_tokens * model["price_per_1k_input"] +
                output_tokens * model["price_per_1k_output"]) / 1000

# 使用示例
catalog = ModelCatalog()
model = catalog.best_for_scenario("content_creation")
print(f"推荐模型: {model['id']} (tier={model['tier']})")
cost = catalog.estimate_cost(model["id"], 1000, 500)
print(f"预估成本: ¥{cost:.4f}")

import org.springframework.ai.openai.OpenAiChatModel;
import org.springframework.ai.openai.OpenAiChatOptions;
import org.springframework.context.annotation.Bean;
import org.springframework.context.annotation.Configuration;

@Configuration
public class ModelCatalogConfig {

    @Bean
    public OpenAiChatModel turboChatModel() {
        return OpenAiChatModel.builder()
            .baseUrl("https://dashscope.aliyuncs.com/compatible-mode/v1")
            .apiKey(System.getenv("DASHSCOPE_API_KEY"))
            .defaultOptions(OpenAiChatOptions.builder()
                .model("qwen-turbo")
                .maxTokens(2048)
                .build())
            .build();
    }

    @Bean

        return OpenAiChatModel.builder()

            .defaultOptions(OpenAiChatOptions.builder()
                .model("qwen-plus")
                .maxTokens(4096)
                .build())
            .build();

常见问题（FAQ）

能力矩阵应该谁来维护？多久更新一次？
由平台团队维护，至少每季度更新一次。厂商发布新模型版本时（如 qwen2.5 升级）应立即更新。建议将 model-catalog.yaml 纳入 Git 仓库，变更走 PR 审批流程。
开源模型如何加入能力矩阵？
私有化部署的开源模型在 catalog 中标记 provider=self-hosted。能力评估需要跑 Golden Set 评测，确认质量后再加入路由候选。通常开源模型作为商用模型的 fallback 或沙箱环境使用。
多模态模型（VL）的上下文窗口为什么比文本模型小？
视觉模型需要处理图像 token（一张图通常占几百到几千 tokens），相同上下文窗口下能承载的文字量更少。S5 场景中长文档优先走文本模型，仅在需要看图时路由 VL 模型。
GPT-4o 和 Qwen 系列在矩阵中的位置如何决定？
GPT-4o 作为对标基准（benchmark），不进入默认生产路由。通过定期跑 Golden Set 对比两者质量，当 Qwen 在某个场景明显落后时才考虑使用 GPT-4o 作为补充。

本章结论

  • 能力矩阵：capability matrix 是路由策略的基础数据，必须定期维护。
  • 商用 vs 开源：commercial model 生产主力，open source llm 补充沙箱与离线。
  • 价格分层：price tier 指导路由优先级，贵模型留给高价值场景。
  • 上下文：context window 决定文档适配性，需前置 token 计数。

级联与 Fallback

Cascade routing（级联路由） 先尝试小模型，不满足预期再升档：

python 
# 级联路由实现
from openai import OpenAI
import re

dashscope = OpenAI(
    base_url="https://dashscope.aliyuncs.com/compatible-mode/v1",
    api_key="YOUR_KEY",
)

TIERS = [
    {"model": "qwen-turbo", "cost": 0.0003},
    {"model": "qwen-plus", "cost": 0.0008},
    {"model": "qwen-max", "cost": 0.02},
]

def quality_check(response: str) -> bool:
    """检查输出质量是否达标"""
    if len(response) < 10:
        return False
    if "抱歉" in response or "我不知道" in response:
        return False
    return True

def cascade_infer(prompt: str, min_quality: bool = True):
    """级联推理：先试便宜的，不达标再升档"""
    for tier in TIERS:
        resp = dashscope.chat.completions.create(
            model=tier["model"],
            messages=[{"role": "user", "content": prompt}],
        )
        result = resp.choices[0].message.content
        if not min_quality or quality_check(result):
            print(f"使用 {tier['model']}，花费 ¥{tier['cost']:.4f}")
            return result
        print(f"{tier['model']} 质量不达标，升档")
    return result  # 最后一个模型无论如何返回

# 使用
result = cascade_infer("解释一下量子计算的基本原理")
print(result[:200])

Fallback 链 处理模型不可用场景：

java 
// Spring AI — 带 fallback 链的路由服务
import org.springframework.ai.chat.client.ChatClient;
import org.springframework.ai.openai.OpenAiChatModel;
import org.springframework.stereotype.Service;
import org.slf4j.Logger;
import org.slf4j.LoggerFactory;

@Service
public class RoutingService {

    private static final Logger log = LoggerFactory.getLogger(RoutingService.class);
    private final OpenAiChatModel turboModel;
    private final OpenAiChatModel plusModel;
    private final OpenAiChatModel vlModel;

    public RoutingService(
            OpenAiChatModel turboModel,
            OpenAiChatModel plusModel,
            OpenAiChatModel vlModel) {
        this.turboModel = turboModel;
        this.plusModel = plusModel;
        this.vlModel = vlModel;
    }

    /**
     * 带 fallback 链的路由推理
     */
    public String route(String intent, String message, boolean hasImage) {
        // 首先选择主模型
        OpenAiChatModel primary;
        String modelName;

        if (hasImage) {
            primary = vlModel;
            modelName = "qwen-vl-plus";
        } else if ("creation".equals(intent)) {
            primary = plusModel;
            modelName = "qwen-plus";
        } else {
            primary = turboModel;
            modelName = "qwen-turbo";
        }

        // 尝试主模型，失败时 fallback
        Exception lastError = null;
        for (int attempt = 0; attempt < 2; attempt++) {
            try {
                return ChatClient.builder(primary).build()
                    .prompt().user(message).call().content();
            } catch (Exception e) {
                log.warn("{} 推理失败: {}", modelName, e.getMessage());
                lastError = e;
                // fallback 到 turbo
                primary = turboModel;
                modelName = "qwen-turbo";
            }
        }
        throw new RuntimeException("所有模型均不可用", lastError);
    }
}
先想 10 秒：级联路由中质量检查 false 但模型输出其实正常，怎么避免？

质量检查规则不能太严格。建议用「模型自身评价」：让模型自评置信度，或只用客观指标（长度、关键词等）。生产环境中用人工抽标矫正质检模型。

路由策略配置与管理

Router policy 通过 model selector 组件执行：

策略描述S5 适用场景
intent routing 按意图分类选择模型 FAQ vs 创作 vs 附件问答
cascade routing 先小模型，不达标升档 文案生成（先 turbo 草稿再 plus 精修）
fallback chain 模型不可用时切换备选 所有场景的降级保障
primary backup 主备切换 DashScope 故障时切 OpenAI
常见误区
  • 路由策略不做日志 — 不知道请求走了哪条路径，无法排查故障。
  • Fallback 链无限重试 — 所有模型都失败时应直接报错，而不是循环尝试。
  • 质量检查开销过高 — 每次升档都调用大模型检查质量，成本反而更高。

常见问题（FAQ）

意图分类的准确率要多少才够？
建议目标 90%+。低于 90% 时，低质量路由可能将 FAQ 错误路由到 plus 导致成本上升，或将创作路由到 turbo 导致质量下降。可以通过人机协作持续优化分类器。
级联路由会导致响应变慢吗？
会。每级升档增加一次推理延迟。建议给每级设超时上限，并在总延迟超时前返回当前最佳结果，避免用户无限等待。
Fallback 链应该多深？
推荐 2-3 级：主模型 → 低档模型 → 缓存结果 → 错误消息。太深的 fallback 链会显著增加延迟且成功率递减。
路由策略如何做 A/B 测试？
在 routing-rules.yaml 中支持多版本策略，通过 header 或租户 ID 分流。详见 路由效果评测 章节。

本章结论

  • 意图路由：intent routing 按请求场景分配最优模型。
  • 级联：cascade routing 先便宜后贵，质量不达标升档。
  • Fallback：fallback chain 保障模型不可用时的服务连续性。

import yaml

    base_url="https://dashscope.aliyuncs.com/compatible-mode/v1",
    api_key="YOUR_DASHSCOPE_KEY",
)

class RouteRequest(BaseModel):
    intent: str
    message: str
    has_attachment: bool = False

class RouteResponse(BaseModel):
    model: str
    content: str

@app.post("/v1/route", response_model=RouteResponse)
def route(req: RouteRequest):
    rules = CONFIG["routing"]["rules"]
    selected = next(
        (r for r in rules if r["intent"] == req.intent),
        rules[-1]  # 默认
    )
    model = selected["model"]
    try:
        resp = client.chat.completions.create(
            model=model,
            messages=[{"role": "user", "content": req.message}],
        )
        return RouteResponse(model=model, content=resp.choices[0].message.content)
    except Exception:
        for fb in selected.get("fallback", []):
            try:
                resp = client.chat.completions.create(
                    model=fb, messages=[{"role": "user", "content": req.message}],
                )
                return RouteResponse(model=fb, content=resp.choices[0].message.content)
            except Exception:
                continue
        raise HTTPException(503, "所有模型不可用")

import time
import yaml
from pathlib import Path

class RouteConfig:
    """热重载路由配置"""
    def __init__(self, path="routing-rules.yaml"):
        self.path = Path(path)
        self._mtime = 0
        self._rules = self._load()

    def _load(self):
        mtime = self.path.stat().st_mtime
        if mtime > self._mtime:
            with open(self.path) as f:
                self._rules = yaml.safe_load(f)
                self._mtime = mtime
        return self._rules

    def get_model(self, intent: str) -> str:
        self._load()
        for rule in self._rules["routing"]["rules"]:
            if rule["intent"] == intent:
                return rule["model"]
        return "qwen-turbo"

常见问题（FAQ）

双栈路由如何保证路由规则同步？
推荐将 routing-rules.yaml 放在共享存储（如 NFS、Git 子模块），双栈定时轮询或监听文件变更。更成熟的方案是使用配置中心（Nacos/Etcd），双栈通过 SDK 监听变更。
Python 路由服务挂了对 Spring 有影响吗？
Spring 应缓存路由规则，路由服务不可用时使用缓存规则降级。Python 路由服务本身应多副本部署，避免单点故障。
路由决策应该放在客户端还是服务端？
服务端。客户端（IDE 插件、前端）不应做路由决策，因为它们可能被篡改。路由决策在 BFF 层完成，客户端只透传意图标识。
双栈路由如何监控和审计？
每个路由请求打印日志：{intent, model_selected, cost_estimate, latency}。双栈日志汇聚到同一 ELK/Prometheus，按 model 和 intent 维度聚合。详见 路由治理。

def eval_routing(strategy_name: str, routing_rules: dict):
    """评测一个路由策略"""
    results = {"total": 0, "passed": 0, "cost": 0.0, "models": {}}

    for item in GOLDEN_SET:
        # 按路由规则选择模型
        rule = next(
            r for r in routing_rules["routing"]["rules"]
            if r["intent"] == item["intent"]
        )
        model = rule["model"]
        results["models"][model] = results["models"].get(model, 0) + 1

        t0 = time.time()
        resp = client.chat.completions.create(
            model=model,
            messages=[{"role": "user", "content": item["prompt"]}],
        )
        content = resp.choices[0].message.content
        latency = time.time() - t0

        results["total"] += 1
        # 质量检查
        if item["assert"]:
            if item["assert"] in content:
                results["passed"] += 1
        else:
            results["passed"] += 1 if len(content) > 20 else 0

        # 成本估算（示例价格）
        input_tokens = resp.usage.prompt_tokens
        output_tokens = resp.usage.completion_tokens
        results["cost"] += (input_tokens * 0.0003 + output_tokens * 0.0006) / 1000

    print(f"\n=== 策略: {strategy_name} ===")
    print(f"通过率: {results['passed']}/{results['total']} ({results['passed']/results['total']*100:.0f}%)")
    print(f"总成本: ¥{results['cost']:.4f}")
    print(f"模型分布: {results['models']}")
    return results

# ab_test.py — 在线 A/B 对比
import random, time, statistics

class ABRouter:
    """A/B 路由测试器"""
    def __init__(self, strategy_a: dict, strategy_b: dict, a_weight: float = 0.5):
        self.strategies = {"A": strategy_a, "B": strategy_b}
        self.a_weight = a_weight
        self.stats = {"A": {"count": 0, "latencies": [], "costs": []},
                      "B": {"count": 0, "latencies": [], "costs": []}}

    def route(self, intent: str, prompt: str):
        group = "A" if random.random() < self.a_weight else "B"
        rules = self.strategies[group]["routing"]["rules"]
        rule = next(r for r in rules if r["intent"] == intent)

        t0 = time.time()
        resp = client.chat.completions.create(
            model=rule["model"],
            messages=[{"role": "user", "content": prompt}],
        )
        latency = time.time() - t0

        self.stats[group]["count"] += 1
        self.stats[group]["latencies"].append(latency)
        return resp.choices[0].message.content, group

    def report(self):
        for group, s in self.stats.items():
            avg_latency = statistics.mean(s["latencies"]) if s["latencies"] else 0
            print(f"{group}: {s['count']} 请求, 平均延迟 {avg_latency*1000:.0f}ms")

# 使用
ab = ABRouter(strategy_a=rules, strategy_b=rules)
for _ in range(100):
    ab.route("faq", "公司的年假政策")
ab.report()

# vision_rag.py — 图文解析与向量化
import pytesseract
from pdf2image import convert_from_path
from openai import OpenAI
import base64, io

client = OpenAI(base_url="https://dashscope.aliyuncs.com/compatible-mode/v1")

class VisionRAGPipeline:
    def __init__(self):
        self.chunks = []

    def parse_pdf(self, pdf_path: str, dpi: int = 200):
        images = convert_from_path(pdf_path, dpi=dpi)
        for page_num, img in enumerate(images):
            # OCR提取文本
            text = pytesseract.image_to_string(img, lang="chi_sim+eng")
            if text.strip():
                self.chunks.append({
                    "type": "text", "content": text.strip(),
                    "page": page_num, "source": "ocr"
                })
            # VL模型描述图片布局
            buf = io.BytesIO()
            img.save(buf, format="PNG")
            b64 = base64.b64encode(buf.getvalue()).decode()
            resp = client.chat.completions.create(
                model="qwen-vl-plus",
                messages=[{"role": "user", "content": [
                    {"type": "image_url", "image_url": {"url": f"data:image/png;base64,{b64}"}},
                    {"type": "text", "text": "详细描述此页面的布局、图片、图表和文字结构"}
                ]}]
            )
            desc = resp.choices[0].message.content
            self.chunks.append({
                "type": "image_desc", "content": desc,
                "page": page_num, "source": "vl"
            })
        return self.chunks

    def embed_all(self):
        """将所有chunk向量化"""
        vectors = []
        for c in self.chunks:
            resp = client.embeddings.create(
                model="text-embedding-v3", input=c["content"][:512]
            )
            c["vector"] = resp.data[0].embedding
            vectors.append(c)
        return vectors

OCR识别错误如何影响检索？

VL模型描述图片的延迟多高？

图文向量能不能用同一个模型？
可以。图文都用text-embedding-v3向量化（图片先转文字描述）。统一向量空间保证跨模态检索质量。如果用多模态Embedding模型（如CLIP），可以直接向量化图片，但中文场景text-embedding-v3效果更好。

# audio_asr.py — 语音转文字
from openai import OpenAI
from pydub import AudioSegment

client = OpenAI(base_url="https://dashscope.aliyuncs.com/compatible-mode/v1")

def transcribe_audio(audio_path: str) -> str:
    """DashScope语音识别"""
    with open(audio_path, "rb") as f:
        transcript = client.audio.transcriptions.create(
            model="whisper-1", file=f, response_format="text",
        )
    return transcript

def process_long_audio(audio_path: str, chunk_minutes: int = 10):
    """长音频分片处理"""
    audio = AudioSegment.from_file(audio_path)
    chunk_ms = chunk_minutes * 60 * 1000
    segments = []
    for i, start in enumerate(range(0, len(audio), chunk_ms)):
        chunk = audio[start:start + chunk_ms]
        chunk_path = f"/tmp/audio_chunk_{i}.mp3"
        chunk.export(chunk_path, format="mp3")
        text = transcribe_audio(chunk_path)
        segments.append({"segment": i + 1, "text": text, "start_sec": start // 1000})
    return segments

def summarize_meeting(segments: list) -> str:
    """会议摘要生成"""
    full_text = "\n".join([s["text"] for s in segments])
    resp = client.chat.completions.create(
        model="qwen-plus",
        messages=[{"role": "user", "content": f"请总结以下会议内容：\n{full_text}"}]
    )
    return resp.choices[0].message.content

视频帧采样与摘要

视频处理通过提取关键帧，用VL模型理解每一帧的内容，最后汇总为视频摘要。

Python：视频帧采样

python
# video_summary.py — 视频摘要
import cv2, base64, os
from openai import OpenAI

client = OpenAI(base_url="https://dashscope.aliyuncs.com/compatible-mode/v1")

def extract_frames(video_path: str, interval_sec: int = 10, max_frames: int = 10):
    """提取视频关键帧"""
    cap = cv2.VideoCapture(video_path)
    fps = cap.get(cv2.CAP_PROP_FPS)
    frames = []
    count = 0
    while len(frames) < max_frames:
        ret, frame = cap.read()
        if not ret:
            break
        if count % int(fps * interval_sec) == 0:
            path = f"/tmp/frame_{count}.jpg"
            cv2.imwrite(path, frame)
            frames.append(path)
        count += 1
    cap.release()
    return frames

def summarize_video(video_path: str) -> str:
    frames = extract_frames(video_path)
    descriptions = []
    for i, fp in enumerate(frames[:10]):
        with open(fp, "rb") as f:
            b64 = base64.b64encode(f.read()).decode()
        resp = client.chat.completions.create(
            model="qwen-vl-plus",
            messages=[{"role": "user", "content": [
                {"type": "image_url", "image_url": {"url": f"data:image/jpg;base64,{b64}"}},
                {"type": "text", "text": f"描述第{i+1}帧的画面内容"}
            ]}]
        )
        descriptions.append(resp.choices[0].message.content)

    summary = client.chat.completions.create(
        model="qwen-plus",
        messages=[{"role": "user", "content": f"汇总以下帧描述为视频摘要:\n{descriptions}"}]
    )
    return summary.choices[0].message.content

DashScope和OpenAI的多模态API有什么差异？

# corpassist_attachment_qa.py — 完整管线
import os, tempfile, base64
from openai import OpenAI
import pytesseract
from pdf2image import convert_from_path

client = OpenAI(base_url="https://dashscope.aliyuncs.com/compatible-mode/v1")

class AttachmentQAPipeline:
    def __init__(self):
        self.chunks = []

    def process(self, file_path: str):
        ext = os.path.splitext(file_path)[1].lower()
        if ext == ".pdf":
            self._process_pdf(file_path)
        elif ext in (".jpg", ".png", ".jpeg", ".webp"):
            self._process_image(file_path)
        elif ext in (".mp3", ".wav", ".m4a", ".ogg"):
            self._process_audio(file_path)
        else:
            raise ValueError(f"不支持的格式: {ext}")

    def _process_pdf(self, path: str):
        images = convert_from_path(path, dpi=200)
        for i, img in enumerate(images):
            text = pytesseract.image_to_string(img, lang="chi_sim+eng")
            if text.strip():
                self.chunks.append({"source": f"p{i}", "text": text.strip(), "type": "text"})

    def _process_image(self, path: str):
        with open(path, "rb") as f:
            b64 = base64.b64encode(f.read()).decode()
        resp = client.chat.completions.create(
            model="qwen-vl-plus",
            messages=[{"role": "user", "content": [
                {"type": "image_url", "image_url": {"url": f"data:image/jpeg;base64,{b64}"}},
                {"type": "text", "text": "请详细描述这张图片的内容、文字和布局"}
            ]}]
        )
        self.chunks.append({
            "source": "img", "text": resp.choices[0].message.content, "type": "vision"
        })

    def _process_audio(self, path: str):
        with open(path, "rb") as f:
            transcript = client.audio.transcriptions.create(
                model="whisper-1", file=f, response_format="text",
            )
        self.chunks.append({"source": "audio", "text": transcript, "type": "audio"})

    def ask(self, question: str) -> str:
        context = "\n".join([c["text"][:500] for c in self.chunks[:5]])
        # 简单路由：短问答用turbo，复杂用plus
        model = "qwen-turbo" if len(question) < 30 else "qwen-plus"
        resp = client.chat.completions.create(
            model=model,
            messages=[
                {"role": "system", "content": "基于附件内容回答。如果找不到答案请诚实说明。"},
                {"role": "user", "content": f"附件内容:\n{context}\n\n问题: {question}"}
            ],
        )
        return resp.choices[0].message.content

# golden_eval.py — Golden Set评测
from typing import List, Dict

class GoldenEvaluator:
    def __init__(self, qa_pairs: List[Dict]):
        self.qa_pairs = qa_pairs

    def evaluate(self, pipeline: AttachmentQAPipeline) -> Dict:
        passed = 0
        results = []
        for item in self.qa_pairs:
            pipeline.process(item["file"])
            answer = pipeline.ask(item["question"])
            is_correct = item["expected"] in answer
            if is_correct:
                passed += 1
            results.append({
                "question": item["question"],
                "expected": item["expected"],
                "got": answer[:100],
                "pass": is_correct,
            })
        pass_rate = passed / len(self.qa_pairs)
        return {"pass_rate": pass_rate, "passed": passed, "total": len(self.qa_pairs), "results": results}

# cost_router.py — 成本感知路由
from openai import OpenAI
import hashlib, json, time

client = OpenAI(base_url="https://dashscope.aliyuncs.com/compatible-mode/v1")

MODEL_COST = {
    "qwen-turbo": 0.0003,  # ¥/1K tokens
    "qwen-plus": 0.002,
    "qwen-vl-plus": 0.008,
}

class CostAwareRouter:
    def __init__(self, daily_budget: float = 10.0):
        self.daily_cost = 0.0
        self.daily_budget = daily_budget
        self.exact_cache = {}
        self.semantic_cache = []

    def classify(self, prompt: str) -> str:
        resp = client.chat.completions.create(
            model="qwen-turbo",
            messages=[{"role": "user", "content": f"分类请求: {prompt}\n返回faq/creation/vision之一"}],
            max_tokens=10, temperature=0,
        )
        return resp.choices[0].message.content.strip()

    def route(self, prompt: str, needs_vision: bool = False) -> str:
        # 1. 精确缓存
        key = hashlib.md5(prompt.encode()).hexdigest()
        if key in self.exact_cache:
            return self.exact_cache[key]

        # 2. 语义缓存检查
        for cached_q, cached_a in self.semantic_cache:
            if self._similar(prompt, cached_q) > 0.92:
                return cached_a

        # 3. 预算检查
        if self.daily_cost >= self.daily_budget:
            return "【成本控制】日预算已超限，请稍后重试"

        # 4. 路由决定
        if needs_vision:
            model = "qwen-vl-plus"
        else:
            intent = self.classify(prompt)
            model = {"faq": "qwen-turbo", "creation": "qwen-plus", "vision": "qwen-vl-plus"}.get(intent, "qwen-turbo")

        resp = client.chat.completions.create(model=model, messages=[{"role": "user", "content": prompt}])
        result = resp.choices[0].message.content

        # 5. 更新成本
        tokens = resp.usage.prompt_tokens
        cost = tokens / 1000 * MODEL_COST.get(model, 0.002)
        self.daily_cost += cost

        # 6. 缓存
        self.exact_cache[key] = result
        self.semantic_cache.append((prompt, result))
        return result

    def _similar(self, a: str, b: str) -> float:
        """简单文本相似度"""
        set_a, set_b = set(a), set(b)
        return len(set_a & set_b) / len(set_a | set_b) if set_a | set_b else 0

# semantic_cache.py — 语义缓存
import numpy as np

class SemanticCache:
    def __init__(self, threshold: float = 0.92, max_size: int = 1000):
        self.threshold = threshold
        self.max_size = max_size
        self.keys = []    # 向量
        self.values = []  # 回答

    def _embed(self, text: str) -> np.ndarray:
        resp = client.embeddings.create(model="text-embedding-v3", input=text)
        return np.array(resp.data[0].embedding)

    def get(self, query: str) -> str | None:
        q_vec = self._embed(query)
        for k_vec, val in zip(self.keys, self.values):
            sim = np.dot(q_vec, k_vec) / (np.linalg.norm(q_vec) * np.linalg.norm(k_vec))
            if sim >= self.threshold:
                return val
        return None

    def put(self, query: str, answer: str):
        if len(self.keys) >= self.max_size:
            self.keys.pop(0)
            self.values.pop(0)
        self.keys.append(self._embed(query))
        self.values.append(answer)

四级降级：主模型→备模型→备区域→缓存/提示。

Python：降级链实现

python
# degrade_chain.py — 多级降级链
from openai import OpenAI, APIStatusError, APITimeoutError
import time

class DegradeChain:
    def __init__(self):
        self.primary = OpenAI(base_url="https://dashscope.aliyuncs.com/compatible-mode/v1")
        self.backup_region = OpenAI(base_url="https://dashscope-shanghai.aliyuncs.com/compatible-mode/v1")
        self.cache = {}

    def infer(self, prompt: str) -> str:
        last_error = None

        # L1: 主模型 qwen-plus 主区域
        for attempt in range(2):
            try:
                resp = self.primary.chat.completions.create(
                    model="qwen-plus", messages=[{"role": "user", "content": prompt}],
                    timeout=30,
                )
                return resp.choices[0].message.content
            except APIStatusError as e:
                last_error = e
                if e.status_code == 429:
                    time.sleep(2 ** attempt)
                else:
                    break
            except APITimeoutError as e:
                last_error = e
                break

        # L2: 降级到 qwen-turbo 主区域
        try:
            resp = self.primary.chat.completions.create(
                model="qwen-turbo", messages=[{"role": "user", "content": prompt}],
                timeout=15,
            )
            return f"[降级turbo] {resp.choices[0].message.content}"
        except Exception as e:
            last_error = e

        # L3: 切换到备区域
        try:
            resp = self.backup_region.chat.completions.create(
                model="qwen-turbo", messages=[{"role": "user", "content": prompt}],
                timeout=15,
            )
            return f"[降级备区域] {resp.choices[0].message.content}"
        except Exception as e:
            last_error = e

        # L4: 缓存或提示
        cached = self.cache.get(prompt[:50])
        if cached:
            return f"[缓存结果] {cached}"

        return "服务暂时不可用，请稍后重试"

# retry_with_backoff.py — 指数退避
import random, time

def retry_with_backoff(fn, max_retries: int = 3, base_delay: float = 1.0):
    for attempt in range(max_retries):
        try:
            return fn()
        except APIStatusError as e:
            if e.status_code == 429:
                delay = base_delay * (2 ** attempt) + random.uniform(0, 0.5)
                time.sleep(delay)
            else:
                raise
        except APITimeoutError:
            delay = base_delay * (2 ** attempt)
            time.sleep(delay)
    raise Exception("全部重试失败")

常见问题（FAQ）

降级链中降级到更便宜的模型会不会影响质量？
降级场景下「可用性优先于质量」。qwen-turbo处理FAQ/简单问答质量足够。如果是对质量敏感的创作场景，降级策略应返回缓存或提示稍后重试。
熔断的阈值怎么设？
推荐：50%失败率+10次调用窗口+30s半开。先保守（50%），观察后调整。太低会频繁熔断影响可用性，太高则保护作用不足。
区域容灾需要两地部署吗？
DashScope多Region天然支持。只需要配置两个base_url，一个为主Region，一个为备Region。备Region只做fallback，日常流量不分配。

  • 四级降级：主模型→备模型→备区域→缓存，逐级保障可用性。

  • 熔断保护：50%失败率触发熔断，30s后尝试半开恢复。
  • 区域容灾：主备两Region，备Region仅降级时使用。
  • 缓存兜底：降级到末级时返回缓存结果或友好提示。

# policy_loader.py — 策略加载与回滚
import yaml, os
from typing import Optional

class PolicyManager:
    def __init__(self, policy_dir: str = "./policies"):
        self.policy_dir = policy_dir
        self.current_version: Optional[str] = None
        self.history = []

    def load(self, version: str) -> dict:
        path = os.path.join(self.policy_dir, f"routing-rules.{version}.yaml")
        with open(path) as f:
            policy = yaml.safe_load(f)
        self.current_version = version
        self.history.append({"version": version, "action": "load"})
        return policy

    def rollback(self) -> dict:
        """回滚到上一版本"""
        if len(self.history) < 2:
            raise ValueError("没有可回滚的版本")
        prev_version = self.history[-2]["version"]
        return self.load(prev_version)

    def validate(self, policy: dict) -> bool:
        """校验策略合法性"""
        if "version" not in policy:
            return False
        if "routing" not in policy:
            return False
        if "rules" not in policy["routing"]:
            return False
        for rule in policy["routing"]["rules"]:
            if "intent" not in rule or "model" not in rule:
                return False
        return True

import json, time, uuid

class RouteAuditLogger:
    def __init__(self, log_path: str = "route_audit.log"):
        self.log_path = log_path

    def log_call(self, user_id: str, intent: str, model: str,
                 cost: float, latency_ms: int, degraded: bool = False):
        entry = {
            "type": "call",
            "id": uuid.uuid4().hex[:12],
            "timestamp": time.time(),
            "user_id": user_id,
            "intent": intent,
            "model": model,
            "cost": cost,
            "latency_ms": latency_ms,
            "degraded": degraded,
        }
        self._write(entry)

    def log_change(self, operator: str, from_v: str, to_v: str, reason: str):
        entry = {
            "type": "change",
            "timestamp": time.time(),
            "operator": operator,
            "from_version": from_v,
            "to_version": to_v,
            "reason": reason,
        }
        self._write(entry)

    def query(self, model: str = None, user_id: str = None,
              limit: int = 100) -> list:
        results = []
        with open(self.log_path) as f:
            for line in f:
                entry = json.loads(line)
                if model and entry.get("model") != model:
                    continue
                if user_id and entry.get("user_id") != user_id:
                    continue
                results.append(entry)
        return results[-limit:]

    def _write(self, entry: dict):
        with open(self.log_path, "a") as f:
            f.write(json.dumps(entry) + "\n")

常见问题（FAQ）

审计日志占用存储怎么办？
按时间分区存储，保留最近90天可在线查询，更早的数据归档到廉价存储。每条审计记录约200字节，日均5万次调用约10MB/天，90天约900MB，完全可控。
策略回滚有风险吗？
有。回滚可能引入旧bug。推荐：1) 回滚前做CI评测；2) 灰度发布新版策略（10%→50%→100%）；3) 回滚后监控指标（失败率/延迟/成本）确认正常。
多人同时修改策略怎么处理？
Git天然处理冲突。两个PR同时修改routing-rules.yaml，先合并的生效，后合并的需要解决冲突。冲突通常发生在同一intent的不同规则上。推荐分工：不同团队负责不同intent的规则。

// GatewayConfig.java — 网关配置
@Configuration
public class GatewayConfig {

    @Bean
    public RouteLocator s5Routes(RouteLocatorBuilder builder) {
        return builder.routes()
            // S5 Chat 路由
            .route("s5-chat", r -> r.path("/api/s5/chat/**")
                .filters(f -> f
                    .circuitBreaker(config -> config
                        .setFallbackUri("forward:/fallback/s5")
                        .setName("s5CB"))
                    .requestRateLimiter(config -> config
                        .setRateLimiter(redisRateLimiter()))
                    .addRequestHeader("X-Route-Version", "v2.4"))
                .uri("lb://s5-router"))

            // 多模态路由
            .route("s5-vision", r -> r.path("/api/s5/vision/**")
                .filters(f -> f
                    .circuitBreaker(config -> config
                        .setFallbackUri("forward:/fallback/vision")
                        .setName("visionCB")))
                .uri("lb://s5-vision-service"))

            // 附件上传路由
            .route("s5-upload", r -> r.path("/api/s5/attachment/**")
                .uri("lb://s5-attachment-service"))
            .build();
    }

    @Bean
    public RedisRateLimiter redisRateLimiter() {
        return new RedisRateLimiter(100, 200);  // 100 req/s, 200 burst
    }
}

from fastapi import FastAPI, Header, HTTPException
from typing import Optional

app = FastAPI()

INTENT_MODEL_MAP = {
    "faq": "qwen-turbo",
    "creation": "qwen-plus",
    "vision": "qwen-vl-plus",
    "audio": "qwen-audio-turbo",
}

@app.post("/api/bff/route")
async def bff_route(
    intent: str,
    user_id: str = Header(None),
    x_route_version: str = Header("v2.4"),
):
    model = INTENT_MODEL_MAP.get(intent)
    if not model:
        raise HTTPException(400, f"未知意图: {intent}")

    return {
        "model": model,
        "version": x_route_version,
        "endpoint": f"https://dashscope.aliyuncs.com/compatible-mode/v1",
    }

网关层特征与BFF模式

BFF integration将路由决策放在BFF层。前端只传意图标识（如intent=faq），BFF根据策略决定后端模型。

功能网关层实现BFF层实现
鉴权JWT/OAuth2统一验证透传user_id到业务
限流令牌桶/Redis Rate Limiter业务级限流（按用户）
路由按路径分发到服务按意图分发到模型
熔断服务级Circuit Breaker模型级超时控制
审计请求日志调用审计（含成本）
不推荐
  • 前端直接调模型API（暴露Key）
  • 每个服务自己实现限流和熔断
  • 路由策略散落在各业务代码中
  • 无统一入口，难以做审计
推荐
  • 网关统一入口+鉴权+限流
  • 网关层熔断+降级配置
  • BFF层负责意图→模型路由
  • 审计日志统一收集
CorpAssist · S5网关集成 CorpAssist S5通过Spring Cloud Gateway统一入口。前端请求先过网关鉴权，令牌桶限流（100req/s）。网关根据Path分发到S5服务：/api/s5/chat→S5 Router，/api/s5/vision→VL Service。BFF层根据intent参数决定用turbo/plus/vl模型。网关层配置熔断器，S5服务不可用时自动降级。

常见问题（FAQ）

网关层做路由和BFF层做路由有什么区别？
网关层路由是按服务/路径分发（如/chat→S5、/vision→VL）。BFF层路由是按业务意图分发（如faq→qwen-turbo、creation→qwen-plus）。推荐双层：网关负责服务级分发，BFF负责模型级分发。
网关层的限流和熔断怎么和业务联动？
网关层限流保护后端不被突发流量打爆。熔断器在后端响应慢或连续错误时断开。联动：网关熔断后返回降级页面，BFF层收到熔断信号后自动切换模型策略。
网关集成后如何做灰度发布？
网关支持权重路由：10%流量走新版路由策略，90%走旧版。通过Spring Cloud Gateway的Weight路由谓词实现。灰度期间监控错误率和延迟，确认稳定后逐步提升权重。

  • 双层路由：网关层按服务分发，BFF层按意图分发。
  • 熔断降级：网关层Resilience4j熔断器，后端不可用时自动降级。
  • 灰度发布：网关权重路由支持路由策略的灰度发布。
