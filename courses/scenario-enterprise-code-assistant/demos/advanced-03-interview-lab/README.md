# Demo：面试五连问准备

[章节](../../index.html#ch-advanced-03-interview)

## 实验目标

1. 准备五连问的回答框架
2. 练习画系统架构图
3. 模拟面试场景

## 步骤一：五连问回答提纲

| # | 问题 | 回答框架 |
|---|------|---------|
| 1 | 设计企业代码助手 | 四层架构 + 关键决策 + 权衡 |
| 2 | 超大仓库上下文 | 三级文件选择 + 预算管理 + AST 切片 |
| 3 | 准确率怎么提升 | 定义→度量→优化（检索/上下文/模型） |
| 4 | 数据安全怎么保障 | 最小数据 + 加密 + 隐私模式 + 不落盘 |
| 5 | 延迟优化方案 | 全链路预算 + 缓存 + 模型路由 + vLLM |

## 步骤二：画架构图

在纸上画出代码助手的系统架构图，包含以下元素：

- 客户端层（IDE/Web）
- API 网关层（LB/Auth/Rate）
- 服务层（Completion/QA/Review）
- 索引层（AST/Embedding/Milvus）
- 基础设施（GPU/Redis/MQ）

## 步骤三：模拟面试

每个问题准备 3 个关键要点和 1 个数据支撑。

## 验证标准

- 每个问题有清晰的三段式回答框架
- 架构图包含四层 + 基础设施
- 回答中包含至少 2 个具体数据指标
