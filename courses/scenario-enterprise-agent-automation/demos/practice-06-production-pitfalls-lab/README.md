# Demo：生产坑：规划、工具与记忆

[章节](../../index.html#ch-practice-06-production-pitfalls)

## 目标

排查并修复 Agent 生产环境中的四大常见坑：无观察循环、步数预算超标、工具假执行、记忆溢出。

## 练习内容

1. **Pitfall 修复方案**（pitfall-fixes.md）：针对 4 个生产坑各写一个检测方法和修复策略
2. **Observe Guard 实现**：编写强制观察守卫函数，阻止无观察的 Agent 继续执行
3. **假执行检测器**：实现工具响应签名验证和调用链追踪

## 验收标准

- 4 个 pitfall 的检测方法都是可代码化的（非人工检查）
- Observe Guard 能阻断连续 2 步无观察的 Agent 执行
- 假执行检测器能通过签名/时间戳识别伪造的工具结果

## 修复优先级

1. 无观察循环 → 强制每步 Observe Edge
2. 步数超标 → max_steps 硬限制 + 连续重复检测
3. 工具假执行 → 签名校验 + 调用 chain 追踪
4. 记忆溢出 → 三层分层 + 滑动窗口 + 摘要归档
