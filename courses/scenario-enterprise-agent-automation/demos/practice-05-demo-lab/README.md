# Demo：完整 Agent 演示链路

[章节](../../index.html#ch-practice-05-demo)

## 目标

搭建可演示的 Agent 端到端链路，编写录屏脚本，配置步数/Token 监控。

## 练习内容

1. **演示脚本**（demo-script.md）：编写请假场景的完整演示脚本，包含前提条件、操作步骤、预期结果、验收标准
2. **监控埋点**（metrics.md）：在 Agent 执行中埋点记录每步的 step、tokens、tool_latency
3. **Mock 环境搭建**：用 Mock 工具模拟外部 API，确保演示不依赖真实系统

## 验收标准

- 演示脚本至少包含 6 个步骤，每个步骤有预期结果
- 监控数据包含总步数、总 Token、成功率、p99 延迟
- Mock 环境能模拟 3 个办公工具的响应

## 演示场景

| 场景 | 步骤数 | Token 预算 | 审批次数 |
|------|--------|-----------|---------|
| 请假流程 | 6-8 | 4000 | 1 |
| 报销流程 | 10-12 | 6000 | 1 |
| 工单创建 | 8-10 | 5000 | 0 |
