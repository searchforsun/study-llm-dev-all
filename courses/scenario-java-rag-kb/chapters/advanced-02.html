成本优化方向：

  • Prompt 是最大头（45%）——优化 System Prompt 长度（从 800 Tokens 压缩到 450 Tokens，节省约 44%）
  • 回答输出（32%）——控制 max-tokens（从 1024 降到 512，对简单问答不影响质量）
  • Rerank（16%）——对高频查询做缓存，减少 GPU 调用次数

优化后的日均成本可降至约 ¥120（降幅 34%），年节省约 ¥22,000。

面试可能问

Q1: 召回率从 60% 优化到 92% 的过程中，最大的挑战是什么？

最大的挑战不是技术实现，而是评测体系的建立。一开始没有黄金评测集，只能靠人工抽检评估，每周才能知道一次效果。后来花了一周时间建立 200 条黄金评测集（找业务部门同事标标注），之后每次优化 1 小时内就能得到 Recall@K 的量化反馈。没有评测体系，所有优化都是盲目的。

Q2: Token 成本优化中，「Prompt 压缩」具体怎么做的？

三步压缩：① 去掉 System Prompt 中的冗余描述（从「你是一个企业知识库问答助手，你基于以下文档回答…」精简为「基于文档回答，未覆盖请说明」）；② 使用短的指令格式（bullets 代替 paragraph）；③ 去掉非核心的约束示例（如只在最后加一句「引用格式：[来源]」而不是给出 3 个示例）。压缩后 Prompt 从 800 Tokens 降到 450，回答质量无显著变化。
