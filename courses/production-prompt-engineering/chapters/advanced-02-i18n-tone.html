为什么要学本章

ToB 客服与对外内容必须「像同一家公司说话」。模型默认语气随机、易踩广告法与行业禁用语。本章把语调与合规写进 Prompt 与配置，并与 Prompt 注入攻防 的输出过滤、内容生成（S5） 的风格链衔接。

本章先记住 5 件事
  • 语调是产品资产，不是「让模型客气一点」的随口要求。
  • 合规分「生成前约束」与「生成后审核」两层。
  • 多语言用独立模板，不要指望模型自动翻译 System。
  • locale prompt 变量要在网关层注入，避免用户伪造 locale。
  • 与 PromptOps 的变更流程绑定，语调改动也要走 CR。

品牌语调

CorpAssist · 客服 S2 统一口吻 同一工单场景，A 模型回答过于口语，B 模型过于生硬。产品要求：专业、简洁、先共情再给步骤，禁止网络梗与夸张承诺。

把 brand tone 写成 5～8 条可测规则（而非形容词堆砌），并配 2 条正反 few-shot：

yaml 
brand_tone:
  zh-CN:
    persona: warm-professional
    rules:
      - 使用「您」，不用「亲」
      - 先一句共情，再给 1-3 步操作
      - 禁止承诺「100%」「绝对」
    few_shot:
      good: "理解您着急。请先在工作台提交工单编号，我们将在 2 小时内响应。"
      bad: "亲，包治百病，马上搞定！"

与 Prompt 解剖与角色设计 中的 CorpAssist 人设叠加：System 先定角色，再插入 tone 块。

persona适用场景禁用
warm-professionalS2 客服、员工通知网络梗、过度口语
formal-executive高管摘要、合规函表情符号、夸张承诺
marketing-warmS5 对外内容（仍受 forbidden 约束）绝对化促销语
坏 tone 卡
  • 「专业、友好、高效」无操作定义
  • 无 few-shot，模型自由发挥
  • 与 compliance 拒答模板冲突
好 tone 卡
  • 5～8 条可测规则 + 正反例
  • semver 版本化，CR 可 diff
  • 与 forbidden 列表同源引用

合规话术

compliance script 覆盖拒答、免责声明、敏感行业表述。生成 Prompt 中写「禁止列表」，生成后用规则引擎二次扫描。

场景合规要求Prompt 落点
医疗/金融暗示不得给出诊断或投资建议System 边界 + 拒答模板
绝对化用语广告法禁用「最」「第一」等forbidden phrase 列表
隐私不复述其他用户数据与注入防护共用 output filter
markdown 
若用户询问投资荐股、医疗诊断：使用合规拒答模板 COMPLIANCE-REFUSE-01，
不得自行编造免责声明。
模板 ID用途触发条件
COMPLIANCE-REFUSE-01投资/医疗类拒答意图分类命中敏感域
COMPLIANCE-SLA-01服务时效说明用户追问具体到账/治疗时间
COMPLIANCE-PRIVACY-01隐私边界索取他人订单/员工信息
CorpAssist · 合规拒答统一 用户问「这只股票能买吗」。Agent 不得给投资建议；必须输出 COMPLIANCE-REFUSE-01（zh-CN/en-US 各一套正文），而非模型即兴「仅供参考」——后者不可审计且易漏免责声明。

多语言

multilingual 生产实践：每种 locale 一套 Jinja/PromptTemplate，共享变量 schema，不共享自然语言正文。 locale prompt 由 BFF 从用户档案读取，禁止由用户消息指定。

多语言模板路由
flowchart LR
  Req[请求] --> BFF[BFF 解析 locale]
  BFF --> Tzh[prompts/zh-CN/]
  BFF --> Ten[prompts/en-US/]
  Tzh --> LLM[LLM]
  Ten --> LLM

评测需分 locale golden set；英文拒答率下降不能说明中文也可发布。

locale模板路径golden 集
zh-CNprompts/zh-CN/cs-agent-system.j2eval/cs-golden-zh.jsonl
en-USprompts/en-US/cs-agent-system.j2eval/cs-golden-en.jsonl
ja-JPprompts/ja-JP/…（可选扩展）独立集，禁止复用 en 通过结果
java 
// Spring · locale 路由（BFF 注入 locale，非 user 消息）
PromptTemplate systemTpl = promptRegistry.load(
    "classpath:prompts/" + locale + "/cs-agent-system.st");
Map<String, Object> vars = Map.of(
    "brand_tone", toneService.forLocale(locale),

);
坏做法
  • 单套中文 System，user 末尾写「请用英文」
  • 机器翻译 tone 卡上线
  • locale 由用户消息参数指定
好做法
  • prompts/zh-CN/ 与 prompts/en-US/ 独立
  • 分 locale golden + tone audit
  • BFF 从用户档案读 locale
先想 10 秒：能否用「请用英文回答」替代独立 en-US 模板？

不建议。System 中的合规话术、拒答模板与 forbidden phrase 需按 locale 维护；用户消息无法可靠指定语言，且翻译 System 会漂移语调与法律表述。

审核

tone audit = 离线 Rubric + 在线抽检。用 review tone Prompt 让强模型按 1～5 分打「是否符合语调卡」，人工复核 5% 样本。

环节频率通过标准
发版前每 CR 一次语调分 ≥ 4.0 且零条禁用语命中
线上每日 1%违规率 < 0.1% 否则回滚 Prompt
markdown 
## review-tone prompt（节选）
按 brand_tone 规则对下列回复打 1～5 分，输出 JSON：
{ "tone_score": 4.2, "violations": [], "suggest_fix": "" }
若命中 forbidden phrase，tone_score 上限为 2。
Rubric 维度1 分5 分
称谓与共情无共情、口语化「您」+ 先理解再给步骤
承诺边界绝对化/夸张可核实时限与范围
合规命中 forbidden零禁用语
简洁冗长跑题1～3 步可执行
CorpAssist · tone audit 阻断发版 客服 S2 tone 卡 PATCH 后，发版前 review-tone 抽检 200 条：平均分 3.7（< 4.0），3 条命中「绝对保证」。CR 驳回，恢复 tone 块并补 few-shot 后再跑 golden。

先想 10 秒：S5 营销 tone 更「促销」，能否不共用 forbidden 列表？

不能拆合规底线。促销风格可调 persona 与结构，但广告法禁用语（绝对化、未授权优惠）必须在 shared forbidden 统一维护；否则 S5 与 S2 合规口径漂移。

语调 CR 须同步更新：brand_tone.yaml、分 locale 模板、review-tone golden 与 PromptOps registry 的 last_ab 字段。发版前跑 zh/en 各 50 条 tone 子集，任一 locale 未过线则阻断合并。

动手练习

为 CorpAssist 客服 S2 编写可版本化的 brand tone 与合规拒答模板。

操作步骤

  1. 在 demos/advanced-02-i18n-tone-lab/tone-rules.md 写 5 条 brand tone 规则 + 1 组正反例（对照「品牌语调」YAML 示例）。
  2. 添加 10 条 forbidden phrase 并说明触发动作（对照「合规话术」表：替换/拒答）。
  3. 为 zh-CN / en-US 各写 3 句合规拒答模板（独立正文，参照「多语言」节独立模板原则）。
  4. 写一条 review-tone JSON 输出样例（参照「审核」节 Rubric 四维）。
  5. 完成本章测验。

判断练习

  1. 产品要求「更活泼」，能否只在 user 末尾加「请活泼一点」？

    参考答案

    否。语调属于 System 可版本化资产；user 追加指令不可审计、易与合规冲突。应改 tone 卡 + few-shot，并走 CR。

  2. 英文 golden 全过、中文未测，能否合并发版？

    参考答案

    否。每 locale 独立模板与评测集；英文通过不代表中文拒答率与禁用语命中达标。

  3. tone 卡与 compliance 拒答模板冲突（tone 要求「热情承诺」、合规禁止绝对化），以谁为准？

    参考答案

    合规优先。tone 可调 warmth 与结构，不得 override forbidden 与 COMPLIANCE 模板。冲突时改 tone 卡 few-shot，而非 weaken 合规句。
