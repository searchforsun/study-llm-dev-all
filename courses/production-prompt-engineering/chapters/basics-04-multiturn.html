历史裁剪入门

多轮对话时 history trim 决定哪些 Assistant/User 进入下一轮 Prompt。CorpAssist 默认「最近 6 轮 + 滚动摘要」，避免 history 挤占 few-shot 与 RAG。

历史裁剪策略
flowchart TB\n  Full[全量历史] --> Win[滑动窗口]\n  Win --> Sum[超长则摘要]\n  Sum --> Inject[注入 User 前缀]

先窗口后摘要，摘要 prompt 独立版本化。

python 
MAX_TURNS = 6
def trim_history(messages):
    return messages[-MAX_TURNS*2:]

sliding window 与上下文课一致：System 不被裁剪，history 从旧到新丢弃。

Spring Advisor vs LangChain 预览

spring advisor 在调用链前后织入记忆；LangChain 用 langchain memory 包装 Runnable。双栈需统一 session_id。

java 
@Bean
ChatMemory chatMemory() { return new MessageWindowChatMemory(10); }
// Advisor 将 memory 注入 Prompt
python 
from langchain.memory import ConversationBufferWindowMemory
memory = ConversationBufferWindowMemory(k=6)
