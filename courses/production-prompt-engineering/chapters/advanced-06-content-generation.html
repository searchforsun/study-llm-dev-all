风格与结构 Prompt

style structure prompt 明确：受众、渠道（公众号/邮件）、段落结构（钩子-要点-CTA）。继承 品牌语调 的 tone 卡，并增加结构约束。

markdown 
## 任务
根据 outline 撰写 {{channel}} 文稿。

## 风格（不可偏离）
- 语调：{{tone_persona}}
- 结构：标题 → 3 段要点 → 行动号召
- 禁止：绝对化用语、未证实数据

## 输入
outline: {{outline}}
facts: {{verified_facts}}

长文规划与去重

长文采用两阶段：longform plan prompt 生成章节大纲 → 按章生成 → 合并。dedup prompt 在合并后要求模型标注重复句并改写，或用 embedding 相似度 > 0.92 自动重写。

S5 长文 Prompt 链
flowchart LR
  Plan[plan prompt] --> Sec1[section 1]
  Plan --> Sec2[section 2]
  Sec1 --> Merge[merge]
  Sec2 --> Merge
  Merge --> Dedup[dedup prompt]
  Dedup --> Out[终稿]

质量评估流水线

quality pipeline：生成 → review prompt 打分 → 低于阈值则重写（最多 N 次）。Rubric 维度：事实一致、结构完整、语调、可读性。

阶段Prompt 角色输出
生成style + section草稿
评审review promptJSON 分数 + 修改建议
修复repair 仅改扣分段终稿候选

内容安全审核

content safety prompt 与规则引擎双轨：模型自检敏感主题 + 外部内容安全 API。命中则拒发或转人工，与 输出 filter 共用配置。

个性化参数

personalize var 由 BFF 注入，禁止用户随意覆盖：

  • audience：员工 / 客户 / 高管
  • channel：钉钉 / 邮件 / 官网
  • max_words：控制成本与阅读时长
  • locale：联动 i18n 模板

面试题：S5 与 S1 RAG 的 Prompt 最大差异？

S1 强调仅依据检索、必须引用；S5 强调可控创造性（风格/结构/个性化），事实块用 verified_facts 注入而非自由发挥。二者都要安全审核，但 S5 额外要质量流水线与 dedup。
