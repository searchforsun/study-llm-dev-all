坏例分析

面试题 · CorpAssist RAG 引用消失 发版 rag-qa-v3.1 后，faithfulness 从 0.89 降到 0.72，用户反馈「答案像编的」。日志显示 Prompt token 变短，检索 chunk 数未变。

bad prompt triage 检查顺序（先排除非 Prompt）：

  1. 检索：top_k、索引版本、ACL 是否变化
  2. 模型：是否换模型或温度
  3. Prompt：System 是否删了「仅依据上下文/引用格式」
  4. 数据：golden set 是否仍代表生产分布
信号更可能根因验证方式
引用块消失Prompt 删 citation 指令diff v3.0 vs v3.1 System
工具乱调工具描述含糊或示例误导见 Agent 与工具 Prompt
语调漂移tone 卡未随 CR 更新见 语气、合规与多语言
注入成功率升防护层被绕过见 Prompt 注入攻防

改写策略

rewrite strategy：一次只改一层（先 System 边界，再 few-shot，最后变量），每步跑 golden 子集 50 条，避免「大改不可测」。

改写反模式
  • 同时改模型 + Prompt + 检索参数
  • 无 before 快照，无法回滚
  • 用 2 条手工样例宣布修好
推荐
  • 恢复 引用格式 与拒答句到 System 顶部
  • A/B 5% 流量验证 metric delta
  • 保留 v3.0 tag 作 instant rollback
markdown 
## v3.1 → v3.1.1 修复（仅 System）
+ 回答必须包含 [来源:文件名#段落]，无检索依据时使用拒答模板。
+ 不得补充检索上下文外的政策细节。

先想 10 秒：faithfulness 降但 citation 率未变，还能怪 Prompt 吗？

可能。模型可能「编造看似合理但无 chunk 支撑」的表述；需同时看引用准确率与人工 spot check。也可能 golden 分布漂移，应交叉验证检索与数据版本。

答题模板

用 answer template 五段式（约 5 分钟）：

  1. 现象：谁反馈、哪版 Prompt、哪条指标跌
  2. 假设：列 2～3 个根因（检索 / Prompt / 模型）
  3. 验证：diff、trace、golden 对比
  4. 改动：before/after Prompt 片段 + 为何只改这一层
  5. 结果：metric delta + 回滚预案

debug prompt 技巧：在 staging 加「请先列出你将引用的 chunk_id，再生成答案」的 CoT 中间步，便于定位是检索空还是生成违例。

答辩 / 面试口述题
  1. RAG 发版后 faithfulness 骤降，你如何 10 分钟内判断是不是 Prompt 问题？
  2. 举一例 before after：你改过哪条 System 使引用恢复？
  3. Agent 乱调工单工具，你会改工具描述还是 System？顺序是什么？
  4. 如何用 50 条 golden 子集做 Prompt 小步迭代而不污染全量基线？
  5. 描述一次 Prompt 回滚与 A/B 5% 验证的完整话术。

案例二 · Agent 工具误调

面试题 · 客服 Agent 误创建退款 用户仅查询订单，Agent 却调用 create_refund。trace 显示工具 description 与 query_order 过于相似，且 System 未写「查询类意图禁止写操作工具」。

排障顺序：确认意图分类 → diff 工具 description → 检查 System 工具白名单分组（read vs write）。改写策略：先收紧 tool desc spec 边界，再补 System「只读查询不得调用 mutating 工具」，golden 子集验证 tool_acc。

markdown 
## before（tool description 含糊）
create_refund: 处理订单退款
query_order: 查询订单

## after
create_refund: 仅当用户明确确认退款且 order_id 已校验时调用；禁止用于查询。
query_order: 只读；po_number 格式 PO-YYYY-NNN；勿与 create_refund 混用。
指标修复前修复后
tool 选择准确率0.780.94
误创建退款次数/日120

五段式口述示例（S1 faithfulness 下跌）：「v3.1 发版后 faithfulness 0.89→0.72；假设 System 裁剪引用指令；diff 证实 context-only 段缺失；恢复 v3.1.1 仅补 System 两行；faithfulness 回 0.88，保留 v3.0 tag 作 rollback。」

案例三 · S5 语调漂移

面试题 · 营销文案过于促销 S5 内容生成出现「全网最低价」等禁用语。trace 显示 s5-content-v1.1 合并 tone 块时误删 forbidden-phrase include。

排障：检查 shared tone-compliance.yaml 是否仍被 Jinja include；对比 brand tone 与 S5 模板 diff。修复：恢复 include + 跑 content-golden 禁用语子集；线上 1% 抽检 tone audit。
