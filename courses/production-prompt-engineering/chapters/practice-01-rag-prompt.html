引用格式

CorpAssist · 产假制度问答 企业知识库 RAGS1行业说法Enterprise RAG · Knowledge Base QA
检索增强生成回答制度/文档，带引用溯源；面试最高频系统设计题 员工问「二胎产假多少天」。检索返回 3 条 HR 制度片段；回答必须在句末标注 chunk cite，以便合规部门核对是否引用了 2024 版而非过期 PDF。

生产 citation format 需满足：人可读（文件名/章节）、机可解析（正则或 JSON 字段）、与索引一致（chunk_id 来自 rag-service）。推荐在检索侧预格式化为带 ID 的块，再注入模板。

格式要素示例用途
块边界<chunk id="hr-v3#c42">…</chunk>模型对齐引用来源
行内引用158 天 [hr-v3#c42]用户可见溯源
拒答时无引用块 → 不输出虚假 cite避免伪造出处
python 
# Python · LangChain + Jinja2（rag-service / dialog-orchestrator）
from jinja2 import Template
from langchain_core.prompts import ChatPromptTemplate

RAG_SYSTEM_JINJA = Template("""\
你是 CorpAssist 企业知识助手（模板版本 {{ prompt_version }}）。
## 引用规则（citation-format）
- 每个事实性陈述句末必须标注 [doc_id#chunk_id]，与下方 <chunk> 的 id 一致。
- 禁止引用未出现在上下文中的 chunk_id。
{% if refusal_phrase %}
## 拒答
上下文不足以回答时，仅输出：{{ refusal_phrase }}
{% endif %}
""")

def format_chunks(chunks: list[dict]) -> str:
    lines = []
    for c in chunks:
        lines.append(
            f'<chunk id="{c["doc_id"]}#{c["chunk_id"]}">\n{c["text"]}\n</chunk>'
        )
    return "\n".join(lines)

user_tpl = ChatPromptTemplate.from_messages([
    ("system", RAG_SYSTEM_JINJA.render(
        prompt_version="rag-v2.3.0",
        refusal_phrase="根据现有资料无法确定，建议联系 HR 或提交工单。",
    )),
    ("human", "<context>\n{context}\n</context>\n\n问题：{question}"),
])
# chain = user_tpl | chat_model
java 
// Spring AI · dialog-orchestrator
@Bean
PromptTemplate corpAssistRagSystemPrompt() {
  return new PromptTemplate("""
    你是 CorpAssist 企业知识助手（模板版本 {promptVersion}）。
    ## 引用规则
    - 句末标注 [doc_id#chunk_id]，与 context 中 chunk id 一致。
    - 禁止伪造引用。
    ## 拒答
    无依据时仅输出：{refusalPhrase}
    """);
}

public String buildRagUserMessage(List<RagChunk> chunks, String question) {
  String context = chunks.stream()
      .map(c -> "<chunk id=\"" + c.docId() + "#" + c.chunkId() + "\">\n"
          + c.text() + "\n</chunk>")
      .collect(Collectors.joining("\n"));
  return "<context>\n" + context + "\n</context>\n\n问题：" + question;
}
坏 Prompt
  • 「请尽量引用来源」——格式不固定，无法解析
  • 引用写「根据公司规定」无 chunk_id
  • 在 user 里才提引用，system 未约束
好 Prompt
  • 固定 [doc_id#chunk_id] + XML 块边界
  • 检索侧注入带 id 的 context，模型只选已有 id
  • 双栈模板 Git 版本号 rag-v2.3.0 一致

拒答

refusal prompt 不是「别回答」，而是可预期的 no answer 路径：当 top-k 分数低于阈值、或用户问的是检索库外领域（如股票行情），模型应输出固定拒答句并建议下一步（转人工、提工单、换关键词）。

触发条件CorpAssist 行为Prompt 要点
max_score < 0.55 不调用生成，直接返回拒答模板 编排层短路，节省 Token
有 context 但不足以结论 LLM 输出固定 no answer 句 system 中列举 2～3 条合法拒答示例
用户要求超出角色（写代码、医疗诊断） 拒答 + 边界说明 与 边界声明 一致
RAG 拒答决策流
flowchart TD
  Q[用户问题] --> R[rag-service 检索]
  R --> S{max_score >= 阈值?}
  S -->|否| NA[固定 no-answer 文案]
  S -->|是| P[拼装 rag-system-prompt + context]
  P --> L[LLM 生成]
  L --> C{输出含 chunk cite?}
  C -->|否且为事实句| R2[二次拒答或降级]
  C -->|是| OK[流式返回 + 审计 cite]

拒答可在编排层（检索分低）或生成层（context 不足）触发；两层文案需一致。

坏 Prompt
  • 「不知道就随便说说」
  • 拒答句每次措辞不同，无法监控
  • 低分仍强行生成，faithfulness 崩盘
好 Prompt
  • 唯一拒答句：根据现有资料无法确定…
  • Few-shot 含 1 条「上下文不足 → 拒答」示例
  • 检索阈值与 Prompt 拒答语义对齐

仅依据上下文

CorpAssist · 跨库误答 检索只返回「年假」片段，用户问「病假扣薪吗」。若模型用预训练常识补全，HR 将收到错误政策解读——必须在 System 写死 context only。

context only 是 rag system prompt 的核心约束：禁止「根据一般经验」「通常情况下」等表述；禁止引用 context 外法条编号。配合 temperature 0～0.2 与 top_p 限制，降低补全倾向。

yaml 
# CorpAssist RAG 生成参数（llm-gateway 路由 rag-qa 场景）
corpassist:
  prompts:
    rag_qa:
      version: rag-v2.3.0
      system_ref: classpath:prompts/rag-system-v2.3.st
      params:
        temperature: 0.1
        top_p: 0.9
      context_only_rules:
        - "禁止引用 <context> 外信息"
        - "禁止「我认为」「一般来说」"
        - "数字、日期必须来自 chunk 原文"
先想 10 秒：用户消息末尾写「仅依据上文」够吗？

不够。user 可被多轮历史稀释；且注入攻击可能在 user 中覆盖指令。context-only 必须写在 system，并在编排层把 context 与 user 问题分块注入。

幻觉抑制

hallucination guard = Prompt 约束 + 生成参数 + 后验校验。Prompt 侧：拒答、context-only、引用格式；工程侧：对输出做 cite 存在性检查、关键数字与 chunk 文本模糊匹配；评测侧：faithfulness 集监控。

坏 Prompt
  • temperature 0.8 + 「详细展开」
  • 无 cite 校验，前端仍展示「AI 回答」
  • 幻觉样例未进回归集
好 Prompt
  • 低 temperature + 句末强制 cite
  • 编排层 strip 无 cite 的事实句或整段拒答
  • 每周跑 faithfulness + 引用覆盖率

本章结论

四条对应四个 section，作为 S1 RAG Prompt 的共同语言。

  • 引用格式：固定 [doc_id#chunk_id] + XML chunk 块；双栈模板同版本、同规则。
  • 拒答：refusal prompt 固定句式；检索低分可编排层短路。
  • 仅依据上下文：context only 写 system；低 temperature；禁止常识补全。
  • 幻觉抑制：Prompt + cite 校验 + faithfulness 评测闭环。

判断练习

  1. 产品希望「没检索到也尽量回答」，以提升满意度。是否修改 refusal-prompt？

    参考答案

    不应取消拒答。可优化检索与引导话术，但 S1 合规要求无依据必须 no answer；否则 faithfulness 与审计无法通过。

  2. 模型常在回答末尾编造 [hr-v3#c99]，context 中无 c99。Prompt 侧还能做什么？

    参考答案

    强化「禁止引用未出现 chunk」+ Few-shot 反例；编排层对 cite 做存在性校验，无效则触发拒答或重试；评测集加入伪造 cite 样例。
