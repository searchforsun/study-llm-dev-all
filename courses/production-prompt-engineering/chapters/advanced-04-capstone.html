已完成实战：CorpAssist Prompt 套件

约 58 分钟 · 阶段：生产治理 · 能力：交付 prompt suite（S1/S2/S5）、version repo、eval bind 与 delivery pack

为什么要学本章

Capstone 把本课散落能力收成可上线资产包：three scenarios 各一套模板 + 评测 + 文档。面试与项目答辩常问「你如何交付企业级 Prompt 套件」——本章给出目录结构与验收清单。

学完你能
  • 组装 S1/S2/S5 三套 template pack
  • 建立 Git version repo 与 golden bind
  • 输出 capstone prompt 交付说明（回滚、指标、负责人）
本章先记住 5 件事
  • S1 来自 RAG 专用 Prompt：引用、拒答、仅依据上下文。
  • S2 来自 Agent 与工具 Prompt：工具描述、计划、失败恢复。
  • S5 来自 内容生成与创作自动化：风格链、长文、安全审核。
  • 三套共享：注入防护（注入攻防）、语调合规（语气合规）、PromptOps（协作流程）。
  • 交付 = 模板 + eval + registry + 发布记录，缺一不可。

版本库

version repo 约定：语义化版本 MAJOR.MINOR.PATCH——破坏兼容升 MAJOR；新增变量升 MINOR；文案微调升 PATCH。生产只引用 tag，不引用 main 浮动分支。

评测绑定

每个模板必须 eval bind 独立 golden set；共享 injection 子集。指标示例：

  • S1：faithfulness、引用准确率、拒答准确率
  • S2：工具选择准确率、任务完成率
  • S5：语调分、禁用语命中率、段落重复率（dedup）
markdown 
eval/
  rag-golden.jsonl      # 绑定 s1-rag-qa-v3
  cs-golden.jsonl       # 绑定 s2-cs-agent-v2
  content-golden.jsonl  # 绑定 s5-content-style-v1
  injection.jsonl       # 三场景共用

交付

delivery pack 清单（面试可直接背）：

  1. 三套模板 + shared 配置（Git tag）
  2. registry.yaml 与 Prompt 说明文档
  3. 评测报告（发版前全量 pass）
  4. 回滚脚本（上一 tag 一键切换）
  5. 负责人与 on-call 联系（Prompt 变更）

验收勾选 demos/advanced-04-capstone-lab/suite-checklist.md：S1 RAG / S2 客服 / S5 内容三项。

  • 版本库 + eval bind + delivery pack 是交付最小集。
  • 下一步：深化 S5 — 内容生成与创作自动化；排障 — 面试排障案例。
