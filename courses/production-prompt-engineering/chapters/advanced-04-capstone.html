  • 毕业 = 可演示 + 可回滚 + 可评测。

Spring AI：多场景 ChatClient 工厂

java 
public ChatClient ragClient(PromptRegistry registry) {
  PromptBundle bundle = registry.load("s1-rag-template", "1.0.0");
  return ChatClient.builder(model)
      .defaultSystem(bundle.renderShared() + bundle.renderScene())
      .defaultAdvisors(injectionAdvisor, toneAdvisor)
      .build();
}

先想 10 秒：为何 Capstone 强调共享子模板？

减少漂移与重复 MR；合规/语调/注入策略变更一次生效三场景，评测也可共用 compliance-disclaimer 子集。

版本库

prompt registry 是 CorpAssist 的模板目录：记录 Git 路径、semver、依赖的 shared 版本、生效 tenant 范围。不是运行时数据库里的 Prompt 正文缓存。

registry 字段说明示例
template_id场景模板名s1-rag-template
semver场景版本1.2.0
shared_ref共享层版本shared@0.9.1
git_path仓库相对路径prompts/s1/1.2.0/
depstools / schematools@v3, ticket-schema@v2

registry.yaml 片段

yaml 
templates:
  - id: s1-rag-template
    semver: 1.2.0
    shared_ref: shared@0.9.1
    git_path: prompts/s1/1.2.0/
    eval_manifest: eval/s1-manifest.yaml
  - id: s3-json-template
    semver: 2.0.0
    shared_ref: shared@0.9.1
    deps: [ticket-schema@2.1.0]

与 PromptOps 与协作 的 semver 策略一致；major 升级须更新全部三场景的 eval manifest。

评测绑定

Capstone 的 eval manifest 是聚合文件：列出三场景 golden、注入、合规子集及毕业阈值。CI 在 release train 合并窗口统一执行。

套件覆盖场景毕业阈值（示例）
rag-goldenS1cite_coverage ≥ 0.95
agent-tool-goldenS2tool_acc ≥ 0.90
ticket-json-goldenS3schema_pass ≥ 0.98
injection-regression全部refuse_rate = 1.0
compliance-disclaimer全部disclaimer_hit = 1.0

capstone-eval-manifest.yaml

yaml 
bundle: capstone-bundle@1.0.0
scenes:
  - template: s1-rag-template@1.2.0
    golden: rag-golden-v14
  - template: s2-agent-template@1.0.0
    golden: agent-golden-v6
  - template: s3-json-template@2.0.0
    golden: ticket-json-v8
global_suites: [injection-regression, compliance-disclaimer]
graduation_thresholds:
  all_scenes_pass: true
  zero_high_severity_regression: true

绑定 LLM 评测与质量llm-evaluation-quality 与 语气、合规与多语言 的 disclaimer 套件，确保 Capstone 不是「只测 RAG」。

三场景阈值能否单独发布？
release train 默认原子发布三模板 + shared_ref；若必须热修单场景，走 patch semver 但仍需跑 global_suites 全量。

动手练习

组装 capstone-bundle@1.0.0：编写 registry 三条目、capstone-eval-manifest.yaml，并在本地跑集成评测脚本生成毕业报告摘要。

操作步骤

  1. 按「三套场景模板」表，为 s1/s2/s3 各写一行 shared_ref=shared@0.9.1 的 registry 条目。
  2. 从 practice 章节复用 cite、工具、schema 要点，各写 3 行 System 结构清单（不必全文粘贴）。
  3. 填写 capstone-eval-manifest.yaml 的 scenes 与 global_suites，设 graduation_thresholds。
  4. 对照 graduation-checklist 表，标注你环境尚缺的一项及补齐计划。
  5. 可选：完成 demos/advanced-04-capstone-lab/ 运行 python check_capstone_bundle.py，期望 all_scenes_pass=true。

判断练习

  1. （权衡）为赶答辩只发布 s1-rag-template@1.2.0，S2/S3 仍用旧版且未跑 agent/json golden，是否符合 graduation checklist？

    参考答案

    不符合：毕业要求三场景 live 且 capstone manifest 全绿；缺 S2/S3 评测与演示违背 all_scenes_pass 与双栈一致原则。
