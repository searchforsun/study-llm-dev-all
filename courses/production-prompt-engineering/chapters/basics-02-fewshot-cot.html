为什么要学本章

格式不稳定时，Few-shot 与 CoT 是低成本杠杆；但示例过时与推理外露是常见生产事故。本章建立可版本化的双栈模板与成本意识。

学完你能
  • 为工单分类挑选 3 条覆盖边界的 few-shot
  • 设计隐藏 CoT 的三步推理模板
  • 估算 few-shot+CoT 的 token 增量并设停止条件
  • 双栈共享 YAML 示例库
本章先记住 3 件事
  • 示例要少而精，覆盖边界
  • CoT 对外隐藏，步数有上限
  • 成本用仪表盘驱动增减

示例选择

few-shot 通过示范输入输出形状，降低模型对 CorpAssist 工单、拒答话术的「格式猜测」。生产里示例不是越多越好，而要代表边界：正常、拒答、缺 Tool、多实体。

CorpAssist · 工单分类 需将员工描述映射为 ticket_type：咨询/投诉/审批。选 3 条示例覆盖「模糊描述」「跨部门」「明确单号」三类。
原则说明反例
多样性覆盖主要意图簇5 条全是「查订单」
新鲜度与当前 API/字段同步示例仍含已废弃字段 legacy_id
匿名化脱敏真实 PII示例带真实手机号
长度单示例 < 400 tokens贴整页政策 PDF

example selection 可静态维护在 YAML，也可按意图动态检索示例库（注意与 RAG 检索隔离，避免示例污染事实层）。

yaml 
few_shot_examples:
  - input: "客户说发货慢，要投诉"
    output: {"ticket_type":"complaint","need_order_id":true}
  - input: "帮我看看年假还剩几天"
    output: {"ticket_type":"hr_query","escalate":false}
  - input: "没有单号，只说很急"
    output: {"ticket_type":"unknown","reply":"请提供订单号 SO- 开头"}
先想 10 秒：为何示例要与评测 badcase 同源维护？

badcase 反映真实失败分布；把修复后的对话抽成 few-shot，可形成「评测—示例」飞轮，避免示例脱离生产。

思维链

chain of thought（CoT）要求模型先输出推理步骤再给出结论，适合 CorpAssist 多条件判断（是否越权、是否需 Tool）。对外部用户可见回复时，应使用隐藏 CoT 或结构化 reasoning 字段，仅日志留存。

CoT 与对外回复分离
flowchart LR
    User[User 问题] --> CoT[内部 reasoning]
    CoT --> Decision[工具/拒答决策]
    Decision --> Answer[用户可见 answer]

推理链不直接进入客服气泡，除非产品明确要求展示。

Python：在 Prompt 中启用分步推理

python 
COT_SUFFIX = """
在回答前先在  标签内用 3 步分析：
1) 意图 2) 是否需工具 3) 风险
再输出  给用户。"""

prompt = ChatPromptTemplate.from_messages([
    ("system", SYSTEM + COT_SUFFIX),
    ("human", "{question}"),
])

Java：Spring AI 结构化推理字段

java 
@StructuredOutput
public record TicketDecision(
    String reasoning,
    String ticketType,
    boolean needOrderId
) {}

hidden cot 在网关层剥离 <reasoning>，防止泄露内部规则；与安全课「输出过滤」一致。

不推荐
  • 对用户气泡展示完整 CoT
  • 无步数上限，推理冗长爆窗
推荐
  • 固定 3 步模板
  • reasoning 仅审计日志

成本权衡

每增加 1 条 few-shot 或 1 轮 CoT，都会线性增加 prompt token cost。CorpAssist 日调用 50 万次时，示例从 3 条涨到 8 条可能每月多数千美元，且延迟上升。

策略收益成本适用
Zero-shot + 强 System最低 tokens格式漂移风险稳定单意图
Few-shot 3 条格式稳定+600~1200 tokens分类/JSON
CoT 3 步复杂决策更准+200~800 output tokens多条件审批
动态示例检索按需示例检索延迟意图种类多

用 cost dashboard 按场景看 P50/P95 tokens；当准确率边际收益 < 0.5% 时停止加示例。A/B 指标见 A/B 与迭代流程。

常见误区
  • 用 CoT 解决本该用 Tool 的事实问题
  • 示例与线上字段不一致导致「学会错误格式」

双栈模板

dual template 指同一份 prompts/ticket_classify.yaml 被 LangChain 与 Spring AI 加载，差异仅在占位符语法适配层。

python 
from langchain_core.prompts import FewShotChatMessagePromptTemplate
examples = load_yaml("ticket_classify.yaml")["few_shot_examples"]
few_shot = FewShotChatMessagePromptTemplate(examples=examples)
java 
@Value("classpath:prompts/ticket_classify.yaml")
Resource promptResource;
// 解析为 Message 列表注入 Prompt
