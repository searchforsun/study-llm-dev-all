pytest 用例自动生成的 Prompt 模式与 Given-When-Then 模板

AI 生成测试的核心是将测试需求结构化为可重复的 Prompt 模式。标准的测试生成 Prompt 包含三个部分：被测函数签名——函数名称、参数类型和返回值类型，AI 从签名中推断出需要 Mock 的依赖和需要验证的返回值；测试场景列表——按 Given-When-Then 格式列出正常路径、异常路径和边界条件，每条场景对应一个测试函数；Mock 策略说明——哪些参数是外部依赖需要 Mock（如 AsyncSession、httpx.AsyncClient），哪些参数是纯数据可以直接传入。使用结构化 Prompt 后测试有效率达到 85%（定义：测试通过并能验证至少一个具体行为）。

python 
# AI 测试 Prompt 模板 — Given-When-Then 结构
"""
函数: get_permissions(db: AsyncSession, user_id: str) -> list[Permission]
依赖: db 是 AsyncSession 需要 AsyncMock
场景列表:
1. [正常] 用户有 3 条权限 -> 返回长度为 3 的列表
2. [正常] 用户无权限 -> 返回空列表
3. [异常] user_id 为空字符串 -> 返回空列表（不抛异常）
4. [异常] 数据库连接失败 -> 抛出 DatabaseError
5. [边界] 用户有 100 条权限 -> 正确返回 100 条
"""
import pytest
from unittest.mock import AsyncMock, patch

@pytest.mark.asyncio
async def test_get_permissions_normal():
    """Given: 用户有 3 条权限记录
       When: 查询用户权限
       Then: 返回 3 条权限"""
    mock_db = AsyncMock(spec=AsyncSession)
    mock_result = AsyncMock()
    mock_result.scalars.return_value.all.return_value = [
        Permission(kb_id="kb-1", access_level="read"),
        Permission(kb_id="kb-2", access_level="write"),
        Permission(kb_id="kb-3", access_level="admin"),
    ]
    mock_db.execute.return_value = mock_result
    permissions = await get_permissions(mock_db, "user-123")
    assert len(permissions) == 3
    assert permissions[0].kb_id == "kb-1"

测试生成流程：从函数签名出发，AI 自动推断测试场景和 Mock 策略，生成审查后得到可用的测试用例。

边界条件覆盖与异常路径测试策略

AI 倾向于只覆盖 Happy Path——因为训练数据中 70% 的测试用例都是正常路径。需要显式在 Prompt 中要求边界条件和异常路径。常用的边界条件枚举包括：空值（空字符串、None、空列表）、极值（最大值、最小值、0）、非法格式（错误的类型、超长的字符串、无效的 ID 格式）、并发场景（多次提交相同请求、超时后的重试）。异常路径包括：外部依赖失败（数据库连接超时、API 返回 500）、权限不足（未认证、无操作权限）、资源不存在（查询 ID 不存在）。
