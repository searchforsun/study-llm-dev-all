为什么要学本章

本章是 AI-native 开发工具链实战 的收官：前序 Model-as-Judge 代码质量门禁 解决「合入的代码是否够好」，本章解决「工具链本身是否会把代码与数据送出信任域」。金融、政务类 CorpAssist 客户会审计 code leakage risk 与 data residency。

本章先记住 3 件事
  • 默认不信任公有云：除非合同样确「零训练 + 区域限制」。
  • 审计要可关联：会话、PR、request_id、模型版本一条链。
  • 合规是持续过程：工具链变更（新 Skill、新 MCP）要重新评估。

私有模型部署方案

评估 code leakage risk 后再选型 private LLM deploy。Python 轨 CorpAssist 研发工具链推荐三层：

L1 网关代理

企业统一 OpenAI-compatible 网关；IDE 只配内网 URL；记录 Token 与调用方。

L2 自托管推理

vLLM / TGI 部署代码专用模型；Judge 与 Coder 可分池。

L3 离线/air-gap

仅内网文档与本地模型；用于涉密交付环境。

IDE 指向私有端点

json 
{
  "openaiApiBase": "https://llm-gw.corp.internal/v1",
  "openaiApiKey": "${CORP_LLM_TOKEN}",
  "models": {
    "default": "corpassist-coder-32k",
    "judge": "corpassist-judge-8k"
  },
  "telemetry": {
    "enabled": true,
    "endpoint": "https://otel.corp.internal/v1/traces"
  }
}

与 AI 辅助编码工具对比 衔接：采购时要求供应商支持自带端点而非仅 SaaS。

私有部署常见误区

误区后果纠正
Coder 公有云 + Judge 私有混用无评估代码出境、评审在内统一数据分级策略
只换 endpoint 不做网络隔离横向移动风险VPC + 专用 Runner
认为 Privacy Mode 等于合规索引仍含密钥片段.cursorignore + 扫描
金融客户要求 air-gap，IDE 还能用 AI 吗？
可用离线或专线模型；禁止默认云索引。CorpAssist 实践：开发机仅连内网 gw；CI 使用自托管 Runner；审计日志不出域。

审计日志与溯源

一次 AI 辅助合并应可追溯：

  1. 谁（开发者 enterprise id）
  2. 何时（会话 / PR 时间戳）
  3. 用了什么（模型 id、Skill 版本、Prompt hash）
  4. 改了什么（commit SHA、diff 统计）
  5. 结果（Judge 分、pytest、是否 override）
json 
{
  "event": "ai_pr_review",
  "request_id": "ca-9f2a-20260601",
  "actor": "sunachao@corp",
  "model": "corpassist-judge-8k@2026-05-01",
  "skill": "corpassist-pr-review@1.2.0",
  "prompt_sha256": "a1b2…",
  "pr": 4182,
  "outcome": "fail",
  "weighted_total": 3.1,
  "override": null
}

完整 audit trail 写入 SIEM，保留期不少于客户合同要求（常见 180～365 天）。与网关 request_id 对齐，便于与运行时 RAG 请求串联。跨境客户须核对 data residency 与推理区域一致。

年度复审使用 compliance checklist，证据包提供给内审与客户尽调。

审计溯源常见误区

误区后果纠正
只存 PR 评论不存 Prompt 哈希无法复现 Judge 结论prompt_sha256 字段
request_id 未贯通网关与 IDE事故无法串联统一 ca- 前缀规范
日志含 diff 明文 PII二次泄露仅存统计与哈希
override Judge 失败是否需要额外审计？
是。记录 override 人、原因、关联 ticket；保留 180 天以上。CorpAssist 规定：安全/契约类 blocker 禁止 override，仅性能类可申请。内审抽样时须能由 request_id 在 5 分钟内定位到 PR、模型版本与 Prompt 哈希。

将 SIEM 告警与 团队级 AI 编码度量与治理 的合规层指标对齐，可避免「审计有日志、运营无看板」的两张皮。

季度演练应覆盖：误将含密钥 diff 送模、Judge 被注入指令、以及跨境客户要求切换驻留区域三类场景，并验证 Runbook 可在 30 分钟内执行完毕。

演练纪要须归档至 docs/security/ai-drills/，作为 compliance checklist 第 7 项证据。

与 data residency 冲突时优先哪条？
合同驻留 > 工具便利。若 Coder 必须在境内，Judge 也不得将 diff 发往境外；必要时全链路 air-gap。

事件响应 24h 清单（泄露疑似）

  1. 冻结相关开发者 Cloud 会话与 API Key 轮换
  2. 从 SIEM 按 request_id / 时间窗导出审计事件
  3. 比对泄露范围：路径、行数、是否含 PII
  4. 通知法务与客户成功（按合同 SLA）
  5. 复盘：更新 ignore、Skill 审批、培训记录
MCP 服务器如何纳入合规？
MCP 等同「可执行 Skill 扩展」：须内部 registry 登记、代码评审、明确网络出站域。禁止个人随意添加访问生产库的 MCP。
审计日志能否存 Prompt 全文？
涉密环境存 hash + 加密对象存储引用；一般研发存 hash 即可复现结构。全文留存需单独审批与缩短保留期。

常见坑

  • 只买 SaaS 不配私有端点：合同写境内，实际 IDE 仍连公网。
  • ignore 列表过时：新敏感目录未加入，索引泄露。
  • 审计只存 Prompt hash 却无法复现：事故调查困难，需存 rubric 版本与模型 id。
  • Skill 绕过 registry：个人 MCP 成后门。

采购与工程协同清单

  • 法务：DPA、data residency、零训练条款
  • 安全：code leakage risk 评估、DLP、事件响应
  • 平台：private LLM deploy 容量与 SLO
  • 研发：audit trail 字段与 request_id 规范
  • 全员：compliance checklist 培训与年度签署

动手练习

为 CorpAssist 仓库起草 .cursorignore 与 3 条审计事件字段，并完成合规清单自检。

操作步骤

  1. 根据 Mermaid 信任边界图，标出你们项目 2 条「误配置到 SaaS」路径。
  2. 编写 .cursorignore 至少包含 tests/fixtures/ 与 config/prod/。
  3. 补全审计 JSON 示例：为 override 写一种合法非 null 场景。
  4. 从合规表选 3 项，写明你们团队当前「证据」存放位置。
  5. 可选：在 demos/advanced-04-lab/ 提交 checklist 勾选 PR 模板片段。

判断练习

  1. 为省成本，Coder 用公有云、Judge 用私有云，是否满足「代码不出境」？

    参考答案

    不满足。Coder 阶段 diff 与上下文可能已出境；若合同要求全链路境内，Coder 与 Judge 均须走私有端点，或做脱敏/摘要后再传公有云（需法务评估）。
