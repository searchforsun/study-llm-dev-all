AI 辅助 RAG 管线开发

目标：为 CorpAssist S1 知识库实现「 ingest → embed → milvus → retrieve → synthesize」最小链路。把架构图、接口契约与依赖版本写入 Prompt，让 AI 生成模块骨架，你用 1 个下午接好配置与 secrets。

CorpAssist RAG MVP
flowchart LR
    IN[PDF ingest] --> CH[chunk]
    CH --> EM[embed batch]
    EM --> MV[(Milvus)]
    Q[query API] --> RET[retrieve+filter tenant]
    RET --> SYN[LLM synthesize]
    SYN --> OUT[answer+citations]
python
# app/rag/pipeline.py — AI 生成骨架经人工调整
from dataclasses import dataclass

@dataclass
class RagPipeline:
    retriever: "TenantRetriever"
    llm: "ChatClient"

    async def arun(self, query: str, top_k: int = 3) -> dict:
        chunks = await self.retriever.aretrieve(query=query, top_k=top_k)
        chunks = [c for c in chunks if c.text and c.text.strip()]
        context = "\n---\n".join(c.text for c in chunks)
        answer = await self.llm.acomplete(system=RAG_SYSTEM, user=build_user(query, context))
        return {
            "answer": answer,
            "citations": [{"doc_id": c.doc_id, "snippet": c.text[:200]} for c in chunks],
        }

# FastAPI 路由 — 与 pipeline 同 PR 由 AI 生成
from fastapi import APIRouter, Depends
from app.rag.schemas import RagQueryRequest, RagQueryResponse
from app.rag.pipeline import RagPipeline, get_pipeline

router = APIRouter(prefix="/v1/rag", tags=["rag"])

@router.post("/query", response_model=RagQueryResponse)
async def rag_query(body: RagQueryRequest, pipeline: RagPipeline = Depends(get_pipeline)):
    return await pipeline.arun(query=body.query, top_k=body.top_k)

Agent 工作流生成

S3 办公自动化需要「查知识库 → 填表单 → 人工审批」多步流。可用 AI 生成 LangGraph 状态图草图，再按团队 Rules 加 interrupt 与步数上限。对外仍通过 FastAPI 暴露同步/流式接口，与 Java BFF 契约一致。

python
# app/agent/graph.py 示意 — AI 生成后须人工加预算与白名单
from langgraph.graph import StateGraph, END

def build_corpassist_agent():
    g = StateGraph(dict)
    g.add_node("retrieve", retrieve_node)
    g.add_node("draft_form", draft_form_node)
    g.add_node("human_approve", human_approve_node)
    g.set_entry_point("retrieve")
    g.add_edge("retrieve", "draft_form")
    g.add_conditional_edges("draft_form", need_approval, {"yes": "human_approve", "no": END})
    g.add_edge("human_approve", END)
    return g.compile(interrupt_before=["human_approve"])

Prompt 要点：列出工具白名单（只读 HR API、禁止 shell）、最大步数 8、必须写 request_id 到 state。生成后对照 自定义 AI Agent 工作流设计 做治理加固。

评测脚本自动编写

RAG 质量不能靠「感觉」。让 AI 从 PRD 抽取 30 条 golden Q/A，生成 evals/rag_golden.jsonl 与跑分脚本，输出 faithfulness、 citation 命中率。

python
# scripts/run_rag_eval.py
import json, asyncio
from app.rag.pipeline import RagPipeline

async def main():
    pipeline = RagPipeline.from_settings()
    rows = [json.loads(l) for l in open("evals/rag_golden.jsonl")]
    ok = 0
    for row in rows:
        out = await pipeline.arun(query=row["question"], top_k=3)
        hit = any(k in out["answer"] for k in row.get("must_contain", []))
        ok += int(hit)
    print(f"hit_rate={ok/len(rows):.2%}")

if __name__ == "__main__":
    asyncio.run(main())

CI 夜间 job 跑 eval（可调用真实模型），PR 内只跑 mock 快测。AI 还可生成「失败样例分析报告」模板，供周五质量例会使用。

hit_rate 90% 能否直接上线？

需看失败是否集中在合规高风险问题；检查 citations 是否伪造；对比人工评测子集。评测脚本本身也要 code review。

Agent 草图能否跳过人工 interrupt？
演示环境也不行。CorpAssist S3 办公自动化涉及写表单，必须 interrupt_before 人工审批；AI 生成图后由架构师加白名单与步数上限。
eval 只有 3 条 golden 够吗？
不够。lab 要求扩至 20 条并覆盖「未找到」类降级；上线前还需 staging 真实模型抽检与 badcase 飞轮。
dev-metrics 只报周期缩短够吗？
不够。必须同时报 7d 缺陷、覆盖率、hit_rate 与 openapi-diff，证明门禁未降低。见正文「效率数据对比」表。
Agent 与 RAG 能否同一个 PR 交付？
可以，但 eval 与 pytest 须分模块；Agent 图须单独 Review 白名单与 interrupt，避免与 RAG 路由混在一个 giant diff 里。

本章结论

  • AI 加速的是骨架、测试、文档与评测脚本，架构决策仍由人负责
  • RAG + Agent + eval 三件套应在同一迭代内闭环
  • 效率数据要同时看周期与缺陷率，避免片面追求速度
  • 进阶治理见 advanced 阶段的自定义 Agent 工作流

面试可能问

如何证明 AI 提升了效率而不是制造技术债？

展示 dev-metrics：周期缩短且 7d 缺陷未恶化；覆盖率与 eval 命中率上升；openapi-diff 零 breaking。技术债用 ADR 与重构 budget 管理。

RAG MVP 最先做哪一段？

retrieve+synthesize 闭环 + mock 测试；ingest 可用手工小 corpus；eval 与 tenant 过滤从第一天纳入，避免后期返工。
