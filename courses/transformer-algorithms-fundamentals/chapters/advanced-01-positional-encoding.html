为什么要学本章

Self-Attention 本身置换不变——打乱 token 顺序，注意力权重矩阵只是行列重排，模型无法区分「第 3 个词」与「第 300 个词」。位置信息必须显式注入；而 2023～2026 主流开源 LLM（LLaMA、Qwen、DeepSeek）几乎清一色采用 RoPE。面试常问「为何不用原始 Transformer 正弦编码」「ALiBi 与 RoPE 谁更适合外推」——本章建立概念级直觉，不涉及预训练推导与 CUDA 实现。

本章先记住 3 件事
  • 绝对 vs 相对：绝对编码告诉模型「我在第 i 位」；相对编码强调「我与你相距 Δ」——长文外推更依赖后者。
  • RoPE 是主流：在 Q/K 上旋转向量，使注意力分数只依赖相对位置；LLaMA 系默认选择。
  • 外推 ≠ 免费午餐：训练 8K、推理 128K 需 YaRN/NTK 等应用侧策略；见 长上下文技术概览，本章不讲微调推导。

正弦位置编码

原始 Transformer（Vaswani et al., 2017）在嵌入层相加固定 正弦位置编码：偶数维用 sin，奇数维用 cos，不同维度对应不同波长。其设计动机是：对任意固定偏移 Δ，PE(pos+Δ) 可表示为 PE(pos) 的线性函数，从而隐含相对位置信息。

CorpAssist · 制度 PDF 切块后的顺序 同一政策文档切成 40 个 chunk 再拼接进 prompt 时，模型需知道「第 12 段在讲请假流程、第 13 段在讲审批链」——若缺少位置信号，Attention 可能把相隔很远的两条款误当作相邻条款比对。位置编码 就是给每个 token 打上「序列坐标」。

正弦公式（概念级）

PE(pos, 2i) = sin(pos / 100002i/d)
PE(pos, 2i+1) = cos(pos / 100002i/d)

pos 为位置索引；i 为维度对索引；d 为模型维度。最终输入：x = TokenEmbed + PE(pos)

方案 注入方式 是否可学习 长于训练长度时 代表模型
正弦绝对 PE 嵌入层相加 否（固定） 可计算任意 pos，但分布外推质量下降 原始 Transformer、早期 BERT
可学习绝对 PE 查表相加 是 无法直接超出训练最大长度 GPT-2、部分 Encoder
相对位置偏置 注意力分数 + bias(i−j) 可学习或固定 依赖 bias 表或 ALiBi 斜率 T5、Transformer-XL
RoPE 旋转 Q/K 否（结构固定） 配合 NTK/YaRN 外推 LLaMA、Qwen、DeepSeek
面试误区 · 避免
  • 说「正弦 PE 不能表达相对位置」——其实设计目标就包含相对性，只是长外推仍弱
  • 把「可学习 PE」说成万能——超长序列会直接越界
  • 声称「位置编码在 FFN 里」——标准 Transformer 在嵌入或 Q/K 注入
推荐表述
  • Attention 置换不变，必须额外注入顺序信息
  • 绝对 PE 加在 embedding；RoPE 改 Q/K 几何结构
  • CorpAssist 选模型时看有效上下文与位置方案，而非背公式
FAQ：BERT 也用位置编码，为何说 GPT 系更关心外推？

BERT 编码器常处理固定 512 token 分类/句对；GPT 类 Decoder 在推理时要自回归展开，KV Cache 随序列增长，上下文长度 直接决定能否一次读完整份制度 PDF。应用岗应关注 API 标注的 context window 与实测 p95，而非推导 PE 梯度。

相对位置与 ALiBi

相对位置 强调 token 对 (i, j) 的距离 Δ = i − j，而非各自绝对坐标。T5 在 Attention logits 上加可学习 bias(Δ)；ALiBi（Attention with Linear Biases, Press et al.）则用固定线性斜率惩罚远距离键，使模型在未见过更长序列的训练下，仍能在推理时保持较稳定的注意力衰减模式。

ALiBi 核心直觉

对第 m 个 query 与第 n 个 key，在缩放点积分数上添加 −m·|m−n|（m 为 head 专属斜率，越靠后的 head 斜率越大）。效果：距离越远，分数被扣得越多，形成inductive bias 偏向局部上下文。与 RoPE 相比，ALiBi 不修改 Q/K 向量本身，实现简单，曾用于 MPT、BLOOM 等；但 2024 后主流开源 LLM 仍以 RoPE 为主。

先想 10 秒：MPT 用 ALiBi 宣传 65K context，CorpAssist 能否直接把所有制度 chunk 塞进一个 prompt？

不能仅凭窗口数字决策。还需考虑：检索质量（无关 chunk 越多幻觉越高）、延迟与 Token 成本、以及模型在超长分布上的实测引用准确率。长窗口是必要条件，不是充分条件——见 长上下文技术概览 与 RAG 取舍节。

RoPE 旋转直觉

RoPE（Rotary Position Embedding）把位置信息通过复平面旋转注入 Query 与 Key：将相邻维度两两配对，按位置 pos 旋转角度 θ(pos)。神奇之处在于：旋转后的 q·k 内积只依赖 (pos_q − pos_k)，即相对位置，且与绝对起点无关。

二维旋转示意（单对维度）

python 
import numpy as np

def rotate2d(x_pair, pos, base=10000.0, dim_pair_idx=0):
    """概念演示：对 (x_{2i}, x_{2i+1}) 施加位置旋转"""
    freq = 1.0 / (base ** (2 * dim_pair_idx / 64))  # 示意 d=64
    angle = pos * freq
    c, s = np.cos(angle), np.sin(angle)
    x0, x1 = x_pair
    return np.array([x0 * c - x1 * s, x0 * s + x1 * c])

# 关键性质（口述面试）：
# rotate(q, m) · rotate(k, n) 的标量积 ∝ f(q, k, m-n)
# 相对距离 m-n 决定注意力，而非绝对 m、n

高维时对各维度对使用不同旋转频率（波长从短到长），低频维度捕获长距离相对关系，高频维度捕获局部邻接——与正弦 PE 的频率分解思想一脉相承，但作用在 Q/K 而非 embedding 加法。

面试追问要点回答勿深入（mustNotCover）
RoPE 为何只旋 Q/K，不旋 V？ 相对位置应影响「谁关注谁」的匹配；V 承载内容聚合，位置已通过权重体现 手写 CUDA 核
与 Multi-Head 关系？ 每个 head 独立 Q/K，RoPE 逐 head 应用；GQA 共享 K 时旋转仍按位置一致 Megatron 并行切分
推理增量解码？ 新 token 位置 pos，仅对新 Q/K 算旋转角；历史 KV 已带位置信息可缓存 KV Cache 内核实现

Self-Attention 本身不知道顺序；RoPE 在 Q/K 上做位置相关旋转，使注意力分数只依赖相对距离。LLaMA/Qwen 默认方案，配合 KV Cache 做增量解码。

  1. 问题：置换不变 → 必须注入位置。
  2. 绝对 PE 加 embedding，超长外推弱；可学习 PE 有最大长度墙。
  3. RoPE：复平面旋转 Q/K，内积只依赖 pos_q − pos_k。
  4. 不同维度对不同波长，兼顾局部与远程。
  5. 工程：与 GQA、Flash Attention、KV Cache 正交；外推靠 YaRN/NTK（见 长上下文技术概览）。

长上下文外推面试题

训练时常见 4K～8K，产品却宣传 128K～1M。位置外推 指在不重新预训练（本课不讲预训练）的前提下，让模型在更长序列上仍可用的推理侧技巧。应用岗需会解释概念与业务取舍，而非推导 loss。

高频面试题与答法骨架

题目核心答点CorpAssist 落地
128K 窗口是否取代 RAG？ 不能取代。成本 ∝ 序列长度；无关文本越多幻觉越高；检索仍控证据边界 制度库 200 万 chunk 必须检索；长窗适合「少而精」的拼接

FAQ：NTK 缩放 需要改权重吗？

常见推理时调整 RoPE 的 base 或应用 YaRN 插值策略，不改动已下载的 FP16 权重张量；属于推理框架 / 模型配置层。若面试官追问训练级扩展，应诚实划界：「本岗位关注推理配置与评测，预训练扩展不在我职责内」——对齐 mustNotCover。

位置编码速查

  • 正弦 PE：固定、可外推计算但质量衰减
  • 可学习 PE：有最大长度上限
  • ALiBi：线性偏置，免训练外推友好
  • RoPE：旋转 Q/K，相对距离进内积；主流默认
  • 外推：NTK / YaRN；配合 RAG，不替代检索

本章结论

  • 正弦位置编码：在嵌入层注入绝对坐标，隐含相对性但长外推仍弱；可学习 PE 受最大长度限制。
  • 相对位置与 ALiBi：用距离惩罚远距离注意力；ALiBi 线性偏置实现简单、外推省心，但生态次于 RoPE。
  • RoPE 旋转直觉：旋转 Q/K 使内积只依赖相对距离；LLaMA/Qwen 默认，与 KV Cache 增量解码兼容。
  • 长上下文外推面试题：长窗不取代 RAG；外推是推理侧技巧；评测要用长文档 golden set，不只看榜单短题。
