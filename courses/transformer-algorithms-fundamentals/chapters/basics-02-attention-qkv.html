本章先记住 4 件事
  • Q/K/V 都是学出来的投影：同一嵌入 X 乘以不同权重矩阵 WQ, WK, WV。
  • 相似度 = Q 与 K 的点积：越大表示「更相关」，再经 softmax 变概率分布。
  • 输出 = 权重的 V 加权和：相当于按相关性从「内容库」里取料。
  • 除以 √dk 防梯度爆炸：维度高时点积方差大，softmax 易极端化。

查询-键-值直觉

CorpAssist · 制度问答 用户问「驻外补贴能否与差旅叠加？」模型在组织每个新 Token 的答案时，需要决定：上下文里哪几句制度更值得参考。Attention 把这一过程变成可微分的矩阵运算。

注意力（Attention） 的核心抽象来自键值存储（Key-Value Store）：

  • Query（查询）：当前位置「想问什么」——例如正在生成「叠加」一词时，对上下文的提问向量。
  • Key（键）：每个上下文位置贴的「索引标签」——用于与 Query 算相似度。
  • Value（值）：每个位置实际携带的「内容」——相似度高则多取一些。

图书馆类比

Attention ≈ 软检索
  • Q = 你手里的问题卡片
  • K = 书架上每本书的标题/标签
  • 相似度 = Q·KT（点积）
  • V = 书的内容片段；输出 = 按相似度加权拼读
硬检索（RAG 工程层）
  • Query = 用户问题 Embedding
  • Key = 向量库文档 Embedding
  • Top-K 硬选 K 条 chunk
  • 无梯度、在模型外部完成

CorpAssist 的 RAG 在模型外做硬 Top-K；Transformer 内部每层都在做软 Attention——对所有 Token 赋予连续权重，且端到端可训练。

三矩阵形状速记（面试白板）

  • 输入 X：(n, d_model)
  • Q、K：(n, d_k)；V：(n, d_v)
  • 权重 A：(n, n)，每行和 = 1
  • 输出 O：(n, d_v)，常令 d_v = d_model

若面试官追问「Q 和 K 能否相同」：矩阵可相同但语义角色不同——Q 问「我要什么」，K 答「我有什么标签」；权重由二者交互产生，不是简单复制 X。

下列三点是阅读后续公式的锚点：① Q/K/V 同源不同投影；② 相似度 = 点积；③ 输出 = Value 的凸组合。

从嵌入到 Q/K/V 的线性投影

python 
# X: (seq_len, d_model) — 嵌入 + 位置编码后的序列
Q = X @ W_q   # (seq_len, d_k)
K = X @ W_k   # (seq_len, d_k)
V = X @ W_v   # (seq_len, d_v)
# W_q, W_k, W_v 为可学习参数；d_k 常取 d_model / num_heads

同一 Token 向量 X 经不同矩阵「扮演」不同角色——这是 Q/K/V 与 RAG 里「问题向量 / 文档向量」共用同一 Embedding 模型但用途不同的对照点。

Scaled Dot-Product 公式

原论文给出的 缩放点积注意力 是 Transformer 的默认注意力核：

核心公式（手写板版本）

latex 
Attention(Q, K, V) = softmax( Q @ K^T / sqrt(d_k) ) @ V

形状：
  Q: (n, d_k)   — n 个查询，每个 d_k 维
  K: (n, d_k)   — n 个键
  V: (n, d_v)   — n 个值
  Q @ K^T: (n, n) — 每对位置的相似度
  softmax 沿最后一维（对每个 Query 看所有 Key）
  输出: (n, d_v)

分步拆解

  1. 打分：S = QKT，Sij 表示第 i 个 Query 对第 j 个 Key 的相关性。
  2. 缩放：S' = S / √dk。当 dk=64 时除以 8，防止点积过大使 softmax 饱和。
  3. 归一化：A = softmax(S')，每行和为 1，得到注意力权重矩阵。
  4. 聚合：输出 = AV，按权重对 Value 行向量加权求和。
Scaled Dot-Product 数据流
flowchart LR
  X[嵌入 X] --> WQ[W_q]
  X --> WK[W_k]
  X --> WV[W_v]
  WQ --> Q[Query]
  WK --> K[Key]
  WV --> V[Value]
  Q --> DOT["Q @ K^T / sqrt(d_k)"]
  K --> DOT
  DOT --> SM[softmax 得权重 A]
  SM --> OUT["A @ V → 输出"]
  V --> OUT

Self-Attention 时 Q、K、V 同源序列 X；Cross-Attention 时 Q 来自解码器、K/V 来自编码器（见 多头与 Mask）。

NumPy 最小实现

python 
import numpy as np

def scaled_dot_product_attention(Q, K, V):
    d_k = Q.shape[-1]
    scores = Q @ K.T / np.sqrt(d_k)          # (n, n)
    weights = softmax(scores, axis=-1)       # 每行和为 1
    return weights @ V, weights

def softmax(x, axis=-1):
    x = x - np.max(x, axis=axis, keepdims=True)
    e = np.exp(x)
    return e / e.sum(axis=axis, keepdims=True)
先想 10 秒：若去掉 sqrt(d_k) 缩放，训练初期可能出现什么现象？

点积方差随 dk 增大而增大，softmax 输出接近 one-hot，梯度几乎只流向个别位置，注意力过于尖锐、难以学习多样对齐。缩放把分数拉回合理量级，使梯度更稳定。

注意力权重可视化

权重矩阵 A ∈ ℝn×n 是理解模型「在看哪里」的最直观产物。行 i 表示第 i 个 Token 作为 Query 时，对全文各位置的注意力分布。

热力图读法

轴含义CorpAssist 例
行（Query 位置） 正在处理 / 生成的 Token 生成「叠加」时那一行
列（Key 位置） 被参考的上下文 Token 「驻外补贴」「差旅报销」列权重高
颜色深浅 权重 0～1 深色 = 模型认为更相关

可视化注意事项

  • 单层单头热力图只是近似解释，多层多头叠加后人类难以完全还原决策。
  • 权重高 ≠ 因果证明；制度合规审计仍以 RAG 引用与人工复核为准。
  • 调试 CorpAssist badcase 时，可视化可辅助发现「模型是否关注到检索片段」——若权重全在 user 问题而忽略 chunk，可能是拼接或 Mask 配置错误。

调试 CorpAssist badcase 时的读图步骤

  1. 确认检索 chunk 是否已进入 prompt（RAG 链路）。
  2. 打开最后一层或指定层热力图，看生成关键 Token 对应行。
  3. 若权重集中在 user 问题而制度列接近 0 → 检查拼接顺序、Mask 或截断。
  4. 若权重分散均匀 → 可能 chunk 过长无关内容多，需收紧 Top-K 或改写 system。
能否用注意力权重直接做合规审计证据？

不建议。权重是内部浮点、随 checkpoint 变化；合规应依赖检索命中记录、引用条文 ID、回答文本。权重可视化仅用于研发排障。

与检索相似度类比

CorpAssist RAG 用 向量检索：query_embedding 与 doc_embedding 算 cosine 或内积，取 Top-K。Attention 打分 QKT 在数学上同为内积相似度（向量 L2 归一化后即 cosine 相关）。

对照表

维度RAG 向量检索Transformer Attention
发生位置 模型外、检索服务 模型内、每层每头
选择方式 Top-K 硬截断 softmax 软权重（全序列）
Q/K 来源 独立 Embedding 模型 同一前向中的 WQ, WK
训练 对比学习 / 双塔 与下游任务端到端反传
CorpAssist 角色 缩小进 prompt 的素材 生成时动态融合已进上下文 Token

双塔检索 vs 单塔 Attention（口述）

text 
RAG 双塔（离线）:
  score_i = cosine(Embed_query(q), Embed_doc(d_i))
  取 Top-K → 拼入 prompt

Self-Attention（在线、每层）:
  score_ij = Q_i · K_j / sqrt(d_k)
  softmax 得权重 → 加权 V_j

工程组合：RAG 负责「库外知识进上下文」；Attention 负责「上下文内动态融合」。

理解这层类比后，面试题「Attention 与 RAG 有何异同」可答：数学上都是相似度加权，工程上分工不同、应组合使用。下一章 Transformer 块 把 Attention 嵌入完整层结构。

组合策略（CorpAssist 制度问答推荐）

双层检索阅读

外层 RAG：向量+BM25 硬选 4～8 段进上下文；内层每层 Self-Attention：在已进上下文的 Token 间软融合。缺外层则知识进不来；缺内层则模型无法动态组织已给材料的措辞。

记住 缩放点积 四步：投影 Q/K/V → 打分 → softmax → 加权 V；面试手写到此即可，细节见 矩阵视角 与 NumPy 实验。

对 Java 后端：你实现的是 RAG 检索排序与 prompt 拼装；Attention 在 GPU 上的 matmul 由推理框架完成——但懂公式才能解释延迟与 badcase。

本章结论

每条对应上一节 h3，用于评审前 30 秒复述。

  • Q/K/V 直觉：Query 提问、Key 索引、Value 内容；均由 X 线性投影得到。
  • Scaled Dot-Product：softmax(QKT/√dk)V；缩放稳定 softmax 梯度。
  • 可视化：权重矩阵行列对应 Query/Key 位置；用于排障非合规证据。
  • 检索类比：内积相似度同源；RAG 硬检索在外、Attention 软融合在内。

判断练习

  1. （找错）实习生实现 Attention 时写 scores = Q @ K.T 省略了 / sqrt(d_k)，且抱怨「注意力全挤在第一个 Token」。最可能原因与修复？

    参考答案

    未缩放使 logits 过大，softmax 饱和为近似 one-hot，梯度几乎只更新单一位置。修复：恢复除以 √dk；并检查 Q/K 初始化是否过大。这与本章「缩放因子」动机一致。
