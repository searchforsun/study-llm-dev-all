手写 Attention 口述

高频白板题：写出 缩放点积注意力 并解释每步维度。应用岗答到矩阵形状与因果 Mask 即可，勿展开 CUDA tile。

四步口述模板

Attention(Q, K, V) = softmax(QKT / √dk) V

  1. 投影：Q, K, V 由输入 X 乘以 WQ, WK, WV 得到，shape (n, d)
  2. 打分：S = QKT / √dk，表达 token 两两相似度
  3. Mask（可选）：padding 置 −∞；因果 Mask 遮未来 token
  4. 聚合：权重 A = softmax(S)，每行和为 1；输出 = AV
Self-Attention 数据流
flowchart LR
  X[输入 X n×d] --> QKV[线性层得 Q,K,V]
  QKV --> SCORE["QKᵀ / √d_k"]
  SCORE --> MASK[Mask 可选]
  MASK --> SM[softmax]
  SM --> OUT["A·V → 输出"]

面试时先画此图，再写公式；比单写公式更有结构感。

追问要点CorpAssist 例子
为何除以 √d_k？ 维度大时点积方差大，softmax 趋 one-hot，梯度变小 长 chunk 多时仍靠缩放稳定训练（概念）
多头 做什么？ 多子空间并行关注不同关系，再拼接投影 一词多义：「苹果」公司 vs 水果
与检索相似度？ Q·K 类似 query-doc 打分，V 像文档内容加权 RAG 外挂检索是显式版「找相关段」
先想 10 秒：softmax 前行和为何必须 = 1？

保证输出是 V 的凸组合，可解释为「按关注度加权平均」；数值上避免尺度任意放大。若 mask 后全 -inf，需处理 NaN（工程实现细节，可口述「全 mask 行置零输出」）。

GPT vs BERT 对比题

二者代表 Decoder-only 与 Encoder-only 两条路线，GPT 自回归 因果 Mask 做生成；BERT 双向上下文做表示，MLM 训练（本课不推 loss）。CorpAssist：生成用 GPT 类，向量检索常用 Encoder。

维度 GPT（Decoder-only） BERT（Encoder-only）
注意力 因果 Mask，只看左侧 双向，全上下文
训练目标 Next-token 预测 MLM（掩码语言模型）
典型用途 对话、写作、代码补全 分类、向量 embedding、Rerank 特征
CorpAssist 制度问答最终生成 bge-m3 等检索 embedding
面试句 「生成式 LLM 主干」 「检索与表示层」
答错信号
  • 「BERT 也能做长文生成所以取代 GPT」——生成需自回归，BERT 非为此设计
  • 「GPT 不能做 embedding」——可用末层 hidden state，但非最优路线
  • 混淆 T5（Encoder-Decoder）与 GPT
加分句
  • 「RAG = Encoder 检索 + Decoder 生成」
  • 「同一 CorpAssist 请求可经两套模型：小 embedding + 大生成」
  • 「选型看任务，不迷信单一架构」

复杂度与优化题

Self-Attention 对序列长度 n 复杂度 O(n²)（算力与内存）。推理优化：KV Cache 避免重复算历史 K/V；Flash Attention 融合 kernel 减 HBM 读写；滑动窗口 / 稀疏 Attention 限制感受野（见 长上下文技术概览）。

优化解决复杂度/资源应用岗关注
朴素 Attention 基线 O(n²) 算力，O(n) KV 存 长 prompt 贵
KV Cache 解码步复用历史 K/V 每步 O(n) 增存 TTFT 与并发显存
Flash Attention IO 融合 同渐进复杂度，常数更小 p95 延迟改善
GQA/MQA 减 K/V head 数 降 KV 体积 见 架构对照
滑动窗口 限制注意力范围 O(n·w) 远程依赖需 RAG

幻觉/长文失效算法解释

幻觉 不只有「模型胡说」：从算法视角，训练目标（预测下一 token）并不保证事实正确；上下文过长时注意力分散、位置外推弱化，远程证据权重下降。应用侧用 RAG 约束证据、评测集监控引用率，而非指望改 Attention 公式一劳永逸。

现象算法层解释（概念）工程缓解
无检索胡编制度条款 生成分布偏向流畅而非可验证 RAG + 强制引用 + 拒答
长 prompt 漏中间段 「Lost in the Middle」注意力偏首尾 减 K、Rerank、分段摘要
超长外推掉点 RoPE 分布外，远程依赖弱 YaRN + 长文档 golden set
工具参数编造 Decoder 仍做 token 预测 JSON schema、函数调用 API
答辩误区
  • 「换更大模型就没有幻觉」——仅降低概率
  • 「Attention 可视化能完全解释错误」——相关非因果
  • 承诺「微调一定消除幻觉」——需数据与评测（不讲 RLHF 推导）
推荐结构
  • 承认生成目标与事实一致性不等价
  • 给 RAG + 护栏 + 评测三板斧
  • 用引用率、幻觉率指标闭环

面试 TOP10 · 题干与参考答案要点
  1. 手写/口述 Self-Attention 公式与四步。
    • 答点：QKT/√dk → mask → softmax → 乘 V；行和为 1；与检索相似度类比。
  2. GPT 与 BERT 的核心区别？CorpAssist 各用在哪？
    • 答点：因果 vs 双向；生成 vs 表示；Decoder 问答 + Encoder embedding。
  3. Attention 复杂度？长文档为何贵？
    • 答点：O(n²)；prompt token 增 → 算力与 KV 增；对策 RAG 减 n、Flash、GQA。
  4. KV Cache 是什么？为何能加速自回归？
    • 答点：缓存历史 K/V；新 token 只算一步 Q 与历史 K 注意力；显存换时间。
  5. RoPE 与 ALiBi 如何帮助长上下文？
    • 答点：RoPE 相对距离进内积 + YaRN 外推；ALiBi 线性偏置免训练延伸；见 位置编码演进：绝对/相对/RoPE。
  6. MoE 总参数与激活参数区别？
    • 答点：top-k 专家路由；成本看激活参数；Mixtral vs DeepSeek 共享专家。
  7. 128K 窗口能否取代 RAG？
    • 答点：不能；成本、幻觉、溯源；检索定证据长窗补连贯。
  8. 幻觉的算法层原因与工程缓解？
    • 答点：MLE 目标≠事实；长文注意力分散；RAG+引用+评测。
  9. Flash Attention 解决什么？是近似 Attention 吗？
    • 答点：IO 优化，数学等价；降常数延迟；非 CUDA 面试可止于此。
  10. 遇「讲讲 RLHF / 预训练」怎么答？
    • 答点：诚实 mustNotCover；转应用岗：评测、RAG、网关、成本；STAR 案例证明交付。
