本章先记住 4 件事
  • 双向上下文：Encoder 自注意力无因果掩码，整句互相可见（训练时用 MLM 防泄露）。
  • 输出是向量：句级/段级 embedding 用于相似度，不是逐 token 生成。
  • RAG 标配：检索用 Encoder；回答用 Decoder-only Chat 模型。
  • 别混用：把 Chat 模型当 embedding 模型通常效果差、成本高。

MLM 与双向上下文

CorpAssist · 制度段落入库 rag-index 将 HR 制度 PDF 分块为 512 token 段落。每段经 bge-large-zh（BERT 类 Encoder）编码为 1024 维向量。用户问「产假天数」时，query 也用同一 Encoder 编码，在向量库做 ANN 检索——这要求表示对整句语义双向理解，而非自回归续写。

BERT（Bidirectional Encoder Representations from Transformers）使用仅 Encoder 栈：多层双向 Self-Attention + FFN。预训练任务为 Masked Language Model（MLM）：随机 mask 15% token，模型根据左右两侧上下文预测被 mask 词。与 GPT 的 Next-Token 不同，MLM 需要双向可见性，故不使用因果掩码。

BERT Encoder 与 MLM
flowchart LR
  subgraph enc [Encoder Stack]
    E1[Token + Segment + Position]
    E2[Bi-directional Self-Attention]
    E3[FFN x L layers]
  end
  E1 --> E2 --> E3
  E3 --> MLM[MLM Head 预测 mask 位]
  E3 --> POOL[句向量 pooling]

推理检索时不用 MLM Head，而取最后一层 hidden states 做 pooling 得 embedding。

先想 10 秒：能否用 GPT 做 MLM 式双向填空？

标准 GPT 因果掩码禁止看右侧 token，无法直接做 BERT 式 MLM。有研究用特殊训练或 UniLM 混合掩码，但 CorpAssist 检索仍选成熟 Embedding 模型 更稳妥。

Embedding 输出用途

Encoder 最后一层输出 shape 约为 (seq_len, hidden_dim)。检索常用句级 pooling：

  • [CLS] token：BERT 原设计，取第 0 位向量
  • Mean pooling：对非 pad 位置求平均（bge、E5 常见）
  • Last token：部分新模型取最后有效 token

向量经 L2 归一化后，用内积或余弦相似度检索。CorpAssist rag-index 存储 chunk_id, embedding, metadata，query 向量与库内向量做 top-k ANN。

检索服务伪代码

python 
# CorpAssist rag-service 概念流程（Encoder 路线）
from embedding_client import encode  # 调用 bge / text-embedding API

def index_chunks(chunks: list[str], collection: str) -> None:
    vectors = encode(chunks, model="bge-large-zh-v1.5", normalize=True)
    vector_store.upsert(collection, ids=chunk_ids, vectors=vectors)

def retrieve(query: str, collection: str, top_k: int = 6) -> list[dict]:
    q_vec = encode([query], model="bge-large-zh-v1.5", normalize=True)[0]
    hits = vector_store.search(collection, q_vec, top_k=top_k)
    return [{"chunk_id": h.id, "text": h.text, "score": h.score} for h in hits]
输出类型用途CorpAssist 示例
段向量 ANN 检索 制度库 top-6 片段
Query 向量 与段向量相似度 用户问题 embedding
Cross-encoder 分数 精排（可选第二层） 对 top-50 做 pairwise 打分
Token 级表示 NER、分类（非本课 RAG 主路径） 工单意图离线标注
反模式
  • 用 Chat 模型 last hidden 当 embedding，无对比学习
  • 检索与入库用不同模型维度
  • 不做 normalize 导致内积尺度漂移
推荐
  • 统一 embedding 模型版本与 chunk 版本 别名
  • 评测集固定 query-doc 相关性标签
  • 大库用 HNSW/IVF + 定期 rebuild

与 RAG 向量模型关系

现代 RAG 向量模型多继承 BERT Encoder 架构，经对比学习（In-batch negatives、难负例）微调，使语义相近的 query-doc 向量接近。与原始 BERT MLM 不同，但双向 Encoder 骨干相同。CorpAssist 典型链路：

CorpAssist RAG：Encoder 检索 + Decoder 生成
flowchart LR
  DOC[制度 PDF 分块] --> ENC1[Encoder Embedding]
  ENC1 --> IDX[(rag-index)]
  Q[用户问题] --> ENC2[同一 Encoder]
  ENC2 --> IDX
  IDX --> CITE[top-k 引用块]
  CITE --> PROMPT[编排层拼 messages]
  PROMPT --> DEC[Decoder-only Chat 生成]

两阶段、两模型：检索质量上限由 Encoder 决定；最终措辞由 Decoder 决定。

常见增强：Hybrid 检索（BM25 + 向量）、Reranker（Cross-encoder，仍多为 Encoder 变体）。这些都不替代最终 Chat 生成，但影响引用是否准确——引用错则 Decoder 再强也会幻觉。

text-embedding-3 与 bge 选型差异？
看维度、中文效果、延迟、私有化部署与许可证。CorpAssist 若数据不出域，优先私有化 bge/m3e；多语言场景可看 E5/multilingual。务必在 golden set 上测 Recall@k，而非只看 MTEB 榜单。
ColBERT 等多向量方案要懂到什么程度？
应用岗知道「token 级 late interaction 提精度、增存储与延迟」即可。默认单向量 ANN 够用后再优化；本课 advanced 长上下文章会涉及检索与窗口取舍。

何时选 Encoder 类模型

任务推荐架构理由
语义检索 / 去重 / 聚类 Encoder embedding 固定维向量，ANN 友好；双向语义
问答生成 / 摘要 / Agent Decoder-only Chat 自回归生成长文本
分类 / NER / 情感 Encoder + 分类头 整句表示 + 小头微调
Seq2Seq 翻译（传统） Encoder-Decoder（T5/BART） CorpAssist 少见；多被 Chat 替代
精排 rerank Cross-encoder（双句输入） 精度高、只适合小候选集

选型决策树（面试口述）

text 
需要输出长文本给用户？
  ├─ 是 → Decoder-only Chat（Qwen/GPT/LLaMA）
  └─ 否 → 需要固定长度向量做相似度？
        ├─ 是 → Encoder Embedding（bge/E5/厂商 embedding API）
        └─ 否 → 结构化标签 → Encoder + 分类头

CorpAssist 面试常问：「为什么 RAG 不直接用 GPT embedding？」标准答：GPT 因果训练目标优化的是续写而非语义相似度；专用 embedding 经对比学习，Recall@k 更稳，且维度与索引 pipeline 成熟。

快速对比：GPT vs BERT 一句话

GPT：因果 Decoder，猜下一个词，擅长生成。BERT：双向 Encoder，MLM 表示学习，擅长句向量与理解类任务。CorpAssist：BERT 路线检索，GPT 路线回答。

重建索引时必须整库同一 embedding 模型与维度；混用旧向量会导致检索分数不可比。编排层在 debug 模式可记录 query 与 top-1 的 cosine，便于 badcase 归因是「检索错」还是「生成错」。

本章结论

  • MLM 与双向：Encoder 无因果掩码，MLM 学上下文表示。
  • Embedding 输出：pooling 得句向量，入库 ANN；query 同模型编码。
  • RAG 关系：检索 Encoder + 生成 Decoder，两模型分工明确。
  • 选型：相似度/检索选 Encoder；对话生成选 Decoder-only。
