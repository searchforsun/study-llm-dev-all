本章先记住 4 件事
  • Scale：除以 sqrt(d_k) 防点积过大 softmax 饱和。
  • Softmax 按行：对每个 query 位置，在所有 key 上归一化。
  • Mask：未来位置置 -inf（实现用 -1e9）再 softmax → 权重 0。
  • 输出：权重矩阵乘 V 得加权 value 组合。

实现 Scaled Dot-Product

给定序列长度 seq_len 与模型维 d_model，设 Q/K/V 形状均为 (seq_len, d_model)（单头简化）。注意力分数 S = Q K^T / sqrt(d_k)，权重 A = softmax(S)，输出 O = A V。

核心公式（LaTeX）

latex 
\mathrm{Attention}(Q,K,V)=\mathrm{softmax}\left(\frac{QK^\top}{\sqrt{d_k}}\right)V

NumPy 实现（与 demo 一致）

python 
import numpy as np

def softmax(x, axis=-1):
    shifted = x - np.max(x, axis=axis, keepdims=True)
    exp = np.exp(shifted)
    return exp / np.sum(exp, axis=axis, keepdims=True)

def scaled_dot_product_attention(q, k, v, mask=None):
    d_k = q.shape[-1]
    scores = q @ k.T / np.sqrt(d_k)
    if mask is not None:
        scores = np.where(mask, scores, -1e9)
    weights = softmax(scores, axis=-1)
    return weights @ v, weights

CorpAssist 类比：RAG 引用 token 在 GPT 前缀中，因果 mask 保证生成第 t 个答案 token 时不能「偷看」尚未生成的后续答案 token；但对已写入前缀的引用，只要位置 ≤ 当前，即可 attend。

张量shape（单头）含义
Q(seq, d_k)「我要查什么」
K(seq, d_k)「可被匹配的键」
V(seq, d_v)「取回的内容」
weights(seq, seq)对每个 query 行 softmax 和为 1
output(seq, d_v)加权后的表示
先想 10 秒：为何不除以 d_k 而除以 sqrt(d_k)？

点积方差随 d_k 增大而增大，softmax 会极端 one-hot，梯度变小。缩放使分数分布更稳定，与 缩放点积 一致。

因果 Mask 实验

因果 Mask 为下三角矩阵 M：M[i,j]=1 当 j≤i，否则 mask 掉。实现上用 np.tril 生成 bool mask，在 softmax 前把禁止位置设为 -1e9。

4×4 因果 Mask（1=可见）
flowchart TB

    direction TB
    R1["行0: 1 0 0 0"]
    R2["行1: 1 1 0 0"]
    R3["行2: 1 1 1 0"]
    R4["行3: 1 1 1 1"]
  end

行 i 只能 attend 到列 j≤i；对应 GPT 自回归「不看未来」。

生成 mask 与调用

python 
def causal_mask(seq_len):
    return np.tril(np.ones((seq_len, seq_len), dtype=bool))

seq_len = 4
mask = causal_mask(seq_len)
out, weights = scaled_dot_product_attention(q, k, v, mask=mask)
# 上三角应 ≈ 0
assert np.all(weights[np.triu_indices(seq_len, k=1)] < 1e-6)
常见错误
  • mask 用 0/1 直接乘 scores（softmax 后非零）
  • softmax 沿错轴（列而非行）
  • 忘记 scale，权重 one-hot 极端
正确做法
  • 禁止位 large negative 再 softmax
  • axis=-1 对 key 维归一化
  • 验收 weights.sum(axis=1) ≈ 1

口述：手写 Attention 四步
  1. 线性投影得 Q、K、V
  2. scores = QK^T / sqrt(d_k)，加 mask 若有
  3. weights = softmax(scores, dim=-1)
  4. output = weights @ V

手算例题（seq_len=2, d_k=2）

设 Q=K=V（省略投影）：q1=[1,0], q2=[0,1]。无 mask 时 scores 为 QK^T；加因果 mask 后第 1 行只 attend 位置 0。可在纸面算 softmax 验证行和为 1——与 demo 打印一致。

步骤CorpAssist 工程映射
Q/K/V 投影 模型权重 Wq/Wk/Wv；应用层不可见
Scale + softmax 数值稳定性影响长上下文精度
Causal mask Chat 流式生成不泄露未来 token
O = AV hidden state 传入 FFN；堆 L 层
demo 中 rng seed=42 的作用？
固定随机 Q/K/V 投影，使每次运行打印的权重矩阵一致，便于对照讲义与同学之间比对。生产模型权重由训练决定，非随机初始化。
为何 lab 不用 PyTorch 而用 NumPy？
NumPy 无 GPU 依赖，适合课程离线验收；公式与矩阵维度一目了然。确认直觉后再用 PyTorch/Flash 做规模化和加速。
如何将本 lab 扩展到多头？
在 Q/K/V 增加 head 维，对每个 head 独立调用 scaled_dot_product_attention，concat 后乘 W_o。CorpAssist 线上模型 head 数见 config 的 num_attention_heads。
Attention 手算检查清单
  1. scores shape = (seq, seq)
  2. 每行 softmax 和 = 1
  3. causal：上三角权重 ≈ 0
  4. output shape = (seq, d_v)

本章结论

  • Scaled Dot-Product：scale + softmax + 乘 V，NumPy 可完整实现。
  • 因果 Mask：下三角可见，上三角权重为 0，对应 GPT decode。
  • PyTorch：scaled_dot_product_attention 数值对照，Flash 为加速实现。
  • 验收：每行权重和为 1；causal 上三角 ≈ 0。
