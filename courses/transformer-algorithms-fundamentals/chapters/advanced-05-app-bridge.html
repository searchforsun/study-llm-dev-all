懂架构如何助力 RAG/Agent

RAG 不是「向量库 + 大模型」五个字；Agent 也不是「多调几个 API」。懂 Transformer 帮你在正确层做决策：

本课知识点RAG 设计影响Agent 设计影响
GPT vs BERT 检索 embedding 与生成模型分离 规划用 Decoder；工具结果再注入上下文
O(n²) 与 KV Cache 控制 Top-K chunk 总 token 步数预算；长工具输出要摘要
RoPE / 长上下文 长窗是拼接预算非全库 多轮历史要摘要或结构化状态
MoE 激活参数 主生成模型成本估算 大小模型分工：规划大、执行小
幻觉与 Lost in the Middle Rerank、减 K、强制引用 工具结果校验，避免模型编造参数
因果 Mask 生成路径单向；检索可双向 ReAct 循环是多次自回归调用
CorpAssist · 从算法到工程的决策链 业务要求「100 页制度 PDF 问答」→ 算法认知：全量 Attention O(n²) 不可接受 → 工程决策：混合检索 Top-6 + Rerank + 生成模型 32K 窗 + 引用 chunk_id → 指标：引用率、P95、幻觉率。缺任何一环都可能做成「能 demo 不能上线」。

FAQ：Python 栈与 Java 栈如何选？

算法本课双栈无关。应用阶段按岗位选 RAG 系统（Python 工程实践）rag-system-py 或 RAG 系统（Spring AI 工程实践）rag-system-java；Spring 背景优先 Java 轨，检索/Python 生态常作子服务。
