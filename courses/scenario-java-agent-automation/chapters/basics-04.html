已完成记忆与工具生态（Spring AI）

约 30 分钟 · 阶段：Agent 基础 · 场景：S3 Agent 自动化

为什么学这章

Agent 需要"记忆"来维持对话上下文、学习用户偏好、跨会话保持状态。Spring AI 提供了 ChatMemory（短期记忆）和向量存储（长期记忆）两层记忆体系。同时，Agent 的能力边界由工具决定，理解 MCP 工具注册和权限管理是 Agent 扩展的关键。

学完后能做什么

配置 Spring AI ChatMemory 的滑动窗口和摘要模式；用向量存储实现跨会话长期记忆；注册和调用 MCP 工具；配置工具的权限白名单和调用审计。

本章要点速记

1) Spring AI ChatMemory 有四种实现：InMemory/SlidingWindow/MessageHistory/Summarize；2) 长期记忆使用向量数据库（如 Milvus）存储用户偏好和历史决策；3) MCP（Model Context Protocol）提供标准化的工具注册接口，Spring AI Alibaba 支持 MCP 客户端；4) 工具权限白名单按用户角色限定可调用的工具范围；5) CorpAssist 短期记忆配置为 Summarize 模式（60% Token 节省），长期记忆基于 Milvus 向量检索。

Spring ChatMemory 短期记忆

短期记忆维持当前对话的上下文。Spring AI 提供了四种策略，每种适用于不同的场景和 Token 预算。

java
// Spring AI 短期记忆配置
@Configuration
public class MemoryConfig {

    @Bean
    public ChatMemory shortTermMemory() {
        // Summarize 模式：每 5 轮对话压缩一次历史
        return ChatMemory.builder()
            .strategy(ChatMemory.Strategy.SUMMARIZE)
            .maxMessages(50)           // 保留最近 50 条消息
            .summarizeThreshold(10)    // 超过 10 条触发摘要
            .summarizeTokenLimit(1024) // 摘要最大 Token 数
            .build();
    }

    @Bean
    public ChatClient memoryChatClient(ChatClient.Builder builder,
                                        ChatMemory chatMemory) {
        return builder
            .defaultSystem("你是 CorpAssist 企业办公助手。")
            .chatMemory(chatMemory)
            .build();
    }
}

// 滑动窗口模式（备用）
@Configuration
@Profile("sliding-window")
public class SlidingWindowConfig {

    @Bean
    public ChatMemory slidingWindowMemory() {
        return ChatMemory.builder()
            .strategy(ChatMemory.Strategy.SLIDING_WINDOW)
            .windowSize(20)            // 保留最近 20 条消息
            .build();
    }
}

CorpAssist 场景

员工小张在 CorpAssist 中连续请求："查我的工单"→"看下还有没有未完成的"→"帮我给项目经理发个进度消息"。没有短期记忆的话，Agent 每次都需要重新问"你是谁""哪个项目"。有了 Summarize 模式的 ChatMemory，Agent 自动将前三轮对话压缩为"用户是小张，正在查看工单状态并想给项目经理发进度消息"，后续对话直接使用摘要作为上下文。每次摘要约 200 Tokens，相比完整对话历史节省 60% Token。

向量存储长期记忆

长期记忆需要跨会话保持，用于记住用户的偏好、历史决策、已执行的任务模板。Spring AI 通过 VectorStore 抽象支持多种向量数据库。

java
// 长期记忆：用户偏好存储
@Component
public class LongTermMemory {

    private final VectorStore vectorStore;

    public LongTermMemory(VectorStore vectorStore) {
        this.vectorStore = vectorStore;
    }

    public void rememberUserPreference(String userId, String key, String value) {
        Document doc = new Document(
            "preference:" + userId + ":" + key,
            Map.of("userId", userId, "key", key, "value", value, "type", "preference")
        );
        vectorStore.add(List.of(doc));
    }

    public List<Document> recallUserPreferences(String userId, String query) {
        // 构建搜索请求
        SearchRequest request = SearchRequest.builder()
            .query("用户 " + userId + " 的偏好: " + query)
            .topK(5)
            .similarityThreshold(0.7)
            .filterExpression("userId == '" + userId + "' && type == 'preference'")
            .build();
        return vectorStore.similaritySearch(request);
    }

    public void rememberTaskHistory(String userId, String taskType,
                                     String taskResult) {
        Document doc = new Document(
            "task:" + userId + ":" + taskType + ":" + Instant.now(),
            Map.of("userId", userId, "taskType", taskType,
                   "result", taskResult, "timestamp", Instant.now().toString())
        );
        vectorStore.add(List.of(doc));
    }
}

仅有短期记忆

用户每次打开 CorpAssist 都是"新会话"，需要重新介绍自己、重新说明偏好。Agent 无法从历史中学习用户的写作风格和常用模板。

短期+长期记忆

Agent 在用户登录时自动加载历史偏好："小张，上次你用了 Markdown 格式的周报模板，这次是否继续使用？" 用户无需重复说明，体验连续。

MCP Client 工具注册

MCP（Model Context Protocol）是 Anthropic 提出的模型上下文协议。Spring AI Alibaba 支持 MCP 客户端，可以通过标准化的 HTTP 或 SSE 接口注册和使用外部工具。

java
// MCP Client 配置
@Configuration
public class McpConfig {

    @Bean
    public McpClient corpMcpClient() {
        return McpClient.builder()
            // 注册 MCP 工具服务器
            .server("calendar", "http://calendar-service:8081/mcp")
            .server("email", "http://email-service:8082/mcp")
            .server("ticket", "http://ticket-service:8083/mcp")
            .timeout(Duration.ofSeconds(30))
            .build();
    }

    @Bean
    public List<ToolCallback> mcpToolCallbacks(McpClient mcpClient) {
        // 将 MCP 工具注册为 Spring AI ToolCallback
        return mcpClient.listTools().stream()
            .map(toolDef -> new McpToolCallback(toolDef, mcpClient))
            .collect(Collectors.toList());
    }
}

// 自定义 MCP 工具适配器
public class McpToolCallback implements ToolCallback {

    private final McpToolDefinition toolDef;
    private final McpClient mcpClient;

    @Override
    public String getName() {
        return toolDef.server() + "_" + toolDef.name();
    }

    @Override
    public String getDescription() {
        return toolDef.description();
    }

    @Override
    public JsonSchema getInputSchema() {
        return toolDef.inputSchema();
    }

    @Override
    public String call(String functionInput) {
        return mcpClient.callTool(toolDef.server(), toolDef.name(), functionInput);
    }
}

CorpAssist 场景

企业内已有日历服务（Java）、邮件服务（Java）、工单服务（Python）。通过 MCP 协议，这三个服务可以统一注册到 CorpAssist Agent：CalendarService 暴露 createEvent/queryEvents/getFreeSlots；EmailService 暴露 sendEmail/queryEmails/createDraft；TicketService 暴露 createTicket/queryTicket/assignTicket。Agent 可以跨服务编排任务如"查一下明天下午的空闲时间→预约会议室→发送会议邀请给团队成员→创建跟进工单"。

工具权限白名单

不是所有用户都能调用所有工具。CorpAssist 使用分层权限模型，按用户角色控制可用的工具。

角色可用工具审批要求审计级别
普通员工查用户、查日程、查工单、写周报发邮件需确认基础
部门经理以上全部 + 批量发通知、创建项目批量操作需审批详细
系统管理员全部工具敏感操作需审批完整
外部用户仅查公共日程、提交工单全部审批完整
java
// 工具权限白名单
@Component
public class ToolPermissionChecker {

    private final Map<String, List<String>> roleTools = Map.of(
        "EMPLOYEE", List.of("getUserInfo", "querySchedule", "queryTicket", "generateReport"),
        "MANAGER", List.of("getUserInfo", "querySchedule", "queryTicket",
            "generateReport", "batchNotify", "createProject", "sendEmail"),
        "ADMIN", List.of("*"),  // 通配符
        "EXTERNAL", List.of("queryPublicSchedule", "submitTicket")
    );

    public void checkPermission(String userId, String toolName) {
        String role = getUserRole(userId);
        List<String> allowed = roleTools.get(role);
        if (allowed == null || (!allowed.contains(toolName) && !allowed.contains("*"))) {
            throw new PermissionDeniedException(
                "用户 " + userId + "（角色:" + role + "）无权调用 " + toolName
            );
        }
    }

    public void auditCall(String userId, String toolName,
                           String params, String result) {
        // 记录审计日志
        AuditLog log = new AuditLog(userId, toolName, params,
            result, Instant.now());
        auditLogger.save(log);
    }
}
Q: MCP 工具和 @Tool 注解的区别是什么？

@Tool 注解是 Spring AI 原生的工具定义方式，方法直接写在 Java 代码中，适合内部工具。MCP 工具是通过标准协议调用的外部服务，适合跨团队、跨语言的服务集成。CorpAssist 实践中：核心工具（如 sendEmail、queryTicket）用 @Tool 注解直接实现，扩展工具（如 hrService、financeService）通过 MCP 注册。MCP 的优势在于：1) 工具服务器可以独立部署和扩展；2) 支持多种语言实现；3) 协议标准化，工具定义自动生成。

本章结论

  • Spring AI ChatMemory 四种策略中，Summarize 模式压缩历史为摘要，节省约 60% Token
  • 长期记忆基于向量存储（Milvus），通过语义搜索检索用户偏好和历史任务模板
  • MCP Client 标准化外部工具注册，支持跨语言服务集成到 Spring AI Agent
  • 工具权限白名单按角色控制可用工具，防止越权调用，每次调用记录审计日志
  • CorpAssist 采用四层记忆体系：短期记忆（会话上下文）+ 长期记忆（用户偏好）+ 任务记忆（历史模板）+ 审计记忆（操作追溯）

面试可能问

Summarize 模式的摘要丢失了重要信息怎么办？

三种应对策略：1) 摘要保留关键字段——在 Summarize Prompt 中指定"保留以下信息：用户身份、任务目标、已完成步骤、决策原因"，确保这些关键信息不会被压缩掉；2) 分级压缩——短期记忆（最近 10 轮）保留完整对话，中期记忆（10-30 轮）使用摘要，长期记忆（30 轮以上）仅保留关键决策点；3) 用户手动保留——用户可以标记某条消息为"重要"，标记的消息不会被压缩。CorpAssist 使用方案 2，根据经验 80% 的上下文来自最近 10 轮对话，摘要丢失重要信息的概率约 2%。

MCP 工具的延迟比 @Tool 高多少？如何优化？

MCP 工具比 @Tool 多一次网络调用（HTTP/SSE），典型的延迟差异：@Tool 调用约 5-20ms（同进程方法调用），MCP 工具约 50-200ms（网络往返 + 序列化）。优化策略：1) 连接池——MCP Client 使用 HTTP 连接池（keep-alive）减少连接建立时间；2) 本地缓存——对于频繁调用的工具（如 getUserInfo），使用本地缓存减少重复请求；3) 批量处理——MCP 协议支持批量调用，将多个工具请求合并为一次网络往返；4) 预连接——Agent 初始化时建立 MCP 连接，避免首次调用时的手续延迟。优化后 MCP 工具延迟可降至 20-50ms。

长期记忆的向量检索效果不好（检索不到相关记忆）怎么办？

四步优化：1) 元数据过滤——搜索时使用 filterExpression 限定用户范围（如 userId=='u123'），避免检索到其他用户的记忆；2) Embedding 模型优化——使用 domain-specific 的 Embedding 模型（如 bge-large-zh）替代通用模型，提升办公场景的语义匹配效果；3) 混合检索——结合关键词匹配（BM25）和向量检索，先精确匹配用户 ID+偏好键，再语义搜索；4) 记忆刷新——Agent 每次生成回答后，主动将关键的交互结果写入长期记忆，确保"刚刚使用过的偏好"一定在最新数据中。

场景：用户说"按上次的方式处理"，Agent 如何理解"上次的方式"？

需要从长期记忆中检索用户的历史任务记录：1) Agent 在 System Prompt 中加入"如果用户提到'上次'或'之前'，请先查询长期记忆中的历史任务"；2) Agent 调用 longTermMemory.recallUserPreferences 查询"用户 u123 在最近 7 天内同类型任务的执行记录"；3) 检索到的记录包含 taskType、参数、执行方式，Agent 参照上次的参数和方式执行；4) 如果检索不到，Agent 询问用户"我没有找到上次的记录，请描述一下您希望的处理方式"。CorpAssist 的实现中，每个任务执行结果的 metadata 会记录 taskType、parameters、template，使后续可以按相同方式快速重执行。

场景：管理员误操作调用了 deleteProject 工具，如何通过权限系统阻止？

三层防护：1) 调用前检查——ToolPermissionChecker.checkPermission 检查用户角色是否有 deleteProject 权限，管理员角色默认有该权限，但 deleteProject 被标记为 HIGH_RISK，需要额外审批；2) 高风险确认——被标记为 HIGH_RISK 的工具调用时，Agent 自动暂停执行并请求用户二次确认"您确定要删除项目 PRJ-001 吗？此操作不可撤销。"；3) 二次确认——用户确认后，还需要输入验证码或进行生物识别。CorpAssist 的 HIGH_RISK 工具列表包括 deleteProject、batchDeleteUsers、updateSalaryInfo 等 8 个敏感操作。
