## 目标

用 CorpAssist HR 制度 v3.2 场景，熟悉**采集 → 发布 → 监控 → 退役**四阶段及 ops handoff 检查点。

## 步骤

1. 打开 [`lifecycle-stages.md`](lifecycle-stages.md)。
2. 在四个阶段下各勾选至少 1 项与本章正文表/Mermaid 对应的检查点。
3. 在「HR v3.2 批次」备注区写明：该批次当前处于哪一阶段、下一交接角色是谁。

## 验收

- [ ] 四阶段均有勾选，且无空节
- [ ] 备注中点名 **alias / version_tag** 或 **ops handoff** 至少其一
- [ ] 能口述 Python 管线与 Spring VectorStore 的交界（alias 切换）
