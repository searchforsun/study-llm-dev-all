本章先记住 3 件事
  • 覆盖率：有满意答案的 query 占比，不是 chunk 数量。
  • 空答要分型：zero hit / filter 全拒 / LLM 无依据拒答，Remediation 不同。
  • 热点反哺 ingest：Top query 无 doc 命中 → 优先补库或修 ACL。

覆盖率

CorpAssist · 季度复盘 Q1 共 12,400 次 KB 问答；运营抽样 400 条标注「是否由 KB 充分回答」。覆盖率 = 满意数 / 抽样数。若仅 68% 且「报销流程」类 recall gap 集中，则优先补 ingest 而非调 prompt。

coverage rate 衡量「用户问题被 KB 支撑的程度」。CorpAssist 采用分层抽样 + 意图标签，避免被高频简单 FAQ 拉高均值。

指标定义

指标公式目标（S1）
coverage_rate satisfied_samples / total_samples ≥ 75%
recall_gap_rate queries_with_no_relevant_hit / total_queries ≤ 8%
intent_coverage 按 intent 维度单独统计 HR ≥ 80%, Finance ≥ 70%
python 
# corpassist/metrics/coverage.py
def daily_coverage(logs: list[RagLog]) -> CoverageReport:
    sampled = stratified_sample(logs, by="intent", n=200)
    satisfied = sum(1 for x in sampled if x.user_feedback == "helpful")
    gap = sum(1 for x in logs if x.max_score < 0.55 and x.hit_count == 0)
    return CoverageReport(
        rate=satisfied / len(sampled),
        recall_gap_rate=gap / len(logs),
    )

def classify_empty(log: RagLog) -> str:
    if log.hit_count_before_filter > 0 and log.hit_count_after_filter == 0:
        return "ACL_FILTER"
    if log.hit_count == 0 and log.max_score < 0.55:
        return "ZERO_HIT"
    return "LLM_REFUSAL"

先想 10 秒：chunk 数从 10 万增至 15 万，coverage_rate 一定上升吗？

参考思路

不一定；若新增多为低价值重复 chunk，recall gap 可能不变。应看 intent 维度 coverage 与 recall_gap。

空答

empty answer 在 CorpAssist 分三类，Remediation 不同：

  • Type A · zero hit：检索 score 全低于阈值，recall gap → 补 doc 或改分块
  • Type B · ACL filter：有 doc 但 filter 后 0 条 → 修 ACL 或用户权限
  • Type C · LLM 拒答：有 chunk 但模型判定不足以回答 → 改 prompt 或补摘要元数据
混为一谈
  • 空答率升高就加 chunk
  • 不记录 filter 后 hit_count
  • 无 Type 标签的聚合 KPI
分型治理
  • 日志带 empty_type 枚举
  • Type B 联动 权限章 审计
  • Type A 进 top query 补库 backlog

空答日志字段

yaml 
rag_log:
  query_id: q-8821
  intent: expense_reimburse
  hit_count_before_filter: 12
  hit_count_after_filter: 0
  empty_type: ACL_FILTER
  max_score: 0.71
  tenant_id: corp-a
  acl_filter: "tenant_id == corp-a and ..."

热点

hot doc 与 top query 驱动运营优先级：高频 query 对应 doc 陈旧或缺失时，business impact 最大。

热点反哺闭环
flowchart LR
  Logs[RAG 日志] --> Agg[Top query 聚合]
  Agg --> Gap{recall gap?}
  Gap -->|是| Ingest[优先 ingest]
  Gap -->|否 stale| Quality[质量审核]
  Agg --> Hot[Hot doc 排行]

热点 doc 同时进入 kb dashboard 与 review queue 优先级加权。

本章结论

  • 覆盖率：分层抽样 + intent 维度；与 chunk 总量脱钩。
  • 空答：zero hit / ACL / LLM 三分型；日志带 filter 前后 hit_count。
  • 热点：Top query 与 hot doc 驱动 ingest 与质量优先级。
