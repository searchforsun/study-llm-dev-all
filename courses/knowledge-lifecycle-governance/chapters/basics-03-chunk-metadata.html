为什么要学本章

ACL 泄露、检索无法按部门过滤，根因常是入库时没写 metadata。统一 chunk spec 是双栈共用契约。

学完你能
  • 设计 corpassist schema 必填字段
  • 配置 acl tags 与 doc type tag
  • 保证 Python 写入与 Spring filter 表达式字段一致
本章先记住 3 件事
  • chunk_id 稳定可复现，便于增量 upsert。
  • acl_tags 在入库时写入，检索层 filter，非事后过滤。
  • metadata norm 版本化，变更需 re-embed 或迁移计划。

规范

metadata norm 规定字段类型、枚举与必填项：

字段类型用途
chunk idstring主键；格式 {doc_id}#{page}#{seq}
index_versionstring蓝绿发布（索引版本与回滚）
tenant_idstring多租户隔离
updated_atISO8601stale 检测

分块策略对比

不推荐
  • 固定 512 token 切标题中间

标签体系

tag system 支持检索过滤与运营统计：

  • 业务域：domain:hr、domain:finance
  • 密级：classification:internal
  • ACL：acl:dept-hr、acl:mgmt-only
标签 → Milvus filter
flowchart LR
  META[chunk metadata] --> FILTER["filter: acl in user.acl"]
  FILTER --> HITS[合法 hits]
  META --> STATS[运营聚合 by domain]
先想 10 秒：为何 acl_tags 必须在入库时写入，而不是检索后打标？

检索后打标意味着无权限 chunk 已进入候选集，存在日志与缓存泄露风险；filter 依赖入库时的标量字段索引。

CorpAssist schema

c7 metadata 双栈共享 JSON Schema：

json
{
  "$schema": "https://corpassist/schema/chunk/v1.2",
  "required": ["chunk_id", "tenant_id", "acl_tags", "source_uri", "index_version"],
  "properties": {
    "chunk_id": { "type": "string" },
    "acl_tags": { "type": "array", "items": { "type": "string" } },
    "doc_type": { "enum": ["policy", "faq", "procedure"] },
    "embedding_model": { "type": "string" }
  }
}

Spring VectorStore filter 示例

java
// 检索层注入 ACL — 非 LLM 事后过滤
SearchRequest request = SearchRequest.query(userQuery)
  .withTopK(8)
  .withFilterExpression(
    "tenant_id == '" + tenantId + "' && acl_tags IN " + userAclJson);
