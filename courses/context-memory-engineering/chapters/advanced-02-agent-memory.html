为什么要学本章

Agent 场景对记忆有特殊要求：长任务需要 Checkpoint 支持中断恢复；多 Agent 协作需要共享黑板（shared blackboard）；每个 Agent 的 mem scope 需要差异化管理。本章将 CorpAssist agent memory 扩展到 Agent 场景。

本章先记住 3 件事

  • Thread ID 对齐：Agent thread_id 必须与 session_id 一致才能共享会话。

LangGraph checkpoint：Agent 状态持久化

langgraph checkpoint 是 LangGraph 的 Agent 状态持久化机制。checkpoint store 存储到 PostgreSQL 或 Redis。

Agent Checkpoint 生命周期
sequenceDiagram
    participant Agent as CorpAssist Agent
    participant Graph as LangGraph
    participant Store as Checkpoint Store

    Agent->>Graph: 开始任务 (thread_id)
    Note over Graph: Step 1: 查询订单
    Graph->>Store: 保存 Checkpoint 1
    Note over Graph: Step 2: 查询库存
    Graph->>Store: 保存 Checkpoint 2
    Note over Graph: Step 3: 生成回复 (中断)
    Graph->>Store: 保存 Checkpoint 3

    Agent-->>Graph: 恢复任务 (thread_id)
    Graph->>Store: 加载 Checkpoint 3
    Note over Graph: Step 4: 继续执行
    Graph->>Store: 保存 Checkpoint 4
    Graph-->>Agent: 任务完成

Agent 任务在每一步后保存 Checkpoint，中断后从最新 Checkpoint 恢复。

python 
# LangGraph Checkpoint - Agent 状态持久化
from langgraph.checkpoint.redis import RedisSaver
from langgraph.graph import StateGraph

# 定义 Agent 状态
class AgentState(TypedDict):
    messages: list
    order_context: dict
    user_id: str
    step: int

# 配置 Checkpointer
checkpointer = RedisSaver(
    url="redis://localhost:6379",
    ttl=86400  # 24h
)

# 构建 Agent Graph
graph = StateGraph(AgentState)
# ... 添加节点和边 ...

# 编译时绑定 Checkpointer
agent = graph.compile(checkpointer=checkpointer)

# 运行 Agent，thread_id 与 session_id 对齐
result = agent.invoke(
    {"messages": [user_msg], "user_id": user_id},
    config={"configurable": {"thread_id": f"thread:{session_id}"}}
)

# 恢复 Agent
result = agent.invoke(
    None,  # 继续之前的输入
    config={"configurable": {"thread_id": f"thread:{session_id}"}}
)
CorpAssist · Agent 客服场景 用户提交了一个复杂工单：「查询订单 20240501 状态，如果已发货则预约改地址，同时生成一个投诉工单」。这个任务需要 5-8 步 Agent 操作。用户在第 3 步时关闭了页面。20 分钟后重新打开，Agent 从最后一个 Checkpoint 恢复，继续完成后续操作，用户无需重复之前的对话。

共享黑板（shared blackboard） + 私有记忆分离：协作信息共享，执行细节隔离。

python 
# 共享黑板（shared blackboard）实现
from typing import Any
import json

class SharedBlackboard:
    """Redis 实现的共享黑板（shared blackboard）"""
    def __init__(self, redis_client, session_id: str):
        self.redis = redis_client
        self.key = f"corpassist:blackboard:{session_id}"

    def write(self, key: str, value: Any, agent: str):
        data = json.dumps({"value": value, "writer": agent, "scope": "shared"})
        self.redis.hset(self.key, key, data)
        self.redis.expire(self.key, 86400)

    def read(self, key: str) -> Any:
        data = self.redis.hget(self.key, key)
        if data:
            return json.loads(data)["value"]
        return None

    def get_all(self) -> dict:
        result = {}
        for k, v in self.redis.hgetall(self.key).items():
            parsed = json.loads(v)
            result[k.decode()] = parsed["value"]
        return result


class PrivateMemory:
    """Agent 私有记忆"""
    def __init__(self, redis_client, agent_id: str, session_id: str):
        self.key = f"corpassist:private:{agent_id}:{session_id}"
        self.redis = redis_client

    def write_log(self, step: int, log: str):
        self.redis.rpush(self.key, f"[Step {step}] {log}")
        self.redis.expire(self.key, 86400)

Java 实现：多 Agent 共享

java 
// 多 Agent 共享记忆 - Java 实现
@Service
public class AgentMemoryService {

    private final RedisTemplate<String, Object> redis;

    public AgentMemoryService(RedisTemplate<String, Object> redis) {
        this.redis = redis;
    }

    public void writeToBlackboard(String sessionId, String key,
                                   Object value, String agent) {
        String redisKey = "corpassist:blackboard:" + sessionId;
        Map<String, Object> data = Map.of(
            "value", value, "writer", agent, "scope", "shared"
        );
        redis.opsForHash().put(redisKey, key, data);
        redis.expire(redisKey, 24, TimeUnit.HOURS);
    }

    public Object readFromBlackboard(String sessionId, String key) {
        String redisKey = "corpassist:blackboard:" + sessionId;
        Map<Object, Object> data = redis.opsForHash().entries(redisKey);
        return data.get(key);
    }
}

Thread/Scope 管理与 Agent 恢复

thread state 管理 Agent 的执行线程。resume agent 从 Checkpoint 恢复 Agent 执行。

概念说明与 session 的关系
ThreadAgent 的一次执行实例thread_id 与 session_id 对齐（如 thread:{session_id}）
Checkpoint某一步的状态快照按 thread_id 索引
Blackboard多 Agent 共享上下文按 session_id 索引
Scope记忆可见范围global/agent/step 三级
不推荐
  • 无 Checkpoint，Agent 重跑全部步骤
  • 所有 Agent 共享全部记忆，无隔离
  • thread_id 随意生成，无法关联会话
  • Agent 失败后无恢复路径
推荐
  • 每步后保存 Checkpoint，支持恢复
  • 共享黑板（shared blackboard） + 私有记忆分离
  • thread_id 与 session_id 对齐，step 存于 checkpoint 元数据
  • 注册 Resume Handler 处理恢复
检查理解：thread_id 为什么需要对齐 session_id？

当用户从 Agent 切回普通对话，或从一个 Agent 切换到另一个 Agent，session_id 保证用户上下文不丢失。thread_id 与 session_id 对齐后，LangGraph 按 thread 恢复；具体 step 由 checkpoint 元数据记录。

常见问题（FAQ）

Checkpoint 存储在哪里？性能如何？
LangGraph 支持 Redis 和 PostgreSQL。Redis 适合高性能场景（< 10ms 读写），PostgreSQL 适合需要复杂查询的场景。推荐 Redis 用于 Checkpoint，PostgreSQL 用于历史记录。
多个 Agent 同时写入黑板（shared blackboard）会冲突吗？
黑板（shared blackboard）使用 Redis Hash，每个字段独立操作。不同 Agent 写不同 key 不会冲突。如果写同一 key，最后写入者获胜。需要事务性写入时使用 Redis Watch 或 Lua 脚本。

本章结论

  • LangGraph Checkpoint：Agent 每步后保存状态，支持中断恢复，通过 RedisSaver 持久化。
  • 共享黑板（shared blackboard）：Redis 实现的共享上下文，多 Agent 可通过黑板（shared blackboard）交换信息。
  • 私有记忆：每个 Agent 维护私有执行日志和中间状态，不共享。
  • Thread ID 对齐：thread_id 与 session_id 对齐，checkpoint 元数据记录 step，保证跨 Agent 会话连续。
  • Scope 管理：记忆分为 global/session/agent 三级可见范围。
