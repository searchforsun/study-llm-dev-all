SAA 上下文压缩与编辑

saa compress edit 提供了两种核心能力：上下文压缩和记忆编辑。通过 saa advisor 实现自动注入。

SAA Advisor 链：请求经过压缩→编辑→安全三道处理后才到达 LLM。

java 
// SAA 上下文压缩与编辑 Advisor 配置
import com.alibaba.cloud.ai.advisor.ContextCompressAdvisor;
import com.alibaba.cloud.ai.advisor.ContextEditingAdvisor;

@Configuration
public class SaaAdvisorConfig {

    @Bean
    public ContextCompressAdvisor compressAdvisor() {
        return ContextCompressAdvisor.builder()
            .withCompressStrategy("summary")       // 摘要压缩
            .withTokenThreshold(0.8f)               // 80% 触发
            .withMaxTokens(32000)
            .withKeepSystemPrompt(true)
            .withProtectedKeys(List.of(             // 保护关键字段
                "order_id", "user_id", "membership"
            ))
            .build();
    }

    @Bean
    public ContextEditingAdvisor editingAdvisor() {
        return ContextEditingAdvisor.builder()
            .withEditStrategies(List.of(
                new TruncateLongFieldStrategy(500),  // 字段截断
                new RemoveDuplicatedContent(),       // 去重
                new RedactPIIContent()               // PII 脱敏
            ))
            .build();
    }
}

CorpAssist 集成 SAA

java 
// CorpAssist 集成 SAA Advisor
@Service
public class CorpAssistSaaService {

    private final ChatClient chatClient;

    public CorpAssistSaaService(ChatClient.Builder builder,
                                 ContextCompressAdvisor compress,
                                 ContextEditingAdvisor edit) {
        this.chatClient = builder
            .defaultAdvisors(compress, edit)
            .build();
    }

    public String chat(String tenant, String userId, String message) {
        return chatClient.prompt()
            .user(message)
            .advisors(a -> a.param("tenant", tenant)
                            .param("user_id", userId))
            .call()
            .content();
    }
}
CorpAssist · SAA 集成场景 CorpAssist 客服系统使用 SAA 的 ContextCompressAdvisor 自动管理上下文窗口。当历史对话超过 80% 预算时，Advisor 自动执行摘要压缩。同时 ContextEditingAdvisor 截断过长的工具输出字段，防止单个字段撑爆窗口。

上下文限制与安全边界

context limit 定义了上下文的硬性边界。edit memory 提供了上下文内容的编辑替换能力。

yaml 
# SAA 上下文限制配置
spring:
  ai:
    alibaba:
      context:
        # 上下文限制
        limits:
          max-tokens: 32000
          max-history-turns: 20
          max-retrieval-chunks: 8
          max-tool-output-length: 2000

        # 编辑策略
        editing:
          truncate-long-fields: true
          max-field-length: 500
          remove-duplicates: true
          redact-pii: true
          pii-patterns:
            - "\\d{17}[\\dXx]"    # 身份证
            - "1[3-9]\\d{9}"       # 手机号

        # 审计
        audit:
          enabled: true
          log-all-edits: true
不推荐
  • JVM 内存无限增长，无上下文限制
  • 工具输出不截断，原文全量注入
  • 无 PII 脱敏，敏感信息进入 LLM
  • Advisor 链顺序错误，先安全检查再压缩
推荐
  • SAA context limits 配置硬性边界
  • 工具输出截断 + PII 脱敏
  • 先压缩后安全，顺序合理
  • 审计日志记录所有编辑操作

Graph 状态管理

graph state 是 SAA 的对话状态管理机制。state reducer 定义了状态如何合并更新。

Graph 状态与记忆分离
sequenceDiagram
    participant User as 用户
    participant Graph as StateGraph
    participant Mem as 记忆系统
    participant LLM as LLM

    User->>Graph: 输入消息
    Graph->>Mem: 加载状态
    Mem-->>Graph: 会话/长期记忆
    Graph->>Graph: State Reducer 合并
    Graph->>LLM: 组装后的上下文
    LLM-->>Graph: 生成回复
    Graph->>Mem: 更新状态
    Mem->>Mem: Checkpoint 保存
    Graph-->>User: 返回结果

Graph 状态与记忆分离：状态使用 Reducer 管理更新，记忆通过 Checkpoint 持久化。

java 
// SAA Graph 状态管理
import com.alibaba.cloud.ai.graph.StateGraph;
import com.alibaba.cloud.ai.graph.state.AgentState;
import com.alibaba.cloud.ai.graph.state.StateReducer;

// 定义对话状态
public class ConversationState extends AgentState {
    public ConversationState(Map<String, Object> initData) {
        super(initData);
    }

    public String getUserId() { return get("user_id").toString(); }
    public List<String> getMessages() { return get("messages"); }
    public Map<String, Object> getMemory() { return get("memory"); }
}

// 构建 Graph
StateGraph<ConversationState> graph = new StateGraph<>(
    ConversationState::new,
    StateReducer.builder()
        .addReducer("messages", StateReducer::concatAndTrim)  // 消息追加+截断
        .addReducer("memory", StateReducer::lastWriteWins)    // 记忆最后写入者获胜
        .addReducer("user_id", StateReducer::keepOriginal)    // 用户 ID 保留原始值
        .build()
);

graph checkpoint 支持将状态持久化到外部存储，实现对话中断后恢复。

java 
// Graph Checkpoint 持久化
graph.setCheckpointStore(new RedisCheckpointStore(redisConnection));
// thread_id 与 session_id 对齐
graph.execute(conversationState, "thread:" + sessionId);
检查理解：State Reducer 和记忆系统的区别？

State Reducer 管理 Graph 执行的当前状态（如当前轮次的消息列表），是短暂的。记忆系统管理用户的长期知识（如用户偏好、事实），是持久的。两者分离，各司其职。

常见问题（FAQ）

SAA Advisor 和 LangGraph 有什么异同？
SAA Advisor 是 Spring 生态的拦截器模式，适合 Java 单体应用。LangGraph 是 Python 生态的图执行引擎，适合复杂 Agent 编排。两者在概念上类似但生态不同。
Graph Checkpoint 和 LangGraph Checkpoint 如何对应？
两者概念一致：都是持久化对话状态以支持中断恢复。SAA 使用 CheckpointStore 接口，LangGraph 使用 Checkpointer 类。thread_id 和 session_id 应对齐。

本章结论

  • SAA 压缩/编辑：通过 ContextCompressAdvisor 和 ContextEditingAdvisor 自动管理上下文。
  • Context Limits：硬性边界配置，防止意外爆窗和数据泄漏。
  • Graph 状态：State Reducer 管理状态合并，与记忆系统分离。
  • Checkpoint：状态持久化到 Redis，支持中断恢复。
  • Java 生态优势：SAA 将上下文工程能力以 Spring 风格注入，降低集成成本。
