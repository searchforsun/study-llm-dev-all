为什么要学本章

LLM 自动提取的事实可能出错。错误记忆如果不加纠正，会在每次召回中持续误导 LLM。必须引入人工审核（human loop）机制：人工审批重要事实的写入、纠正错误事实、记录所有变更。

学完你能

  • 实现 memory edit 编辑功能
  • 实现 audit trail mem 审计日志
  • 设计 override 覆盖机制
本章先记住 3 件事
  • 不是所有记忆都需要审批：低置信度的写入需要人工审核，高置信度的自动批准。
  • 审计日志不可篡改：所有记忆变更记录 must be append-only。
  • 纠正需要版本信息：每次编辑记录 before/after，方便回滚。

审批流程：Human Approval for Memory Writes

human approval 确保重要的事实写入前经过人工审核。基于置信度阈值设计审批规则。

记忆写入审批流程
flowchart TB
    Start[事实提取完成] --> Check{置信度 >= 0.9?}
    Check -->|是| Auto[自动写入长期记忆]
    Check -->|否| Review[进入人工审批队列]

    Review --> Human{人工审核}
    Human -->|批准| Write[写入长期记忆]
    Human -->|拒绝| Discard[丢弃该事实]
    Human -->|修改| Edit[编辑后写入]

    Auto --> Log[记录审计日志]
    Write --> Log
    Discard --> Log
    Edit --> Log

# 审批队列实现
from enum import Enum
from datetime import datetime

class ApprovalStatus(Enum):
    PENDING = "pending"
    APPROVED = "approved"
    REJECTED = "rejected"
    EDITED = "edited"

class MemoryApprovalQueue:
    def __init__(self):
        self.pending_reviews = []

    def submit_for_review(self, fact: MemoryFact):
        self.pending_reviews.append({
            "fact": fact,
            "status": ApprovalStatus.PENDING,
            "submitted_at": datetime.now(),
            "reviewed_by": None,
            "review_notes": ""
        })

    def approve(self, fact_id: str, reviewer: str):
        entry = self._find(fact_id)
        entry["status"] = ApprovalStatus.APPROVED
        entry["reviewed_by"] = reviewer
        self._write_to_long_term(entry["fact"])

    def reject(self, fact_id: str, reviewer: str, reason: str):
        entry = self._find(fact_id)
        entry["status"] = ApprovalStatus.REJECTED
        entry["reviewed_by"] = reviewer
        entry["review_notes"] = reason

    def edit_and_approve(self, fact_id: str, reviewer: str,
                         edited_content: str):
        entry = self._find(fact_id)
        old_fact = entry["fact"]
        entry["fact"].content = edited_content
        entry["status"] = ApprovalStatus.EDITED
        entry["reviewed_by"] = reviewer
        self._write_to_long_term(entry["fact"])
        self._audit_change(old_fact, entry["fact"], reviewer)

记忆编辑：纠正错误与版本管理

correct fact 允许人工修复错误记忆。memory edit 记录了修改历史。

python 
# 记忆编辑与版本管理
class MemoryEditService:
    def edit_fact(self, fact_id: str, new_content: str,
                  editor: str, reason: str) -> dict:
        old = self._get_fact(fact_id)

        # 创建新版本
        new_version = {
            "fact_id": fact_id,
            "version": old["version"] + 1,
            "content": new_content,
            "changed_by": editor,
            "changed_at": datetime.now(),
            "reason": reason
        }

        # 覆盖当前事实
        self._update_fact(fact_id, new_version)

        # 记录审计
        self._audit_entry({
            "event": "fact_edited",
            "fact_id": fact_id,
            "before": old["content"],
            "after": new_content,
            "editor": editor,
            "reason": reason
        })

        return {"success": True, "new_version": new_version["version"]}

    def rollback_fact(self, fact_id: str, target_version: int,
                      editor: str):
        """回滚到指定版本"""
        version = self._get_version(fact_id, target_version)
        self._update_fact(fact_id, version)
        self._audit_entry({
            "event": "fact_rollback",
            "fact_id": fact_id,
            "rolled_back_to": target_version,
            "editor": editor
        })

java 
// 记忆编辑 - Java 实现
@Service
public class MemoryEditService {
    private final FactRepository factRepo;
    private final AuditLogRepository auditRepo;

    @Transactional
    public FactVersion editFact(String factId, String newContent,
                                 String editor, String reason) {
        MemoryFact old = factRepo.findById(factId)
            .orElseThrow(() -> new FactNotFoundException(factId));

        FactVersion newVersion = new FactVersion(
            factId, old.getVersion() + 1, newContent,
            editor, reason, Instant.now()
        );

        factRepo.updateContent(factId, newContent, newVersion.getVersion());

        auditRepo.save(new AuditEntry(
            "fact_edited", factId,
            old.getContent(), newContent, editor, reason
        ));

        return newVersion;
    }
}

审计日志：不可篡改的变更记录

audit trail mem 记录了所有记忆的创建、审批、编辑和删除操作。审计日志是 append-only 的，不可篡改。

审计事件触发条件记录内容
fact_createdLLM 提取新事实事实内容、置信度、来源轮次
fact_approved人工批准写入审批人、审批时间、备注
fact_rejected人工拒绝写入审批人、拒绝原因
fact_edited人工编辑事实修改前内容、修改后内容、编辑器、原因
fact_deletedGDPR 删除或手动删除删除者、删除范围
python 
# 审计日志存储 - append-only
class AuditLogStore:
    def __init__(self, db):
        self.db = db
        self.db.execute("""
            CREATE TABLE IF NOT EXISTS memory_audit_log (
                id INTEGER PRIMARY KEY AUTOINCREMENT,
                event_type TEXT NOT NULL,
                fact_id TEXT NOT NULL,
                before_content TEXT,
                after_content TEXT,
                operator TEXT NOT NULL,
                reason TEXT,
                created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
            )
        """)

    def append(self, entry: dict):
        # append-only，禁止 DELETE 和 UPDATE
        self.db.execute("""
            INSERT INTO memory_audit_log
            (event_type, fact_id, before_content, after_content,
             operator, reason)
            VALUES (?, ?, ?, ?, ?, ?)
        """, (
            entry["event"], entry["fact_id"],
            entry.get("before"), entry.get("after"),
            entry["operator"], entry.get("reason")
        ))

    def query_by_fact(self, fact_id: str) -> list:
        """查询某个事实的所有变更历史"""
        return self.db.execute(
            "SELECT * FROM memory_audit_log WHERE fact_id = ? ORDER BY id",
            (fact_id,)
        ).fetchall()
不推荐
  • LLM 提取的事实直接写入，无需审核
  • 记忆可被任意覆盖，无版本记录
  • 审计日志可被删除或修改
  • 无编辑人记录，无法追责
推荐
  • 置信度 < 0.9 的事实需人工审核
  • 每次编辑保留 before/after 版本
  • 审计日志 append-only，不可修改
  • 记录操作人、时间和原因
检查理解：审计日志为什么必须是 append-only？

如果审计日志可以被修改，就失去了审计的意义。恶意操作者可以删除自己的操作痕迹。Append-only 保证一旦写入，永久存在。

常见问题（FAQ）

审核队列太长怎么办？
可动态调整置信度阈值。如果审核资源充裕，降低到 0.8；如果积压严重，提高到 0.95。同时提供批量审核功能。
如何找到需要审核的待办项？
提供一个 Dashboard 接口，按时间排序展示 PENDING 状态的待审批事实。支持按用户、置信度范围过滤。

本章结论

  • 审批流程：置信度 >= 0.9 自动写入，否则人工审核。降低错误记忆风险。
  • 记忆编辑：支持编辑和回滚，每次变更保留版本信息。
  • 审计日志：Append-only 的变更记录，不可篡改，支持按事实查询全量历史。
  • Override 覆盖：人工编辑的优先级高于自动提取，确保最终控制权在人手中。
  • Human Loop：人在回路不是妥协，而是对 LLM 记忆质量的必要保障。
