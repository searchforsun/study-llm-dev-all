# Demo：记忆的成本与延迟

[返回章节](../../index.html#ch-advanced-03-cost-latency)

## 目标

分析记忆系统的读写放大问题，实现会话内事实缓存和懒加载策略，评估缓存命中率对 P95 延迟的改善。

## 前置准备

- Python 3.8+
- Redis 实例

## 步骤

1. **实现 LatencyMonitor**：记录每次记忆操作的延迟，按操作类型统计读写放大倍数。输出类似「1 次请求 = 5 次后端操作」的分析报告。

2. **实现 SessionFactCache**：会话内缓存已召回的长期记忆，TTL 5 分钟。缓存命中时跳过向量搜索。

3. **实现 LazyMemoryLoader**：添加判断逻辑，简单问候语跳过长期记忆召回，操作意图才触发召回。

4. **对比实验**：分别测量无缓存、有缓存（TTL 5min）、懒加载三种场景下的 P95 延迟。输出对比表和成本分析。

## 预期输出

缓存命中后 P95 延迟降低 ≥ 50%。懒加载减少 40% 不必要召回。成本分析报告量化优化效果。

## 验收清单

- [ ] 读写放大倍数可测量
- [ ] 缓存命中率 ≥ 60%
- [ ] P95 延迟降低 ≥ 50%
- [ ] 成本分析报告量化
