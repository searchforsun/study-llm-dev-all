循环检测

检测相同 tool+args 重复 N 次、或 messages 哈希不变。触发后强制 END 或 handoff。

python
def guard_node(state):
    sig = last_tool_signature(state)
    if state.get("repeat_count", 0) >= 2:
        return {"route": "handoff", "reason": "loop_detected"}
    return {}

预算

Step budget 与 token budget 双限；state.steps 每 super-step +1，超限走降级。

场景max steps说明
S2 客服8多数 2–4 步
S3 办公15需中间 reflection 节点

fallback

Tool failure：retryable 限 2 次 → 小模型摘要错误 → handoff。配置见 failure-policy.yaml。

降级链
flowchart TD
  F[工具失败] --> R{retryable?}
  R -->|是| RT[重试≤2]
  R -->|否| FB[fallback 话术]
  RT --> L{仍失败?}
  L -->|是| HO[handoff_human]

本章结论

  • 循环检测：重复签名/无进展即停。
  • 预算：步数+Token 硬上限。
  • fallback：重试→话术→转人工。
