Interrupt/Resume 机制

LangGraph 的 human in loop 通过 interrupt_before 参数实现：在执行到指定节点前暂停，保存当前状态，等待外部输入后从断点继续。

人机协作流程图
sequenceDiagram
    participant Agent as Agent
    participant Human as 审批人
    participant System as 系统

    Agent->>Agent: 思考：需要执行 refund 工具
    Agent->>System: interrupt_before("tools")
    System->>Human: 发送审批请求\n[工具: refund, 参数: T001]
    Human-->>System: 审批通过
    System->>Agent: resume() 继续执行
    Agent->>Agent: 执行 refund 工具
    Agent-->>Human: 退款成功

Python：Interrupt/Resume 实现

python 
from langgraph.graph import StateGraph, END
from langgraph.checkpoint import MemorySaver

# 定义需要审批的工具
APPROVAL_REQUIRED_TOOLS = ["refund_order", "delete_order", "transfer_money"]

def tool_node_with_approval(state: AgentState) -> AgentState:
    """需要审批的工具节点"""
    last_message = state["messages"][-1]
    for tool_call in last_message.tool_calls:
        tool_name = tool_call["name"]
        if tool_name in APPROVAL_REQUIRED_TOOLS:
            # 保存审批请求信息到 state
            state["pending_approval"] = {
                "tool": tool_name,
                "args": tool_call["args"],
                "status": "pending"
            }
            # 触发 interrupt_before 会在此处暂停
            # 等待人工审批
        result = execute_tool(tool_call)
        state["messages"].append({"role": "tool", "content": str(result)})
    return state

# 构建图，设置中断
graph = StateGraph(AgentState)
graph.add_node("agent", agent_node)
graph.add_node("tools", tool_node_with_approval)
graph.set_entry_point("agent")
graph.add_conditional_edges(
    "agent",
    should_continue,
    {"tools": "tools", "end": END}
)
graph.add_edge("tools", "agent")

# 编译时指定中断位置
app = graph.compile(
    checkpointer=MemorySaver(),
    interrupt_before=["tools"]  # 在 tools 节点前中断
)

# 执行
config = {"configurable": {"thread_id": "session_001"}}
# 第一次 invoke：Agent 思考后会中断在 tools 前
result = app.invoke({"messages": [user_msg]}, config)

# 查看中断状态
pending = result.get("pending_approval")
if pending:
    print(f"等待审批：{pending['tool']} - {pending['args']}")
    # 人工审批通过后
    app.update_state(config, {"pending_approval": {"status": "approved"}})
    # 恢复执行
    result = app.invoke(None, config)

审计追踪

每一次人机交互都需要完整记录，用于合规审查和问题追溯。

python 
class AuditLogger:
    """审计日志记录器"""

    def log_interrupt(self, session_id: str, node: str, state: dict):
        """记录中断事件"""
        audit_entry = {
            "event_type": "interrupt",
            "session_id": session_id,
            "node": node,
            "state_snapshot": state,
            "timestamp": datetime.utcnow().isoformat()
        }
        self._write(audit_entry)

    def log_resume(self, session_id: str, decision: str, reviewer: str):
        """记录恢复事件"""
        audit_entry = {
            "event_type": "resume",
            "session_id": session_id,
            "decision": decision,  # approved / rejected
            "reviewer": reviewer,
            "timestamp": datetime.utcnow().isoformat()
        }
        self._write(audit_entry)

    def _write(self, entry: dict):
        # 写入数据库或日志系统

人工转接策略

当 Agent 无法处理用户请求时，需要智能地转接人工。以下策略决定何时转接：

转人工决策树
flowchart TD
    Q1[Agent 是否能理解用户意图？] -->|否| Transfer[转人工]
    Q1 -->|是| Q2[是否在已有工具能力范围内？]
    Q2 -->|否| Transfer
    Q2 -->|是| Q3[工具调用是否连续失败？]
    Q3 -->|是| Transfer
    Q3 -->|否| Q4[用户是否要求转人工？]
    Q4 -->|是| Transfer
    Q4 -->|否| Continue[Agent 继续处理]

转人工决策树确保 Agent 在能力不足时及时升级，避免让用户等待过久。

不推荐
  • Agent 耗尽步数后直接报错
  • 所有操作都要求人工确认
  • 无审计日志，无法追溯决策过程
推荐
  • Agent 无法处理时优雅转人工
  • 仅高危操作需人工确认
  • 全量审计日志可追溯

常见问题（FAQ）

中断后 Agent 的状态会丢失吗？
不会。LangGraph 的 Checkpoint 机制在中断时自动保存完整状态。恢复时从 Checkpoint 加载状态，从断点继续执行。即使进程重启，只要使用持久化 Checkpoint（如 SqliteSaver、RedisSaver），状态也不会丢失。
审批超时后应该怎么处理？
超时策略取决于业务场景：① 查询类操作可以自动超时通过（返回已存在的数据）；② 资金类操作应该超时拒绝并通知用户；③ 关键操作可以升级到更高审批层级。建议在审批请求中明确标注超时策略。
转人工时如何保证上下文完整传递？
转人工时需要将完整对话历史、Agent 推理轨迹、已调用的工具结果打包传递给人工客服。可以序列化为 JSON 格式，在客服系统中展示 Agent 的决策链。建议包含 Agent 的思考过程，帮助客服快速理解背景。

本章结论

  • Interrupt/Resume：LangGraph 原生支持，在高危操作前中断等待审批。
  • 审批分级：L1 自动、L2 快速、L3 高级，根据操作风险级别设置。
  • 审计追踪：完整记录每次中断和恢复操作，用于合规审查。
  • 转人工策略：决策树决定何时转接，保证上下文完整传递。
  • Checkpoint：中断时保存完整状态，恢复时不丢失上下文。
