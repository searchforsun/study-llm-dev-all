为什么要学本章

Agent 的「行动」层就是工具。schema 模糊会导致模型传错参；错误返回过长会淹没上下文；未校验的返回值会被模型误读为成功。本章是 NS2 的落地接口。

学完你能
  • 用 Pydantic 定义 structured tool 并写清 description
  • 设计 tool error 返回格式，避免 Agent 死循环重试
  • 对照 SAA ToolCallback 说明双栈工具契约一致性的要点
本章先记住 3 件事
  • Description 即文档：模型靠描述选工具；写「何时用/何时不用」。
  • Args 先校验：arg validation 在 Pydantic 层失败，不要把脏参数打到业务 API。
  • Return 要短：tool return 截断 + 结构化，便于 Observation 进 state。

定义

CorpAssist S2 · 订单工具集 客服 Agent 仅允许：get_order、create_ticket、handoff_human。每个工具一个文件/模块，禁止「万能 search_db」。

LangChain tool 把 Python 函数包装成模型可调的 Action。生产上工具是白名单，不是「模型想到什么写什么」。

最小工具示例

python
from langchain_core.tools import tool
from pydantic import BaseModel, Field

class GetOrderInput(BaseModel):
    order_id: str = Field(..., description="纯数字订单号，不含 # 前缀")

@tool("get_order", args_schema=GetOrderInput)
def get_order(order_id: str) -> str:
    """查询订单物流与状态。用户问「到哪了/进度」时使用；不用于改地址或退款。"""
    data = order_api.fetch(order_id)  # 真实 HTTP
    return truncate_json(data, max_chars=1200)

schema

tool schema 来自 Pydantic/JSON Schema，决定模型如何填参。schema 与业务 API 契约同源（可从 OpenAPI 生成）。

反例 vs 推荐：参数描述

不推荐
python
class BadInput(BaseModel):
    id: str  # 无 description，模型常传「#8821」含符号
推荐
python
class GoodInput(BaseModel):
    order_id: str = Field(
        pattern=r"^\d{6,12}$",
        description="6-12 位数字订单号；去掉用户输入的 # 前缀",
    )

注册到 LangGraph 节点

python
TOOLS = [get_order, create_ticket, handoff_human]
tool_node = ToolNode(TOOLS)  # 预绑定，运行时不可动态增删

错误

工具失败是常态：超时、4xx、下游维护。Observation 里必须让模型知道失败原因且可行动（重试/换工具/转人工），而不是抛未捕获栈追踪。

推荐错误 envelope

json
{
  "ok": false,
  "error_code": "ORDER_NOT_FOUND",
  "message": "订单 8821 不存在",
  "retryable": false,
  "suggested_action": "ask_user_confirm_order_id"
}
错误类型retryableAgent 行为
超时 / 503true有限重试 → 仍失败则降级
404 订单不存在false澄清 order_id，禁止盲重试
403 无权限false拒答或转人工
工具失败 → Agent 分支
flowchart TD
  T[Tool 调用] --> OK{ok?}
  OK -->|是| OBS[写入 Observation]
  OK -->|否| R{retryable?}
  R -->|是| RET[重试计数+1]
  R -->|否| ASK[澄清/转人工]
  RET --> LIM{超限?}
  LIM -->|是| HO[handoff_human]

本章结论

  • 定义：工具窄接口 + 白名单；description 写清使用边界。
  • schema：Pydantic 校验与 OpenAPI 对齐，减少脏参数。
  • 错误：结构化 error envelope，区分可重试与应澄清/降级。
