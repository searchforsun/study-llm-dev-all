本章先记住 3 件事
  • 节点 = 处理单元：agent_node（思考）和 tool_node（行动）是两个核心节点。
  • 条件边 = 路由器：根据模型是否调用工具决定走工具节点还是直接结束。
  • 状态循环：LangGraph 在节点间传递 State 对象，每个节点读取和更新状态。

LangGraph 中的 ReAct 实现

LangGraph 的 StateGraph 是实现 ReAct 循环的容器。它定义了三个要素：State（在节点间传递的数据结构）、Node（处理逻辑的单元）、Edge（控制节点间的流转路径）。

LangGraph ReAct 状态图
flowchart LR
    START([开始]) --> AgentNode[Agent 节点\n思考+推理]
    AgentNode -->|有工具调用| ToolNode[工具节点\n执行工具]
    AgentNode -->|无工具调用| END([结束])
    ToolNode --> AgentNode

ReAct 循环在 Agent 节点和 Tool 节点之间交替，直到 Agent 决定直接回答。

Python：StateGraph ReAct 完整实现

python 
from typing import TypedDict, Literal, Sequence
from langgraph.graph import StateGraph, END
from langgraph.prebuilt import ToolExecutor
from langchain_core.messages import BaseMessage, HumanMessage, AIMessage
import json

class AgentState(TypedDict):
    messages: Sequence[BaseMessage]
    step: int
    max_steps: int

def agent_node(state: AgentState, llm, tools) -> AgentState:
    messages = state["messages"]
    response = llm.invoke(messages, tools=tools)
    state["messages"].append(response)
    state["step"] = state.get("step", 0) + 1
    return state

def tool_node(state: AgentState, tool_executor: ToolExecutor) -> AgentState:
    last_message = state["messages"][-1]
    for tool_call in last_message.tool_calls:
        try:
            result = tool_executor.invoke(tool_call)
            state["messages"].append({
                "role": "tool", "content": str(result),
                "tool_call_id": tool_call["id"]
            })
        except Exception as e:
            state["messages"].append({
                "role": "tool", "content": f"工具调用失败：{str(e)}",
                "tool_call_id": tool_call["id"]
            })
    return state

def should_continue(state: AgentState) -> Literal["tools", "end"]:
    if state.get("step", 0) >= state.get("max_steps", 15):
        return "end"
    last_message = state["messages"][-1]
    if hasattr(last_message, "tool_calls") and last_message.tool_calls:
        return "tools"
    return "end"

def build_react_agent(llm, tools):
    tool_executor = ToolExecutor(tools)
    graph = StateGraph(AgentState)
    graph.add_node("agent", lambda s: agent_node(s, llm, tools))
    graph.add_node("tools", lambda s: tool_node(s, tool_executor))
    graph.set_entry_point("agent")
    graph.add_conditional_edges(
        "agent", should_continue,
        {"tools": "tools", "end": END}
    )
    graph.add_edge("tools", "agent")
    return graph.compile()

react_agent = build_react_agent(llm, [query_order_tool, get_user_tool])
result = react_agent.invoke({
    "messages": [HumanMessage(content="我的订单 T001 到哪了？")],
    "step": 0, "max_steps": 15
})

@Configuration
public class ReactGraphConfig {
    @Bean
    public Graph reactGraph(ChatModel chatModel, List<ToolCallback> tools) {
        return Graph.builder()
            .addNode("agent", context -> {
                List<Message> messages = context.getState("messages");
                ChatResponse response = chatModel.call(
                    new Prompt(messages, tools));
                context.setState("messages",
                    append(context.getState("messages"), response.getResult()));
                context.setState("step",
                    (int)context.getStateOrDefault("step", 0) + 1);
                return context;
            })
            .addNode("tools", context -> {
                Message last = lastMessage(context.getState("messages"));
                for (ToolCall tc : last.getToolCalls()) {
                    String result = executeTool(tc, tools);
                    context.setState("messages",
                        append(context.getState("messages"),
                            new ToolResponseMessage(result, tc.id())));
                }
                return context;
            })
            .setEntryPoint("agent")
            .addConditionalEdge("agent", context -> {
                int step = (int)context.getStateOrDefault("step", 0);
                if (step >= (int)context.getStateOrDefault("maxSteps", 15))
                    return "end";
                return lastMessage(context.getState("messages")).hasToolCalls()
                    ? "tools" : "end";
            }, Map.of("tools", "tools", "end", "__END__"))
            .addEdge("tools", "agent")
            .build();
    }
}
先想 10 秒：如果 Agent 节点连续调用同一个工具 5 次，会发生什么？

要么是工具返回结果有问题导致模型无法推进，要么是工具副作用导致模型反复调用。should_continue 的步数检测会在此类场景中终止。步数上限是 ReAct 的核心安全带。

步数控制与终止条件

ReAct 循环可能因为多种原因需要终止。除了 max_steps，还需要考虑重复调用、连续失败等检测。

终止条件实现方式推荐值
最大步数step >= max_steps10-15
最大执行时间计时器超时30-60 秒
重复工具检测相同工具相同参数连续 N 次3 次
无意义循环消息内容相似度 > 90%2 次
工具全部失败连续失败次数超限3 次
python 
def should_continue_advanced(state: AgentState) -> str:
    """增强的终止条件检测"""
    if state.get("step", 0) >= state.get("max_steps", 15):
        return "end"
    messages = state["messages"]
    # 重复工具调用检测
    tool_msgs = [m for m in messages
                 if m.get("role") == "assistant" and m.get("tool_calls")]
    if len(tool_msgs) >= 3:
        recent_calls = []
        for m in tool_msgs[-3:]:
            calls = [(tc["name"], json.dumps(tc["args"], sort_keys=True))
                     for tc in m["tool_calls"]]
            recent_calls.append(calls)
        if recent_calls[0] == recent_calls[1] == recent_calls[2]:
            return "end"  # 连续 3 次相同调用，终止
    # 连续失败检测
    tool_results = [m for m in messages if m.get("role") == "tool"]
    recent_fails = [m for m in tool_results[-3:]
                    if "失败" in m.get("content", "")]
    if len(recent_fails) >= 3:
        return "end"
    # 正常判断
    last = messages[-1]
    if hasattr(last, "tool_calls") and last.tool_calls:
        return "tools"
    return "end"

CorpAssist S2 客服 Agent 完整实现

S2 客服 Agent 需要处理订单查询、退款处理和人工转接三种核心场景。

python 
from langchain_core.tools import tool

@tool
def query_customer_order(order_id: str) -> str:
    """查询客户订单状态。输入订单号返回物流、预计送达。"""
    order = order_service.query(order_id)
    return f"订单 {order_id}: {order.status}, 物流: {order.logistics}, 预计: {order.eta}"

@tool
def process_refund(order_id: str, reason: str) -> str:
    """发起退款申请。需要订单号和退款原因，提交后等待审批。"""
    refund_id = refund_service.create(order_id, reason)
    return f"退款已提交，编号 {refund_id}，等待审批"

@tool
def transfer_to_human(issue: str) -> str:
    """转接人工客服。Agent 无法处理或用户要求时调用。"""
    ticket_id = ticket_service.create_escalation(issue)
    return f"已转接人工客服，工单号 {ticket_id}，请稍候"

s2_agent = build_react_agent(llm, [query_customer_order, process_refund, transfer_to_human])

response = s2_agent.invoke({
    "messages": [HumanMessage(content="我想查一下订单 T001，还没收到")],
    "step": 0, "max_steps": 10
})
print(response["messages"][-1].content)
不推荐
  • Agent 和工具节点混在一个函数
  • 不设 max_steps
  • 无重复检测
  • 工具异常抛到外层
推荐
  • Agent 和 Tool 独立节点
  • 多维度终止条件
  • 异常捕获返回友好信息
  • 每步记录状态用于回放

常见问题（FAQ）

LangGraph StateGraph 和普通 DAG 有什么不同？
关键区别在于循环。DAG 不支持循环，ReAct 的核心是思考->行动->再思考的循环。StateGraph 通过条件边支持这种循环，且 State 对象持久化支持中断恢复。
多个工具可以并行调用吗？
LangGraph 支持并行工具调用。模型可在一次响应中请求多个 tool_call，Tool 节点串行或并行执行。见 多 Agent 协作。
ReAct 循环如何插入人工审批？
使用 interrupt_before 在指定节点前中断。详见 人机协作。

本章结论

  • StateGraph：Agent 节点和 Tool 节点通过条件边构成循环。
  • 条件边：根据模型是否调用工具判断走向，步数上限是强制终止条件。
  • 多维度终止：步数、时间、重复调用、连续失败联合保护。
  • S2 客服：订单查询、退款处理、转人工三种工具覆盖常见场景。
  • 双栈一致：Python StateGraph 和 Java Graph API 概念一一对应。
