为什么要学本章

Agent 需要标准化的协议来发现和调用外部工具（MCP），以及与其他 Agent 通信协作（A2A）。MCP（Model Context Protocol）是 Anthropic 提出的工具集成标准，A2A（Agent-to-Agent）是 Google 提出的 Agent 间通信协议。两者共同构成 Agent 生态的互联互通基础。

本章先记住 3 件事
  • MCP = 工具标准化：让任何 Agent 都能发现和调用任何工具。
  • A2A = Agent 通信：让不同平台、不同框架的 Agent 能互相通信协作。
  • 两者互补：MCP 解决 Agent-工具集成，A2A 解决 Agent-Agent 集成。

MCP 协议原理

MCP（Model Context Protocol）定义了 Agent 与外部工具之间的标准接口。它采用 Client-Server 架构：MCP Server 提供工具能力，MCP Client（Agent）发现和调用这些工具。

MCP 协议架构
flowchart LR
    subgraph Client[MCP Client]
        Agent[Agent]
        ToolRegistry[工具注册表]
    end
    subgraph Server[MCP Server]
        Tool1[订单查询工具]
        Tool2[工单管理工具]
        Tool3[知识库工具]
    end
    Client -->|1. ListTools| Server
    Server -->|工具列表| Client
    Client -->|2. CallTool| Server
    Server -->|工具结果| Client

MCP 客户端通过 ListTools 发现工具列表，通过 CallTool 调用具体工具。

MCP 操作描述对应 HTTP
ListTools获取 MCP Server 提供的工具列表GET /tools
CallTool调用指定工具并传入参数POST /tools/{name}/call
GetTool获取单个工具的详细 SchemaGET /tools/{name}

Python：MCP Server 实现

python 
from mcp.server import Server, stdio_server
from mcp.types import Tool, TextContent
import json

# 创建 MCP Server
server = Server("corpassist-tools")

@server.list_tools()
async def list_tools() -> list[Tool]:
    return [
        Tool(
            name="query_order",
            description="查询订单状态。输入订单号返回物流信息。",
            inputSchema={
                "type": "object",
                "properties": {
                    "order_id": {
                        "type": "string",
                        "description": "订单号，如 T2024-001"
                    }
                },
                "required": ["order_id"]
            }
        ),
        Tool(
            name="process_refund",
            description="发起退款申请。需要订单号和原因。",
            inputSchema={
                "type": "object",
                "properties": {
                    "order_id": {"type": "string"},
                    "reason": {"type": "string"}
                },
                "required": ["order_id", "reason"]
            }
        )
    ]

@server.call_tool()
async def call_tool(name: str, arguments: dict) -> list[TextContent]:
    if name == "query_order":
        result = order_service.query(arguments["order_id"])
        return [TextContent(type="text", text=str(result))]
    elif name == "process_refund":
        refund_id = refund_service.create(
            arguments["order_id"],
            arguments["reason"]
        )
        return [TextContent(type="text", text=f"退款 {refund_id} 已提交")]
    raise ValueError(f"Unknown tool: {name}")

# 启动 Server（stdio 模式）
async def run_server():
    async with stdio_server(server) as server_io:
        await server_io.run()

Python：MCP Client 集成到 Agent

python 
from mcp import ClientSession, StdioClient

class MCPToolAdapter:
    """将 MCP 工具适配为 Agent 可用工具"""

    def __init__(self, server_command: str):
        self.server_command = server_command
        self.session = None
        self.tools = []

    async def connect(self):
        process = await create_process(self.server_command)
        self.session = await ClientSession(process.stdout, process.stdin)
        await self.session.initialize()
        # 获取工具列表
        tools_result = await self.session.list_tools()
        self.tools = tools_result.tools

    async def execute(self, tool_name: str, arguments: dict) -> str:
        result = await self.session.call_tool(tool_name, arguments)
        return "".join(c.text for c in result.content)

# 在 Agent 中使用
mcp_adapter = MCPToolAdapter("python mcp_server.py")
await mcp_adapter.connect()
for mcp_tool in mcp_adapter.tools:
    # 动态创建 LangChain 工具
    tools.append(create_mcp_tool(mcp_tool, mcp_adapter))
不推荐
  • 工具硬编码在 Agent 代码中
  • 每个 Agent 有独立的工具集成方式
  • MCP Server 无鉴权
推荐
  • 通过 MCP 协议动态发现工具
  • 所有 Agent 使用统一工具集成接口
  • MCP Server 配置 OAuth2 鉴权

A2A Agent-to-Agent 协议

A2A 是 Google 提出的 Agent 间通信协议，解决不同平台、不同框架的 Agent 之间如何发现、连接和协作。A2A 定义了 Agent Card（能力声明）、Task（任务单元）和 Message（通信消息）等核心概念。

json 
// Agent Card — Agent 的能力声明
{
  "agent_name": "corpassist-s2",
  "version": "2.0",
  "capabilities": [
    {
      "name": "order_query",
      "description": "查询订单状态",
      "input_schema": {...},
      "output_schema": {...}
    },
    {
      "name": "ticket_management",
      "description": "管理客服工单",
      "input_schema": {...}
    }
  ],
  "communication": {
    "protocol": "a2a",
    "endpoint": "https://agent.example.com/a2a",
    "auth": "oauth2"
  }
}

在 CorpAssist 场景中，A2A 让 S2 客服 Agent 可以调用 S3 办公自动化 Agent 的能力——例如客服 Agent 需要查询员工组织架构时，通过 A2A 协议向办公 Agent 发起请求。

常见问题（FAQ）

MCP 和 Function Calling 是什么关系？
Function Calling 是 LLM 的能力——模型声明需要调用某个函数。MCP 是工具的标准——定义工具如何被发现和调用。两者是互补的：MCP 提供工具注册和发现，Function Calling 提供模型层面的调用声明。Agent 框架（LangGraph、Spring AI）将两者结合起来。
A2A 和 MCP 的区别是什么？
MCP 是 Agent-工具协议，解决 Agent 如何调用外部工具。A2A 是 Agent-Agent 协议，解决 Agent 之间如何通信协作。MCP 适合工具集成（查询 API、操作数据库），A2A 适合 Agent 间的任务委派和协同（客服 Agent 请求财务 Agent 协助）。
MCP Server 如何保证安全？
MCP Server 需要实现鉴权和权限控制。标准做法：① 使用 OAuth2 或 API Key 认证；② 每个工具定义调用权限级别；③ 工具调用记录审计日志；④ 敏感操作需要二次确认。MCP 协议本身不强制安全措施，需要在 Server 层实现。

本章结论

  • MCP 协议：标准化工具集成，Client-Server 架构，支持动态工具发现。
  • A2A 协议：标准化 Agent 间通信，支持跨平台 Agent 协作。
  • MCP + A2A：两者互补，MCP 管工具，A2A 管 Agent。
  • 双栈实现：Python MCP SDK 和 Java Spring AI MCP 概念一致。
  • 安全考量：MCP Server 需要实现鉴权、权限和审计。
