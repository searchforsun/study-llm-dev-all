为什么要学本章

Agent 的评估比传统软件更困难——输出是自然语言而非结构化结果。agent evaluation 需要结合离线评测（已知正确答案的数据集）和在线监控（用户反馈和行为指标）。没有评测体系，你无法知道 Agent 是变好了还是变差了。

学完你能
  • 构建 Agent 离线评测数据集和自动化流水线
  • 定义 CSAT、转人工率等在线业务指标
  • 设计回归测试防止 Agent 退化
  • 使用 LangSmith 进行评测管理
本章先记住 3 件事
  • 离线评测：用标注好的数据集测试 Agent 输出质量，发现问题在上线前。
  • 在线指标：真实用户反馈（CSAT）和行为指标（转人工率）反映生产表现。
  • 回归测试：每次 Prompt 修改、工具变更后运行，防止「修复一个 bug 引入三个新 bug」。

离线评测与数据集

离线评测的核心是高质量的数据集。每个测试用例包含：用户输入、期望的工具调用、期望的最终回答。

Python：离线评测引擎

python 
class AgentEvaluator:
    """Agent 离线评测引擎"""

    def __init__(self, agent, test_cases: list):
        self.agent = agent
        self.test_cases = test_cases

    def evaluate(self) -> dict:
        results = {"passed": 0, "failed": 0, "details": []}
        for case in self.test_cases:
            result = self._evaluate_single(case)
            results["details"].append(result)
            if result["passed"]:
                results["passed"] += 1
            else:
                results["failed"] += 1
        results["pass_rate"] = results["passed"] / len(self.test_cases)
        return results

    def _evaluate_single(self, case: dict) -> dict:
        response = self.agent.invoke({"messages": [HumanMessage(content=case["input"])]})

        checks = {
            "tool_correct": self._check_tools(response, case.get("expected_tools", [])),
            "response_quality": self._check_response(response, case.get("expected_response_contains", [])),
        }
        passed = all(checks.values())
        return {"id": case["id"], "passed": passed, "checks": checks}

    def _check_tools(self, response, expected_tools):
        actual_tools = [m.name for m in response["messages"] if hasattr(m, "name")]
        return all(t in actual_tools for t in expected_tools)

    def _check_response(self, response, expected_contains):
        last_msg = response["messages"][-1].content
        return all(kw in last_msg for kw in expected_contains)

LangSmith 数据集管理

LangSmith 提供了数据集创建、评测运行和结果对比的功能。你可以在 LangSmith 中创建数据集，将 Agent 运行结果自动记录到数据集，然后运行对比评测。

python 
from langsmith import Client, Dataset

client = Client()

# 创建数据集
dataset = client.create_dataset(
    "corpassist-s2-eval",
    description="CorpAssist S2 客服 Agent 评测集"
)

# 添加示例
client.create_example(
    inputs={"input": "查订单 T001"},
    outputs={
        "expected_tools": ["query_order"],
        "expected_response": "已发货"
    },
    dataset_id=dataset.id
)

# 运行评测
def evaluate_agent(example):
    result = agent.invoke({"messages": [HumanMessage(content=example["input"])]})
    return {"response": result["messages"][-1].content}

results = client.run_on_dataset(
    dataset_name="corpassist-s2-eval",
    llm_or_chain_fn=evaluate_agent,
    project_name="s2-v2-eval"
)

在线指标与监控

离线评测无法覆盖所有生产场景。在线指标反映 Agent 在真实用户环境中的表现。

首次解决率未再次提问的会话比> 70%24h 内回归分析

先想 10 秒：CSAT 高但转人工率也高，说明什么？

说明 Agent 可能只处理了简单问题（用户满意），但稍微复杂的问题就转人工了。需要检查转人工的触发原因：是 Agent 能力不足、工具缺失还是 System Prompt 过于保守导致过早转交。

回归测试与 CI

回归测试确保 Agent 变更不会引入退化。每次 Prompt 修改、工具变更或模型升级后都应运行回归测试。

yaml 
# .github/workflows/agent-eval.yml
name: Agent Evaluation
on: [pull_request]
jobs:
  evaluate:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - name: Run Agent Eval
        run: |
          python -m pytest tests/evaluation/ -v --junitxml=results.xml
      - name: Check Quality Gate
        run: |
          python scripts/check_quality_gate.py \
            --pass-rate 0.85 \
            --max-regression 0.05
      - name: Upload Results
        uses: actions/upload-artifact@v4
        with:
          name: eval-results
          path: results.xml

常见问题（FAQ）

评测数据集的标注工作量太大了，有什么省力的方法？
可以先用 LLM 生成初始数据集（合成数据），然后人工抽样审核修正。也可以从生产日志中采样用户问题和 Agent 回答，由人工标注是否合格。建议从 50 个用例开始，覆盖核心场景，然后逐步扩展到 200-500 个。
Agent 的回答是自然语言，如何自动化判断是否正确？
常用的方法有：① 关键词/正则匹配（简单但不够准确）；② LLM-as-Judge（用另一个 LLM 打分，准确率 85%+）；③ 语义相似度（用 embedding 比较回答和标准答案的相似度）。建议组合使用——先用规则做预检，再用 LLM 判分。
在线指标多久看一次比较合适？
CSAT 和转人工率建议每日看，因为样本量太小时指标波动大。延迟和 Token 消耗建议实时监控，设置告警阈值。每周汇总趋势分析，发现 Agent 表现是否在退化。每次模型升级或 Prompt 变更后需要密切观察 24-48 小时。

本章结论

  • 三维评测：离线（数据集）、在线（用户指标）、CI（回归测试）。
  • 数据集设计：包含输入、期望工具、期望输出三个要素。
  • LLM-as-Judge：用 LLM 判分是自动化评估自然语言输出的有效方法。
  • 在线指标：CSAT、转人工率、首次解决率反映生产表现。
  • CI 集成：每次变动自动运行评测，设置质量门禁。
