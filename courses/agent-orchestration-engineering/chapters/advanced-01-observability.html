为什么要学本章

Agent 的决策过程是非确定性的——同样的输入可能产生不同的输出。传统的日志监控无法捕捉 Agent 的推理轨迹。observability 通过 Trace、Metrics 和 Logging 三大支柱让 Agent 的内部决策过程透明化，是生产 Agent 不可或缺的基础设施。

学完你能
  • 使用 LangSmith 和 OpenTelemetry 追踪 Agent 执行轨迹
  • 设计 Agent 的可观测性指标
  • 实现 Agent 执行回放用于调试
  • 通过 Trace 定位 Agent 故障根因
本章先记住 3 件事
  • Trace = 调用链：记录 LLM 调用、工具调用、推理步骤的完整轨迹。
  • Span = 调用单元：每个 LLM 调用或工具调用是一个 Span，包含耗时和元数据。
  • Metrics = 聚合指标：延迟、Token 消耗、错误率的聚合视图。

三大支柱协同工作：Trace 记录单次完整执行，Metrics 聚合全局趋势，Logging 提供细节。

LangSmith Trace 集成

LangSmith 是 LangGraph/LangChain 生态的官方可观测平台，自动捕获 Agent 的每一步执行。在 LangChain 集成下，只需配置 API Key 即可启用。

Python：LangSmith 配置

python 
import os
from langsmith import traceable
from langchain_core.tracers import LangSmithTracer

# 配置环境变量
os.environ["LANGSMITH_TRACING"] = "true"
os.environ["LANGSMITH_API_KEY"] = "ls__your_key"
os.environ["LANGSMITH_PROJECT"] = "corpassist-s2"

# 手动追踪特定函数
@traceable(name="query_order_tool", run_type="tool")
def query_order(order_id: str) -> str:
    result = order_service.query(order_id)
    return f"订单 {order_id}: {result.status}"

# 追踪 Agent 执行
@traceable(name="s2_agent_run", run_type="chain")
def run_agent(user_input: str) -> str:
    response = agent.invoke({"messages": [HumanMessage(content=user_input)]})
    return response["messages"][-1].content

# 在 LangSmith 中可以看到：
# - 每次 LLM 调用的输入输出和 Token 数
# - 每次工具调用的参数和结果
# - 完整的推理链和步数
# - 执行时间和延迟分布

Python：自定义 Span

python 
from langsmith import traceable
from langsmith.run_trees import RunTree

# 手动创建追踪树
def trace_agent_execution(user_input: str):
    root = RunTree(
        name="agent_execution",
        run_type="chain",
        inputs={"input": user_input}
    )

    # LLM 调用 Span
    llm_span = root.create_child(
        name="llm_call",
        run_type="llm",
        inputs={"messages": [...]}
    )
    llm_span.end(outputs={"response": "..."})
    llm_span.post()

    # 工具调用 Span
    tool_span = root.create_child(
        name="tool_call",
        run_type="tool",
        inputs={"tool": "query_order", "args": {"order_id": "T001"}}
    )
    tool_span.end(outputs={"result": "已发货"})
    tool_span.post()

    root.end(outputs={"final_response": "..."})
    root.post()
不推荐
  • 无 Trace，出问题只能看日志猜
  • 只采集错误不采集成功 Trace
  • 不关联 LLM 调用和工具调用
推荐
  • 全量 Trace 自动采集
  • 成功 + 失败 Trace 都采集
  • 父子 Span 关联完整调用链

OpenTelemetry 集成

如果使用 Java 技术栈或不依赖 LangChain，可以通过 OpenTelemetry 实现 Agent 可观测性。

java 
@Component
public class AgentObservability {
    private static final Tracer tracer =
        GlobalOpenTelemetry.getTracer("corpassist-agent");

    public String executeWithTrace(String userId, String message) {
        Span span = tracer.spanBuilder("agent.execute")
            .setAttribute("user.id", userId)
            .setAttribute("message.length", message.length())
            .startSpan();

        try (Scope scope = span.makeCurrent()) {
            // LLM 调用子 Span
            Span llmSpan = tracer.spanBuilder("agent.llm_call")
                .setAttribute("model", "qwen-max")
                .startSpan();
            String llmResponse = chatModel.call(message);
            llmSpan.end();

            // 工具调用子 Span
            Span toolSpan = tracer.spanBuilder("agent.tool_call")
                .setAttribute("tool", "query_order")
                .setAttribute("args", "T001")
                .startSpan();
            String toolResult = queryOrder("T001");
            toolSpan.end();

            return llmResponse;
        } catch (Exception e) {
            span.recordException(e);
            span.setStatus(StatusCode.ERROR);
            throw e;
        } finally {
            span.end();
        }
    }
}

关键指标与监控

指标含义告警阈值
Agent 执行延迟 P5050% 的执行在多少毫秒内完成> 5s 告警
Agent 执行延迟 P9999% 的执行在多少毫秒内完成> 15s 告警
每执行 Token 消耗平均每次执行消耗多少 Token日环比 > 20% 告警
工具调用失败率工具调用失败的百分比> 5% 告警
死循环触发率达到 max_steps 的执行比例> 10% 告警
平均步数每次执行的平均 ReAct 步数日均变化 > 1 步

调试与回放

Trace 不仅用于监控，还可以用于调试。通过回放 Trace 可以重现 Agent 的执行过程，定位问题步骤。

python 
from langsmith import Client

client = Client()

# 根据条件查询 Trace
runs = client.list_runs(
    project_name="corpassist-s2",
    execution_order="desc",
    limit=10
)

for run in runs:
    print(f"Run ID: {run.id}")
    print(f"Input: {run.inputs}")
    print(f"Output: {run.outputs}")
    print(f"Error: {run.error}")
    print(f"Latency: {run.latency_ms}ms")
    print(f"Token Usage: {run.prompt_tokens + run.completion_tokens}")

    # 获取子 Span
    child_runs = client.list_runs(
        parent_run_id=run.id,
        execution_order="asc"
    )
    for child in child_runs:
        print(f"  - {child.name}: {child.latency_ms}ms")

常见问题（FAQ）

全量 Trace 会不会影响性能？
Trace 采集是异步的，通常不会阻塞 Agent 执行。LangSmith 的 Python SDK 使用后台线程发送数据，延迟增加在毫秒级。如果对性能要求极高，可以采样（如 10% 的请求采集 Trace）或只采集异常 Trace。
Trace 数据量太大怎么办？
LangSmith 和 OpenTelemetry 都支持采样策略。建议：成功 Trace 采样 10%，失败 Trace 全量采集。也可以通过分级存储——热数据存 7 天，冷数据归档到 S3。Trace 的保留策略需要根据合规要求确定。
Spring AI Alibaba 怎么集成 Trace？
Spring AI Alibaba 支持 Micrometer 和 OpenTelemetry 集成。通过添加 micrometer-registry-prometheus 和 opentelemetry-spring-boot-starter 依赖，自动采集指标和 Trace。默认集成了 LLM 调用和工具调用的 Span。参考 可靠性工程。

本章结论

  • 三大支柱：Trace 记录轨迹、Metrics 聚合趋势、Logging 提供细节。
  • LangSmith：LangChain/LangGraph 生态官方可观测平台，零配置集成。
  • OpenTelemetry：Java 栈标准可观测方案，Spring AI Alibaba 原生支持。
  • 关键指标：延迟、Token、错误率、步数、死循环触发率。
  • 调试回放：通过 Trace 回放 Agent 执行过程，定位故障步骤。
