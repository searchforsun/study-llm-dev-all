FastAPI 入口（双场景）

python 
@app.post("/v1/agent/{scene}/chat")
async def chat(scene: Literal["s2", "s3"], req: ChatReq, user=Depends(auth)):
    graph = GRAPHS[scene]
    config = {
        "configurable": {
            "thread_id": req.thread_id,
            "role": user.role,
            "user_id": user.id,
        }
    }
    if req.stream:
        return EventSourceResponse(stream_graph(graph, req.message, config))
    return await graph.ainvoke({"messages": [("user", req.message)]}, config)

指标

agent metrics 分体验、可靠、成本三类；与 LangSmith experiment 与 Prometheus 双写。

指标定义S2 目标示例告警
csat metric 会话结束 1～5 分 ≥ 4.2 周环比 -0.2
transfer rate 转人工 / 总会话 ≤ 18% > 25% 1h
tool_failure_rate tool 错误 / 调用 < 2% MCP 宕机
avg_steps / cost 每会话步数与 Token steps ≤ 8 成本 +30%
eval_pass_rate regression dataset ≥ 92% gate fail

先想 10 秒：eval_pass_rate 92% 但 CSAT 跌，先查什么？

查 dataset 是否覆盖最新促销话术/新 SKU；trace replay CSAT 低分会话，看是回复语气、步数过早终止还是 tool 误调。指标冲突时用户体验 KPI 优先，但须排除评测集过时。

文档

交付物：openapi agent spec、Spring 集成指南、on-call runbook。文档与代码同 PR，避免「代码上了 runbook 下周补」。

runbook 结构（扩展 demo）

markdown 
# demos/advanced-05-capstone-lab/runbook.md
## 1. 服务概览
- FastAPI :8080, graphs: s2, s3
- 依赖: Redis checkpoint, MCP order-service

## 2. 告警 playbook
| 告警 | 首步 | 升级 |
| transfer_rate high | 查 LangSmith intent 分布 | on-call TL |
| tool_failure_rate | 查 MCP 健康 | 切 fallback 话术 |

## 3. 灰度与回滚
- 5% → 24h → 100%; 回滚 helm revision N-1

## 4. 联调
- Spring: X-Session-Id, POST /v1/agent/s2/chat

API 文档要点

  • thread_id：续聊与 checkpoint 键；Spring 须持久化。
  • stream=true：sse stream 事件含 tool_start/tool_end。
  • 错误码：429 步数耗尽、403 tool denied、503 MCP 不可用。
生产陷阱
  • 只交付 Docker 镜像：无 OpenAPI、无 runbook，Spring 联调反复 3 天。
  • 指标未接告警：CSAT 跌一周才发现。
  • 跳过 eval gate：灰度直接 50% 流量——违反 eval gate 纪律。

本章结论

  • 端到端：S2/S3 分 graph，共享 FastAPI；checklist 覆盖 MCP、interrupt、预算、obs。
  • 指标：CSAT、转人工、tool 失败、成本、eval pass；看板 + 告警绑定灰度。
  • 文档：OpenAPI、Spring 联调、runbook 与代码同发布；eval gate 先于放量。

  5. 设计 eval gate：handoff_rate 不得较 baseline +2%，写在 PR 模板一句。
  6. 可选：对照 demos/practice-06-api-lab/openapi-snippet.yaml 合并到 capstone。

判断练习

  1. （发布）eval 90% pass，PM 要求今晚全量。你如何决策？

    参考答案

    拒绝直接全量：未达 92% 门槛或需书面豁免；最低 5% 灰度 + 看板 24h。强调 production agent 纪律与回滚预案。

  2. （双栈）Spring 调通 chat，但未传 thread_id。会有什么问题？

    参考答案

    无法续聊与 interrupt resume；每次新会话，checkpoint 失效；S3 长任务中断后不能恢复。须在 api doc 标必填并集成测试。
