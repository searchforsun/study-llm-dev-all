为什么要学本章

CorpAssist 工具来自 CRM、工单、知识库等多系统。MCP server 统一暴露 capability list，避免每个 Agent 手写 HTTP 胶水。

学完你能
  • 用 MCP client 拉取工具并转成 LangChain tools
  • 配置 mcp auth 与 timeout
  • 理解 stdio MCP 与 HTTP 部署差异

接入

Tool bridge：启动时连接 MCP，缓存 schema，注册到 ToolNode。

json
{
  "mcpServers": {
    "corp-ticket": {
      "command": "uvx",
      "args": ["corp-mcp-ticket", "--stdio"],
      "env": {"TICKET_API": "${TICKET_API}"}
    }
  }
}

配置见 demos/practice-04-mcp-tools-lab/mcp-config.json。

鉴权

Agent 进程持有 service account；MCP 层映射用户 ACL，禁止把用户 JWT 裸传给不可信 server。External tool 仍走白名单。

超时

每个 MCP call 独立 timeout + 总 budget；超时 Observation 用 retryable envelope（见 basics-02）。

不推荐
  • 默认无限等待 MCP
推荐
  • connect 2s / call 5s，与 Spring 网关一致

本章结论

  • 接入：MCP 统一工具发现与 schema。
  • 鉴权：服务身份 + 用户 ACL 映射。
  • 超时：分级 timeout，失败可观测。
