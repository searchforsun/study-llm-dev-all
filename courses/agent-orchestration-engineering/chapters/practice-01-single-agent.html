意图路由与工具调用

Router 用小模型或规则先分类，再限制 bind_tools 列表，降低误调 refund_execute 概率。

python
INTENT_TOOLS = {
    "track_order": [get_order, create_ticket],
    "refund_inquiry": [get_order, refund_policy, handoff_human],
    "complaint": [create_ticket, handoff_human],
}

def router_node(state):
    intent = classify_intent(state["messages"][-1])
    return {"intent": intent, "allowed_tools": INTENT_TOOLS[intent]}

人工转接降级

触发条件：步数超限、用户明确要求、工具连续失败、低 CSAT 预测。走 handoff_human 写工单队列，返回固定话术模板。

不推荐
  • 模型自由说「已转人工」无队列
推荐
  • 工具返回 ticket_id；Spring 侧工单系统可查

评测指标 CSAT/转人工率

指标定义门禁示意
CSAT会话结束后 thumbs up 比例≥ 78% 发版
transfer ratehandoff / 总会话≤ 25%（可业务调）
工具成功率ok=true 的 tool call 比例≥ 95%

坏例进 golden set（NS1），发版前回归 — 见《评测、实验与用户反馈闭环》。

本章结论

  • S2 Agent：router 限工具 + ReAct 环 + handoff 节点。
  • 意图与工具：DST 槽位规则校验 + 模型理解分离。
  • 降级：明确触发条件，handoff 必须写系统。
  • 指标：CSAT/转人工率/工具成功率作发版门禁。
