本章先记住 4 件事
  • StateGraph：图的状态类型与字段合并规则是一等公民。
  • 节点：纯函数 (state) -> partial state，不要做隐式全局副作用。
  • 边：固定边 + 条件边决定 ReAct 何时结束。
  • 编译：compile() 产出可部署对象；生产必带 checkpointer。

StateGraph

CorpAssist 最小客服图 状态至少包含：messages（对话）、step_count（步数预算）、handoff（是否已转人工）。StateGraph 规定这些字段如何随节点更新。
python 
from typing import Annotated
from typing_extensions import TypedDict
from langgraph.graph import StateGraph, START, END
from langgraph.graph.message import add_messages

class AgentState(TypedDict):
    messages: Annotated[list, add_messages]
    step_count: int
    handoff: bool

add_messages reducer 表示新消息追加而非覆盖——适合 ReAct 中 tool 与 assistant 消息交替。自定义字段（如 step_count）默认覆盖，需在节点内显式 +1。

字段类型S2 用途
messages BaseMessage 列表 用户输入、tool observation、最终回复
step_count int 与 步数预算 联动
handoff bool 条件边直达 END，跳过 tools
python 
builder = StateGraph(AgentState)

节点

graph node 是同步函数，接收当前 agent state，返回要合并的局部更新。

python 
from langgraph.prebuilt import ToolNode

def agent_node(state: AgentState) -> dict:
    step = state.get("step_count", 0) + 1
    response = llm_with_tools.invoke(state["messages"])
    return {"messages": [response], "step_count": step}

tool_node = ToolNode(TOOLS)

builder.add_node("agent", agent_node)
builder.add_node("tools", tool_node)

节点职责划分

节点输入输出禁止
agent messages assistant 消息（可含 tool_calls） 直接 HTTP 调 CRM（应走 tools）
tools tool_calls tool 消息 observation 修改与工具无关的业务状态
handoff_human（可选） messages handoff: true + 固定话术 继续调用业务 API
节点里写数据库合适吗？

副作用应集中、可测试；优先在 tool 内访问外部系统，节点只做编排与 state 变换。

边

graph edge 描述控制流。ReAct 典型结构：START → agent，agent → tools | END，tools → agent。

python 
def route_after_agent(state: AgentState) -> str:
    if state.get("handoff"):
        return "end"
    if state.get("step_count", 0) >= MAX_STEPS:
        return "end"
    last = state["messages"][-1]
    if getattr(last, "tool_calls", None):
        return "tools"
    return "end"

builder.add_edge(START, "agent")
builder.add_conditional_edges(
    "agent",
    route_after_agent,
    {"tools": "tools", "end": END},
)
builder.add_edge("tools", "agent")
条件边（对照 demos/graph-sketch.md）
flowchart TD
  START((START)) --> agent[agent 节点]
  agent -->|tool_calls| tools[tools 节点]
  agent -->|无 tool / 达预算| END((END))
  tools --> agent

entry node 为 agent；conditional edge 实现 ReAct 分支。

不推荐
  • tools 后直接 END，无法再规划
  • 条件边返回未在映射表注册的字符串
推荐
  • tools → agent 形成闭环
  • 显式 MAX_STEPS 与 handoff 分支

编译

compile graph 将 builder 固化为可执行图，并可挂载 checkpointer、interrupt 点（后续章节）。

python 
from langgraph.checkpoint.memory import MemorySaver

checkpointer = MemorySaver()
app = builder.compile(checkpointer=checkpointer)

# FastAPI 路由中
# app.invoke(input, {"configurable": {"thread_id": session_id}})
# app.astream_events(...)  # SSE

本章结论

  • StateGraph：TypedDict + reducer 定义可恢复的 agent state。
  • 节点：agent 负责 LLM，tools 负责 tool use，职责单一。
  • 边：条件边实现 ReAct 闭环与结束条件。
  • 编译：compile(checkpointer=...) 得到生产可 invoke 的图应用。

判断练习

  1. tools → END 且不再回到 agent，对 ReAct 有何影响？

    参考答案

    模型无法根据 observation 二次规划，易直接输出未消化工具结果的回复；应 tools → agent。

  2. 未 compile 的 builder 能否在 FastAPI 请求里并发 invoke？

    参考答案

    不能；需 compile graph 得到线程安全的 Runnable，并在进程内单例复用。
