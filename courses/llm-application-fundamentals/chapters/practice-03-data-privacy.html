安全红线（本章贯穿）
  • 默认最小出网：未标注 dataClass 的请求不得调用公有云 API。
  • 日志≠调试场：生产日志禁止记录完整 prompt、身份证、手机号、银行卡、合同正文。
  • 密钥与分级分离：API Key 在网关；业务服务只传分级标签与脱敏后文本。
  • 采购书面化：「不用于训练」「数据驻留境内」须写入合同附件，不能仅靠口头承诺。

数据分级

CorpAssist · 安全评审 智能客服坐席粘贴了客户手机号与订单号；制度问答索引含未公开人事制度 PDF。架构师被要求：为每条进入 LLM 的上下文打数据分级标签，网关按标签路由，审计能追溯「谁在何时把哪一级数据送到了哪个 Provider」。

CorpAssist 采用与集团信息安全对齐的四级分类（L1～L4）。应用工程师的职责不是重新定义分级，而是在 API 与消息队列中携带可机读的 dataClass，并让编排层、RAG 检索与模型路由一致遵守。L3 制度片段可经网关出网；L4 仅允许内网 Embedding 与私有化推理。分级决定「能去哪台模型」，而不是事后由法务补批。

出网与私有化

出网在 LLM 场景下指：用户或系统数据离开企业可控边界，进入模型厂商或第三方公有云的计算与存储域（含「专属实例」在厂商机房内的情况）。私有化指推理与向量检索发生在企业 VPC、专有云或离线机房。应用侧须同时回答两个问题：数据有没有出去？ 与 出去的是不是已脱敏/摘要？

三种部署形态（复习 + 合规视角）

形态数据驻留适用分级CorpAssist 注意点
公有云 API 逻辑出网；依合同可能境内区域 L1；经审批的 L2 须「不用于训练」条款；限流与内容安全
专属 / VPC 专线实例 逻辑隔离，运维在厂商侧 L2～L3 采购检查项须含审计接口、补丁 SLA

关键：脱敏在路由之前；审计记录分级、Provider、requestId，不记录完整 prompt。L4 应在策略引擎被阻断，而非依赖模型「自觉保密」。

与模型选型的衔接

路由键 scene + dataClass 与 模型选型 中多 Provider 图一致。新增业务模块时，先定分级与出网路径，再选 Provider，避免「先接 API 再补合规」。

yaml 
# llm-gateway 路由片段（CorpAssist 示例）
routes:
  - match: { dataClass: [L1, L2], scene: faq }
    provider: dashscope
    require: [contract_enterprise, log_redaction_on]
  - match: { dataClass: [L3, L4], scene: knowledge }
    provider: qwen-private-vllm
    egress: forbidden_public_internet
  - match: { dataClass: [L4] }
    action: deny
    message: "L4 数据不得进入通用 LLM，请走人工审批工作流"

日志脱敏

可观测性需要 Trace 与日志，但日志系统权限往往宽于 LLM 网关。一次把完整 prompt、客户身份证号写入 ELK，等于把 L3/L4 复制到更多存储。CorpAssist 约定：结构化日志只记元数据；若必须采样调试，走独立脱敏管道、短期留存、权限审批。

哪些字段必须脱敏或禁止落盘

字段类型示例日志策略
PII 手机、身份证、邮箱、银行卡 掩码或哈希；禁止明文
凭证 API Key、OAuth token、密码 永不记录；仅记录 keyId 后四位
对话正文 user/assistant 完整 content 生产默认不记；采样须分级 ≤ L2 且脱敏
RAG 片段 检索到的制度段落 只记 docId、chunkId、分级

「企业版」是否自动满足所有检查项？
否。企业版通常包含专属配额与部分内容安全，但「不用于训练」「境内驻留」仍须在合同逐条确认。应用侧应保留检查表签字版，而非仅保存销售 PPT。
开源自托管是否就不用采购表？
权重开源不等于合规免责。你仍需评估GPU 机房物理安全、镜像漏洞、备份介质分级，并在检查表上勾选「自托管责任方：甲方运维」对应项。

本章结论

合规 = 分级标签 + 出网前脱敏 + 安全日志 + 书面采购条款，四件事缺一不可。

  • 数据分级：L1～L4 决定路由与存储边界；模块级默认分级须在需求阶段确定。
  • 出网与私有化：脱敏先于路由；L4 阻断进通用 LLM，而非依赖模型自觉。
  • 日志脱敏：生产日志记元数据与指纹，不记 PII 与完整 prompt。
  • 企业采购检查项：训练数据、驻留、审计、SLA、兼容接口须可勾选、可归档。

判断练习

  1. 为排查幻觉，团队在 Kibana 中索引了完整 prompt 与 model 回复，保留 90 天。从数据分级与日志脱敏角度，最大问题是什么？

    参考答案

    扩大 L3/L4 暴露面：日志系统权限通常宽于 LLM 网关，90 天留存等于把对话正文复制到更多介质，违反最小必要原则。应改为指纹、分级、docId 与采样脱敏；评测用黄金集走独立加密存储与审批访问。

  2. 厂商销售口头承诺「绝对不会用你们数据训练」。是否足以让 L3 制度全文走公有云 API？

    参考答案

    不足。L3 制度全文默认应走私有化或专属实例；口头承诺不能替代 DPA/合同附件中的 opt-out、驻留区域与审计条款。还需信息安全与法务书面审批，而非销售话术。
