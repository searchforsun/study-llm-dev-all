实战：首次打通模型调用

约 35 分钟 · 阶段：应用架构与能力拼图 · 能力：用 curl / Python / Spring AI 完成首次 OpenAI 兼容 调用并读懂响应

为什么要学本章

API 契约与集成方式 讲了契约与拓扑；本章在你的机器上跑通第一条请求。CorpAssist 后续 Python 编排与 Spring AI 微服务共用同一网关地址，先会用 curl 验通，再复制到双栈客户端，能少走半天联调弯路。

学完你能
  • 用 curl 完成非流式与 流式 调用，并从 JSON / SSE 中取出回复与 usage
  • 用 Python（httpx 或 OpenAI SDK）编写可运行的最小客户端
  • 配置 Spring AI ChatClient Bean，指向公司 LLM 网关
  • 对照双栈在超时、流式、Token 计费上的共同配置项
动手前准备
  • 一个 OpenAI 兼容 端点：通义 compatible-mode、DeepSeek、或 CorpAssist llm-gateway（见 Demo 目录 .env.example）
  • 环境变量：CORPASSIST_LLM_BASE_URL、CORPASSIST_LLM_KEY、CORPASSIST_LLM_MODEL
  • Python 3.10+（可选 pip install httpx openai）；Java 17+ 与 Spring Boot 3.x（Spring AI 章节预览）

curl 调用

CorpAssist · 联调第一步 网关刚部署到测试环境，运维给你 Key 与 base_url。在改任何业务代码前，用 curl 证明链路可达，并确认 model 路由与 usage 字段符合采购合同。

所有后续客户端最终都变成同一条 HTTP：POST {base_url}/chat/completions。curl 无依赖、可脚本化，适合写进 CI 冒烟或 on-call 手册。

非流式：一次拿完整 JSON

bash 
# Linux / macOS / Git Bash
export CORPASSIST_LLM_BASE_URL="https://dashscope.aliyuncs.com/compatible-mode/v1"
export CORPASSIST_LLM_KEY="sk-your-key"
export CORPASSIST_LLM_MODEL="qwen-plus"

curl -s "${CORPASSIST_LLM_BASE_URL%/}/chat/completions" \
  -H "Authorization: Bearer ${CORPASSIST_LLM_KEY}" \
  -H "Content-Type: application/json" \
  -H "X-Request-Id: $(uuidgen 2>/dev/null || cat /proc/sys/kernel/random/uuid)" \
  -d '{
    "model": "'"${CORPASSIST_LLM_MODEL}"'",
    "stream": false,
    "temperature": 0.2,
    "max_tokens": 256,
    "messages": [
      {"role": "system", "content": "你是 CorpAssist 企业助手，回答须简洁。"},
      {"role": "user", "content": "年假最多可休几天？"}
    ]
  }' | jq -r '.choices[0].message.content, .usage'

成功时关注三处：choices[0].message.content（正文）、usage.prompt_tokens / completion_tokens（Token 计费依据）、id（排障时提供给网关日志）。

流式：SSE 逐块输出

bash 
curl -N "${CORPASSIST_LLM_BASE_URL%/}/chat/completions" \
  -H "Authorization: Bearer ${CORPASSIST_LLM_KEY}" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "'"${CORPASSIST_LLM_MODEL}"'",
    "stream": true,
    "messages": [{"role": "user", "content": "用三句话介绍 RAG"}]
  }'

加 -N 禁用 curl 缓冲，才能实时看到 data: {...} 行与结尾 data: [DONE]。流式场景下 streaming 的 delta.content 需客户端拼接；最后一包常带 usage（依厂商略有差异，见 API 契约与集成方式）。

Windows PowerShell

Demo 目录提供 chat-non-stream.ps1 / chat-stream.ps1，逻辑与上述 bash 等价；加载 .env 后执行即可。

Python 最小客户端

CorpAssist 的 Python 编排服务（评测脚本、离线摘要、RAG 流水线）推荐两种写法：httpx（显式 HTTP，与 curl 一一对应）或 官方 OpenAI SDK（改 base_url 即可连 OpenAI 兼容 网关）。

方式 A：httpx 异步客户端

python 
#!/usr/bin/env python3
"""CorpAssist 最小调用 — httpx（与 curl 同形）。"""
from __future__ import annotations

import asyncio
import os
import sys

import httpx

BASE_URL = os.environ.get("CORPASSIST_LLM_BASE_URL", "").rstrip("/")
API_KEY = os.environ.get("CORPASSIST_LLM_KEY", "")
MODEL = os.environ.get("CORPASSIST_LLM_MODEL", "qwen-plus")


async def chat_once(user_text: str, *, stream: bool = False) -> str | None:
    if not BASE_URL or not API_KEY:
        raise RuntimeError("请设置 CORPASSIST_LLM_BASE_URL 与 CORPASSIST_LLM_KEY")

    timeout = httpx.Timeout(60.0, connect=5.0)
    headers = {"Authorization": f"Bearer {API_KEY}"}

    async with httpx.AsyncClient(base_url=BASE_URL, headers=headers, timeout=timeout) as client:
        if not stream:
            r = await client.post(
                "/chat/completions",
                json={
                    "model": MODEL,
                    "stream": False,
                    "messages": [
                        {"role": "system", "content": "你是 CorpAssist 企业助手。"},
                        {"role": "user", "content": user_text},
                    ],
                },
            )
            r.raise_for_status()
            data = r.json()
            usage = data.get("usage", {})
            print(
                f"usage: prompt={usage.get('prompt_tokens')} "
                f"completion={usage.get('completion_tokens')}",
                file=sys.stderr,
            )
            return data["choices"][0]["message"]["content"]

        async with client.stream(
            "POST",
            "/chat/completions",
            json={
                "model": MODEL,
                "stream": True,
                "messages": [{"role": "user", "content": user_text}],
            },
        ) as r:
            r.raise_for_status()
            parts: list[str] = []
            async for line in r.aiter_lines():
                if not line.startswith("data: "):
                    continue
                payload = line[6:].strip()
                if payload == "[DONE]":
                    break
                import json

                chunk = json.loads(payload)
                delta = chunk["choices"][0].get("delta", {})
                if text := delta.get("content"):
                    parts.append(text)
                    print(text, end="", flush=True)
            print()
            return "".join(parts)


if __name__ == "__main__":
    q = sys.argv[1] if len(sys.argv) > 1 else "用一句话解释 OpenAI 兼容 API"
    print(asyncio.run(chat_once(q, stream="--stream" in sys.argv)))

方式 B：OpenAI SDK

python 
#!/usr/bin/env python3
"""CorpAssist 最小调用 — openai 包（base_url 指向网关）。"""
import os
from openai import OpenAI

client = OpenAI(
    api_key=os.environ["CORPASSIST_LLM_KEY"],
    base_url=os.environ["CORPASSIST_LLM_BASE_URL"].rstrip("/"),
)

resp = client.chat.completions.create(
    model=os.environ.get("CORPASSIST_LLM_MODEL", "qwen-plus"),
    messages=[{"role": "user", "content": "CorpAssist 制度问答和客服各适合流式还是非流式？"}],
    stream=False,
    max_tokens=256,
)
print(resp.choices[0].message.content)
print(resp.usage)

运行：python demos/practice-05-first-call-lab/ask_httpx.py "你的问题"；流式加 --stream。SDK 版见同目录 ask_openai_sdk.py。

Spring AI ChatClient 预览

Java 侧 CorpAssist 微服务通过 Spring AI 1.x 的 ChatClient 访问同一网关。配置 spring.ai.openai.base-url 与 API Key 后，业务代码只写 prompt，不绑死某一国内厂商 SDK。

application.yml（网关 + 模型）

yaml 
spring:
  ai:
    openai:
      api-key: ${CORPASSIST_LLM_KEY}
      base-url: ${CORPASSIST_LLM_BASE_URL:https://llm-gateway.corpassist.internal/v1}
      chat:
        options:
          model: ${CORPASSIST_LLM_MODEL:corpassist-fast}
          temperature: 0.2
          max-tokens: 512

ChatClient Bean 与调用

java 
package com.corpassist.llm;

import org.springframework.ai.chat.client.ChatClient;
import org.springframework.context.annotation.Bean;
import org.springframework.context.annotation.Configuration;
import org.springframework.web.bind.annotation.GetMapping;
import org.springframework.web.bind.annotation.RequestParam;
import org.springframework.web.bind.annotation.RestController;
import reactor.core.publisher.Flux;

@Configuration
class CorpAssistLlmConfig {

  @Bean
  ChatClient corpAssistChatClient(ChatClient.Builder builder) {
    return builder
        .defaultSystem("你是 CorpAssist 企业助手，回答须简洁并提示信息来源类型。")
        .build();
  }
}

@RestController
class CorpAssistChatSmokeController {

  private final ChatClient chatClient;

  CorpAssistChatSmokeController(ChatClient corpAssistChatClient) {
    this.chatClient = corpAssistChatClient;
  }

  /** 非流式 — 对齐 curl stream:false */
  @GetMapping("/internal/llm/smoke")
  String smoke(@RequestParam(defaultValue = "ping") String q) {
    return chatClient.prompt().user(q).call().content();
  }

  /** 流式 — 对齐 curl stream:true + SSE */
  @GetMapping(value = "/internal/llm/smoke-stream", produces = "text/event-stream")
  Flux<String> smokeStream(@RequestParam String q) {
    return chatClient.prompt().user(q).stream().content();
  }
}

本地验证：启动 Spring Boot 后访问 /internal/llm/smoke?q=年假几天；流式端点用浏览器或 curl -N 观察逐段输出。生产环境此类接口应加内网鉴权，勿对公网暴露。

双栈对照

Python 编排与 Spring AI 微服务在 CorpAssist 中长期并存。首次打通后，用下表统一环境变量 / 配置项，避免「curl 能通、Java 报 401」类问题。

首次调用 · 请求路径（非流式）
sequenceDiagram
  participant Dev as 开发者脚本/curl
  participant Py as Python httpx 或 OpenAI SDK
  participant Java as Spring AI ChatClient
  participant GW as llm-gateway OpenAI 兼容
  participant LLM as 模型厂商或私有化集群

  Dev->>GW: POST /v1/chat/completions JSON
  Py->>GW: POST /chat/completions 同契约
  Java->>GW: Spring AI 转 chat/completions
  GW->>LLM: 路由 model + 鉴权
  LLM-->>GW: choices + usage
  GW-->>Dev: JSON 完整响应
  GW-->>Py: JSON 完整响应
  GW-->>Java: 映射为 ChatResponse

三端共享 base_url、api-key、model；差异在客户端如何解析 流式 chunk 与记录 Token usage。

维度curlPythonSpring AI
端点 $BASE/chat/completions base_url + /chat/completions spring.ai.openai.base-url
鉴权 Authorization: Bearer 同上 / SDK api_key spring.ai.openai.api-key
非流式 "stream": false stream=False / .create(...) .call().content()
流式 curl -N + SSE 行 client.stream 解析 data: .stream().content() → Flux
Token 用量 jq .usage resp.usage / JSON usage ChatResponse.getMetadata().getUsage()
典型用途 冒烟、on-call RAG 流水线、评测、脚本 BFF 邻域服务、同步 API
联调反模式
  • curl 用公网 DashScope URL，Java 仍指向旧内网网关
  • Python 未设读超时，流式长回答挂死线程池
  • 只看 content 不看 usage，无法核对 Token 账单
推荐
  • 三端共用 Demo .env 或 K8s Secret 同一套变量名
  • 先非流式打通，再开 streaming 验证 SSE
  • 请求带 X-Request-Id，与网关日志关联
与进阶章节衔接

文本对话跑通后，CorpAssist 附件问答会引入图文 / PDF 等多模态输入，仍可能走同一网关的不同 model 或专用端点。预习见 多模态应用入门；费用与窗口仍回到 Token、上下文窗口与费用。

本章结论

首次打通 = 同一 OpenAI 兼容契约，三种客户端各跑通一次。

  • curl：最快验证 Key、model、usage；流式务必 -N。
  • Python：httpx 与 curl 同构；OpenAI SDK 改 base_url 即可换厂商。
  • Spring AI：ChatClient + spring.ai.openai.* 指向网关，call / stream 对应非流式与 SSE。
  • 双栈：统一环境变量与请求 ID；先非流式再流式；多模态见进阶章。

动手练习

操作步骤

  1. 进入 demos/practice-05-first-call-lab/，复制 .env.example 为 .env 并填入真实 Key 与 BASE_URL。
  2. 执行 chat-non-stream.sh（或 PowerShell 版），确认响应含 choices 与 usage。
  3. 执行 chat-stream.sh，肉眼看至少 3 个 SSE data: 与 [DONE]。
  4. 运行 ask_httpx.py 与 ask_openai_sdk.py，对比 stderr 中的 usage 是否接近。
  5. 填写 dual-stack-compare.md 中你负责栈的一列；Java 同学补充 application.example.yml 与 Controller 片段。

判断练习

  1. curl 非流式已成功，但 Spring Boot 报 401。最应先核对哪两项？

    参考答案

    spring.ai.openai.api-key 是否与 curl 的 Bearer 一致；base-url 是否多了/少了 /v1 或与 curl 的 CORPASSIST_LLM_BASE_URL 完全一致（勿一个走公网一个走内网）。

  2. Python 流式脚本已打印半段回答后进程被 kill，厂商是否一定未计费？

    参考答案

    不一定。已生成的输出 Token 通常仍计费；勿盲目重试同一轮。应新开会话或显式续写策略，见 API 契约 流式中断说明。
