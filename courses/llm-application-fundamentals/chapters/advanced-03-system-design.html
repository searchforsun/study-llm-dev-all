假设输入

参数取值说明
员工数5,000潜在用户池
DAU 比例20%≈ 1,000 日活
人均日提问8 次制度/采购高频查阅
峰值系数10× 均值上班前 30 分钟集中

推导（口算友好）

  1. 日均请求：1,000 × 8 = 8,000 次/天。
  2. 均值 QPS：8,000 / 86,400 ≈ 0.09（很低——说明瓶颈在峰值与单次延迟，非均值吞吐）。
  3. 峰值 QPS：若 20% 流量落在 30 分钟：8,000 × 0.2 / 1,800 ≈ 0.9；再乘突发系数 ≈ 5～10 QPS。面试可保守报 50 QPS 作 3 年规划，便于讨论水平扩展。
  4. 日 Token：8,000 × (4,500 + 600) ≈ 4.08×10⁷ Token/天。
  5. 月 Token：≈ 12.2×10⁸ Token/月（按 30 天）。按 input ¥0.002/1K、output ¥0.006/1K 粗算（示例单价），月费数量级 ¥7～9 万——与 NFR 预算对齐，需网关按部门限流。
  6. 向量存储：2×10⁶ × 1,024 × 4 byte ≈ 8 GB 向量本体 + 元数据与 HNSW 索引开销（常按 1.5～2× 估）→ 规划 16～20 GB 内存型或 SSD 向量库分片。
  7. Embedding 离线：每周增量 5 万 chunk × 512 token → 一次性 2.56×10⁷ Token 索引成本，应走异步队列而非在线路径。

# CorpAssist S1 容量粗算（面试白板可手算，此处为可运行核对脚本）
# 用法：python estimate_s1_capacity.py

DAU = 1000
QUESTIONS_PER_USER = 8
PEAK_BURST = 10          # 相对均值 QPS 的突发倍数
INPUT_TOKENS = 4500
OUTPUT_TOKENS = 600
CHUNKS = 2_000_000
DIM = 1024
PRICE_IN_PER_1K = 0.002  # 示例单价，元
PRICE_OUT_PER_1K = 0.006

daily_requests = DAU * QUESTIONS_PER_USER
avg_qps = daily_requests / 86400
peak_qps = avg_qps * PEAK_BURST
daily_tokens = daily_requests * (INPUT_TOKENS + OUTPUT_TOKENS)
monthly_tokens = daily_tokens * 30
monthly_cost_yuan = (
    daily_requests * 30 * INPUT_TOKENS / 1000 * PRICE_IN_PER_1K
    + daily_requests * 30 * OUTPUT_TOKENS / 1000 * PRICE_OUT_PER_1K
)
vector_gb = CHUNKS * DIM * 4 / (1024**3)

print(f"daily_requests={daily_requests:,}")
print(f"avg_qps={avg_qps:.3f}, peak_qps~={peak_qps:.1f} (burst x{PEAK_BURST})")
print(f"monthly_tokens={monthly_tokens:.2e}")
print(f"monthly_cost_yuan~={monthly_cost_yuan:,.0f}")
print(f"vector_raw_gb={vector_gb:.1f} (+ index overhead plan 16-20GB)")

一次在线问答路径（与拓扑对照）

  1. 钉钉 → Ingress → BFF：校验 OAuth2，注入 tenant_id、trace_id。
  2. BFF → orchestrator：携带 session_id 与 user 问题。
  3. orchestrator → rag-service（Python）：混合检索 200ms 级，返回 6 条 chunk_id。
  4. orchestrator 拼装 messages → gateway POST /v1/chat/completions，stream: true。
  5. Gateway → 模型 API：流式 chunk 经 BFF 写 SSE；末包 usage 写 ClickHouse。

判断练习

  1. 面试官说「我们 3 万人、峰值 200 QPS，仍用单实例 rag-service 可以吗？」

    参考答案

    不可以作为生产方案。200 QPS ×（检索 + 编排等待）远超单实例并发；需 rag-service HPA、向量库分片、gateway RPM 配额与 orchestrator 水平扩展。面试中应重算 QPS 并给出副本数数量级。

  2. 产品希望「答案不必引用，只要快」——五段式框架里哪些段落必须调整？

    参考答案

    需求：FR-1 从强制引用改为可选；方案：可简化 Citation Prompt 与评测流水线；权衡：质量 vs 延迟权重变化；风险：幻觉合规上升；指标：从引用准确率转向事实一致率与拒答率。说明需求变更会 cascade 到全架构。
