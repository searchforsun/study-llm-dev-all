已完成面试 TOP5 应用场景与 CorpAssist 模块

约 35 分钟 · 阶段：应用架构与能力拼图 · 能力：把面试 TOP5 场景映射到 CorpAssist 模块与后续课程路线图

为什么要学本章

JD 与系统设计面试很少问「Transformer 有几层」，而常问「设计企业知识库问答」「智能客服如何降级」「Agent 与对话应用有何不同」。本章把 outline-specs.json 中 interviewScenarios 的 TOP5 与 CorpAssist S1～S5 对齐，并给出场景落地课（scenario-enterprise-*）与能力拼图课的分工，避免「只会调 API、说不清项目边界」。

学完你能
  • 用一张总表说明 S1～S5 各自对应哪门 scenario-enterprise-* 课与 2～3 门能力课
  • 针对任一场景，口述 1 道高频面试题 + 核心技术栈关键词
  • 在双轨学习路线图上标出「下一阶段应优先补的课」
本章先记住 3 件事
  • 场景课（scenario-enterprise-*）负责端到端业务交付与面试答辩；能力课负责 RAG / Prompt / Agent 等可复用积木。
  • S1 知识库 几乎必考 — 首选 RAG，而非一上来微调。
  • S2 客服 与 S3 Agent 都涉及工具，但 S2 偏会话与话术，S3 偏规划与多步自动化。

S1 企业 RAG 知识库

CorpAssist · S1 制度 / 手册 / 标书问答 员工问「年假能否顺延」「采购流程第几步要审批」——答案必须可引用制度条文，且知识库每周增量更新。

面试出现概率约 100%（见 REFERENCE.md · 面试 TOP5 与 interviewScenarios.scenarios[0]）。CorpAssist 将 S1 作为其他模块的检索上下文供给方。

维度要点
核心能力 RAG：分块 → Embedding → 混合检索 → Rerank → 带引用生成
企业落地课 企业落地 · RAG 知识库问答（scenario-enterprise-rag-kb）
能力拼图课 rag-system-engineering、retrieval-vector-platform、knowledge-lifecycle-governance、llm-evaluation-quality、production-prompt-engineering
面试口述锚点 RAG vs 微调选型；分块大小；检索不相关如何排障；召回率 / MRR 如何评
示例面试题（摘自 outline-spec）
  1. 设计企业内部知识库问答系统的整体架构？
  2. 分块大小怎么选？语义分块为何更好？
  3. 如何解决检索不相关？用过哪些 Rerank？
先想 10 秒：业务要求「每条回答必须带制度出处」，应主要改哪一层？

数据层检索 + 生成层引用约束（Citation Prompt）+ 评测集校验引用字段；不是单纯调高 temperature。

S2 智能客服

CorpAssist · S2 工单 / 多轮客服 客户问订单物流、要求退款；系统需识别意图、查订单工具、在答不上时转人工，且话术符合公司政策。

概率约 90%。与 S1 的差异：强调多轮上下文、意图路由与服务质量指标，而非只做静态文档问答。

维度要点
核心能力 提示词模板、会话状态、工具调用、流式首字延迟
企业落地课 企业落地 · 智能客服系统（scenario-enterprise-customer-service）
能力拼图课 context-memory-engineering、production-prompt-engineering、agent-orchestration-engineering、spring-ai-engineering、llm-application-backend、llm-evaluation-quality、security-compliance-engineering
与 S1 协作 政策类问题常从 S1 检索片段注入 S2 的 Prompt，避免整库 PDF 进上下文
示例面试题
  1. 设计电商智能客服需要哪些核心模块？
  2. 多轮对话如何做上下文与指代消解？
  3. 模型答不上时的降级策略？

S3 Agent 自动化

CorpAssist · S3 办公自动化 「帮我查本周审批、给经理发邮件并记一条工单」——需要规划多步、调用 ERP / 通知等工具，并受步数与权限约束。

概率约 85%。面试常考 Agent 与普通对话应用的区别：规划、工具、状态、预算。

维度要点
核心能力 ReAct / 任务分解、工具白名单、短期+长期记忆、失败重试与步数上限
企业落地课 企业落地 · Agent 任务自动化（scenario-enterprise-agent-automation）
能力拼图课 agent-orchestration-engineering、spring-ai-alibaba-engineering、context-memory-engineering、rag-system-engineering
与 S2 对比 S2 以「一轮用户意图 → 回复」为主；S3 以「多步计划 → 多次工具」为主，审计粒度更细
S2 对话 vs S3 Agent（概念对比）
flowchart LR
  subgraph s2["S2 智能客服"]
    U1[用户消息] --> IR[意图/路由]
    IR --> RAG2[可选 RAG]
    IR --> T2[工具 1～2 次]
    T2 --> R1[回复 + 转人工]

S3 的编排层通常更长生命周期，需单独配置 max-steps 与工具权限。

示例面试题
  1. Agent 与普通大模型应用的区别？
  2. 工具调用失败怎么办？
  3. 记忆系统如何设计？过载与不准怎么解？

S4 代码辅助

CorpAssist · S4 内部代码助手 基于公司内部 Spring 模板与 SQL 规范做补全与问答；代码不出内网，延迟敏感。

概率约 80%。除通用 RAG 外，强调代码库索引、长文件切片与安全扫描。

维度要点
核心能力 代码 RAG、仓库范围、补全 Prompt、低延迟推理
企业落地课 企业落地 · 代码助手（scenario-enterprise-code-assistant）
能力拼图课 context-memory-engineering、rag-system-engineering、python-engineering-for-llm、llm-serving-for-applications、observability-reliability-ops
示例面试题
  1. 如何理解整个代码库上下文？
  2. 推理延迟高如何优化？
  3. 如何检测安全漏洞与 bug？

S5 内容生成

CorpAssist · S5 内容工坊 运营生成营销话术、周报、邮件；需风格模板、审核流水线，并防止违规内容出网。

概率约 75%。与 S1 的「准确性」不同，S5 更关注风格、结构、合规与批量效率。

维度要点
核心能力 风格 Prompt、长文规划、多模态素材、生成+审核
企业落地课 企业落地 · 内容生成工坊（scenario-enterprise-content-studio）
能力拼图课 production-prompt-engineering、multimodel-routing-multimodal、security-compliance-engineering、llm-evaluation-quality
示例面试题
  1. 如何生成符合特定风格的内容？
  2. 如何防止有害违规内容？
  3. 如何评估生成内容质量？

与 LLM 应用分层架构 的衔接

五大场景共用同一套接入层 / 编排层 / 数据层：S1 偏重数据层；S2/S5 偏重编排层 Prompt；S3 偏重编排层 Agent 循环；S4 横跨数据层（代码索引）与接入层（低延迟）。上一章若已学分层，本章把分层「落到具体 JD 场景」。

自检：你计划先攻克 S1 还是 S3？需要提前补哪 2 门能力课？

多数应用岗先 S1（RAG 全链路 + 评测）；若 JD 强调自动化办公，则先补 agent-orchestration-engineering 再进 S3 场景课。

本章结论

  • S1 知识库：面试最高频；scenario-enterprise-rag-kb + RAG 能力课；首选检索增强而非微调。
  • S2 客服：多轮、意图、工具与转人工；Prompt 与记忆工程为主，常消费 S1 检索结果。
  • S3 Agent：多步规划与工具链；与 S2 相比状态与审计更复杂，对应 scenario-enterprise-agent-automation。
  • S4 代码：代码库 RAG + 延迟与安全；scenario-enterprise-code-assistant 与 serving 课衔接。
  • S5 内容：风格模板与审核流水线；production-prompt-engineering 与多模态课为主。
  • 路线图：场景课负责端到端交付；能力课按阶段二→三→三B 递进；真源在 interviewScenarios。

判断练习

  1. 排障顺序：CorpAssist 制度问答命中率低。同事依次提出 ① 立刻 LoRA 微调 ② 检查分块与混合检索 ③ 把 200 页 PDF 每次全塞进 Prompt ④ 上 Rerank。请按推荐处理顺序排列（写序号）。

    参考答案

    ② → ④ → 停用 ③ → 最后评估 ①。先修数据层检索与分块；再加 Rerank；禁止全量 PDF 进 Prompt；微调需标注与合规，不应作为第一步。

  2. 权衡：JD 要求「办公流程自动化（审批+邮件+工单）」与「电商智能客服」二选一作为首个 CorpAssist 子项目，应优先映射 S2 还是 S3？对应哪门 scenario-enterprise-* 课？

    参考答案

    S3 → scenario-enterprise-agent-automation。办公流属于多步工具编排；电商客服才是 S2 / scenario-enterprise-customer-service。
