为什么要学本章

CorpAssist 不只有 HR 政策：还有 IT 手册、财务制度等多个 知识库。若所有问题都打向默认 collection，会串库回答或越权引用。本章在 practice-03 混合检索之上，用 意图路由 选库，并在检索层施加 acl filter 与 租户隔离。

学完你能
  • 设计 kb-routing.yaml 映射意图 → collection
  • 在 Python/Spring 检索前应用 部门范围 与 tenant filter
  • 说明 router chain 与单一 RAG 链的关系
  • 用 curl 带不同 X-Tenant-Id / 问题类型验证不串库
本章先记住 3 件事
  • 先路由后检索：选错库比检索算法差更致命。
  • 权限在检索层：Prompt 禁止越权不能替代 metadata filter。
  • 配置真源：新增库只改 yaml，不改 Java/Python 分支地狱。

意图路由：选 HR 还是 IT

CorpAssist · 企业助手入口 员工问「VPN 连不上」应路由 it 库；问「产假天数」应路由 hr 库。路由可规则（关键词）或小模型分类器，结果写入 kb 字段供检索 filter 使用。

Intent router 输出 {intent, kb, confidence}。低置信度时走澄清或默认库（需产品定义）。kb routing 配置应与混合检索、RAG 链解耦，便于运营增库。

多库 RAG 路由
flowchart TB
  Q[用户问题] --> IR[Intent Router]
  IR -->|hr| R1[HR Retriever]
  IR -->|it| R2[IT Retriever]
  R1 --> ACL[ACL filter]
  R2 --> ACL
  ACL --> RAG[RAG 生成]

每个 Retriever 指向不同 collection，但共用同一套 ACL 规则引擎。

kb-routing.yaml（Demo 真源）

yaml 
# demos/practice-04-multi-kb-lab/kb-routing.yaml
knowledge_bases:
  hr:
    collection: corpassist-hr-v2
    intents: [leave, benefit, policy]
    keywords: [年假, 产假, 人事]
  it:
    collection: corpassist-it-v1
    intents: [vpn, account, device]
    keywords: [VPN, 密码, 工单]
  finance:
    collection: corpassist-fin-v1
    intents: [expense, invoice]
    keywords: [报销, 发票]

default_kb: hr
router:
  mode: keyword_first   # keyword_first | classifier
  min_confidence: 0.6

权限：acl filter 在检索层

ACL filter 将用户 JWT 中的 tenant_id、dept_ids、roles 转为向量库/倒排 filter。与 幻觉治理 的拒答互补：检不到优于检到无权限文档。

Python：路由 + ACL 片段

python 
def build_filter(user, kb: str) -> dict:
    return {
        "tenant_id": user.tenant_id,
        "kb": kb,
        "dept": {"$in": user.dept_ids},
    }

def route_and_ask(question: str, user) -> dict:
    kb = route_kb(question)  # 读 kb-routing.yaml
    retriever = get_retriever(cfg.knowledge_bases[kb].collection)
    docs = retriever.invoke(question, filter=build_filter(user, kb))
    return rag_with_docs(question, docs)

Spring：filterExpression 与请求头

java 
// 从 SecurityContext 取 tenantId、deptIds

    "tenant_id == '%s' && kb == '%s' && dept in [%s]",

权限字段来源检索层表达
tenant_id JWT / X-Tenant-Id 等值 filter
kb 意图路由结果 等值 filter，防串库
dept HR 主数据同步 in 列表 filter
roles IAM 敏感库额外门槛（如 finance）
FAQ：路由错了怎么办？

产品层：低置信度澄清；技术层：记录 routed_kb 与 sources 供评测；运营层：在 yaml 补关键词。不要用「检索全库」兜底。

租户：multi-tenant 隔离

Multi-tenant 要求：索引 metadata 必含 tenant_id；禁止跨租户默认库。Tenant isolation 需在 ingest 与 query 双侧校验——CorpAssist 为集团多法人场景，corp-a 与 corp-b 的制度库物理或逻辑隔离。

CorpAssist · 集团多法人 同一套 RAG 服务承载多个子公司租户；查询时必须从网关透传 tenant_id，检索 filter 强制匹配，禁止用「默认租户」吞掉缺失头。

ingest 侧：租户写入 metadata

python 
def index_chunk(chunk: dict, tenant_id: str, kb: str):
    meta = {
        **chunk["meta"],
        "tenant_id": tenant_id,
        "kb": kb,
        "doc_id": chunk["doc_id"],
    }
    vector_store.upsert(ids=[chunk["doc_id"]], metadatas=[meta], texts=[chunk["text"]])

Spring：拒绝缺失租户

java 
@PostMapping("/ask")
AskResponse ask(@RequestBody AskRequest req,
                @RequestHeader("X-Tenant-Id") String tenantId) {
  if (tenantId == null || tenantId.isBlank()) {
    throw new ResponseStatusException(HttpStatus.BAD_REQUEST, "tenant required");
  }
  // filter 必须含 tenant_id，禁止 fallback 到全库
}
租户泄漏风险
  • 未传 X-Tenant-Id 仍返回 corp-a 文档
  • ingest 漏写 tenant，查询无法过滤
  • 默认库合并多租户 chunk
隔离标准
  • 请求无 tenant → 400，不检索
  • metadata 缺 tenant → 索引 job 失败
  • 评测集含 acl_context.tenant（practice-05）
先想 10 秒：租户隔离放在 Prompt 里写「请勿泄露其他公司数据」够吗？

不够。LLM 无法阻止已检入的无权限片段；必须在 Retriever / SearchRequest 的 filter 层阻断，并审计越权检索（advanced-05）。

配置：增库不改代码

新增「法务库」时，理想 diff 仅为 kb-routing.yaml + 新 collection 的 ingest job，而非改 5 个 Java 类。Spring 用 @ConfigurationProperties 绑定；Python 启动时 load yaml。kb config 是真源，代码只实现「读配置 → 选 Retriever → 套 ACL」。

Python：加载路由配置

python 
import yaml
from pathlib import Path

def load_routing(path: str = "kb-routing.yaml") -> dict:
    return yaml.safe_load(Path(path).read_text())

def route_kb(question: str, cfg: dict) -> str:
    if cfg["router"]["mode"] == "keyword_first":
        for kb, spec in cfg["knowledge_bases"].items():
            if any(kw in question for kw in spec.get("keywords", [])):
                return kb
    return cfg["default_kb"]

Spring：ConfigurationProperties 绑定

java 
@ConfigurationProperties(prefix = "corpassist.rag")
public record RagRoutingProperties(
    String routingConfig,
    boolean enforceAcl,
    Map<String, KnowledgeBaseSpec> knowledgeBases
) {}
yaml

FAQ：新增法务库最少改哪些文件？

① kb-routing.yaml 增加 legal 条目；② Python ingest 注册新 collection；③ 混合检索 hybrid-config.json 若按库分配置则加一份；④ 黄金集加 5～10 条法务问法（practice-05）。Java Controller 不应新增 if (legal) 分支。

本章结论

  • 路由：kb-routing.yaml 定义库、意图、关键词；先路由后检索。
  • 权限：acl filter 在 Retriever/SearchRequest，含 tenant + dept + kb。
  • 租户：metadata 与请求头一致，禁止跨租户 fallback 全库。
  • 配置：增库以 yaml 为真源，双栈共享路由文件。

判断练习

  1. （排障）用户有 hr 角色，却检索到 finance 库文档。列出 yaml、filter、ingest metadata 各 1 个排查点。

    参考答案

    yaml：关键词/意图是否误匹配 finance；filter：是否缺少 kb 条件；ingest：chunk 是否错标 kb 或 tenant。

  2. （排障）路由正确但 sources 为空。ACL 与混合检索配置中优先检查哪 2 项？

    参考答案

    filter 中 dept/tenant 是否过严；collection 名是否与 kb-routing 中该库一致（非 default 库名）。
