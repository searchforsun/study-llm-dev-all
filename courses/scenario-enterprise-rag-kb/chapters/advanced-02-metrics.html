本章先记住 3 件事
  • 效果叙事要有数字和归因：「MRR 提升 25pp」不如「BM25 混合 +12pp，Rerank +10pp，分块调优 +3pp」。
  • 成本和 SLA 必须一起讲：只说效果不说成本的方案无法落地。
  • 交付清单是答辩的 checkmate：一条一条列出来验收，比泛泛而谈有力得多。

优化叙事 Python 报告生成

python 
# optimization_report.py — 优化报告生成
class OptimizationReport:
    """生成优化旅程报告"""

    def __init__(self):
        self.milestones = []

    def add_milestone(self, name: str, mrr: float, recall: float,
                      latency: int, cost: str, reason: str):
        self.milestones.append({
            "name": name, "mrr": mrr, "recall": recall,
            "latency": latency, "cost": cost, "reason": reason,
        })

    def generate(self) -> str:
        """生成 Markdown 报告"""
        lines = ["# CorpAssist 优化旅程报告\n"]
        lines.append(f"| 阶段 | MRR | Recall@5 | P95 | 成本 | 归因 |")
        lines.append(f"|------|-----|----------|-----|------|------|")

        for m in self.milestones:
            lines.append(
                f"| {m['name']} | {m['mrr']:.2f} | {m['recall']:.2f} | "
                f"{m['latency']}ms | {m['cost']} | {m['reason']} |"
            )

        # 计算总提升
        first = self.milestones[0]
        last = self.milestones[-1]
        lines.append(f"\n## 总提升\n")
        lines.append(f"- MRR: {first['mrr']:.2f} → {last['mrr']:.2f} (+{(last['mrr']-first['mrr'])*100:.0f}pp)")
        lines.append(f"- Recall@5: {first['recall']:.2f} → {last['recall']:.2f}")
        lines.append(f"- P95 延迟: {first['latency']}ms → {last['latency']}ms")
        lines.append(f"- 总成本: 增量约 ¥{self._total_cost()}/月")

        return "\n".join(lines)

    def _total_cost(self) -> int:
        costs = [int(m['cost'].replace('+', '').replace('¥', '').replace('/月', '').split()[0])
                 for m in self.milestones if m['cost'] != '0' and m['cost'] != '—']
        return sum(costs)


# 生成 CorpAssist 优化报告
report = OptimizationReport()
report.add_milestone("基线", 0.60, 0.71, 120, "—", "纯向量检索")
report.add_milestone("+BM25", 0.72, 0.82, 200, "+¥500/月", "BM25 混合")
report.add_milestone("+Rerank", 0.79, 0.87, 450, "+¥800/月", "BGE-Reranker")
report.add_milestone("分块调优", 0.82, 0.89, 460, "+5% token", "chunk=768")
report.add_milestone("+Rewrite", 0.84, 0.90, 600, "+¥200/月", "Query Rewrite")
report.add_milestone("ACL", 0.85, 0.91, 610, "0", "移除无权限噪音")
print(report.generate())

SLA 监控代码

java 
// SlaMonitor.java — SLA 监控
@Component
public class SlaMonitor {

    private final MeterRegistry registry;

    @Scheduled(fixedRate = 60000)
    public void checkSla() {
        // P95 延迟检查
        Timer timer = registry.find("rag.total.latency").timer();
        double p95 = timer.percentile(0.95);
        if (p95 > 2000) {
            alertService.sendAlert("SLA_BREACH",
                "P95 latency " + p95 + "ms exceeds 2000ms threshold");
        }

        // MRR 检查
        double mrr = registry.find("rag.mrr").gauge().value();
        if (mrr < 0.80) {
            alertService.sendAlert("SLA_BREACH",
                "MRR " + mrr + " below 0.80 threshold");
        }
    }

    @EventListener
    public void onRequest(RequestCompleteEvent event) {
        registry.counter("rag.requests.total").increment();
        registry.timer("rag.total.latency")
            .record(event.getDuration());
    }
}

常见问题（FAQ）

答辩时，评委问「成本这么高，能再降吗？」怎么回答？
分三条路径回答：1) 基础设施——将 GPU Reranker 改为 CPU 版本（MRR 降约 0.05，成本降 ¥800/月）；2) LLM 调用——用 gpt-4o-mini 替代已是最优，但可加缓存命中高频问题（预计减少 20% 调用）；3) 维护人力——0.5 人年不可压缩，但可通过文档化和自动化减少。核心观点：当前成本对应 MRR=0.85 的最佳性价比。
优化故事中 BM25 贡献最大（+12pp），是不是更应该关注关键词检索？
这说明企业知识库中精确关键词匹配（制度编号、产品型号）占比较高。是的，对于企业场景，关键词检索的贡献往往被低估。但不要忽略向量检索的长尾价值——对于语义匹配问题（「我还没休完的假怎么办」 vs 「年假余额处理」），向量检索不可或缺。两者是互补关系。
SLA 的 P95 延迟包含网络传输吗？
包含，P95 延迟是从用户发起请求到收到完整答案的全链路时间。CorpAssist 在用户侧（浏览器/App）和服务端（Spring AI）分别埋点。服务端 P95 通常比用户侧少 200-500ms（网络传输差异）。SLA 承诺的是服务端 P95。
交付清单中「Demo 演示」怎么做？
准备 5 个有代表性的问答场景：1) 精确匹配（「制度 HR-001 号文件」）；2) 语义匹配（「我还没休的年假怎么办」）；3) 跨文档关联（「社保和公积金有什么区别」）；4) 拒答场景（敏感问题）；5) 多轮对话（follow-up 问题）。每个场景展示答案和引用来源。

本章结论

  • 效果叙事要可归因：BM25 +12pp、Rerank +7pp、分块 +3pp、Rewrite +2pp、ACL +1pp，每条优化都有数字支撑。
  • 成本模型做决策参考：千人企业月均 ¥23,000（基础设施 + LLM 调用 + 维护人力），每种优化有对应的成本增量。
  • SLA 是客户信任的基础：5 项指标明确承诺，Prometheus 监控并自动告警。
  • 交付清单五维度覆盖：效果/架构/安全/成本/运维，11 个条目逐项验收。

    参考答案

    回答： 1) 我们有独立的训练集（200 条）和验证集（50 条），所有优化结果基于验证集，未在训练集上调参数；2) 我们做了 weekly 稳定性测试——连续 4 周 MRR 波动 < 0.02；3) 可以现场抽取 10 条未标注的实时查询，Demo 验证效果。
