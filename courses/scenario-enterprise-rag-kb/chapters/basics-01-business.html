合规边界三层模型：来源过滤 → 访问控制 → 使用场景限制，缺一不可。

java 
// Spring AI 中通过 FilterExpression 实现合规过滤
@Bean
public QuestionAnswerAdvisor complianceAdvisor(VectorStore vectorStore) {
    return QuestionAnswerAdvisor.builder(vectorStore)
        .searchRequest(SearchRequest.builder()
            .filterExpression("allowed_roles CONTAINS 'employee'")
            .topK(5)
            .build())
        .build();
}

合规边界检查清单

  • 来源合规：哪些文档禁止入库（薪资、未审批、过期）？
  • 角色合规：不同角色能看哪些文档？
  • 场景合规：知识库的答案用于哪些场景（内部问答 vs 客户展示）？
  • 审计合规：谁在什么时间问了什么、看到了什么？
  • 数据驻留：文档数据存储在哪个区域？是否符合 GDPR / 个人信息保护法？

ROI 量化模型

kb roi（知识库投资回报率）是企业 RAG 项目立项的核心依据。CorpAssist 推荐使用以下公式：

ROI 计算公式
text 
年节省人天 = 员工数 × 人均日检索次数 × 每次节省时间(小时) / 8

ROI = (年节省人天 × 人均日薪) / (年系统成本)

其中：
- 年系统成本 = 硬件/云资源 + LLM API 调用费 + 维护人力
- 基准命中率 = 人工检索命中率（约 40%）
- RAG 命中率目标 ≥ 85%
- 节省时间 = 人工检索耗时 × (RAG 命中率 - 基准命中率)

ROI 模型 Python 实现

python 
# roi_model.py — CorpAssist ROI 计算器
def calculate_roi(
    employee_count: int = 1000,
    daily_searches_per_employee: float = 0.5,
    manual_search_minutes: float = 15.0,
    rag_hit_rate: float = 0.85,
    manual_hit_rate: float = 0.40,
    daily_rate_per_person: float = 800.0,  # 人均日薪(元)
    annual_system_cost: float = 300000.0,  # 年系统成本(元)
) -> dict:
    """计算 CorpAssist 知识库 ROI"""
    # 每次检索节省的时间（考虑命中率差异）
    time_saved_per_search = manual_search_minutes * (rag_hit_rate - manual_hit_rate)

    # 年总检索次数
    total_annual_searches = employee_count * daily_searches_per_employee * 250  # 250 工作日

    # 年节省人天
    saved_person_days = (total_annual_searches * time_saved_per_search / 60) / 8

    # 年节省金额
    annual_savings = saved_person_days * daily_rate_per_person

    # ROI
    roi = annual_savings / annual_system_cost

    return {
        "saved_person_days": round(saved_person_days, 1),
        "annual_savings": round(annual_savings, 2),
        "annual_cost": annual_system_cost,
        "roi": round(roi, 2),
        "time_saved_per_search_min": round(time_saved_per_search, 1),
    }

if __name__ == "__main__":
    result = calculate_roi()
    print(f"年节省人天: {result['saved_person_days']}")
    print(f"年节省金额: {result['annual_savings']:.2f} 元")
    print(f"ROI: {result['roi']}x")

常见问题（FAQ）

ROI 模型中的「年系统成本」包含哪些项？
主要包含三部分：(1) 基础设施——向量数据库（如 Milvus/Pinecone）和 GPU 资源年费；(2) LLM API 调用费——按 token 计费，CorpAssist 使用 gpt-4o-mini 时约为 0.15 元/次调用；(3) 维护人力——约 0.5 人年的运维成本。三部分合计约 20-40 万/年（千人规模）。
「合规边界」和「权限控制」有什么区别？
合规边界是策略层——决定哪些文档能进知识库、能被哪些场景使用。权限控制是执行层——在检索时根据用户角色过滤结果。边界在前（文档审核时确定），控制在后（问答执行时落实）。CorpAssist 的两层设计让合规策略与工程实现解耦。详见 权限、审计与多租户。
知识库上线后，如何持续衡量 ROI？
建议建立三个核心指标：(1) 检索命中率——每次问答中 top-5 是否包含正确答案；(2) 员工采纳率——日活员工数 / 全员数；(3) 节省人天——通过问卷估算「之前找多久，现在找多久」。CorpAssist 运营看板自动汇总这三个指标，按月出报表。
CorpAssist 与传统企业搜索（如 Elasticsearch）有何不同？
传统搜索依赖关键词匹配，无法理解「下个月的年假我还没休完怎么办」这类自然语言问题。CorpAssist 的 RAG 架构通过语义向量检索+LLM 推理，理解用户真实意图并给出有引用来源的具体答案，而非返回文档列表。详见 混合检索与 Rerank。

本章结论

四条主线串起全章内容，帮你从业务视角理解 CorpAssist 知识库的价值定位。

  • 四大痛点驱动立项：信息分散、版本混乱、检索低效、合规风险——这是每一个企业知识库项目的共同起点。
  • 合规边界是系统红线：来源合规、访问合规、使用合规三层模型，让知识库在安全范围内发挥作用。
  • ROI 量化为沟通桥梁：使用 ROI 公式（年节省人天 × 人均日薪 / 年系统成本）将业务价值量化，千人规模下 ROI 可达 3x 以上。
  • 四大模块形成飞轮：文档管理 → 索引构建 → 问答服务 → 运营管理，Python 索引 + Spring 问答的双栈架构协作。
