本章先记住 3 件事
  • 拒绝比幻觉好一万倍：一句「我没有在知识库中找到相关信息」比编造答案安全得多。
  • 引用溯源是反幻觉的第一道防线：强制模型引用原文，没有引用不生成。
  • 置信度不是银弹：即使是高置信度答案，在合规敏感场景仍需人工抽检。

拒答策略设计

refusal policy 定义了 CorpAssist 在什么情况下应该拒绝回答。企业场景中，「不回答」比「错误回答」更安全。

CorpAssist · 拒答场景 员工问「CEO 的薪资是多少？」——该信息不在知识库中。员工问「这个产品多少钱」——知识库中有价格表但版本过期了。员工问「帮我写一份离职证明」——公司政策不允许 AI 生成法律文件。

三层拒答检查：检索相关度 → 引用覆盖率 → 合规敏感词。

Python 端拒答判断

python 
# refusal_check.py — CorpAssist 拒答判断
class RefusalPolicy:
    """三层拒答策略"""

    SENSITIVE_PATTERNS = [
        r"薪资|工资|salary",
        r"密码|password",
        r"身份证|id_card",
        r"银行账号|bank_account",
    ]

    def __init__(self, score_threshold=0.5, citation_threshold=0.8):
        self.score_threshold = score_threshold
        self.citation_threshold = citation_threshold

    def should_refuse(self, query: str, retrieval_scores: list,
                      citation_count: int, total_sentences: int) -> tuple:
        """返回 (是否拒答, 原因)"""

        # 1. 检索相关度检查
        max_score = max(retrieval_scores) if retrieval_scores else 0
        if max_score < self.score_threshold:
            return True, "NO_RELEVANT_INFO"

        # 2. 引用覆盖率检查
        coverage = citation_count / total_sentences if total_sentences > 0 else 0
        if coverage < self.citation_threshold:
            return True, "INSUFFICIENT_CITATIONS"

        # 3. 合规敏感词检查
        import re
        for pattern in self.SENSITIVE_PATTERNS:
            if re.search(pattern, query, re.IGNORECASE):
                return True, "SENSITIVE_QUERY"

        return False, ""

    def get_refuse_message(self, reason: str) -> str:
        messages = {
            "NO_RELEVANT_INFO": "抱歉，我在知识库中未找到与您问题相关的信息。请尝试换个问法或联系知识库管理员。",
            "INSUFFICIENT_CITATIONS": "我找到了一些可能相关的信息，但不足以形成完整答案。请将问题描述得更具体一些。",
            "SENSITIVE_QUERY": "该问题涉及个人敏感信息，不在 CorpAssist 知识库的服务范围内。",
        }
        return messages.get(reason, "抱歉，暂时无法回答该问题。")

Spring AI 拒答拦截

java 
// RefusalAdvisor.java — Spring AI 拒答 Advisor
@Component
public class RefusalAdvisor implements Advisor {

    @Override
    public AdvisedResponse advise(AdvisedRequest request) {
        List docs = request.retrievedDocuments();

        // 检索相关度检查
        if (docs == null || docs.isEmpty()) {
            return new AdvisedResponse(
                "抱歉，我在知识库中未找到相关信息。", request.params());
        }

        double maxScore = docs.stream()
            .mapToDouble(d -> d.getMetadata().get("score") instanceof Number
                ? ((Number) d.getMetadata().get("score")).doubleValue() : 0)
            .max().orElse(0);

        if (maxScore < 0.5) {
            return new AdvisedResponse(
                "抱歉，找到的信息相关度不足。", request.params());
        }

        // 合规敏感词检查
        String query = request.userText();
        List sensitiveWords = List.of("薪资", "密码", "身份证");
        for (String word : sensitiveWords) {
            if (query.contains(word)) {
                return new AdvisedResponse(
                    "该问题涉及敏感信息，不在服务范围内。", request.params());
            }
        }

        return request.next(); // 继续 Advisor 链
    }
}

强制引用与置信度评分

citation required 要求每个答案必须包含至少一个引用来源。confidence score 综合检索分数和 LLM 自评，给出答案可信度。

python 
# confidence_scorer.py — 置信度评分
class ConfidenceScorer:
    """综合检索分数和 LLM 自评计算置信度"""

    def __init__(self):
        self.retrieval_weight = 0.6
        self.llm_weight = 0.4

    def score(self, retrieval_scores: list, llm_confidence: float) -> dict:
        """
        计算置信度
        Args:
            retrieval_scores: 各 chunk 的检索相似度得分
            llm_confidence: LLM 自评置信度 (0-1)
        Returns:
            {"score": float, "level": str, "action": str}
        """
        avg_retrieval = sum(retrieval_scores) / len(retrieval_scores) if retrieval_scores else 0
        combined = self.retrieval_weight * avg_retrieval + self.llm_weight * llm_confidence

        if combined >= 0.8:
            return {"score": combined, "level": "high", "action": "direct_answer"}
        elif combined >= 0.6:
            return {"score": combined, "level": "medium", "action": "low_confidence_review"}
        elif combined >= 0.4:
            return {"score": combined, "level": "low", "action": "human_review"}
        else:
            return {"score": combined, "level": "very_low", "action": "refuse"}

# 置信度调优
scorer = ConfidenceScorer()
result = scorer.score([0.85, 0.72, 0.68], 0.90)
print(f"置信度: {result['score']:.2f} ({result['level']}) → {result['action']}")
# 输出: 置信度: 0.81 (high) → direct_answer

// ConfidenceFilter.java — 置信度过滤
@Component
public class ConfidenceFilter {

    public AnswerResponse filter(AnswerResponse response) {
        double score = response.getConfidenceScore();
        if (score >= 0.8) {
            return response; // 直接返回
        } else if (score >= 0.5) {
            // 附加低置信度提示
            return response.withWarning("本答案置信度较低，建议人工核实。");
        } else {
            // 转入人工复核队列
            humanReviewQueue.enqueue(response);
            return AnswerResponse.refuse(
                "我找到了一些可能相关的信息，但无法确认准确性。已将问题转交知识库管理员。");
        }
    }
}

人工复核队列

human review queue 是 CorpAssist 的兜底机制。当置信度低于阈值时，答案不直接返回给用户，而是进入人工审核队列。

人工复核工作流
flowchart LR
    Q[低置信度答案] --> A[进入复核队列\n状态: pending]
    A --> B{管理员审核}
    B -->|通过| C[标记 verified\n发送给用户]
    B -->|拒绝| D[标记 rejected\n通知用户未找到]
    B -->|修改| E[管理员编辑答案\n标记 edited]
    E --> C
    C --> F[加入 goodcase 集]
    D --> G[更新拒答策略\n防止类似误判]

复核队列让管理员审批评为 low confidence 的答案，审核过的 good case 回馈到评测集。

python 
# review_queue.py — 人工复核队列
from datetime import datetime
from typing import Optional

class ReviewItem:
    def __init__(self, query: str, answer: str,
                 retrieval_scores: list, confidence: float):
        self.id = f"REV-{datetime.now().strftime('%Y%m%d%H%M%S')}"
        self.query = query
        self.answer = answer
        self.retrieval_scores = retrieval_scores
        self.confidence = confidence
        self.status = "pending"  # pending/reviewed/approved/rejected
        self.reviewer = None
        self.review_note = ""
        self.created_at = datetime.now()

class ReviewQueue:
    """人工复核队列管理"""

    def __init__(self):
        self.items = []

    def enqueue(self, item: ReviewItem):
        self.items.append(item)
        self._notify_admin(item)

    def review(self, item_id: str, reviewer: str,
               action: str, note: str = "") -> bool:
        for item in self.items:
            if item.id == item_id and item.status == "pending":
                item.status = action  # approved/rejected
                item.reviewer = reviewer
                item.review_note = note
                return True
        return False

    def get_pending(self) -> list:
        return [i for i in self.items if i.status == "pending"]

    def _notify_admin(self, item: ReviewItem):
        """发送审核通知（钉钉/邮件）"""
        print(f"[NOTIFY] 新复核项 {item.id}: {item.query[:50]}...")

常见问题（FAQ）

拒答率太高（>30%）怎么办？
拒答率高通常是检索质量或阈值设置问题。排查方向：1) 降低 score_threshold（从 0.5 调至 0.4 观察变化）；2) 检查索引覆盖率——是否有些文档没有被索引；3) 分析拒答原因分布——如果是「NO_RELEVANT_INFO」多，说明检索召回不够；如果是「SENSITIVE_QUERY」多，说明敏感词列表过严。
引用强制是否会影响回答的自然度？
会影响，但这是企业场景必须接受的成本。CorpAssist 的做法是：在答案末尾集中展示引用来源（而非在句中标引用编号），保持正文自然。「后置引用」方式经用户测试满意率从 62% 提升至 89%。
人工复核队列的 SLA 怎么定？
CorpAssist 建议：普通问题 4 小时内审核完毕，紧急问题（打标 urgent）30 分钟内审核。如果队列积压超过 50 条，自动告警知识库管理员。审核后的优质答案（good case）每周汇总补充到评测集和调优数据中。
LLM 自评置信度可靠吗？
不完全可靠。LLM 对自己生成的答案倾向于高置信度（约 0.3 的偏差）。CorpAssist 将 LLM 自评权重设为 0.4，检索分数权重设为 0.6，以检索证据为主。同时增加「反方验证」：让 LLM 从检索文档中找不支持答案的证据，降低自评偏差。

本章结论

  • 三层拒答策略保底线：检索相关度 → 引用覆盖率 → 合规敏感词，任何一层不通过即拒答。
  • 强制引用反幻觉：引用覆盖率达 80% 以上才允许生成答案，答案中必须有可核验的引用。
  • 置信度驱动分级响应：high→直接回答，medium→加警告，low→人工复核，very_low→拒答。
  • 人工复核是兜底机制：审核过的 good case 反馈到评测集和拒答策略优化，形成治理闭环。

判断练习

  1. 拒答后用户投诉「知识库什么都不知道」，业务方要求降低拒答率。你如何平衡答和拒？

    参考答案

    三管齐下： 1) 优化拒答消息——从「我找不到」改成「我找到了几条相关信息但不完全匹配，您是否想了解这些？」；2) 添加「探索模式」——当拒答时，显示相关但匹配不足的文档摘要给用户；3) 后台记录拒答原因，每周分析 TOP 拒答 query，补充文档或调优检索。

  2. 有一个低置信度答案被管理员核准后发布，后来发现答案是错的。怎么从流程上防止再次发生？

    参考答案

    1) 增加「双人审核」——低置信度答案需要两名管理员分别核准；2) 增加审核质量抽查——每月随机抽 10% 的已审核答案做二次复核；3) 建立「答案召回」能力——用户可以对答案点踩，点踩后自动重新进入审核队列。
