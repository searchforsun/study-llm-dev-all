可自动化的办公任务分类

办公自动化是CorpAssist S3核心场景。按复杂度分四层：信息查询(45%)、数据录入(30%)、流程审批(15%)、跨系统编排(10%)。

信息查询：RAG方案，Agent检索向量知识库，LLM生成回答附引用。数据录入：结构化输出JSON Schema，API写入业务系统。流程审批：StateGraph编排多步骤状态流转。跨系统编排：Plan-and-Execute动态规划执行路径。

数据录入填表单结构化输出JSON Schema+API
流程审批请假审批Multi-AgentStateGraph+interrupt

CorpAssist请假场景Agent查余额->填单->提交->通知。3分钟vs15分钟，提升80%效率。
资金操作为什么不能全自动？

合规要求人工复核。LLM概率输出不可100%准确。LangGraph interrupt实现暂停->审批->恢复。

四类任务的实施顺序？

从低到高：信息查询(见效快)->数据录入(价值高)->流程审批(影响大)->跨系统编排(技术难)。

人机协同的三层边界

CorpAssist遵循分层授权。自动化等级由风险系数、影响范围、审计要求决定。执行成本越低越可自动，影响范围越大越需审批。

低风险(检索)：全自动，错误只影响单用户。中风险(创建)：Agent生成草稿，用户确认。高风险(资金/权限)：Agent准备材料，有权限人员执行。

S3模块全景

S3由三大子系统构成：编排层(LangGraph StateGraph)、工具层(MCP协议)、记忆层(Redis Checkpoint+Milvus向量库)。

编排层定义intent_detect->tool_execute->human_approval->summarize流程。每个节点是Python async函数。Checkpointer每步自动保存。工具层用JSON Schema标准化接口。新增工具只需5行配置。

S3和S1知识库RAG的主要区别？

S1是RAG线性检索->生成。S3有Tool循环+审批，用条件路由边实现分支循环。S3可以有循环，S1是单向前馈。

ROI量化

某集团6个月试点数据。ROI基于工时统计和人力成本$50/h。隐性收益：满意度+35%、错误率5%->0.3%、全量审计可追溯。

隐性收益半年后显现。建议ROI预留15-20%隐性收益系数。运营成本：模型调用约$0.003/次，资源费约$50/月。

python
def calc_roi(m, a, v, r=50, infra=0.003):
    mc = (m/60)*r*v
    ac = (a/60)*r*v + infra*v
    sv = mc - ac
    pct = round((1-a/m)*100, 1)
    return {'savings': round(sv,2), 'pct': pct}

print(calc_roi(15, 3, 200))

面试可能问

自动化技术边界？

信息查询、数据录入可自动，资金操作必须审批。interrupt机制兜底。

ROI公式？

(人工-Agent)/人工*100%，含人力成本、月频次、运营成本。

必须审批的场景？

报销/付款/权限变更等高危操作。audit_level=full+审批流。
