为什么要学本章

企业级 Agent 不能完全自主决策——付款、删除数据、修改权限等高风险操作必须经过人工审批。如何让 Agent 在审批节点优雅中断、等待人工确认、再无缝恢复执行，是生产部署的必要条件。人工审批节点 是 LangGraph 的核心能力之一。

学完你能
  • 在 LangGraph 工作流中设置人工审批中断点
  • 实现审批超时和自动降级策略
  • 用 Webhook 或轮询方式接收人工审批结果
  • 设计审批 Dashboard 的前后端交互
  • 实现审批完成后自动恢复工作流执行
本章先记住 3 件事
  • 中断 != 失败：工作流在审批节点暂停，等待外部输入后继续，不是崩溃。
  • 审批必须有超时：审批人可能忘记、离职、度假——超时后自动降级或通知备选审批人。
  • 审计必须完整：谁的审批、什么时间、看了什么数据、同意了什么——全程记录。

人工审批节点的设计与实现

CorpAssist · 大额付款审批 发票金额超过 5 万元时，Agent 不能自动付款，必须暂停等待财务经理审批。审批人通过 Web 界面查看发票详情，点击「通过」或「驳回」，Agent 恢复执行。
人工审批中断与恢复流程
sequenceDiagram
    participant W as 工作流
    participant DB as 审批数据库
    participant API as 审批API
    participant U as 审批人

    W->>W: 执行到审批节点
    W->>DB: 保存审批请求(状态: pending)
    W-->>W: 中断等待

    Note over W,U: 异步等待阶段

    API->>U: 发送审批通知(邮件/IM)
    U->>API: 查看审批详情
    U->>API: 点击「通过」
    API->>DB: 更新状态(approved)
    API->>W: 恢复执行信号
    W->>DB: 读取审批结果
    W->>W: 继续执行后续节点
    W->>DB: 记录完整审批链路

工作流在审批节点暂停，将请求持久化到数据库。审批人通过 API 查看和操作，完成后触发工作流恢复。

审批节点实现

python 
# approval_node.py — 人工审批节点完整实现
from pydantic import BaseModel, Field
from typing import Optional, Literal
from datetime import datetime
from fastapi import FastAPI, HTTPException
import uuid
import asyncio

# ---- 数据模型 ----
class ApprovalRequest(BaseModel):
    """审批请求"""
    request_id: str = Field(default_factory=lambda: f"APP-{uuid.uuid4().hex[:8]}")
    workflow_id: str
    thread_id: str
    title: str
    description: str
    amount: Optional[float] = None
    requester: str
    status: Literal["pending", "approved", "rejected", "timeout"] = "pending"
    assigned_to: str
    created_at: str = Field(default_factory=lambda: datetime.utcnow().isoformat())
    decided_at: Optional[str] = None
    comment: str = ""

class ApprovalDecision(BaseModel):
    """审批决定"""
    request_id: str
    decision: Literal["approved", "rejected"]
    reviewer: str
    comment: str = ""

# ---- 审批存储 ----
class ApprovalStore:
    """审批请求存储（生产用 PostgreSQL）"""
    def __init__(self):
        self._store: dict[str, ApprovalRequest] = {}
        self._pending_events: dict[str, asyncio.Event] = {}

    async def create_request(self, req: ApprovalRequest) -> ApprovalRequest:
        self._store[req.request_id] = req
        self._pending_events[req.request_id] = asyncio.Event()
        return req

    async def get_request(self, request_id: str) -> Optional[ApprovalRequest]:
        return self._store.get(request_id)

    async def decide(self, decision: ApprovalDecision) -> ApprovalRequest:
        req = self._store.get(decision.request_id)
        if not req:
            raise HTTPException(404, "审批请求不存在")
        req.status = decision.decision
        req.decided_at = datetime.utcnow().isoformat()
        req.comment = decision.comment
        # 通知等待的工作流
        event = self._pending_events.get(decision.request_id)
        if event:
            event.set()
        return req

    async def wait_for_decision(self, request_id: str, timeout: int = 3600) -> ApprovalRequest:
        """等待审批结果（带超时）"""
        event = self._pending_events.get(request_id)
        if not event:
            raise ValueError(f"未知审批请求: {request_id}")
        try:
            await asyncio.wait_for(event.wait(), timeout=timeout)
        except asyncio.TimeoutError:
            req = self._store[request_id]
            req.status = "timeout"
            return req
        return self._store[request_id]

store = ApprovalStore()

# ---- 审批 API ----
app = FastAPI(title="CorpAssist 审批服务")

@app.post("/approval/requests")
async def create_approval(req: ApprovalRequest):
    """创建审批请求"""
    return await store.create_request(req)

@app.get("/approval/requests/{request_id}")
async def get_approval(request_id: str):
    """查询审批请求"""
    req = await store.get_request(request_id)
    if not req:
        raise HTTPException(404, "未找到")
    return req

@app.post("/approval/decisions")
async def submit_decision(decision: ApprovalDecision):
    """提交审批决定"""
    return await store.decide(decision)

@app.get("/approval/pending")
async def list_pending():
    """列出所有待审批请求"""
    return [r for r in store._store.values() if r.status == "pending"]

# ---- Agent 工作流中的审批节点 ----
async def human_approval_node(state: dict, store: ApprovalStore, timeout: int = 3600) -> dict:
    """LangGraph 人工审批节点"""
    req = ApprovalRequest(
        workflow_id=state.get("workflow_id", ""),
        thread_id=state.get("thread_id", ""),
        title=state.get("approval_title", "请审批"),
        description=state.get("approval_description", ""),
        amount=state.get("amount"),
        requester=state.get("user_id", "system"),
        assigned_to=state.get("approver", "manager"),
    )
    await store.create_request(req)
    print(f"[审批] 请求已创建: {req.request_id}, 等待 {req.assigned_to} 审批")

    result = await store.wait_for_decision(req.request_id, timeout=timeout)
    print(f"[审批] 结果: {result.status}, 意见: {result.comment}")
    return {
        "approval_status": result.status,
        "approval_comment": result.comment,
        "approval_time": result.decided_at,
    }
不推荐：同步阻塞
  • 审批人在终端输入 y/n，阻塞 Agent 进程
  • 审批人不在线时工作流永远卡住
  • 无超时机制，审批人忘记就永远等待
  • 无审计日志，谁审批了什么无记录
推荐：异步 + 超时 + 审计
  • FastAPI 提供审批 Web 界面，审批人随时操作
  • asyncio.Event 实现非阻塞等待
  • 支持超时自动降级（通知备选审批人）
  • 完整的审批日志：谁、什么时间、什么决定

超时策略与备选审批人

审批超时是最容易被忽视的生产坑。CorpAssist 使用三级超时策略：

级别超时时间行为
L1 提醒 30 分钟 IM/邮件提醒审批人「有待审批请求」
L2 升级 2 小时 通知备选审批人（审批人的上级）
L3 超时 24 小时 标记为超时拒绝，通知申请人和所有审批链路上的人
python 
# approval_timeout.py — 审批超时处理
import asyncio

class TimeoutManager:
    """审批超时管理器"""

    def __init__(self, store: ApprovalStore, notifier):
        self.store = store
        self.notifier = notifier  # 通知服务（邮件/IM）

    async def monitor(self, request_id: str):
        """监控审批超时，执行升级策略"""
        await asyncio.sleep(30 * 60)  # 30 分钟
        req = await self.store.get_request(request_id)
        if req and req.status == "pending":
            await self.notifier.send(req.assigned_to, f"【提醒】审批 {req.request_id} 待处理")

        await asyncio.sleep(90 * 60)  # 再等 1.5 小时（共 2 小时）
        req = await self.store.get_request(request_id)
        if req and req.status == "pending":
            backup_approver = self._get_backup(req.assigned_to)
            await self.notifier.send(backup_approver, f"【升级】审批 {req.request_id} 已超时 2 小时，请处理")

        await asyncio.sleep(22 * 3600)  # 再等 22 小时（共 24 小时）
        req = await self.store.get_request(request_id)
        if req and req.status == "pending":
            req.status = "timeout"
            await self.notifier.send(req.requester, f"【超时】审批 {req.request_id} 已超时 24 小时，自动拒绝")

    def _get_backup(self, approver: str) -> str:
        """获取备选审批人（实际工程中查组织架构）"""
        backup_map = {"manager": "director", "director": "vp"}
        return backup_map.get(approver, "admin")

常见问题（FAQ）

审批中断时，工作流占用的内存和连接怎么释放？
LangGraph 的检查点机制会将状态持久化到数据库，然后工作流进程可以安全退出。恢复时从检查点重新加载状态。这意味审批中断期间不消耗内存或连接资源。这是 LangGraph 相比自己写 while True 等待的最大优势。
如果审批人在审批时看到了过期数据怎么办？
审批请求中应该包含数据的时间戳和有效期。如果审批时发现数据已过期（如库存已变），Agent 应触发重新评估流程。推荐在审批界面显示「数据截止时间：xxx」，并在恢复执行时做数据新鲜度检查。
同一个审批人可以同时审批多个请求吗？
可以。每个审批请求是独立的 request_id，互不干扰。审批 Dashboard 显示所有待审批列表，点开每个查看详情。需要注意并发安全性——同一个审批人对同一请求的两次操作，以第一次为准（幂等）。

本章结论

  • 中断 != 失败：LangGraph 检查点持久化状态，工作流进程可安全退出，恢复时重新加载。
  • 异步审批 API：FastAPI 提供审批 REST 接口，asyncio.Event 实现非阻塞等待。
  • 三级超时策略：30min 提醒 → 2h 升级 → 24h 超时拒绝。每个级别有明确的业务行为。
  • 完整审计：谁审批、何时、看了哪些数据、结果如何——全程记录不可篡改。
  • 数据新鲜度：审批时检查数据有效期，过期数据触发重新评估。
