本章先记住 3 件事
  • 无依赖可并行：两个工具调用如果输入不互相依赖，就应该并行执行以节省时间。
  • 错误不扩散：一个工具失败不应影响其他独立工具的调用结果。
  • 超时是硬约束：每个工具调用必须设超时，避免一个慢工具拖慢整个编排。

工具编排模式：串行、并行与混合

并行工具调用实现

python 
# parallel_tools.py — 并行工具编排
import asyncio
import time
from typing import Dict, Any, List
from datetime import datetime, timedelta

class ToolExecutor:
    """并行工具执行器"""

    async def execute_parallel(self, tool_calls: List[Dict[str, Any]]) -> Dict[str, Any]:
        """并行执行多个工具调用"""
        async def run_single(call: Dict[str, Any]):
            tool_name = call["tool"]
            params = call.get("params", {})
            timeout = call.get("timeout", 10)
            try:
                result = await asyncio.wait_for(
                    self._invoke_tool(tool_name, params),
                    timeout=timeout,
                )
                return {"tool": tool_name, "success": True, "result": result}
            except asyncio.TimeoutError:
                return {"tool": tool_name, "success": False, "error": "超时"}
            except Exception as e:
                return {"tool": tool_name, "success": False, "error": str(e)}

        tasks = [run_single(call) for call in tool_calls]
        results = await asyncio.gather(*tasks, return_exceptions=False)
        return {"parallel_results": results}

    async def _invoke_tool(self, tool_name: str, params: dict) -> Any:
        """实际调用工具（模拟）"""
        if tool_name == "check_calendar":
            await asyncio.sleep(0.5)
            return {"free_slots": ["14:00-15:00", "15:00-16:00"]}
        elif tool_name == "check_room":
            await asyncio.sleep(0.3)
            return {"available_rooms": ["A201", "B305"]}
        elif tool_name == "create_event":
            await asyncio.sleep(0.5)
            return {"event_id": "EVT-001", "status": "created"}
        elif tool_name == "send_email":
            await asyncio.sleep(1.0)
            return {"message_id": "MSG-001", "status": "sent"}
        raise ValueError(f"未知工具: {tool_name}")

    def sync_execute_parallel(self, tool_calls: List[Dict]) -> Dict:
        """同步版本的并行执行"""
        return asyncio.run(self.execute_parallel(tool_calls))

# 使用示例
if __name__ == "__main__":
    executor = ToolExecutor()

    # 并行查询日历和会议室
    parallel_calls = [
        {"tool": "check_calendar", "params": {"date": "2026-06-03"}, "timeout": 5},
        {"tool": "check_room", "params": {"capacity": 10}, "timeout": 5},
    ]
    results = executor.sync_execute_parallel(parallel_calls)
    print(f"并行结果: {results}")
不推荐：串行执行独立工具
  • 先查日历(0.5s) → 再查会议室(0.3s) → 总耗时 0.8s
  • 后一个工具不需要前一个的结果，却串行等待
  • 总耗时 = 各工具耗时之和
推荐：并行执行独立工具
  • 查日历和查会议室同时开始
  • 总耗时 = max(0.5s, 0.3s) = 0.5s
  • 节省 37.5% 的时间
先想 10 秒：如果并行查日历成功(0.5s)但查会议室超时(5s)，你是等还是继续？

取决于业务：如果会议必须有会议室，则整个编排失败回退；如果会议室是可选项（线上会议），则用日历结果继续。设计时每个工具要有独立的超时和「是否必须」标记。

串行编排与错误处理

当工具调用有数据依赖时，必须串行执行。串行编排的关键是错误传播和幂等性。

python 
# serial_pipeline.py — 串行工具编排与错误处理
from dataclasses import dataclass
from typing import Optional, Callable
import time

@dataclass
class PipelineStep:
    name: str
    func: Callable
    retry_count: int = 2
    timeout: int = 30
    critical: bool = True  # 是否关键步骤

class ToolPipeline:
    """串行工具管线"""

    def __init__(self, steps: list[PipelineStep]):
        self.steps = steps

    def execute(self, context: dict) -> dict:
        """顺序执行所有步骤，context 在步骤间传递"""
        for step in self.steps:
            print(f"[{step.name}] 开始执行")
            result = self._execute_with_retry(step, context)
            if result["success"]:
                context[step.name] = result["data"]
            elif step.critical:
                raise RuntimeError(f"关键步骤 {step.name} 失败: {result['error']}")
            else:
                context[step.name] = {"error": result["error"]}
        return context

    def _execute_with_retry(self, step: PipelineStep, context: dict) -> dict:
        for attempt in range(step.retry_count + 1):
            try:
                start = time.time()
                data = step.func(context)
                elapsed = time.time() - start
                return {"success": True, "data": data, "elapsed": elapsed}
            except Exception as e:
                if attempt < step.retry_count:
                    wait = 2 ** attempt
                    print(f"[{step.name}] 重试 {attempt+1}/{step.retry_count} (等待 {wait}s)")
                    time.sleep(wait)
                else:
                    return {"success": False, "error": str(e)}

# 步骤函数
def create_calendar_event(ctx: dict) -> dict:
    """创建日历事件"""
    free_slots = ctx.get("check_calendar", {}).get("free_slots", [])
    room = ctx.get("check_room", {})
    return {"event_id": "EVT-001", "time": free_slots[0], "room": room.get("available_rooms", [None])[0]}

def send_notification(ctx: dict) -> str:
    """发送通知"""
    event = ctx.get("create_calendar_event", {})
    return f"会议通知已发送，事件ID: {event.get('event_id')}"

# 构建管线
pipeline = ToolPipeline([
    PipelineStep("check_calendar", lambda ctx: {"free_slots": ["14:00-15:00"]}, timeout=5),
    PipelineStep("check_room", lambda ctx: {"available_rooms": ["A201"]}, timeout=5),
    PipelineStep("create_calendar_event", create_calendar_event, critical=True),
    PipelineStep("send_notification", send_notification, critical=False),
])

result = pipeline.execute({})
print(f"管线执行结果: {result}")

多工具编排中的幂等性

多次调用同一工具不能产生副作用。CorpAssist 为每个工具调用生成幂等键（idempotency key），服务端检测到相同幂等键时直接返回上次结果。

工具类型幂等策略示例
查询类 天然幂等（多次查询结果不变） 查日历、查库存
创建类 幂等键去重 创建工单、创建事件
通知类 消息ID去重 发送邮件、发送短信

常见问题（FAQ）

并行执行多个工具时，Rate Limit 怎么控制？
使用信号量（Semaphore）控制并发数。CorpAssist 为每个 API 设置独立的最大并发限制（如日历 API 最大 5 并发），用 asyncio.Semaphore 实现。超过限制的任务排队等待。
工具 A 和工具 B 都需要调用同一个下游 API，如何避免重复调用？
在编排层做结果缓存。同一个请求周期内（同一 thread_id），相同入参的工具调用直接返回缓存结果。使用 context 共享缓存字典。
部分成功场景怎么处理？比如邮件发送成功但创建事件失败？
需要补偿事务。CorpAssist 维护工具调用的「待补偿列表」：如果后续步骤失败，自动调用各工具的回滚函数（如取消事件、撤回邮件）。LangGraph 的条件边可以实现补偿流程。

本章结论

  • 编排三模式：并行（无依赖）、串行（有依赖）、混合（先并行后串行）。
  • 并行用 asyncio：asyncio.gather + 独立超时控制，总耗时 = max(各工具耗时)。
  • 串行要幂等：幂等键防止重复调用导致数据错乱。
  • 错误隔离：非关键步骤失败不影响全局，关键步骤失败触发回滚。
  • Rate Limit：Semaphore 控制并发数，避免触发 API 限流。
