为什么要学本章

ReAct (Reasoning + Acting) 是 Agent 最核心的设计模式——它让 LLM 不只是回答问题，而是能思考-行动-观察地完成任务。CorpAssist 的发票处理、数据同步、审批催办等 Agent 都基于 ReAct 模式构建。不理解 ReAct，就无法理解 Agent。

学完你能

  • 画出 ReAct 循环的完整流程图，标注 Thought → Action → Observation → ... → Final Answer
  • 配置 Agent 的 max_iterations、early_stopping_method 等关键参数
  • 编写自定义工具函数并通过 tool decorator 注册到 Agent
  • 理解 ReAct 的局限：幻觉、循环、工具选择错误及其应对
本章先记住 3 件事
  • ReAct = 推理 + 行动：LLM 先生成思考（Thought），再决定调用什么工具（Action），看到结果后继续推理。
  • 工具是 Agent 的手脚：Agent 本身只做推理，所有对外操作都通过注册的工具函数完成。
  • 无限循环是最大风险：必须设置 max_iterations 和 early_stopping，否则 Agent 可能在工具调用中死循环。

ReAct 模式原理：思考-行动-观察循环

ReAct 模式由 Shunyu Yao 在 2022 年提出，核心思想是将推理（Reasoning）和行动（Acting）交织进行。LLM 在每一步生成结构化的思维链，决定是否需要调用外部工具，观察结果后续推，最终给出答案。

CorpAssist · 数据同步 Agent 某零售企业需要跨系统同步商品信息：从 ERP 系统查询商品价格 → 从仓储系统查库存 → 从物流系统查配送状态 → 汇总到一张报表。Agent 需要分多步执行，每步依赖上一步的结果。
ReAct 循环：数据同步 Agent 示例
sequenceDiagram
    participant LLM as LLM (推理)
    participant Tool as 工具层
    participant ERP as ERP 系统
    participant WH as 仓储系统
    participant Log as 物流系统

    LLM->>LLM: Thought: 需要查询商品价格
    LLM->>Tool: Action: query_price("SKU-001")
    Tool->>ERP: GET /api/price?sku=SKU-001
    ERP-->>Tool: {"price": 299.00}
    Tool-->>LLM: Observation: 价格 299.00 元
    LLM->>LLM: Thought: 还需要查库存
    LLM->>Tool: Action: query_stock("SKU-001")
    Tool->>WH: GET /api/stock?sku=SKU-001
    WH-->>Tool: {"available": 150}
    Tool-->>LLM: Observation: 库存 150 件
    LLM->>LLM: Thought: 所有信息已收集
    LLM-->>用户: Final Answer: 商品 SKU-001 售价 299 元，库存 150 件

    Note over LLM,Tool: 每步 Agent 决定是否继续或结束

ReAct 循环中，LLM 交替生成 Thought 和 Action，工具返回 Observation，直到 LLM 判断可以给出 Final Answer。

ReAct Agent 的最小实现

python 
# react_agent.py — CorpAssist ReAct Agent 最小实现
from langchain_openai import ChatOpenAI
from langchain.agents import create_react_agent, AgentExecutor
from langchain.agents import tool
from langchain_core.prompts import PromptTemplate

# 1. 定义工具函数
@tool
def query_price(sku: str) -> str:
    """查询商品价格，输入 SKU 编码，返回价格信息"""
    price_map = {"SKU-001": "299.00", "SKU-002": "159.00", "SKU-003": "89.00"}
    return price_map.get(sku, "未知商品")

@tool
def query_stock(sku: str) -> str:
    """查询商品库存，输入 SKU 编码，返回库存数量"""
    stock_map = {"SKU-001": "150", "SKU-002": "32", "SKU-003": "0"}
    return stock_map.get(sku, "未知商品")

# 2. 构建 ReAct Prompt
REACT_PROMPT = PromptTemplate.from_template(
    "你是一个企业数据助手 CorpAssist。请使用可用工具查询信息并回答问题。\n\n"
    "可用工具：\n{tools}\n\n"
    "工具名称：{tool_names}\n"
    "用户问题：{input}\n\n"
    "请按以下格式回答：\n"
    "Thought: 下一步要做什么\n"
    "Action: 工具名称\n"
    "Action Input: 工具输入参数\n"
    "Observation: 工具返回结果\n"
    "...（重复 Thought/Action/Observation）\n"
    "Thought: 我已收集到足够信息\n"
    "Final Answer: 最终答案\n\n"
    "历史记录：\n{agent_scratchpad}"
)

# 3. 创建 Agent
def create_corpassist_agent():
    llm = ChatOpenAI(model="gpt-4o-mini", temperature=0)
    tools = [query_price, query_stock]
    agent = create_react_agent(llm=llm, tools=tools, prompt=REACT_PROMPT)
    agent_executor = AgentExecutor(
        agent=agent, tools=tools,
        max_iterations=5, early_stopping_method="force",
        handle_parsing_errors=True, verbose=True,
    )
    return agent_executor

if __name__ == "__main__":
    agent = create_corpassist_agent()
    result = agent.invoke({"input": "SKU-001 的商品价格和库存是多少？"})
    print(f"答案：{result['output']}")
不推荐
  • 不给 Agent 设置 max_iterations，默认无限循环
  • 工具函数返回空字符串或不规范格式
  • 用字符串拼接构造 Prompt，无结构化模板
  • 不处理 parsing_errors，Agent 解析失败时静默崩溃
推荐
  • 设置 max_iterations=5~15，配合 early_stopping_method
  • 工具函数返回格式化的 JSON 字符串，含错误信息
  • 使用 PromptTemplate 结构化模板，维护方便
  • handle_parsing_errors=True，输出格式错误时自动重试
先想 10 秒：max_iterations=5 时如果 Agent 在第 5 步还没收集全信息会怎样？

AgentExecutor 会强制停止并返回当前收集到的信息作为 Final Answer。生产环境推荐 "force" 避免无限循环。

Agent 配置参数详解

ReAct Agent 的行为由一组关键参数控制。理解每个参数的影响是生产部署的前提。

参数默认值说明CorpAssist 推荐值
max_iterations 15 Agent 最大思考-行动循环次数 10~20
early_stopping_method "force" 超限后处理方式：force 直接返回，generate 用已有信息生成 "force"
handle_parsing_errors False LLM 输出格式不符合 Action 规范时是否自动重试 True
max_execution_time None Agent 最大执行时间（秒），超时强制终止 60~120
return_intermediate_steps False 是否在结果中返回完整的中间步骤 True（调试）/ False（生产）
trim_intermediate_steps -1 保留最近 N 条中间步骤 50

生产级 Agent 配置

python 
# advanced_agent.py — 生产级 Agent 配置
from langchain_openai import ChatOpenAI
from langchain.agents import create_react_agent, AgentExecutor, tool
from langchain_core.prompts import PromptTemplate
from langchain_core.callbacks import BaseCallbackHandler

class AgentMonitorCallback(BaseCallbackHandler):
    """Agent 运行时监控回调"""
    def on_agent_action(self, action, **kwargs) -> None:
        print(f"[Agent Action] {action.tool}({action.tool_input})")
    def on_agent_finish(self, finish, **kwargs) -> None:
        print(f"[Agent Finish] {finish.return_values['output'][:100]}...")

def create_production_agent():
    llm = ChatOpenAI(model="gpt-4o-mini", temperature=0, max_tokens=4096)
    @tool
    def query_erp(sql: str) -> str:
        """执行 ERP 只读查询，输入 SQL 返回 JSON 结果"""
        if sql.strip().upper().startswith(("SELECT", "WITH")):
            return '[{"id": 1, "name": "商品A", "price": 299.00}]'
        return '{"error": "仅支持只读查询"}'
    @tool
    def send_notification(recipient: str, message: str) -> str:
        """发送系统通知"""
        return f"通知已发送给 {recipient}"
    tools = [query_erp, send_notification]
    agent = create_react_agent(llm, tools, PromptTemplate.from_template(
        "你是一个 CorpAssist 企业数据助手。\n工具：{tools}\n工具名称：{tool_names}\n问题：{input}\n\n{agent_scratchpad}"
    ))
    executor = AgentExecutor(
        agent=agent, tools=tools,
        max_iterations=10, max_execution_time=120,
        early_stopping_method="force", handle_parsing_errors=True,
        return_intermediate_steps=True, trim_intermediate_steps=50,
        callbacks=[AgentMonitorCallback()],
    )
    return executor

ReAct 常见局限与应对

问题表现解决方案
循环陷阱 反复调用同一个工具，结果不变 设置 max_iterations；Prompt 中加入「重复结果请停止」
工具选择错误 应该用 search 时用了 calculator 工具描述写精确，包含输入输出格式示例
幻觉参数 Agent 编造工具返回结果 启用 Agent 可观测性 对比日志；Prompt 强调必须真实调用
上下文超长 中间步骤超过 Token 限制 trim_intermediate_steps=50；使用大窗口模型

常见问题（FAQ）

ReAct Agent 的 Action 输出格式必须是 JSON 吗？
LangChain 的 create_react_agent 期望的 Action 格式是 Action: 工具名称\nAction Input: 输入参数——纯文本格式。工具函数的输入输出推荐用 JSON，方便 LLM 解析。如果使用 OpenAI function calling，底层走的是 Tool 调用协议，格式由 API 自动处理。
为什么我的 Agent 总是在第 3 步就停止，明明还有工具需要调用？
检查三个配置：1) max_iterations 是否过小；2) Prompt 中有没有「尽快结束」的隐式指令；3) 工具返回的 Observation 是否让 LLM 误以为任务已完成。推荐在 Prompt 中加入「思考完成后，请逐一调用所有需要的工具」。
ReAct 能并行调用多个工具吗？
标准的 ReAct 是串行的——一次只能调用一个工具。如果需要并行（如同时查价格和库存），需要在工具函数内部做批量操作，或者使用 LangGraph 的 parallel 节点（见 practice-02 多工具协作）。
如何防止 Agent 调用不该调用的工具？
在工具层实现权限检查。不要在 Agent 层做权限控制——Prompt 可能被注入绕过。在工具函数内部调用 工具沙箱 做权限校验，同时将高度敏感的操作从工具列表中彻底移除。

本章结论

  • ReAct = 推理 + 行动交织：Agent 交替进行 Thought → Action → Observation 循环，直到收集足够信息给出 Final Answer。
  • 工具是 Agent 的边界：Agent 只能通过注册的工具函数与外部交互，工具描述质量决定 Agent 行为准确性。
  • 生产必须设限：max_iterations、max_execution_time、handle_parsing_errors 是防止 Agent 失控的三大护栏。
  • ReAct 适合中短流程：步数 < 20 时效果最佳，复杂长流程推荐 LangGraph。
  • 工具描述要精确：包含输入输出格式、单位、边界条件、错误示例。

操作步骤

  1. 编写 2-3 个工具函数（订单查询、客户信息查询、物流查询），使用 @tool 装饰器。
  2. 构建 create_react_agent，配置 max_iterations=5。
  3. 测试 Agent 对多步查询问题的行为，观察工具调用顺序。
  4. 将 max_iterations 改为 1，观察 Agent 在步数不足时的行为。
  5. 运行 return_intermediate_steps=True 版本，分析 Agent 的每一步 Thought 质量。

判断练习

  1. Agent 查询库存后得到 stock=0，但仍回复「库存充足」。问题出在哪个环节？

    参考答案

    LLM 幻觉：Agent 可能忽略了 Observation 中的 stock=0。排查：1) 检查 intermediate_steps 确认 LLM 确实看到了 Observation；2) Prompt 中加入「必须根据 Observation 中的实际数据回答」；3) 对比 LangSmith Trace 查看实际上下文。
