Spring Cloud Gateway 配置

yaml
spring:
  cloud:
    gateway:
      routes:
        - id: cs-intent-router
          uri: lb://cs-intent-router
          predicates:
            - Path=/api/v1/intent/**
          filters:
            - name: RequestRateLimiter
              args:
                redis-rate-limiter.replenishRate: 100
                redis-rate-limiter.burstCapacity: 200
        - id: cs-chat-engine
          uri: lb://cs-chat-engine
          predicates:
            - Path=/api/v1/chat/**
          filters:
            - name: CircuitBreaker
              args:
                name: csChatEngineCB
                fallbackUri: forward:/fallback/chat
        - id: cs-tools
          uri: lb://cs-tools
          predicates:
            - Path=/api/v1/tools/**
        - id: cs-agent-ws
          uri: lb:ws://cs-agent-ws
          predicates:
            - Path=/ws/agent/**

模块间数据流与接口契约

java
// 意图路由 -> 对话引擎 契约
public record IntentResult(
    String sessionId,
    String intent,         // order_query | return | refund | policy
    double confidence,     // 0.0 ~ 1.0
    Map slots,  // {orderId: "ORD-123"}
    @Nullable String fallbackReason
) {}

// 对话引擎 -> 工具层 契约
public record ToolCallRequest(
    String sessionId,
    String toolName,       // queryOrder | createReturn | changeAddress
    Map parameters,
    String idempotencyKey
) {}

// 人工坐席转接 契约
public record HandoffRequest(
    String sessionId,
    String userId,
    String reason,          // low_confidence | sensitive_content | user_request
    List chatHistory,
    @Nullable String preferredAgentId
) {}

面试可能问

Q1: 意图路由和对话引擎为什么是两个独立模块？

职责分离：意图路由专注分类（输入 -> intent），对话引擎专注生成（intent + context -> 回答）。独立模块的好处：① 可以独立部署和扩缩（意图路由需要 GPU 分类器，对话引擎需要 LLM）；② 独立升级（换分类模型不影响对话流程）；③ 更清晰的可观测性（分别监控分类准确率和回答质量）。
