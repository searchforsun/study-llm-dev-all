为什么学这章

项目交付时业务方最关心两个数字：解决率和成本。用数据讲优化故事比讲技术细节更有说服力。本章教你如何构建从 70% 到 85% 的解决率提升叙事。

学完后能做什么

能构建解决率从 70% 到 85% 的优化叙事；能分阶段归因 Token 成本；能使用 STAR 模板回答面试行为题；能准备 CorpAssist 客服上线 Checklist。

面试 STAR 模板

text
S (Situation): CorpAssist 客服系统为某电商平台提供服务，日均 3000 次对话，解决率 70%。
T (Task): 需要将解决率提升到 80%+，同时控制 Token 成本。
A (Action):
  1. 引入 LLM 意图分类器（结构化输出）替代纯规则引擎
  2. 实现 @Tool 工具调用（订单查询/退货/改地址）
  3. 配置 ChatMemory + 上下文摘要压缩管理多轮对话
  4. 建立坏例库驱动回归测试
R (Result):
  - 解决率从 70% 提升到 85%
  - 转人工率从 35% 降至 22%
  - Token 成本控制在日均 ¥300
