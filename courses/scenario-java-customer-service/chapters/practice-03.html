Spring AI ReactAgent 对话流程

java
@Service
public class CustomerServiceAgent {

    private final ChatClient chatClient;
    private final ChatMemory chatMemory;
    private final IntentClassifier intentClassifier;

    private static final int MAX_STEPS = 8;

    public CustomerServiceAgent(ChatClient.Builder builder,
                                 ChatMemory chatMemory,
                                 IntentClassifier intentClassifier,
                                 OrderTools orderTools,
                                 ReturnTools returnTools) {

        this.chatMemory = chatMemory;
        this.intentClassifier = intentClassifier;

        this.chatClient = builder
            .defaultSystem("""
                你是一个智能客服助手。你的工作流程：
                1. 理解用户问题，确定意图
                2. 如果需要查数据，使用 available functions
                3. 基于工具返回结果，生成友好、准确的回答
                4. 如果用户要求退货/退款，先查订单再创建退货
                5. 如果无法处理，主动提出转人工

                可用工具：
                - query_order: 查订单状态和物流
                - create_return: 创建退货申请
                - change_address: 修改收货地址

                注意：
                - 每次调用前先确认用户意图
                - 工具返回错误时，用通俗语言解释给用户
                - 涉及金额的决策须用户明确确认
                """)
            .defaultTools(orderTools, returnTools)
            .defaultAdvisors(
                new ConversationAdvisor(chatMemory),
                new SafeOutputAdvisor()
            )
            .build();
    }

    public Flux chat(String sessionId, String userMessage) {
        return chatClient.prompt()
            .user(userMessage)
            .advisors(a -> a.param("chat_memory_conversation_id", sessionId))
            .stream()
            .content();
    }
}

工具节点与 LLM 节点的协作

ReactAgent 在每次迭代中执行「思考 -> 行动 -> 观察」循环。Spring AI 通过 ChatClient 的 defaultTools() 注册可用工具，LLM 自主决策何时调用哪个工具。

ReactAgent 思考-行动-观察循环
sequenceDiagram
    participant USER as 用户
    participant LLM as LLM
    participant TOOL as 工具层
    participant API as 业务系统

    USER->>LLM: 我的订单还没到
    LLM->>LLM: 思考：用户意图是查订单
    LLM->>TOOL: 行动：query_order(orderId="")
    TOOL->>API: 查订单 API
    API-->>TOOL: 返回物流信息
    TOOL-->>LLM: 观察：订单已发货，在途
    LLM->>LLM: 思考：基于结果生成回答
    LLM-->>USER: 您的订单已于昨日发出，预计2-3天到达
    USER->>LLM: 那我要退货
    LLM->>LLM: 思考：意图转为退货
    LLM->>TOOL: 行动：query_order(确认订单)
    TOOL-->>LLM: 订单可退货
    LLM->>LLM: 思考：需要用户确认退货原因
    LLM-->>USER: 好的，请问您退货的原因是什么？

SSE 流式响应的端到端链路

java
@RestController
@RequestMapping("/api/v1/cs")
public class CustomerServiceController {

    private final CustomerServiceAgent agent;

    @PostMapping(value = "/chat/stream",
                 produces = MediaType.TEXT_EVENT_STREAM_VALUE)
    public Flux> chatStream(
            @RequestBody ChatRequest request,
            @AuthenticationPrincipal UserPrincipal user) {

        return agent.chat(request.sessionId(), request.message())
            .map(token -> ServerSentEvent.builder()
                .event("token")
                .data(token)
                .build())
            .concatWithValues(
                ServerSentEvent.builder()
                    .event("done")
                    .data("[DONE]")
                    .build());
    }
}

record ChatRequest(String sessionId, String message) {}
