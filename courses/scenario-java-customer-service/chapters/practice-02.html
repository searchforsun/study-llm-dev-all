为什么学这章

智能客服的核心价值在于——不只回答问题，还能替用户做事（查订单、创建退货、改地址）。Spring AI 的 @Tool 注解是 Java 侧工具调用的核心技术。

学完后能做什么

能使用 @Tool 注解定义业务工具并自动生成 JSON Schema；能通过 Bean Validation 实现参数校验；能使用 Redis 分布式锁保证工具幂等性；能实现工具失败的重试与降级策略。

@Tool 注解定义业务工具

java
@Service
public class OrderTools {

    @Tool(name = "query_order", description = "查询订单状态和物流信息")
    public OrderInfo queryOrder(
            @ToolParam(description = "订单编号") String orderId,
            @ToolParam(description = "查询用户手机号") String phone) {

        log.info("Querying order: orderId={}, phone={}", orderId, phone);

        // 调用订单系统 API
        OrderResponse response = orderApiClient.query(orderId, phone);

        return new OrderInfo(
            response.orderId(),
            response.status(),       // PENDING/SHIPPED/DELIVERED/RETURNED
            response.courier(),
            response.trackingNo(),
            response.estimatedDelivery()
        );
    }

    @Tool(name = "create_return", description = "创建退货申请")
    public ReturnResult createReturn(
            @ToolParam(description = "订单编号") String orderId,
            @ToolParam(description = "退货原因") String reason,
            @ToolParam(description = "退货商品ID列表") List itemIds) {

        ReturnRequest request = new ReturnRequest(orderId, reason, itemIds);
        return returnApiClient.create(request);
    }

    @Tool(name = "change_address", description = "修改收货地址")
    public AddressResult changeAddress(
            @ToolParam(description = "订单编号") String orderId,
            @ToolParam(description = "新收货地址") String newAddress,
            @ToolParam(description = "新收件人") String recipient,
            @ToolParam(description = "新联系电话") String phone) {

        return addressApiClient.update(orderId, newAddress, recipient, phone);
    }
}

Spring AI 的 @Tool 注解自动将方法描述和参数描述生成为 OpenAI Function Calling 格式的 JSON Schema。这意味着你不需要手动写 JSON Schema —— 注解驱动就够了。

参数校验和幂等性保证

java
@Service
public class IdempotentToolService {

    private final RedisTemplate redisTemplate;

    private static final long IDEMPOTENT_TTL_SECONDS = 3600; // 1 小时

    public  T executeWithIdempotency(
            String toolName,
            Map params,
            Supplier operation) {

        // 1. 生成幂等 Key
        String idempotencyKey = generateKey(toolName, params);

        // 2. Redis 分布式锁（防止并发执行两次）
        String lockKey = "lock:" + idempotencyKey;
        Boolean locked = redisTemplate.opsForValue()
            .setIfAbsent(lockKey, "executing",
                Duration.ofSeconds(30));

        if (Boolean.FALSE.equals(locked)) {
            // 另一个请求正在执行相同操作
            throw new IdempotentConflictException(
                "操作正在执行中，请稍后再试");
        }

        try {
            // 3. 检查是否已执行过
            String result = redisTemplate.opsForValue()
                .get("result:" + idempotencyKey);
            if (result != null) {
                return deserialize(result);
            }

            // 4. 执行工具
            T operationResult = operation.get();

            // 5. 缓存结果
            redisTemplate.opsForValue().set(
                "result:" + idempotencyKey,
                serialize(operationResult),
                Duration.ofSeconds(IDEMPOTENT_TTL_SECONDS));

            return operationResult;

        } finally {
            redisTemplate.delete(lockKey);
        }
    }

    private String generateKey(String toolName, Map params) {
        String paramStr = new TreeMap<>(params).entrySet().stream()
            .map(e -> e.getKey() + "=" + e.getValue())
            .collect(Collectors.joining("&"));
        return DigestUtils.md5Hex(toolName + ":" + paramStr);
    }
}

工具失败的重试与降级

java
@Service
public class ResilientToolExecutor {

    private final RetryTemplate retryTemplate;
    private final MeterRegistry meterRegistry;

    public ResilientToolExecutor() {
        this.retryTemplate = RetryTemplate.builder()
            .maxAttempts(3)
            .exponentialBackoff(Duration.ofMillis(200), 2, Duration.ofSeconds(5))
            .retryOn(IOException.class, TimeoutException.class)
            .notRetryOn(IllegalArgumentException.class)  // 参数错误不重试
            .build();
    }

    public  T executeWithRetry(String toolName, Supplier operation) {
        try {
            return retryTemplate.execute(context -> {
                log.info("Tool {} attempt {}",
                    toolName, context.getRetryCount() + 1);
                return operation.get();
            });
        } catch (Exception e) {
            meterRegistry.counter("tool.failure",
                "tool", toolName,
                "error", e.getClass().getSimpleName())
                .increment();

            // 降级：返回错误消息供 LLM 使用
            throw new ToolExecutionException(
                "工具 " + toolName + " 执行失败：" + e.getMessage(),
                "请告诉用户：系统暂时无法完成这个操作，请稍后再试或联系人工客服。"
            );
        }
    }
}

本章结论

  • @Tool 注解声明式注册业务工具，自动生成 Function Calling JSON Schema
  • Redis 分布式锁 + 幂等 Key 保证工具不会重复执行
  • RetryTemplate 指数退避重试，参数错误不重试，3 次失败后降级给 LLM 友好提示
