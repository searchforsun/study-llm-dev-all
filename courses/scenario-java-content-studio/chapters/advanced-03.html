Q1: 如何控制 AI 生成内容的风格？

考点：风格控制不仅仅是"写正式一点"的指令，而是系统化的参数体系和模板管理。

好回答

五维语调参数 + Few-shot 示例 + 模板版本管理。语调参数（正式度/情感度/专业度/简洁度/幽默度）控制风格方向；Few-shot 示例提供 2-3 个"好的"例子让 LLM 模仿；模板用 Git 管理，每次修改有 PR Review 和版本 diff。这样即使换模型（从 qwen-max 到 qwen-plus），风格仍然一致。

差回答

在 Prompt 里写"请用正式语气"就行。

Q2: 长文生成如何保证质量？

考点：分段生成时的连贯性和一致性控制。

java
// 面试回答话术
//
// "三步法：Outline→Segment→Assemble。
// 1. 先生成大纲（LLM 输出 JSON 格式的章节列表）
// 2. 分段生成，每章注入前章摘要作为上下文
// 3. 拼接后做连贯性检测（相邻段落向量相似度 > 0.3）
//
// 关键优化：每段的第一个句子要"承上启下"。
// 我们在 Prompt 中明确要求：'段首句需要引用上一章的结论'，
// 这样即使分段生成，读起来也像一次性写的。
//
// 去重也很重要——我们用 Jaccard 相似度比较
// 不同段落的句子，相似度 > 0.7 的合并或重写。
// 检测到重复后调用 LLM 重新表达。
//
// 效果：5000 字长篇的连贯性评分从 3.2 提升到 4.5。"
//
// 实现的 Spring AI 组件:
// ChatClient (outline和segment各自独立)
// plusClient (连贯性检测用推理模型)
// VectorStore (存储段落 embedding 供相似度比较)

Q4: 内容安全怎么保障？

考点：多层防护 + 行业合规 + 留痕审计。

bash
# 面试回答话术
#
# "四层安全防护：
# 1. 敏感词过滤——精确匹配 + 正则匹配 + AI 变体检测
#    （如用拼音、谐音绕过的敏感词）
# 2. 有害内容检测——LLM 检查暴力/色情/歧视内容，
#    Advisor 链在生成前拦截
# 3. 行业合规引擎——金融/医疗/教育等行业特有规则
#    （如金融禁止'保本'宣传，医疗禁止'包治百病'）
# 4. 人工审核——Flowable 工作流管理审核流程，
#    高风险内容自动拦截，中等风险送审，低风险放行
#
# 所有操作记录审计日志：
# INSERT INTO content_audit_log (content_id, action, snapshot, result)
# VALUES ('CT-001', 'GENERATE', '{...}', 'INTERCEPTED')
# 每次违规处理都有完整的操作追溯。"

Q5: 个性化怎么做？

考点：用户画像 + 动态模板注入 + A/B 验证的完整链路。

java
// 面试回答话术
//
// "三步个性化策略：
// 1. 用户画像——Redis Hash 存储行业/角色/偏好风格/历史互动。
//    30 天过期，每次互动更新。
// 2. 动态模板注入——根据画像选择不同的 Prompt 变量值。
//    电商用户: '限时优惠，立即抢购'
//    企业用户: '降本增效，驱动增长'
//    技术用户: '最新功能，提升效率'
// 3. A/B 验证——50% 流量走版本 A，50% 走版本 B。
//    对比点击率、转化率、NPS。
//
// 效果数据：个性化后点击率 +35%（2.3%→3.1%），
// 转化率 +50%（0.8%→1.2%），NPS +12 分。
//
// 实现要点：
// UserProfileService - Redis Hash 操作
// DynamicTemplateInjector - 画像到模板变量的映射
// AbExperimentManager - 用户 ID 哈希分组 + 指标采集"
