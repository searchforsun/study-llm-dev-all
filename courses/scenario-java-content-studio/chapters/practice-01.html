约 30 分钟 · 阶段：Java 栈实战 · 场景：S5 内容生成工坊

为什么学这章

企业内容生成往往需要批量产出（一次生成 50 条产品描述、100 条客户回访文案）。Spring Batch 提供事务性批处理能力，结合 MQ 任务队列实现可靠的大规模内容生成。

学完后能做什么

用 Spring Batch 实现批量内容生成 Job；配置 MQ 任务队列拆分任务；实现进度追踪（已生成/总数/失败数）；实现失败重试和部分完成处理。

本章要点速记

1) Spring Batch Job = Step(Reader→Processor→Writer)，Processor 中调用 LLM 生成；2) MQ 任务队列实现任务的拆分和调度（如 100 条 split 成 10 个 batch 各 10 条）；3) 进度追踪通过 JobExecution 的 ExecutionContext 存储每批的生成结果；4) 失败重试：Step 级别最多重试 3 次，跳过不可恢复的错误。

// Spring Batch 配置
@Configuration
public class BatchContentJobConfig {

    @Bean
    public Job batchContentJob(JobRepository jobRepository,
                                Step generateStep, Step summaryStep) {
        return new JobBuilder("batchContentJob", jobRepository)
            .start(generateStep)
            .next(summaryStep)
            .build();
    }

    @Bean
    public Step generateStep(JobRepository jobRepository,
                              PlatformTransactionManager transactionManager) {
        return new StepBuilder("generateStep", jobRepository)
            .<ContentInput, ContentOutput>chunk(10, transactionManager)
            .reader(contentReader())
            .processor(contentProcessor())
            .writer(contentWriter())
            .faultTolerant()
            .retryLimit(3)
            .retry(TimeoutException.class)
            .skipLimit(5)
            .skip(IllegalArgumentException.class)
            .build();
    }

    @Bean
    public ItemProcessor<ContentInput, ContentOutput> contentProcessor() {
        return input -> {
            // 模板填充 + LLM 调用
            String prompt = fillTemplate(input.getTemplateId(), input.getVariables());
            String content = llmService.generate(prompt);
            return new ContentOutput(input.getId(), content, "SUCCESS");
        };
    }

    @Bean
    public ItemWriter<ContentOutput> contentWriter() {
        return items -> {
            for (ContentOutput item : items) {
                contentRepository.save(item);
                progressTracker.increment(item.getId());
            }
        };
    }
}

// MQ 任务拆分
@Component
public class ContentTaskDispatcher {

    private final RabbitTemplate rabbitTemplate;

    public void dispatchBatch(List<ContentInput> inputs) {
        // 拆分为每个 10 条的子任务
        List<List<ContentInput>> batches = partition(inputs, 10);
        String batchId = UUID.randomUUID().toString();

        for (int i = 0; i < batches.size(); i++) {
            ContentTask task = new ContentTask(
                batchId, i, batches.get(i));
            rabbitTemplate.convertAndSend(
                "content.generate.queue", task);
        }
    }

    @RabbitListener(queues = "content.generate.queue")
    public void handleTask(ContentTask task) {
        try {
            for (ContentInput input : task.getInputs()) {
                String content = llmService.generate(
                    fillTemplate(input.getTemplateId(), input.getVariables()));
                contentRepository.save(new ContentOutput(
                    input.getId(), content, "SUCCESS"));
                progressTracker.update(task.getBatchId(),
                    task.getTaskIndex(), 1, 0);
            }
        } catch (Exception e) {
            progressTracker.update(task.getBatchId(),
                task.getTaskIndex(), 0, task.getInputs().size());
        }
    }
}

// 进度追踪
@Component
public class ProgressTracker {

    private final RedisTemplate<String, String> redis;

    public void initBatch(String batchId, int total) {
        redis.opsForValue().set("batch:" + batchId + ":total",
            String.valueOf(total));
        redis.opsForValue().set("batch:" + batchId + ":completed", "0");
        redis.opsForValue().set("batch:" + batchId + ":failed", "0");
    }

    public void update(String batchId, int taskIdx, int success, int fail) {
        redis.opsForValue().increment("batch:" + batchId + ":completed", success);
        redis.opsForValue().increment("batch:" + batchId + ":failed", fail);
    }

    public Progress getProgress(String batchId) {
        int total = Integer.parseInt(
            redis.opsForValue().get("batch:" + batchId + ":total"));
        int completed = Integer.parseInt(
            redis.opsForValue().get("batch:" + batchId + ":completed"));
        int failed = Integer.parseInt(
            redis.opsForValue().get("batch:" + batchId + ":failed"));
        return new Progress(total, completed, failed,
            (double) (completed + failed) / total * 100);
    }
}

本章结论

  • Spring Batch Chunk=10 批量处理，Processor 中调用 LLM 生成
  • MQ 任务队列拆分 100 条为 10 个子任务并行处理
  • Redis 追踪进度：总任务数/已完成/已失败
  • 失败重试 3 次（可恢复错误），参数错误直接跳过
