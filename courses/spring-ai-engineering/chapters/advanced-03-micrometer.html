指标埋点

CorpAssist 推荐三类 llm metrics（见 demo metrics-list.md）：

  • corpassist.chat.requests — Counter，tag: model, status
  • corpassist.chat.tokens — Counter，tag: model, type=prompt|completion
  • corpassist.chat.latency — Timer，发布 histogram 供 P95

LlmMetricsAdvisor

java
@Component
public class LlmMetricsAdvisor implements CallAroundAdvisor {

  private final MeterRegistry registry;

  @Override
  public AdvisedResponse aroundCall(AdvisedRequest request, CallAroundAdvisorChain chain) {
    String model = String.valueOf(request.adviseContext().getOrDefault("model", "unknown"));
    String tenant = String.valueOf(request.adviseContext().getOrDefault("tenantId", "unknown"));
    Timer.Sample sample = Timer.start(registry);
    try {
      AdvisedResponse resp = chain.nextAroundCall(request);
      registry.counter("corpassist.chat.requests",
          "model", model, "tenant", tenant, "status", "ok").increment();
      recordTokens(resp, model, tenant);
      return resp;
    } catch (Exception ex) {
      registry.counter("corpassist.chat.requests",
          "model", model, "tenant", tenant, "status", "error").increment();
      throw ex;
    } finally {
      sample.stop(Timer.builder("corpassist.chat.latency")
          .tag("model", model)
          .tag("tenant", tenant)
          .register(registry));
    }
  }

  private void recordTokens(AdvisedResponse resp, String model, String tenant) {
    var usage = resp.response().getMetadata().getUsage();
    if (usage != null) {
      registry.counter("corpassist.chat.tokens",
          "model", model, "tenant", tenant, "type", "prompt")
          .increment(usage.getPromptTokens());
      registry.counter("corpassist.chat.tokens",
          "model", model, "tenant", tenant, "type", "completion")
          .increment(usage.getCompletionTokens());
    }
  }
}

Tracing

单次问答：Gateway → Spring Controller → Python /search → ChatModel → Tool。Micrometer Observation 为每段创建 span，调用 Python 时注入 W3C traceparent。

Observation 包装 ChatService

java
@Service
public class ChatService {

  private final ObservationRegistry observationRegistry;
  private final ChatClient chatClient;

  public String chat(String tenantId, String message) {
    return Observation.createNotStarted("corpassist.chat", observationRegistry)
        .lowCardinalityKeyValue("tenant", tenantId)
        .highCardinalityKeyValue("message.length", String.valueOf(message.length()))
        .observe(() -> chatClient.prompt()
            .advisors(a -> a.param("tenantId", tenantId))
            .user(message)
            .call()
            .content());
  }
}

TraceIdFilter + RestClient 传播

生产环境应使用 Micrometer Tracing 的 Propagator 注入 W3C traceparent（格式 00-{traceId}-{spanId}-01），勿手写 UUID 充当 traceparent。X-Request-Id 为业务关联 ID，可与分布式 trace 并存。

java
@Component
public class TraceIdFilter extends OncePerRequestFilter {
  private final Propagator propagator;
  private final Tracer tracer;

  @Override
  protected void doFilterInternal(HttpServletRequest req, HttpServletResponse res,
      FilterChain chain) throws ServletException, IOException {
    Span span = propagator.extract(req, HttpServletRequest::getHeader)
        .name("http.request").start();
    try (Tracer.SpanInScope ws = tracer.withSpan(span)) {
      String requestId = Optional.ofNullable(req.getHeader("X-Request-Id"))
          .orElse(UUID.randomUUID().toString());
      MDC.put("traceId", span.context().traceId());
      res.setHeader("X-Request-Id", requestId);
      chain.doFilter(req, res);
    } finally {
      span.end();
      MDC.clear();
    }
  }
}

@Bean
RestClient pythonSearchClient(Propagator propagator, Tracer tracer) {
  return RestClient.builder()
      .baseUrl("${corpassist.python.search-url}")
      .requestInterceptor((req, body, exec) -> {
        tracer.currentSpan().ifPresent(span ->
            propagator.inject(span.context(), req.getHeaders(),
                (headers, key, value) -> headers.set(key, value)));
        return exec.execute(req, body);
      })
      .build();
}

自检：生产 sampling 是否应永远 1.0？

高流量通常 0.1–0.3；预发可 1.0。Counter/Timer 指标应全量，不受 trace 采样影响。

判断练习

  1. 只打日志不打 Metrics，能否满足生产 SLA 复盘？

    参考答案

    不能。 日志难聚合 P95 与 Token 日趋势；Metrics + Dashboard 是标准做法。

  2. 把 trace sampling 设为 0 以节省存储，是否影响 Counter？

    参考答案

    不影响 Counter/Timer；但排障样本变少，预发仍建议提高 sampling。

  3. reasoning 档 P95 超标，第一反应是扩容 Pod？

    参考答案

    应先看 NS3 路由是否过多走 reasoning、缓存命中率、网关延迟。
