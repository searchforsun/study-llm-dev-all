DocumentReader：从制度 PDF 到 Document

CorpAssist · S2 制度库更新 每季度 HR 上传新版《员工手册》PDF。Python 批处理可用 LangChain Loader；Java 侧 Spring AI 提供 PagePdfDocumentReader、TikaDocumentReader 等，统一输出 List<Document>。

Document reader 负责格式解析与初始 metadata（文件名、页码）。CorpAssist 约定在 reader 后立刻补齐 dept、docVersion、tenantId。

java 
@Service
public class HandbookIngestService {
  private final VectorStore vectorStore;

  public int ingestPdf(Path pdfPath, String dept, String version) {
    PagePdfDocumentReader reader = new PagePdfDocumentReader(pdfPath.toString());
    List<Document> pages = reader.get();
    pages.forEach(doc -> {
      doc.getMetadata().put("dept", dept);
      doc.getMetadata().put("docVersion", version);
      doc.getMetadata().put("source", pdfPath.getFileName().toString());
    });
    List<Document> chunks = splitter.split(pages);
    vectorStore.add(chunks);
    return chunks.size();
  }
}
整 PDF 单向量
  • 一次 embed 整本书
  • 检索命中整本 PDF 噪声大
  • 无法精确 citation 到条款
Reader + Splitter
  • 按页/段读取再切块
  • metadata 保留 page、source
  • 引用可定位到具体 chunk id
FAQ：大文件 ingest 要不要放请求线程？

不要。使用 @Async 或消息队列触发 ingest 任务，在线 API 只读 VectorStore。Demo 的 rag-flow.md 第 1 步即离线上传。

Splitter：chunk size 与 overlap

Text splitter 将长 Document 切为适合 embedding 的片段。CorpAssist 与 Python 轨统一：500 tokens / overlap 50（见 Demo rag-flow.md）。

参数偏小CorpAssist 默认偏大
chunk size 召回细、易断章 500 上下文全、噪声多
overlap 边界丢信息 50 存储与成本上升
java 
@Bean
TokenTextSplitter handbookSplitter() {
  return new TokenTextSplitter(500, 50, 5, 10_000, true);
}

overlap 保证跨段问题（如「年假+审批流程」）仍能在相邻 chunk 中被检索到。

建议 {source}-{page}-{index}，写入 metadata；回答时 citation 直接返回 id 与 source 链接。

RetrievalAugmentation：QuestionAnswerAdvisor

Retrieval augmentation 在模型生成前注入检索片段。Spring AI 1.0 推荐 QuestionAnswerAdvisor 挂到 ChatClient 的 Advisor 链。

java 
@Configuration
public class RagAdvisorConfig {
  @Bean
  ChatClient ragChatClient(ChatModel chatModel, VectorStore vectorStore) {
    var qaAdvisor = QuestionAnswerAdvisor.builder(vectorStore)
        .searchRequest(SearchRequest.builder().topK(4).build())
        .build();
    return ChatClient.builder(chatModel)
        .defaultAdvisors(qaAdvisor)
        .build();
  }
}

@Service
public class CorpAssistRagService {
  private final ChatClient ragChatClient;

  public String ask(String question, String dept) {
    return ragChatClient.prompt()
        .advisors(a -> a.param(QuestionAnswerAdvisor.FILTER_EXPRESSION,
            "dept == '" + dept + "'"))
        .user(question)
        .call()
        .content();
  }
}

引用溯源：citation 与审计

企业场景不能只返回答案文本。Source attribution 要求列出引用的 document id、source 文件名与可选页码，供员工点开原文核对。

java 
public record RagResponse(String answer, List<Citation> citations) {}

public record Citation(String documentId, String source, String excerpt) {}

@Service
public class RagResponseMapper {
  public RagResponse map(ChatResponse response, List<Document> retrieved) {
    List<Citation> cites = retrieved.stream()
        .map(d -> new Citation(
            d.getId(),
            (String) d.getMetadata().get("source"),
            d.getText().substring(0, Math.min(120, d.getText().length()))))
        .toList();
    return new RagResponse(response.getResult().getOutput().getText(), cites);
  }
}
无 citation
  • 模型 hallucinate 制度条款
  • 合规审计无法复核
  • 用户信任度低
结构化 citations
  • 返回 id + source + 摘要
  • 日志记录 retrieval set
  • 评测可对比 gold chunk id

Prompt 侧约束（system message 片段）

properties 
corpassist.rag.system=你是 CorpAssist 制度助手。仅依据检索上下文回答；
若上下文不足，明确说「未找到相关规定」；勿编造条款编号。

FAQ：RAG Advisor 与手写 prompt 拼接谁优先？

优先 Advisor：检索、模板、filter 由框架统一；业务仅配置 SearchRequest 与 system message。手写拼接难测试且易漏 filter。

本章结论

每条对应上一节 h3。

  • DocumentReader：统一输出 Document；ingest 时补齐 dept/version/source metadata。
  • Splitter：500/50 与 Python 同源；稳定 chunk id 服务 citation。
  • RetrievalAugmentation：QuestionAnswerAdvisor + filter；ingest 与在线问答分离。
  • 引用溯源：RagResponse 带 citations；system prompt 禁止无依据编造。
