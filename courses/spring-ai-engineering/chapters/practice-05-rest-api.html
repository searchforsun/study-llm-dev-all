Controller 设计：分层与 DTO

CorpAssist · S2 API 网关后 门户 BFF 调用 /v1/chat（同步）与 /v1/chat/stream（SSE）。Controller 注入 CorpAssistChatService，聚合 Memory、RAG、Tool（见前四章）。
java 
@RestController
@RequestMapping("/v1/chat")
@Validated
public class ChatController {
  private final CorpAssistChatService chatService;

  @PostMapping
  public ChatReply chat(@Valid @RequestBody ChatRequest req,
      @AuthenticationPrincipal CorpUser user) {
    return chatService.chat(user, req);
  }

  @PostMapping("/rag")
  public RagResponse rag(@Valid @RequestBody RagRequest req,
      @AuthenticationPrincipal CorpUser user) {
    return chatService.rag(user, req);
  }
}

public record ChatRequest(
    @NotBlank @Size(max = 4000) String message,
    String conversationId) {}

public record ChatReply(String conversationId, String answer) {}

/v1/rag/queryPOST JSONRAG + citations（与 advanced-05 一致）

SSE 流式：Flux 与事件格式

Spring MVC 6 / Boot 3 可直接返回 Flux<ServerSentEvent<String>> 或 text/event-stream。Demo chat-api.http 指向 POST /v1/chat/stream。

java 
@PostMapping(value = "/stream", produces = MediaType.TEXT_EVENT_STREAM_VALUE)
public Flux<ServerSentEvent<String>> stream(
    @Valid @RequestBody ChatRequest req,
    @AuthenticationPrincipal CorpUser user) {
  return chatService.stream(user, req)
      .map(token -> ServerSentEvent.builder(token).event("token").build())
      .concatWith(Flux.just(ServerSentEvent.builder("[DONE]").event("done").build()))
      .onErrorResume(ex -> Flux.just(
          ServerSentEvent.builder("模型暂时不可用").event("error").build()));
}
SSE 事件流
sequenceDiagram
  participant C as 客户端
  participant API as ChatController
  participant S as ChatService
  participant LLM as ChatModel Flux
  C->>API: POST /stream
  API->>S: stream()
  S->>LLM: ChatClient.stream
  loop tokens
    LLM-->>S: chunk
    S-->>C: event:token data:...
  end
  S-->>C: event:done
  • event 类型：token、done、error，便于前端区分。
  • 网关：Nginx proxy_buffering off；超时大于模型首 token 延迟。
  • 与 ChatClient 与 ChatModel 的 Flux<String> 直接对接。

全局异常：ProblemDetail 与错误码

@RestControllerAdvice 将校验失败、模型超时、Tool 异常映射为 RFC 9457 ProblemDetail，避免泄露 stack trace。

java 
@RestControllerAdvice
public class CorpAssistExceptionHandler {

  @ExceptionHandler(MethodArgumentNotValidException.class)
  ResponseEntity<ProblemDetail> validation(MethodArgumentNotValidException ex) {
    ProblemDetail pd = ProblemDetail.forStatus(HttpStatus.BAD_REQUEST);
    pd.setTitle("请求参数无效");
    pd.setProperty("code", "VALIDATION_ERROR");
    return ResponseEntity.badRequest().body(pd);
  }

  @ExceptionHandler(UncheckedIOException.class)
  ResponseEntity<ProblemDetail> llmTimeout(UncheckedIOException ex) {
    ProblemDetail pd = ProblemDetail.forStatus(HttpStatus.GATEWAY_TIMEOUT);
    pd.setTitle("模型调用超时");
    pd.setProperty("code", "LLM_TIMEOUT");
    return ResponseEntity.status(504).body(pd);
  }
}
500 + 原始异常信息
  • 暴露内部类名
  • BFF 无法分类重试
  • 不符合企业 API 规范
ProblemDetail + code
  • VALIDATION_ERROR / LLM_TIMEOUT
  • SSE error event 同码
  • 日志含 traceId（advanced 可观测）

FAQ：ProblemDetail 与 SSE error event 如何统一 code？

定义 CorpAssistErrorCodes 枚举；同步 API 写入 ProblemDetail.properties.code，SSE 的 error event data 用同一 JSON 字段，BFF 一套解析逻辑。

端点方法说明
/v1/chatPOST同步全文
/v1/chat/streamPOSTSSE token 流
/v1/rag/queryPOST带 citations 的 RAG

Controller 只做鉴权、DTO 校验与 ChatClient 委托；护栏逻辑放在 Advisor 链而非 if-else 堆在 REST 层。

FAQ：OpenAPI 如何标注 citations 必填？

RagResponse schema 对 citations[] 设 required；springdoc 生成示例时附带 pol-001 样例，方便 BFF 契约测试。
