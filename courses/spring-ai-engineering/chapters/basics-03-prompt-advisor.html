已完成Prompt 模板与 Advisor

约 40 分钟 · 阶段：Spring AI 核心抽象 · 能力：用 PromptTemplate 与 Advisor 链构建可测试 Prompt 管线

为什么要学本章

把 prompt 硬编码在 Service 里，会导致无法版本化、无法单测、无法复用。CorpAssist S2 客服的 RAG 系统提示、NS2 工具调用前的安全指令，都应通过 prompt template 与 advisor chain 管理；advisor order 错了等于护栏失效。

学完你能
  • 用 PromptTemplate 与 .st 文件管理带 placeholder 的 RAG 提示
  • 区分 system message 与 user message，并说明各自在 S2/NS2 中的职责
  • 设计 testable prompt：快照测试模板渲染结果，不调用真实 LLM
本章先记住 3 件事
  • 模板：变量用 {context}、{query} 等占位，渲染与调用分离。
  • 消息角色：System 定边界，User 承载员工提问，Assistant 由模型生成。
  • Advisor：在 call 前后织入 RAG、记忆、护栏；顺序影响 NS2 工具是否被注入攻击。

PromptTemplate

prompt template 把「提示词文档」从 Java 字符串挪到可审查的文件（如 classpath:/prompts/rag-system.st），便于产品/合规一起评审。

text
你是 CorpAssist 企业知识库助手（S2）。
仅根据下列检索片段回答；无依据时回答「未在制度库中找到」。
引用格式：[doc_id] 片段摘要

上下文：
{context}

用户：{query}
java
@Bean
PromptTemplate ragPromptTemplate(
    @Value("classpath:/prompts/rag-system.st") Resource resource) {
  return new PromptTemplate(resource);
}

public String renderUserMessage(String context, String query) {
  return ragPromptTemplate.render(Map.of(
      "context", context,
      "query", query));
}
占位符来源CorpAssist 含义
{context}VectorStore 检索 Top-K制度片段拼接（practice RAG 章）
{query}员工原始问题门户 POST body
{sessionId}网关透传Memory Advisor 使用（practice 章）
不推荐
  • Service 内 200 行字符串拼接
  • 模板改一字需重新编译整包
  • 无渲染单测，只靠肉眼 diff
推荐
  • .st 文件进 Git，PR 可 review
  • 渲染结果做 testable prompt 快照
  • 变量缺失时单测失败（Map 缺 key）
先想 10 秒：PromptTemplate 与 ChatClient.user() 谁先谁后？

先 render() 得到最终 user 文本（或整段 Prompt），再 chatClient.prompt().user(rendered).call()。

Advisor 链

advisor chain 在模型调用前后插入横切逻辑：检索增强、会话记忆、输入过滤、输出审核。CorpAssist NS2 强调工具调用前的白名单与注入防护，常作为靠前的 Advisor。

Advisor 执行顺序（概念）
flowchart LR
  IN[用户请求]
  A1[InputSanitizer]
  A2[RetrievalAdvisor]
  A3[ChatMemoryAdvisor]
  CM[ChatModel]
  A4[OutputModeration]
  OUT[响应]
  IN --> A1 --> A2 --> A3 --> CM --> A4 --> OUT

advisor order：先净化输入，再 RAG，再记忆；模型输出后做审核。advanced 生产护栏章细化实现。

java
@Bean
ChatClient ragChatClient(ChatModel chatModel, VectorStore vectorStore) {
  var retrieval = new QuestionAnswerAdvisor(vectorStore, SearchRequest.defaults());
  return ChatClient.builder(chatModel)
      .defaultAdvisors(retrieval)
      .defaultSystem("你是 CorpAssist S2 助手，禁止编造制度条款。")
      .build();
}
Advisor 类型典型作用阶段
QuestionAnswerAdvisor检索片段注入 promptpractice RAG
MessageChatMemoryAdvisor多轮历史拼入practice Memory
自定义 SafeInputAdvisor越狱/注入检测advanced 护栏
ToolCallingAdvisorNS2 工具循环practice Tools
CorpAssist · NS2 · 工具与护栏 员工问「帮我导出全部客户手机号」——InputSanitizer 应拦截；合法问「查工单 WO-1024」才进入 ToolCallingAdvisor。顺序错误（先 Tool 后 Sanitizer）可能让恶意 prompt 直接触发工具。
before

改 Prompt、加检索上下文、加载 Memory。

after

审核输出、打码 PII、写审计日志。

链式组合

.defaultAdvisors(a, b, c) 顺序即执行顺序。

系统/用户消息

OpenAI 风格对话由多条 message chat 组成。Spring AI 用 .system() / .user() 映射到不同角色。

角色谁写CorpAssist 示例
system message 平台模板 「仅依据 context 回答；无依据说未找到」
user message 员工 + 渲染变量 含 {context} 与 {query} 的完整 user 段
assistant 模型历史 Memory Advisor 注入上轮回复

在 ChatClient.builder 上 .defaultSystem(...)，适合全局不变的 S2 人格与合规边界。

java
chatClient.prompt()
    .user(renderedUserMessage)
    .call();

多租户或 A/B 时每次 .system(dynamicSystem) 覆盖默认。

java
chatClient.prompt()
    .system(tenantPolicyLoader.forTenant(tenantId))
    .user(renderedUserMessage)
    .call();

与 Python 对照：messages=[{"role":"system",...},{"role":"user",...}] 与 Spring 的 system/user 划分一致，便于 python parity 评测。

可测试 Prompt

testable prompt 指：单元测试只验证模板渲染与 Advisor 是否改写了 Prompt，不必每次打真实 LLM。

java
@Test
void ragTemplate_rendersPlaceholders() {
  var template = new PromptTemplate("""
      上下文：{context}
      用户：{query}
      """);
  String rendered = template.render(Map.of(
      "context", "[doc-1] 年假 5 天",
      "query", "还剩几天"));
  assertThat(rendered).contains("[doc-1]").contains("还剩几天");
}
测试层测什么不依赖
模板渲染占位符替换、缺 key 失败ChatModel
Advisor 单元mock Request/Response 改 Prompt真实 HTTP
集成@MockBean ChatModel 固定返回厂商配额
不推荐
  • CI 每条用例打 OpenAI 计费
  • 只断言「返回非空」
  • 模板改字无快照回归
推荐
  • 黄金文件存渲染结果（advanced 快照章）
  • Advisor 顺序用专用测试固定
  • 与 Python 轨共用 JSON fixture 测 prompt 文本
PromptTemplate 与 StringTemplate 选哪个？
Spring AI 的 PromptTemplate 与 ChatClient 集成更好；复杂逻辑可先在 Java 算变量，再传入简单模板。
NS2 工具 prompt 放 system 还是 user？
工具说明与边界放 system；具体工具调用上下文由框架生成 tool 消息（practice Tools 章）。

本章结论

  • PromptTemplate：提示词文件化，{context}、{query} 等变量可 review、可单测。
  • Advisor 链：RAG/Memory/护栏/工具按顺序织入；NS2 先净化再 Tool。
  • 系统/用户消息：System 定边界，User 承载渲染后的问题与检索上下文。
  • 可测试 Prompt：渲染快照 + Mock ChatModel，CI 不烧配额。
