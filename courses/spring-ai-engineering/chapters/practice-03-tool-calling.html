@Tool 定义：从 POJO 到模型可见工具

CorpAssist · S2 查年假 员工问「我还剩几天年假？」模型应调用 leaveBalance 而非编造数字。工具类放在 corpassist-ai-chat 模块，由 ChatClient 注册。
java 
@Component
public class HrTools {
  private final LeaveService leaveService;

  @Tool(description = "查询当前登录员工的年假剩余天数")
  public int leaveBalance(
      @ToolParam(description = "员工工号，如 E1024") String employeeId) {
    return leaveService.remainingDays(employeeId);
  }

  @Tool(description = "创建 IT 工单，非幂等，需用户确认后调用")
  public TicketResult createTicket(
      @ToolParam(description = "工单标题") String title,
      @ToolParam(description = "问题描述") String body) {
    return ticketService.create(title, body);
  }
}
java 
@Bean
ChatClient toolChatClient(ChatModel chatModel, HrTools hrTools) {
  return ChatClient.builder(chatModel)
      .defaultTools(hrTools)
      .build();
}

// 调用
toolChatClient.prompt()
    .user("帮我查一下 E1024 还剩多少年假")
    .call()
    .content();

参数 schema：描述、校验与类型

模型依据 schema 生成 tool_call 参数。@ToolParam 的 description 要业务化（工号格式、枚举值），减少 parse 失败。复杂对象可用 record + 校验注解。

类型Spring AI 映射CorpAssist 示例
StringstringemployeeId、title
int / IntegerintegertopK（内部工具）
booleanbooleanconfirmCreate

模糊 description
  • @ToolParam String id 无说明
  • 模型传姓名而非工号
  • 下游 404 变成 tool error
可执行 description
  • 「员工工号 E 开头 + 数字」
  • Service 层 @Valid 二次校验
  • 错误 message 回灌模型重试
java 
public record CreateTicketRequest(
    @NotBlank @Size(max = 80) String title,
    @NotBlank String body,
    @NotNull TicketPriority priority) {}

@Tool(description = "创建工单，priority 为 LOW|MEDIUM|HIGH")
public TicketResult createTicketValidated(CreateTicketRequest req) {
  return ticketService.create(req);
}
自检：createTicket 为何要在 ToolWhitelist 标「非幂等」？

模型可能重复调用导致多个工单；需 confirm 参数或业务 idempotency key，见 Demo ToolWhitelist.md。

错误重试：tool error 与模型回灌

工具失败时，应返回结构化错误给模型，而非抛 500 到用户。Spring AI 将 tool 异常序列化为 tool result；可配合 Advisor 限制重试次数。

java 
@Tool(description = "查询年假")
public String leaveBalanceSafe(String employeeId) {
  try {
    int days = leaveService.remainingDays(employeeId);
    return "REMAINING:" + days;
  } catch (EmployeeNotFoundException ex) {
    return "ERROR:工号不存在，请让用户提供 E 开头工号";
  } catch (TimeoutException ex) {
    return "ERROR:HR 系统超时，请稍后重试";
  }
}
  • 可重试：超时、503 — 模型可再次调用（上限 2 次）。
  • 不可重试：参数非法 — 让模型向用户澄清。
  • 写操作：createTicket 失败时不自动 retry，避免重复单。
异常直抛到 Controller
  • 用户看到 stack trace
  • 模型无法自我纠正
  • 审计缺少 tool_call 记录
ERROR: 前缀回灌
  • 模型改写为用户友好话术
  • 日志保留 tool_call id
  • 结合 暴露 REST 与 SSE 统一 ProblemDetail

  • 多步规划、长时任务、外部插件

延伸：Tool 错误如何反馈给模型？

返回结构化 tool error（业务 code + 可读 message），禁止把 ERP stack trace 塞回 tool result；retry 仅对幂等工具且限次。

本章结论

  • @Tool 定义：Component + defaultTools；ChatClient 自动完成 tool_call 循环。
  • 参数 schema：@ToolParam description 要可执行；record + @Valid 二次校验。
  • 错误重试：ERROR 前缀回灌模型；写操作慎 retry。
  • Agent 衔接：Spring 管企业内短链 Tool；Agent 管多步；OpenAPI bridge 对齐白名单。
