代码助手项目交付不仅是代码，还包括效果评估报告、SLA 承诺、运维手册、演示视频。CorpAssist 的交付 checklist 帮助团队系统化整理交付物，确保项目可部署、可运维、可量化，避免「代码写好但不知道跑没跑对」的局面。

本章先记住 3 件事
  • 交付 = 代码 + 文档 + 数据：不只是 API 端点，还有如何证明它好用。

  • 运维手册要：新运维 1 小时内可独立操作，包含常见故障排障步骤。

Python 交付物检查器

# delivery_checker.py — CorpAssist 交付物检查器
from dataclasses import dataclass
from typing import List, Optional
from datetime import datetime

@dataclass
class DeliveryItem:
    name: str
    content: List[str]
    acceptance_criteria: str
    status: str = "pending"  # pending / done / verified

@dataclass
class DeliveryReport:
    project: str
    version: str
    date: str
    items: List[DeliveryItem]
    sla: dict
    eval_summary: dict

class DeliveryChecker:
    """交付物检查器"""

    REQUIRED_ITEMS = {
        "index_pipeline": DeliveryItem(
            name="索引流水线",
            content=["Git 增量索引脚本", "全量重索引脚本", "Milvus Schema", "权限配置"],
            acceptance_criteria="增量 < 60s, 全量 < 30min",
        ),
        "completion_api": DeliveryItem(
            name="补全 API",
            content=["FIM 格式 Prompt", "SSE 流式端点", "风格约束参数", "质量网关集成"],
            acceptance_criteria="TTFT < 200ms, P99 < 2s",
        ),
        "qa_api": DeliveryItem(
            name="问答 API",
            content=["检索+生成端点", "引用格式化", "SSE 流式", "score_threshold 配置"],
            acceptance_criteria="语义准确率 ≥80%, 引用准确率 ≥90%",
        ),
        "eval_report": DeliveryItem(
            name="效果报告",
            content=["四维评估数据", "基线对比", "迭代记录", "A/B 实验结果"],
            acceptance_criteria="采纳率 ≥40%, 语法准确率 ≥95%",
        ),
        "ops_manual": DeliveryItem(
            name="运维手册",
            content=["部署步骤", "监控项列表", "告警规则", "排障流程", "回滚方案"],
            acceptance_criteria="新运维 1h 内可独立操作",
        ),
    }

    DEFAULT_SLA = {
        "availability": "99.9% (月故障 ≤ 43min)",
        "p99_latency": "< 500ms",
        "p95_latency": "< 200ms",
        "throughput": "100 req/s",
    }

    def __init__(self):
        self.items = list(self.REQUIRED_ITEMS.values())

    def mark_done(self, item_name: str):
        for item in self.items:
            if item.name == item_name:
                item.status = "done"

    def mark_verified(self, item_name: str):
        for item in self.items:
            if item.name == item_name:
                item.status = "verified"

    def all_verified(self) -> bool:
        return all(i.status == "verified" for i in self.items)

    def generate_report(self, eval_summary: dict, project: str = "CorpAssist", version: str = "1.0.0") -> DeliveryReport:
        return DeliveryReport(
            project=project,
            version=version,
            date=datetime.now().isoformat(),
            items=self.items,
            sla=self.DEFAULT_SLA,
            eval_summary=eval_summary,
        )

    def print_summary(self):
        print(f"\n=== 交付物检查清单 ===")
        for item in self.items:
            icon = "✓" if item.status == "verified" else "○" if item.status == "done" else "✗"
            print(f"  {icon} {item.name} [{item.status}]")
            for c in item.content:
                print(f"    - {c}")
            print(f"    验收: {item.acceptance_criteria}")
        print(f"\n总进度: {sum(1 for i in self.items if i.status == 'verified')}/{len(self.items)}")
        print(f"全部完成: {'是' if self.all_verified() else '否'}")


checker = DeliveryChecker()
checker.mark_verified("索引流水线")
checker.mark_verified("补全 API")
checker.mark_done("问答 API")
checker.print_summary()

# 交付报告示例
report = checker.generate_report({
    "syntax_accuracy": 96.2,
    "semantics_accuracy": 82.5,
    "adoption_rate": 45.1,
    "security_violation_rate": 1.8,
    "eval_dataset_size": 150,
}, version="2.1.0")
print(f"\n交付版本: {report.version} | 日期: {report.date[:10]} | 采纳率: {report.eval_summary['adoption_rate']}%")

效果报告模板

效果报告是交付评审的核心依据，包含以下部分：

  1. 项目概览：项目名称、版本、交付日期、团队
  2. 四维评估对比：语法准确率（基线 92% → 当前 96%）、语义准确率（65% → 82%）、采纳率（28% → 45%）、安全违规率（5% → 2%）
  3. 用户覆盖：日活用户数、日请求量、平均采纳次数/人/天
  4. 稳定性：本月可用性、P99 延迟、P50 延迟、错误率
  5. A/B 实验摘要：最近 3 次实验的变更内容、效果对比、统计显著性
  6. 待优化项：已知问题列表、优先级、排期

报告数据来源于自动化流水线，每周一自动生成并发送给相关方。CorpAssist 的经验：业务方最关注采纳率和用户覆盖（多少人用），技术方最关注语义准确率和 P99 延迟。

运维手册章节结构

章节内容
1. 系统概述系统架构图、依赖组件、网络拓扑
2. 部署指南环境要求、部署步骤、配置参数、首次启动
3. 监控项Prometheus 指标列表、Grafana 看板截图、关键告警规则
4. 告警响应P0/P1/P2 告警响应流程、值班联系方式
5. 常见故障排查模型 API 限流、Redis 宕机、vLLM OOM、Milvus 连接超时
6. 回滚方案版本回滚步骤、数据恢复流程
7. 容量规划当前配额、瓶颈分析、扩缩容方案
SLA 承诺

代码助手服务的 SLA：可用性 99.9%（非计划停机月累计 ≤43 分钟）；补全 API P99 延迟 <500ms；问答 API P99 延迟 <2s；日处理能力 ≥10 万次请求。SLA 未达标时按服务合同条款进行服务费减免。

本章结论

  • 五大交付物：索引流水线 + 补全 API + 问答 API + 效果报告 + 运维手册。
  • SLA 量化：可用性 99.9%、P99 < 500ms、采纳率 ≥40%。
  • 验收标准明确：每个交付物有具体的可度量验收条件。
  • 交付流程：功能验证 → 效果跑分 → 运维文档 → 演示录制 → 交付评审。
