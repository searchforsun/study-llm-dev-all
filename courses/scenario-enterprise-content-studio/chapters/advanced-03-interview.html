本章先记住 3 件事
  • 面试考的是工程思维：不是「用什么模型」，而是「为什么这么设计」。
  • 每问都有分层答案：先说原理，再说工程实现，最后说生产坑。
  • 系统设计要画图：五连问最后一问是系统设计，准备好画架构图。

第一问：风格控制

style question 的典型问法：「如何确保 AI 生成的内容始终保持品牌风格一致？」

层次答案要点参考章节
语调卡（Tone Card） 5-8 条品牌语调规则，yaml 化注入 system prompt 风格控制与模板库
Few-shot 范例 2-5 篇精品文案隐式传递风格 风格控制与模板库
模板版本管理 semver 三号段 + 灰度迁移 风格控制与模板库
质量评估闭环 LLM-as-Judge 评分驱动模板迭代 内容质量评估

加分回答：提到「风格控制不是一次完成——上线后通过评估飞轮持续优化语调卡和 few-shot」。

第三问：质量评估

eval question 的典型问法：「如何评估 AI 生成内容的质量？没有标注数据怎么办？」

参考答案框架：

  1. 五维 Rubric：品牌一致、事实准确、可读性、合规、创意。
  2. LLM-as-Judge：用独立模型（如 qwen-turbo）按 Rubric 打分。
  3. 人工校准：每周抽检 30% 低分内容人工评分，校准自动分的偏差。
  4. 坏例驱动：低分内容入库，分类归因，驱动模板迭代。
  5. 回归测试：坏例构成回归集，每次模板更新后验证。

加分回答：提到「评估不是目的——评估飞轮驱动模板优化才是」。

第四问：安全合规

security content qa 的典型问法：「生成内容涉及敏感信息或合规风险，你怎么防护？」

低分答案
  • 「加一个敏感词过滤」
  • 「让模型自己注意」
  • 「审核一下就好了」
高分答案
  • 三层审核：自动（敏感词+Moderation API）→ 人工 → 驳回重写
  • 行业词库分层：通用/广告法/医疗/金融
  • 对抗样本定期测试审核鲁棒性
  • 全链路审计日志，生成到发布可追溯

常见问题（FAQ）

五连问面试大概多长时间？
通常 45-60 分钟。前四问各 8-10 分钟，最后一问系统设计 15-20 分钟。回答时保持结构：先说核心思路，再说工程细节，最后提一个实际踩过的坑。
如果没有实际落地经验怎么办？
用本课的 CorpAssist 场景作为「项目经验」来回答。关键是要讲清楚设计决策——为什么分段生成、为什么双栈——比有没有上过线更重要。
面试官追问「具体用哪个模型」怎么办？
回答原则：选型取决于场景。FAQ 快答用 qwen-turbo（低成本），任务型用 qwen-plus（高能力），评分用独立模型。强调选型逻辑而非模型名称本身。

本章结论

  • 风格控制：语调卡 + few-shot + 模板版本 + 评估飞轮。
  • 长文生成：大纲→分段→过渡句→去重，而非一次生成。
  • 质量评估：五维 Rubric + LLM-as-Judge + 坏例驱动迭代。
  • 安全合规：三层审核 + 行业词库 + 对抗样本 + 审计留痕。
  • 系统设计：Python 核心 + Spring BFF + MQ 解耦的经典架构。
