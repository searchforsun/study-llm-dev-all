为什么要学本章

内容工坊的 Python 生成侧和 Spring 编排侧如何协作？dual stack studio 定义了清晰的双栈分工：Python 专注 LLM 调用与模板渲染，Spring 负责 BFF 编排、鉴权与会话管理。本章打通两端的 API 契约。

学完你能
  • 实现 Python FastAPI 生成服务（python generate）
  • 用 Spring WebFlux 编排调用 Python 服务（spring orchestrate）
  • 使用 SSE 实现流式输出到前端
  • 编写双栈契约测试确保 API 兼容
本章先记住 3 件事
  • Python 负责「生成」：模板渲染、LLM 调用、审核，因为这些生态成熟、迭代快。
  • Spring 负责「编排」：BFF 聚合、用户鉴权、会话管理、指标收集，因为 Java 企业生态更稳。
  • 契约测试不可省略：contract test 确保 Python API 升级不会 break Spring BFF。

Python FastAPI 生成服务

python generate 提供两个核心 API：/v1/generate 单条生成与 /v1/batch 批量提交。

Python：FastAPI 服务

python 
# CorpAssist FastAPI 生成服务
from fastapi import FastAPI, HTTPException
from pydantic import BaseModel
import asyncio

app = FastAPI(title="CorpAssist Content Studio")

class GenerateRequest(BaseModel):
    template_id: str
    variables: dict
    stream: bool = False

class GenerateResponse(BaseModel):
    job_id: str
    content: str | None
    score: dict | None
    audit_result: str

@app.post("/v1/generate")
async def generate(req: GenerateRequest):
    """单条文案生成"""
    prompt = engine.render(req.template_id, req.variables)
    content = await llm_client.chat_async(prompt)
    audit_result = auditor.audit(content)
    score = scorer.score(content)

    return GenerateResponse(
        job_id=str(uuid.uuid4()),
        content=content,
        score=score,
        audit_result=audit_result[0].value
    )

@app.post("/v1/generate/stream")
async def generate_stream(req: GenerateRequest):
    """流式生成"""
    prompt = engine.render(req.template_id, req.variables)

    async def event_stream():
        async for chunk in llm_client.chat_stream(prompt):
            yield f"data: {json.dumps({'text': chunk})}\n\n"
        yield "data: [DONE]\n\n"

    return StreamingResponse(event_stream(),
                             media_type="text/event-stream")

@app.post("/v1/batch")
async def create_batch(req: BatchRequest):
    """提交批量生成任务"""
    batch_id = producer.publish_batch(req.template_id, req.variables_csv)
    return {"batch_id": batch_id, "status": "submitted"}
先想 10 秒：为什么不直接用 Python 全栈，还需要 Spring BFF？

Python 全栈在鉴权、会话管理、企业合规（审计日志、租户隔离）方面生态不如 Java。Spring BFF 作为坚固的前端网关，Python 服务可以专注于 LLM 核心能力。

Spring BFF 编排

spring orchestrate 负责将前端的请求编排到 Python 服务，并通过 SSE 流式返回。

Java：Spring WebFlux 编排

java 
// Spring BFF 编排 Python 生成服务
@RestController
@RequestMapping("/api/v1")
public class ContentBffController {

    private final WebClient pythonClient;

    @PostMapping("/generate")
    public Mono<GenerateResponse> generate(@RequestBody GenerateRequest req) {
        return pythonClient.post()
            .uri("/v1/generate")
            .bodyValue(req)
            .retrieve()
            .bodyToMono(GenerateResponse.class);
    }

    @GetMapping(value = "/generate/stream", produces = MediaType.TEXT_EVENT_STREAM_VALUE)
    public Flux<ServerSentEvent<String>> generateStream(
            @RequestParam String templateId,
            @RequestParam Map<String, String> vars) {

        return pythonClient.post()
            .uri("/v1/generate/stream")
            .bodyValue(new GenerateRequest(templateId, vars, true))
            .retrieve()
            .bodyToFlux(String.class)
            .map(data -> ServerSentEvent.<String>builder(data).build())
            .doOnError(e -> log.error("Stream error", e));
    }
}

record GenerateRequest(String templateId,
                       Map<String, String> variables,
                       boolean stream) {}
record GenerateResponse(String jobId, String content,
                        Map<String, Double> score,
                        String auditResult) {}

bff compose 负责组合多个 Python 服务的结果。例如「生成 + 审核 + 评分」可以并行调 Python 的三个独立服务，Spring 等待所有结果后返回。

不推荐
  • Spring 直接调 LLM，与 Python 竞争
  • 无契约测试，API 升级频繁 break
  • 批量生成在 Spring 侧实现
推荐
  • Spring 只编排不生成
  • 契约测试覆盖所有双栈 API
  • 批量在 Python 侧通过 MQ 实现

SSE 流式输出与契约测试

sse studio 使用 Server-Sent Events 实现流式输出到前端。相比 WebSocket，SSE 更轻量且浏览器原生支持。

contract test 通过 Spring Cloud Contract 或 PACT 框架，确保 Python API 的输入输出格式与 Spring BFF 的期望一致。

常见问题（FAQ）

Python 和 Spring 之间用什么协议通信？
HTTP/REST，通过 Spring WebClient（非阻塞）。批量场景通过 RabbitMQ 解耦，不直接 HTTP 调用。
双栈部署时要考虑什么？
Python 服务需要 GPU 实例（如 P100/V100 按需），Spring BFF 只需要普通 CPU 实例。网络延迟：双栈之间最好同区域部署，延迟 < 5ms。
API 升级如何做灰度？
Python 服务保留 2 个 API 版本（/v1/generate 和 /v2/generate），Spring BFF 通过配置切换版本。旧版本废弃前保留至少 1 个月。

本章结论

  • 双栈分工：Python 生成 + Spring 编排，各司其职。
  • Python FastAPI：/v1/generate 单条 + /v1/batch 批量 + SSE 流式。
  • Spring BFF：鉴权、编排、SSE 推送，WebClient 非阻塞调用。
  • SSE 流式：浏览器原生支持的流式输出协议。
  • 契约测试：确保双栈 API 的兼容性。
