图文混合生成：流程与分工

image text gen 不是让 LLM 同时生成文字和图片，而是文字先写、图片后配——文案确定后，从中提取关键词标签，用标签检索品牌素材库中最匹配的图片。

CorpAssist · 促销海报图文生成（30 秒） 运营生成一条 SKU 促销文案：「春季焕肤精华，限时 8 折」。模板渲染后，从文案中提取关键词「精华」「春季」「焕肤」，在品牌素材库中检索到一张浅绿色调的产品场景图。系统检查该图 license=commercial 且到期日在 2026-12-31 之后，自动匹配为海报配图。整个过程 2 分钟，传统方式需设计师 1 小时。

Visual RAG：图文双模态检索

visual rag 是对品牌素材库进行语义检索的技术方案。与纯文本 RAG 不同，Visual RAG 需要将图片的视觉特征和文本标签同时编码为向量，实现图文双模态搜索。

不推荐
  • 只靠文件名关键词匹配图片（精确匹配，召回率低）
  • 从网上随意搜索图片（版权风险）
  • 每张图手动配标签（不可扩展）
  • 无图片审核，低质/违规图片混入
推荐
  • CLIP 双模态 embedding（文本+图片统一向量空间）
  • 品牌素材库统一管理，标注 license 元数据
  • 自动标签 + 人工校准
  • 图片审核前置（NSFW + 品牌一致）

Python：Visual RAG 检索

python 
# CorpAssist Visual RAG 素材检索
import numpy as np
from clip_client import ClipClient  # CLIP embedding

class VisualRAG:
    def __init__(self, vector_db_url: str):
        self.client = ClipClient(vector_db_url)

    def index_asset(self, image_path: str, tags: list[str],
                    license: str, expiry: str):
        """索引品牌素材"""
        embedding = self.client.encode_image(image_path)
        metadata = {"tags": tags, "license": license, "expiry": expiry}
        self.client.insert(embedding, metadata)

    def search(self, text_query: str, top_k: int = 3,
               license_filter: str = "commercial") -> list:
        """按文本搜索可用素材"""
        query_emb = self.client.encode_text(text_query)
        results = self.client.search(query_emb, top_k * 2)
        # 版权过滤
        valid = [r for r in results
                 if r.metadata.get("license") == license_filter
                 and r.metadata.get("expiry", "9999") > "2026-12-31"]
        return valid[:top_k]

    def auto_tag(self, image_path: str) -> list[str]:
        """自动生成图片标签"""
        return self.client.classify(image_path,
                                    ["产品图", "场景图", "人物图",
                                     "插画", "背景图"])

rag = VisualRAG("http://localhost:8000")
matches = rag.search("春季护肤品 清新场景", top_k=2)

先想 10 秒：CLIP 双模态检索相比纯标签匹配的核心优势是什么？

CLIP 能理解语义而非字面匹配。「清新自然」可以匹配到浅绿色调的产品场景图，即使图片标签中没有「清新」二字。召回率提升 40%+。

版权管理与 API 选型

copyright check 是内容工坊合规的关键环节。每张配图必须记录：来源、授权范围、到期日。商用素材建议统一采购正版图库（如视觉中国、Shutterstock）并建立品牌专属素材库。

stock library 的选型标准：

  • 商用授权：确认授权范围含「企业营销推广」。
  • 标签准确：图片自带结构化标签，方便 Visual RAG 检索。
  • API 集成：支持通过 API 直接搜索和下载，无需人工操作。

mm api choice 是图文生成 API 的选型决策：

API适合场景成本生成速度
通义万相 商品图、海报背景 低（¥0.02/张） 2-5s
Stable Diffusion 创意插画、抽象风格 中（自建 GPU） 3-10s
Midjourney 高品质品牌视觉 高（$10-30/月） 10-30s
DALL-E 3 概念图、原型图 中（¥0.08/张） 5-15s

选型原则：存量用素材库检索，增量用生成 API。品牌已有历史图片的，优先通过 Visual RAG 复用；需要全新视觉的，走生成 API。

常见问题（FAQ）

Visual RAG 的素材库需要多大？
初始 200-500 张高质量品牌图片即可覆盖 80% 的配图需求。关键是标签质量而非数量。建议每月新增 50-100 张，建立持续索引。
生成图片的版权归谁？
各平台政策不同。通义万相生成的图片商用版权归用户；Midjourney 付费版商用版权归用户；Stable Diffusion 开源模型需注意训练数据的版权。商用前务必阅读服务条款。
图文生成时，图片和文案怎么保证风格一致？
通过共同标签体系——文案的语调卡（清新/高冷/专业）与图片的色调标签（浅色/深色/暖色）共享同一分类体系。同一条模板产生的文案和配图在标签上自然对齐。
素材库接入成本高吗？
初始阶段可以用向量数据库的免费层（如 Milvus 50 万向量免费）加上 CLIP 开源模型（免费），搭建 Visual RAG 基本没有成本。主要投入在标签标注上。

判断练习

  1. 设计师投诉 AI 配的图「风格不对」，但你用的已经是品牌素材库。可能的原因？

    参考答案

    标签体系与文案提取的关键词不一致。文案说「清爽」但素材库中无「清爽」标签。解决：统一标签体系，或在文案→标签的映射中加入同义词扩展。

  2. 发现一张图片 license 即将到期，已用于 50 篇已发布内容。如何处理？

    参考答案

    立即标记该图片为 expired，替换已发布内容中的这张图。自动触发重新检索，用同标签的其他图片替换。框架应在设计时就支持图片的灰度替换。
