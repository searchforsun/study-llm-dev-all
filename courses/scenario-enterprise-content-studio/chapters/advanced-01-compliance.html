为什么要学本章

内容发布涉及行业合规（广告法、医疗广告、金融宣传）与平台安全（涉政、色情、暴力）。compliance content 不是可选项——每个 ToB 内容工坊必须过合规审计。本章建立「规则 + 模型 + 留痕」三层防护。

学完你能
  • 建立行业敏感词库与 AC 自动机扫描
  • 配置 industry rule 行业合规规则引擎
  • 实现全链路审计留痕（audit trail）
本章先记住 3 件事
  • 行业规则不同：医疗、金融、食品各有专属禁用语，不能用通用词库覆盖。
  • 有害内容不止敏感词：harmful block 需要覆盖隐形违规（隐喻、谐音、上下文违规）。
  • 留痕是合规底线：从生成到发布的每步操作必须有记录，可追溯、不可篡改。

敏感词库与行业规则

sensitive word 检测使用 AC 自动机实现 O(n) 匹配。词库分为通用层和行业层：

层级内容维护频率覆盖场景
通用违规 涉政、色情、暴力、赌博 月度更新 所有内容
广告法 「最」「第一」「国家级」「纯天然」 季度更新 营销推文
医疗 「治愈」「根治」「特效」「秘方」 半年更新 健康/医美内容
金融 「保本」「稳赚」「零风险」 半年更新 理财/保险内容
品牌自定义 竞品名、品牌禁止词 按需 所有品牌内容

industry rule 是可配置的合规规则引擎。支持正则表达式和条件规则：

yaml 
industry_rules:
  medical:
    - pattern: "(治愈|根治|特效|秘方)"
      action: block
      message: "医疗广告禁用承诺性疗效"
    - pattern: "(纯天然|无副作用|安全无毒)"
      action: flag
      message: "需药监局批文审核"

  finance:
    - pattern: "(保本|稳赚|零风险|收益保证)"
      action: block
      message: "金融宣传禁止保证收益"
    - condition: "有收益率且无'历史收益不代表未来'"
      action: flag

有害内容拦截与 Moderation

harmful block 在敏感词匹配之外，还需 Moderation API 检测语义级违规（如暗示、隐喻、对抗性文本）：

有害内容三层拦截架构
flowchart TB
    A[生成内容] --> B[敏感词 AC 自动机
O(n) 匹配]
    B -->|命中 Block| C[直接拦截 + 告警]
    B -->|通过| D[Moderation API
语义检测]
    D -->|有害 > 0.8| C
    D -->|可疑 0.5-0.8| E[人工复核]
    D -->|安全| F[合规通过]

    G[行业规则引擎] --> B

三阶段拦截：词级别→语义级别→人工兜底。

不推荐
  • 只有敏感词匹配，不调 Moderation API
  • 违规内容直接删除无告警
  • 所有行业用同一套词库
推荐
  • AC 自动机 + Moderation API 双检测
  • 违规内容记录+告警+通知运营
  • 按行业加载专属词库

审计留痕

audit trail 记录每篇内容从生成到发布的全生命周期。合规审计时可以说「每条内容都有据可查」。

content log 的字段设计：

  • 生成阶段：模板 ID + 版本、渲染变量、LLM model + 参数、原始 prompt + 输出
  • 审核阶段：自动审核结果 + 命中规则、人工审核人 + 操作时间 + 结论
  • 重写阶段：驳回原因、重写轮次、修改前后 diff
  • 发布阶段：发布时间、渠道、操作人

审计日志存储

python 
# 审计日志写入
class AuditLogger:
    def __init__(self, db):
        self.db = db

    def log(self, content_id: str, stage: str,
            action: str, detail: dict, operator: str = "system"):
        """记录审计日志"""
        entry = {
            "content_id": content_id,
            "stage": stage,          # generate/audit/rewrite/publish
            "action": action,
            "detail": json.dumps(detail, ensure_ascii=False),
            "operator": operator,
            "timestamp": datetime.utcnow().isoformat()
        }
        self.db.insert("audit_log", entry)

    def get_trail(self, content_id: str) -> list:
        """获取完整审计链"""
        return self.db.query(
            "SELECT * FROM audit_log WHERE content_id=? ORDER BY timestamp",
            [content_id])

常见问题（FAQ）

行业规则更新后，已发布的历史内容需要重新审核吗？
取决于行业。广告法更新后，通常需要对在架内容做「合规重扫描」，标记违规内容下架。医药行业要求 24 小时内处理。
Moderation API 的误判率多高？
主流 Moderation API（如通义、OpenAI）的误判率在 1-3%。建议结合规则引擎降低误杀——只有规则 + API 都标记才拦截。
审计日志需要存多久？
行业法规通常要求 1-3 年。金融行业要求 5 年。推荐启用日志归档，90 天内的日志在线查询，更早的归档到对象存储。

本章结论

  • 敏感词库：通用 + 行业 + 品牌三层，AC 自动机 O(n) 匹配。
  • 行业规则引擎：医疗/金融/食品各专属规则，可配置。
  • 有害拦截：规则 + Moderation API 双检测，人工兜底。
  • 审计留痕：全链路日志，从生成到发布每步可追溯。
  • 合规下线：规则更新后，重新扫描在架内容。
