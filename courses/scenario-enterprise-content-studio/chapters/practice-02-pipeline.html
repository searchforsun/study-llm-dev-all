三层审核流水线设计

audit pipeline 由自动审核、人工复核、驳回重写三个环节串联，每个环节有明确的进入和退出条件：

自动审核：敏感词 + Moderation API

content safety 通过多层检测实现自动准入：

  • 敏感词匹配：AC 自动机匹配行业词库（广告法禁用词、医疗违禁词等）。
  • Moderation API：调用通义 Moderation 接口检测涉政、色情、暴力等有害内容。
  • 品牌规则：自定义品牌规则（如不得提及竞品、不得使用「第一」「最」等极限词）。

Python：自动审核引擎

python 
# CorpAssist 自动审核引擎
from acora import AhoCorasick
from enum import Enum

class AuditResult(Enum):
    PASS = "pass"
    FLAG = "flag"    # 进人工复核
    BLOCK = "block"  # 直接拦截

class AutoAuditor:
    def __init__(self, word_lists: list[str]):
        self.ac = AhoCorasick()
        for wl in word_lists:
            with open(wl) as f:
                self.ac.add_words([w.strip() for w in f])
        self.ac.build()

    def scan_sensitive_words(self, text: str) -> list[str]:
        return list(self.ac.findall(text))

    def audit(self, text: str) -> tuple[AuditResult, list[str]]:
        # 敏感词检测
        found = self.scan_sensitive_words(text)
        if any(w in FIRE_LIST for w in found):
            return AuditResult.BLOCK, found

        # 调用 Moderation API（伪代码）
        moderate_result = call_moderation_api(text)
        if moderate_result["label"] == "harmful":
            return AuditResult.BLOCK, ["moderation: harmful"]
        if moderate_result["score"] > 0.7:
            return AuditResult.FLAG, ["moderation: suspicious"]

        return AuditResult.PASS, []

    def batch_audit(self, items: list) -> list:
        return [{"id": item["id"],
                 "result": self.audit(item["content"])[0].value,
                 "reasons": self.audit(item["content"])[1]}
                for item in items]

Java：Spring 审核配置

java 
// Spring 审核流水线配置
@Component
public class AuditPipeline {

    private final List<AuditStep> steps = List.of(
        new SensitiveWordStep(),
        new ModerationApiStep(),
        new BrandRuleStep()
    );

    public AuditResult execute(ContentItem item) {
        for (AuditStep step : steps) {
            AuditResult r = step.check(item);
            if (r != AuditResult.PASS) {
                item.addAuditLog(step.name(), r);
                return r;  // 非 PASS 即终止流水线
            }
        }
        return AuditResult.PASS;
    }
}

interface AuditStep {
    String name();
    AuditResult check(ContentItem item);
}
先想 10 秒：自动审核的误杀率（好内容被拦）和漏杀率（违规内容通过）哪个更难接受？

漏杀率更难接受——违规内容发布出去可能造成品牌声誉损失或法律风险。宁可误杀偏高，通过人工复核兜底。

人工复核与驳回重写

human review 是审核流程中唯一的人工环节。设计要点：

  • 待审队列：按优先级排序（疑似违规 > 品牌调性 > 常规抽检）。
  • 审核面板：显示原文 + AI 生成内容 + 自动审核标记，一次对比。
  • 驳回原因：结构化选项（用词不当、事实错误、语调不符）+ 自由输入。

reject rewrite 的闭环流程：驳回原因 → LLM 根据原因重写 → 重新走审核流水线。

不推荐
  • 驳回原因填写「不行，重写」，无具体指向
  • 重写后直接发布，不走审核
  • 无重写次数限制，陷入死循环
  • 人工复核无 SLA，积压越来越多
推荐
  • 驳回原因结构化：类别 + 具体描述
  • 重写后必须重新审核
  • 最多重写 3 次，超限转人工写
  • 审核队列 SLA：1 小时内响应

class RewriteLoop:
    MAX_RETRIES = 3

    def rewrite(self, content: str,
                reject_reason: str, attempt: int) -> str:
        prompt = f"""你是一位文案编辑。以下内容被驳回，请根据原因修改。

原内容：{content}

驳回原因：{reject_reason}

修改要求：
1. 仅修改驳回原因指出的问题
2. 保持品牌语调一致
3. 不要改变核心信息

请输出修改后的完整文案。"""
        return llm_client.chat(prompt)

    def review_loop(self, item: dict) -> str:
        content = item["content"]
        for i in range(self.MAX_RETRIES):
            result = pipeline.execute(content)
            if result == "pass":
                return content
            if result == "block":
                raise ValueError("内容被拦截")
            # flag 或 reject：从审核结果中获取驳回原因
            reason = item.get("reject_reason", "不符合要求")
            content = self.rewrite(content, reason, i + 1)
        # 超限，转人工处理

常见问题（FAQ）

自动审核的敏感词词库怎么维护？
建立三层词库：① 行业合规层（广告法、医疗法规）——法务维护，季度更新；② 品牌自定义层（竞品名、品牌违禁词）——运营维护；③ 实时热词层（突发舆情相关词）——AI 自动标注+人工确认。
人工复核队列积压怎么处理？
设置三级 SLA：队列深度 < 50 条正常流转；50-200 条触发告警，自动加急最早进入的 20 条；> 200 条触发熔断——暂停批量生成，优先处理积压。
驳回重写会不会越改越差？
会。限制重写次数（≤3 次），每次重写只修改驳回原因指出的问题，不改变整体结构。如果 3 次后仍未通过，说明模板本身有问题，需要人工调整模板而不是继续改内容。

本章结论

  • 三层审核：自动 → 人工 → 驳回重写，每条内容有完整状态机跟踪。
  • 自动审核：敏感词 AC 自动机 + Moderation API + 品牌规则，拦截 90% 违规。
  • 驳回重写：最多 3 轮，驳回原因结构化，重写后重新审核。
  • SLA 保障：复核队列三级告警，防止积压。
  • 审计追溯：生成→审核→发布全链路日志。
