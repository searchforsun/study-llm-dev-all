OpenAPI 导出

CorpAssist · Java 客户端生成 Spring 团队要用 openapi-generator 生成 CorpassistPythonClient。你需要在每次 MINOR 发版前提交 openapi.json 到契约仓库，或 CI 自动上传制品库。

运行时导出（推荐联调）

bash 
# 服务启动后
curl -s http://localhost:8000/openapi.json -o openapi.json
# 或访问 /docs  UI 下载

离线脚本导出（CI 无启动服务时）

python 
import json
from corpassist.api.main import app

schema = app.openapi()
# 统一 info.version 与 pyproject
schema["info"]["version"] = "0.2.0"
Path("artifacts/openapi.json").write_text(
    json.dumps(schema, indent=2, ensure_ascii=False),
    encoding="utf-8",
)
导出方式优点注意
HTTP /openapi.json 与运行配置一致（含动态 servers） 需服务已启动且路由已注册
app.openapi() 脚本 CI 快速、可 diff lifespan 副作用需在 import 时避免连真 DB

导出物应包含：/health、/api/v1/rag/search、/api/v1/index/jobs 及统一错误 schema（code、message、request_id）。

FAQ：openapi.json 要不要提交 Git？

推荐提交到 artifacts/openapi.json 或独立 contract-repo。PR 对 openapi 做 diff 可让 Java 同事提前知晓字段变更；若仅运行时导出，则 CI artifact 至少保留 30 天供下载。

自定义 OpenAPI metadata

CorpAssist 导出物应包含稳定的 info 与 servers，便于 Java 客户端生成与 staging 联调：

python 
def custom_openapi(app: FastAPI) -> dict:
    if app.openapi_schema:
        return app.openapi_schema
    schema = get_openapi(
        title="CorpAssist Python API",
        version=settings.app_version,
        routes=app.routes,
    )
    schema["servers"] = [{"url": settings.public_base_url}]
    app.openapi_schema = schema
    return schema
字段来源用途
info.versionpyproject.toml与 Docker tag / semver 对齐
servers[0].urlCORPASSIST_PUBLIC_BASE_URLBFF 生成 Client basePath
components.schemasPydantic modelsJava DTO 生成真源

与 Spring 契约对齐

Contract alignment 不是「口头说字段一样」，而是可自动化的 schema diff + 评审。CorpAssist 约定：

  • JSON 字段 snake_case（Python 原生）；Java 用 @JsonNaming(SnakeCaseStrategy.class) 或 OpenAPI generator 配置。
  • 日期时间统一 ISO-8601 字符串；枚举值两边共用常量表（见 双栈协作规范）。
  • 错误响应结构对齐 Spring ProblemDetail 或 CorpAssist 统一 ErrorBody。
契约评审流程
flowchart LR
  py["Python 导出 openapi.json"]
  diff["openapi-diff / oasdiff"]
  java["Java 团队评审"]
  repo["contract-repo 合并"]
  py --> diff --> java --> repo

MINOR 变更：diff 仅新增 optional 字段 → 自动通过；删除/改类型 → 需 MAJOR 与 BFF 排期。

漂移风险
  • Python 改 Pydantic 未导出 OpenAPI
  • Java 手写 DTO 与文档不一致
  • 仅测 happy path，422 结构未对齐
对齐做法
  • CI 提交 openapi.json artifact
  • Consumer driven：BFF 提 PR 改契约
  • Contract test 校验样例 JSON

集成测试

Integration test 验证「HTTP 进 → 路由 → Mock LLM → JSON 出」，不替代单元测试，但能在发版前捕获路由注册与序列化问题。

TestClient + MockTransport

python 
import pytest
from httpx import ASGITransport, AsyncClient, MockTransport, Response

@pytest.fixture
def mock_llm_transport():
    def handler(request):
        return Response(200, json={"data": [{"embedding": [0.1, 0.2]}]})
    return MockTransport(handler)

@pytest.mark.asyncio
async def test_search_returns_hits(app_with_mocks):
    transport = ASGITransport(app=app_with_mocks)
    async with AsyncClient(transport=transport, base_url="http://test") as ac:
        r = await ac.post("/api/v1/rag/search", json={"query": "政策", "top_k": 3})
    assert r.status_code == 200
    body = r.json()
    assert "hits" in body
    assert body["request_id"]

集成测试应覆盖：200 成功、422 校验失败、401 缺 API Key、503 下游超时（Mock 延迟）。与 Java 侧 Testcontainers 调 Python 容器是下一层 E2E，本章以 Python 仓内集成为主。

先想 10 秒：集成测试为何仍要 Mock LLM，而不是连 staging 网关？

CI 需稳定、无密钥、无配额；staging 联调放在交付检查单的「环境验证」步，而非每次 PR。

测试分层与覆盖率目标

层级范围Mock 策略CI 频率
单元 Pydantic、纯函数、AppError 无 I/O 每次 PR
集成 FastAPI 路由 + 依赖注入 MockTransport / 假 Redis 每次 PR
契约 样例 JSON vs openapi.json 固定 fixture MINOR 发版前
E2E BFF → Python 容器 staging 网关或 Mock LLM 交付检查单

集成测试覆盖率目标：核心路由（search、index job、health）语句覆盖 ≥ 80%；异常分支至少覆盖 422、401、502 三类。

API 版本与 semver 对照

CorpAssist 采用 URL 版本前缀 /api/v1；破坏性变更升 v2，非破坏性变更仅 bump MINOR：

变更类型示例semverURL 版本
新增 optional 字段SearchResponse 加 trace_idMINOR保持 v1
删除字段 / 改类型top_k 从 int 改 stringMAJOR升 v2 或并行
新增端点POST /api/v1/rag/reindexMINOR保持 v1
错误码新增RAG_INDEX_BUSYMINOR保持 v1

发版 PR 须同时更新 CHANGELOG.md、pyproject version 与 openapi.json 的 info.version，三处一致方可合并 release 分支。

判断练习

  1. （场景）BFF 联调时 search 返回 200 但 Java 反序列化失败：Python 返回 camelCase requestId，契约写 snake_case。应改哪一侧、如何避免再发？

    参考答案

    Python 侧 Pydantic 应配置 alias 或统一 snake_case 输出；CI 加 contract test 对比样例 JSON 与 openapi.json schema；发版前 schema diff 拦截。
