错误码统一

CorpAssist · 前端同一 toast 用户搜索知识库无结果时，BFF 与直连 Python 应返回相同 code: RAG_NOT_FOUND，前端一套 i18n 映射。若 Python 返回 not_found、Java 返回 RAG_NOT_FOUND，联调会陷入 endless 对齐会议。

FastAPI exception_handler

python 
from fastapi import Request
from fastapi.responses import JSONResponse

@app.exception_handler(AppError)
async def app_error_handler(request: Request, exc: AppError) -> JSONResponse:
    rid = request.state.request_id
    return JSONResponse(
        status_code=exc.http_status,
        content={"code": exc.code, "message": exc.message, "request_id": rid},
    )

日志侧：structlog 记录 code、request_id；Java Logback MDC 同键，便于 ELK request_id:"uuid" 联查（MDC trace）。

错误码变更流程
  1. 在共享 error_codes 表新增一行并 PR 双栈评审
  2. Python 增加 AppError 子类 + handler 映射
  3. Java common 模块增加枚举 + i18n 文案
  4. 补充 contract test 样例 JSON

DTO 映射

DTO mapping 解决三类差异：命名（snake/camel）、可选字段默认值、枚举与日期格式。原则：OpenAPI schema 为真源，避免手写两套模型漂移。

python 
class SearchRequest(BaseModel):
    model_config = ConfigDict(extra="forbid")
    query: str = Field(min_length=1, max_length=2000)
    top_k: int = Field(default=5, ge=1, le=50)
    # JSON 输出 snake_case（默认）
java 
@JsonNaming(PropertyNamingStrategies.SnakeCaseStrategy.class)
public record SearchRequest(
    @NotBlank String query,
    @Min(1) @Max(50) int topK  // Java 字段 camel，JSON snake
) {}
双份手写
  • Python 改字段未通知 Java
  • BFF 层手动 Map 漏字段
  • 枚举 int vs string 不一致
契约驱动
  • openapi.json → Java generator
  • 共享 corpassist-schemas 包（advanced-03）
  • contract test 固定样例 JSON

可选字段与 null 语义

双栈常见漂移点：filters: null vs 省略字段 vs {}。CorpAssist 约定：

  • OpenAPI 中 optional 字段用 nullable: true 或 Python Field(default=None) 显式声明。
  • JSON 输出省略 None 值（Pydantic model_dump(exclude_none=True)），Java 侧用 Optional 接收。
  • 禁止 Python 返回 null 字符串或空对象混用 — contract test 固定 3 组样例 JSON 校验。
python 
class SearchResponse(BaseModel):
    hits: list[Hit]
    request_id: str
    trace_id: str | None = None

    def to_json(self) -> dict:
        return self.model_dump(exclude_none=True)  # 无 trace 时不输出 null

联调清单

Joint debug 标准流程减少「在我机器上能跑」。CorpAssist 推荐每次 MINOR 发版前走一遍：

  1. 环境 — compose 起 Python + redis + mock-llm；BFF 指向 http://corpassist-python:8000。
  2. 契约 — 拉取同 tag 的 openapi.json，oasdiff 与 BFF 依赖版本一致。
  3. 冒烟 — GET /health；带 X-Request-Id 调 POST /api/v1/rag/search（定稿路径；practice 章 /v1/rag/query 经 BFF 映射）。
  4. 日志 — 用同一 request_id 在 Python JSON 日志与 Java MDC 日志各查到一条。
  5. 异常 — 触发 RAG_NOT_FOUND 与 LLM_RATE_LIMIT（Mock），核对 body.code。
  6. 长任务 — BFF 提交 index job，轮询至 succeeded（advanced-02）。

内网联调应配置中间件：缺 header 则生成并回传；生产 BFF 必须转发，便于 BFF pattern 统一入口追踪。

答辩题：为何 Python 服务不并入 Maven 父工程？

工具链（uv/Docker）、发布节奏与运行时不同；集成靠 shared contract 而非编译耦合。Python 专注 RAG/LLM；Spring 专注网关、账号与 Spring MVC 生态。
