为什么要学本章

没有 mock LLM 的测试会在 CI 里烧钱且不稳定。CorpAssist-python 需在合并前验证路由、依赖与错误映射，并与 Spring 侧共享 OpenAPI 契约。本章建立可重复、零外部网络的测试基线。

学完你能
  • 用 pytest 组织测试目录与 fixture
  • 用 httpx ASGITransport 测 async FastAPI，无需真实端口
  • 用 MockTransport 模拟 LLM JSON 响应
  • 入门 契约测试，与 Java 消费方对齐 schema
本章先记住 3 件事
  • CI 零外网：测试不得调用真实模型网关；mock 固定 JSON。
  • 测行为不测实现：断言 HTTP 状态与响应字段，少测私有函数。
  • 契约先于联调：OpenAPI/schema 快照失败则阻断发布。

pytest

CorpAssist · PR 门禁 每次 PR 触发 GitHub Actions：ruff、mypy、pytest。测试必须在 2 分钟内跑完且不依赖 VPN 访问内网 LLM。

pytest 是 Python 事实标准测试框架，类比 JUnit 5 + Spring Boot Test。CorpAssist 约定测试目录 tests/，文件命名 test_*.py，与源码分离。

基础测试结构

python 
# tests/test_health.py
import pytest

@pytest.mark.parametrize("path", ["/health"])
def test_paths_exist(path):
    assert path.startswith("/")

使用 parametrize 覆盖多组输入；coverage 报告可选，PR 门槛建议核心包 ≥80%。

TestClient

FastAPI 基于 ASGI，可用 httpx 的 ASGITransport 在进程内调用 app，无需 uvicorn 起端口。同步与异步测试均支持。

异步 TestClient 测 /health

python 
import pytest
from httpx import ASGITransport, AsyncClient
from main import app

@pytest.fixture
async def client():
    transport = ASGITransport(app=app)
    async with AsyncClient(transport=transport, base_url="http://test") as ac:
        yield ac

@pytest.mark.asyncio
async def test_health_ok(client):
    resp = await client.get("/health")
    assert resp.status_code == 200
    body = resp.json()
    assert body["status"] == "ok"

dependency_overrides 注入 mock Settings

python 
from main import app, get_settings

@pytest.fixture
def override_settings():
    class FakeSettings:
        service_name = "test-service"
    app.dependency_overrides[get_settings] = lambda: FakeSettings()
    yield
    app.dependency_overrides.clear()

类比 Spring @MockBean + @WebMvcTest：只替换被测边界外的依赖。

先想 10 秒：为何测试用 ASGITransport 而非 localhost:8000？

避免端口冲突与启动慢；测试与 app 同进程，fixture 可安全 override 依赖。集成环境仍保留少量 e2e 打真实端口。

mock LLM

LLM 响应体大、耗时长、带随机性。单元/集成测试应通过 httpx mock 返回固定 JSON，验证业务解析与错误分支。

MockTransport 固定 Chat 响应

python 
import httpx
import pytest

MOCK_CHAT_JSON = {
    "id": "chatcmpl-demo",
    "choices": [{"index": 0, "message": {"role": "assistant", "content": "mock answer"}}],
    "usage": {"prompt_tokens": 1, "completion_tokens": 2},
}

def handler(request: httpx.Request) -> httpx.Response:
    return httpx.Response(200, json=MOCK_CHAT_JSON)

@pytest.mark.asyncio
async def test_llm_client_parses_mock():
    transport = httpx.MockTransport(handler)
    async with httpx.AsyncClient(transport=transport, base_url="http://llm") as client:
        resp = await client.post("/v1/chat/completions", json={"model": "demo"})
    assert resp.json()["choices"][0]["message"]["content"] == "mock answer"
测试分层：mock LLM 所在位置
flowchart TB
  test["pytest"]
  api["FastAPI route"]
  svc["LlmService"]
  mock["MockTransport"]
  test --> api
  api --> svc
  svc --> mock
  mock -.->|不触达| real["真实网关"]

路由测试走完整 handler；仅 HTTP 边界被 mock。

  • 429/503 场景：handler 返回对应状态码，断言重试或错误映射。
  • 禁止在 CI secret 里放真实 API Key 做「集成测试」。

契约测试入门

契约测试 保证 Python 发布的 OpenAPI 与 Spring BFF 消费方期望一致。入门做法：导出 schema 快照，PR diff 失败则人工审查。

OpenAPI schema 快照

python 
import json
from main import app

def test_openapi_schema_snapshot(snapshot):
    schema = app.openapi()
    # 或使用 pytest-snapshot / 黄金文件
    assert schema["info"]["title"] == "CorpAssist Python"
    paths = schema["paths"]
    assert "/health" in paths
    assert "/v1/rag/query" in paths
    rag_post = paths["/v1/rag/query"]["post"]
    assert "RagQueryRequest" in json.dumps(rag_post)
层级测什么工具
单元Pydantic 解析、工具函数pytest
APIHTTP 状态与 JSON 字段ASGITransport
契约OpenAPI paths/schemasschema 快照
e2eCompose 全栈见 容器化与本地 compose

进阶：consumer driven 契约由 Java 团队提供期望 JSON，Python CI 校验（实战：CorpAssist Python 服务 API 展开）。

错误分支也要测

python 
@pytest.mark.asyncio
async def test_rag_query_validation_error(client):
    resp = await client.post("/v1/rag/query", json={"query": ""})
    assert resp.status_code == 422
    detail = resp.json()["detail"]
    assert any("query" in str(e.get("loc", [])) for e in detail)

（场景）测试应覆盖 422 校验、401 鉴权（见 中间件、鉴权与 CORS）与 504 超时映射，而非只测 happy path。

  • pytest + ASGITransport：路由与校验
  • MockTransport：LLM 客户端单元测试
  • OpenAPI path 断言：契约回归
  • docker compose e2e（见 容器化与本地 compose）
  • Spring @SpringBootTest 调 Python /health
  • schema diff 与 Java 消费方对齐

本章结论

每条对应上一节 h3，测试策略四要素复述。

动手练习

在 demos/practice-03-testing-lab/ 运行 pytest，并对照正文补一条 mock LLM 的 MockTransport 测试与 OpenAPI path 断言。

操作步骤

  1. 阅读「异步 TestClient 测 /health」代码块，确认 Demo test_health.py 中断言与之一致。
  2. 按「MockTransport 固定 Chat 响应」示例新增 test_mock_llm.py，断言 content == "mock answer"。
  3. 对照「测试分层」Mermaid，在笔记中标出 mock 应插在 LlmService 还是 route 层。
  4. 运行 pytest -q，期望至少 2 passed；加 --cov=main（可选）查看覆盖率。

判断练习

  1. 下列哪条违反 CorpAssist CI 测试约定？A) MockTransport 返回 200 JSON；B) PR 测试读取 CORPASSIST_LLM_API_KEY 调真实网关；C) schema 快照比对 /health path。

    参考答案

    选 B。CI 不得依赖真实 LLM 与外网密钥；费用、 flaky 与密钥泄露风险均不可接受。A、C 均为推荐做法。

  2. 测试里忘记 app.dependency_overrides.clear() 会导致什么 flaky 现象？

    参考答案

    后续测试仍使用被 override 的 fake Settings，顺序敏感导致「单独跑过、全套失败」。fixture 应在 teardown 清除 overrides。
