API Key

CorpAssist · 服务间调用 BFF 调 Python RAG 时携带共享密钥 X-API-Key，防止内网其他服务误调用。密钥来自环境变量，不进 Git。

API Key 适合 machine-to-machine；比 JWT 简单，适合 BFF → Python 固定信任链。校验失败返回 401，body 与 Java AppError 格式对齐（见 异常、日志与配置）。

Depends 校验 API Key

python 
from fastapi import Depends, HTTPException, Security
from fastapi.security import APIKeyHeader

_api_key_header = APIKeyHeader(name="X-API-Key", auto_error=False)

async def verify_api_key(
    key: str | None = Security(_api_key_header),
    settings: CorpAssistSettings = Depends(get_settings),
) -> None:
    if not key or key != settings.internal_api_key:
        raise HTTPException(status_code=401, detail={"code": "UNAUTHORIZED", "message": "Invalid API key"})

FAQ：API Key 与 OAuth2 用户登录冲突吗？

不冲突。API Key 标识服务（BFF→Python）；用户 OAuth2 在 Spring 完成。Python 可同时校验 Key（服务信任）与 JWT（用户身份），见下一节。

JWT 对接 Spring

用户身份由 Spring 鉴权后，BFF 将 JWT 转发给 Python（或换发内部 token）。Python 侧验证签名、issuer、exp，提取 sub / tenant_id 供审计。

解析 JWT claims（HS256 示例）

python 
import jwt
from fastapi import Header, HTTPException

async def get_current_user(authorization: str | None = Header(default=None)) -> dict:
    if not authorization or not authorization.startswith("Bearer "):
        raise HTTPException(status_code=401, detail="Missing bearer token")
    token = authorization.removeprefix("Bearer ").strip()
    try:
        payload = jwt.decode(
            token,
            settings.jwt_secret,
            algorithms=["HS256"],
            issuer="corpassist-spring",
        )
    except jwt.PyJWTError as exc:
        raise HTTPException(status_code=401, detail="Invalid token") from exc
    return {"user_id": payload["sub"], "tenant": payload.get("tenant_id")}

生产可用 RS256 + JWK 集；issuer / audience 必须与 Spring Security 配置一致。双栈联调清单见 双栈协作规范。

Spring 与 Python JWT 字段对照

ClaimSpring 侧Python 使用
isscorpassist-springjwt.decode(..., issuer=...) 必校验

FAQ：Python 还要再登录吗？

通常不。BFF 已完成 OAuth2/Session，Python 信任内网 mTLS + API Key + 短 JWT。勿在 Python 重复实现完整 OIDC，除非服务直接对公网。

请求 ID

request_id 贯穿 Spring MDC、Python structlog 与 ELK。中间件：读入 X-Request-Id，无则生成 UUID，写回响应头并绑定日志上下文。

RequestIdMiddleware

python 
import uuid
from starlette.middleware.base import BaseHTTPMiddleware
from starlette.requests import Request

class RequestIdMiddleware(BaseHTTPMiddleware):
    async def dispatch(self, request: Request, call_next):
        rid = request.headers.get("X-Request-Id") or str(uuid.uuid4())
        # structlog.contextvars.bind_contextvars(request_id=rid)
        response = await call_next(request)
        response.headers["X-Request-Id"] = rid
        return response

app.add_middleware(RequestIdMiddleware)

structlog 绑定 request_id

python 
import structlog

class RequestIdMiddleware(BaseHTTPMiddleware):
    async def dispatch(self, request: Request, call_next):
        rid = request.headers.get("X-Request-Id") or str(uuid.uuid4())
        structlog.contextvars.clear_contextvars()
        structlog.contextvars.bind_contextvars(request_id=rid)
        response = await call_next(request)
        response.headers["X-Request-Id"] = rid
        return response

与 异常、日志与配置 中 structlog JSON 字段一致，便于 ELK 与 Java MDC trace 联查。

方便客户端与 BFF 关联错误响应；若无入站 id，Python 生成的 id 需回传以便 Spring 日志对齐。

超时

LLM 与 RAG 链路耗时长，但 HTTP 连接不能无限挂起。timeout 分层：网关全局、Python middleware、httpx 客户端 per-request。

中间件超时（示意）

python 
import asyncio
from starlette.middleware.base import BaseHTTPMiddleware
from starlette.responses import JSONResponse

class TimeoutMiddleware(BaseHTTPMiddleware):
    def __init__(self, app, seconds: float = 60.0):
        super().__init__(app)
        self.seconds = seconds

    async def dispatch(self, request, call_next):
        try:
            return await asyncio.wait_for(call_next(request), timeout=self.seconds)
        except asyncio.TimeoutError:
            return JSONResponse(
                status_code=504,
                content={"code": "GATEWAY_TIMEOUT", "message": "Request timed out"},
            )
  • 流式 SSE 端点应排除或放宽全局 timeout，改用 idle timeout。
  • httpx timeout=httpx.Timeout(30.0, connect=5.0) 控制上游 LLM。
  • 与 Spring Cloud Gateway response-timeout 协调，避免一层先断一层仍等。

CORS 与 security header（本地）

python 
from fastapi.middleware.cors import CORSMiddleware

if settings.env == "local":
    app.add_middleware(
        CORSMiddleware,
        allow_origins=["http://localhost:5173"],
        allow_methods=["GET", "POST"],
        allow_headers=["Authorization", "X-API-Key", "X-Request-Id"],
    )

生产环境 Python 服务通常仅内网可达，CORS 由 Spring BFF 对外处理。可选添加 security header（X-Content-Type-Options 等）于网关层。

  • Spring Gateway response-timeout
  • Python TimeoutMiddleware 整请求上限
  • SSE 路由需排除或单独 idle 超时
  • httpx.Timeout(30.0, connect=5.0)
  • 仅约束上游 LLM HTTP
  • 与 middleware 超时不重复叠加过短

答辩题：为何 API Key 与 JWT 在 CorpAssist 里并存？各解决什么问题？

API Key 标识服务间调用（BFF→Python）；JWT 携带用户/租户身份供审计与多租户 RAG。二者校验层不同，勿用单一方案替代。

判断练习

  1. （排障）BFF 日志有 request_id，Python 日志没有。最可能漏了哪一步？

    参考答案

    RequestIdMiddleware 未注册，或 structlog 未 bind_contextvars(request_id=...)。BFF 已转发头，但 Python 未读头/未写入日志上下文。

  2. （找错）RAG 路由未加 API Key Depends，但 /health 返回 401。最可能误改了什么？

    参考答案

    把 verify_api_key 挂到了 app 全局 middleware 或根 router，而非仅 /v1/rag 子路由。health 探活不应要求 Key；应改为 router 级 dependencies 或按 path 前缀排除。
