约 50 分钟 · 阶段：生产与治理 · 能力：用 LLaMA-Factory 训练、评估、合并 Llama/Qwen 系列模型

为什么要学本章

LLaMA-Factory 是当前最流行的开源微调框架，支持 Llama、Qwen、Mistral 等主流模型。它封装了数据准备、LoRA/QLoRA 训练、评测和合并的完整流水线，是应用工程师替代手写 Python 训练脚本的最优选择。

学完你能
  • 用 LLaMA-Factory 命令行完成 LoRA/QLoRA 训练
  • 配置 YAML 训练文件：数据集、模型、超参、量化
  • 用 LLaMA-Factory 内置评测功能评估微调效果
  • 导出并合并 adapter 用于生产部署
本章先记住 3 件事
  • LLaMA-Factory 封装流水线：数据集 → 训练 → 评测 → 导出，一条命令完成。
  • 支持多种训练方法：LoRA、QLoRA、Full FT、DPO、PPO 全部内置。
  • 一键评测：内置 MMLU、C-Eval、BLEU/ROUGE 等评测集。

LLaMA-Factory 架构与安装

CorpAssist · 用 LLaMA-Factory 微调（90 秒） CorpAssist 团队已有清洗后的 JSONL 语料。不想手写 Python 训练脚本，希望用一个开箱即用的工具完成微调。LLaMA-Factory 提供 CLI 和 Web UI 两种界面，只需配置 YAML 文件即可启动训练。
bash 
# 安装 LLaMA-Factory
git clone https://github.com/hiyouga/LLaMA-Factory.git
cd LLaMA-Factory
pip install -e ".[torch,metrics]"

# 验证安装
llamafactory-cli version

# 启动 Web UI（可选）
llamafactory-cli webui

数据集注册与格式配置

LLaMA-Factory 使用 dataset_info.json 管理数据集。需要将 CorpAssist 语料注册其中。

dataset_info.json 配置

json 
{
  "corpassist_sft": {
    "file_name": "corpassist_v2_scrubbed.jsonl",
    "formatting": "sharegpt",
    "columns": {
      "messages": "conversations",
      "system": "system"
    },
    "tags": {
      "role_tag": "from",
      "content_tag": "value",
      "user_tag": "human",
      "assistant_tag": "gpt"
    }
  },
  "corpassist_dpo": {
    "file_name": "corpassist_dpo_v1.jsonl",
    "formatting": "sharegpt",
    "columns": {
      "messages": "conversations",
      "chosen": "chosen",
      "rejected": "rejected"
    },
    "tags": {
      "role_tag": "from",
      "content_tag": "value",
      "user_tag": "human",
      "assistant_tag": "gpt"
    }
  }
}

formatting: "sharegpt" 对应多轮对话格式，"alpaca" 对应三字段格式。LLaMA-Factory 支持两种主流格式，自动适配不同模型的 chat template。

训练配置：YAML 配置详解

yaml 
# corpassist_lora.yaml — LLaMA-Factory LoRA 训练配置
# 模型配置
model_name_or_path: Qwen/Qwen2.5-7B-Instruct
trust_remote_code: true

# 方法
stage: sft                          # sft / dpo / ppo
do_train: true
finetuning_type: lora               # lora / full / freeze
lora_target: all                    # 或指定: q_proj,k_proj,v_proj,o_proj
lora_rank: 16
lora_alpha: 32
lora_dropout: 0.05

# 数据集
dataset: corpassist_sft             # 对应 dataset_info.json 中注册的名称
template: qwen                      # 模型模板
cutoff_len: 2048                    # 最大序列长度
overwrite_cache: true
preprocessing_num_workers: 4

# 训练参数
per_device_train_batch_size: 4
gradient_accumulation_steps: 4
learning_rate: 2.0e-4
num_train_epochs: 3.0
lr_scheduler_type: cosine
warmup_ratio: 0.1
bf16: true

# 日志与保存
logging_steps: 10
save_steps: 100
save_total_limit: 2
output_dir: outputs/corpassist-lora-v2

# 量化（QLoRA）
quantization_bit: 4                  # 4-bit 量化训练
quantization_method: bitsandbytes    # bitsandbytes / hqq / eetq

# 报告
report_to: wandb
run_name: corpassist-lora-v2

内置评测与导出

LLaMA-Factory 内置了多种评测方法，训练完成后可直接评测。

yaml 
# corpassist_eval.yaml — 评测配置
model_name_or_path: outputs/corpassist-lora-v2
finetuning_type: lora
template: qwen

# 评测数据
dataset: corpassist_sft
eval_dataset: corpassist_holdout   # holdout 集

# 评测参数
do_eval: true
per_device_eval_batch_size: 8
max_new_tokens: 512
top_p: 0.9
temperature: 0.6

# 预测输出
predict_with_generate: true
generation_max_length: 512
output_dir: outputs/corpassist-lora-v2-eval

# MMLU / C-Eval 评测
evaluation_args:
  tasks:
    - mmlu
    - ceval
    - bleu
    - rouge
bash 
# 运行评测
llamafactory-cli eval corpassist_eval.yaml

# 导出合并模型
llamafactory-cli export corpassist_export.yaml

# 合并配置示例（export 子命令）
# model_name_or_path: Qwen/Qwen2.5-7B-Instruct
# adapter_name_or_path: outputs/corpassist-lora-v2
# export_dir: outputs/corpassist-lora-v2-merged
# export_size: 2
# export_legacy_format: false

导出配置示例

yaml 
# corpassist_export.yaml — 合并导出配置
model_name_or_path: Qwen/Qwen2.5-7B-Instruct
adapter_name_or_path: outputs/corpassist-lora-v2
export_dir: outputs/corpassist-lora-v2-merged
export_size: 2
export_legacy_format: false
template: qwen

多模型支持与 DPO 训练

LLaMA-Factory 不仅支持 SFT/LoRA，也支持 DPO 训练。以下是一个 DPO 训练配置：

yaml 
# corpassist_dpo.yaml — DPO 训练配置
model_name_or_path: Qwen/Qwen2.5-7B-Instruct
stage: dpo
do_train: true
finetuning_type: lora
lora_target: all
lora_rank: 8                    # DPO 通常用小 rank

dataset: corpassist_dpo          # DPO 偏好数据集
template: qwen
cutoff_len: 2048

pref_beta: 0.1                   # DPO beta 参数
pref_loss: sigmoid               # sigmoid / simpo / kto

per_device_train_batch_size: 2
gradient_accumulation_steps: 8
learning_rate: 5.0e-6            # DPO lr 比 SFT 小
num_train_epochs: 3.0

bf16: true
output_dir: outputs/corpassist-dpo-v1
report_to: wandb
先想 10 秒：LLaMA-Factory 和自己写 Python 训练脚本比，最大的优势是什么？

可复现性 + 零样板代码。YAML 配置 + 子命令确保了每个实验的完整参数被记录，不需要在长脚本中找超参定义。同时支持多数据集注册、模型模板自动适配、内置评测。

本章结论

  • LLaMA-Factory：开源微调框架，YAML 配置 + CLI 子命令。
  • 数据集注册：dataset_info.json 管理多个数据集。
  • 训练配置：支持 LoRA/QLoRA/Full FT/DPO，一行命令启动。
  • 内置评测：MMLU、C-Eval、BLEU/ROUGE 一键评估。
  • 导出合并：export 子命令直接产出合并模型。
