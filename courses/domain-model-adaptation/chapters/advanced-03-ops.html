为什么要学本章

微调模型上线后不是终点。数据漂移 会导致模型效果随时间下降，质量漂移 需要 再训练触发 机制。没有 模型监控，你会在用户说「这个 AI 变笨了」才发现问题。

学完你能

  • 检测 数据漂移：用户输入分布变化 vs 训练分布
  • 设定 再训练触发条件（定时 + 指标驱动）
  • 实现 反馈阈值告警
本章先记住 3 件事
  • 效果会衰减：业务变化 → 用户输入变化 → 模型回答质量下降。
  • 监控先行：上线第一天就部署监控，漂移数据是再训练的基础。
  • 再训练不是 bug fix：用数据飞轮驱动，而非临时修 bug。

模型监控：指标与告警

CorpAssist · 运维监控（90 秒） CorpAssist v2 上线后运行稳定，但第 3 周发现 CSAT 从 4.2 降到 3.8。排查发现：公司发布了新的人事制度，但模型训练语料中没有包含新制度。用户提问「新休假政策」时模型回答的是旧政策。这就是典型的数据漂移。

模型监控 应该覆盖以下维度：

# model_monitor.py — CorpAssist 模型监控
import time, json, statistics
from collections import deque
from typing import Dict, List

class CorpAssistMonitor:
    """CorpAssist 模型监控器"""

    def __init__(self, window_size: int = 1000):
        self.latencies = deque(maxlen=window_size)
        self.csat_scores = deque(maxlen=5000)
        self.safety_violations = 0
        self.total_queries = 0

    def record_query(self, start_time: float, response: str, csat: float = None):
        """记录一次查询的监控数据"""
        latency = time.time() - start_time
        self.latencies.append(latency)
        self.total_queries += 1

        if csat is not None:
            self.csat_scores.append(csat)

    def get_metrics(self) -> Dict:
        """获取当前监控指标"""
        if not self.latencies:
            return {"error": "no data"}

        sorted_lat = sorted(self.latencies)
        p50 = sorted_lat[len(sorted_lat) // 2]
        p99 = sorted_lat[int(len(sorted_lat) * 0.99)]

        avg_csat = statistics.mean(self.csat_scores) if self.csat_scores else 0

        return {
            "p50_latency": round(p50, 3),
            "p99_latency": round(p99, 3),
            "avg_csat": round(avg_csat, 2),
            "total_queries": self.total_queries,
            "safety_rate": round(self.safety_violations / max(self.total_queries, 1), 4),
        }

    def check_alerts(self) -> List[str]:
        """检查是否需要告警"""
        alerts = []
        metrics = self.get_metrics()
        if "error" in metrics:
            return alerts
        if metrics["p99_latency"] > 2.0:
            alerts.append(f"[P1] P99 延迟 {metrics['p99_latency']}s > 2s")
        if metrics["avg_csat"] < 3.5 and self.total_queries > 100:
            alerts.append(f"[P0] CSAT {metrics['avg_csat']} < 3.5")
        if metrics["safety_rate"] > 0.005:
            alerts.append(f"[P0] 安全违规率 {metrics['safety_rate']} > 0.5%")
        return alerts

数据漂移检测

数据漂移 检测比较线上用户输入的 Embedding 分布与训练集的 Embedding 分布。当分布差异（KL 散度或 Wasserstein 距离）超过阈值时触发告警。

python 
# drift_detection.py — 数据漂移检测
import numpy as np
from sklearn.feature_extraction.text import TfidfVectorizer
from scipy.spatial.distance import jensenshannon
from typing import List, Optional

class DriftDetector:
    """检测用户输入分布与训练分布之间的漂移"""

    def __init__(self, train_texts: List[str]):
        self.vectorizer = TfidfVectorizer(max_features=1000)
        # 训练集分布（参考分布）
        self.train_tfidf = self.vectorizer.fit_transform(train_texts)
        self.train_dist = np.mean(self.train_tfidf.toarray(), axis=0)

    def detect(self, online_texts: List[str], threshold: float = 0.1) -> Optional[dict]:
        """检测线上输入与训练集的分布差异"""
        online_tfidf = self.vectorizer.transform(online_texts)
        online_dist = np.mean(online_tfidf.toarray(), axis=0)

        # 计算 JS 散度
        js_div = jensenshannon(self.train_dist, online_dist)

        result = {
            "js_divergence": round(float(js_div), 4),
            "drift_detected": js_div > threshold,
            "sample_size": len(online_texts),
        }

        if js_div > threshold:
            result["alert"] = f"数据漂移: JS={js_div:.4f} > {threshold}"
        return result

# 使用示例
detector = DriftDetector(train_texts)
result = detector.detect(online_texts[-100:], threshold=0.1)
print(result)

再训练触发条件

再训练触发 是运维的核心决策机制。不应等模型「彻底不行」才训练，也不应频繁训练浪费资源。

yaml 
# retrain_policy.yaml — CorpAssist 再训练策略
retrain:
  # 定时策略
  schedule:
    type: cron
    expression: "0 0 1 * *"     # 每月 1 号凌晨
    reason: "定期迭代"

  # 指标触发
  triggers:
    - metric: csat_weekly_avg
      operator: lt
      threshold: 3.5
      cooldown_days: 14          # 触发后 14 天内不重复触发
      reason: "CSAT 低于阈值"

    - metric: drift_js_divergence
      operator: gt
      threshold: 0.15
      cooldown_days: 7
      reason: "输入分布漂移"

    - metric: transfer_rate
      operator: gt
      threshold: 0.12
      cooldown_days: 14
      reason: "人工转接率过高"

  # 触发后的动作
  on_trigger:
    - action: notify_slack
      channel: "#corpassist-alerts"
    - action: collect_badcases
      count: 200
    - action: create_ft_ticket
      assignee: ml-team

微调模型运维 的黄金指标：CSAT 趋势是 业务健康度，延迟趋势是 技术健康度，漂移指标是 模型时效性。

本章结论

  • 监控：延迟 + CSAT + 安全违规 + 人工转接率 四维监控。
  • 漂移检测：Embedding 分布的 JS 散度，阈值 0.1。
  • 再训练触发：定时（每月）+ 指标驱动（CSAT/漂移/转接率）。
  • 告警分级：P0 即时处理，P1 工作时间处理。
