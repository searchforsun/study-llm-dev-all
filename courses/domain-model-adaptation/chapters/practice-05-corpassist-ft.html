项目全景：CorpAssist v2 领域适配

CorpAssist · 领域适配 capstone（120 秒） CorpAssist v1 采用基座模型 + RAG + Prompt 的架构。用户反馈：回答太「AI 味」，缺少企业客服的温暖感。项目目标：通过 LoRA 微调实现话术对齐，让 CorpAssist 回答听起来像「专业的企业客服」而非「通用 AI 助手」。

话术规范设计

垂直话术 是领域适配的核心资产。CorpAssist 的话术规范包含五个维度：

维度要求正面示例负面示例
开场白固定：「您好，我是 CorpAssist 小C」您好，我是 CorpAssist 小C你好，有什么问题？
语气专业、温和、礼貌非常抱歉给您带来不便不好意思啊（过于随意）
流程投诉：确认→记录→方案→满意确认请问您遇到了什么问题？我来为您记录你投诉什么？说清楚
安全不输出 PII、不怂恿投诉我会为您记录并转交相关负责人建议你打 12345 投诉
结束语固定：「感谢您的咨询」感谢您的咨询，祝您工作愉快拜拜（过于随意）

品牌语调 需要团队共同评审并形成文档。话术规范文档是 SFT 数据标注的黄金标准。

markdown 
# CorpAssist 话术规范 v2.0

## 开场白
所有回答必须以「您好，我是 CorpAssist 小C」开头。

## 投诉处理四步法
1. 确认情绪：「非常抱歉给您带来不好的体验」
2. 记录问题：「请告诉我具体发生的场景」
3. 给出方案：「我将为您记录并转交相关部门」
4. 确认满意：「请问您还有其他问题需要处理吗？」

## 禁止表达
- ❌ 建议投诉到外部机构
- ❌ 输出任何个人联系方式
- ❌ 对公司政策做负面评价

端到端交付流水线

交付流水线 将前几章的各个环节串起来。以下是一个完整的交付脚本：

python 
# corpassist_delivery.py — CorpAssist 端到端交付
import subprocess, json, yaml
from pathlib import Path

class CorpAssistDelivery:
    """CorpAssist 领域适配交付管理器"""

    def __init__(self, config_path: str = "delivery_config.yaml"):
        with open(config_path) as f:
            self.config = yaml.safe_load(f)
        self.steps = []

    def step_data_prep(self):
        """Step 1: 数据准备与清洗"""
        print("[1/7] 语料准备...")
        subprocess.run(["python", "validate_jsonl.py", self.config["data_path"]])
        subprocess.run(["python", "dedup.py"])
        subprocess.run(["python", "quality_scan.py"])
        self.steps.append("data_ready")

    def step_training(self):
        """Step 2: LoRA 训练"""
        print("[2/7] LoRA 训练...")
        subprocess.run([
            "python", "train_lora.py",
            "--config", self.config["train_config"],
            "--output", self.config["output_dir"],
        ])
        self.steps.append("trained")

    def step_eval(self):
        """Step 3: 评测对比"""
        print("[3/7] 评测...")
        subprocess.run(["python", "gate_check.py",
            "--base", self.config["base_metrics"],
            "--ft", self.config["ft_metrics"],
        ])
        self.steps.append("evaluated")

    def step_merge(self):
        """Step 4: 合并 adapter"""
        print("[4/7] 合并 adapter...")
        subprocess.run(["python", "merge_lora.py"])
        self.steps.append("merged")

    def step_deploy_canary(self):
        """Step 5: 5% 灰度"""
        print("[5/7] 5% 灰度部署...")
        # 更新 Nginx / 环境配置
        self.steps.append("canary")

    def step_scale_up(self):
        """Step 6: 逐步放量"""
        print("[6/7] 逐步放量...")
        self.steps.append("scaled")

    def step_monitor(self):
        """Step 7: 持续监控"""
        print("[7/7] 持续监控 & 飞轮...")
        self.steps.append("monitoring")

    def run_all(self):
        for step_name in self.config["steps"]:
            getattr(self, f"step_{step_name}")()
        print(f"交付完成! 完成步骤: {len(self.steps)}")

if __name__ == "__main__":
    delivery = CorpAssistDelivery()
    delivery.run_all()

项目指标 分为技术指标和业务指标两层。技术指标在评测集上测量，业务指标在线上环境中跟踪。

常见误区
  • 跳过灰度直接全量：微调效果在评测集上好不代表线上效果好。
  • 只关注技术指标忽略业务指标：CSAT 没提升，技术指标再高也是失败。
  • 交付物不全：只有 adapter 没有文档、指标报告和回滚方案。
  • 忽略飞轮启动：上线后不启动数据收集，下一轮迭代没有数据基础。

A/B 灰度策略

A/B 灰度 是微调上线的标准流程。推荐的三阶段放量策略：

阶段流量比例持续时间观察指标晋级条件
Canary5%24 小时安全违规率、错误率违规率 < 0.1%
小范围20%48 小时CSAT、人工转接率CSAT ↑ > 3%
大范围50% → 100%72 小时全量指标CSAT ↑ > 5%
yaml 
# rollout_config.yaml — A/B 灰度配置
rollout:
  stages:
    - name: canary
      traffic: 0.05  # 5%
      duration_hours: 24
      gates:
        - error_rate < 0.01
        - safety_violations < 0.001
    - name: small
      traffic: 0.20  # 20%
      duration_hours: 48
      gates:
        - csat_improvement > 0.03
        - transfer_rate < 0.03
    - name: full
      traffic: 1.00  # 100%
      duration_hours: 72
      gates:
        - csat_improvement > 0.05
  rollback:
    condition: any_gate_failed
    action: automatic_rollback_to_v1

本章结论

  • 话术规范：垂直话术是领域适配的核心资产，需形成文档。
  • 端到端交付：7 阶段流水线，每阶段有验收标准。
  • 指标双轨：技术指标（合规率、Win Rate）+ 业务指标（CSAT）。
  • 交付清单：数据 → 训练 → 评测 → 部署 四类检查项。
  • 灰度策略：5% → 20% → 100%，每阶段门禁检查。
