为什么要学本章

CorpAssist 话术微调在双栈里有两条路径：百炼托管训练，或本地/云 GPU 用 Python 跑 peft train。Python 轨让你完全掌控语料、超参与 checkpoint，产物可直接上传 DashScope 或走 merge → vLLM。若只会调 API 不会写训练脚本，微调评测 与回滚都缺少可复现实验基线。

本章先记住 3 件事
  • HF/PEFT：基座 frozen，只训 LoRA 低秩矩阵；超参与 SFT 与 LoRA 概念 中的 rank/alpha 一致。
  • 脚本：语料格式对齐 语料构建与清洗 的 JSONL；训练前跑 holdout 划分。
  • 产物：adapter_model.safetensors + adapter_config.json 是制品，不是「某个 checkpoint 文件夹里的一堆 pt」。

HF/PEFT

CorpAssist · 本地 LoRA 实验（90 秒） 数据组交付 corpassist-tone-v1.jsonl（2140 条，Alpaca 格式）。ML 工程师在 A10 单卡上基于 Qwen/Qwen2.5-7B-Instruct 做 r=16 LoRA SFT，3 epoch，预计 2 小时。目标：holdout win rate 相对基座 +10%，再决定是否上传百炼或 merge 自托管。

python ft 栈的核心库：transformers（模型与 Trainer）、peft（LoRA 注入）、accelerate（分布式/混合精度）、datasets（JSONL 加载）。与概念章不同，本章关注可运行的最小工程，而非 rank 公式推导。

LoRA 注入：最小 Python 片段

python 
from transformers import AutoModelForCausalLM, AutoTokenizer
from peft import LoraConfig, get_peft_model, TaskType

BASE = "Qwen/Qwen2.5-7B-Instruct"

tokenizer = AutoTokenizer.from_pretrained(BASE, trust_remote_code=True)
model = AutoModelForCausalLM.from_pretrained(
    BASE,
    torch_dtype="auto",
    device_map="auto",
    trust_remote_code=True,
)

lora_config = LoraConfig(
    task_type=TaskType.CAUSAL_LM,
    r=16,
    lora_alpha=32,
    lora_dropout=0.05,
    target_modules=["q_proj", "k_proj", "v_proj", "o_proj"],
    bias="none",
)
model = get_peft_model(model, lora_config)
model.print_trainable_parameters()
# trainable params: ~40M / 7.6B (~0.5%)

PEFT 训练数据流

HF/PEFT 训练栈
flowchart TD
  Jsonl[corpassist-tone-v1.jsonl] --> Load[datasets.load_dataset]
  Load --> Map[apply_chat_template tokenize]
  Map --> Trainer[HF Trainer + PEFT model]
  Trainer --> Ckpt[checkpoint-N/]
  Ckpt --> Export[save_pretrained adapter]
  Export --> Art[adapter artifact]

语料格式见 语料构建与清洗；评测 gate 见 微调评测。

不推荐
  • 全参数微调 7B（单卡 OOM 或极慢）
  • target_modules 只训 q_proj 却期望大幅风格变化
  • 不用 chat template，手工拼 prompt 与线上一致性差
  • 训练集 = 评测集，holdout 虚高
推荐
  • LoRA r=16、alpha=32 为 CorpAssist 默认起点
  • 冻结基座，只保存 adapter 权重
  • 训练日志含 dataset id、git commit、seed
  • 每 epoch 在 holdout 上跑 win rate 抽检
先想 10 秒：为什么 PEFT 只保存 adapter，而不是整个 7B 模型？

Adapter 通常 < 200MB，便于版本管理与上传；基座权重可从 Hugging Face 或内网镜像按需加载。全量保存会重复存储 14GB+ 且难以 diff。

脚本

lora pipeline 在工程上体现为一条可重复执行的 train_lora.py + shell 入口。CorpAssist 脚本须参数化：基座路径、语料路径、输出目录、LoRA 超参、训练步数，便于 CI 与飞轮再训练。

语料预处理（对齐 JSONL 格式）

python 
def format_example(example, tokenizer):
    messages = [
        {"role": "system", "content": example["system"]},
        {"role": "user", "content": example["input"]},
        {"role": "assistant", "content": example["output"]},
    ]
    text = tokenizer.apply_chat_template(
        messages, tokenize=False, add_generation_prompt=False
    )
    return tokenizer(text, truncation=True, max_length=2048)

# datasets.map(format_example, remove_columns=raw.columns)

TrainingArguments 关键项

参数CorpAssist 推荐说明
per_device_train_batch_size2–4配合 gradient_accumulation
gradient_accumulation_steps8等效 batch 16–32
learning_rate2e-4LoRA 常用区间 1e-4 ~ 3e-4
num_train_epochs3过拟合则降至 2 或加 early stop
save_strategyepoch每 epoch 留 checkpoint
logging_steps10对接 wandb / tensorboard
bf16true（A10+）省显存、加速

完整训练入口（节选）

python 
from transformers import TrainingArguments, Trainer, DataCollatorForSeq2Seq

training_args = TrainingArguments(
    output_dir="./outputs/corpassist-tone-v1",
    per_device_train_batch_size=4,
    gradient_accumulation_steps=4,
    learning_rate=2e-4,
    num_train_epochs=3,
    save_strategy="epoch",
    logging_steps=10,
    report_to="wandb",
    run_name="corpassist-tone-v1-r16",
    bf16=True,
)

trainer = Trainer(
    model=model,
    args=training_args,
    train_dataset=train_ds,
    eval_dataset=eval_ds,
    data_collator=DataCollatorForSeq2Seq(tokenizer, pad_to_multiple_of=8),
)
trainer.train()
model.save_pretrained("./adapters/corpassist-tone-v1")
tokenizer.save_pretrained("./adapters/corpassist-tone-v1")

产物

训练结束后，adapter artifact 是下游 DashScope 上传、merge、registry 登记的唯一输入。须区分「训练过程 checkpoint」与「发布用 adapter 目录」。

发布目录标准结构

text 
adapters/corpassist-tone-v1/
├── adapter_config.json      # LoRA rank、target_modules、base model 名
├── adapter_model.safetensors # 可训练权重（safetensors 格式）
├── tokenizer_config.json    # 若 save_pretrained 一并导出
├── special_tokens_map.json
├── training_manifest.yaml   # 建议自建：dataset id、eval、wandb run

常见误区
  • 只留 checkpoint-3000 — 未执行 save_pretrained，PEFT 格式不完整。
  • base_model 路径写本地绝对路径 — 换机器加载失败；应写 HF hub id 或相对路径规范。
  • 无 eval 快照就标记 prod-ready — 违反 微调评测 gate。
  • 产物未入制品库 — 仅在本机目录，同事无法复现部署。

本章结论

双栈实践 Python 轨：从 HF/PEFT 到可登记 adapter，为 DashScope 或 vLLM 提供统一上游。

  • HF/PEFT：transformers + peft + accelerate；LoRA 注入基座，只训低秩矩阵。
  • 脚本：chat template 对齐线上；TrainingArguments 参数化；shell 一键启动。
  • 产物：adapter_model.safetensors + manifest；checkpoint 与发布目录分离。
  • 衔接：holdout 过 gate 后再走 DashScope 定制模型 或 merge 路径。
  • 下一步：上传百炼并 Spring 切换 model id。

判断练习

  1. 同事训练后只提交了 checkpoint-3/ 文件夹（含 optimizer.pt），未执行 model.save_pretrained。上传 DashScope 失败。排障第一步应做什么？

    参考答案

    从最佳 checkpoint 重新加载 PEFT 模型，执行 model.save_pretrained("./adapters/xxx") 导出标准 adapter 目录（含 adapter_model.safetensors 与 adapter_config.json）。Trainer checkpoint 含 optimizer 状态，不是百炼/vLLM 期望的 PEFT 发布格式。

  2. 下列 TrainingArguments 哪组最可能导致 CorpAssist 话术过拟合、holdout 下降？

    参考答案

    num_train_epochs=10 + 仅 2000 条语料 + 无 eval_dataset early stopping。epoch 过多在小数据集上记忆训练集，holdout win rate 会下降。应 2–3 epoch 并监控 eval loss / win rate。
