为什么要学本章

CorpAssist capstone 上线后，LoRA adapter 与 DashScope 定制模型会像代码制品一样频繁迭代。NS1 飞轮 每轮偏好数据训练都会产出新 ft version。若没有 model registry 与 rollback model 门禁，一次错误切换就会把客服话术打回基座模型水平。

学完你能
  • 为 CorpAssist 微调产物设计注册元数据（语料版本、基座、评测 win rate、合规签核）
  • 用 canary model + ab model 做 5%→20%→100% 晋级，并写 Spring/DashScope 切换配置
  • 把 preference flywheel 新批次与版本号绑定，避免「换了模型却不知道用了哪份语料」
本章先记住 3 件事
  • 注册：每个 adapter 是可追溯制品，不是「某个同事机器上的文件夹」。
  • 切换：生产流量走 model alias，底层 id 可灰度替换。
  • 治理：评测未过、合规未签、飞轮语料未归档 — 三者任一缺失则禁止晋级。

注册：把 adapter 当制品入库

CorpAssist · v2.3 话术 adapter 注册（90 秒） 数据组交付 corpassist-tone-v2.3-lora：基于 Qwen2.5-7B、语料 jsonl-20260528、holdout win rate 67%。MLOps 在 registry 创建记录，绑定 DashScope 上传 job id 与内部 artifact 路径，等待评测与合规 gate 通过后才能挂到 prod alias。

应用侧微调工程的「注册」不是算法论文里的模型发布，而是可追溯制品登记：谁训练的、用什么语料、相对基座提升多少、能否回滚。Registry 可以是 MLflow Model Registry、内部 Git+S3、或 DashScope 控制台中的定制模型列表 — 关键是统一元数据 schema。

注册元数据最小字段

字段示例治理用途
ft version corpassist-tone-v2.3 与语料批次、飞轮轮次一一对应
artifact tag lora-r16-a32-3epoch 训练超参与 checkpoint 定位
base model qwen2.5-7b-instruct 禁止 silent 换基座
eval snapshot win rate 67%、违规率 0.2% rollback gate 对比依据
dataset id jsonl-20260528-dedup 审计与再训练溯源
deploy target DashScope ft-xxx / vLLM merged 双栈路径明确

注册流程：Python 产物 → Registry

微调制品注册与签核
flowchart TD
  Train[LoRA 训练完成] --> Art[adapter artifact]
  Art --> Meta[写入 registry 元数据]
  Meta --> Eval{holdout 优于 base?}
  Eval -->|否| Reject[标记 rejected]
  Eval -->|是| Safety[合规/偏见抽检]
  Safety -->|过| Staged[staged 状态]
  Safety -->|否| Reject
  Staged --> Canary[可挂 canary alias]

评测与合规是注册后的状态机，不是训练脚本里的 print。详见 微调评测 与 合规与偏见。

Registry 记录示例（YAML）

yaml 
# demos/advanced-01-version-lab/version-policy.md 扩展
model_id: corpassist-tone-v2.3
status: staged
base_model: qwen2.5-7b-instruct

  type: lora
  path: s3://ft-artifacts/corpassist/v2.3/adapter_model.safetensors
  tag: lora-r16-a32-3epoch
dataset:
  id: jsonl-20260528-dedup
  rows: 2140
  flywheel_round: ns1-r7
eval:
  holdout_win_rate: 0.67
  baseline_win_rate: 0.52
  violation_rate: 0.002
  regression_suite: pass
deploy:
  dashscope_model_id: ft-corpassist-v2.3-xxxxx
  spring_alias: corpassist-chat-prod
signoffs:
  ml_owner: zhang@corp
  compliance: li@corp
  approved_at: 2026-05-28T14:00:00Z
不推荐
  • 只存 adapter 文件，无 dataset / eval 绑定
  • 同事口头说「这个更好」就替换 prod
  • DashScope id 与内部版本号无映射表
  • 飞轮新语料训练后覆盖旧 artifact 不留档
推荐
  • Registry 为唯一真源，Spring 只读 alias
  • 每次注册自动生成 changelog
  • NS1 飞轮 轮次写入 flywheel_round
  • rejected 版本保留 90 天供审计
先想 10 秒：没有 registry 时，线上 bug 你怎么证明「昨天用的是哪个 adapter」？

通常不能。Registry 的价值是事后审计与一键回滚，不是给 MLOps 看的花架子。

切换：alias、灰度与 A/B

生产环境不应硬编码 DashScope ft-xxx 或 vLLM 路径。对外暴露 model alias（如 corpassist-chat-prod），由配置中心或 Spring 属性解析到具体 version switch 目标。切换分三层：配置切换（改 alias 指向）、流量灰度（canary percent）、实验 A/B（对比业务指标）。

切换策略对照

策略流量比例适用回滚成本
canary model 5% → 20% → 50% 话术微调、低风险域 改配置，秒级
ab model 固定 50/50 或 90/10 需统计显著的业务指标 需实验平台记录
蓝绿 0% 或 100% 大版本、合并权重后 切回旧 alias
rollback model 100% 指回上一 stable 违规率/spike、CSAT 跌 预注册 prod-1 快照

Canary 晋级条件（与 Demo 对齐）

yaml 
# 生产治理：advanced-01-version-lab/version-policy.md
models:
  production: ft-corpassist-v2.2
  canary: ft-corpassist-v2.3
  canary_percent: 5

promotion_gates:
  min_canary_hours: 24
  max_violation_rate_delta: 0.001
  min_csat_delta: -0.02        # 不得低于 -2%
  min_sample_size: 500
  auto_rollback_on_breach: true

rollback:
  stable_snapshot: ft-corpassist-v2.2
  notify: ml-oncall@corp

NS1 飞轮驱动的版本迭代

飞轮批次 → 新版本 → 灰度
flowchart LR
  Prod[线上 v2.2] --> Bad[坏例采集]
  Bad --> Label[标注 ns1-r8]
  Label --> Train[SFT v2.3]
  Train --> Reg[registry staged]
  Reg --> Canary[5% canary]
  Canary -->|gate pass| Prod2[100% v2.3]
  Canary -->|gate fail| Roll[rollback v2.2]

每一轮 NS1 飞轮 产出新版本号，而非覆盖旧 adapter。详见 偏好数据飞轮（应用侧）。

A/B 实验：工程侧要记录什么

  • 分流键：userId 或 sessionId 哈希，保证同用户稳定分组。
  • 暴露日志：{request_id, model_version, variant} 写入 OLAP，供 CSAT / 违规率按组聚合。
  • 护栏指标：违规率、PII 泄露、平均延迟；任一超阈自动 rollback model。
  • 与 RAG 共存：A/B 只换生成模型，检索链路基线一致，避免混淆变量。
常见误区
  • Canary 只看 loss — 训练 loss 降不代表 CSAT 升；必须看 holdout 与线上护栏。
  • 同时改 Prompt 和模型 — A/B 无法归因；一次只变一个因子。
  • 无 rollback 快照 — 晋级前把当前 prod 注册为 stable_snapshot。
  • 飞轮语料未归档就切换 — 出事无法复现训练集。

生产治理：谁有权切换、何时禁止

版本切换是生产变更，不是数据科学家的本地实验。建议 RACI：

ML 工程

训练、注册、提交 canary 申请；无 prod 写权限。

MLOps / SRE

执行 alias 切换、监控护栏、一键 rollback。

合规

抽检违规率、签核 staged → canary。

产品

定义 A/B 成功指标（CSAT、转人工率）。

禁止切换的硬条件

条件说明
holdout win rate ≤ baseline评测 gate 未过
regression suite 失败核心话术模板退化
合规未签核见 合规与偏见
大促封网期变更窗口策略
canary 护栏触发自动 rollback，人工复核后再提

本章结论

生产与治理阶段的第一块拼图：让 CorpAssist 微调产物可登记、可灰度、可回滚。

  • 注册：adapter 入库须绑定语料、评测、飞轮轮次与 deploy id；registry 是唯一真源。
  • 切换：prod 走 alias；canary 5% 起，护栏指标 breach 则 rollback model。
  • NS1 飞轮：每轮标注数据 → 新版本号 → staged → canary，禁止 silent 覆盖。
  • 双栈：DashScope ft-xxx 与自托管 vLLM 均须映射到同一 logical version。
  • 下一步：版本稳定后算训练与推理成本 ROI，见 训练与推理成本。
