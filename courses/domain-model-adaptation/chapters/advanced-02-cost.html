为什么要学本章

微调不是免费的。GPU 租约费、云端训练费、标注人力、部署成本加起来可能远超预期。不懂 成本估算 和 ROI 分析，项目可能在上线前被财务叫停。

学完你能
  • 估算一次微调项目的总成本（GPU + 标注 + 部署 + 人力）
  • 对比 竞价实例、按需实例和预留实例的成本差异
  • 计算微调项目的 ROI：人力节省 vs 微调成本
  • 设立 预算上限 和成本优化策略
本章先记住 3 件事
  • GPU 是大头：一次 7B LoRA 训练约 2-8 小时，A100 每小时 ~1-3 美元。
  • 标注成本常被低估：2000 条客服语料的人工标注费可能超过 GPU 费。
  • ROI = 节省 / 成本：用 CSAT 提升导致的人工客服减少量衡量微调价值。

GPU 成本优化：实例选择

GPU 租赁 策略直接影响训练成本。以下是三种主流方案对比：

方案A100-80G 价格适用场景风险
按需实例~$3.5/小时临时实验、快速验证成本高
竞价实例~$1.0/小时（70% off）可中断的训练作业随时被回收
预留实例~$1.8/小时（1年）持续部署 + 定期训练锁定承诺
python 
# cost_estimate.py — 训练成本估算
from typing import Optional

TRAINING_COST = {
    "qwen2.5-7b-lora": {
        "gpu_hours": 4,          # 3 epoch, 2000 条
        "gpu_type": "A100-80G",
        "on_demand": 3.5,        # $/hour
        "spot": 1.0,
    },
    "qwen2.5-7b-qlora": {
        "gpu_hours": 3,
        "gpu_type": "RTX4090",
        "on_demand": 1.5,
        "spot": 0.4,
    },
    "qwen2.5-14b-qlora": {
        "gpu_hours": 8,
        "gpu_type": "A100-80G",
        "on_demand": 3.5,
        "spot": 1.0,
    },
}

def estimate_training_cost(
    model_scale: str,
    instance_type: str = "spot",
    num_runs: int = 1,
) -> Optional[dict]:
    """估算训练成本"""
    config = TRAINING_COST.get(model_scale)
    if not config:
        return None

    hourly = config["on_demand"] if instance_type == "on_demand" else config["spot"]
    total = hourly * config["gpu_hours"] * num_runs
    return {
        "model": model_scale,
        "gpu_type": config["gpu_type"],
        "hours": config["gpu_hours"] * num_runs,
        "hourly_rate": hourly,
        "total_cost": round(total, 2),
    }

print(estimate_training_cost("qwen2.5-7b-lora", "spot"))
# {"total_cost": 4.0, ...} — 4 美元一次训练
先想 10 秒：竞价实例训练中途被回收怎么办？

使用 checkpoint 机制。每 N 步保存一次 checkpoint，实例回收后从最新 checkpoint 恢复训练。LLaMA-Factory 和 HF Trainer 都原生支持。

云端训练 vs 自建 GPU

云端训练 的灵活性和自建 GPU 的成本优势需要权衡。

维度云 GPU（A100）自建 GPU
初始投入0$10,000-30,000/卡
每小时成本$1-3.5$0.5-1（电费+折旧）
弹性即开即用固定资源
运维云厂商负责自行管理
GPU 利用率按需付费闲置即浪费

经验法则：每周训练 > 40 小时 → 考虑预留实例或自建；< 40 小时 → 按需或竞价实例更划算。

推理成本：微调后推理成本变化

推理成本差异：微调模型推理成本与基座模型几乎相同（因为参数量没变），但 LoRA 模型合并后和基座推理成本完全一致。QLoRA 4-bit 量化模型推理成本更低。

部署方式模型大小推理延迟成本/百万 token
基座 bf1614 GB~50ms~$0.5
微调后 bf1614 GB~50ms（相同）~$0.5
AWQ 4-bit4.5 GB~40ms~$0.2
GGUF Q44 GB~60ms（CPU）~$0.1

推理成本优化建议：使用 AWQ 量化 + vLLM 部署，与基座模型成本相同，但 CSAT 提升带来的业务价值远超推理成本。

ROI 计算：微调值不值

微调 ROI 的计算公式：ROI = (节省成本 - 微调总成本) / 微调总成本。CorpAssist 场景中，ROI 主要来自「人工客服的减少」。

python 
# roi_calculator.py — ROI 计算
def calculate_roi(
    ft_cost: float,               # 微调总成本
    monthly_queries: int,         # 月查询量
    csat_improvement: float,      # CSAT 提升比例
    human_cost_per_query: float,  # 人工客服单次成本
    monthly_deploy_cost: float,   # 部署成本
    months: int = 12,             # 评估周期
) -> dict:
    """计算微调 ROI"""
    # 微调提升后减少的人工查询量
    reduced_queries = monthly_queries * csat_improvement * 0.3  # CSAT ↑ 的 30% 转化为人工减少
    monthly_savings = reduced_queries * human_cost_per_query

    total_savings = monthly_savings * months
    total_cost = ft_cost + monthly_deploy_cost * months
    roi = (total_savings - total_cost) / total_cost

    return {
        "ft_cost": ft_cost,
        "monthly_savings": monthly_savings,
        "total_savings_12m": total_savings,
        "total_cost_12m": total_cost,
        "roi": f"{roi:.0%}",
        "payback_months": round(ft_cost / monthly_savings, 1),
    }

# CorpAssist 示例
roi = calculate_roi(
    ft_cost=1500,
    monthly_queries=50000,
    csat_improvement=0.08,
    human_cost_per_query=0.5,  # $0.5/次人工处理
    monthly_deploy_cost=400,
)
print(f"ROI: {roi['roi']}, 回本周期: {roi['payback_months']} 月")
# ROI: ~120%, 回本周期: ~2.5 月
常见误区
  • 只算 GPU 成本忽略人力：标注和 ML 工程师时间通常占总成本 50%+。
  • 不计算回本周期：只问「贵不贵」不问「多久回本」。
  • 部署成本持续计算：模型部署是按月持续付费，非一次性投入。
  • 忽略失败成本：微调失败/效果不佳的浪费也应计入预算。

成本优化实战：CorpAssist 预算方案

结合 CorpAssist 场景，制定一个 3 个月的微调项目预算：

项目第 1 月第 2 月第 3 月
GPU 训练80（首次训练）50（1 次迭代）50（1 次迭代）
数据标注600200200
模型部署（A100 持续）600600600
ML 工程师1000300300
月合计$2,280$1,150$1,150
3 个月总计$4,580

成本优化方案：

  • 用竞价实例：训练成本从 $80 降至 $25/次，3 个月省 ~$80。
  • 用 QLoRA：可在 RTX 4090（$1.5/h）上训练，省 50%。
  • 合并模型后部署 GGUF Q4：推理显存减少 60%，单卡部署成本降 40%。
  • 标注用 RLAIF 辅助：首月标注成本从 $600 降至 $300。
先想 10 秒：如果预算真的非常有限（<$1000），你的方案怎么调整？

① 用 QLoRA + 竞价实例训练（$10/次）② 部署用开源 GGUF 在 CPU 上跑（不租 GPU）③ 标注用 LLM 自动生成 + 人工抽检 10%。总预算 < $500/月。

本章结论

  • 成本构成：GPU + 标注 + 人力 + 部署 + 评测五类成本。
  • GPU 优化：竞价实例省 70%，QLoRA 省显存。
  • 推理成本：微调后推理成本不变，量化后更低。
  • ROI 计算：从 CSAT 提升到人工成本节省，计算投资回报。
