为什么要学本章

微调项目最常见的失败模式：没有基线对比就上线。没有 baseline compare，就无法判断提升来自微调还是数据噪声；没有 rollback gate，一旦新模型效果回退，只能全体回滚。本章教你建立可复现的评测流水线。

学完你能
  • 设计 holdout set 并计算微调前后 win rate
  • 用 BLEU / ROUGE-L 等自动指标 + LLM Judge 做多维评估
  • 设立 regression test 门禁，阻止效果回退的模型晋级
本章先记住 3 件事
  • 无基线不微调：微调前必须有基座模型在 holdout 集上的准确率基线。
  • Win rate 是金标准：人工 / LLM Judge 对比新旧模型的胜率，最贴近业务感知。
  • 回滚门禁：新模型在关键能力上低于基线 → 自动阻止上线。

评测体系设计：多维对比框架

微调评测 不是单一的准确率数字，而是一个多维框架。CorpAssist 场景需要同时关注四个方面：

CorpAssist · 微调评测（90 秒） CorpAssist v2.3 话术 adapter 训练完成。需要回答三个问题：① 话术合规率是否提升？② 通用能力（如数学、推理）是否下降？③ 基座模型 + RAG 的事实准确率是否保持不变？

基座和微调模型必须在同一 holdout 集上对比。

Holdout 集设计与 Win Rate 计算

Holdout set 是评测的锚。它必须满足三个条件：

  • 不参与训练：与训练集无重叠（去重检查）。
  • 覆盖所有类别：查询、投诉、操作指导、转接四类都有代表。
  • 静态版本化：锁定 holdout 集版本，跨训练轮次可比。

Win Rate 计算方法

Win rate 是最直观的微调效果指标：在 N 条测试用例上，微调模型的回答「优于」或「不劣于」基座模型的比例。判断方式可以是人工标注或 LLM Judge。

python 
# win_rate.py — 计算微调 vs 基座 Win Rate
from typing import List, Dict
import json

def compute_win_rate(
    base_outputs: List[str],
    ft_outputs: List[str],
    judge_fn,  # 判断函数: (base, ft, reference) -> "base" | "ft" | "tie"
) -> Dict:
    """计算微调模型相对基座的 win rate"""
    wins = ties = losses = 0
    for base, ft in zip(base_outputs, ft_outputs):
        result = judge_fn(base, ft)
        if result == "ft":
            wins += 1
        elif result == "tie":
            ties += 1
        else:
            losses += 1
    total = wins + ties + losses
    return {
        "win_rate": wins / total,
        "tie_rate": ties / total,
        "loss_rate": losses / total,
        "total": total,
    }

# 用 LLM 做 Judge
from openai import OpenAI

def llm_judge(base: str, ft: str, reference: str) -> str:
    """让 LLM 判断哪个回答更好"""
    client = OpenAI()
    prompt = f"""参考标准回答：{reference}

回答A（基座）：{base}
回答B（微调）：{ft}

哪个回答更符合 CorpAssist 客服话术标准？
回复 A、B 或 TIE。"""
    resp = client.chat.completions.create(
        model="qwen-plus",
        messages=[{"role": "user", "content": prompt}],
        temperature=0,
    )
    result = resp.choices[0].message.content.strip()
    if "B" in result and "A" not in result:
        return "ft"
    elif "A" in result and "B" not in result:
        return "base"
    return "tie"

自动指标：BLEU / ROUGE-L

BLEU 和 ROUGE-L 是 NLP 最经典的自动评测指标：

  • BLEU（双语评估替补）：基于 n-gram 精确率，适合翻译、生成式任务的精确度衡量。
  • ROUGE-L（面向召回率的评估）：基于最长公共子序列（LCS），适合摘要、问答等需要内容覆盖的任务。
python 
# auto_metrics.py — BLEU / ROUGE-L 计算
from nltk.translate.bleu_score import sentence_bleu, SmoothingFunction
from rouge_chinese import Rouge  # pip install rouge-chinese
from typing import List
import json

def evaluate_bleu(predictions: List[str], references: List[str]) -> float:
    """计算 BLEU-4（带平滑）"""
    smoothie = SmoothingFunction().method4
    scores = []
    for pred, ref in zip(predictions, references):
        score = sentence_bleu(
            [list(ref)], list(pred),
            weights=(0.25, 0.25, 0.25, 0.25),
            smoothing_function=smoothie,
        )
        scores.append(score)
    return sum(scores) / len(scores)

def evaluate_rouge(predictions: List[str], references: List[str]) -> dict:
    """计算 ROUGE-1, ROUGE-2, ROUGE-L"""
    rouge = Rouge()
    scores = rouge.get_scores(predictions, references, avg=True)
    return {
        "rouge-1": scores["rouge-1"]["f"],
        "rouge-2": scores["rouge-2"]["f"],
        "rouge-l": scores["rouge-l"]["f"],
    }

if __name__ == "__main__":
    predictions = ["您好，我是 CorpAssist 小C，请问有什么可以帮您？"]
    references = ["您好，我是 CorpAssist 小C，很高兴为您服务。"]
    bleu = evaluate_bleu(predictions, references)
    rouge = evaluate_rouge(predictions, references)
    print(f"BLEU: {bleu:.4f}")
    print(f"ROUGE: {rouge}")
先想 10 秒：BLEU 和 ROUGE-L 各自的优势和局限是什么？

BLEU 关注精确率——生成的内容有多少词出现在参考中；ROUGE-L 关注召回率——参考中的内容有多少被生成覆盖。两者互补，但对同义表达不敏感，需要结合 LLM Judge 使用。

LLM Judge：自动判断回答质量

LLM Judge 用大模型作为裁判评估回答质量，是当前最实用的自动评测方法。优势：对同义表达鲁棒，可自定义评分维度。

不推荐
  • 只用 BLEU/ROUGE 做唯一指标
  • 用微调模型本身做 Judge（位置偏差）
  • Judge prompt 不包含具体评分标准
推荐
  • BLEU/ROUGE + LLM Judge + 人工抽检
  • 用更强的模型（如 qwen-max）做 Judge
  • Judge prompt 明确定义评分维度（话术/准确/安全）
python 
# llm_judge.py — 多维 LLM Judge
from openai import OpenAI

client = OpenAI()

CORPASSIST_JUDGE_PROMPT = """你是一个专业的客服质量评估员。请评估以下客服回答。

用户问题：{question}
客服回答：{answer}

请对以下四个维度分别打分（1-5分）：
1. 话术合规（是否使用标准开场、礼貌用语）
2. 内容准确（信息是否正确）
3. 流程完整（是否包含必要步骤）
4. 安全合规（是否避免敏感/违规表述）

输出格式：JSON {{"话术合规": N, "内容准确": N, "流程完整": N, "安全合规": N}}"""

def llm_judge_score(question: str, answer: str) -> dict:
    resp = client.chat.completions.create(
        model="qwen-plus",
        messages=[{"role": "user", "content": CORPASSIST_JUDGE_PROMPT.format(
            question=question, answer=answer
        )}],
        response_format={"type": "json_object"},
        temperature=0,
    )
    return json.loads(resp.choices[0].message.content)

回滚门禁：自动阻止效果回退

rollback gate 是微调上线的安全阀门。新模型必须通过三项门禁检查：

yaml 
# eval_gate.yaml — CorpAssist 回滚门禁配置
gates:
  # 门禁 1：话术合规不低于基线
  tone_compliance:
    metric: tone_match_rate
    threshold: 0.95
    operator: ge  # >= base score
    on_fail: block

  # 门禁 2：事实准确率不降超过 3%
  fact_accuracy:
    metric: rouge_l_f1
    threshold: -0.03  # 允许最多下降 3%
    operator: delta_ge
    on_fail: warn

  # 门禁 3：安全违规不增加
  safety:
    metric: violation_rate
    threshold: 0.01
    operator: le  # <= base rate + 1%
    on_fail: block

  # 门禁 4：通用能力不降超过 5%
  general_capability:
    metric: ceval_subset
    threshold: -0.05
    operator: delta_ge
    on_fail: warn

门禁检查脚本：

python 
# gate_check.py — 回滚门禁检查
import yaml, json

def check_gates(base_metrics: dict, ft_metrics: dict, config_path: str) -> dict:
    with open(config_path) as f:
        config = yaml.safe_load(f)

    results = {"passed": True, "gates": []}
    for gate_name, gate_cfg in config["gates"].items():
        base_val = base_metrics[gate_cfg["metric"]]
        ft_val = ft_metrics[gate_cfg["metric"]]
        diff = ft_val - base_val

        if gate_cfg["operator"] == "ge":
            ok = ft_val >= gate_cfg["threshold"]
        elif gate_cfg["operator"] == "le":
            ok = ft_val <= gate_cfg["threshold"]
        elif gate_cfg["operator"] == "delta_ge":
            ok = diff >= gate_cfg["threshold"]
        else:
            ok = True

        if not ok and gate_cfg["on_fail"] == "block":
            results["passed"] = False

        results["gates"].append({
            "name": gate_name,
            "base": base_val, "ft": ft_val, "diff": diff,
            "passed": ok, "action": gate_cfg["on_fail"],
        })
    return results

if __name__ == "__main__":
    base = {"tone_match_rate": 0.82, "rouge_l_f1": 0.88, "violation_rate": 0.02}
    ft = {"tone_match_rate": 0.96, "rouge_l_f1": 0.86, "violation_rate": 0.01}
    result = check_gates(base, ft, "eval_gate.yaml")
    print(json.dumps(result, indent=2, ensure_ascii=False))

本章结论

  • 评测框架：话术合规 + 事实准确 + 流程完整 + 通用能力 四维对比。
  • Holdout 集：静态版本化、类别覆盖、与训练集无重叠。
  • Win Rate：微调 vs 基座的胜率，LLM Judge 可自动化。
  • 自动指标：BLEU + ROUGE-L 互补，但需结合 LLM Judge 和人工抽检。
  • 回滚门禁：YAML 配置 + Python 脚本自动阻止效果回退。
