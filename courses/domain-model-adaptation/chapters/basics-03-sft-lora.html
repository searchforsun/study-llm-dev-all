为什么要学本章

SFT（监督微调）和 LoRA（低秩适配）是当前应用侧微调最主流的两种技术。SFT 让模型学会特定行为，LoRA 把训练成本降到全量微调的 1%。不懂 trainable params 和 GPU memory 的关系，就选不对 r 和 target_modules。

本章先记住 3 件事
  • LoRA 不改变基座：只训练一个小 adapter，基座权重冻结。
  • r 决定能力上限：r 越大 adapter 容量越大，但显存和过拟合风险也增大。
  • QLoRA 省显存：4-bit 量化 + LoRA，单卡 24GB 可微调 7B 模型。

SFT 原理：监督微调的本质

SFT（Supervised Fine-Tuning） 是用标注好的 (instruction, output) 对基座模型做有监督训练。训练目标是最小化输出与标注之间的交叉熵损失。通俗理解：SFT 让模型学会「在这种输入下，应该这样回答」。

CorpAssist · 话术 SFT（90 秒） 基座模型 Qwen2.5-7B-Instruct 的默认回答风格是通用助手的「你好，有什么可以帮助你的？」。通过 SFT，我们希望它学会 CorpAssist 的专业客服话术：使用固定开场白、遵循投诉处理四步流程、结尾带满意度确认。

LoRA 原理：低秩分解

LoRA（Low-Rank Adaptation） 的核心思想：冻结预训练权重 W，在每层插入一个可训练的低秩分解矩阵 BA。对于 W ∈ R^(d×k)，LoRA 学习 B ∈ R^(d×r) 和 A ∈ R^(r×k)，其中 r << min(d, k)。

数学上：h = Wx + BAx。推理时可将 BA 合并回 W（零额外延迟）。

LoRA 低秩分解示意

    end

r << min(d, k)，训练参数量从 d×k 降至 d×r + r×k。

PEFT 参数详解

PEFT 参数 直接影响 LoRA 的容量和训练效率。以下是 CorpAssist 场景的推荐配置：

yaml 
# lora_config.yaml — CorpAssist 话术微调配置
model_name_or_path: Qwen/Qwen2.5-7B-Instruct

# LoRA 参数
lora:
  r: 16               # 秩（rank），越大容量越大。8/16/32 常见
  lora_alpha: 32      # scaling 系数，通常 alpha=2*r
  target_modules:     # 目标模块，通常 Q/K/V/O
    - q_proj
    - k_proj
    - v_proj
    - o_proj
  lora_dropout: 0.05  # 防过拟合，话术微调建议 0.05

# 训练参数
training:
  per_device_train_batch_size: 4
  gradient_accumulation_steps: 4
  learning_rate: 2e-4
  num_train_epochs: 3
  max_seq_length: 2048
  bf16: true          # bfloat16 混合精度

# 量化（QLoRA）
quantization:
  load_in_4bit: false  # true = QLoRA, false = LoRA
  bnb_4bit_compute_dtype: bfloat16
  bnb_4bit_quant_type: nf4
参数含义推荐值效果
r低秩分解的秩8-32r↑ → 容量↑，显存↑，过拟合风险↑
alphaScaling 系数2*r控制 adapter 对输出的影响强度
target_modules插 adapter 的层Q/K/V/O选更多层→容量↑但速度↓
dropout随机失活率0.05-0.1防过拟合，小数据集用大值
先想 10 秒：r=8 和 r=32 分别适合什么场景？

r=8：语料 < 500 条或简单话术对齐（开场白、结束语），不易过拟合。r=32：复杂行为学习（多步骤流程、投诉话术），需要更大容量但需要更多数据防过拟合。

QLoRA：4-bit 量化微调

QLoRA 在 LoRA 基础上加了两项技术：4-bit NormalFloat (NF4) 量化基座权重 + 双阶段反量化（计算时反量化为 bf16）。效果：7B 模型微调显存从 ~16GB 降至 ~8GB，单卡 RTX 4090 即可训练。

不推荐
  • 7B 模型全量微调用 V100 (32GB) — OOM
  • QLoRA 用 r=256 — 量化精度损失 + 大 rank 矛盾
  • LoRA 训练后不 merge 直接推理—代码冗余
推荐
  • 7B 用 QLoRA，r <= 32，单卡 4090 可训
  • 13B-70B 必须 QLoRA，4-bit + r=16
  • 训练后 merge adapter 到基座，推理无额外组件

显存估算公式

LoRA 训练显存 ≈ 模型参数量 × 2 (bf16) + 梯度 × 2 + 优化器状态 × (4 或 8) + adapter 参数

简化估算（以 Qwen2.5-7B 为例）：

方法基座精度训练显存推荐 GPU
全量微调bf16~56 GBA100-80G
LoRAbf16~16 GBA100-40G / RTX 4090 (24G)
QLoRA4-bit NF4~8 GBRTX 4090 / RTX 3090

Adapter 管理与合并

adapter weight 是 LoRA 训练的产物——一个很小的权重文件（7B 模型 r=16 时约 16MB）。生产部署时可以选择：

  • 不合并：用 PEFT 加载 adapter + 基座，灵活切换不同 adapter。
  • 合并：用 merge_and_unload() 将 adapter 合并回基座，推理框架无需 PEFT 依赖。
python 
# merge_adapter.py — LoRA adapter 合并与保存
from transformers import AutoModelForCausalLM, AutoTokenizer
from peft import PeftModel

base_model_name = "Qwen/Qwen2.5-7B-Instruct"
adapter_path = "./outputs/corpassist-tone-v1/checkpoint-500"

# 加载基座 + adapter
model = AutoModelForCausalLM.from_pretrained(
    base_model_name,
    torch_dtype="auto",
    device_map="auto",
)
model = PeftModel.from_pretrained(model, adapter_path)

# 合并并卸载 adapter
merged_model = model.merge_and_unload()
merged_model.save_pretrained("./outputs/corpassist-tone-v1-merged")
tokenizer.save_pretrained("./outputs/corpassist-tone-v1-merged")

# 推理测试
from transformers import pipeline
pipe = pipeline("text-generation", model=merged_model, tokenizer=tokenizer)
result = pipe("经理，我对食堂卫生不满意")
print(result[0]["generated_text"])

效果预期：LoRA 能做什么、不能做什么

效果预期 管理是微调工程的重要环节。LoRA 不是万能的：

能力LoRA 能做的LoRA 不能做的
话术风格学会特定开场、结束语、语气消除基座模型的固有偏见
流程固化学会多步骤输出模板学会全新的逻辑推理能力
知识注入记住少量高频知识（< 1000 条）大量知识注入（须用 RAG）
能力提升在某类任务上提升表现全面超越基座模型
灾难性遗忘通过混合通用语料缓解完全避免遗忘（LoRA 仍会遗忘）
常见误区
  • LoRA = 全量微调的平替：不对，LoRA 是低秩近似，容量有限。
  • r 越大越好：r 过大容易过拟合，尤其语料不足时。
  • QLoRA 效果等于 LoRA：4-bit 量化有精度损失，通常效果略低于 bf16 LoRA。
  • 训练后不验证直接上线：必须做基座 vs adapter 的对比评测。

本章结论

  • SFT：用标注数据监督微调，让模型学会特定行为。
  • LoRA：低秩分解适配器，训练参数量减少 99%，效果接近全量微调。
  • QLoRA：4-bit 量化 + LoRA，单卡 24GB 可训练 7B 模型。
  • PEFT 参数：r=16, alpha=32, target_modules=Q/K/V/O 是 CorpAssist 推荐起点。
  • 效果预期：LoRA 擅长行为对齐，不适合大量知识注入。
