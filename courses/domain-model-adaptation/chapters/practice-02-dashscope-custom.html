数据集上传与格式要求

数据集上传 是百炼微调的第一步。百炼支持 SFT 和 DPO 两种训练类型的 JSONL 格式。

SFT 数据格式（百炼要求）

jsonl 
{"messages": [{"role": "user", "content": "我今年入职满一年，有多少天年假？"}, {"role": "assistant", "content": "您好，我是 CorpAssist 小C。根据公司规定，入职满一年的员工可享受 5 天带薪年假。"}]}
{"messages": [{"role": "user", "content": "食堂投诉电话是多少？"}, {"role": "assistant", "content": "非常抱歉给您带来不好的体验。食堂投诉请拨打 1234，或直接在 OA 系统提交投诉单。我是 CorpAssist 小C，需要我帮您转接吗？"}]}

百炼 CLI 上传脚本

python 
# upload_to_bailian.py — 上传数据集到百炼
import dashscope
from dashscope import DataSet
import json
import os

# 配置 API Key
dashscope.api_key = os.getenv("DASHSCOPE_API_KEY")

def convert_to_bailian_format(input_path: str, output_path: str):
    """将 CorpAssist 三字段格式转为百炼 messages 格式"""
    records = []
    with open(input_path, "r", encoding="utf-8") as f:
        for line in f:
            obj = json.loads(line)
            user_content = f"{obj['instruction']}: {obj['input']}" if obj.get("input") else obj["instruction"]
            records.append({
                "messages": [
                    {"role": "user", "content": user_content},
                    {"role": "assistant", "content": obj["output"]},
                ]
            })
    with open(output_path, "w", encoding="utf-8") as f:
        for r in records:
            f.write(json.dumps(r, ensure_ascii=False) + "\n")
    print(f"转换完成: {len(records)} 条 -> {output_path}")

# 上传数据集

启动训练与获取 Model ID

在百炼平台训练 只需以下步骤：

  1. 登录 百炼 Model Studio 控制台
  2. 进入「模型训练」→「创建训练任务」
  3. 选择基座模型（如 qwen2.5-7b-instruct）
  4. 选择数据集
  5. 配置超参：learning_rate、epoch、batch_size
  6. 提交训练，等待完成
  7. 获取 model id（如 ft-202605281234-abc）
百炼微调流程

    UploadDS --> Config[配置超参]

百炼平台将训练、部署、API 调用串联为托管服务。

百炼训练配置示例

yaml 
# bailian_train_config.yaml — 百炼训练配置
train_config:
  model: qwen2.5-7b-instruct     # 基座模型
  dataset: corpassist-tone-v2     # 数据集名称
  train_type: sft                 # sft / dpo
  hyperparameters:
    learning_rate: 1e-5
    num_train_epochs: 3
    per_device_train_batch_size: 8
    max_seq_length: 2048
  output:
    model_id: auto_generated      # 训练完成后自动生成
  deploy:
    auto_deploy: true             # 训练完成后自动部署
    replica: 2                    # 部署 2 个副本

部署与 API 调用

部署定制模型 后，DashScope 会生成一个 API endpoint。你可以通过 HTTP 或 SDK 调用。

python 
# call_bailian.py — 调用百炼定制模型
from openai import OpenAI
import os

client = OpenAI(
    api_key=os.getenv("DASHSCOPE_API_KEY"),
    base_url="https://dashscope.aliyuncs.com/compatible-mode/v1",
)

# 替换为你的 model id
MODEL_ID = "ft-202605281234-abc"  # 训练完成后获取

def corpassist_query(user_input: str) -> str:
    """调用 CorpAssist 定制模型"""
    response = client.chat.completions.create(
        model=MODEL_ID,
        messages=[{"role": "user", "content": user_input}],
        temperature=0.3,
        max_tokens=512,
    )
    return response.choices[0].message.content

# 测试
print(corpassist_query("我想请 3 天事假"))
# 输出: "您好，我是 CorpAssist 小C。事假申请...""

Spring AI 调用定制模型

Spring AI 提供了与 DashScope 集成的 starter，可以在 Spring Boot 应用中像调用本地 Service 一样调用百炼定制模型。

yaml 
# application.yaml — Spring AI + DashScope 配置
spring:
  ai:
    dashscope:
      api-key: ${DASHSCOPE_API_KEY}
      chat:
        options:
          model: ft-202605281234-abc   # 定制模型 ID
          temperature: 0.3
          max-tokens: 512
java 
// CorpAssistService.java — Spring AI 调用定制模型
@Service
public class CorpAssistService {
    private final DashScopeChatModel chatModel;

    public CorpAssistService(DashScopeChatModel chatModel) {
        this.chatModel = chatModel;
    }

    public String ask(String userQuestion) {
        Prompt prompt = new Prompt(
            userQuestion,
            DashScopeChatOptions.builder()
                .withModel("ft-202605281234-abc")
                .withTemperature(0.3f)
                .build()
        );
        ChatResponse response = chatModel.call(prompt);
        return response.getResult().getOutput().getContent();
    }
}

// 控制器
@RestController
@RequestMapping("/corpassist")
public class CorpAssistController {
    @Autowired
    private CorpAssistService service;

    @PostMapping("/ask")
    public String ask(@RequestBody String question) {
        return service.ask(question);
    }
}

模型管理：版本更新与回滚

Model ID 是百炼定制模型的唯一标识。每次训练产出新 model id，只需在配置文件中更新 model id 即可切换到新版本。回滚同理——改回旧 model id。

yaml 
# 环境配置切换 model id
spring:
  ai:
    dashscope:
      chat:
        options:
          model: ${CORPASSIST_MODEL_ID:ft-202605281234-abc}

# 生产环境: CORPASSIST_MODEL_ID=ft-20260528-v2-def
# 回滚: CORPASSIST_MODEL_ID=ft-20260520-v1-xyz
常见误区
  • 训练完不部署直接调用：训练完成后 model id 不会自动变成可调用 endpoint，必须手动部署。
  • Spring 配置不匹配：model id 在 Spring AI 中通过 model 参数指定，不是 model id。
  • 用基座模型 id 调用定制版：基座和定制 model id 不同，直接调用基座不会获得微调效果。
  • 忽略训练账单：百炼按训练时长收费，大模型训练 3 小时可能花费数百元。

本章结论

  • 托管微调：百炼平台提供一站式训练 + 部署，无需 GPU 运维。
  • 数据集格式：百炼要求 messages 格式，需从三字段转换。
  • Model ID：训练产出的唯一标识，用于 API 调用和配置切换。
  • Spring AI：原生集成，通过配置即可切换定制模型。
