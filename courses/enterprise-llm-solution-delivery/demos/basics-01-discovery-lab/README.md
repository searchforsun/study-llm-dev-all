# Demo：需求调研与范围

[返回章节](../index.html#ch-basics-01-discovery)

## 目标

完成 CorpAssist 虚构客户「华北制造」的 discovery 纪要、MVP 边界表与 risk register 初版。

## 验收标准

- [ ] `discovery-notes.md` 含三方 KPI（业务/IT/安全）
- [ ] 至少 3 条 Out of Scope 及理由
- [ ] 至少 3 条 High 风险及责任人
- [ ] 纪要 48h 内回传客户确认的习惯已建立

## 产出物

| 文件 | 说明 | 模板来源 |
|------|------|----------|
| `discovery-notes.md` | 访谈纪要（三方 KPI + 数据源 + 签字人） | 正文模板 |
| `scope-boundary.md` | MVP 范围 + Out of Scope 表 | 正文决策表 |
| `risk-register.yaml` | Risk Register 初版（≥6 条） | 正文 yaml 模板 |

## 操作步骤

1. 复制正文访谈纪要模板，补全业务 KPI、数据源与签字人
2. 在边界决策表中新增一行 Out of Scope
3. 从风险表选 3 条标为 High，并写缓解措施与责任人
4. （可选）用 Mermaid 向同事口述 Discovery → MVP 冻结路径
