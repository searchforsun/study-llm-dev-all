为什么要学本章

CorpAssist capstone layer1 评审方案文档后，layer-3 方案交付总答辩要求你在 30~45 分钟内串讲面试最高频五类场景（S1~S5），且每场景至少 1 个真实生产坑与对应治理手段。本章把场景课、双栈 workshop 与交付课压成一张可背诵的 top5 defense 总表。

学完你能
  • 按 S1~S5 各输出「架构 3 层 + 指标 2 个 + pitfall per scenario 1 条」答辩卡片
  • 用 three mountains（NS1 飞轮 / NS2 确定性 / NS3 成本）自评每个场景是否达标
  • 把同一 CorpAssist 案例写成交付轨 + 技术轨两条面试叙事（衔接 简历与面试叙事）
  • 完成 demos/advanced-06-top5-defense-lab/top5-cheat-sheet.md 与章节测验
本章先记住 4 件事
  • 答辩顺序：先业务价值与边界 → 架构分层 → 指标 → 生产坑 → 三座大山如何兜底。
  • 生产坑必须「可观测」：能说清怎么发现、怎么止血、怎么进飞轮（见 坏例复盘与飞轮设计）。
  • 双轨叙事共享事实源：交付轨讲 PoC/试点/合同；技术轨讲 RAG/Agent/路由，数字一致。
  • S1 必修：选修 ≥1 门场景课，但 TOP5 总答辩仍须能口述 S2~S5 的架构骨架。

S1 企业 RAG 架构与指标

S1 · 企业知识库 RAG 制度/合同/运维手册问答，回答必须带引用；内网 ACL 按部门隔离。面试常问：混合检索、评测集、索引更新策略。

s1 rag defense 核心：检索增强 + 引用溯源 + 权限在检索层生效。CorpAssist 双栈下，Spring BFF 传 acl_scope，Python 检索服务在向量/关键词两路均过滤。

S1 企业 RAG 分层架构
flowchart TB
  subgraph ingest [索引链路]
    OSS[文档 OSS] --> Parse[解析 OCR/表格]
    Parse --> Chunk[分块+元数据]
    Chunk --> Embed[Embedding]
    Embed --> VDB[(向量库)]
  end
  subgraph query [在线问答]
    Q[用户问题] --> Hyb[混合检索 BM25+向量]
    Hyb --> ACL[ACL 过滤]
    ACL --> Rerank[重排]
    Rerank --> Gen[LLM 生成+引用]
  end
  VDB --> Hyb
  Gen --> Cite[引用 doc_id 回传 BFF]

解析与索引在 Python；BFF 只做鉴权、限流与审计日志。索引版本变更必须触发 re-embed 计划。

指标目标示例北极星

本场景必答生产坑（答辩高光）

坑：脏 PDF / 扫描件
  • 无 OCR/表格还原 → 召回上限锁死在 60%
  • 客户质疑「模型不行」实为数据管道缺失
防御
  • discovery 阶段签「可解析文档占比」基线
  • 解析质量进 golden set 分层（扫描件子集）
  • PoC 演示前先跑 50 页样本解析报告

另两条高频坑（口述即可）：ACL 未在检索层过滤；纯向量忽略制度 Graph 关联。延伸：scenario-enterprise-rag-kb 场景课。

S2 智能客服与降级

s2 cs defense：多轮对话 + 意图识别 + 工具查单 + 可预期转人工。概率模型管「理解」，gray release 与规则管「状态与合规」。

S2 客服：DST + 降级路径
flowchart LR
  User[用户消息] --> Route[意图路由]
  Route -->|FAQ| RAG[RAG 知识库]
  Route -->|交易| Tool[订单/物流 API]
  Route -->|情绪/复杂| Human[转人工队列]
  Tool --> Validate[规则校验槽位]
  Validate -->|失败| Human
  RAG --> Guard[输出审核]
  Guard --> Reply[回复]

DST 状态机在 Spring；Python 可选做意图分类。改单、退款类动作必须规则双校验。

指标与降级策略

指标说明北极星
首轮解决率未转人工即结案占比NS2
转人工率试点 <18%NS2
幻觉改单率应为 0（审计）NS2
视觉调用成本截图分级后下降NS3

本场景必答生产坑

纯 LLM 管理对话状态导致幻觉改单：模型自行「确认退款」但未调支付 API。防御：DST 槽位 + 工具结果回写 + 高风险意图强制转人工；会话日志与工单系统对账。

FAQ：截图识图是否一律上大模型？

否。先分级：小图/二维码走 OCR；复杂 UI 再走视觉模型，并计入 NS3 路由预算。避免「用户截图未分级直接调贵视觉模型」。

S3 Agent 工作流

s3 agent defense：办公流程自动化（审批汇总、跨系统填单）。面试关注规划步数、工具真执行、观察-反思循环。

S3 Agent：执行-观察-反思
flowchart TB
  Plan[规划 ≤3 步] --> Act[工具调用]
  Act --> Obs[观察结果截断]
  Obs --> Reflect{是否达成?}
  Reflect -->|否且步数余量| Plan
  Reflect -->|是| Done[输出]
  Reflect -->|否且无预算| Fallback[降级: 人工/模板]
  Act -->|失败| Fallback

步数预算与工具超时写进契约；工具返回超长必须摘要再进上下文。

本场景必答生产坑

规划超过 3 步无执行-观察-反思 → 偏航。防御：显式 ReAct 循环；工具失败走降级；记忆窗口分层（工作记忆 vs 长期记忆检索），避免「记忆无限堆积」。

yaml 
# Agent 运行预算（OpenAPI 扩展）
x-agent-policy:
  max_steps: 3
  tool_timeout_ms: 8000
  observation_max_tokens: 1500
  on_tool_failure: fallback_human_task

S4 代码库 RAG

s4 code defense：Repo RAG + FIM 补全 + LSP/测试验证闭环。企业内网代码不出域，禁止整库塞 prompt。

架构要点

  • 索引：AST/符号表 + 文件路径；按 repo/branch 版本化
  • 检索：query → 相关文件片段（Top-K 严格上限）
  • 生成：FIM 格式与 IDE 插件对齐
  • 验证：LSP 诊断 / 单测片段执行（NS2 确定性）
能力Spring 轨Python 轨
鉴权与审计Git 权限映射、请求日志—
索引与检索触发 jobAST 分块、向量索引
补全 API网关限流FIM 推理

本场景必答生产坑

整库塞 prompt → Token 爆炸与成本失控。防御：检索 Top-K + 路径过滤；大文件只送符号摘要；NS3 上对补全请求做小模型路由。

S5 内容生成流水线

s5 content defense：营销/运营文案、海报脚本、多风格模板。企业场景必须接真实库存/政策 API，娱乐子场景另论品牌安全。

S5 内容流水线（企业营销默认轨）
flowchart LR
  Brief[营销 Brief] --> Gen[LLM 草稿]
  Gen --> Policy[政策/库存 API 校验]
  Policy -->|失败| Revise[约束重生成]
  Policy -->|通过| Moderation[合规审核]
  Moderation --> Publish[发布通道]
  Moderation -->|拦截| Human[人工复核]

「过度承诺」在 Policy 节点拦截；对抗样本进 NS1 回归集。

本场景必答生产坑

营销过度承诺未接真实库存/政策 API：模型写出「全场五折」但 ERP 无活动。防御：生成后结构化抽取承诺点 → 规则/API 校验 → 不通过则重写或人工审核。

次要坑：缺解码约束致违规素材；千人千无记忆分层。延伸：scenario-enterprise-content-studio。

每场景必答 1 个生产坑

答辩评委常问：「你说上过生产，举一个线上翻车怎么修的？」下表是 TOP5 最小必答坑 + 止血 + 飞轮入口，与 courses.json → interviewScenarioIndex 对齐。

场景 必答生产坑（一句话） 典型止血 进飞轮动作

三座大山 Rubric（飞轮/确定性/成本）

three mountains 对应学习路径 NS1 模型与数据飞轮 · NS2 概率→确定性桥梁 · NS3 成本与延迟权衡。总答辩要求：每个场景至少覆盖其重点北极星 2 项中的 1 项，并说明工程化手段。

TOP5 × 三座大山 评分 Rubric（自评用）

场景 NS1 飞轮（评测/数据） NS2 确定性（规则/工具） NS3 成本（路由/缓存） 答辩及格线（各选 1 条能说清）
S1 golden set + 引用率；索引版本 eval ACL 检索层强制 语义缓存、rerank 小模型 NS1+NS3 各 1 措施
S2 对话 bad case 周报 DST+API 双校验 意图路由到小模型；截图分级 NS2 必达
S3 轨迹评测集 工具真执行+超时 步数预算、模型分层 NS2+NS1
S4 补全质量 benchmark LSP/测试验证 Top-K、FIM 小模型 NS2+NS3
S5 合规对抗集 政策 API 硬校验 草稿用大模型、审核用小模型 NS1+NS2
markdown 
## 三座大山 0~2 分 Rubric（每场景自评）
| 分 | NS1 飞轮 | NS2 确定性 | NS3 成本 |
|----|----------|------------|----------|
| 0 | 无评测集、无 bad case 记录 | 纯 Prompt 赌概率 | 无监控、无限流 |
| 1 | 有 golden set 但未入门禁 | 部分 API，无降级 | 仅账单事后看 |
| 2 | 坏例→golden→发版门禁闭环 | 规则+工具+审计闭环 | 路由/缓存/预算前置 |

**毕业门槛**：5 场景中至少 3 场景有 1 项 NS 达到 2 分，且 S1 必达 NS1≥1。

与 layer-3 关系：若 NS1 仅停留在 PoC 演示准确率，未接 release gate，则总答辩「飞轮」维度不及格。

双轨复合型叙事

同一 CorpAssist 项目，面试时要能切换交付负责人与双栈工程师两种话术，事实数据一致。衔接 dual track resume 与 compound profile。

技术轨 Spring BFF + Python RAG；混合检索+ACL；P95 1.8s 架构图、OpenAPI、评测、生产坑 「我只画 PPT」

本章结论

  • S1 RAG：混合检索+ACL+引用；必答脏 PDF/OCR 与 re-embed 策略。
  • S2 客服：DST+工具+转人工；必答幻觉改单与规则双校验。
  • S3 Agent：步数预算+真执行+反思；必答偏航与工具降级。
  • S4 代码：AST RAG+FIM+验证；必答 Token 爆炸与 Top-K。
  • S5 内容：审核流水线+政策 API；必答过度承诺。
  • 生产坑：每场景 1 坑讲清发现—止血—飞轮。
  • 三座大山：NS1/NS2/NS3 用 Rubric 自评，S1 飞轮不可缺。
  • 双轨叙事：交付轨与技术轨共享指标，STAR 分轨写 Action。

判断练习

  1. （场景）面试官问 S1：「你们 RAG 准确率 90%，怎么保证上线不翻车？」你只回答换更大模型。是否足够？

    参考答案

    不够。应补充：golden set 分层、扫描件/OCR、ACL、发版门禁、坏例飞轮；90% 需说明数据集版本与是否 frozen UAT 集。

  2. （场景）选修只做 S2 客服，总答辩可否跳过 S1？

    参考答案

    不可。S1 为必修场景课；总答辩须能口述 S1 架构骨架与 1 个生产坑，可声明深度实践在 S2。
