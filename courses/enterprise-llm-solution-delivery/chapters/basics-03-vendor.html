私有化

private deploy 适用于不出域、固定 QPS、可接受 CapEx。CorpAssist 私有化常见组件：推理服务、向量库、解析集群。

GPU 估算方法论

gpu estimate 公式：所需显存 = 模型参数量 × 精度系数 × 并发度 × 安全系数。示例：

python
# GPU 估算示例 — 7B 模型，INT4 量化，8 并发
model_params_b = 7        # 7B 模型
precision_bytes = 0.5     # INT4 = 0.5 bytes/param
concurrency = 8
safety_factor = 1.3

vram_per_model = model_params_b * precision_bytes  # 3.5 GB
vram_total = vram_per_model * concurrency * safety_factor
print(f"估算显存: {vram_total:.1f} GB")
# 输出：估算显存: 36.4 GB → 推荐 1 × A100-40G 或 2 × L20

招标要点

tender point 要把「能验收」写进技术卷，与 PoC success criteria 同语言。procurement 流程中，技术方案评分通常占 40-60%。

招标技术条款清单

  • 模型与版本：明确指定模型规格（如 qwen-plus 或同参数级国内模型），允许升级的流程与回归义务
  • 指标：引用率、P95、可用性；附 golden set 规模与标注方式
  • 安全：ACL、审计日志保留、工具调用白名单、渗透测试
  • 知识产权：客户数据、微调权重归属、方案文档版权
  • SLA 与赔偿：可用性、响应时间、赔偿计算方式
  • 运维移交：文档、培训、runbook、值班过渡期
markdown
## 招标技术条款（节选）

### 模型与推理
- 基础模型：qwen-plus 或同参数级国产模型（需提供对比测试报告）
- 模型升级：乙方需在升级前 14 天通知，并提供回归测试报告
- GPU 方案：见附件 B 显存估算表

### 验收指标
- K1：UAT golden set ≥ 200 条，准确率 ≥ 85%
- K2：引用可追溯率 ≥ 95%
- K3：P95 响应时间 ≤ 2.0s（内网）
- K4：月度可用性 ≥ 99.5%（计划内维护除外）

### 安全与合规
- 数据不出域：所有推理在客户私有云/机房完成
- 审计日志留存 ≥ 180 天
- 渗透测试每年一次，修复 P0 漏洞 ≤ 48h

### 知识产权
- 客户数据与标注数据归客户所有
- 交付方案文档归客户所有
- 乙方通用组件库归乙方所有

商务希望写「业界领先准确率」？
改为可测指标 + eval_report，避免 过度承诺。举例：「准确率 ≥ 85%（按合同附件 A 的评测方法）」。
客户要求同时支持 3 家国产芯片？
建议评估适配成本与收益。多芯片适配需要每个芯片做模型量化和性能调优，通常需要 2-4 周/芯片。建议一期选 1 家主推，二期再扩展。
招标中如何评分厂商的模型能力？
建议采用五分制评分卡：① 基础能力（通用任务准确率 30%）② 场景适配（在客户 golden set 上的表现 30%）③ 服务与 SLA（响应时间/可用性 20%）④ 成本（Token 单价/TCO 10%）⑤ 安全合规（数据保护/审计 10%）。PoC 测试报告中的准确率和延迟数据是评分核心依据。
厂商「模型能力免费，只收服务费」怎么评估？
需要谨慎。免费模型通常意味着：① 无 SLA 保障 ② 模型随时可能变更或下架 ③ 商业化后价格不可预期。建议在合同中增加「模型版本锁定 + 价格保护期 + 迁移协助」条款，避免被低价锁定后被动接受涨价。

国产化适配 Checklist

对于国企客户，国产化适配是必须项。以下检查清单确保适配顺利：

  • 确认目标芯片（昇腾/寒武纪/海光）与模型的兼容性（官方适配列表）
  • 确认推理框架（vLLM/TGI/SG）在目标芯片上的支持程度
  • 评估模型量化后的精度损失（在 golden set 上对比 FP16 vs INT4）
  • 确认运维工具链（监控/CICD/日志）与国产化环境的兼容性
  • 预留 4-8 周适配窗口用于调试与回归测试

采购流程与商务谈判

procurement 流程中，技术团队需要为商务团队提供明确的选型建议与风险提示。CorpAssist 的采购流程通常分四步：技术调研 → 厂商对比 → PoC 验证 → 商务谈判。

采购流程各阶段交付物

阶段交付物技术团队输入
技术调研厂商对比矩阵模型规格、API 兼容性、数据合规
厂商对比TCO 估算表公有云 vs 私有化三年总成本
PoC 验证选型测试报告Golden set 准确率、延迟、稳定性
商务谈判SLA 条款建议可用性、赔偿、模型升级策略

商务谈判中的技术要点

  • SLA 写入合同：不要只在口头承诺。明确可用性 ≥ 99.5%、P95 ≤ 2.0s、赔偿计算方式。
  • 模型版本管理：厂商升级模型版本时，需提前 14 天通知并提供兼容性说明。
  • 数据安全：厂商不得将客户数据用于模型训练（写入 DPA 数据保护协议）。
  • 撤退路径：合同应包括「厂商停止服务时客户数据导出方案与迁移协助」条款。
采购常见失误
  • 只看单价不看 TCO（三年总成本）
  • 未考虑模型升级的回归成本
  • SLA 不写赔偿条款

  • PoC 测试报告作为合同附件

FAQ：供应商锁定风险评估方法？

从三个维度评估：模型迁移成本（Prompt 是否跨模型兼容/需重新评测）、平台迁移成本（工作流/知识库是否可导出）、数据迁移成本（向量库/日志是否标准格式）。每项按低/中/高评分，总分决定锁定风险等级。

多家厂商同时 PoC 的资源分配？

建议最多 3 家并行。资源分配：第 1 家 40%（基准）、第 2 家 35%（对比）、第 3 家 25%（参考）。避免超过 3 家——每家的适配、评测、沟通成本都很高。PoC 周期统一为 2 周。

CorpAssist 厂商选型实战案例

某省国企 CorpAssist 项目在厂商选型时面临的三方博弈：业务部门希望用国际模型（GPT-4o）追求最佳效果，IT 部门要求必须国产化（昇腾 + 通义），财务部门要求预算控制在 80 万以内。

选型决策过程

方案准确率成本合规结论
GPT-4o 公有云92%¥12 万/年数据出境风险被合规一票否决
通义 qwen-plus 公有云88%¥5 万/年合规PoC 阶段推荐
通义私有化 + 昇腾86%¥65 万（3 年）完全合规二期推荐

最终决策：一期用通义 DashScope 公有云快速上线（满足业务刚需），二期根据业务量评估切换私有化。IT 部门同步启动昇腾适配预研。这个案例说明：厂商选型不是「选最好的」，而是「选在当前约束下最可行的」。

供应商关系与长期合作

厂商选型不是一次性决策。CorpAssist 运维期需要与模型供应商保持良好合作关系，包括：定期 Review SLA 达标情况、提前沟通模型升级计划、反馈生产问题。建议：每季度与厂商开一次运营回顾会，汇总 Token 消耗、延迟趋势、故障记录。

sla vendor 管理的最佳实践：建立厂商 SLA 对照表，每次故障后记录厂商侧原因与修复时长，用于季度 SLA 评审和续约谈判依据。如果厂商连续两个季度 SLA 不达标，启动备选厂商切换计划。

管理动作频率参与方产出
SLA 评审月度交付团队 + 厂商客户经理SLA 达标报告
运营回顾季度双方技术 + 商务改进计划
合同续约评估年度采购 + 技术 + 法务续约/切换决策

与供应商的关系核心：不要依赖单一供应商。即使一期选了通义，也要在架构层面预留切换其他模型的能力（通过模型网关抽象层）。这不代表不信任厂商，而是企业的基本风险管理。

本章结论

  • 通义：PoC 优先公有云 + 企业账号与 Token 预算。按场景选模型（plus/turbo）。
  • 私有化：不出域时选型；TCO 含 GPU 与运维。决策树辅助快速判断。
  • 招标要点：可验收指标、安全与 IP；与 PoC 脚本一致；国产化适配需评估迁移成本。
