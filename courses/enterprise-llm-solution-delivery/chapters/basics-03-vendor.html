私有化

private deploy 适用于不出域、固定 QPS、可接受 CapEx。CorpAssist 私有化常见组件：推理服务、向量库、解析集群。

私有化部署拓扑（简化）
flowchart TB
  subgraph DC[客户机房]
    GW[API 网关] --> Inf[推理集群]
    Inf --> Emb[Embedding]
    Job[解析 Job] --> VDB[(向量库)]
  end
  Portal[门户] --> GW

与公有云差异：运维、补丁、gpu estimate 由客户或集成商承担。

触发条件建议
数据不出域硬约束私有化或专有云
QPS 低、快速验证公有云 PoC
长期高 QPS建模 TCO 再比公有云

招标要点

tender point 要把「能验收」写进技术卷，与 PoC success criteria 同语言。

  • 模型与版本：允许升级的流程与回归义务
  • 指标：引用率、P95、可用性；附 golden set 规模
  • 安全：ACL、审计日志保留、工具调用白名单
  • 知识产权：客户数据、微调权重归属
markdown
## 招标技术条款（节选）
- 验收：UAT golden set ≥ 80 条，引用率 ≥ 75%，P95 ≤ 3s
- SLA：月度可用性 ≥ 99.5%（计划内维护除外）
- 私有化：GPU 型号与数量见附件 B，集成商提供 7×24 运维或联合 oncall
商务希望写「业界领先准确率」？
改为可测指标 + eval_report，避免 过度承诺。

本章结论

  • 通义：PoC 优先公有云 + 企业账号与 Token 预算。
  • 私有化：不出域时选型；TCO 含 GPU 与运维。
  • 招标要点：可验收指标、安全与 IP；与 PoC 脚本一致。
