本章先记住 3 件事
  • 字符串 model id：dashscope:qwen-plus 由 ModelRegistry 解析，读 DASHSCOPE_API_KEY。
  • Formatter 必须配对：DashScope 用 DashScopeChatFormatter，否则 tool_call 解析失败。
  • 路由降本：复杂推理 qwen-max，日常 plus，分类/摘要 turbo。

DashScopeChatModel 配置

DashScopeChatModel 位于 io.agentscope.core.model，支持流式、多模态与 reasoning 模式。CorpAssist 办公场景默认 qwen-plus + stream(true) 配合 AG-UI 事件流。

Model 层与 ReActAgent
flowchart LR
    Agent[ReActAgent] --> FMT[DashScopeChatFormatter]
    FMT --> CM[DashScopeChatModel]
    CM --> API[DashScope API]
    MR[ModelRegistry] -.->|dashscope:qwen-plus| CM

显式 Builder 或 Registry 字符串两种入口，Agent 逻辑不变。

显式 Builder（生产精细控制）

java 
import io.agentscope.core.formatter.dashscope.DashScopeChatFormatter;
import io.agentscope.core.model.DashScopeChatModel;
import io.agentscope.core.model.GenerateOptions;

DashScopeChatModel model = DashScopeChatModel.builder()
        .apiKey(System.getenv("DASHSCOPE_API_KEY"))
        .modelName("qwen-plus")
        .stream(true)
        .formatter(new DashScopeChatFormatter())
        .defaultOptions(GenerateOptions.builder()
                .parallelToolCalls(false)
                .maxTokens(8192)
                .build())
        .build();

ReActAgent agent = ReActAgent.builder()
        .name("CorpAssist")
        .sysPrompt("你是 CorpAssist 办公助手。")
        .model(model)
        .build();

ModelRegistry 字符串（推荐快速启动）

java 
ReActAgent agent = ReActAgent.builder()

先想 10 秒：何时不用字符串而用显式 Builder？

需要自定义 baseUrl（私有化网关）、超时、GenerateOptions（thinkingBudget）、或固定 credential 对象时。CorpAssist staging 走内网 compatible-mode 时常用 Builder。

模型适用CorpAssist 场景
qwen-max复杂推理、长报告季度分析 Planner
qwen-plus日常办公、工具调用S3 默认办公助手
qwen-turbo分类、摘要、路由意图识别 Agent

retry + fallback 模型

DashScope 偶发 5xx 或限流时，ReActAgent.Builder 提供 maxRetries 与 fallbackModel，与工具层重试分开计数。

java 
ReActAgent agent = ReActAgent.builder()
        .name("CorpAssistResilient")
        .sysPrompt("你是 CorpAssist 办公助手。")
        .model("dashscope:qwen-plus")
        .maxRetries(3)
        .fallbackModel("dashscope:qwen-turbo")
        .maxIters(10)
        .build();
反模式
  • fallback 到不可用模型且无告警
  • retry 与 maxIters 混淆（层不同）
  • 静默降级到 turbo 却未记录 OTel
推荐
  • primary=plus，fallback=turbo + 告警
  • Middleware 记录 model.fallback 事件
  • fallback 后仍满足 NS1 质量下限 eval
CorpAssist · 高峰限流 活动日 DashScope 429 激增。maxRetries=3 吸收瞬断；连续失败后 fallback turbo 保证可用性，同时 Prometheus 计数 agentscope_model_fallback_total 供 NS3 评审。

多模型路由

多模型路由 按任务类型选择 ChatModel：意图分类走 turbo，主对话走 plus，规划走 max。Java 栈常见模式是双 Agent（分类器 + Worker），而非单 Agent 内硬编码。

CorpAssist 模型路由
flowchart TD
    Input[用户输入] --> C[qwen-turbo 分类 Agent]
    C -->|simple| P[qwen-plus Worker]
    C -->|complex| M[qwen-max Planner]
    M --> W[qwen-plus Worker + 工具]
    P --> Out[回复]
    W --> Out

路由节点用最小模型；主链路按复杂度升级。

双 Agent 路由示例

java 
ReActAgent classifier = ReActAgent.builder()
        .name("intent-router")
        .sysPrompt("只输出 simple 或 complex。")
        .model("dashscope:qwen-turbo")
        .maxIters(1)
        .build();

ReActAgent worker = ReActAgent.builder()
        .name("corp-worker")
        .sysPrompt("你是 CorpAssist 办公助手。")
        .model("dashscope:qwen-plus")
        .toolkit(toolkit)
        .maxIters(8)
        .build();

Msg label = classifier.call(List.of(new UserMessage(userText))).block();
String route = label.getTextContent().toLowerCase().contains("complex")
        ? "max-planner" : "plus-worker";
能否在 ModelRegistry 注册 CorpAssist 自定义别名？
可以。ModelRegistry.registerFactory("corp:.*", ...) 将 corp:office-plus 映射到内部 DashScope 配置，便于配置中心统一下发而无需改 Agent 代码。

与 NS3 成本权衡

北星 NS3 要求 Agent 成本可度量、可优化。AgentScope Java 的 ChatUsage 经事件流汇总到 OTel，配合 maxIters、模型路由与 Session compaction 降本。

杠杆做法NS3 影响
maxIters8～12 封顶限制 worst-case 步数
模型路由turbo 做分类/摘要Token 单价降 60%+
Session compaction见 basics-04输入 Token 线性下降
stream + 早停UI 满足即中断减少无效生成 Token

usage 采集（streamEvents）

java 
import io.agentscope.core.event.AgentEventType;
import io.agentscope.core.event.ReplyEndEvent;

agent.streamEvents(new UserMessage("导出周报"))
    .doOnNext(evt -> {
        if (evt.getType() == AgentEventType.REPLY_END) {
            var usage = ((ReplyEndEvent) evt).getUsage();
            System.out.println("NS3 tokens=" + usage.getTotalTokens());
        }
    })
    .blockLast();
CorpAssist · NS3 评审 答辩说明：S3 主 Agent 默认 plus——「90% 任务 8 步内完成，max 仅用于 Planner，月均 Token 降 35%」。须同时展示 eval 质量曲线，避免单指标优化。

本章结论

  • DashScope 配置：Builder 精细控制或 ModelRegistry 字符串快速启动。
