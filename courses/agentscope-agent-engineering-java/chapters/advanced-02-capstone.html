java 
@RestController
@RequestMapping("/agent")
public class CapstoneAgentController {

    private final ReActAgent capstoneAgent;

    public CapstoneAgentController(CapstoneAgentConfig config) {
        this.capstoneAgent = ReActAgent.builder()
            .name("CorpAssist-Capstone")
            .systemPrompt("""
                你是 CorpAssist 办公助手。先查制度再订会议室，最后发邮件。
                信息足够则直接回答，不要多余工具调用。""")
            .model(config.dashScopeModel())
            .tools(config.capstoneTools())  // searchPolicy, bookMeeting, sendEmail
            .maxIterations(5)
            .build();
    }

    @PostMapping(value = "/chat", produces = MediaType.TEXT_EVENT_STREAM_VALUE)
    public Flux<ServerSentEvent<String>> chat(
            @RequestBody ChatRequest req,
            @RequestHeader("X-Session-Id") String sessionId) {
        Msg userMsg = Msg.builder()
            .role(MsgRole.USER)
            .textContent(req.message())
            .build();
        RuntimeContext ctx = RuntimeContext.builder()
            .sessionId(sessionId)
            .userId(req.userId())
            .build();
        return capstoneAgent.streamEvents(userMsg, ctx)
            .map(ev -> ServerSentEvent.builder(ev.type()).data(ev.payload()).build());
    }
}

@Component
public class CapstoneTools {

    @Tool(description = "查询公司制度，返回 policy_id 与摘要正文")
    public String searchPolicy(@ToolParam("keyword") String keyword) {
        return """
            {"policy_id":"POL-2024-ANNUAL","summary":"年假按工龄计算，需提前3天申请。"}
            """;
    }

    @Tool(description = "预订会议室，返回 confirmation_id 与 room")
    public String bookMeeting(
            @ToolParam("room") String room,
            @ToolParam("startTime") String startTime) {
        return """
            {"confirmation_id":"BK-88421","room":"%s","status":"confirmed"}
            """.formatted(room);
    }

    @Tool(description = "发送确认邮件给参会人")
    public String sendEmail(@ToolParam("confirmationId") String confirmationId) {
        return """
            {"email_id":"EM-99102","status":"sent","ref":"%s"}
            """.formatted(confirmationId);
    }
}
Capstone 反模式
  • Sub-agent 堆 3 层做简单订会议室
  • 工具返回纯文本「ok」导致模型重复调用
  • 无 sessionId，每轮丢失上下文
推荐

  • Mock JSON 含 confirmation_id 等结构化字段
  • golden set 断言 must_call_tools 顺序

步数/Token 监控

步数预算 与 Token 监控 是 CorpAssist NS3 成本可控的核心。Capstone 通过 Micrometer 暴露 agent.steps、agent.tokens，与 Python 平行课 metrics 语义对齐。离线评测使用 golden set 统计 pass rate 与平均步数。

指标采集点Capstone 门禁Actuator 路径
ReAct 步数每次 tool_call 后 incrementavg ≤ 4，max ≤ 5/actuator/metrics/agent.steps
Token 用量DashScope 响应 usage单请求 prompt+completion 可记录/actuator/metrics/agent.tokens
成功率golden set 评测≥ 90%eval 脚本输出 metrics.json
P95 延迟Controller 计时≤ 15s/actuator/metrics/http.server.requests
java 
@Component
public class AgentMetricsListener implements AgentEventListener {

    private final Counter stepCounter;
    private final Counter tokenCounter;

    public AgentMetricsListener(MeterRegistry registry) {
        this.stepCounter = Counter.builder("agent.steps")
            .description("ReAct tool invocation count")
            .register(registry);
        this.tokenCounter = Counter.builder("agent.tokens")
            .description("Total tokens from DashScope usage")
            .register(registry);
    }

    @Override
    public void onToolCall(ToolCallEvent event) {
        stepCounter.increment();
    }

    @Override
    public void onModelResponse(ModelResponseEvent event) {
        if (event.usage() != null) {
            tokenCounter.increment(event.usage().totalTokens());
        }
    }
}

配置 management.endpoints.web.exposure.include=health,metrics,prometheus 后，Prometheus 可抓取 agent_steps_total 与 agent_tokens_total，为后续 可观测性与可靠性运维observability-reliability-ops-java 打基础。

avg steps 4.8、success rate 92% 能否通过？
不能。 成功率达标但 avg_steps > 4.0。需优化工具 description 或 systemPrompt，减少无效 ReAct 轮次后再答辩。
Token 监控在 Mock 模式下还有意义吗？
有。即使工具 Mock，模型仍消耗 Token；Capstone 应记录真实 DashScope 调用量，为 NS3 成本估算提供样本。无 API Key 时可跳过 token 断言，步数门禁仍必填。

当任务超出 短链路 能力——步数 >10、跨会话恢复、多文件批处理、Sub-agent 并行长时运行——应升级到 AgentScope Harness 长任务工程（Java）agentscope-harness-engineering-java（或 Python 平行课）。不要在 ReAct 上无限堆叠 maxIterations。
