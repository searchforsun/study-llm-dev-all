为什么要学本章

CorpAssist 知识库问答不能仅靠静态 RAG 管线——Agent 需自主决定何时检索、检索什么、是否多轮。Agentic RAG 把 RAG 系统（Java）rag-system-java 的检索链或百炼 grounding 封装为 Agent 工具，并配合 引用溯源 落地生产。

学完你能
  • 用 DashScopeDocumentRetriever / grounding 接入 CorpAssist 知识库
  • 衔接 rag-system-java 的 VectorStore 为 ReActAgent 工具
  • 在回答中输出可点击的文档引用 [n]
  • 对照 Python Agentic RAG 双栈方案
本章先记住 3 件事
  • 检索是工具不是前置步骤：Agent 按需调用，避免每次都全库检索。
  • 引用必须绑 chunk metadata：doc_id、page、score 写入回答。
  • 低置信度拒答：score 低于阈值时拒绝编造，建议转人工。

百炼 grounding 接入

阿里云百炼提供托管知识库检索（grounding）：DashScopeDocumentRetriever 按 indexName 检索文本切片，无需自建 VectorStore。CorpAssist 快速上线可先用百炼知识库；深度定制索引管线则衔接 rag-system-java 自建 VectorStore。

CorpAssist · 企业知识库 Agent 员工问：「年假最多几天？需要提前多久申请？」KB Agent 应先调用 search_knowledge_base，基于 grounding 返回的 chunk 回答，并在末尾列出 [1] 员工手册 §3.2 引用。

grounding 检索作为 @Tool 注册；Agent 自主决定是否调用。

java 
@Component
public class KnowledgeBaseTools {
    private static final String INDEX = "corpassist-hr-kb";
    private final DashScopeApi dashScopeApi;

    @Tool(name = "search_knowledge_base",
          description = "检索 CorpAssist 企业知识库。输入自然语言问题，返回带 source/page/score 的片段。")
    public String searchKnowledgeBase(@ToolParam(name = "query") String query) {
        DocumentRetriever retriever = new DashScopeDocumentRetriever(dashScopeApi,
            DashScopeDocumentRetrieverOptions.builder()
                .withIndexName(INDEX)
                .withTopK(4)
                .build());
        List<Document> docs = retriever.retrieve(new Query(query));
        if (docs.isEmpty()) {
            return "未找到相关文档，请换关键词或转人工。";
        }
        StringBuilder sb = new StringBuilder();
        for (int i = 0; i < docs.size(); i++) {
            Document d = docs.get(i);
            sb.append(String.format("[%d] source=%s page=%s score=%s%n%s%n---%n",
                i + 1,
                d.getMetadata().get("source"),
                d.getMetadata().get("page"),
                d.getMetadata().get("score"),
                d.getText().substring(0, Math.min(400, d.getText().length()))));
        }
        return sb.toString();
    }
}
百炼 grounding 与自建 VectorStore 如何选型？

grounding 适合快速 PoC、索引由百炼控制台管理；rag-system-java 自建 VectorStore 适合混合检索、权限分库、离线评测闭环。CorpAssist 生产常两者并存：HR 制度用 grounding，技术文档用 Milvus。

衔接 rag-system-java

在 rag-system-java · Spring AI RAG Pipelinerag-system-java 中你已用 DocumentReader → TokenTextSplitter → VectorStore → QuestionAnswerAdvisor 构建管线。本章将该 VectorStore 的 similaritySearch 包装为 Agent 工具，而非固定 Advisor 前置检索——实现 Agentic 按需检索。

java 
@Component
public class VectorStoreKbTool {
    private final VectorStore vectorStore;  // 来自 rag-system-java demo

    @Tool(name = "search_knowledge_base")
    public String search(@ToolParam(name = "query") String query) {
        List<Document> docs = vectorStore.similaritySearch(
            SearchRequest.builder().query(query).topK(4).build());
        return formatCitations(docs);
    }
}

ReActAgent kbAgent = ReActAgent.builder()
    .name("CorpAssistKB")
    .model(dashScopeModel)
    .sysPrompt("""
        你是 CorpAssist 知识库助手。
        1. 涉及公司制度/政策的事实性问题，必须先调用 search_knowledge_base
        2. 回答用 [1][2] 标注引用；末尾输出 ### 参考文献
        3. 工具无结果或低置信度时拒答，可建议转人工

    .toolkit(Toolkit.builder().tool(vectorStoreKbTool).build())
    .maxIters(6)
    .build();

引用溯源

引用溯源 要求每条事实对应 chunk metadata。工具返回结构化 JSON 或固定模板，Agent 回答末尾附「参考文献」列表，前端将 source 渲染为可点击链接（doc_id → 原文预览）。

java 
public record Citation(int refId, String source, String page, String snippet) {}

public String formatCitationsJson(List<Document> docs) {
    List<Citation> citations = new ArrayList<>();
    for (int i = 0; i < docs.size(); i++) {
        Document d = docs.get(i);
        citations.add(new Citation(i + 1,
            (String) d.getMetadata().get("source"),
            String.valueOf(d.getMetadata().get("page")),
            d.getText().substring(0, Math.min(300, d.getText().length()))));
    }
    return new ObjectMapper().writeValueAsString(Map.of(
        "citations", citations));
}
// sys_prompt：正文用 [1][2]；末尾 ### 参考文献 含 source 与 page

java 
private static final double MIN_SCORE = 0.65;

@Tool(name = "search_knowledge_base")
public String searchWithThreshold(@ToolParam(name = "query") String query) {
    List<Document> docs = vectorStore.similaritySearch(
        SearchRequest.builder().query(query).topK(4).build());
    List<Document> filtered = docs.stream()
        .filter(d -> score(d) >= MIN_SCORE)
        .toList();
    if (filtered.isEmpty()) {
        return "检索置信度均低于 " + MIN_SCORE + "，无法可靠回答。请转人工或细化问题。";
    }
    return formatCitations(filtered);
}
Agentic RAG 如何与 rag-system-java 评估指标对齐？
复用 rag-system-java 的 faithfulness / answer_relevance 评测集：Agentic 路径在 golden 中增加 must_call_tools: ["search_knowledge_base"] 断言。引用准确率单独测 citation 与 chunk 对应关系。
高 QPS 纯 FAQ 入口还用 Agentic RAG 吗？
高 QPS 纯 FAQ 优先固定 QuestionAnswerAdvisor 链 + 小模型，延迟更低。Agentic RAG 适合混合意图（检索+工单+邮件）的 CorpAssist 统一入口。
CorpAssist · 制度问答拒答 用户问「火星移民政策」——检索 score 均 <0.65，工具返回拒答文案，Agent 不编造数字并建议联系 HR。审计记录 tool 空结果与拒答原因。
双栈 golden set 如何共用 must_call_tools 断言？

在评测 JSON 中为政策类问题添加 must_call_tools: ["search_knowledge_base"]；Python 与 Java 轨使用相同工具名与 citation Schema，eval 脚本可共用用例集，仅替换 Agent 启动命令。

  • 百炼 grounding：DashScopeDocumentRetriever 快速接入托管 KB。
