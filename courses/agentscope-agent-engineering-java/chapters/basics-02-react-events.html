ReActAgent.builder 模式

AgentScope Java 2.0 的 ReActAgent.builder 是创建 Agent 的标准入口：模型生成 → 解析 tool_call → 执行 @Tool → 将观察写回 Session → 再次调用模型，直到输出最终文本或达到 maxIterations。与 SAA Graph 需手写 agent/tools 节点不同，循环对外不可见，通过 streamEvents 暴露中间态。

AgentScope Java ReAct 循环
sequenceDiagram
    participant U as User
    participant A as ReActAgent
    participant M as DashScopeChatModel
    participant T as @Tool

    U->>A: Msg (USER)
    loop maxIterations
        A->>M: 推理请求
        M-->>A: thinking / tool_call / text
        alt 有 tool_call
            A->>T: 执行工具
            T-->>A: ToolResultBlock
        else 纯文本
            A-->>U: 最终 Msg
        end
    end

ReAct 循环在 Agent 内部闭合；外部通过 streamEvents 观察每一步。

ReActAgent.builder 完整配置

java 
import io.agentscope.core.ReActAgent;
import io.agentscope.core.model.DashScopeChatModel;
import io.agentscope.core.middleware.LoggingMiddleware;

ReActAgent agent = ReActAgent.builder()
    .name("S3-Assistant")
    .systemPrompt("你是 CorpAssist 办公助手，可查询日历。")
    .model(DashScopeChatModel.builder()
        .modelName("qwen-plus")
        .apiKey(System.getenv("DASHSCOPE_API_KEY"))
        .maxRetries(2)
        .fallbackModelName("qwen-turbo")
        .build())
    .tools(calendarTools)              // @Tool 注册的工具列表
    .maxIterations(10)                 // 对应 Graph 回环步数上限
    .middleware(new LoggingMiddleware())
    .build();
先想 10 秒：maxIterations 与 Graph conditional_edge 回环有何不同？

Graph 通过图的边显式回到 agent 节点；AgentScope Java 在 Agent 内部计数迭代，超限直接终止并返回错误或部分结果，无需在图里画回边。

builder 参数行为CorpAssist 建议
模型输出纯文本正常结束 ReAct展示最终回复
达到 maxIterations强制停止，可能部分完成提示用户缩小问题
工具连续失败模型可能反复重试@Tool 返回结构化错误
用户 cancelRealtime Steering 中断见 practice-03 HITL

ContentBlock 与 streamEvents

ContentBlock 是 AgentScope Java 的统一内容单元：text、file、image、model thinking、tool_result 均以 Typed Block 表达。消息不再只是字符串，而是 Block 列表。streamEvents() 将模型输出与工具执行拆成细粒度事件，CorpAssist 前端可据此展示「正在查日历…」进度条。

ContentBlock / Event 类别含义UI 映射
TextBlock / TextDeltaEvent流式可见文本打字机效果
ThinkingBlock模型推理过程（若模型支持）可折叠「思考过程」
ToolCallStartEvent工具名与参数工具调用卡片
ToolCallEndEvent工具结果与耗时更新卡片状态
ReplyEndEvent整轮回复边界启用下一轮输入

消费 streamEvents 事件流

java 
import io.agentscope.core.event.*;

StringBuilder buffer = new StringBuilder();
agent.streamEvents(userMsg, ctx).subscribe(event -> {
    switch (event) {
        case TextDeltaEvent e -> {
            buffer.append(e.delta());
            System.out.print(e.delta());
        }
        case ToolCallStartEvent e ->
            System.out.printf("%n[工具] %s(%s)%n", e.name(), e.args());
        case ToolCallEndEvent e ->
            System.out.printf("[完成] %s 耗时 %dms%n", e.name(), e.durationMs());
        case ReplyEndEvent e ->
            System.out.println("--- 本轮完成 ---");
        default -> { /* OTel span 记录 */ }
    }
});
反模式
  • 只用 call().block() 阻塞等待，前端无进度
  • 忽略 ToolCallStartEvent，用户不知 Agent 在做什么
  • ThinkingBlock 直接展示给终端用户（可能泄露推理链）
推荐
  • SSE / WebSocket 转发 streamEvents 到 CorpAssist UI
  • 工具事件显示名称与耗时，隐藏敏感参数
  • thinking 默认折叠，仅 debug 模式开启

Spring Boot SSE 桥接片段

java 
@PostMapping(value = "/chat/stream", produces = MediaType.TEXT_EVENT_STREAM_VALUE)
public Flux<ServerSentEvent<String>> chatStream(@RequestBody ChatRequest req) {
    Msg userMsg = Msg.builder().role(MsgRole.USER).textContent(req.question()).build();
    RuntimeContext ctx = RuntimeContext.builder()
        .sessionId(req.sessionId()).userId(req.userId()).build();
    return agent.streamEvents(userMsg, ctx)
        .map(evt -> ServerSentEvent.builder(evt.getClass().getSimpleName())
            .data(JsonUtils.toJson(evt)).build());
}

Human-in-the-loop 事件

AgentScope Java 2.0 将权限与人工审批提升为一等公民：Permission 三态（allow / approve / deny）在工具执行前由框架判定，需要人工确认时发射 ApprovalRequiredEvent，外部系统完成审批后恢复任务。这比 Graph 的 interrupt 更结构化，适合 CorpAssist 工单创建、邮件发送等敏感操作。

CorpAssist HITL 事件流
sequenceDiagram
    participant A as ReActAgent
    participant P as PermissionEngine
    participant U as 审批人
    participant T as @Tool

    A->>P: 工具调用请求
    P-->>A: approve 需要人工
    A-->>U: ApprovalRequiredEvent
    U->>A: 审批结果 allow/deny
    alt allow
        A->>T: 执行工具
    else deny
        A-->>U: 拒绝说明
    end

HITL 通过事件暂停 ReAct 循环，外部执行循环恢复任务。

处理 ApprovalRequiredEvent

java 
agent.streamEvents(userMsg, ctx).subscribe(event -> {
    switch (event) {
        case ApprovalRequiredEvent e -> {
            // 推送到 CorpAssist 审批队列
            approvalService.enqueue(e.toolName(), e.args(), e.approvalId());
            ui.showApprovalCard(e.approvalId(), e.toolName());
        }
        case ToolCallStartEvent e -> ui.showTool(e.name());
        default -> {}
    }
});

// 审批完成后恢复（外部执行循环）
agent.resumeWithApproval(approvalId, ApprovalDecision.ALLOW).block();
Permission 三态含义CorpAssist 示例
allow自动放行queryCalendar 只读查询
approve需人工确认sendEmail 外发邮件
deny禁止调用deleteTicket 高危操作
CorpAssist · S3 邮件外发审批 用户说「给张三发会议提醒」后，UI 依次展示：ToolCallStart(sendEmail) → ApprovalRequiredEvent → 审批人点击允许 → ToolCallEnd → TextDelta 确认已发送。详见 Events + Permissions 与 HITL。

本章结论

  • ReActAgent.builder：链式配置模型、工具、Middleware 与 maxIterations，内置 ReAct 循环。
  • ContentBlock + streamEvents：Typed Block 与 Event 支持流式 UI 与审计。
  • HITL 事件：allow/approve/deny 三态 + ApprovalRequiredEvent 实现人工审批。
  • Python 对照：reply_stream 与 streamEvents 语义对齐，双栈统一 trace_id。

Python 侧对照：平行课 ReActAgent 与事件流agentscope-agent-engineering 用 reply_stream + EventType；Java 课用 streamEvents + sealed Event 类。前端组件可共用 SSE 协议，仅在网关层做字段映射，业务 Agent 逻辑不必双写。

判断练习

  1. （排障）用户反馈 CorpAssist UI 长时间空白，后端日志显示 ToolCallStartEvent 后无 End。最可能的根因与排查顺序是？
