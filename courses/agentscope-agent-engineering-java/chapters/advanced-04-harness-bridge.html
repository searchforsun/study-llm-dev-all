ReActAgent vs HarnessAgent

ReActAgent 适合目标明确、工具链短、可在单 Session 内完成的 CorpAssist 任务。HarnessAgent 在 ReAct 之上增加计划-执行-检查循环、持久化 Workspace 与断点续跑，适合步数不可预估的长任务。

维度ReActAgentHarnessAgent
典型步数≤ 5～8（Capstone ≤5）10～数百，动态规划
任务结构单目标链式工具调用多阶段子任务 + 检查点
状态Session + Msg 历史Workspace 文件 + 任务 DAG 状态
失败恢复重试或新 Session从检查点续跑
沙箱工具各自隔离统一 Workspace 预览与执行
CorpAssist 示例订会议室 + 发邮件批量整理纪要 → 汇总 PPT
ReActAgent 与 HarnessAgent 运行时对比

        U1[User Msg] --> R[ReAct 循环]

ReAct 是单环工具循环；Harness 是计划-执行-检查-检查点外环。

java 
// ReActAgent — Capstone 短链路（advanced-02）
var reactAgent = ReActAgent.builder()
    .name("CorpAssist-Short")
    .maxIterations(5)
    .tools(List.of(searchPolicy, bookMeeting, sendEmail))
    .build();

// HarnessAgent — 长任务（导读，完整见 harness-engineering-java 课）
var harnessAgent = HarnessAgent.builder()
    .name("CorpAssist-Quarterly-Report")
    .workspace(WorkspaceConfig.builder()
        .rootPath("/var/corp-assist/ws/{taskId}")
        .maxDiskMb(512)
        .allowPreview(true)
        .build())
    .planner(dashScopePlannerModel)
    .executor(reactExecutor)          // 子步骤仍可用 ReAct
    .checkpointStore(redisCheckpoint)
    .maxPhases(20)
    .build();

// 同一 Spring Boot 应用可按路由分流
@PostMapping("/agent/short")  /* → reactAgent */
@PostMapping("/agent/long")    /* → harnessAgent */

先想 10 秒：Harness 内部的 Executor 还能用 ReActAgent 吗？

可以。Harness 负责宏观计划与检查点；每个子任务可由 ReActAgent 执行——不是二选一，是分层组合。

Workspace 与沙箱预览

Workspace 是 Harness 的持久化工作目录：中间产物（Markdown 摘要、CSV、脚本）写入 Workspace，Checker 读取验证，最终 Deliverable 经沙箱预览网关供用户确认后再导出。

能力说明CorpAssist 安全要求
路径隔离/ws/{tenant}/{taskId}/禁止跨 tenant 读文件
磁盘配额maxDiskMb per task防止 Agent 写满节点
预览网关只读 HTTP 展示 html/md不直接暴露 OS 路径
沙箱执行可选容器 / WASM 跑脚本生产禁止任意 shell
审计文件 create/modify 事件对接 SIEM 与 HITL
Workspace 与预览网关
flowchart LR
    HA[HarnessAgent] --> WS[(Workspace  volume)]
    WS --> PV[Preview API 只读]
    PV --> UI[CorpAssist 前端预览]
    HA --> SB[Sandbox Executor 可选]
    SB --> WS
    Audit[Audit Log] --> WS

agentscope:

    workspace:
      root: /data/corp-assist/workspace
      max-disk-mb: 512
      retention-days: 7
    preview:
      enabled: true
      base-url: https://corp-assist.internal/preview
      allowed-extensions: [md, html, csv, json]
    sandbox:
      mode: disabled          # 导读阶段；Harness 课开启 container
      # mode: container
      # image: corp-assist/sandbox-runner:1.0
Workspace 反模式
  • Agent 直接写 /tmp 无 tenant 隔离
  • 预览 API 映射任意绝对路径（路径遍历）
  • 长任务无 retention，磁盘打满
推荐
  • taskId + tenantId 双键隔离
  • Preview 仅 serve 白名单扩展名
  • 7 天 retention + 配额告警

何时升级 Harness 课

完成 advanced-02 Capstone 后，用下列信号判断是否需要进入 HarnessAgent 专项课。满足 ≥2 条强烈建议升级，而非继续增大 maxIterations。升级前须完成 Harness 检查点 与 Workspace 配额设计评审。

#升级触发条件短链路 ReAct 为何不够
1预估步数 > 10 或不可预估maxIterations 截断导致半成品
2任务时长 > 30s，用户需离开再回来单 Session SSE 易断连
3多文件中间产物需人工预览ReAct 无 Workspace 管理
4子任务失败需从阶段 N 续跑Session 历史无法表达 DAG 状态
5Sub-agent 并行 > 3 且有时长差异主管 sync 编排复杂度爆炸
6需沙箱跑用户上传脚本@Tool 直连 JVM 风险过高
升级决策流
flowchart TD
    Cap[Capstone 短链路完成] --> Q1{步数/时长超阈值?}
    Q1 -->|否| Stay[继续 ReAct + OPS 监控]
    Q1 -->|是| Q2{需 Workspace/检查点?}
    Q2 -->|是| Har[agentscope-harness-engineering-java]

与 observability 课衔接

长任务上线后，ReAct Capstone 阶段的 agent.steps 不足够——需要任务级 trace、阶段耗时、Workspace 配额与失败率 SLO。observability 课衔接 点在以下指标与告警规则。

指标 / 信号ReAct Capstone 已有Harness + OPS 需补充
trace_id 透传Controller → AgentPlanner/Executor/Checker 各阶段 span
步数agent.stepsharness.phase.duration、checkpoint.count
Tokenagent.tokens按 phase 分摊 cost
失败golden set pass rate阶段失败率、续跑成功率
资源—Workspace 磁盘、沙箱 CPU
告警P95 > 15s任务 stalled > 5min、quota 80%
java 
// observability 衔接 — Harness 阶段 Micrometer（导读）
@Component
public class HarnessMetrics {
    private final Timer phaseTimer;
    private final Counter checkpointCounter;

    public HarnessMetrics(MeterRegistry registry) {
        this.phaseTimer = Timer.builder("harness.phase.duration")
            .tag("phase", "plan|execute|check")
            .register(registry);
        this.checkpointCounter = Counter.builder("harness.checkpoint.count")
            .register(registry);
    }

    public void recordPhase(String phase, Duration d) {
        phaseTimer.record(d);
    }
}

判断练习

  1. 团队将 CorpAssist 长任务 maxIterations 从 5 调到 50，仍频繁截断——是否应继续调到 200？

    参考答案

    不应。 步数不可预估且已 >10 是 Harness 升级信号。继续堆 maxIterations 会恶化延迟、Token 成本与 SSE 断连风险；应进入 Harness 课做计划分解与检查点。

  2. （找错配置）下列哪条 Workspace 配置在生产环境风险最高？ A) max-disk-mb: 512 B) preview 允许 .. 路径 C) retention-days: 7

    参考答案

    B。 Preview 必须禁止路径遍历，仅映射 taskId 下白名单文件。A/C 合理，需配合 quota 告警。
