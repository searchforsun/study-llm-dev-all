为什么要学本章

CorpAssist Java 轨已在前面章节用 AgentScope Java 2.0 手写 ReActAgent。进入生产后，团队希望与现有 Spring AI Alibaba 工程体系spring-ai-alibaba-engineering 统一：依赖管理、自动配置、Graph 编排与 OpenAPI 契约导出。spring-ai-alibaba-starter-agentscope 把 AgentScope Agent 包装为 Spring Bean，可作为 SAA Graph 节点 与 Python Agent Service 双栈联调。

学完你能

  • 在 SAA Graph 工作流中嵌入 AgentScope 节点，与审批/RAG 节点串联

  • 用 WebClient 联调 Python Agent Service，验证双栈契约
本章先记住 3 件事
  • Starter ≠ 替代 AgentScope API：底层仍是 ReActAgent，Starter 管 Bean 生命周期与 Graph 适配。
  • Graph 节点要有明确 I/O：输入 state 字段、输出写入 state，避免隐式全局变量。
  • OpenAPI 是双栈契约：Java 导出 schema，Python Agent Service 按同一字段联调。

AgentScopeAgent 包装

CorpAssist · dialog-orchestrator 接入 架构组要求：办公 Agent 仍用 AgentScope Java 事件流与 Sub-agent，但部署形态与 SAA ReactAgent 模块一致——通过 Starter 自动配置 AgentScopeAgent Bean，供 Controller 与 Graph 共用同一实例。

AgentScopeAgent 包装 将 ReActAgent.builder() 产物注册为 Spring Bean，并桥接 SAA 的 ToolCallback、Session 与观测钩子。与手写配置相比，Starter 统一版本对齐（AgentScope Java 2.0 + SAA 1.1.x）并暴露 agentscope.* 配置前缀。

xml
<dependency>
  <groupId>com.alibaba.cloud.ai</groupId>
  <artifactId>spring-ai-alibaba-starter-agentscope</artifactId>
</dependency>
java
@Configuration
public class CorpAssistAgentConfig {

    @Bean
    public ReActAgent corpAssistReActAgent(DashScopeChatModel chatModel,
            List<ToolCallback> corpTools) {
        return ReActAgent.builder()
            .name("corp-assist")
            .chatModel(chatModel)
            .tools(corpTools)
            .systemPrompt("""
                你是 CorpAssist 办公助手。工具：日历、邮件、工单。
                外部写操作须等待 HITL 审批事件后再执行。
                """)
            .maxIterations(10)
            .build();
    }

    @Bean
    public AgentScopeAgent agentScopeAgent(ReActAgent corpAssistReActAgent) {
        return AgentScopeAgent.from(corpAssistReActAgent)
            .withEventStreaming(true)
            .build();
    }
}
不推荐
  • Controller 与 Graph 各 new 一个 ReActAgent，Session 不一致
  • 跳过 Starter，手工拼 DashScope + AgentScope 版本易冲突
  • 在包装层吞掉 AgentScope Event，BFF 无法 SSE 转发
推荐
  • 单例 AgentScopeAgent Bean，Graph 与 REST 共用
  • agentscope.max-iterations 与 YAML 统一配置
  • 保留 streamEvents() 供 SSE 端点复用

嵌入 SAA Graph 工作流

CorpAssist 典型链路：意图路由 → AgentScope 推理 → 审批节点 → 汇总。AgentScope 节点作为 Graph 中的一个 可调用节点，读取 AgentState 中的 userMessage，写回 agentResult 与 toolTrace。

Graph 嵌入 AgentScope 节点
flowchart LR
    Start([用户请求]) --> Route{意图路由}
    Route -->|办公自动化| ASNode[AgentScope 节点]
    Route -->|纯 FAQ| RAG[RAG 节点]
    ASNode --> HITL{需审批?}
    HITL -->|是| Approve[人工审批]
    HITL -->|否| Done[汇总响应]
    Approve --> Done
    ASNode --> Session[(Session Store)]

AgentScope 节点负责 ReAct 循环；Graph 负责分支、审批与补偿。

java
@Bean
public Graph corpAssistGraph(AgentScopeAgent agentScopeAgent) {
    return Graph.builder()
        .addNode("route", this::routeIntent)
        .addNode("agentscope", state -> {
            String msg = state.get("userMessage", String.class);
            AgentResult result = agentScopeAgent.run(msg, state.getSessionId());
            state.put("agentResult", result.content());
            state.put("steps", result.steps());
            return state;
        })
        .addNode("hitl", this::checkApproval)
        .addEdge("route", "agentscope", s -> "office".equals(s.get("intent")))
        .addEdge("agentscope", "hitl")
        .build();
}
State 键类型写入节点说明
userMessageString入口用户自然语言
intentStringrouteoffice / faq / ticket
agentResultStringagentscopeAgent 最终文本
stepsIntegeragentscopeReAct 步数，供监控
approvalStatusEnumhitlallow / pending / deny

OpenAPI 契约导出

Java BFF 与 Python Agent Service 需要对齐 REST 字段。SpringDoc 可扫描 Controller，也可维护独立 openapi-agent.yaml 作为 SSOT——双栈 CI 用同一文件做契约测试。

yaml
# openapi-agent.yaml 核心片段
paths:
  /api/agent/run:
    post:
      requestBody:
        content:
          application/json:
            schema:
              properties:
                sessionId: { type: string }
                message: { type: string }
                maxIterations: { type: integer, default: 10 }
      responses:
        "200":
          content:
            application/json:
              schema:
                properties:
                  content: { type: string }
                  steps: { type: integer }
                  traceId: { type: string }
java
@RestController
@RequestMapping("/api/agent")
public class AgentController {

    private final AgentScopeAgent agentScopeAgent;

    @PostMapping("/run")
    public AgentRunResponse run(@RequestBody AgentRunRequest req) {
        AgentResult result = agentScopeAgent.run(req.message(), req.sessionId());
        return new AgentRunResponse(
            result.content(), result.steps(), TraceContext.currentTraceId());
    }
}

Python Agent Service 联调

CorpAssist 双栈中，部分 Sub-agent（如复杂 RAG）仍跑在 Python AgentScope。Java Graph 在特定分支通过 WebClient 调用 Python /v1/chat/stream，请求体字段与 OpenAPI 保持一致，并透传 X-Tenant-Id、X-Trace-Id。

Java Graph ↔ Python Agent Service
sequenceDiagram
    participant Graph as SAA Graph
    participant Java as AgentScopeAgent
    participant Py as Python agent-service
    Graph->>Java: 主路径 ReAct
    Graph->>Py: 分支：复杂 RAG Sub-agent
    Py-->>Graph: SSE events → 聚合为 state
    Java-->>Graph: agentResult + steps
java
@Service
public class PythonAgentClient {
    private final WebClient webClient;

    public Flux<AgentEvent> streamFromPython(String tenantId, AgentRunRequest req) {
        return webClient.post()
            .uri("/v1/chat/stream")
            .header("X-Tenant-Id", tenantId)
            .header("X-Trace-Id", TraceContext.currentTraceId())
            .bodyValue(Map.of(
                "session_id", req.sessionId(),
                "message", req.message()))
            .retrieve()
            .bodyToFlux(String.class)
            .filter(line -> line.startsWith("data:"))
            .map(this::parseEvent);
    }
}

本章结论

  • Starter 包装：单 Bean 供 REST 与 Graph 共用，保留 Event 流。
  • Graph 节点：显式 state 键，AgentScope 只管 ReAct，Graph 管分支与 HITL。
  • OpenAPI：双栈契约 SSOT，字段与 Python snake_case 做映射层。
  • 联调：WebClient + trace/tenant 透传，SSE 聚合回 Graph state。
