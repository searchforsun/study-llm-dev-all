日历/邮件/工单工具

CorpAssist 办公助手需区分查日程、发邮件、建工单三类意图。Java 栈用 @Tool 注解 + @ToolParam，框架自动生成 JSON Schema 供模型选择。

CorpAssist · S3 办公自动化 员工：「帮我把明天 10 点会议室 A 加到日历，并邮件通知项目组。」单 Agent 需依次调用 create_calendar_event 与 send_email，参数缺失时追问而非臆造。
工具典型场景必填参数权限
create_calendar_event会议预约title, startTime日历写
send_email通知、摘要to, subject, body邮箱发
create_ticketIT 报障category, description工单写
transfer_to_human复杂审批reason无副作用

OfficeTools 注册

java 
import io.agentscope.core.tool.Tool;
import io.agentscope.core.tool.ToolParam;
import io.agentscope.core.tool.Toolkit;

public class OfficeTools {
    @Tool(name = "create_calendar_event",
          description = "创建日历事件。title 标题，startTime ISO8601。")
    public String createCalendarEvent(
            @ToolParam(name = "title") String title,
            @ToolParam(name = "startTime") String startTime) {
        return "已创建日历 EVT-001：" + title + " @ " + startTime;
    }

    @Tool(name = "send_email", description = "发送邮件。")
    public String sendEmail(
            @ToolParam(name = "to") String to,
            @ToolParam(name = "subject") String subject,
            @ToolParam(name = "body") String body) {
        return "邮件已发送至 " + to;
    }
}

Toolkit toolkit = new Toolkit();
toolkit.registerTool(new OfficeTools());
不推荐
  • @Tool description 为空
  • 失败抛 RuntimeException 中断循环
  • 一个 mega-tool 包打天下
推荐
  • 每工具单一职责 + 参数说明
  • 失败返回可读错误字符串
  • 敏感操作预留 HITL（practice-03）

ReActAgent 完整链路

完整链路 = 用户 Msg → 推理 → tool_call → 工具执行 → 再推理 → 最终回复。Spring Boot 中可将 ReActAgent 注册为单例 Bean，HTTP 层传入 RuntimeContext。

单 Agent 工具路由
flowchart TD
    User[用户请求] --> ReAct[ReActAgent]
    ReAct -->|查日程| Cal[create_calendar_event]
    ReAct -->|发邮件| Mail[send_email]
    ReAct -->|建工单| Ticket[create_ticket]
    ReAct -->|无法处理| Human[transfer_to_human]
    Cal --> ReAct
    Mail --> ReAct
    Ticket --> ReAct
    Human --> End([结束])

ReActAgent 在循环内根据工具 Schema 选择下一步。

构建与调用

java 
ReActAgent agent = ReActAgent.builder()
        .name("CorpAssistOffice")
        .sysPrompt("""
            你是 CorpAssist 办公助手。优先用工具完成请求；
            缺参数时追问；无法处理时调用 transfer_to_human。
            """)
        .model("dashscope:qwen-plus")
        .toolkit(toolkit)
        .maxIters(8)
        .build();

Msg result = agent.call(
        List.of(new UserMessage(
            "帮我把周五 10 点产品评审加到日历，并邮件通知 team@corp.com")),
        RuntimeContext.builder().sessionId("office-demo").build())
    .block();
System.out.println(result.getTextContent());
先想 10 秒：为何不单独写一个「意图分类 LLM」再调 Agent？

多一跳 LLM 增加延迟与 NS3 成本，且分类错误会级联。ReActAgent 在单循环内完成意图识别与工具调用，streamEvents 可观测每一步。

事件流观测

上线前须用 streamEvents 验证 tool_call 序列，避免模型跳过工具直接编造结果。

java 
import io.agentscope.core.event.AgentEventType;
import io.agentscope.core.event.ToolCallStartEvent;

agent.streamEvents(new UserMessage("查下周会议并发邮件"))
    .doOnNext(evt -> {
        if (evt.getType() == AgentEventType.TOOL_CALL_START) {
            System.out.println("[tool] " + ((ToolCallStartEvent) evt).getToolName());
        }
    })
    .blockLast();

降级与预算控制

transfer_to_human 是显式工具。触发条件：用户明确要求、连续工具失败、maxIters 将耗尽。在 sysPrompt 中约定预算规则，生产可用 Middleware 注入。

参数推荐值作用
maxIters8–12ReAct 最大轮次
maxRetries2模型调用重试
单次工具超时15s防止外部 API 拖死循环
Session compaction见 basics-04控制 context Token
java 
@Tool(name = "transfer_to_human", description = "转接人工。预算耗尽或无法完成时调用。")
public String transferToHuman(@ToolParam(name = "reason") String reason) {
    return "已转人工，原因：" + reason;
}

ReActAgent agent = ReActAgent.builder()
        .name("CorpAssistOffice")
        .sysPrompt("最多 8 步；即将超限必须调用 transfer_to_human。")
        .model("dashscope:qwen-plus")
        .toolkit(toolkit)
        .maxIters(8)
        .build();
降级触发决策
flowchart TD
    Start[ReAct 循环] --> Fail{连续工具失败?}
    Fail -->|是| Human[transfer_to_human]
    Fail -->|否| Budget{步数将超限?}
    Budget -->|是| Human
    Budget -->|否| Continue[继续推理]

降级是显式工具调用，事件流可审计触发原因。

Spring Boot 集成要点

将 ReActAgent 声明为 @Bean 单例，Controller 从 Header 取 X-Session-Id / X-User-Id 构造 RuntimeContext。DashScope API Key 走 K8s Secret 注入环境变量，与 spring-ai-alibaba-engineering 课密钥管理一致。

java 
@PostMapping("/api/office/chat")
public Mono<OfficeReply> chat(@RequestBody ChatRequest req,
        @RequestHeader("X-Session-Id") String sessionId,
        @RequestHeader("X-User-Id") String userId) {
    RuntimeContext ctx = RuntimeContext.builder()
            .sessionId(sessionId).userId(userId).build();
    return agent.call(List.of(new UserMessage(req.message())), ctx)
            .map(msg -> new OfficeReply(msg.getTextContent()));
}

单 Agent 何时升级到 Sub-agent？
当任务需多角色专业分工、或 Supervisor 需汇总多专家中间结果时，升级到 practice-02 Sub-agent。单 Agent 工具数 ≤6 且步骤 ≤8 时不必过早拆分。
