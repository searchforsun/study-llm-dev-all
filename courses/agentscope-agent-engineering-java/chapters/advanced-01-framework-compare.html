为什么要学本章

CorpAssist S3 办公自动化在 Java 侧可能已有 SAA Graph 编排、LangGraph4j PoC 或 Python CrewAI 原型。AgentScope Java 2.0 提供 ReActAgent + Sub-agent + streamEvents 的 AOP 路径——本章系统对比 LangGraph4j、SAA Graph 与 AgentScope Java，给出可落地的 选型决策矩阵 与迁移评估方法，并与 Python 平行课 框架选型对比agentscope-agent-engineering 对照。

本章先记住 3 件事
  • Java 栈无银弹：单 ReAct 用 AgentScope Java 或 SAA ReactAgent 均可；复杂 checkpoint 图用 Graph；LangGraph4j 适合已有 Python LangGraph 资产对照。
  • Python 框架 ≠ Java 生产：CrewAI / AutoGen 可作 PoC，CorpAssist Java 生产仍以 AgentScope Java + SAA 为主。
  • 迁移看状态与契约：Graph State / checkpoint 与 AgentScope Msg + Session 是最大差异点。

AgentScope Java 以 Agent + Msg 为中心；SAA Graph 与 LangGraph4j 以显式状态图为中心，checkpoint 语义更成熟。

java 
// 同一任务「查制度并订会议室」三框架写法对照（示意）

// --- AgentScope Java ---
var agent = ReActAgent.builder()
    .name("CorpAssist-S3")
    .systemPrompt("先查制度再订会议室，信息足够则直接回答。")
    .model(dashScopeModel)
    .tools(List.of(searchPolicyTool, bookMeetingTool))
    .maxIterations(8)
    .build();
Msg reply = agent.call(userMsg, ctx).block();

// --- SAA Graph（复习 agent-orchestration-java）---
StateGraph graph = new StateGraph(AgentState::new);
graph.addNode("agent", agentNode);
graph.addNode("tools", toolNode);
graph.addConditionalEdges("agent", routeFn, Map.of("tools", "tools", "end", END));
CompiledGraph compiled = graph.compile(checkpointSaver);

// --- LangGraph4j（与 Python LangGraph 心智一致）---
StateGraph<AgentState> lg = new StateGraph<>(AgentState.SCHEMA);
lg.addNode("agent", agentNode);
lg.addEdge(START, "agent");
CompiledGraph<AgentState> app = lg.compile(checkpointer);
选型误区
  • 因「LangGraph 名字熟」忽视团队已有 SAA Graph checkpoint 资产
  • 用 Graph 思维硬套 ReActAgent，手写 agent/tools 边
  • 把框架能力当 CorpAssist NS1 SLA 保证
推荐思路
  • 按场景复杂度、checkpoint 需求与 streamEvents 选型
  • 保留 Graph 子图，外层 AgentScope ReActAgent 路由
  • 工具 HTTP 契约统一，编排层可并存
先想 10 秒：CorpAssist 带 Redis checkpoint 的审批流，为何不必全部重写为 AgentScope Sub-agent？

审批链步数固定、interrupt 语义成熟，SAA Graph checkpoint 资产可直接复用；外层 ReActAgent 仅在需要多 Agent 协作与 streamEvents 审计时封装 Graph 为 @Tool，渐进迁移风险更低。

AgentScope Java 与 spring-ai-alibaba-starter-agentscope 是什么关系？
Starter 将 ReActAgent 包装为 SAA Graph 节点（见 spring-ai-alibaba-starter-agentscope 集成），实现「Graph 确定性 + AgentScope ReAct」混合编排，不是二选一。
LangGraph4j 与 SAA Graph 如何选？
团队已有 Python LangGraph 课成果、需 Java 对照选 LangGraph4j；已深度使用 Spring AI Alibaba 生态、要 Redis checkpoint 与 HITL 节点选 SAA Graph。两者可共存于不同子场景。

CrewAI 与 Microsoft AutoGen 在 Python 侧以角色委托与多 Agent 对话见长，适合 1～2 周演示原型。CorpAssist Java 生产栈不以它们为主框架，但架构评审时常被提及，需能说明边界。

迁移成本评估

从 LangGraph4j 或 SAA Graph 迁移到 AgentScope Java 时，主要工作量在状态模型、工具注册与checkpoint / Session 三块。使用下表估算 人日。

迁移项现状AgentScope Java 目标估算人日风险
单 ReAct AgentGraph ReactAgent 节点ReActAgent.builder + Msg1～2低
多节点 StateGraph8+ Node/EdgeSub-agent 或 Graph @Tool5～12高：需重设计协作语义
CheckpointRedis GraphSaverSessionStore + Msg 历史3～6中：interrupt 语义差异
工具层@ToolCallback@Tool + MCP2～3低：OpenAPI schema 可复用
Trace / 指标Actuator + 自定义streamEvents + Micrometer2～4中：排障习惯迁移
评测集JUnit + JSONgolden set + eval 脚本2～4低
yaml 
# migration-acceptance.yaml — LangGraph4j → AgentScope Java 验收门禁
acceptance:
  golden_set:
    pass_rate_min: 0.90          # 不低于迁移前 - 2%
    cases: 10
  latency:
    p95_max_ms: 15000            # ≤ 迁移前 × 1.1
  steps:
    avg_max: 4.0                 # ReAct 均值 ± 1 步
  observability:
    require_trace_id: true
    require_agent_steps_metric: true
  rollback:
    feature_flag: corpassist.agent.provider
    values: [langgraph4j, agentscope-java]
CorpAssist 应该统一一个框架还是多框架并存？
工具层统一，编排层可并存。 日历/工单 HTTP 工具只写一份；S2 可继续 SAA Graph，S3 多 Agent 用 AgentScope Java。BFF 通过 OpenAPI 契约屏蔽差异。
迁移第一步做什么？
导出 golden set 基线指标（pass rate、avg steps、P95），映射 State.messages → Msg 列表与 thread_id → session_id，再选一条 happy path 做 ReActAgent 等价实现 PoC。

本章结论

  • AgentScope Java：Msg + ReActAgent + Sub-agent + streamEvents，适合 S3 多 Agent 与 MCP。
  • SAA Graph：checkpoint + HITL 节点最成熟，适合确定性审批子图。
  • LangGraph4j：与 Python LangGraph 同构，适合双栈对照与已有课成果。
  • CrewAI/AutoGen：Python PoC 快，Java 生产需重写并统一工具契约。
  • 迁移：重点评估 State、Session、Trace 三块人日与 golden set 门禁。

Python 侧对照：平行课以 AgentScope Python vs LangGraph vs CrewAI 为主；Java 课以 AgentScope Java vs SAA Graph vs LangGraph4j 为主。双栈联调时保持 golden set JSON 与 trace_id 一致，不必统一编排入口类名。

判断练习

  1. CorpAssist 已在 SAA Graph 实现带 Redis checkpoint 的审批子图。架构组要求外层改为 AgentScope Java Sub-agent 协作。下列哪种集成最合理？

    参考答案

    将 Graph 编译子图封装为 AgentScope @Tool 或 Starter 节点，由主管 ReActAgent 在需要审批时调用。保留 checkpoint 资产，外层获得 Sub-agent 与 streamEvents。不应重写 checkpoint 逻辑，也不应让 Graph 与 ReActAgent 同时驱动同一条 ReAct 循环。

  2. 「CrewAI 多角色天然适合 CorpAssist S3 Java 生产，应直接上生产」——是否正确？

    参考答案

    不正确。 CrewAI 是 Python 框架，适合快速 PoC；S3 Java 生产需要 Permissions、Micrometer 步数预算与 golden set 门禁。建议 PoC 后迁移 AgentScope Java Sub-agent，工具契约经 OpenAPI 统一。
