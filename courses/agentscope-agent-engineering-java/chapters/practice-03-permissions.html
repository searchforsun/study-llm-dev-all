本章先记住 3 件事
  • approve = 暂停等确认：敏感工具触发审批事件后 Agent 暂停。
  • 改参 = 新 ToolResult：用户修改 args 后以 ToolResultBlock 恢复 Agent。
  • 审计不可删：DENY 决裁同样须落库，保留 180 天。

allow/approve/deny 三态

AgentScope Java 权限系统对每次 tool_call 给出三种决策：allow 直接执行；approve 暂停并 emit 待确认事件；deny 拒绝并返回错误 ToolResult 给 LLM。规则来源：静态 PermissionRule、工具 category、输入分析（如外部域名检测）。

CorpAssist · 外发邮件审批 Agent 提议 send_email(to=partner@external.com)，权限引擎返回 approve；前端展示参数表单，用户选 ALLOW / MODIFY / DENY 后，Spring Controller 调用 agent.call(toolResultMsg) 恢复执行。
Permission 三态事件流
sequenceDiagram
    participant U as 用户/审批员
    participant C as Spring Controller
    participant A as ReActAgent
    participant T as 工具
    A->>A: 推理 → tool_call send_email
    A-->>C: PermissionApproveEvent
    C-->>U: SSE 展示待审批参数
    U->>C: ALLOW / MODIFY / DENY
    C->>A: call(toolResultMsg)
    alt ALLOW
        A->>T: 执行工具
    else DENY
        A->>A: 带拒绝原因继续推理
    end

approve 暂停在 Acting 前；决裁通过 call() 恢复，非轮询。

决裁行为CorpAssist 示例
ALLOW按原 args 执行确认发送会议纪要给项目组
MODIFY替换 args 再执行修改收件人列表后发送
DENY返回错误 ToolResult拒绝向个人邮箱发内部文档
java 
ReActAgent agent = ReActAgent.builder()
    .name("CorpAssistOffice")
    .model(dashScopeModel)
    .toolkit(Toolkit.builder()
        .tool(sendEmailTool, ToolOptions.builder()
            .permission(PermissionRule.builder()
                .decision(PermissionDecision.APPROVE)  // 敏感：需审批
                .build())
            .build())
        .tool(queryCalendarTool, ToolOptions.builder()
            .permission(PermissionRule.allow())       // 只读：直接执行
            .build())
        .build())
    .hook(new ToolConfirmationHook(Set.of("send_email", "create_ticket")))
    .build();

// streamEvents 消费审批事件
agent.streamEvents(userMsg).subscribe(evt -> {
    if (evt instanceof PermissionApproveEvent p) {
        ui.showApprovalForm(p.toolName(), p.toolArgs());
    }
});
权限反模式
  • 所有工具都 approve，体验极差
  • 前端超时自动 ALLOW
  • deny 不写审计记录
分级策略
  • 只读查询 allow 默认
  • 外发/删改/权限变更 approve
  • 超时 DENY 而非自动执行
PostReasoningEvent.stopAgent() 与 Permission approve 有何区别？

Hook stopAgent() 是通用暂停点（PostReasoning/PostActing）；Permission approve 绑在 tool 粒度，自动识别敏感工具并 emit 结构化审批事件。CorpAssist 推荐 Permission 规则 + Hook 兜底危险工具白名单。

人工审批审计

人工审批审计 须记录：session_id、agent_name、tool_name、original_args、modified_args、decision、operator_id、timestamp、trace_id。CorpAssist NS2 合规要求保留 180 天，append-only 存储，与 OTel span 关联。

java 
@Service
public class HitlAuditService {
    private final Path auditFile = Paths.get("audit.jsonl");

    public void record(HitlAuditEntry entry) throws IOException {
        String line = new ObjectMapper().writeValueAsString(Map.of(
            "ts", Instant.now().toString(),
            "session_id", entry.sessionId(),
            "tool", entry.toolName(),
            "original_args", entry.originalArgs(),
            "modified_args", entry.modifiedArgs(),
            "decision", entry.decision(),      // ALLOW | MODIFY | DENY
            "operator", entry.operatorId(),
            "trace_id", entry.traceId()
        ));
        Files.writeString(auditFile, line + "\n",
            StandardOpenOption.CREATE, StandardOpenOption.APPEND);
    }
}

// Controller 决裁回调
@PostMapping("/chat/confirm")
public Flux confirmTool(@RequestBody ToolConfirmRequest req) {
    auditService.record(buildEntry(req));
    Msg toolResult = req.confirmed()
        ? buildAllowResult(req)
        : buildDenyResult(req.reason());
    return agent.streamEvents(toolResult).map(ChatEvent::from);
}
字段必填说明
session_id是关联用户会话
tool_name是待审批工具名
original_args是模型提议参数 JSON
modified_argsMODIFY 时用户改后参数
decision是ALLOW / MODIFY / DENY
trace_id推荐OTel 关联排障
CorpAssist · 合规抽查 安全团队问：「上周是否有 Agent 向个人邮箱发内部文档？」审计员查 audit.jsonl 中 decision=DENY 与 MODIFY 记录——DENY 记录不可省略，否则无法追溯拦截事件。

外部执行循环

外部执行循环 适用于工具逻辑完全在外部系统（采购审批、银行转账、人工填表）。工具抛出 ToolSuspendException 或注册为 Schema-only ExternalTool，Agent 返回 GenerateReason.TOOL_SUSPENDED，宿主应用执行外部流程后以 ToolResultBlock 恢复。

java 
@Tool(name = "submit_purchase_order", description = "提交采购单到 ERP 审批流")
public ToolResultBlock submitPurchaseOrder(
        @ToolParam(name = "amount") double amount,
        @ToolParam(name = "vendor") String vendor) {
    throw new ToolSuspendException(
        "Awaiting ERP approval workflow for vendor=" + vendor + " amount=" + amount);
}

// 宿主恢复
Msg response = agent.call(userMsg).block();
if (response.getGenerateReason() == GenerateReason.TOOL_SUSPENDED) {
    List<ToolUseBlock> pending = response.getContentBlocks(ToolUseBlock.class);
    // 外部 ERP 完成后：
    ToolUseBlock toolUse = pending.get(0);
    Msg toolResult = Msg.builder()
        .role(MsgRole.TOOL)
        .content(ToolResultBlock.of(toolUse.getId(), toolUse.getName(),
            TextBlock.builder().text("ERP approved: PO-2026-001").build()))
        .build();
    response = agent.call(toolResult).block();
}
approve 与 TOOL_SUSPENDED 如何选择？
approve 适合「用户在前端点确认/改参」的同步 HITL；TOOL_SUSPENDED 适合「逻辑在外部系统异步跑完再回传」的长事务。CorpAssist 单次发邮件用 approve；大额采购用 external + ERP 工作流。
CorpAssist 场景机制恢复方式
单次发邮件确认Permission approveController confirm API
ERP 采购审批ToolSuspendExceptionWebhook 回传 ToolResult
Schema-only 远程 APIExternalTool宿主执行后 call(result)

与 SAA 审批节点对照

Spring AI Alibaba Graph 的 interrupt_before / 人工节点与 AgentScope Java HITL 概念对齐：Graph 在节点前持久化 checkpoint；AgentScope 在 tool 权限 approve 时 emit 事件暂停。双栈 CorpAssist 可约定：Java Graph 跑长事务审批流，AgentScope ReActAgent 跑轻量 approve，审计 JSON Schema 统一。

AgentScope 侧避免
  • approve 事件不落库
  • MODIFY 后不写 modified_args
  • TOOL_SUSPENDED 后丢失 pending tool id
SAA 侧对齐
  • Graph interrupt ↔ Permission approve 暂停点
  • 统一 audit schema 跨栈
  • SSE 推送待审批到同一前端组件

AgentScope approve 与 LangGraph interrupt_before 有何异同？
同：都在敏感步骤前暂停等人输入。异：LangGraph interrupt 绑 checkpoint 节点；AgentScope approve 绑 tool Permission，粒度更细到单次 tool_call。CorpAssist 双栈可统一前端审批 UX，后端分别 resume。

本章结论

  • 三态权限：allow 直执行、approve 暂停、deny 拒绝。
  • 审计：原参/改参/决裁/操作者全量记录，含 DENY。
  • 外部循环：ToolSuspendException + call(toolResult) 恢复。
  • SAA 对照：Graph interrupt 与 approve 可双栈统一 UX。
  • NS2：敏感工具默认 approve，而非事后追责。

判断练习

  1. 排障：Agent 返回 TOOL_SUSPENDED 后团队直接 agent.call() 不带 ToolResult，Agent 反复 suspend。正确恢复顺序是？

    参考答案

    先从 response 取出 pending ToolUseBlock（含 tool id），外部执行完成后构造对应 ToolResultBlock（id 必须匹配），再 agent.call(toolResultMsg)。空 call() 无法消掉 suspended 状态，PendingToolRecoveryHook 可能补错误结果导致偏航。
