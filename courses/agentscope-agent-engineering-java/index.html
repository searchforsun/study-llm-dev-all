AOP vs Workflow 哲学

AOP（Agent-Oriented Programming） 把 Agent 视为程序基本单元：每个 ReActAgent 拥有名称、系统提示、ChatModel、工具集与 Session，通过 Msg 传递协作。与 SAA Graph 的「状态 + 节点 + 边」不同，AOP 强调异步事件流与可插拔 Middleware，调试依赖 streamEvents() 而非图遍历日志。

AgentScope Java AOP 组件关系
flowchart TB
    User[User Msg] --> Agent[ReActAgent]
    Agent --> Model[DashScopeChatModel]
    Agent --> Tools[@Tool / MCP]
    Agent --> Session[Session + StateStore]
    Agent --> Events[streamEvents]
    Events --> UI[CorpAssist SSE]
    Agent --> Sub[Sub-agent Markdown]

AgentScope Java 2.0 以 ReActAgent 为中心，模型、工具、会话、事件均为可插拔组件。

Graph 思维硬套
  • 先画 CompiledGraph 再强行映射 ReActAgent API
  • 把 Msg 当成普通 Map 塞进 Graph State
  • 忽略 streamEvents，只等 call().block() 最终字符串
AOP 原生写法
  • 先定义 Agent 角色与 @Tool，再考虑 Sub-agent 或嵌入 Graph Tool
  • 用 UserMessage 构造用户输入（或 agent.call(List.of(...), ctx)）
  • 消费 Typed Event 驱动 UI 与审计

先想 10 秒：AOP 与 Graph Workflow 最大的工程差异是什么？

Graph 以显式节点与 State 传递为主，流程结构在编译期可见；AOP 以 Agent 内部 ReAct 循环为主，中间态通过事件流暴露，多 Agent 协作靠 Sub-agent 声明而非画边。

AgentScope Java 还需要自己画 Graph 吗？
单 Agent ReAct 不需要；多 Agent 可用 Markdown 声明 Sub-agent。仅当嵌入既有 SAA Graph 审批子流程时，Graph 仍在外部维护并由 AgentScope Tool 调用。
AOP 是否排斥确定性工作流？
不排斥。Middleware 可注入固定步骤；AOP 强调 Agent 作为组合单元，而非禁止预定义流程。CorpAssist 审批链可 Graph + HITL 节点实现，外层由 ReActAgent 路由。

ReActAgent.builder 模式

AgentScope Java 2.0 的 ReActAgent.builder 是创建 Agent 的标准入口：模型生成 → 解析 tool_call → 执行 @Tool → 将观察写回 Session → 再次调用模型，直到输出最终文本或达到 maxIters。与 SAA Graph 需手写 agent/tools 节点不同，循环对外不可见，通过 streamEvents 暴露中间态。

AgentScope Java ReAct 循环
sequenceDiagram
    participant U as User
    participant A as ReActAgent
    participant M as DashScopeChatModel
    participant T as @Tool

    U->>A: Msg (USER)
    loop maxIters
        A->>M: 推理请求
        M-->>A: thinking / tool_call / text
        alt 有 tool_call
            A->>T: 执行工具
            T-->>A: ToolResultBlock
        else 纯文本
            A-->>U: 最终 Msg
        end
    end

ReAct 循环在 Agent 内部闭合；外部通过 streamEvents 观察每一步。

ContentBlock 与 streamEvents

ContentBlock 是 AgentScope Java 的统一内容单元：text、file、image、model thinking、tool_result 均以 Typed Block 表达。消息不再只是字符串，而是 Block 列表。streamEvents() 将模型输出与工具执行拆成细粒度事件，CorpAssist 前端可据此展示「正在查日历…」进度条。

ContentBlock / Event 类别含义UI 映射
TextBlock / TextDeltaEvent流式可见文本打字机效果
ThinkingBlock模型推理过程（若模型支持）可折叠「思考过程」
ToolCallStartEvent工具名与参数工具调用卡片
ToolCallEndEvent工具结果与耗时更新卡片状态
ReplyEndEvent整轮回复边界启用下一轮输入

消费 streamEvents 事件流

java 
import io.agentscope.core.event.*;

StringBuilder buffer = new StringBuilder();
agent.streamEvents(userMsg, ctx).subscribe(event -> {
    switch (event) {
        case TextDeltaEvent e -> {
            buffer.append(e.delta());
            System.out.print(e.delta());
        }
        case ToolCallStartEvent e ->
            System.out.printf("%n[工具] %s(%s)%n", e.name(), e.args());
        case ToolCallEndEvent e ->
            System.out.printf("[完成] %s 耗时 %dms%n", e.name(), e.durationMs());
        case ReplyEndEvent e ->
            System.out.println("--- 本轮完成 ---");
        default -> { /* OTel span 记录 */ }
    }
});
反模式
  • 只用 call().block() 阻塞等待，前端无进度
  • 忽略 ToolCallStartEvent，用户不知 Agent 在做什么
  • ThinkingBlock 直接展示给终端用户（可能泄露推理链）
推荐
  • SSE / WebSocket 转发 streamEvents 到 CorpAssist UI
  • 工具事件显示名称与耗时，隐藏敏感参数
  • thinking 默认折叠，仅 debug 模式开启

Session 生命周期

Session 生命周期 由「一次 call() 的 RuntimeContext」与「跨调用的 AgentStateStore」共同定义：调用开始加载状态，调用结束写回；同 Session 并发请求会被框架串行化。

sequenceDiagram
    participant BFF as Spring BFF
    participant A as ReActAgent
    participant Store as AgentStateStore

    BFF->>A: call(msgs, RuntimeContext)
    A->>Store: load(userId, sessionId)
    Store-->>A: AgentState
    A->>A: ReAct 推理 + 工具
    A->>Store: save(AgentState)
    A-->>BFF: Msg 回复

本章先记住 3 件事
  • approve = 暂停等确认：敏感工具触发审批事件后 Agent 暂停。
  • 改参 = 新 ToolResult：用户修改 args 后以 ToolResultBlock 恢复 Agent。

allow/approve/deny 三态

AgentScope Java 权限系统对每次 tool_call 给出三种决策：allow 直接执行；approve 暂停并 emit 待确认事件；deny 拒绝并返回错误 ToolResult 给 LLM。规则来源：静态 PermissionRule、工具 category、输入分析（如外部域名检测）。

CorpAssist · 外发邮件审批 Agent 提议 send_email(to=partner@external.com)，权限引擎返回 approve；前端展示参数表单，用户选 ALLOW / MODIFY / DENY 后，Spring Controller 调用 agent.call(toolResultMsg) 恢复执行。

@Component
public class KnowledgeBaseTools {
    private static final String INDEX = "corpassist-hr-kb";
    private final DashScopeApi dashScopeApi;

    @Tool(name = "search_knowledge_base",
          description = "检索 CorpAssist 企业知识库。输入自然语言问题，返回带 source/page/score 的片段。")
    public String searchKnowledgeBase(@ToolParam(name = "query") String query) {
        DocumentRetriever retriever = new DashScopeDocumentRetriever(dashScopeApi,
            DashScopeDocumentRetrieverOptions.builder()
                .withIndexName(INDEX)
                .withTopK(4)
                .build());
        List<Document> docs = retriever.retrieve(new Query(query));
        if (docs.isEmpty()) {
            return "未找到相关文档，请换关键词或转人工。";
        }
        StringBuilder sb = new StringBuilder();
        for (int i = 0; i < docs.size(); i++) {
            Document d = docs.get(i);
            sb.append(String.format("[%d] source=%s page=%s score=%s%n%s%n---%n",
                i + 1,
                d.getMetadata().get("source"),
                d.getMetadata().get("page"),
                d.getMetadata().get("score"),
                d.getText().substring(0, Math.min(400, d.getText().length()))));
        }
        return sb.toString();
    }
}

public record Citation(int refId, String source, String page, String snippet) {}

public String formatCitationsJson(List<Document> docs) {
    List<Citation> citations = new ArrayList<>();
    for (int i = 0; i < docs.size(); i++) {
        Document d = docs.get(i);
        citations.add(new Citation(i + 1,
            (String) d.getMetadata().get("source"),
            String.valueOf(d.getMetadata().get("page")),
            d.getText().substring(0, Math.min(300, d.getText().length()))));
    }
    return new ObjectMapper().writeValueAsString(Map.of(
        "citations", citations));
}

private static final double MIN_SCORE = 0.65;

@Tool(name = "search_knowledge_base")
public String searchWithThreshold(@ToolParam(name = "query") String query) {
    List<Document> docs = vectorStore.similaritySearch(
        SearchRequest.builder().query(query).topK(4).build());
    List<Document> filtered = docs.stream()
        .filter(d -> score(d) >= MIN_SCORE)
        .toList();
    if (filtered.isEmpty()) {
        return "检索置信度均低于 " + MIN_SCORE + "，无法可靠回答。请转人工或细化问题。";

@Bean
public Graph corpAssistGraph(AgentScopeAgent agentScopeAgent) {
    return Graph.builder()
        .addNode("route", this::routeIntent)
        .addNode("agentscope", state -> {
            String msg = state.get("userMessage", String.class);
            AgentResult result = agentScopeAgent.run(msg, state.getSessionId());
            state.put("agentResult", result.content());
            state.put("steps", result.steps());
            return state;

        .addNode("hitl", this::checkApproval)
        .addEdge("route", "agentscope", s -> "office".equals(s.get("intent")))
        .addEdge("agentscope", "hitl")
        .build();
}

@Service
public class PythonAgentClient {
    private final WebClient webClient;

    public Flux<AgentEvent> streamFromPython(String tenantId, AgentRunRequest req) {
        return webClient.post()
            .uri("/v1/chat/stream")
            .header("X-Tenant-Id", tenantId)
            .header("X-Trace-Id", TraceContext.currentTraceId())
            .bodyValue(Map.of(
                "session_id", req.sessionId(),
                "message", req.message()))
            .retrieve()
            .bodyToFlux(String.class)
            .filter(line -> line.startsWith("data:"))
            .map(this::parseEvent);
    }
}

@RestController
@RequestMapping("/agent")
public class CapstoneAgentController {

    private final ReActAgent capstoneAgent;

    public CapstoneAgentController(CapstoneAgentConfig config) {
        this.capstoneAgent = ReActAgent.builder()
            .name("CorpAssist-Capstone")
            .systemPrompt("""
                你是 CorpAssist 办公助手。先查制度再订会议室，最后发邮件。
                信息足够则直接回答，不要多余工具调用。""")
            .model(config.dashScopeModel())
            .tools(config.capstoneTools())  // searchPolicy, bookMeeting, sendEmail
            .maxIterations(5)
            .build();
    }

    @PostMapping(value = "/chat", produces = MediaType.TEXT_EVENT_STREAM_VALUE)
    public Flux<ServerSentEvent<String>> chat(
            @RequestBody ChatRequest req,
            @RequestHeader("X-Session-Id") String sessionId) {
        Msg userMsg = Msg.builder()
            .role(MsgRole.USER)
            .textContent(req.message())
            .build();
        RuntimeContext ctx = RuntimeContext.builder()
            .sessionId(sessionId)
            .userId(req.userId())
            .build();
        return capstoneAgent.streamEvents(userMsg, ctx)
            .map(ev -> ServerSentEvent.builder(ev.type()).data(ev.payload()).build());
    }
}

@Component
public class AgentMetricsListener implements AgentEventListener {

    private final Counter stepCounter;
    private final Counter tokenCounter;

    public AgentMetricsListener(MeterRegistry registry) {
        this.stepCounter = Counter.builder("agent.steps")
            .description("ReAct tool invocation count")
            .register(registry);
        this.tokenCounter = Counter.builder("agent.tokens")
            .description("Total tokens from DashScope usage")
            .register(registry);
    }

    @Override
    public void onToolCall(ToolCallEvent event) {
        stepCounter.increment();
    }

    @Override
    public void onModelResponse(ModelResponseEvent event) {
        if (event.usage() != null) {
            tokenCounter.increment(event.usage().totalTokens());
        }
    }
}

事件流断连

CorpAssist BFF 通过 TEXT_EVENT_STREAM 消费 AgentScope Java streamEvents()。断连表现为客户端收不到后续 tool_call 事件，或 Gateway 返回 504。与 Python FastAPI SSE 类似，但 Java 侧常见 Reactor 背压与 Spring Cloud Gateway 超时。

SSE 事件流断连排查流程
flowchart TD
    Alert[客户端 SSE 中断 / 504] --> Trace[查 trace_id 对应 Event 日志]
    Trace --> Q1{Agent 侧是否仍在 emit?}
    Q1 -->|是| GW[检查 Gateway / Nginx 超时]
    Q1 -->|否| Q2{是否 CancellationException?}
    Q2 -->|是| Client[客户端断开或未读 Flux]
    Q2 -->|否| Q3{Reactor 线程阻塞?}
    Q3 -->|是| Block[工具同步 IO 阻塞 event 线程]
    GW --> Fix1[proxy_read_timeout / response-timeout]
    Client --> Fix2[前端 EventSource reconnect + Last-Event-ID]
    Block --> Fix3[工具改 boundedElastic / 虚拟线程]

工具权限拒绝

工具权限拒绝 发生在 practice-03 的 Permissions 三态：deny 永久拒绝、approve 待人工、allow 放行。生产故障常表现为 Agent 反复尝试同一工具或卡在 HITL 等待。

// PermissionMiddleware — deny 时返回可解析错误，避免 silent loop
@Component
public class CorpAssistPermissionGate implements ToolPermissionEvaluator {

    @Override
    public PermissionDecision evaluate(ToolCallContext ctx) {
        if ("bookMeeting".equals(ctx.toolName()) && !ctx.userRoles().contains("CALENDAR_WRITE")) {
            return PermissionDecision.deny(
                "PERMISSION_DENIED: 用户无 CALENDAR_WRITE，请引导用户申请权限或改查可用会议室");
        }
        if (isSensitive(ctx.toolName())) {
            return PermissionDecision.approve("需人工确认订会议室操作");
        }
        return PermissionDecision.allow();
    }
}

动手练习

填写 Harness 升级检查清单与 observability 衔接表，并对 CorpAssist「Q1 纪要汇总 PPT」需求做 ReAct vs Harness 判定（约 25 分钟）。

操作步骤

  1. 对照「ReActAgent vs HarnessAgent」表，为 Capstone 三个工具场景标注应使用的 Agent 类型。
  2. 在「升级触发条件」表中，为「Q1 40 场纪要 → PPT」需求勾选满足的条目并写结论。
  3. 复制 Workspace yaml 配置块，填写 max-disk-mb 与 retention-days 的合理值并说明理由。
  4. 在「observability 衔接」表补一行：Harness 任务 stalled 5 分钟时的告警规则建议。
  5. 可选：完成 demos/advanced-04-harness-bridge-lab/ 清单验收脚本。

判断练习

  1. 团队将 CorpAssist 长任务 maxIterations 从 5 调到 50，仍频繁截断——是否应继续调到 200？

    参考答案

    不应。 步数不可预估且已 >10 是 Harness 升级信号。继续堆 maxIterations 会恶化延迟、Token 成本与 SSE 断连风险；应进入 Harness 课做计划分解与检查点。

  2. （找错配置）下列哪条 Workspace 配置在生产环境风险最高？ A) max-disk-mb: 512 B) preview 允许 .. 路径 C) retention-days: 7

    参考答案

    B。 Preview 必须禁止路径遍历，仅映射 taskId 下白名单文件。A/C 合理，需配合 quota 告警。
