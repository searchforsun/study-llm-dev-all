为什么要学本章

CorpAssist 知识库问答不能仅靠静态 RAG 管线——Agent 需自主决定何时检索、检索什么、是否多轮。Agentic RAG 把 RAG 系统（Python）rag-system-py 的检索链封装为 Agent 工具，并配合 引用溯源 与 幻觉抑制 策略落地生产。

学完你能
  • 衔接 rag-system-py 的 LCEL/Chroma 检索链为 AgentScope 工具
  • 构建 CorpAssist 知识库问答 ReActAgent
  • 在回答中输出可点击的文档引用
  • 应用检索置信度与拒答策略抑制幻觉
本章先记住 3 件事
  • 检索是工具不是前置步骤：Agent 按需调用，避免每次都全库检索。
  • 引用必须绑 chunk metadata：doc_id、page、score 写入回答。
  • 低置信度拒答：score 低于阈值时拒绝编造，建议转人工或换问法。

衔接 rag-system-py 检索链

在 rag-system-pyrag-system-py 中你已用 LangChain LCEL 构建 检索链（loader → splitter → embedding → Chroma retriever）。本章将该 retriever 包装为 async 工具函数，注册到 CorpAssist 知识库 Agent 的 Toolkit。

CorpAssist · 企业知识库 Agent 员工问：「年假最多几天？需要提前多久申请？」Agent 应先调用 search_knowledge_base，基于检索 chunk 回答，并在末尾列出 [1] 员工手册 §3.2 引用。
Agentic RAG 调用链
flowchart LR
    Q[用户问题] --> Agent[ReActAgent]
    Agent -->|需要事实| Tool[search_knowledge_base]
    Tool --> Retriever[Chroma Retriever\nfrom rag-system-py]
    Retriever --> Chunks[Top-K Chunks + metadata]
    Chunks --> Agent
    Agent --> Ans[带引用的回答]

Agent 自主决定是否检索；多轮对话可多次调用工具 refine query。

python 
from agentscope.agent import ReActAgent
from agentscope.tool import Toolkit, ToolResponse
from langchain_community.vectorstores import Chroma
from langchain_openai import OpenAIEmbeddings

# 复用 rag-system-py 课 demo 构建的 vectorstore
vectorstore = Chroma(persist_directory="./data/chroma", embedding_function=OpenAIEmbeddings())
retriever = vectorstore.as_retriever(search_kwargs={"k": 4})

async def search_knowledge_base(query: str) -> ToolResponse:
    """检索 CorpAssist 企业知识库。输入自然语言问题，返回带 doc_id/page/score 的片段。"""
    docs = retriever.invoke(query)
    if not docs:
        return ToolResponse(content="未找到相关文档，请换关键词或转人工。")
    blocks = []
    for i, doc in enumerate(docs, 1):
        meta = doc.metadata
        score = meta.get("score", "N/A")
        blocks.append(
            f"[{i}] doc={meta.get('source','?')} page={meta.get('page','?')} score={score}\n{doc.page_content[:400]}"
        )
    return ToolResponse(content="\n---\n".join(blocks))

toolkit = Toolkit()
toolkit.register_tool_function(search_knowledge_base)

kb_agent = ReActAgent(
    name="CorpAssistKB",
    sys_prompt="你是知识库助手。回答必须基于 search_knowledge_base 结果，并标注 [n] 引用。无依据时拒答。",
    model=DashScopeChatModel(model_name="qwen-max", api_key="..."),
    formatter=DashScopeChatFormatter(),
    toolkit=toolkit,
    max_iters=6,
)
先想 10 秒：为何不把 retriever 固定放在 Agent 前面做「先检索再回答」？

固定链对纯 FAQ 低延迟，但 CorpAssist 统一入口混合闲聊、工具操作与知识问答——Agentic 模式让 Agent 自主决定是否检索，避免无关问题也全库 embedding 浪费 Token。

知识库问答 Agent

Agentic RAG 与「检索-然后-生成」管线的区别：Agent 可先澄清模糊问题、多次检索、或判断无需检索（闲聊）。CorpAssist KB Agent 的 sys_prompt 应强制：有事实性问题必须先检索；回答格式含引用块。

模式触发适用
单次检索问题明确政策条文查询
多轮检索Agent 改写 query跨文档综合
跳过检索问候/元问题「你能做什么」
python 
KB_SYS_PROMPT = """你是 CorpAssist 知识库助手。
规则：
1. 涉及公司制度/政策/流程的事实性问题，必须先调用 search_knowledge_base
2. 回答正文用 [1][2] 标注引用；末尾输出 ### 参考文献
3. 工具无结果或低置信度时拒答，可建议 transfer_to_human
4. 问候语直接回复，不必检索"""

kb_agent = ReActAgent(
    name="CorpAssistKB",
    sys_prompt=KB_SYS_PROMPT,
    toolkit=toolkit,
    max_iters=6,
)
KB Agent 反模式
  • sys_prompt 未强制先检索
  • 检索与回答用不同模型
  • 无引用也输出具体数字
推荐
  • 事实问题必须先 tool_call
  • 同一 qwen-plus 保证风格一致
  • 每条事实对应 citation ref_id

引用溯源

引用溯源 要求每条事实对应 chunk metadata。工具返回格式应结构化（JSON 或固定模板），Agent 回答末尾附「参考文献」列表，前端可渲染为可点击链接（doc_id → 原文预览）。

python 
import json

async def search_knowledge_base_structured(query: str) -> ToolResponse:
    docs = retriever.invoke(query)
    citations = []
    for i, doc in enumerate(docs, 1):
        citations.append({
            "ref_id": i,
            "source": doc.metadata.get("source"),
            "page": doc.metadata.get("page"),
            "snippet": doc.page_content[:300],
        })
    payload = {"query": query, "citations": citations}
    return ToolResponse(content=json.dumps(payload, ensure_ascii=False))

# sys_prompt 片段：
# 「正文用 [1][2] 标注；末尾输出 ### 参考文献 列表，含 source 与 page」
引用溯源数据流
flowchart LR
    Retriever[Chroma Top-K] --> Tool[ToolResponse JSON]
    Tool --> Agent[ReActAgent 推理]
    Agent --> UI[回答 + 参考文献块]
    UI --> Link[doc_id → 原文预览]

metadata 从 retriever 经工具到 UI，全程不丢失 source/page。

metadata 字段用途前端展示
source文档路径或 doc_id可点击链接
pagePDF 页码跳转预览
score相似度低分灰显/拒答

幻觉抑制策略

幻觉抑制 组合拳：① 检索 score 阈值过滤；② sys_prompt 拒答指令；③ 回答后自检（可选第二 pass）；④ 低置信度转 transfer_to_human。与 rag-system-py 评估章节的 faithfulness 指标对齐。

易幻觉
  • 无检索直接回答政策数字
  • 忽略低 score chunk 硬拼答案
  • 引用号与正文不对应
抑制策略
  • score < 0.65 拒答并建议换问法
  • 工具空结果禁止编造
  • 引用数 ≤ 检索 chunk 数
python 
MIN_SCORE = 0.65

async def search_with_threshold(query: str) -> ToolResponse:
    docs = retriever.invoke(query)
    filtered = [d for d in docs if d.metadata.get("score", 1.0) >= MIN_SCORE]
    if not filtered:
        return ToolResponse(
            content=f"检索置信度均低于 {MIN_SCORE}，无法可靠回答。请转人工或细化问题。"
        )
    return format_citations(filtered)
Agentic RAG 比固定 RAG 链更费 Token，何时仍值得？

当问题类型混合（部分需检索、部分需工具操作）、或需多轮改写 query 时，Agent 按需检索更省平均 Token。纯 FAQ 高 QPS 场景可保留固定链 + 小模型。

Agentic RAG 如何与 rag-system-py 评估指标对齐？
复用 rag-system-py 的 faithfulness / answer_relevance 评测集：Agentic 路径在 golden 中增加 must_call_tools: ["search_knowledge_base"] 断言。引用准确率单独测 citation 与 chunk 对应关系。
多轮检索何时触发？
首次检索 score 偏低或结果不覆盖子问题时，Agent 在 ReAct 第二轮改写 query 再调工具。max_iters=6 限制最多约 2～3 次检索；更多轮应拆分为明确子问题或转人工。
Chroma 与 DashScope Embedding 如何配合？
索引阶段 embedding 模型须与查询阶段一致（见 rag-system-py）。CorpAssist 生产可用 DashScope text-embedding-v3；Agent 工具内只做 retriever.invoke，不在线换 embedding 模型。
CorpAssist · 制度问答拒答 用户问「火星移民政策」——检索 score 均 <0.65，工具返回拒答文案，Agent 不编造数字并建议联系 HR 或换关键词。审计记录 tool 空结果与拒答原因。
静态 RAG 链
  • 所有问题都检索，闲聊也 embedding
  • 无法在同一 Agent 内调工单工具
  • query 改写需额外 pipeline 节点
Agentic RAG
  • 按需检索，混合意图统一入口
  • 检索 + 办公工具同一 ReActAgent
  • 多轮 query 改写原生支持

本章结论

  • 衔接 rag-system-py 检索链：retriever 包装为 Toolkit 工具，复用已有索引。
  • 知识库问答 Agent：Agentic 模式自主决定检索时机与次数。
  • 引用溯源：chunk metadata → [n] 标注 + 参考文献块。
  • 幻觉抑制策略：阈值拒答 + 空结果禁编造 + HITL 兜底。
