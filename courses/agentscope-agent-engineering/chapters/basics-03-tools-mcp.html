为什么要学本章

CorpAssist S3 需连接日历、邮件、工单与外部知识库——工具是 Agent 的「手」。AgentScope 2.0 的 Toolkit 统一注册本地函数与 MCP 远程服务；Skill 把多工具组合为可复用能力包。生产环境必须处理超时、鉴权与错误重试。

  • 接入 MCP Server 并映射到 Agent 工具列表
  • 配置工具超时、API Key 鉴权与 Workspace 边界

本章先记住 3 件事
  • Toolkit 是唯一入口：本地函数与 MCP 工具都注册进 Toolkit。
  • MCP 解耦集成：外部系统通过 MCP 协议暴露，Agent 不直连各 API 形态。
  • 失败可观测：工具错误应结构化返回，触发 Event 而非 silent fail。

工具注册与 Skill 组合

Toolkit 收集 Agent 可调用的工具。用 @tool 装饰器注册 Python 函数，docstring 自动生成 JSON Schema 供模型选择。Skill 将相关工具与提示词打包，例如 CorpAssist「工单 Skill」含 create_ticket / query_ticket。

Toolkit 与 Skill 层次
flowchart TB
    Agent[Agent] --> TK[Toolkit]
    TK --> T1[@tool 本地函数]
    TK --> T2[MCP 远程工具]
    SK[Skill 工单包] --> T1
    SK --> T2
    SK --> SP[Skill System Prompt]

Skill 是逻辑分组；运行时仍通过 Toolkit 统一调度。

注册 CorpAssist 工具

python 
from agentscope.tool import Toolkit, tool

@tool
def create_ticket(title: str, priority: str = "normal") -> str:
    """在 CorpAssist 创建工单。title: 标题；priority: low/normal/high。"""
    ticket_id = f"T-{hash(title) % 10000:04d}"
    return f"工单 {ticket_id} 已创建，优先级 {priority}"

@tool
def query_ticket(ticket_id: str) -> str:
    """查询工单状态。"""
    return f"{ticket_id} 状态：处理中"

toolkit = Toolkit(tools=[create_ticket, query_ticket])
先想 10 秒：工具 docstring 为什么必须写参数说明？

DashScope 等模型靠 schema + 描述选择工具与填参；缺失描述会导致错误调用或幻觉参数，是 CorpAssist 工单类 bug 的常见根因。

Skill 组合示例

python 
# Skill 将工具 + 领域提示打包（概念示例）
TICKET_SKILL_PROMPT = """
你是 CorpAssist 工单专员。创建前确认 title 与 priority。
可用工具：create_ticket, query_ticket。
"""

agent = Agent(
    name="Ticket-Agent",
    system_prompt=TICKET_SKILL_PROMPT,
    toolkit=Toolkit(tools=[create_ticket, query_ticket]),
)
  • Skill 不改变 Toolkit 调度机制，仅缩小模型可见工具集与提示边界。
  • 多 Skill 并存时，按路由 Agent 选择不同 toolkit 实例，避免工具名冲突。

MCP 客户端接入

MCP（Model Context Protocol） 让外部服务以标准方式暴露 tools/resources。AgentScope 2.0 内置 MCP 客户端，可将远程 Server 工具挂载到 Toolkit，避免为每个 SaaS 写适配层。

接入 MCP Server

python 
from agentscope.mcp import MCPClient
from agentscope.tool import Toolkit

# CorpAssist 知识库 MCP（示例配置）
mcp = MCPClient(
    server_url="http://localhost:8765/mcp",
    headers={"Authorization": "Bearer ${KB_MCP_TOKEN}"},
)

async def build_toolkit() -> Toolkit:
    remote_tools = await mcp.list_tools()
    local = [create_ticket, query_ticket]
    return Toolkit(tools=local + remote_tools)
集成方式适用CorpAssist 示例
@tool 本地内部微服务已有 Python SDK工单 API
MCP第三方或跨语言服务Confluence 知识库 MCP
Skill 组合多工具固定编排「会议纪要」Skill

前置课 MCP 概念见 Agent 编排（Python / LangGraph）agent-orchestration-engineering 高级章节。

MCP 配置文件（mcp.json 示例）

json 
{
  "servers": {
    "corpassist-kb": {
      "url": "http://localhost:8765/mcp",
      "headers": {"Authorization": "Bearer ${KB_MCP_TOKEN}"},
      "timeout": 45
    }
  }
}
  • MCP list_tools 返回的工具名可能与本地 @tool 重名，注册前加前缀如 kb_。
  • CorpAssist 知识库 MCP 应只暴露 read/search，不暴露 write（权限最小化）。
MCP 连接失败时 Agent 会怎样？
未设 timeout 时 ReAct 循环可能长时间挂起；应配置 MCPClient timeout 并在工具层返回 TOOL_ERROR retryable=true，让 Agent 提示用户稍后重试而非编造答案。
本地 @tool 与 MCP 工具能否混用？
可以，统一注册到同一 Toolkit。CorpAssist 惯例：内部工单用 @tool，外部 Confluence/日历用 MCP；注意工具名前缀避免冲突。

超时与鉴权

生产工具调用必须设 timeout 与凭证隔离。AgentScope 2.0 支持 Toolkit 级默认超时、单工具 override，以及 Workspace 权限边界（限制文件系统工具路径）。

反模式
  • 工具内硬编码 API Key
  • 无 timeout，线程永久阻塞
  • MCP Token 写进 Git 仓库
推荐
  • 环境变量 + DashScopeCredential 同类模式
  • 默认 timeout=30s，读操作可 60s
  • MCP headers 从 Secret 注入

超时与凭证配置

python 
import os
from agentscope.tool import Toolkit

toolkit = Toolkit(
    tools=[create_ticket, query_ticket],
    default_timeout=30.0,
)

# MCP 鉴权从环境变量读取
mcp_token = os.environ["CORPASSIST_MCP_TOKEN"]
mcp = MCPClient(
    server_url=os.environ["KB_MCP_URL"],
    headers={"Authorization": f"Bearer {mcp_token}"},
    timeout=45.0,
)
工具类型建议 timeout鉴权
本地内存查询5s无
CorpAssist 内部 API30sService Token
MCP 知识库45sBearer Token
批量导出120s + 异步OAuth

Workspace 边界 限制文件类工具只能访问指定目录，防止 Agent 被诱导读取 /etc/passwd 等路径。

错误重试策略

重试策略 需区分可重试错误（5xx、超时）与不可重试（4xx 参数错误）。Agent 层应限制重试次数，避免 ReAct 循环中对同一坏参数无限重试。

工具失败处理流
flowchart TD
    Call[工具调用] --> OK{成功?}
    OK -->|是| Obs[观察写回上下文]
    OK -->|否| Type{错误类型}
    Type -->|超时/5xx| Retry{重试 < 3?}
    Retry -->|是| Call
    Retry -->|否| Degrade[结构化错误 + 降级提示]
    Type -->|4xx/鉴权| Fail[立即失败 + 人工指引]

结构化错误信息帮助模型修正参数，而非空异常。

带重试的工具包装

python 
import asyncio
from functools import wraps

def retry_tool(max_attempts: int = 3, backoff: float = 1.0):
    def decorator(fn):
        @wraps(fn)
        async def wrapper(*args, **kwargs):
            last_err = None
            for attempt in range(max_attempts):
                try:
                    return await fn(*args, **kwargs)
                except (TimeoutError, ConnectionError) as e:
                    last_err = e
                    await asyncio.sleep(backoff * (attempt + 1))
            return f"TOOL_ERROR: {last_err}（已重试 {max_attempts} 次，请换参数或人工处理）"
        return wrapper
    return decorator
CorpAssist · S3 工单创建 模型误传非法 priority 值时，工具返回 400——不应重试，应返回清晰错误让 Agent 改用 enum 值。网络抖动导致 MCP 超时时，最多重试 2 次后提示用户稍后再试。
答辩 / 面试口述题
  1. CorpAssist 如何设计工具白名单与 MCP 权限对齐？
  2. 工具假执行（返回 mock 数据）如何被 Event 与审计发现？
  3. 重试 3 次仍失败时，Agent 应对用户说什么？

结构化错误返回格式

python 
def format_tool_error(code: str, message: str, retryable: bool) -> str:
    return f"TOOL_ERROR code={code} retryable={retryable}: {message}"

# 模型看到 retryable=false 时不应再传相同参数

本章结论

  • 工具注册与 Skill：@tool + Toolkit 统一入口，Skill 打包领域能力。
  • MCP 客户端：标准协议接入外部系统，减少 CorpAssist 适配代码。
  • 超时与鉴权：默认 timeout + 环境变量凭证，禁止硬编码 Secret。
  • 错误重试：区分可重试与不可重试，结构化错误反馈给 ReAct 循环。
