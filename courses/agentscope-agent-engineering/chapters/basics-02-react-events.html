内置 ReAct 循环

AgentScope 2.0 的 ReAct 循环 在 Agent 内部实现：模型生成 → 解析 tool_call → 执行 Toolkit → 将观察写回上下文 → 再次调用模型，直到输出最终文本或达到 max_iters。与 LangGraph 需手写 agent/tools 节点不同，此处循环对外不可见，通过事件暴露中间态。

AgentScope ReAct 循环
sequenceDiagram
    participant U as User
    participant A as Agent
    participant M as DashScopeChatModel
    participant T as Toolkit

    U->>A: UserMsg
    loop max_iters
        A->>M: 推理请求
        M-->>A: thinking / tool_call / text
        alt 有 tool_call
            A->>T: 执行工具
            T-->>A: 观察结果
        else 纯文本
            A-->>U: 最终回复
        end
    end

ReAct 循环在 Agent 内部闭合；外部通过 EventType 观察每一步。

ReActAgent / Agent 初始化

python 
from agentscope.agent import Agent, ReActAgent  # ReActAgent 为兼容别名
from agentscope.model import DashScopeChatModel
from agentscope.tool import Toolkit, tool

@tool
def query_calendar(date: str) -> str:
    """查询 CorpAssist 用户日历，输入 ISO 日期。"""
    return f"{date} 10:00 产品评审会"

agent = Agent(
    name="S3-Assistant",
    model=DashScopeChatModel(model="qwen-plus"),
    toolkit=Toolkit(tools=[query_calendar]),
    max_iters=10,  # 对应 LangGraph maxSteps
)
先想 10 秒：max_iters 与 LangGraph 的 conditional edge 回环有何不同？

LangGraph 通过图的边显式回到 agent 节点；AgentScope 在 Agent 内部计数迭代，超限直接终止并返回错误或部分结果，无需在图里画回边。

终止条件行为CorpAssist 建议
模型输出纯文本正常结束 ReAct展示最终回复
达到 max_iters强制停止，可能部分完成提示用户缩小问题
工具连续失败模型可能反复重试Toolkit 返回 TOOL_ERROR
用户 cancelRealtime Steering 中断见 practice-04 HITL

Event System（text/thinking/tool_call）

Event System 将模型输出与工具执行拆成细粒度事件。常见类型包括 TEXT_BLOCK_DELTA（流式文本）、thinking 相关块、TOOL_CALL_START / TOOL_CALL_END 等。CorpAssist 前端可据此展示「正在查日历…」进度条。

EventType 类别含义UI 映射
REPLY_START / END整轮回复边界会话气泡起止
TEXT_BLOCK_*流式可见文本打字机效果
thinking 块模型推理过程（若模型支持）可折叠「思考过程」
TOOL_CALL_*工具名与参数工具调用卡片

消费 reply_stream 事件流

python 
import asyncio
from agentscope.event import EventType
from agentscope.message import UserMsg

async def stream_to_ui(agent, question: str) -> None:
    buffer = []
    async for evt in agent.reply_stream(UserMsg("user", question)):
        match evt.type:
            case EventType.TEXT_BLOCK_DELTA:
                buffer.append(evt.delta)
                print("".join(buffer), end="", flush=True)
            case EventType.TOOL_CALL_START:
                print(f"\n[工具] {evt.tool_name}({evt.args})")
            case EventType.REPLY_END:
                print("\n--- 完成 ---")

asyncio.run(stream_to_ui(agent, "明天有什么会？"))
反模式
  • 只用 reply() 阻塞等待，前端无进度
  • 忽略 TOOL_CALL 事件，用户不知 Agent 在做什么
  • thinking 块直接展示给终端用户（可能泄露推理链）
推荐
  • SSE / WebSocket 转发 EventType 到 AG-UI
  • 工具事件显示名称与耗时，隐藏敏感参数
  • thinking 默认折叠，仅 debug 模式开启

事件序列完整示例（含 thinking）

python 
# 典型事件顺序（简化）
# REPLY_START → MODEL_CALL_START → TEXT_BLOCK_* / TOOL_CALL_* → REPLY_END

EVENT_UI = {
    "REPLY_START": "显示加载动画",
    "TEXT_BLOCK_DELTA": "追加可见文本",
    "TOOL_CALL_START": "展示工具卡片",
    "TOOL_CALL_END": "更新工具耗时",
    "REPLY_END": "启用下一轮输入",
}

async def render(agent, q: str):
    async for evt in agent.reply_stream(UserMsg("user", q)):
        handler = EVENT_UI.get(evt.type.name)
        if handler:
            log_ui(evt.type.name, handler)
  • thinking 块 在 qwen 部分模型可用，生产 UI 默认不向终端用户展示。
  • 每个 TOOL_CALL 应配对 START/END，缺失 END 是排障重点（见动手练习判断题）。

AG-UI/A2A 对接

AG-UI 是 Agent 前端交互协议；A2A（Agent-to-Agent） 支持跨服务 Agent 互调。AgentScope 2.0 事件流可序列化为 JSON 行推送到 SSE，或由 A2A 网关转发给 Java 侧消费者。

CorpAssist 事件流架构
flowchart LR
    AS[AgentScope Agent] -->|reply_stream| SSE[FastAPI SSE]
    SSE --> AGUI[CorpAssist Web AG-UI]
    AS --> A2A[A2A Gateway]
    A2A --> Java[AgentScope Java Service]
    Java --> Spring[Spring 业务系统]

Python Agent 产生事件，经 SSE 或 A2A 到达多端 UI 与 Java 栈。

SSE 桥接片段（FastAPI）

python 
from fastapi import FastAPI
from fastapi.responses import StreamingResponse
import json

app = FastAPI()

@app.post("/chat/stream")
async def chat_stream(body: dict):
    async def event_generator():
        async for evt in agent.reply_stream(UserMsg("user", body["q"])):
            yield f"data: {json.dumps({'type': evt.type.name, 'payload': evt.to_dict()})}\n\n"
    return StreamingResponse(event_generator(), media_type="text/event-stream")

与 AgentScope Java streamEvents 对照

CorpAssist 双栈部署时，Python 侧 reply_stream 与 Java 侧 streamEvents 应对齐事件名与 payload 结构，便于统一 OTel trace 与前端组件。

能力Python AgentScope 2.0Java AgentScope
流式 APIagent.reply_stream(msg)agent.streamEvents(msg)
文本增量EventType.TEXT_BLOCK_DELTATextDeltaEvent
工具调用TOOL_CALL_STARTToolCallStartEvent
结束REPLY_ENDReplyEndEvent

Java streamEvents 对照片段

java 
// Java 侧消费（与 Python EventType 语义对齐）
agent.streamEvents(userMsg).subscribe(event -> {
    switch (event) {
        case TextDeltaEvent e -> ui.appendText(e.delta());
        case ToolCallStartEvent e -> ui.showTool(e.name(), e.args());
        case ReplyEndEvent e -> ui.finishTurn();
        default -> { /* OTel span 记录 */ }
    }
});

Java 轨详解见 AgentScope 多 Agent 工程实践（Java）agentscope-agent-engineering-java。

Python 与 Java 事件字段不一致怎么办？
在 A2A 网关维护映射表，统一 trace_id、tool_name、delta 字段；集成测试对比同一条 CorpAssist 问句的双栈事件序列。

本章结论

  • 内置 ReAct 循环：Agent 内部完成推理-工具交替，max_iters 防止死循环。
  • Event System：text / thinking / tool_call 事件支持流式 UI 与审计。
  • AG-UI/A2A：SSE 或 A2A 网关将 Python 事件送达 CorpAssist 前端与 Java 服务。
  • streamEvents 对照：双栈统一事件语义，便于混合部署与排障。

判断练习

  1. （排障）用户反馈 CorpAssist UI 长时间空白，后端日志显示 TOOL_CALL_START 后无 END。最可能的根因与排查顺序是？

    参考答案

    工具执行挂起或超时未配置。排查顺序：① 查工具侧 API 延迟与错误；② 确认 Toolkit 是否设置 timeout；③ 检查 reply_stream 消费循环是否 await 阻塞；④ 查看 OTel span 中 TOOL_CALL 段耗时。不应先怀疑模型配额。
