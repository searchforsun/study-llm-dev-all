为什么要学本章

CorpAssist 不能绑定单一模型：客服要快、报告要准、摘要要省——模型无关编程 让你切换 DashScope / OpenAI 而不改 Agent 逻辑。DashScope 通义 是 AgentScope 一等公民；多模型路由 直接服务北星 NS3 成本权衡。

本章先记住 3 件事
  • 换模型 = 换 ChatModel 实例：Agent / Toolkit / Memory API 不变。
  • DashScope 原生：Credential + Formatter 与通义 tool_call 对齐。
  • 路由降本：复杂推理 qwen-max，摘要/分类 qwen-turbo。

模型无关编程

AgentScope 2.0 的 Model 层 抽象异步调用、流式响应与统一 ChatResponse。业务代码依赖 ChatModel 接口，而非 HTTP 细节。与 LangChain 的 ChatModel 理念一致，但 Formatter 与 Event 与 AgentScope Agent 深度集成。

Model 层与 Agent
flowchart LR
    Agent[Agent] --> FMT[Formatter]
    FMT --> CM[ChatModel 接口]
    CM --> DS[DashScopeChatModel]
    CM --> OAI[OpenAIChatModel]
    CM --> Ollama[OllamaChatModel]

切换 Provider 只需替换 ChatModel 与匹配 Formatter。

切换 Provider 示例

python 
import os
from agentscope.model import DashScopeChatModel, OpenAIChatModel
from agentscope.credential import DashScopeCredential, OpenAICredential

def build_model(provider: str = "dashscope"):
    if provider == "dashscope":
        return DashScopeChatModel(
            credential=DashScopeCredential(api_key=os.environ["DASHSCOPE_API_KEY"]),
            model="qwen-plus",
        )
    return OpenAIChatModel(
        credential=OpenAICredential(api_key=os.environ["OPENAI_API_KEY"]),
        model="gpt-4o-mini",
    )

# Agent 构造不变，仅 model= 参数不同
agent = Agent(name="S3", model=build_model("dashscope"), toolkit=toolkit)
先想 10 秒：换模型后为什么要检查 Formatter？

不同 Provider 的 tool_call 与 message 字段格式不同；DashScopeChatFormatter 与 OpenAI Formatter 需与 ChatModel 配对，否则 ReAct 解析 tool_call 会失败。

ProviderChatModelFormatter
DashScopeDashScopeChatModelDashScopeChatFormatter
OpenAIOpenAIChatModelOpenAIChatFormatter
Ollama 本地OllamaChatModel对应 Formatter
反例：模型写死
  • 全链路硬编码 qwen-max，账单失控
  • 换 Provider 时改 Agent / Toolkit / Memory 多处
  • Formatter 与 ChatModel 不配，tool_call 解析失败
推荐：模型无关
  • 仅替换 ChatModel + 匹配 Formatter
  • 路由表按意图选 turbo/plus/max
  • Credential 从环境变量注入，支持 fallback 链

DashScope/通义接入

DashScope 提供 qwen-max / qwen-plus / qwen-turbo 等模型。CorpAssist 默认 plus 平衡质量与成本；AgentScope 2.0 支持 streaming、retry 与 fallback 链。

模型适用CorpAssist 场景
qwen-max复杂推理、长报告S3 季度分析主 Agent
qwen-plus日常办公、工具调用默认办公助手
qwen-turbo分类、摘要、路由Memory 摘要、意图识别

DashScope 完整配置（含 Chat Formatter）

python 
from agentscope.model import DashScopeChatModel
from agentscope.credential import DashScopeCredential
from agentscope.formatter import DashScopeChatFormatter

model = DashScopeChatModel(
    credential=DashScopeCredential(api_key=os.environ["DASHSCOPE_API_KEY"]),
    model="qwen-plus",
    stream=True,
    max_retries=2,
)

agent = Agent(
    name="CorpAssist",
    model=model,
    formatter=DashScopeChatFormatter(),
    toolkit=toolkit,
    max_iters=10,
)
  • stream=True 是 AG-UI 必需，与 basics-02 事件流配合。
  • max_retries=2 应对 DashScope 偶发 5xx，与工具重试层分开计数。
  • Formatter 必须与 ChatModel Provider 配对，否则 tool_call 解析失败。
  • CorpAssist 生产环境 API Key 走 K8s Secret，与工具 MCP Token 同样管理。

多模型路由

多模型路由 按任务类型选择 ChatModel：意图分类走 turbo，主对话走 plus，规划走 max。与 意图路由 结合时，避免为每个意图单独堆叠分类 LLM 造成 NS3 成本级联。

单模型 vs 路由

python 
# 反模式：所有步骤用 qwen-max
agent = Agent(model=DashScopeChatModel(model="qwen-max"), ...)
python 
# 推荐：按角色分配
router = {
    "classify": DashScopeChatModel(model="qwen-turbo"),
    "execute": DashScopeChatModel(model="qwen-plus"),
    "plan": DashScopeChatModel(model="qwen-max"),
}
classifier = Agent(name="router", model=router["classify"], ...)
worker = Agent(name="worker", model=router["execute"], toolkit=toolkit)
CorpAssist 模型路由
flowchart TD
    Input[用户输入] --> C[qwen-turbo 意图分类]
    C -->|简单问答| P[qwen-plus Agent]
    C -->|复杂报告| M[qwen-max Planner]
    M --> W[qwen-plus Worker + 工具]
    P --> Out[回复]
    W --> Out

路由节点用最小模型，主链路按复杂度升级。

意图分类 Agent（turbo）

python 
CLASSIFY_PROMPT = "只输出 simple 或 complex，不要解释。"

classifier = Agent(
    name="intent-router",
    system_prompt=CLASSIFY_PROMPT,
    model=DashScopeChatModel(model="qwen-turbo"),
)

async def route(user_text: str) -> str:
    msg = await classifier.reply(UserMsg("user", user_text))
    label = msg.content.strip().lower()
    return "plan" if "complex" in label else "execute"

多模型路由 应记录每步 model 名称到 OTel attribute，便于 NS3 账单分摊。

CorpAssist · 意图路由 用户问「帮我订会议室」→ turbo 分类为 simple → plus Worker 执行；用户问「对比 Q1/Q2 销售数据并写 executive summary」→ turbo 判 complex → max Planner 分解子任务 → plus Worker 调工具——路由错误时 trace 中 model 字段是第一排查点。

与 NS3 成本权衡

北星 NS3 要求 Agent 成本可度量、可优化。NS3 成本 = Token 费用 + 工具调用次数 × 步数。AgentScope Event 与 usage 字段可汇总到 OTel，配合 max_iters、路由与 Memory 摘要降本。

杠杆做法NS3 影响
max_iters10～15 封顶限制 worst-case 步数
模型路由turbo 做分类/摘要Token 单价下降 60%+
Memory 裁剪见 basics-04输入 Token 线性下降
缓存相同 tool 结果短期缓存减少重复 LLM 调用

fallback 链如何配置？
DashScopeChatModel 可在 primary 失败时 fallback 到备用模型或 Provider；CorpAssist 建议 primary=plus，fallback=turbo 并告警，避免 silent 降级到不可用模型。

NS3 周报指标模板

指标采集点阈值示例
avg_tokens_per_taskREPLY_END usage< 8k
avg_itersAgent 内部计数< 6
max_model_tier路由 OTel90% plus 以下

NS3 评审时须同时展示质量 eval 与成本曲线，避免单指标优化。当 DashScope 主模型偶发 5xx 时，可配置 模型降级链：primary 失败自动切换备用模型并上报告警。

python 
from agentscope.model import DashScopeChatModel

primary = DashScopeChatModel(model="qwen-plus", max_retries=2)
fallback = DashScopeChatModel(model="qwen-turbo", max_retries=1)

async def chat_with_fallback(messages):
    try:
        return await primary.chat(messages=messages)
    except Exception as e:
        otel.record("model.fallback", {"from": "qwen-plus", "to": "qwen-turbo", "err": str(e)})
        return await fallback.chat(messages=messages)

本章结论

  • 模型无关编程：ChatModel 抽象隔离 Provider，Agent 逻辑可移植。
