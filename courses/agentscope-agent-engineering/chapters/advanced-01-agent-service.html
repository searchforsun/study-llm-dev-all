本章先记住 3 件事
  • Agent ≠ LLM Gateway：Agent Service 管推理循环与工具，网关只管模型 Token。
  • SSE 传 Event，不是传 Token：每条 SSE 对应 AgentScope 的 text/thinking/tool_call 事件。
  • 会话状态外置：多实例部署时会话存 Redis，不在进程内存。

REST+SSE 暴露

CorpAssist · S3 办公助手上线 产品要求：Spring dialog-orchestrator 调用 Python agent-service，用户问「帮我订明天下午会议室并通知参会人」——前端需要实时看到 Agent 思考与工具调用进度，最终返回结构化结果。

AgentScope 2.0 执行 ReActAgent 时会产生 Event 流：text（模型输出）、thinking（推理过程，若模型支持）、tool_call / tool_result（工具调用与返回）。服务化时需将内部事件映射为对外协议。

Agent Service 请求路径
sequenceDiagram

同步接口返回最终 Msg；流式接口逐条推送 Event，BFF 可转发给前端 AG-UI。

SSE 事件 payload 约定

json 
{
  "event_type": "tool_call",
  "session_id": "sess-8f3a",
  "step": 2,
  "payload": {
    "tool_name": "book_meeting_room",
    "arguments": {"room": "A301", "time": "2026-06-11T14:00"}
  },
  "trace_id": "tr-abc123"
}
不推荐
  • 把原始 LLM Token 流直接当 Agent SSE（丢失 tool_call 语义）
  • 同步与流式两套 Agent 实例，状态不一致
  • SSE 无 trace_id，排障无法关联
推荐
  • 统一 Agent 执行内核，仅输出层区分 sync/stream
  • 每条 SSE 带 step、trace_id、session_id
  • 流结束发送 event: done 含 usage 汇总

多租户/多会话基础

CorpAssist 企业部署中，HR 与 IT 部门数据隔离；同一用户可能并行打开「订会议室」与「查制度」两个会话。Agent Service 必须在请求层解析 tenant_id，在存储层实现 session 隔离。

维度请求头 / 字段存储键隔离策略
租户X-Tenant-Id{tenant}:config工具白名单、模型路由、Token 预算
用户X-User-Id（BFF 注入）{tenant}:{user}:sessions会话列表、并发上限
会话session_id body{tenant}:{session}:memoryMsg 历史、ReAct 步数计数
追踪X-Trace-Id日志 / OTLP跨 BFF ↔ Agent Service 关联
python 
# session_store.py — Redis 会话隔离示意
from dataclasses import dataclass
import json
import redis

@dataclass
class SessionKey:
    tenant_id: str
    session_id: str

    @property
    def redis_key(self) -> str:
        return f"corpassist:agent:{self.tenant_id}:{self.session_id}:msgs"

class SessionStore:
    def __init__(self, r: redis.Redis, ttl_seconds: int = 86400):
        self.r = r
        self.ttl = ttl_seconds

    def load_messages(self, key: SessionKey) -> list[dict]:
        raw = self.r.get(key.redis_key)
        return json.loads(raw) if raw else []

    def append_message(self, key: SessionKey, msg: dict) -> None:
        msgs = self.load_messages(key)
        msgs.append(msg)
        self.r.setex(key.redis_key, self.ttl, json.dumps(msgs, ensure_ascii=False))

    def purge(self, key: SessionKey) -> None:
        self.r.delete(key.redis_key)

多租户配置示例：IT 租户启用 restart_server 工具，HR 租户禁用；通过 tenant 级 YAML 热加载，避免硬编码在 Agent 代码中。

FastAPI 封装

AgentScope 2.0 推荐在 FastAPI 中托管 Agent 生命周期：启动时注册工具与模型，请求时从 SessionStore 恢复历史、执行 ReAct、持久化结果。

python 
# main.py — AgentScope Agent Service 最小封装
from fastapi import FastAPI, Header, HTTPException
from fastapi.responses import StreamingResponse
from pydantic import BaseModel
import agentscope
from agentscope.agents import ReActAgent
from agentscope.message import Msg
import json
import asyncio

app = FastAPI(title="CorpAssist Agent Service")

class ChatRequest(BaseModel):
    session_id: str
    message: str
    max_iters: int = 8

@app.on_event("startup")
async def init_agentscope():
    agentscope.init(model_configs=[{
        "config_name": "qwen",
        "model_type": "dashscope_chat",
        "model_name": "qwen-max",
    }])

def build_agent(tenant_id: str) -> ReActAgent:
    # 按 tenant 加载工具白名单（示意）
    tools = load_tools_for_tenant(tenant_id)
    return ReActAgent(
        name="corp_assist_s3",
        model_config_name="qwen",
        sys_prompt=load_sys_prompt(tenant_id),
        tools=tools,
        max_iters=8,
    )

@app.post("/v1/chat/stream")
async def chat_stream(
    req: ChatRequest,
    x_tenant_id: str = Header(..., alias="X-Tenant-Id"),
    x_trace_id: str | None = Header(None, alias="X-Trace-Id"),
):
    agent = build_agent(x_tenant_id)
    history = session_store.load_messages(SessionKey(x_tenant_id, req.session_id))
    user_msg = Msg(name="user", content=req.message, role="user")

    async def event_generator():
        step = 0
        async for event in agent.run_stream([*history, user_msg]):
            step += 1
            payload = {
                "event_type": event.type,
                "session_id": req.session_id,
                "step": step,
                "trace_id": x_trace_id,
                "payload": event.to_dict(),
            }
            yield f"data: {json.dumps(payload, ensure_ascii=False)}\n\n"
        yield "event: done\ndata: {}\n\n"

    return StreamingResponse(event_generator(), media_type="text/event-stream")
FastAPI 封装检查清单
  • 启动探针：模型连通 + Redis 连通
  • 请求超时：全局 max_iters + 单工具 timeout
  • 优雅关闭：drain 进行中 SSE 连接
  • 结构化日志：tenant_id、session_id、trace_id 必打

Spring 侧消费契约

Java 轨 dialog-orchestrator 通过 WebClient 消费 Agent Service，需对齐 HTTP 头、SSE 解析与错误码。这与 Spring AI 工程化开发spring-ai-engineering 中的 BFF 模式一致：Spring 管 IAM 与审计，Python 管 Agent 推理。

java 
@Service
public class AgentServiceClient {
    private final WebClient webClient;

    public Flux<AgentEvent> streamChat(String tenantId, String userId,
            String sessionId, String message, String traceId) {
        return webClient.post()
            .uri("/v1/chat/stream")
            .header("X-Tenant-Id", tenantId)
            .header("X-User-Id", userId)
            .header("X-Trace-Id", traceId)
            .bodyValue(Map.of("session_id", sessionId, "message", message))
            .retrieve()
            .bodyToFlux(String.class)
            .filter(line -> line.startsWith("data:"))
            .map(line -> parseAgentEvent(line.substring(5).trim()));
    }
}

常见问题（FAQ）

Agent Service 和 llm-gateway 应该合并还是分开？
分开。 Gateway 负责模型路由、Token 计量与密钥；Agent Service 负责 ReAct 循环、工具编排与会话。合并会导致 Gateway 承担有状态逻辑，难以水平扩展。
SSE 连接断开时 Agent 还在跑怎么办？
客户端断开应触发 cancellation token，AgentScope 侧停止后续 tool_call。若业务要求任务继续（如后台订会议室），改用异步 Job 模式：POST 返回 task_id，轮询或 Webhook 通知结果。
多实例部署时 session 粘滞吗？
不需要。 会话状态存 Redis，任意 Pod 可处理同一 session_id。仅需保证同一 session 并发请求串行化（分布式锁或队列）。

本章结论

  • 双接口：同步 /v1/chat 供评测；SSE /v1/chat/stream 供 UI。
  • Event 映射：SSE payload 对齐 AgentScope text/thinking/tool_call。
  • 多租户：tenant 级工具白名单 + Redis session 外置。
  • FastAPI：统一 Agent 内核，StreamingResponse 转发事件流。
  • Spring 契约：WebClient + 错误码映射 + trace_id 透传。

判断练习

  1. 产品要求「前端直接连 Agent Service SSE，跳过 Spring BFF」——生产环境是否可行？

    参考答案

    不可行。 BFF 负责 IAM 注入 X-User-Id、审计日志与速率限制；前端直连会暴露 tenant 配置、绕过企业 SSO，且无法统一错误码与合规擦除。

  2. 下列 SSE payload 哪条违反本章约定？A) 含 trace_id；B) event_type 为 raw_token；C) 含 step 序号。

    参考答案

    B。 Agent Service 应推送 AgentScope 语义事件（text/thinking/tool_call），不应把 LLM 原始 Token 流当 Agent 事件，否则 BFF 无法展示工具进度。
