AgentScope 以消息为中心；LangGraph 以状态图为中心；CrewAI 以角色任务为中心。

python 
# 同一任务「查制度并总结」三框架写法对照（示意）

# --- AgentScope ---
from agentscope.agents import ReActAgent
from agentscope.message import Msg
agent = ReActAgent(name="assistant", tools=[search_policy, summarize])
result = agent(Msg("user", "查年假制度并三句话总结", "user"))

# --- LangGraph ---
from langgraph.prebuilt import create_react_agent
graph = create_react_agent(model, tools=[search_policy, summarize])
result = graph.invoke({"messages": [("user", "查年假制度并三句话总结")]})

# --- CrewAI ---
from crewai import Agent, Task, Crew
researcher = Agent(role="政策研究员", tools=[search_policy])
writer = Agent(role="摘要员", tools=[summarize])
crew = Crew(agents=[researcher, writer], tasks=[...])
result = crew.kickoff(inputs={"query": "年假制度"})

先想 10 秒：CorpAssist S2 客服已有 LangGraph ReAct，S3 多 Agent 为何仍值得评估 AgentScope？

S2 单链路 ReAct 用 LangGraph 足够；S3 需要 Message Hub 显式 Msg 路由、Realtime Steering HITL 与 DashScope 原生 Event 流——AgentScope AOP 在这些能力上更贴近需求，而非 StateGraph 强行扩展。

三框架的评测与 CI 集成有何差异？
LangGraph 可接 LangSmith Dataset；CrewAI 评测生态弱；AgentScope 2.0 推荐 pytest + JSON golden set 离线门禁。CorpAssist 统一要求 golden pass rate ≥90%，框架差异在采集方式，不在门禁标准。

与 Dify/Coze 低代码对照

Dify / Coze 等低代码平台通过可视化编排 Agent、知识库与工具，适合运营参与配置。与 AgentScope 代码框架是互补关系，不是替代关系。

能力Dify / CozeAgentScope 代码
Prompt / 工作流配置可视化拖拽，运营可改代码 + Git 版本管理
自定义工具HTTP 插件 / 代码节点Python 函数 + MCP 任意复杂度
合规与审计平台依赖，定制难完全自控日志与数据路径
CI/CD 与评测导出 DSL，门禁弱pytest + golden set 原生集成
CorpAssist 双栈通常单栈 SaaSPython Agent + Spring BFF 标准模式

推荐组合：PoC 用 Dify 验证 Prompt → 生产用 AgentScope 重写并接入 Spring 治理；或 Dify 管知识库、AgentScope 管复杂工具链。

CorpAssist · 运营 Prompt 迭代 运营在 Dify 上调整「年假政策问答」Prompt 并 A/B 测点击率；研发 weekly 从 Dify 导出 Prompt 文本，Git 审查后写入 AgentScope sys_prompt 并跑 golden set——低代码管实验，代码管发布与合规。
低代码误用
  • 生产直接跑 Dify workflow 接敏感 MCP
  • Prompt 变更无 Git 审计
  • 低代码与代码框架各维护一套工具 schema
互补模式
  • Dify 仅 PoC / Prompt 实验室
  • AgentScope 管工具链 + HITL + 评测
  • 统一 HTTP 工具契约跨平台复用

常见问题（FAQ）

CorpAssist 应该统一一个框架还是多框架并存？
工具层统一，编排层可并存。 查单、检索等工具只写一份；S2 可继续 LangGraph，S3 用 AgentScope Message Hub。BFF 通过统一 Agent Service 契约屏蔽差异。
Dify 导出的 workflow 能直接跑在 AgentScope 吗？
不能一键导入。 Dify DSL 与 AgentScope 抽象不同。可复用 Prompt 文本与 HTTP 工具 URL，编排逻辑需重写。建议 Dify 仅作 Prompt 实验室。
CrewAI 适合 CorpAssist 生产吗？
不推荐作 S3 生产主框架。 CrewAI 擅长角色扮演原型，但 HITL、步数预算、双栈对照与评测门禁需大量自研。可用于 2 周 PoC，生产迁移到 AgentScope 或 LangGraph。

本章结论

  • AgentScope vs LangGraph vs CrewAI：S3 多 Agent 优先 AgentScope；复杂图与 checkpoint 保留 LangGraph；CrewAI 适原型。
  • 与 Dify/Coze 低代码对照：低代码做 PoC，生产 SLA 靠代码框架。
  • 选型决策矩阵：五维度加权评分，S3 场景 AgentScope 应最高。
  • 迁移成本评估：重点估算 State、checkpoint、Trace 三块人日。
