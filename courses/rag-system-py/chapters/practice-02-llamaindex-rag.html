为什么要学本章

LlamaIndex 是 LangChain 之外最成熟的 RAG 框架，其核心哲学是「索引优先」——通过 IngestionPipeline 精细化控制文档处理流程，使用 Query Engine 封装检索与生成逻辑。本章从 CorpAssist 场景出发，用 LlamaIndex 实现完整的 RAG 管线，并与上一章的 LangChain 实现进行直观对比。

学完你能
  • 用 IngestionPipeline 构建文档预处理流程：解析、切分、嵌入、索引
  • 配置 LlamaIndex Query Engine：retriever + response synthesizer 组合
  • 使用 Router 实现多策略查询路由
  • 客观分析 LangChain 与 LlamaIndex 各自优劣，按场景选型
本章先记住 3 件事
  • LlamaIndex = 索引为中心：所有能力围绕 Index 展开，检索和生成是 Index 的方法。
  • IngestionPipeline 是 ETL 利器：transform 链式组合，缓存中间结果避免重复计算。
  • Router 赋予多策略能力：同一个问题可路由到不同 Index 或 Query Engine。

索引构建：IngestionPipeline 与文档处理

LlamaIndex 把文档到索引的整个过程称为 Ingestion，核心组件是 IngestionPipeline。它像一个 ETL 管道：读取原始文档 → 逐节点解析 → 切分成 Node → 提取 embedding → 写入索引。

CorpAssist · 知识库索引构建 CorpAssist 需将每周更新的员工手册、IT 运维文档和产品说明书自动构建索引。这些文档格式包括 PDF、Markdown 和 Confluence 导出 HTML。LlamaIndex 的 IngestionPipeline 可以统一处理多格式数据源并保持索引一致性。

IngestionPipeline 处理文档 → 索引；Query Engine 完成检索 → 生成。

IngestionPipeline 实战

python 
# ingestion.py — LlamaIndex 文档处理与索引构建
from llama_index.core import (
    SimpleDirectoryReader,
    VectorStoreIndex,
    Settings,
    Document,
)
from llama_index.core.node_parser import SentenceSplitter
from llama_index.core.ingestion import IngestionPipeline
from llama_index.core.extractors import TitleExtractor
from llama_index.embeddings.openai import OpenAIEmbedding
from llama_index.vector_stores.chroma import ChromaVectorStore
from llama_index.core.schema import MetadataMode
import chromadb
from typing import List, Optional

# 全局配置
Settings.embed_model = OpenAIEmbedding(
    model="text-embedding-3-small",
    embed_batch_size=16,
    dimensions=512,
)

def build_ingestion_pipeline(
    chroma_collection_name: str = "corpassist_llama",
    persist_dir: str = "./data/chroma_llama",
) -> IngestionPipeline:
    """构建 IngestionPipeline：读取 → 切分 → 嵌入 → 存储"""
    # Chroma 客户端
    db = chromadb.PersistentClient(path=persist_dir)
    collection = db.get_or_create_collection(
        name=chroma_collection_name,
        metadata={"hnsw:space": "cosine"},
    )
    vector_store = ChromaVectorStore(chroma_collection=collection)

    pipeline = IngestionPipeline(
        transformations=[
            SentenceSplitter(
                chunk_size=1024,
                chunk_overlap=200,
                separator=" ",
                paragraph_separator="\n\n",
                secondary_chunking_regex="[^。！？]+[。！？]?",  # 中文分句
            ),
            TitleExtractor(),
            OpenAIEmbedding(
                model="text-embedding-3-small",
                dimensions=512,
            ),
        ],
        vector_store=vector_store,
    )
    return pipeline

def run_ingestion(doc_dir: str = "./data/docs") -> VectorStoreIndex:
    """执行文档摄取"""
    documents = SimpleDirectoryReader(
        input_dir=doc_dir,
        required_exts=[".pdf", ".md", ".html"],
        recursive=True,
    ).load_data()
    print(f"Loaded {len(documents)} documents")

    pipeline = build_ingestion_pipeline()
    nodes = pipeline.run(documents=documents)
    print(f"Processed into {len(nodes)} nodes")

    # 从 vector store 创建索引
    index = VectorStoreIndex.from_vector_store(
        pipeline.vector_store,
        embed_model=Settings.embed_model,
    )
    return index

if __name__ == "__main__":
    index = run_ingestion("./data/docs")
    print(f"Index ready with {len(index.ref_doc_info)} documents")

from llama_index.core import VectorStoreIndex, StorageContext, load_index_from_storage

def load_or_create_index(
    collection_name: str = "corpassist_llama",
    persist_dir: str = "./data/chroma_llama",
    doc_dir: str = "./data/docs",
) -> VectorStoreIndex:
    """加载已有索引；不存在则创建"""
    db = chromadb.PersistentClient(path=persist_dir)
    collection = db.get_or_create_collection(collection_name)

    if collection.count() > 0:
        print(f"Loading existing index ({collection.count()} vectors)")
        vector_store = ChromaVectorStore(chroma_collection=collection)
        return VectorStoreIndex.from_vector_store(vector_store)

    print("Creating new index...")
    from llama_index.core import SimpleDirectoryReader
    documents = SimpleDirectoryReader(
        input_dir=doc_dir,
        recursive=True,
    ).load_data()
    from ingest import build_ingestion_pipeline
    pipeline = build_ingestion_pipeline(collection_name, persist_dir)
    nodes = pipeline.run(documents=documents)
    return VectorStoreIndex.from_vector_store(pipeline.vector_store)

def get_index_stats(index: VectorStoreIndex) -> dict:
    """获取索引统计信息"""
    stats = {}
    stats["num_docs"] = len(index.ref_doc_info) if index.ref_doc_info else 0
    if hasattr(index, "_vector_store") and hasattr(index._vector_store, "_collection"):
        stats["num_vectors"] = index._vector_store._collection.count()
    stats["embed_model"] = str(index.embed_model)
    return stats

if __name__ == "__main__":
    index = load_or_create_index()
    stats = get_index_stats(index)
    for k, v in stats.items():
        print(f"{k}: {v}")
不推荐
  • 每次重建索引，无缓存机制
  • 全局默认切分器，忽略文档格式差异
  • 同一 embedding 模型不设 dimensions 参数
  • 不保留原始节点元数据
推荐
  • IngestionPipeline 自动缓存已处理的文档
  • 使用专为中英文设计的 SentenceSplitter
  • 显式设置 dimensions=512 降维提效
  • 保留 source/date/author 元数据

Query Engine：检索与生成的组合

LlamaIndex 的 Query Engine 是检索与生成的组合层。它将 retriever、response synthesizer 和 postprocessor 组合为一个可调用的接口。与 LangChain 不同，LlamaIndex 将「检索什么」和「如何生成」分离为独立配置的 RetrieverMode 和 ResponseMode。

组件LangChain 对应LlamaIndex 对应核心差异
检索器 BaseRetriever BaseRetriever / RetrieverMode LlamaIndex 检索器与 Index 强绑定
响应合成 StuffDocumentsChain ResponseSynthesizer + ResponseMode LlamaIndex 5 种 mode：compact/refine/tree/accumulate/compact_accumulate
后处理 cust om callback BaseNodePostprocessor 内置 SimilarityPostprocessor, KeywordPostprocessor

Query Engine 完整实现

python 
# query_engine.py — 多种 Query Engine 配置
from llama_index.core import VectorStoreIndex, Settings
from llama_index.core.query_engine import RetrieverQueryEngine
from llama_index.core.response_synthesizers import ResponseMode
from llama_index.core.postprocessor import (
    SimilarityPostprocessor,
    MetadataReplacementPostProcessor,
)
from llama_index.core.retrievers import VectorIndexRetriever
from llama_index.core.indices.query.query_transform.base import (
    HyDEQueryTransform,
)
from llama_index.core.query_engine import TransformQueryEngine
from llama_index.llms.openai import OpenAI
from llama_index.embeddings.openai import OpenAIEmbedding

Settings.llm = OpenAI(model="gpt-4o-mini", temperature=0.1)
Settings.embed_model = OpenAIEmbedding(model="text-embedding-3-small", dimensions=512)

def create_basic_query_engine(index: VectorStoreIndex) -> RetrieverQueryEngine:
    """基础 Query Engine：similarity 检索 + compact 合成"""
    retriever = VectorIndexRetriever(
        index=index,
        similarity_top_k=5,
    )
    return RetrieverQueryEngine(
        retriever=retriever,
        response_synthesizer=index.as_response_synthesizer(
            response_mode=ResponseMode.COMPACT,
            verbose=True,
        ),
        node_postprocessors=[
            SimilarityPostprocessor(similarity_cutoff=0.7),
        ],
    )

def create_advanced_query_engine(index: VectorStoreIndex) -> RetrieverQueryEngine:
    """进阶 Query Engine：HyDE + tree_summarize"""
    # 基础检索
    base_retriever = VectorIndexRetriever(
        index=index,
        similarity_top_k=10,
    )
    base_engine = RetrieverQueryEngine(
        retriever=base_retriever,
        response_synthesizer=index.as_response_synthesizer(
            response_mode=ResponseMode.TREE_SUMMARIZE,
        ),
        node_postprocessors=[
            SimilarityPostprocessor(similarity_cutoff=0.6),
        ],
    )
    # 使用 HyDE (Hypothetical Document Embeddings) 改进查询
    hyde = HyDEQueryTransform(llm=Settings.llm, include_original=True)
    engine = TransformQueryEngine(base_engine, hyde)
    return engine

def query_corpassist(index: VectorStoreIndex, question: str) -> str:
    """使用 advanced engine 查询"""
    engine = create_advanced_query_engine(index)
    response = engine.query(question)
    return str(response)

if __name__ == "__main__":
    from index_utils import load_or_create_index
    index = load_or_create_index()
    answer = query_corpassist(index, "员工休假政策有哪些更新？")
    print(f"Answer:\n{answer}")
    print(f"\nSource nodes: {len(answer.source_nodes)}")

5 种 ResponseMode 对比

Mode说明Token 效率适合场景
COMPACT将 top-k node 拼接后一次 LLM 调用高默认首选
REFINE逐 node 迭代精炼答案中需要逐段校对
TREE_SUMMARIZE对 node 分组递归摘要后合成低大量分散片段
ACCUMULATE每个 node 独立回答再拼接低多维度问题
COMPACT_ACCUMULATE拼接后分段独立回答中平衡效率与覆盖

Router：多策略查询路由

当 CorpAssist 需要同时处理多种查询类型（如 FAQ 检索、IT 工单查询、政策文档摘要），Router 可以根据问题内容自动选择合适的 Index 或 Query Engine。这是 LlamaIndex 相比 LangChain 更成熟的特性。

python 
# router.py — LlamaIndex Router 实现多策略路由
from llama_index.core import VectorStoreIndex
from llama_index.core.tools import QueryEngineTool, ToolMetadata
from llama_index.core.query_engine import RouterQueryEngine
from llama_index.core.selectors import LLMSingleSelector, PydanticSingleSelector
from llama_index.llms.openai import OpenAI

def build_router_engine(
    hr_index: VectorStoreIndex,
    it_index: VectorStoreIndex,
    policy_index: VectorStoreIndex,
) -> RouterQueryEngine:
    """构建多策略路由引擎"""
    llm = OpenAI(model="gpt-4o-mini", temperature=0)

    # 每个子索引对应一个工具
    hr_tool = QueryEngineTool(
        query_engine=hr_index.as_query_engine(
            similarity_top_k=3,
            response_mode="compact",
        ),
        metadata=ToolMetadata(
            name="hr_knowledge",
            description="员工手册、休假政策、薪酬福利等人力资源相关问题。",
        ),
    )
    it_tool = QueryEngineTool(
        query_engine=it_index.as_query_engine(
            similarity_top_k=5,
            response_mode="tree_summarize",
        ),
        metadata=ToolMetadata(
            name="it_support",
            description="IT 运维、工单系统、网络故障、设备申请等技术支持问题。",
        ),
    )
    policy_tool = QueryEngineTool(
        query_engine=policy_index.as_query_engine(
            similarity_top_k=3,
            response_mode="compact",
        ),
        metadata=ToolMetadata(
            name="company_policy",
            description="公司制度、合规要求、安全规范、数据保护等政策文档。",
        ),
    )

    router = RouterQueryEngine(
        selector=PydanticSingleSelector.from_defaults(llm=llm),
        query_engine_tools=[hr_tool, it_tool, policy_tool],
        verbose=True,
    )
    return router

if __name__ == "__main__":
    # 模拟三个索引
    from llama_index.core import Document
    from llama_index.core import VectorStoreIndex

    hr_docs = [Document(text="年假政策：员工每年享有15天带薪年假。")]
    it_docs = [Document(text="IT 工单流程：通过 ServiceNow 提交。")]
    policy_docs = [Document(text="数据保护政策：禁止将客户数据上传至非授权平台。")]

    hr_index = VectorStoreIndex.from_documents(hr_docs)
    it_index = VectorStoreIndex.from_documents(it_docs)
    policy_index = VectorStoreIndex.from_documents(policy_docs)

    router = build_router_engine(hr_index, it_index, policy_index)

    # 测试路由
    for question in [
        "我今年还剩几天年假？",
        "怎么提交网络故障工单？",
        "客户数据能发给第三方吗？",
    ]:
        print(f"\nQ: {question}")
        response = router.query(question)
        print(f"Selected: {response.metadata.get('selector_result', 'unknown')}")
        print(f"A: {response}")
先想 10 秒：Router 的 LLM 选择器和关键词匹配选择器各有什么优劣？

LLM 选择器（如 PydanticSingleSelector）理解语义，但引入额外延迟和成本。关键词选择器更快但需要预定义规则。CorpAssist 混合场景建议：高频常见问题用关键词，复杂模糊问题用 LLM 选择。

本章结论

  • IngestionPipeline 是 ETL 核心：文档解析→切分→嵌入→索引的全流程自动化，支持缓存和增量更新。
  • Query Engine 封装检索与生成：5 种 ResponseMode 适应不同场景，HyDE 变换可提升检索质量。
  • Router 实现多策略路由：按问题类型自动路由到不同索引，LLM/Pydantic 选择器各有优劣。
  • LangChain vs LlamaIndex：索引管理 LlamaIndex 胜出，Agent/Tool 生态 LangChain 更丰富。
  • 两者可互补：生产环境常见组合 LlamaIndex 索引 + LangChain Agent 管线。

判断练习

  1. IngestionPipeline 处理 1000 份 PDF 耗时 15 分钟。第二天新增了 50 份文档。需要重新处理全部 1050 份吗？

    参考答案

    不需要。IngestionPipeline 支持缓存和增量：设置 IngestionCache 后只有新文档会被处理。需确保文档的 hash 或 mod_time 可唯一标识。增量模式只需处理新增 50 份文档，耗时约 1 分钟。

  2. Router 把 HR 问题错误路由到了 IT 索引。如何改进？

    参考答案

    1) 优化 ToolMetadata 的 description 字段，提供更明确的关键词和排除词；2) 切换 selector 为 LLMSingleSelector 提高语义理解；3) 在 Router 前加一级关键词预分类器（如「年假/考勤→HR」「故障→IT」），降低 LLM 路由错误率。
