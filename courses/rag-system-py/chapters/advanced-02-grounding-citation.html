为什么要学本章

RAG 系统最危险的失败模式不是「答错」，而是「看起来正确的编造」。Grounding（接地/归因）确保生成答案的每个关键声明都可追溯到原始文档。对于 CorpAssist 这样的企业级应用，合规审计要求每个回答都能提供出处。Citation（引用） 溯源是 RAG 从「好玩」到「可信」的必修课。

学完你能
  • 实现从文档 chunk → 答案段落的引用映射机制
  • 构建 groundedness 评分器，自动检测答案中的非归因声明
  • 设计多种 Citation 格式（Markdown 脚注、JSON 引用、行内标注）
  • 建立引用验证验收测试，防止幻觉引用通过 CI
本章先记住 3 件事
  • 每句话都要有来源：答案中的关键事实声明必须链回到具体的 document chunk。
  • Groundedness != Faithfulness：Faithfulness 是「不矛盾」，Groundedness 是「有出处」。
  • 引用也要防幻觉：LLM 可能编造引用，需要用验证器至少检查引用格式和 chunk 存在性。

引用溯源实现：从 Chunk 到答案

引用溯源的核心是建立「生成答案中的声明 ↔ 检索到的文档 chunk」之间的可追溯映射。实现分三步：标记引用位置、记录源 chunk、验证引用合法性。

CorpAssist · 合规审计场景 CorpAssist 的 HR 知识库回答「员工年假政策变更」时，需要标注每个政策条款来自哪个政策文档的哪个版本。合规部门要求：每次回答都能在 30 秒内定位到原始文档段落。

引用溯源管线实现

python 
# citation_pipeline.py — 引用溯源管线
from dataclasses import dataclass, field
from typing import List, Dict, Optional
import json
import re
from langchain_openai import ChatOpenAI, OpenAIEmbeddings
from langchain_chroma import Chroma
from langchain_core.prompts import ChatPromptTemplate
from langchain_core.output_parsers import StrOutputParser
from langchain_core.runnables import RunnableParallel, RunnablePassthrough

@dataclass
class CitationSource:
    """单个引用的源信息"""
    chunk_id: str
    document_name: str
    page_number: Optional[int] = None
    text_snippet: str = ""
    relevance_score: float = 0.0

@dataclass
class CitationResult:
    """带引用的答案"""
    answer: str
    citations: List[CitationSource] = field(default_factory=list)

CITATION_PROMPT = """你是 CorpAssist 企业知识库助手。

基于以下文档片段回答问题。在回答中，对每个关键事实用 [N] 标注来源编号。
N 对应文档片段列表中片段的序号（从 1 开始）。

文档片段：
{context}

要求：
1. 每个关键事实都要标注来源
2. 不要标注没有文档支撑的信息
3. 如果不确定，请说「需要查证」

问题：{question}
回答："""

class CitationRAGPipeline:
    """带引用溯源的 RAG 管线"""

    def __init__(
        self,
        persist_dir: str = "./data/chroma_db",
        embedding_model: str = "text-embedding-3-small",
        llm_model: str = "gpt-4o-mini",
    ):
        embeddings = OpenAIEmbeddings(model=embedding_model, dimensions=512)
        self.vector_store = Chroma(
            persist_directory=persist_dir,
            embedding_function=embeddings,
        )
        self.retriever = self.vector_store.as_retriever(
            search_type="similarity",
            search_kwargs={"k": 5},
        )
        self.llm = ChatOpenAI(model=llm_model, temperature=0.1)
        self.prompt = ChatPromptTemplate.from_template(CITATION_PROMPT)

    def build_citation_chain(self):
        """构建 LCEL 引用管线"""
        def format_with_index(docs):
            """给文档添加序号用于引用"""
            return "\n\n".join(
                f"[{i+1}] {d.page_content}"
                for i, d in enumerate(docs)
            )

        chain = (
            RunnableParallel(
                {
                    "context": self.retriever | format_with_index,
                    "question": RunnablePassthrough(),
                }
            )
            | self.prompt
            | self.llm
            | StrOutputParser()
        )
        return chain

    def ask_with_citations(self, question: str) -> CitationResult:
        """问答并返回带引用信息的结果"""
        # 1. 检索
        docs = self.retriever.invoke(question)

        # 2. 生成带引用的答案
        chain = self.build_citation_chain()
        answer = chain.invoke(question)

        # 3. 解析引用标记并关联源文档
        citations = []
        ref_pattern = r'\[(\d+)\]'
        for match in re.finditer(ref_pattern, answer):
            idx = int(match.group(1)) - 1
            if 0 <= idx < len(docs):
                doc = docs[idx]
                citations.append(CitationSource(
                    chunk_id=doc.metadata.get("chunk_id", f"chunk_{idx}"),
                    document_name=doc.metadata.get("source", "unknown"),
                    page_number=doc.metadata.get("page", None),
                    text_snippet=doc.page_content[:200],
                    relevance_score=getattr(doc, "score", 0.0),
                ))

        return CitationResult(answer=answer, citations=citations)

if __name__ == "__main__":
    pipeline = CitationRAGPipeline()
    result = pipeline.ask_with_citations("员工年假政策是什么？")
    print(f"Answer:\n{result.answer}\n")
    print("Citations:")
    for c in result.citations:
        print(f"  [{c.chunk_id}] {c.document_name} (p.{c.page_number})")
        print(f"      {c.text_snippet[:80]}...")
不推荐
  • 不标注来源，答案不可追溯
  • 引用只是装饰性的 [1][2]，实际无对应 chunk
  • 引用格式不统一，有时用脚注有时用括号
  • 不校验引用的真实性——LLM 自己编引用编号
推荐
  • 每个 [N] 精确对应检索到的 chunk
  • 引用附带 source/document/page 元数据
  • 统一的引用格式（如 [doc_id:chunk_index]）
  • 使用引用验证器检查每个引用是否有效

置信度标注与 Groundedness 评分

Groundedness（归因度） 衡量答案中的声明有多少可以追溯到源文档。与上一章讲到的 RAGAS Faithfulness 不同，Groundedness 更严格：Faithfulness 要求「不矛盾」，Groundedness 要求「有来源」。

概念定义测量方式
Faithfulness 答案不含有与上下文矛盾的声明 LLM 判断（声明 → 上下文校验）
Groundedness 答案关键声明可直接追溯到指定源 chunk LLM 判断（声明 → 具体 chunk 匹配）
Citation Accuracy 引用标注 [N] 确实指向正确的 chunk 精确匹配 + 人工校验

# groundedness.py — Groundedness 评分器
from typing import List, Dict, Tuple
from dataclasses import dataclass
import re
from langchain_openai import ChatOpenAI
from langchain_core.prompts import ChatPromptTemplate

@dataclass
class GroundednessScore:
    """归因评分结果"""
    overall_score: float  # 0.0 - 1.0
    total_claims: int
    grounded_claims: int
    ungrounded_claims: List[str]
    citation_errors: List[str]

CLAIM_EXTRACTION_PROMPT = """提取以下答案中的所有事实声明。
每个声明应该是独立的、可验证的陈述。
以 JSON 数组格式输出：[{"claim": "..."}]

答案：{answer}
"""

GROUNDEDNESS_CHECK_PROMPT = """判断以下声明是否可以被源文档片段支持。

声明：{claim}

源文档片段：
{context}

如果声明完全被源文档支持 -> SUPPORTED
如果声明部分被支持或有细微差异 -> PARTIALLY_SUPPORTED
如果声明与源文档矛盾或源文档无相关信息 -> NOT_SUPPORTED

只输出一个词：SUPPORTED / PARTIALLY_SUPPORTED / NOT_SUPPORTED"""

class GroundednessChecker:
    """归因度检查器"""

    def __init__(self, model: str = "gpt-4o-mini"):
        self.llm = ChatOpenAI(model=model, temperature=0)
        self.extraction_prompt = ChatPromptTemplate.from_template(CLAIM_EXTRACTION_PROMPT)
        self.check_prompt = ChatPromptTemplate.from_template(GROUNDEDNESS_CHECK_PROMPT)

    def extract_claims(self, answer: str) -> List[str]:
        """从答案中提取独立声明"""
        response = self.llm.invoke(
            self.extraction_prompt.format(answer=answer)
        )
        try:
            import json
            claims_data = json.loads(response.content)
            return [item["claim"] for item in claims_data]
        except (json.JSONDecodeError, KeyError):
            # Fallback: 按句子分割
            return [s.strip() for s in re.split(r'[。！？]', answer) if len(s.strip()) > 10]

    def check_claim(self, claim: str, context: str) -> str:
        """检查单个声明的归因状态"""
        response = self.llm.invoke(
            self.check_prompt.format(claim=claim, context=context)
        )
        return response.content.strip()

    def score(
        self, answer: str, context_chunks: List[str]
    ) -> GroundednessScore:
        """完整归因评分"""
        context = "\n\n".join(context_chunks)
        claims = self.extract_claims(answer)
        print(f"Extracted {len(claims)} claims from answer")

        grounded = []
        ungrounded = []
        citation_errors = []

        for claim in claims:
            status = self.check_claim(claim, context)
            if status == "SUPPORTED":
                grounded.append(claim)
            elif status == "PARTIALLY_SUPPORTED":
                grounded.append(claim)  # 部分支持也算通过
            else:
                ungrounded.append(claim)

        # 检查引用标注
        citation_refs = re.findall(r'\[(\d+)\]', answer)
        for ref in citation_refs:
            idx = int(ref)
            if idx < 1 or idx > len(context_chunks):
                citation_errors.append(f"引用 [{idx}] 超出范围 (1-{len(context_chunks)})")

        total = len(claims)
        grounded_count = len(grounded)
        overall = grounded_count / total if total > 0 else 1.0

        return GroundednessScore(
            overall_score=overall,
            total_claims=total,
            grounded_claims=grounded_count,
            ungrounded_claims=ungrounded,
            citation_errors=citation_errors,
        )

if __name__ == "__main__":
    checker = GroundednessChecker()
    answer = "正式员工年假为15天[1]。需提前3个工作日申请[1]。病假需提供医院证明[2]。"
    contexts = [
        "正式员工每年享有15天带薪年假。年假需提前3个工作日申请。",
        "病假需提供医院开具的诊断证明。",
    ]
    result = checker.score(answer, contexts)
    print(f"Groundedness Score: {result.overall_score:.2f}")
    print(f"Claims: {result.grounded_claims}/{result.total_claims} grounded")
    if result.citation_errors:
        print(f"Citation Errors: {result.citation_errors}")
    if result.ungrounded_claims:
        print(f"Ungrounded Claims: {result.ungrounded_claims}")

# confidence_tagging.py — 置信度标注与展示
from dataclasses import dataclass
from typing import List, Optional, Literal

ConfidenceLevel = Literal["high", "medium", "low", "unverifiable"]

@dataclass
class TaggedSegment:
    """带置信度标注的答案段落"""
    text: str
    confidence: ConfidenceLevel
    sources: List[str]  # 来源 chunk ID
    reason: str = ""  # 标注理由

class ConfidenceTagger:
    """答案段落置信度标注器"""

    def __init__(self, groundedness_checker=None):
        self.checker = groundedness_checker

    def tag_answer_segments(
        self, answer: str, source_chunks: List[str]
    ) -> List[TaggedSegment]:
        """将答案分割为段落并标注置信度"""
        segments = []
        # 按引用标记分割
        import re
        parts = re.split(r'(\[\d+\])', answer)
        current_text = ""

        for part in parts:
            if re.match(r'\[\d+\]', part):
                if current_text.strip():
                    confidence = self._infer_confidence(
                        current_text, source_chunks
                    )
                    segments.append(TaggedSegment(
                        text=current_text.strip(),
                        confidence=confidence,
                        sources=[part],
                        reason=self._confidence_reason(confidence),
                    ))
                current_text = ""
            else:
                current_text += part

        if current_text.strip():
            confidence = self._infer_confidence(current_text, source_chunks)
            segments.append(TaggedSegment(
                text=current_text.strip(),
                confidence=confidence,
                sources=[],
                reason=self._confidence_reason(confidence),
            ))

        return segments

    def _infer_confidence(
        self, text: str, chunks: List[str]
    ) -> ConfidenceLevel:
        """推断段落置信度"""
        if not chunks:
            return "unverifiable"

        # 核心规则：有引用且有匹配文本的为 high
        for chunk in chunks:
            if any(word in chunk for word in text.split("，")[0].split("为")[:3]):
                return "high"

        # 默认 medium
        return "medium"

    def _confidence_reason(self, level: ConfidenceLevel) -> str:
        reasons = {
            "high": "有源文档直接支持",
            "medium": "有源文档部分支持",
            "low": "源文档支持不足",
            "unverifiable": "无法找到对应源文档",
        }
        return reasons.get(level, "")

    def format_confident_answer(
        self, segments: List[TaggedSegment]
    ) -> str:
        """格式化为带置信度标记的展示文本"""
        output = []
        for seg in segments:
            badge = {
                "high": "🟢",
                "medium": "🟡",
                "low": "🟠",
                "unverifiable": "🔴",
            }.get(seg.confidence, "⚪")
            output.append(f"{badge} {seg.text} [{seg.confidence}]")
            output.append(f"  来源: {', '.join(seg.sources) if seg.sources else '无'}")
        return "\n".join(output)

if __name__ == "__main__":
    tagger = ConfidenceTagger()
    segments = tagger.tag_answer_segments(
        "正式员工年假为15天[1]。需提前3个工作日申请[1]。",
        ["正式员工每年享有15天带薪年假。年假需提前3个工作日申请。"],
    )
    print(tagger.format_confident_answer(segments))

# citation_validator.py — 引用验证器与验收测试
from typing import List, Dict, Optional, Tuple
from dataclasses import dataclass
import re
import json

@dataclass
class ValidationError:
    """引用验证错误"""
    error_type: str  # "missing_ref", "invalid_ref", "hallucinated_ref", "orphan_ref"
    description: str
    location: str = ""  # 错误位置

class CitationValidator:
    """引用格式与内容验证器"""

    def __init__(self, available_chunks: List[str]):
        self.available_chunks = available_chunks

    def validate_format(self, answer: str) -> List[ValidationError]:
        """检查引用格式是否正确"""
        errors = []

        # 1. 检查引用标记格式 [N]
        ref_pattern = re.findall(r'\[(\d+)\]', answer)
        for ref in ref_pattern:
            idx = int(ref)
            if idx < 1 or idx > len(self.available_chunks):
                errors.append(ValidationError(
                    error_type="invalid_ref",
                    description=f"引用 [{idx}] 超出有效范围 (1-{len(self.available_chunks)})",
                    location=ref,
                ))

        # 2. 检查重复引用
        if len(ref_pattern) != len(set(ref_pattern)):
            # 允许合理重复
            pass

        return errors

    def validate_content(self, answer: str) -> List[ValidationError]:
        """检查引用内容是否合理"""
        errors = []

        refs = re.finditer(r'\[(\d+)\]', answer)
        for match in refs:
            idx = int(match.group(1)) - 1
            chunk = self.available_chunks[idx]
            # 获取引用前的文本
            start = max(0, match.start() - 60)
            preceding_text = answer[start:match.start()]

            # 简单检查：引用前的文本是否能在 chunk 中找到
            key_words = [w for w in preceding_text.split() if len(w) > 2][-5:]
            found = any(word in chunk for word in key_words)
            if not found:
                # 不直接报错（可能 chunk 就是支持段落）
                pass

        return errors

    def detect_hallucinated_refs(
        self, answer: str, context_chunks: List[str]
    ) -> List[ValidationError]:
        """检测幻觉引用 — 引用标注了但内容不在上下文中"""
        errors = []
        refs = re.finditer(r'\[(\d+)\]', answer)
        for match in refs:
            idx = int(match.group(1))
            if idx > len(context_chunks):
                errors.append(ValidationError(
                    error_type="hallucinated_ref",
                    description=f"引用 [{idx}] 指向的 chunk 不存在",
                    location=match.group(),
                ))
        return errors

    def full_validation(
        self, answer: str, context_chunks: List[str]
    ) -> Tuple[bool, List[ValidationError]]:
        """完整引用验证"""
        all_errors = []
        all_errors.extend(self.validate_format(answer))
        all_errors.extend(self.validate_content(answer))
        all_errors.extend(self.detect_hallucinated_refs(answer, context_chunks))
        return len(all_errors) == 0, all_errors

def citation_acceptance_test():
    """引用验收测试"""
    validator = CitationValidator(
        available_chunks=[
            "正式员工年假政策",
            "病假需提供医院证明",
        ]
    )

    test_cases = [
        ("年假为15天[1]，病假需证明[2]", True, "正常引用"),
        ("年假为15天[1]，病假需证明[5]", False, "引用 [5] 超出范围"),
        ("年假为15天，无引用标注", True, "无引用不报错（只是质量建议）"),
        ("年假为15天[1][999]", False, "引用了不存在的 999"),
    ]

    for answer, expected_pass, desc in test_cases:
        passed, errors = validator.full_validation(answer, validator.available_chunks)
        status = "PASS" if passed else "FAIL"
        print(f"[{status}] {desc}: {answer}")
        if errors:
            for e in errors:
                print(f"       {e.error_type}: {e.description}")

if __name__ == "__main__":
    citation_acceptance_test()

  • 每个答案至少包含 1 个引用，最多 5 个

常见问题（FAQ）

Groundedness 评分和 RAGAS Faithfulness 有什么区别？
Faithfulness 检查「有没有矛盾」— 答案说 15 天年假，上下文也提到 15 天，即通过。Groundedness 检查「有无来源」— 答案说 15 天年假，需要标注这信息来自文档 chunk_001。更严格。实践中，建议先保 Faithfulness，再追求 Groundedness。
LLM 经常编造引用编号，怎么防御？
多层防御：1) 在 prompt 中明确要求「只使用提供的文档片段编号」；2) 后处理阶段用 CitationValidator 检查每个 [N] 是否在有效范围；3) 对无法验证的引用从答案中移除并记录告警；4) 设置 CI 门禁，禁止引用错误率 > 5% 的部署。
引用信息应该放在 API 返回的哪个字段？
建议独立于 answer 字段。示例：{"answer": "年假为15天[1]", "citations": [{"id": "1", "source": "hr_policy.pdf", "page": 3, "text": "..."}]}。这样前端可以灵活选择渲染方式（行内标注或脚注），同时后端确保引用数据可被验证。
Groundedness 评分每次都要调用 LLM，成本怎么控制？
不是每次调用都做全量评分。建议策略：1) 开发阶段全量评分；2) 生产环境抽样（如 10% 流量做评分监控）；3) CI 门禁时对评测集做全量评分；4) 出现用户反馈问题时对相关答案做补充评分。

本章结论

  • 引用溯源是可信 RAG 的基础：从文档 chunk → 答案段落建立可追溯的映射，满足合规要求。
  • Groundedness 比 Faithfulness 更严格：Faithfulness 要求不矛盾，Groundedness 要求有出处。
  • 引用验证防幻觉：LLM 会编造引用，必须用验证器检查引用编号和 chunk 存在性。
  • 置信度标注提升透明度：用不同颜色/标记显示每个段落的可验证程度。
  • CI 门禁保证质量：将引用验证和 Groundedness 评分纳入 CI，防止低质量回答上线。

操作步骤

  1. 在 RAG 管线中集成引用标记，让 LLM 生成 [1][2] 格式的答案。
  2. 用 CitationRAGPipeline 获取带引用信息的答案。
  3. 编写 GroundednessChecker 评分 5 个测试问答的归因度。
  4. 用 CitationValidator 检查引用合法性，尝试构造一个幻觉引用并验证它被检出。
  5. 在前端（或 API 返回）中展示带置信度标注的答案。

判断练习

  1. Groundedness Score=1.0，但用户发现答案中一个关键数字（年假 20 天）是错误的。Groundedness 系统为什么没发现？

    参考答案

    Groundedness 检查「有来源」而非「正确」。如果源文档本身就写着 20 天（过时版本），Groundedness 评分会是满分。Groundedness 不能替代数据质量。解决方案：结合数据版本校验 + Faithfulness 检查（看 LLM 是否准确转述了源文档）。

  2. API 返回中，citations 数组包含了 3 个引用，但 answer 文本里只出现了 2 个 [N]。这算问题吗？

    参考答案

    是的，这是「孤立引用」问题。citations 数组和 answer 中的 [N] 应一一对应。多余的 citation 可能是上一个 query 的残留，需要清理。在 CitationValidator 中增加 orphan_ref 检查：citations 数组长度应等于 answer 中 [N] 的数量。
