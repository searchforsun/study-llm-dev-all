为什么要学本章

RAG 系统上线后最大的风险是「你不知道它不知道」。RAGAS 提供了标准化评测框架：忠实度（Faithfulness）、答案相关性（Answer Relevancy）、上下文精度（Context Precision）、上下文召回（Context Recall）。没有评测，调优就是盲调。本章用 评测集 构建 + RAGAS 指标 + Ablation 实验 三步，为 CorpAssist 建立可复现的评测基线。

  • 使用 RAGAS 计算 Faithfulness、Answer Relevancy、Context Precision/Recall
  • 设计 ablation 实验，量化 ch unk_size / top_k / retriever 等参数的影响

本章先记住 3 件事
  • 评测集是评测的起点：质量比数量重要。50 个精心标注的 QA 对 ≥ 500 个自动生成的。
  • RAGAS 4 个核心指标：Faithfulness 防幻觉、Relevancy 保质量、Precision/Recall 评检索。
  • Ablation 实验驱动改进：不改代码，只改参数，看哪个参数变化最敏感。

评测集构建：质量优先的策略

评测集 是 RAG 评测的基石。一个标准评测样本包含三个字段：question（用户问题）、ground_truth（标准答案）、contexts（答案依赖的上下文文档 ID）。构建策略直接影响评测结果的可信度。

CorpAssist · 评测数据集需求 CorpAssist 知识库覆盖员工手册、IT 运维、产品文档三类内容。评测集需要覆盖每类的典型问题，同时包含「难例」— 易混淆、跨文档、多步骤的问题，用于发现检索和生成的边界。

# build_dataset.py — 评测数据集构建
from typing import List, Dict
from dataclasses import dataclass, field
from datasets import Dataset
import json
import random

@dataclass
class EvalSample:
    """评测样本结构"""
    question: str
    ground_truth: str
    contexts: List[str]  # 相关文档 ID 列表
    category: str = "general"  # hr/it/policy
    difficulty: str = "normal"  # easy/normal/hard

@dataclass
class EvalDataset:
    """评测数据集"""
    samples: List[EvalSample] = field(default_factory=list)

    def to_hf_dataset(self, docs_map: Dict[str, str]) -> Dataset:
        """转换为 HuggingFace Dataset 供 RAGAS 使用"""
        data = {
            "question": [],
            "ground_truth": [],
            "contexts": [],
            "category": [],
        }
        for sample in self.samples:
            data["question"].append(sample.question)
            data["ground_truth"].append(sample.ground_truth)
            data["contexts"].append([
                docs_map.get(doc_id, doc_id) for doc_id in sample.contexts
            ])
            data["category"].append(sample.category)
        return Dataset.from_dict(data)

    def save(self, path: str):
        """保存为 JSONL"""
        with open(path, "w", encoding="utf-8") as f:
            for sample in self.samples:
                f.write(json.dumps(sample.__dict__, ensure_ascii=False) + "\n")

    @classmethod
    def load(cls, path: str) -> "EvalDataset":
        """从 JSONL 加载"""
        samples = []
        with open(path, "r", encoding="utf-8") as f:
            for line in f:
                data = json.loads(line)
                samples.append(EvalSample(**data))
        return cls(samples=samples)

合成数据集：LLM 辅助生成

python 
# synthetic_data.py — LLM 辅助合成评测数据
from langchain_openai import ChatOpenAI
from langchain_core.prompts import ChatPromptTemplate
from langchain_chroma import Chroma
from typing import List, Dict
import json
import re

SYNTH_TEMPLATE = """你是一个 RAG 评测数据生成专家。
基于以下文档内容，生成一个问答对（question 和 ground_truth）。
要求：
1. 问题必须忠实于文档内容
2. 答案必须是文档中的完整事实，不要添加外部知识
3. 生成的问题难度应为 {difficulty}
4. 问题应该能被 RAG 系统合理回答

文档内容：
{document}

输出格式（JSON）：
{{"question": "...", "ground_truth": "..."}}"""

def generate_synthetic_samples(
    docs: List[str],
    n_per_doc: int = 3,
    difficulty: str = "normal",
    model: str = "gpt-4o-mini",
) -> List[Dict]:
    """LLM 生成合成评测样本"""
    llm = ChatOpenAI(model=model, temperature=0.7)
    prompt = ChatPromptTemplate.from_template(SYNTH_TEMPLATE)
    chain = prompt | llm

    samples = []
    for doc_text in docs:
        for _ in range(n_per_doc):
            try:
                response = chain.invoke({
                    "document": doc_text[:2000],
                    "difficulty": difficulty,
                })
                content = response.content.strip()
                # 提取 JSON
                json_match = re.search(r'\{.*\}', content, re.DOTALL)
                if json_match:
                    sample = json.loads(json_match.group())
                    samples.append(sample)
            except Exception as e:
                print(f"Error generating sample: {e}")
                continue

    return samples

if __name__ == "__main__":
    # 从向量库获取文档
    from langchain_openai import OpenAIEmbeddings
    store = Chroma(
        persist_directory="./data/chroma_db",
        embedding_function=OpenAIEmbeddings(model="text-embedding-3-small", dimensions=512),
    )
    all_docs = store.get()["documents"]
    print(f"Found {len(all_docs)} documents in vector store")

    samples = generate_synthetic_samples(all_docs[:10], n_per_doc=3)
    with open("./data/synthetic_eval.jsonl", "w") as f:
        for s in samples:
            f.write(json.dumps(s, ensure_ascii=False) + "\n")
    print(f"Generated {len(samples)} synthetic samples")
不推荐
  • 只用 LLM 生成，无人审核，答案可能不准确
  • 所有问题局限在一个文档，不覆盖跨文档场景
  • 评测集一次性构建，不随数据更新
  • 样本量不足 20 条，统计意义有限
推荐
  • 黄金标注 + 合成 = 混合策略，人工审核关键样本
  • 覆盖 easy/normal/hard 三级难度
  • 版本化管理评测集，每次索引更新后重新评估
  • 50+ 黄金样本起步，300+ 合成样本扩展

RAGAS 指标：Faithfulness, Relevancy, Precision, Recall

RAGAS 是 RAG 评测的标准框架，核心指标覆盖 RAG 系统的两个关键环节：检索质量和生成质量。

指标测量对象范围说明
Faithfulness 生成答案 ⭢ 上下文 [0, 1] 答案中的每个声明是否都有上下文支持
Answer Relevancy 生成答案 ⭢ 问题 [0, 1] 答案是否完整且直接回答了问题
Context Precision 检索上下文 ⭢ 问题 [0, 1] 检索到的上下文中相关片段的比例
Context Recall 检索上下文 ⭢ ground_truth [0, 1] 基准答案所需的信息是否都在上下文中

RAGAS 评测管线

python 
# ragas_eval.py — RAGAS 评测管线
from datasets import Dataset
from ragas import evaluate
from ragas.metrics import (
    faithfulness,
    answer_relevancy,
    context_precision,
    context_recall,
)
from ragas.llms import LangchainLLMWrapper
from ragas.embeddings import LangchainEmbeddingsWrapper
from langchain_openai import ChatOpenAI, OpenAIEmbeddings
from typing import List, Dict, Optional
import json

class RAGEvaluator:
    """RAGAS 评测器"""

    def __init__(
        self,
        llm_model: str = "gpt-4o-mini",
        embedding_model: str = "text-embedding-3-small",
    ):
        # 包装 RAGAS 需要的 LLM 和 Embedding
        self.evaluator_llm = LangchainLLMWrapper(
            ChatOpenAI(model=llm_model, temperature=0)
        )
        self.evaluator_emb = LangchainEmbeddingsWrapper(
            OpenAIEmbeddings(model=embedding_model)
        )
        # 配置指标
        self.metrics = [
            faithfulness,
            answer_relevancy,
            context_precision,
            context_recall,
        ]
        for metric in self.metrics:
            metric.__setattr__("llm", self.evaluator_llm)
            if hasattr(metric, "embeddings"):
                metric.__setattr__("embeddings", self.evaluator_emb)

    def evaluate(
        self,
        dataset: Dataset,
        column_map: Optional[Dict] = None,
    ) -> Dict[str, float]:
        """运行完整评测"""
        if column_map is None:
            column_map = {
                "question": "question",
                "ground_truth": "ground_truth",
                "contexts": "contexts",
            }
        result = evaluate(
            dataset=dataset,
            metrics=self.metrics,
            column_map=column_map,
        )
        return dict(result)

    def print_report(self, scores: Dict[str, float]):
        """打印评测报告"""
        print("=" * 50)
        print("RAG 评测报告")
        print("=" * 50)
        for metric, score in scores.items():
            status = "PASS" if score >= 0.7 else "FAIL"
            bar = "█" * int(score * 20) + "░" * (20 - int(score * 20))
            print(f"{metric:25s} {bar} {score:.4f} [{status}]")
        print(f"\n平均得分: {sum(scores.values())/len(scores):.4f}")

if __name__ == "__main__":
    # 加载评测数据集
    from build_dataset import build_corpassist_dataset

    # 构建模拟文档映射
    docs_map = {
        "hr_policy_2024_v2.pdf": "关于员工福利政策，正式员工每年享有15天带薪年假...",
        "hr_leave_policy.md": "年假和事假审批流程分开，需逐级审批...",
        "it_ticket_guide.pdf": "IT工单系统地址: https://servicenow.corpassist.com...",
    }
    ds = build_corpassist_dataset()
    hf_dataset = ds.to_hf_dataset(docs_map)
    print(f"评测数据集: {len(hf_dataset)} 样本")

    # 运行评测
    evaluator = RAGEvaluator()
    # 模拟回答（实际场景中应使用 RAG 管线生成的答案）
    def mock_answers(dataset: Dataset) -> Dataset:
        """模拟 RAG 管线输出"""

            for _ in range(len(data["question"]))
        ]
        return Dataset.from_dict(data)

    hf_dataset_with_answers = mock_answers(hf_dataset)

CI 集成：自动化评测流水线

python 
# ci_eval.py — CI 集成评测脚本
from ragas_eval import RAGEvaluator
from build_dataset import EvalDataset
from datasets import Dataset
import json
import sys
import os

MIN_SCORES = {
    "faithfulness": 0.8,
    "answer_relevancy": 0.7,
    "context_precision": 0.7,
    "context_recall": 0.7,
}

def run_ci_eval(
    dataset_path: str = "./data/eval_dataset.jsonl",
    docs_dir: str = "./data/docs",
    rag_endpoint: str = "http://localhost:8000/ask",
) -> bool:
    """CI 评测入口 - 返回 True 表示通过"""
    import requests

    # 1. 加载评测集
    eval_dataset = EvalDataset.load(dataset_path)
    docs_map = {f"doc_{i}": f"Document {i}" for i in range(10)}
    hf_dataset = eval_dataset.to_hf_dataset(docs_map)

    # 2. 用 RAG 管线生成回答
    answers = []
    contexts_list = []
    for sample in eval_dataset.samples:
        try:
            resp = requests.post(
                rag_endpoint,
                json={"question": sample.question},
                timeout=30,
            )
            data = resp.json()
            answers.append(data.get("answer", ""))
            contexts_list.append(data.get("contexts", []))
        except Exception as e:
            print(f"Error querying RAG: {e}")
            answers.append("")
            contexts_list.append([])

    eval_data = hf_dataset.to_dict()
    eval_data["answer"] = answers
    eval_dataset_with_answers = Dataset.from_dict(eval_data)

    # 3. 运行评测
    evaluator = RAGEvaluator()
    scores = evaluator.evaluate(eval_dataset_with_answers)
    evaluator.print_report(scores)

    # 4. 检查是否通过
    all_pass = True
    for metric, min_score in MIN_SCORES.items():
        if metric in scores and scores[metric] < min_score:
            print(f"FAIL: {metric} = {scores[metric]:.4f} < {min_score}")
            all_pass = False

    # 5. 保存报告
    report = {
        "scores": scores,
        "passed": all_pass,
        "thresholds": MIN_SCORES,
    }
    with open("./data/eval_report.json", "w") as f:
        json.dump(report, f, indent=2, ensure_ascii=False)

    return all_pass

if __name__ == "__main__":
    passed = run_ci_eval()
    sys.exit(0 if passed else 1)

Ablation 实验：参数影响量化

Ablation（消融实验） 通过逐一改变参数观察指标变化，找出对 RAG 质量影响最大的配置项。这是从「凭感觉调参」到「数据驱动调参」的关键手段。

消融实验框架

python 
# ablation.py — 消融实验框架
from itertools import product
from typing import List, Dict, Any
import json
import time
from dataclasses import dataclass, field

@dataclass
class AblationExperiment:
    """消融实验定义"""
    name: str
    chunk_sizes: List[int] = field(default_factory=lambda: [256, 512, 1024, 2048])
    top_ks: List[int] = field(default_factory=lambda: [3, 5, 10])
    retrievers: List[str] = field(default_factory=lambda: ["similarity", "mmr", "ensemble"])
    use_reranker: List[bool] = field(default_factory=lambda: [False, True])
    alphas: List[float] = field(default_factory=lambda: [0.3, 0.5, 0.7])

    def get_combinations(self) -> List[Dict[str, Any]]:
        """获取所有参数组合"""
        params = {
            "chunk_size": self.chunk_sizes,
            "top_k": self.top_ks,
            "retriever": self.retrievers,
            "use_reranker": self.use_reranker,
            "alpha": self.alphas,
        }
        keys = list(params.keys())
        values = list(params.values())
        combinations = []
        for combo in product(*values):
            combinations.append(dict(zip(keys, combo)))
        return combinations

def run_ablation_suite(
    experiment: AblationExperiment,
    eval_dataset_path: str = "./data/eval_dataset.jsonl",
) -> List[Dict]:
    """运行整个消融实验组合"""
    results = []
    combinations = experiment.get_combinations()
    total = len(combinations)
    print(f"Running {total} ablation combinations...")

    # 导入评测器（实际使用时注入配置）
    from ragas_eval import RAGEvaluator
    from build_dataset import EvalDataset

    eval_dataset = EvalDataset.load(eval_dataset_path)
    evaluator = RAGEvaluator()

    for idx, params in enumerate(combinations):
        start = time.perf_counter()
        print(f"\n[{idx+1}/{total}] Params: {params}")

        # 这里应调用实际 RAG 管线获取结果
        # 模拟评测分数
        import random
        scores = {
            "faithfulness": 0.8 + random.uniform(-0.1, 0.1),
            "answer_relevancy": 0.75 + random.uniform(-0.1, 0.1),
            "context_precision": 0.7 + random.uniform(-0.1, 0.1),
            "context_recall": 0.7 + random.uniform(-0.1, 0.1),
        }
        elapsed = time.perf_counter() - start

        result = {
            "params": params,
            "scores": scores,
            "avg_score": sum(scores.values()) / len(scores),
            "latency_s": round(elapsed, 2),
        }
        results.append(result)

        print(f"  avg_score={result['avg_score']:.4f}, latency={result['latency_s']:.2f}s")

    # 按平均分排序
    results.sort(key=lambda r: r["avg_score"], reverse=True)
    return results

def generate_ablation_report(results: List[Dict], output_path: str = "./data/ablation_report.md"):
    """生成消融实验报告"""
    with open(output_path, "w", encoding="utf-8") as f:
        f.write("# Ablation 实验报告\n\n")
        f.write(f"总组合数: {len(results)}\n\n")
        f.write("## Top-5 最佳配置\n\n")
        f.write("| 排名 | chunk_size | top_k | retriever | rerank | alpha | avg_score | latency |\n")
        f.write("|------|-----------|------|-----------|--------|-------|-----------|--------|\n")
        for i, r in enumerate(results[:5]):
            p = r["params"]
            f.write(f"| {i+1} | {p['chunk_size']} | {p['top_k']} | {p['retriever']} | ")
            f.write(f"{p['use_reranker']} | {p['alpha']} | {r['avg_score']:.4f} | {r['latency_s']:.2f}s |\n")

    print(f"Report generated: {output_path}")

if __name__ == "__main__":
    exp = AblationExperiment(
        chunk_sizes=[512, 1024],
        top_ks=[3, 5],
        retrievers=["similarity", "mmr"],
        use_reranker=[False],
        alphas=[0.5],
    )
    print(f"参数组合数: {len(exp.get_combinations())}")
    results = run_ablation_suite(exp)
    generate_ablation_report(results)

常见问题（FAQ）

RAGAS 评测需要调用 LLM，每次评测的成本有多高？
50 个样本的完整评测约需 5-10 美元（gpt-4o-mini）。如果做 ablation 实验（如 20 种参数组合），成本升至 100-200 美元。建议：1) 先用 10 个样本做快速验证；2) 小参数量扫描先用模拟数据；3) 关键对比用完整评测集。
Faithfulness 分数低通常是什么原因？
最常见原因是 LLM 在上下文中找不到答案时「发挥」了。解决方案：1) 优化 prompt 强制要求「如果上下文无信息，请说不知道」；2) 提高检索质量，确保上下文包含答案；3) 降低 temperature（如 0.01）。
Context Precision 和 Context Recall 的区别是什么？
Precision 衡量「检索到的内容有多少是相关的」— 越低说明噪声越多。Recall 衡量「所有相关内容有多少被检索到」— 越低说明遗漏越多。理想系统两者都高，但实际需要在它们之间做权衡。
Ablation 实验中参数组合爆炸怎么办？
全量组合可能很快变成数百种。策略：1) 先用单因素实验（只变一个参数，其他固定）找到敏感参数；2) 对敏感参数做多因素实验；3) 使用 Plackett-Burman 或正交实验设计减少组合数。

本章结论

  • 评测集是基础：黄金标注 + 合成数据，覆盖多类别、多难度，版本化管理。
  • RAGAS 四个核心指标：Faithfulness 防幻觉、Relevancy 保匹配、Precision/Recall 评检索质量。
  • Ablation 驱动调优：逐一改变参数，量化每个变更对指标的影响。
  • CI 门禁：自动化评测纳入 CI 流水线，每次部署前自动检查指标是否达标。
  • 指标解读比绝对值重要：低 Faithfulness 需要看生成侧，低 Recall 需要看检索侧。

动手练习

使用 RAGAS 评测你前几章构建的 RAG 管线（约 30 分钟）。

操作步骤

  1. 构建至少 20 个 QA 对的评测集，覆盖 HR/IT/政策三类。
  2. 用 ragas_eval.py 运行评测，记录 Faithfulness/Relevancy/Precision/Recall。
  3. 修改 chunk_size（256/512/1024）重新评测，观察指标变化。
  4. 运行 ablation.py 中的参数扫描（top_k 和 retriever 类型）。
  5. 生成 ablation 报告，选择最优参数组合。

判断练习

  1. Ablation 实验显示 chunk_size=2048 时 Faithfulness 最高。你会选择这个参数吗？

    参考答案

    不一定。chunk_size=2048 虽然 Faithfulness 高（LLM 看到更多上下文），但可能增加检索噪声，降低 Precision，并因为 token 过多增加成本和延迟。需要综合看：Avg Score + 延迟 + token 成本。如果 2048 只比 1024 高 0.01，但 token 成本翻倍，选择 1024 更合理。

  2. 评测报告显示 Context Recall=1.0 但 Context Precision=0.3。这是好还是坏？

    参考答案

    坏。Recall=1.0 说明所有相关信息都被检索到了，但 Precision=0.3 意味着 70% 的检索结果是噪声。这会稀释 LLM 的注意力（Lost in the Middle 问题）。需要提高 score_threshold 或增加相关性过滤。
