完整 Compose 文件

yaml 
# docker-compose.yml — CorpAssist S4 推理交付包
version: "3.8"
services:
  vllm:
    image: vllm/vllm-openai:latest
    command: >
      --model /models/Qwen2.5-Coder-7B-Instruct-AWQ
      --quantization awq
      --served-model-name corp-coder-7b
      --max-model-len 8192
      --gpu-memory-utilization 0.9
      --enable-prefix-caching
      --host 0.0.0.0 --port 8000
    volumes:
      - ./models:/models:ro
    ports:
      - "8000:8000"
    shm_size: "2gb"
    mem_limit: 32g
    cpus: 8
    deploy:
      resources:
        reservations:
          devices:
            - driver: nvidia
              count: 1
              capabilities: [gpu]
    healthcheck:
      test: ["CMD", "curl", "-f", "http://localhost:8000/v1/models"]
      interval: 30s
      timeout: 10s
      retries: 3
      start_period: 120s
    logging:
      driver: "json-file"
      options:
        max-size: "10m"
        max-file: "3"
    restart: unless-stopped
不推荐
  • 不设 restart policy，进程崩溃后不自动重启
  • 日志不限制大小，磁盘写满
  • healthcheck 无 start_period，启动过程中误杀容器
  • 权重目录写死绝对路径
推荐
  • restart: unless-stopped 保障自愈
  • json-file driver + max-size 避免磁盘写满
  • start_period ≥ 120s 覆盖权重加载时间
  • 权重用相对路径 volume 挂载

验证流水线

Compose verify 分三层：health check（进程存活）→ smoke test（API 契约）→ e2e infer（IDE → BFF → 推理）。

python 
# smoke_test.py — 启动验证脚本
import requests, sys, time

BASE = "http://localhost:8000"

def check_health():
    """Step 1: 健康检查"""
    try:
        r = requests.get(f"{BASE}/v1/models", timeout=10)
        assert r.status_code == 200
        models = r.json()["data"]
        assert any(m["id"] == "corp-coder-7b" for m in models)
        print("[PASS] 健康检查通过，模型 corp-coder-7b 可用")
        return True
    except Exception as e:
        print(f"[FAIL] 健康检查失败: {e}")
        return False

def check_chat():
    """Step 2: Chat completions"""
    try:
        r = requests.post(f"{BASE}/v1/chat/completions", json={
            "model": "corp-coder-7b",
            "messages": [{"role": "user", "content": "1+1="}],
            "max_tokens": 16,
        }, timeout=30)
        assert r.status_code == 200
        content = r.json()["choices"][0]["message"]["content"]
        assert len(content) > 0
        print(f"[PASS] Chat 完成，输出: {content.strip()}")
        return True
    except Exception as e:
        print(f"[FAIL] Chat 失败: {e}")
        return False

def check_stream():
    """Step 3: 流式补全"""
    try:
        r = requests.post(f"{BASE}/v1/chat/completions", json={
            "model": "corp-coder-7b",
            "messages": [{"role": "user", "content": "写一个 word"}],
            "max_tokens": 32,
            "stream": True,
        }, timeout=60, stream=True)
        assert r.status_code == 200
        tokens = []
        for line in r.iter_lines():
            if line:
                decoded = line.decode("utf-8")
                if decoded.startswith("data: ") and decoded != "data: [DONE]":
                    tokens.append(".")
        print(f"[PASS] 流式通过，收到 {len(tokens)} 个 token")
        return True
    except Exception as e:
        print(f"[FAIL] 流式失败: {e}")
        return False

if __name__ == "__main__":
    results = [
        ("Health", check_health()),
        ("Chat", check_chat()),
        ("Stream", check_stream()),
    ]
    for name, passed in results:
        print(f"{'PASS' if passed else 'FAIL'}: {name}")
    sys.exit(0 if all(p for _, p in results) else 1)
检查项命令 / 动作通过标准
vLLM healthy curl /v1/models 或 compose healthcheck HTTP 200，返回模型列表
Chat completions POST /v1/chat/completions 200，content 非空
流式补全 stream=true SSE 有 data 行、TTFT < SLA
Python client OpenAI SDK 调用 200，model 字段正确
Spring client ChatClient.call() 返回非空响应

Spring 端探活 Bean

java 
// Spring Boot 启动后验证推理服务可用
import jakarta.annotation.PostConstruct;
import org.slf4j.Logger;
import org.slf4j.LoggerFactory;
import org.springframework.ai.chat.client.ChatClient;
import org.springframework.stereotype.Component;

@Component
public class InferHealthChecker {

    private static final Logger log = LoggerFactory.getLogger(InferHealthChecker.class);
    private final ChatClient chatClient;

    public InferHealthChecker(ChatClient chatClient) {
        this.chatClient = chatClient;
    }

    @PostConstruct
    public void verifyInferService() {
        try {
            String response = chatClient.prompt()
                .user("返回 OK")
                .call()
                .content();
            log.info("推理服务探活成功: {}", response);
        } catch (Exception e) {
            log.warn("推理服务未就绪，应用将继续启动但推理不可用: {}", e.getMessage());
        }
    }
}
先想 10 秒：health 200 但 chat 500，优先查什么？

模型路径/量化参数错误、GPU OOM、model 名与 served-model-name 不一致；health 只说明进程在听端口，不说明模型已加载成功。

交付清单

delivery-checklist（生产上线门禁）
  • docker compose up -d 启动正常，日志无 CUDA error
  • vLLM 健康检查通过（/v1/models 返回正确模型 ID）
  • Chat completions 非流式调用通过
  • 流式 SSE 调用通过（至少 3 个 token）
  • Python `smoke_test.py` 全部 PASS
  • 双栈 client 指向同一 corp-coder-7b 且 base-url 正确
  • GPU 显存利用率健康（nvidia-smi 确认）
  • 日志已配置 max-size 防止磁盘写满
  • Fallback 云 API 配置正确（私有化异常时降级）
  • TTFT p95 < 500ms（SLA 目标）

常见问题（FAQ）

交付包中的权重太大无法传输怎么办？
AWQ 7B 约 5-6GB，建议使用对象存储（OSS/S3）分发，或使用 Git LFS。如果客户无网络，可以预装到移动硬盘交付。注意：交付后要校验 checksum 防止损坏。
客户环境没有 curl，怎么做 healthcheck？
可以用 Python 的 requests 做 healthcheck，或者使用 wget。Compose healthcheck 也支持 CMD-SHELL 方式：test: ["CMD-SHELL", "python -c \"import requests; requests.get('http://localhost:8000/v1/models')\""]。
双栈 smoke 通过但 IDE 补全仍超时，可能的排查点？
排查链路：IDE 插件 base_url → BFF 网关超时 → 公司代理 → 流式缓冲刷新。建议从 IDE 插件抓包开始，逐层排查到 vLLM 日志。常见问题是网关 read timeout 设得太短，匹配流式场景见 流式章节。
如何监控推理服务的健康状况？
vLLM 提供 /metrics 端点（Prometheus 格式），包含请求数、TTFT、KV cache 使用率等指标。配合 Grafana 仪表板可以全面监控。生产环境建议接入 PagerDuty/钉钉告警。
