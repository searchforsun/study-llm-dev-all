分层答法（2 分钟版）

  1. 契约：OpenAI /v1/chat/completions + FIM；双栈只改 base_url（Python、Spring）。
  2. 引擎：生产 vLLM + AWQ；引擎可比 TGI（advanced-01）。
  3. 发布：model alias + 网关灰度 + 指标回滚（practice-04）。
  4. 容量：并发粗算 + 显存 + GPU 数（advanced-03）。
  5. 混合：敏感代码走私有化，通用对话可 hybrid cloud fallback（混合 Serving）。

坑现象排查对应章节

追问 2: GPU 怎么估？
text 
答: 先估并发 (Little 定律)

再算显存 (权重+KV+余量)
最后 ceil(并/单卡) × 125%（advanced-03）

常见问题（FAQ）

面试中「S4 私有化」的典型场景是什么？
面试官会说「我司需要部署一个代码补全助手，代码不能出内网」。你的回答要覆盖：Serving 模式选择（私有化为主体）、模型量化（7B AWQ）、vLLM + 双栈、灰度发布、容量规划、fallback 策略。
如果面试官追问 vLLM 的源码细节？
坦诚说明「应用架构师层面」，你知道 PagedAttention 的原理（分页管理 KV cache）和 continuous batching 的机制，但算子优化和 kernel 开发不是你的专注领域。
面试中最容易被忽略的环节是什么？
灰度发布和回滚。很多人能画出 vLLM 部署架构，但会在灰度发布和故障回滚的细节上暴露经验不足。务必准备 model alias、网关灰度、rollback 触发条件等话术。
没有 GPU 资源，面试怎么回答私有化部署？
可以讨论 hybrid 方案：常规场景用云 API，敏感场景用私有化（假设有 GPU）。同时讨论 CPU 推理的局限性（llama.cpp GGUF）和边缘场景的适用性。体现架构选型思维比具体资源更重要。
