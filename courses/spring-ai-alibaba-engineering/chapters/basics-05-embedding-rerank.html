Embedding 模型

DashScope 百炼 Embedding API 与 Chat 共用 API Key，模型名通过 spring.ai.dashscope.embedding.options.model 配置。CorpAssist 制度库推荐 text-embedding-v3（维度 1024，支持中英文混合 FAQ）。

yaml
spring:
  ai:
    dashscope:
      api-key: ${AI_DASHSCOPE_API_KEY}
      embedding:
        options:
          model: ${CORPASSIST_EMBEDDING_MODEL:text-embedding-v3}
          dimensions: 1024
模型维度CorpAssist 场景
text-embedding-v31024（可配）制度 FAQ、纪要 chunk（默认）
text-embedding-v21536历史索引兼容；新库勿混用
多模态 embedding视模型带图表制度页；见 vision rag
CorpAssist · S3 · 纪要检索 Agent 收到「上周采购制度里对 Jira 字段的要求」：先对 query 做 Embedding，从 policy-v2026-03 集合检索 Top-20；若仅 embedding 召回含过期版本，靠 metadata effective_date 过滤。
反模式
  • 在线 API 上传 PDF 并同步 embedding（阻塞 Agent）
  • v2/v3 向量混存同一 collection
  • 不设 batch，逐 chunk 调 API 触发 429
推荐
  • 夜间 Job 批量 embedding + 版本号写入 metadata
  • 单 collection 单模型维度；迁移时新 collection
  • EmbeddingModel.embed(List<Document>) 批量 16–32
自检：Agent Memory 与制度库 Embedding 能否共用同一 VectorStore？

不推荐同一 index：Memory 含会话噪声且 TTL 短；制度库需 ACL 与版本。分 collection + tenantId metadata，见 memory budget saa。

DashScope Rerank

向量 Top-K 常含语义相近但无关片段。百炼 gte-rerank（或文档最新 Rerank 模型）对 query–passage 对重打分，Spring 侧可在 VectorStoreDocumentRetriever 后接自定义 DocumentPostProcessor 或检索服务 BFF 调用 Rerank HTTP API。

召回 + Rerank 两阶段
flowchart LR
  Q[用户 query] --> EMB[EmbeddingModel]
  EMB --> VS[VectorStore Top-20]
  VS --> RR[DashScope Rerank]
  RR --> TOP[Top-5 进 Prompt]
  TOP --> LLM[qwen-plus Agent]

S3 Agent 工具 searchPolicy 内部走两阶段；Chat 直连 RAG Advisor 同样适用。

java
// 示意：检索后 Rerank 截断（伪代码）
List<Document> candidates = vectorStore.similaritySearch(
    SearchRequest.query(query).withTopK(20));
List<Document> reranked = dashScopeRerankClient.rerank(query, candidates, 5);
return reranked;
阶段参数S3 建议
向量召回topK=20, threshold=0.65宁可多召回，交给 Rerank 筛
ReranktopN=5进 Agent context；附 doc_id 供 citation
缓存query hash TTL 1h只读制度查询可 semantic cache agent
跳过 Rerank
  • Top-1 向量分 0.72 但段落讲「离职流程」而非「采购审批」
  • Agent 基于错误制度创建 Jira，HITL 才发现
Rerank + 引用
  • 响应带 policy_doc_id 与生效日期
  • Trace span 记录 rerank 前后 score 分布
  • Badcase 入库驱动 Rerank 阈值调参

VectorStore 集成

Spring AI 的 VectorStore 接口与厂商无关；SAA 提供 DashScope EmbeddingModel Bean，存储层常用 PGVector 或 Redis Stack。与 Spring AI 工程化practice-01-vectorstore 相同模式：EmbeddingModel + PgVectorStore + QuestionAnswerAdvisor。

xml
<dependency>
  <groupId>org.springframework.ai</groupId>
  <artifactId>spring-ai-pgvector-store-spring-boot-starter</artifactId>
</dependency>
<dependency>
  <groupId>com.alibaba.cloud.ai</groupId>
  <artifactId>spring-ai-alibaba-starter-dashscope</artifactId>
</dependency>
yaml
spring:
  ai:
    vectorstore:
      pgvector:
        index-type: HNSW
        distance-type: COSINE_DISTANCE
        dimensions: 1024
        table-name: corpassist_policy_chunks

制度库索引

CorpAssist 制度库索引是 S3 Agent 的「只读知识底座」：metadata 必含 tenantId、policy_version、effective_date、acl_roles。索引任务用 tika parser 或 SAA extensions Reader 解析，chunk 策略与 Python 轨 rag-system 对齐（500/50）。

步骤执行栈产出
1 采集数据湖 / SharePoint 导出PDF + manifest.json
2 解析切块Python 或 extensionsDocument 列表 + metadata
3 EmbeddingDashScope batch API向量写入 PGVector
4 发布CI 打 index_versionJava Agent 读最新 version
json
{
  "doc_id": "HR-LEAVE-2026",
  "tenant_id": "corp-hq",
  "policy_version": "2026.03.1",
  "effective_date": "2026-03-01",
  "acl_roles": ["employee", "manager"],
  "chunk_index": 12
}
CorpAssist · 制度版本切换 新规生效日 00:00：索引 Job 写入 policy_version=2026.04；Java 检索 filter effective_date <= today。旧版保留 90 天供审计，Agent 默认不召回过期 chunk。
制度库与 S3 Agent 工具有何关系？
Agent 不直接读 PDF；通过 searchPolicy Tool 调 VectorStore（或 RAG BFF）。Tool 返回截断摘要 + doc_id，完整条文走 citation 链接，符合 tool result truncation。
index_version 如何在 Java 检索侧生效？
Agent Service 启动时读 ConfigMap CORPASSIST_INDEX_VERSION；SearchRequest filter 只查 index_version == current。Python 发布新索引后 bump 版本并滚动 Java Deployment，避免读到半写入向量。

面试快问快答

S3 Agent 为什么需要 Rerank，单靠 Embedding 不够吗？

答：制度与纪要 chunk 语义相近度高，向量 Top-1 常「像但对」。Rerank 对 query-passage 交叉编码，降低 Agent 引用错误制度的概率；配合 metadata 过滤与 citation 可审计。

百炼 Embedding 与 OpenAI embedding 在 Spring AI 里如何切换？

答：业务依赖 EmbeddingModel 接口，不依赖厂商；CorpAssist 通义栈用 SAA DashScope starter。切换需重建向量 index（维度可能不同），不能混存。

判断练习

  1. S3 Agent 收到 PDF 附件后，在 Tool 内同步解析+embedding+检索再调 Jira。有何问题？

    参考答案

    在线路径阻塞、重复 embedding 浪费配额、超时导致 Agent 偏航。应：附件走 extensions 解析→异步索引；在线只检索已发布 index_version。

  2. 为省 Rerank 费用，直接把向量 Top-1 塞进 Prompt。在采购审批场景 acceptable 吗？

    参考答案

    不可接受：合规场景需可解释引用；Top-1 误召回会导致错误审批依据。至少 Rerank + doc_id citation；高风控流程加 HITL。
