API Key 与安全配置

DashScope 使用 api key 进行身份认证。SAA 自动从 spring.ai.dashscope.api-key 读取。严禁硬编码在代码或 Git 仓库中。

环境变量注入方式

yaml 
spring:
  ai:
    dashscope:
      api-key: ${DASHSCOPE_API_KEY}
      chat:
        options:
          model: qwen-plus
      embedding:
        options:
          model: text-embedding-v3
bash 
# 推荐：写入 .env 或 CI/CD 机密
export DASHSCOPE_API_KEY=sk-xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
CorpAssist · 客服 Agent 密钥管理 CorpAssist S3 部署多环境：dev/staging/prod 使用不同 DashScope API Key。生产 Key 具备更高 QPS 配额和更低延迟 SLA。通过 Spring Profile 实现隔离：application-prod.yml 引用 ${DASHSCOPE_PROD_API_KEY}。
不推荐
  • API Key 硬编码在 application.yml 中提交到 Git
  • 开发/生产使用同一 API Key
  • API Key 泄漏后仅删除不轮转
推荐
  • 使用环境变量或阿里云 RAM Role 注入
  • 多环境多 Key 通过 Profile 隔离
  • 定期轮转 + 审计 Key 调用记录

RAM Role 集成（生产推荐）

对于部署在阿里云 ECS/ACK 的生产环境，推荐使用 RAM Role 代替 API Key：

java 
// 使用阿里云凭据管家（Credentials Provider Chain）

模型列表与选型策略

DashScope 提供丰富的 model list。选型需在 能力（智力）与 成本/延迟 间平衡。

模型适用场景智力等级价格（参考）
qwen-turbo 简单问答、分类、提取 低 ¥0.3/百万 token
qwen-plus 客服对话、文档摘要、工具调用 中 ¥1.2/百万 token
qwen-max 复杂推理、代码生成、Agent 编排 高 ¥8.0/百万 token
qwen-omni-turbo 多模态（图像+文本）理解 中 ¥3.0/百万 token
模型选型决策树
flowchart TD
    Task[任务类型] --> Simple{简单分类提取?}
    Simple -->|是| Turbo[qwen-turbo\n低成本]
    Simple -->|否| Chat{客服对话?}
    Chat -->|是| Plus[qwen-plus\n平衡选项]
    Chat -->|否| Complex{需要复杂推理?}
    Complex -->|是| Max[qwen-max\n高智力]
    Complex -->|否| Multi{涉及图片?}
    Multi -->|是| Omni[qwen-omni-turbo]
    Multi -->|否| Plus

根据任务复杂度和输入类型选择模型。CorpAssist 客服默认 qwen-plus，Agent 编排使用 qwen-max。

CorpAssist 推荐配置：主 Agent 使用 qwen-max，子任务（分类/提取）使用 qwen-turbo。通过 tongyi stack 统一管理多模型路由，详见 ReactAgent 基础。

Chat API 与流式调用

SAA 的 stream chat 基于 SSE（Server-Sent Events）实现流式输出。生产环境必须使用流式以降低用户感知延迟。

非流式调用

java 
@Service
public class CorpAssistChatService {

    private final ChatModel chatModel;

    public CorpAssistChatService(ChatModel chatModel) {
        this.chatModel = chatModel;
    }

    public String answer(String userMessage) {
        Prompt prompt = new Prompt(new UserMessage(userMessage));
        ChatResponse response = chatModel.call(prompt);
        return response.getResult().getOutput().getContent();
    }
}

流式调用（推荐）

java 
import reactor.core.publisher.Flux;

@Service
public class CorpAssistStreamService {

    private final StreamingChatModel streamingChatModel;

    public CorpAssistStreamService(StreamingChatModel streamingChatModel) {
        this.streamingChatModel = streamingChatModel;
    }

    public Flux streamAnswer(String userMessage) {
        Prompt prompt = new Prompt(new UserMessage(userMessage));
        return streamingChatModel.stream(prompt)
            .map(response -> response.getResult().getOutput().getContent())
            .filter(text -> text != null);
    }
}

Python 对照：DashScope SDK

python 
import dashscope
from dashscope import Generation

# 非流式
response = Generation.call(
    model='qwen-plus',
    prompt='你好，请介绍一下CorpAssist'
)

# 流式
for chunk in Generation.call(
    model='qwen-plus',
    prompt='你好',
    stream=True
):
    print(chunk.output.text, end='')
先想 10 秒：流式场景下如何计算首 Token 延迟（TTFT）？

TTFT = 从请求发出到收到第一个非空 output token 的时间。通常应在 500ms 以内。通过 Micrometer Timer 埋点记录 dashscope.llm.first_token，详见 观测与调试。

Token 计费与成本控制

DashScope 按 Token 计费，不同类型模型价格差异可达 20 倍。了解 billing 模型是控制成本的基础。

成本项说明估算公式
Input Token 用户消息 + System Prompt + 历史 (System + History + User) * 单价
Output Token 模型生成文本 Output 长度 * 单价（通常 Output 更贵）
缓存命中 同前缀 prompt 缓存 缓存命中仅收 10% 费用

CorpAssist 成本预算样例

java 
// 日成本估算器 - CorpAssist 客服场景
public class CostEstimator {
    // 假设日均 10 万次对话，平均每轮 500 input + 200 output tokens
    static final long DAILY_CONVERSATIONS = 100_000;
    static final int AVG_INPUT_TOKENS = 500;
    static final int AVG_OUTPUT_TOKENS = 200;
    static final double PLUS_INPUT_PRICE = 1.2e-6; // 每 token
    static final double PLUS_OUTPUT_PRICE = 4.0e-6;

    public static double estimateDailyCost() {
        double inputCost = DAILY_CONVERSATIONS * AVG_INPUT_TOKENS * PLUS_INPUT_PRICE;
        double outputCost = DAILY_CONVERSATIONS * AVG_OUTPUT_TOKENS * PLUS_OUTPUT_PRICE;
        return inputCost + outputCost; // 约 ¥140/日
    }
}
成本优化策略
  • 语义缓存：对高频 FAQ 启用 semantic cache，减少重复调用（详见 容灾、限流与降级）
  • 模型分级：简单分类用 qwen-turbo，复杂推理用 qwen-max
  • 上下文压缩：对长历史对话做摘要，减少 Input Token（详见 人在回路与上下文工程）

CorpAssist 场景：DashScope 完整接入

以 CorpAssist 客服 Agent 为例，配置一次完整的 DashScope 接入：

yaml 
# application.yml - CorpAssist 完整配置
spring:
  ai:
    dashscope:
      api-key: ${DASHSCOPE_API_KEY}
      chat:
        options:
          model: qwen-max
          temperature: 0.7
          max-tokens: 2048
      embedding:
        options:
          model: text-embedding-v3
    retry:
      max-attempts: 3
      backoff:
        initial-interval: 1000
        multiplier: 2
java 
// CorpAssist Agent 使用 ChatClient 简化的流式调用
@RestController
public class CorpAssistController {

    private final ChatClient chatClient;

    public CorpAssistController(ChatClient.Builder builder) {
        this.chatClient = builder.build();
    }

    @PostMapping("/corpassist/chat")
    public Flux chat(@RequestBody String message) {
        return chatClient.prompt()
            .system("你是 CorpAssist 客服助手。请用中文回答，保持专业、友好。")
            .user(message)
            .stream()
            .content();
    }
}

常见问题（FAQ）

DashScope 与百炼是什么关系？
DashScope 是阿里云大模型服务平台，百炼（Model Studio）是 DashScope 的商业化品牌。两者 API 兼容，SAA 统一使用 dashscope 配置前缀。详细信息参见 百炼官方文档。
qwen-max 和 qwen-plus 的智力差距有多大？
在复杂推理（数学、代码生成）场景，qwen-max 比 qwen-plus 准确率高约 15-20%。但简单 QA 场景差异不明显。建议：Agent 核心推理用 max，分类提取用 plus，可动态切换。
API Key 被泄露了怎么办？
立即在 DashScope 控制台 删除该 Key 并创建新 Key。同时检查调用记录是否有异常消耗。Key 泄漏是生产事故，建议启用 RAM Role 降低风险。
流式 URL 地址是什么？
SAA 默认使用官方 endpoint：https://dashscope.aliyun.com/api/v1/services/aigc/text-generation/generation。可通过 spring.ai.dashscope.base-url 自定义。

本章结论

DashScope 接入是 SAA 工程的第一步，核心是 API Key 管理、模型选型与流式调用。

  • API Key 管理：使用环境变量注入，多环境隔离，推荐 RAM Role。
  • 模型选型：turbo（低成本）/ plus（均衡）/ max（高智力）三层分级。
  • 流式调用：使用 StreamingChatModel 获得更好用户体验。
  • 成本控制：语义缓存 + 模型分级 + 上下文压缩三重降本。
  • CorpAssist 集成：完整 YAML + Java 配置，开箱即用。
