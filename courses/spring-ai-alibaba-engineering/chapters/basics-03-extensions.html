为什么要学本章

S3 办公 Agent 不仅要「聊天」：员工上传 PDF 请假制度、拍照发票、调用日历 MCP 都需要 saa extensions。若只会 DashScope Starter 而不会选 document parser 与 multimodal vl 模块，RAG 与 OCR 只能手写 SDK，与 Spring AI 抽象脱节。

学完你能
  • 说明 extensions 仓库与主框架的分工，并用 extension bom 管理版本
  • 为制度库 PDF 选用 tika parser 或百炼文档解析能力
  • 配置 qwen-vl 处理扫描件，并了解 community plugin 的接入边界
本章先记住 3 件事
  • 扩展独立发版：DashScope Starter、解析器在 extensions 仓库，BOM 单独 import。
  • 文档进 RAG 先解析：PDF/Word 必须结构化再 Embedding，勿直接把二进制塞 Prompt。
  • 多模态有成本：VL 模型 Token 更贵；S3 场景先 OCR 再文本 Agent 往往更稳。

文档解析

corpassist knowledge 制度库含 PDF、Word、扫描件。extensions 提供基于 Apache Tika / 百炼文档理解的 document parser，输出可切块文本供 VectorStore 索引。

能力适用格式S3 场景
Tika Document ReaderPDF、Office、HTML员工手册、请假制度 PDF
百炼文档解析 API复杂版式、表格含合并单元格的考勤表
自定义 Reader内部 OA 导出格式CorpAssist 专有 JSON 工单 dump
java
// 制度 PDF → 文本块 → Embedding（示意）
@Bean
DocumentReader leavePolicyReader() {
  return new TikaDocumentReader("classpath:policies/leave-policy.pdf");
}

@Bean
VectorStore ingestLeavePolicy(
    DocumentReader reader,
    VectorStore vectorStore,
    EmbeddingModel embeddingModel) {
  List<Document> docs = reader.get();
  TextSplitter splitter = new TokenTextSplitter(512, 50);
  vectorStore.add(splitter.apply(docs));
  return vectorStore;
}
不推荐
  • 把 PDF Base64 直接塞进 Chat Prompt
  • 不切块，整篇制度一次 Embedding
  • 解析失败静默跳过，Agent 仍回答「根据制度…」
推荐
  • Reader → Splitter → VectorStore 标准管道
  • 解析失败入 dead-letter，检索降级
  • 元数据标记 doc_version 便于增量更新
CorpAssist · 请假制度 RAG HR 更新 PDF 后，批处理 Job 重新解析并写入 VectorStore；S3 Agent 在线只检索，不在请求路径做重解析。与 Python 轨索引服务可 OpenAPI 分工（basics-04 双栈）。
先想 10 秒：为何 Agent 工具查制度仍需要 VectorStore？

Token 窗口装不下全文；检索 Top-K 片段 + 引用来源才是可审计答案。文档解析是 RAG 上游，不是 Agent 替代品。

多模态

multimodal vl（如 qwen-vl-plus）可理解图片中的表格与印章。S3 场景：员工拍照上传请假单、报销发票。extensions 与 DashScope 多模态 API 对接 Spring AI 的 Media 类型。

  API --> VL[qwen-vl-plus]
  VL --> JSON[结构化字段 JSON]
  JSON --> TOOL[submit_leave Tool]

vision rag：图片理解 + 文本 Agent 可串联；复杂流程见 multimodel-routing 课。

java
public LeaveDraft parseLeaveForm(byte[] imageBytes) {
  var media = Media.builder()
      .mimeType(MimeTypeUtils.IMAGE_JPEG)
      .data(imageBytes)
      .build();

  String json = chatClient.prompt()
      .user(u -> u.text("提取请假单字段，输出 JSON：姓名、起止日期、事由")
                  .media(media))
      .call()
      .content();
  return objectMapper.readValue(json, LeaveDraft.class);
}
策略优点缺点
端到端 VL实现快，适应版式变化Token 贵、可解释性弱
OCR + 文本模型成本低、可缓存 OCR 结果两阶段流水线运维复杂
专用 OCR 服务 + 规则发票等固定格式最稳新格式需开发
qwen-vl-plus

通用图文理解，适合请假单、截图 FAQ。

text-embedding-v3

纯文本向量；图片须先 OCR/VL 转文本再索引。

Media API

Spring AI 统一 Media 模型，换 VL 模型不改业务接口。

自检：extensions 与 agent-framework 依赖边界？

agent-framework 管 ReAct/Graph 循环；extensions 管模型侧能力与第三方集成。二者版本分别由 alibaba-bom 与 extensions-bom 对齐。

扩展场景推荐模块与 RAG 课衔接
制度 PDF 增量更新Tika Reader + 定时 Jobrag-system-java 索引管道
发票 OCRVL 或专用 OCR 服务multimodel-routing VL 章
工单 MCPcommunity MCP 桥接agent-orchestration-java MCP 章
extensions 升级是否必须停服 Agent？
Reader/Embedding 变更影响索引管道，可滚动重建 VectorStore；在线 Agent 只依赖检索 API。升级前在 staging 跑完整 reindex 验收。
CorpAssist 能否只用 Python 做文档解析、Java 只检索？
可以，且是推荐双栈分工（basics-04）。Python 批处理解析 + 写索引版本；Java extensions 侧保留 Reader 用于小规模热更新或灾备。

面试快问快答

CorpAssist 制度库更新后，Agent 如何感知新版本？

答：批处理重新解析 PDF → 写 VectorStore（带 doc_version 元数据）；在线 Agent 检索时过滤最新 version；可选 Admin 发布 DSL 触发重建（practice-01）。

为何不把所有 extensions 一次性引入？

答：增大攻击面与启动时间；未用模块仍可能拉 transitive 依赖。按场景最小引入，BOM 统一版本。
