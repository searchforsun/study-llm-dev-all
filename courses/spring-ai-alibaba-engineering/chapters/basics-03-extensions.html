本章先记住 3 件事
  • Extensions 不是独立服务：它们作为 SAA 的补充模块，与 Starter 共享同一配置体系。
  • 文档解析先于 RAG：文档解析是 RAG 管线的第一步，质量直接影响检索效果。
  • 多模态是 Agent 的眼睛：视觉模型让 Agent 具备理解图表、截图、手写文字的能力。

Document Parser 文档解析

doc parser 是 RAG 管线的前置模块，将 PDF/Word/Excel 转换为 LLM 友好的 Markdown 格式。

引入依赖

xml 

    com.alibaba.cloud.ai
    spring-ai-alibaba-document-parser

解析 PDF 合同

java 
@Service
public class CorpAssistDocParser {

    private final DocumentParser documentParser;

    public CorpAssistDocParser(DocumentParser documentParser) {
        this.documentParser = documentParser;
    }

    /**
     * 解析 PDF 合同，返回 Markdown 结构化文本
     */
    public String parseContract(InputStream pdfStream, String fileName) {
        Document doc = new Document(pdfStream, fileName);
        List parsed = documentParser.parse(doc);

        return parsed.stream()
            .map(Document::getText)
            .collect(Collectors.joining("\n\n"));
    }

    /**
     * 解析后分块，准备注入 VectorStore
     */
    public List chunkAndEmbed(InputStream pdfStream, String fileName) {
        Document doc = new Document(pdfStream, fileName);
        List parsed = documentParser.parse(doc);

        TokenTextSplitter splitter = new TokenTextSplitter(500, 50, 5, 0);
        return splitter.apply(parsed);
    }
}
不推荐
  • PDF 直接全文塞入 LLM 上下文窗口
  • 不解析排版，纯字符串拼接
  • 不处理表格和图片
推荐
  • 使用 Document Parser 转为 Markdown
  • 保留标题层级、表格结构
  • 解析后分块再注入 RAG 管线
CorpAssist · 合同审批 Agent 员工上传 PDF 采购合同，Agent 自动解析合同条款并提取关键字段（金额、日期、签约方），生成审批摘要，提交人工审批。整个过程从「原始 PDF」到「结构化审批单」完全自动化。

多模态：视觉与 OCR

multimodal 扩展使 Agent 能理解图片内容。包含两个子能力：vision model（视觉理解）和 ocr dashscope（文字识别）。

视觉理解

java 
@Service
public class CorpAssistVisionService {

    private final ChatModel visionModel;

    public CorpAssistVisionService(ChatModel visionModel) {
        this.visionModel = visionModel;
    }

    /**
     * 理解截图中的流程图或 UI
     */
    public String describeImage(String imageUrl) {
        UserMessage message = new UserMessage(
            "请描述这张图片的内容，重点关注文字和数据结构",
            List.of(new Media(MimeTypeUtils.IMAGE_PNG, imageUrl))
        );
        Prompt prompt = new Prompt(message);
        return visionModel.call(prompt).getResult().getOutput().getContent();
    }
}

OCR 文字识别

java 
/**
 * 报销单 OCR 识别 - CorpAssist 场景
 */
public class ReceiptOcrService {

    private final DashScopeOcrService ocrService;

    public ReceiptOcrService(DashScopeOcrService ocrService) {
        this.ocrService = ocrService;
    }

    public OcrResult recognizeReceipt(byte[] imageBytes) {
        // 识别发票/报销单上的文字
        return ocrService.recognize(imageBytes, OcrType.RECEIPT);
    }
}

extension module 的设计模式：每个扩展独立 JAR，通过 SPI 自动注册到 SAA 的 ExtensionRegistry 中。

java 
// 自定义 Extension SPI 示例
public class CorpAssistExtension implements AgentExtension {

    @Override
    public String getName() {
        return "corpassist-oa";
    }

    @Override
    public Set getToolNames() {
        return Set.of("leave_apply", "overtime_apply", "receipt_upload");
    }

    @Override
    public Object execute(String toolName, Map args) {
        // 实现办公自动化工具的调用逻辑
        return switch (toolName) {
            case "leave_apply" -> handleLeaveApply(args);
            case "receipt_upload" -> handleReceiptUpload(args);
            default -> throw new UnsupportedOperationException("未知工具: " + toolName);
        };
    }
}

CorpAssist 场景：多模态审批 Agent

综合各扩展模块，构建一个报销审批 Agent：

java 
@Service
public class ExpenseApprovalAgent {

    private final CorpAssistDocParser docParser;
    private final ReceiptOcrService ocrService;
    private final CorpAssistVisionService visionService;
    private final ChatModel chatModel;

    /**
     * 审批流程：接收附件 → 解析 → 提取关键字段 → 生成审批单
     */
    public ApprovalResult processExpense(ExpenseRequest request) {
        // 1. OCR 识别发票文字
        OcrResult ocrResult = ocrService.recognizeReceipt(request.receiptImage());

        // 2. 如果含 PDF 附件，解析文档
        String docText = request.attachment() != null
            ? docParser.parseContract(new ByteArrayInputStream(request.attachment()), "报销单.pdf")
            : "";

        // 3. 调用 LLM 提取结构化信息
        String extractPrompt = """
            从以下 OCR 结果和文档中提取报销信息：
            总金额、日期、报销人、项目名、是否合规

            OCR结果：
            %s

            文档内容：
            %s
            """.formatted(ocrResult.getText(), docText);

        Prompt prompt = new Prompt(new UserMessage(extractPrompt));
        String extraction = chatModel.call(prompt).getResult().getOutput().getContent();

        // 4. 生成审批摘要，返回给前端等待人工确认
        return ApprovalResult.of(request.requestId(), extraction, Status.PENDING);
    }
}

常见问题（FAQ）

Document Parser 支持哪些文件格式？
支持 PDF（含扫描件）、Word（doc/docx）、Excel（xls/xlsx）、PPT（ppt/pptx）、纯文本。扫描版 PDF 会自动触发 OCR 管线。
Vision 模型支持哪些图片格式？
PNG、JPG、JPEG、WEBP、BMP。支持 URL 和 base64 两种图片传入方式。单图建议小于 20MB。
社区插件如何确保稳定性和安全性？
官方标注"社区维护"的插件需自行评估。建议：检查 GitHub stars、最近更新日期、Issue 响应速度。生产环境使用前进行安全审计和压测。
OCR 识别率能达到多少？
对于标准印刷体，识别率可达 99%+。手写体在 85-95% 之间。建议对关键字段做二次校验（金额正则、日期格式）。

本章结论

  • Extensions 覆盖四大能力：文档解析、视觉理解、OCR、语音合成。
  • Document Parser 是 RAG 前置：解析 PDF/Word 为 Markdown 后分块嵌入。
  • 多模态扩展 Agent 感知：视觉和 OCR 让 Agent 理解图片内容。
  • 社区插件生态：通过 SPI 扩展 Agent 能力，钉钉、搜索、SQL 等。
  • Combo 组合场景：OCR + 文档解析 + LLM 提取，实现报销审批全自动化。
