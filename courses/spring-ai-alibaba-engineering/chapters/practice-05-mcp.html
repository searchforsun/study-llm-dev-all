为什么要学本章

mcp protocol（Model Context Protocol）是 AI 工具接入的行业标准协议。通过 MCP，Agent 可以调用任何遵循该协议的外部服务，而无需为每个工具编写自定义集成代码。CorpAssist 使用 MCP 接入企业内部的 ERP、CRM、OA 系统。

学完你能
  • 理解 MCP 协议的 stdio sse 两种传输模式
  • 使用 tool registry 注册 MCP 工具
  • 配置 mcp security 白名单和鉴权
  • 在 CorpAssist Agent 中集成 MCP 工具
本章先记住 3 件事
  • MCP 是标准化协议：定义了 tools/resources/prompts 三个能力域。
  • 透明传输：支持 stdio（子进程）和 SSE（HTTP）两种模式。
  • 安全是责任：只注册信任的 mcp server，配置严格的权限白名单。

MCP 协议概览

MCP 定义了三个核心 capability list：

能力说明Agent 视角
Tools 可调用的函数，有输入/输出 schema Agent 决定何时调用哪个 Tool
Resources 可读取的数据资源（文件、API 结果等） Agent 按需拉取数据
Prompts 预定义的 Prompt 模板 开发者或 Agent 使用模板生成 Prompt
MCP 在 Agent 架构中的位置
flowchart TB
    Agent[SAA Agent\nReactAgent / Graph] --> MCPClient[MCP Client\nToolRegistry]
    MCPClient --> MCPServer1[MCP Server\nERP 系统]
    MCPClient --> MCPServer2[MCP Server\nCRM 系统]
    MCPClient --> MCPServer3[MCP Server\nOA 审批]

    subgraph MCPClient[MCP 客户端]
        TR[ToolRegistry]
        Auth[鉴权模块]
        Audit[审计日志]
    end

    subgraph MCPServer1[MCP Server]
        T1[query_invoice]
        T2[create_purchase_order]
        R1[/invoices/list]
    end

MCP Client 作为 Agent 和外部服务之间的桥梁：Agent 通过 ToolRegistry 发现可用工具，调用请求经鉴权和审计后转发到 MCP Server。

CorpAssist · ERP 系统集成 CorpAssist 通过 MCP 接入企业 ERP 系统，Agent 能查询发票状态、发起采购审批、查看库存信息。无需为 ERP 写定制 @Tool，只需要一个 MCP Server 适配器。

传输模式：stdio vs SSE

mcp server 支持两种传输方式：

模式通信方式适用场景生命周期
stdio 子进程标准输入输出 本地工具、CLI 工具 随 Agent 进程启停
SSE HTTP Server-Sent Events 远程服务、微服务 独立部署

stdio 模式配置

json 
{
  "mcpServers": {
    "erp-system": {
      "command": "node",
      "args": ["erp-mcp-server/index.js"],
      "env": {
        "ERP_API_KEY": "${ERP_API_KEY}",
        "ERP_BASE_URL": "https://erp.corp.com/api"
      }
    },
    "oa-approval": {
      "command": "python",
      "args": ["-m", "oa_mcp_server"],
      "env": {
        "OA_APP_SECRET": "${OA_APP_SECRET}"
      }
    }
  }
}

SSE 模式配置（生产推荐）

yaml 
spring:
  ai:
    mcp:
      client:
        sse:
          connections:
            - id: erp-mcp
              url: https://erp-mcp.corp.com/sse
              headers:
                Authorization: Bearer ${ERP_MCP_TOKEN}
            - id: crm-mcp
              url: https://crm-mcp.corp.com/sse
              headers:
                Authorization: Bearer ${CRM_MCP_TOKEN}

在 SAA 中使用 MCP 工具

tool registry 自动发现并注册所有已连接 MCP Server 的 Tools。

java 
@Configuration
public class MCPAgentConfig {

    @Bean
    public ReactAgent mcpAgent(
            ChatModel chatModel,
            ToolRegistry toolRegistry) {

        // ToolRegistry 已自动聚合所有 MCP Server 的 Tools
        List allTools = toolRegistry.getAllTools();

        return ReactAgent.builder()
            .chatModel(chatModel)
            .tools(allTools)
            .systemPrompt("你是 CorpAssist 助手，可通过 MCP 工具访问企业系统。")
            .maxSteps(15)
            .build();
    }
}

// 查看已注册的 MCP 工具
@Component
public class MCPInspector {

    private final ToolRegistry toolRegistry;

    public MCPInspector(ToolRegistry toolRegistry) {
        this.toolRegistry = toolRegistry;
    }

    @PostConstruct
    public void listTools() {
        log.info("已注册的 MCP 工具:");
        toolRegistry.getAllTools().forEach(tool ->
            log.info("  - {}: {}", tool.getName(), tool.getDescription())
        );
    }
}
不推荐
  • 为每个外部系统手写 @Tool 适配器
  • 工具调用不记录审计日志
  • 所有 MCP Server 不加鉴权直接连接
推荐
  • 使用 MCP 标准协议统一接入
  • 工具调用通过 Auditor 记录
  • MCP Server 配置 Token 或 mTLS 鉴权

MCP 安全配置

mcp security 通过白名单和鉴权两层防护。

工具白名单

yaml 
spring:
  ai:
    mcp:
      client:
        allowed-tools:     # 白名单，只允许这些工具被 Agent 调用
          - erp-system:query_invoice
          - erp-system:get_invoice_detail
          - oa-approval:query_approval_status
        blocked-tools:     # 黑名单
          - erp-system:delete_invoice
          - erp-system:update_purchase_order

审计日志

java 
@Component
public class MCPAuditor {

    private static final Logger log = LoggerFactory.getLogger(MCPAuditor.class);

    /**
     * 记录所有 MCP 工具调用
     */
    @EventListener
    public void onToolCall(ToolCallEvent event) {
        log.info("MCP Tool Call: server={}, tool={}, args={}, user={}",
            event.serverId(),
            event.toolName(),
            sanitize(event.args()),
            event.userId()
        );
        // 写入审计数据库
        auditRepository.save(new AuditRecord(
            event.executionId(),
            event.serverId(),
            event.toolName(),
            event.args(),
            event.result(),
            event.timestamp()
        ));
    }

    private String sanitize(Map args) {
        // 脱敏敏感字段（如密码、token）
        Map safe = new HashMap<>(args);
        safe.keySet().forEach(k -> {
            if (k.toLowerCase().contains("password")
                || k.toLowerCase().contains("secret")) {
                safe.put(k, "****");
            }
        });
        return safe.toString();
    }
}

自定义 MCP Server

sandbox tool 是安全沙箱中的工具执行环境。创建自定义 MCP Server：

java 
// 自定义 CorpAssist MCP Server
@SpringBootApplication
public class CorpAssistMCPServer {

    public static void main(String[] args) {
        SpringApplication.run(CorpAssistMCPServer.class, args);
    }

    @Bean
    public MCPServer corpassistMCPServer() {
        return MCPServer.builder()
            .serverInfo(new ServerInfo("corpassist-oa", "1.0.0"))
            .addTool(new ToolRegistration(
                "send_notification",
                "发送通知给指定员工",
                """
                {
                  "type": "object",
                  "properties": {
                    "employeeId": {"type": "string"},
                    "title": {"type": "string"},
                    "message": {"type": "string"}
                  },
                  "required": ["employeeId", "title", "message"]
                }
                """,
                this::sendNotification
            ))
            .addTool(new ToolRegistration(
                "query_leave_balance",
                "查询员工剩余假期天数",
                """
                {
                  "type": "object",
                  "properties": {
                    "employeeId": {"type": "string"},
                    "leaveType": {"type": "string"}
                  }
                }
                """,
                this::queryLeaveBalance
            ))
            .build();
    }

    private String sendNotification(Map args) {
        String employeeId = (String) args.get("employeeId");
        String title = (String) args.get("title");
        String message = (String) args.get("message");
        // 调用通知服务
        notificationService.send(employeeId, title, message);
        return "通知发送成功";
    }

    private String queryLeaveBalance(Map args) {
        String employeeId = (String) args.get("employeeId");
        String leaveType = (String) args.get("leaveType");
        return leaveService.getBalance(employeeId, leaveType).toString();
    }
}

常见问题（FAQ）

MCP 和 @Tool 的关系是什么？
@Tool 是 SAA 本地工具定义方式，MCP 是远程工具协议。两者最终都通过 ToolCallback 接口暴露给 Agent。最佳实践：本地简单逻辑用 @Tool，外部系统集成用 MCP。详见 ReactAgent @Tool。
MCP Server 如果挂了怎么办？
Agent 调用时返回错误作为 observation，模型会自行决定是否重试。建议为关键 MCP Server 配置多副本和健康检查。SAA 的 Resilience4j 配置可包装 MCP 调用，参见 容灾、限流与降级。
stdio 和 SSE 模式如何选择？
stdio 适合部署在 Agent 同机或同容器的工具（CLI、脚本）。SSE 适合跨网络、独立部署的微服务。SSE 是生产推荐模式，支持独立扩缩容和权限管控。
MCP 工具返回的数据量有上限吗？
建议 MCP Tool 返回控制在 10KB 以内。大量数据应该通过 Resource 能力暴露，Agent 按需读取。返回超大数据可能撑爆上下文窗口。

本章结论

  • MCP 协议：标准化工具接入协议，支持 Tools/Resources/Prompts 三种能力。
  • 传输模式：stdio 适合本地，SSE 适合远程分布式部署。
  • ToolRegistry：自动发现并聚合所有 MCP Server 的工具。
  • 安全机制：白名单 + 鉴权 + 审计日志三层防护。
  • 自定义 Server：通过 MCPServer Builder 快速创建 MCP 服务。
