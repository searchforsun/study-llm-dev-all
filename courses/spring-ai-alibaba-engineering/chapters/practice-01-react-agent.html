为什么要学本章

react agent 是 SAA 中 Agent 能力的核心。它实现 Reasoning + Acting（ReAct）循环：模型思考→调用工具→观察结果→继续思考。CorpAssist S3 办公 Agent 的本质就是一个 ReactAgent——自主决策任务流程、调用企业工具、处理中间结果。本章从零实现一个 ReactAgent。

学完你能
  • 理解 react loop 的 Observe-Think-Act 三阶段
  • 使用 tool calling 注册自定义工具
  • 配置 step budget 和 stop condition 控制 Agent 行为
  • 为 CorpAssist 实现一个带工单查询工具的 Agent
本章先记住 3 件事
  • ReAct = 思考 + 行动 + 观察：模型不是一次性回答，而是逐轮推理。
  • 工具即函数：@Tool 注解将任意 Java 方法暴露为 Agent 可调用的能力。
  • 步数预算是安全带：没有 maxSteps 的 Agent 可能陷入无限循环。

ReAct 循环原理

react loop 遵循 Observe → Think → Act 循环：

ReAct 一次迭代流程
sequenceDiagram
    participant LLM as ChatModel
    participant Agent as ReactAgent
    participant Tool as 工具

    Agent->>LLM: 发送 System + User 消息
    LLM-->>Agent: 思考结果（thought）\n可能含工具调用请求
    Agent->>Agent: 检查 stop conditions

    alt 需要调用工具
        Agent->>Tool: 执行 @Tool 方法
        Tool-->>Agent: 观察结果（observation）
        Agent->>LLM: 追加 thought+observation
        LLM-->>Agent: 继续思考
    else 直接回答
        Agent-->>User: 返回最终回答
    end

ReAct 循环使 Agent 能在多次工具调用中逐步推理，每次"观察"结果都作为下一轮思考的上下文。

observe think 模式的关键特征是：工具返回结果不会直接拼接到回答中，而是作为"观察"反馈给模型，由模型决定下一步行动。

CorpAssist · 工单状态查询 员工问"我的报销单 #T2024-0381 审批到哪了？"，ReactAgent 先思考需要查工单系统，调用工单查询工具，获取状态"待财务审核"，再思考需要补充什么信息，最终给出完整回答。

@Tool 工具注册

tool calling 通过 @Tool 注解将任意 Spring Bean 方法注册为 Agent 工具。

定义工具

java 
@Component
public class CorpAssistTools {

    private final TicketService ticketService;
    private final EmployeeService employeeService;

    public CorpAssistTools(TicketService ticketService, EmployeeService employeeService) {
        this.ticketService = ticketService;
        this.employeeService = employeeService;
    }

    @Tool(name = "query_ticket", description = "查询工单当前状态，输入工单号")
    public String queryTicket(String ticketId) {
        Ticket ticket = ticketService.findById(ticketId);
        if (ticket == null) {
            return "未找到工单: " + ticketId;
        }
        return "工单 %s 当前状态：%s，处理人：%s，创建时间：%s"
            .formatted(ticket.id(), ticket.status(), ticket.assignee(), ticket.createdAt());
    }

    @Tool(name = "get_employee_info", description = "查询员工信息，输入员工工号")
    public String getEmployeeInfo(String empId) {
        Employee emp = employeeService.findByEmpId(empId);
        if (emp == null) {
            return "未找到员工: " + empId;
        }
        return "员工 %s %s，部门：%s，职位：%s"
            .formatted(emp.empId(), emp.name(), emp.department(), emp.position());
    }
}

创建 ReactAgent

java 
@Configuration
public class CorpAssistAgentConfig {

    @Bean
    public ReactAgent corpassistAgent(
            ChatModel chatModel,
            List toolCallbacks) {

        return ReactAgent.builder()
            .chatModel(chatModel)
            .tools(toolCallbacks)
            .systemPrompt("""
                你是 CorpAssist 办公助手，帮助员工处理日常事务。
                你可以查询工单、查询员工信息、处理请假申请。
                请用中文回答，保持专业和简洁。
                """)
            .maxSteps(10)          // 最大推理步数
            .maxExecutionTime(30)  // 最大执行时间（秒）
            .build();
    }
}
不推荐
  • 不设 maxSteps，Agent 可能无限循环
  • 工具方法没有 description，模型不知道何时调用
  • 工具返回原始 JSON 不加工
推荐
  • 设 maxSteps=10~15，配合超时兜底
  • 每个 @Tool 写清晰的中文 description
  • 工具返回格式化文本，降低模型解析成本

停止条件与步数预算

stop condition 决定 Agent 何时停止循环。SAA 提供多种停止策略：

策略配置方式说明
最大步数 .maxSteps(N) 达到 N 次推理循环后停止，默认 10
最大执行时间 .maxExecutionTime(seconds) 超过秒数后强制停止，默认 60
工具白名单 .allowedTools(...) 仅允许指定工具，防止模型创造工具名
手动确认 .requireConfirmation(true) 高风险工具调用前需要用户确认

高级配置示例

java 
@Bean
public ReactAgent safeCorpassistAgent(ChatModel chatModel, List tools) {
    return ReactAgent.builder()
        .chatModel(chatModel)
        .tools(tools)
        .systemPrompt("你是 CorpAssist 安全模式 Agent。")
        .maxSteps(8)
        .maxExecutionTime(20)
        .allowedTools("query_ticket", "get_employee_info") // 仅允许查询
        .requireConfirmation(true)  // 所有工具需确认
        .build();
}
先想 10 秒：maxSteps=3 的 Agent 能完成"查工单状态→查处理人信息→回复"这个流程吗？

可以。3 步刚好：Step1 思考+查工单，Step2 观察+查处办人，Step3 观察+回复。但如果第二步返回需要多次查询员工部门信息，3 步就不够了。生产环境通常设 10-15 步。

CorpAssist 场景：完整 Agent 服务

java 
@RestController
@RequestMapping("/corpassist/agent")
public class CorpAssistAgentController {

    private final ReactAgent corpassistAgent;

    public CorpAssistAgentController(ReactAgent corpassistAgent) {
        this.corpassistAgent = corpassistAgent;
    }

    @PostMapping("/chat")
    public Mono chat(@RequestBody AgentRequest request) {
        return Mono.fromFuture(corpassistAgent.execute(request.userId(), request.message()))
            .map(result -> new AgentResponse(
                result.output(),
                result.steps(),
                result.tokensUsed(),
                result.executionTimeMs()
            ));
    }

    @PostMapping("/stream")
    public Flux streamChat(@RequestBody AgentRequest request) {
        return Flux.from(corpassistAgent.executeStream(request.userId(), request.message()))
            .map(AgentEvent::getContent)
            .filter(Objects::nonNull);
    }

    // 记录 Agent 执行轨迹
    record AgentResponse(String output, int steps, int tokens, long execMs) {}
    record AgentRequest(String userId, String message) {}
}

agent s3 是 CorpAssist S3 规范中定义的 Agent 服务标准接口，要求每个 Agent 暴露 /chat 和 /stream 两个端点。

常见问题（FAQ）

Agent 调用工具时超时了怎么办？
@Tool 方法本身应设内部超时。如果工具调用超时，ReactAgent 会将超时异常作为 observation 返回给模型，模型可能选择重试或跳过。建议使用 Resilience4j 包装工具调用，详见 容灾、限流与降级。
maxSteps 耗尽后 Agent 会怎样？
返回已收集到的所有信息作为最终回答，日志记录"max steps reached"。不会抛出异常，但回答可能不完整。建议设置告警：当 Agent 频繁达到 maxSteps 时，说明任务复杂度过高或工具设计不合理。
多个工具可以并行调用吗？
当前 ReactAgent 默认串行调用。如需并行，可使用 多 Agent 编排 中的 Parallel Agent 模式。
工具返回太长会不会撑爆上下文？
会。对于返回大量数据的工具（如查全部员工列表），应在工具方法内做截断或分页。全局上下文管理参见 人在回路与上下文工程。

本章结论

  • ReAct 循环：Agent 通过 Observe-Think-Act 多轮推理完成复杂任务。
  • @Tool 注册：注解将 Spring Bean 方法暴露为 Agent 工具，需写清晰描述。
  • 停止条件：maxSteps + timeout 是生产必备的安全措施。
  • LangGraph 对照：SAA ReactAgent 封装了循环，开箱即用。
  • Agent S3 规范：CorpAssist Agent 需暴露 /chat 和 /stream 端点。
