为什么要学本章

ReAct 循环和顺序编排无法表达"如果审批通过走 A 分支，否则走 B 分支"这样的条件逻辑。agent graph 用状态机显式建模业务流程的每个分支，比在 Prompt 里写"if-then"可靠得多。CorpAssist 的审批流程、异常处理、多轮复核都依赖 Graph。

本章先记住 3 件事
  • 节点 = 能力：每个节点是一个处理步骤，可以是 Agent、工具或函数。
  • 边 = 流转：普通边无条件，条件边根据状态选择下一个节点。
  • 状态机思维：Graph 的本质是一个有限状态机（FSM），状态在节点间流动。

SAA 的 state machine 基于有向图建模工作流。核心元素：

请假审批 Graph 使用条件边实现"通过/驳回"分支。每个审批阶段可细分为子状态。

CorpAssist · 请假审批 员工提交请假申请，Graph 依次经过：表单校验 → 上级审批（条件分支：通过/驳回）→ HR 确认（条件分支）→ 通知员工。每个节点是一个 Agent 或规则函数。

构建第一个 Graph

定义状态类

java 
// Graph 中传递的状态对象
public record LeaveState(
    String employeeId,
    String leaveType,      // annual / sick / personal
    LocalDate startDate,
    LocalDate endDate,
    String reason,
    String managerComment,
    String hrComment,
    Status status
) {
    public enum Status {
        SUBMITTED, MANAGER_APPROVED, MANAGER_REJECTED,
        HR_APPROVED, HR_REJECTED, COMPLETED
    }
}

定义节点处理器

java 
@Component
public class LeaveApprovalNodes {

    @GraphNode(name = "validate")
    public LeaveState validate(LeaveState state) {
        // 校验请假表单合法性
        if (state.startDate().isAfter(state.endDate())) {
            throw new GraphException("开始日期不能晚于结束日期");
        }
        if (state.leaveType() == null) {
            throw new GraphException("请选择请假类型");
        }
        return new LeaveState(
            state.employeeId(), state.leaveType(),
            state.startDate(), state.endDate(),
            state.reason(), null, null,
            LeaveState.Status.SUBMITTED
        );
    }

    @GraphNode(name = "manager_approve")
    public LeaveState managerApprove(LeaveState state) {
        // 调用 LLM 生成审批建议（模拟上级审批）
        String comment = "同意，请 HR 确认";
        return new LeaveState(
            state.employeeId(), state.leaveType(),
            state.startDate(), state.endDate(),
            state.reason(), comment, null,
            LeaveState.Status.MANAGER_APPROVED
        );
    }
}

构建并运行 Graph

java 
@Configuration
public class LeaveGraphConfig {

    @Bean
    public Graph leaveGraph(LeaveApprovalNodes nodes) {
        return Graph.builder()
            // 添加节点
            .addNode("validate", nodes::validate)
            .addNode("manager_approve", nodes::managerApprove)
            .addNode("hr_confirm", nodes::hrConfirm)
            .addNode("notify", nodes::notifyEmployee)
            .addNode("reject", nodes::rejectApplication)

            // 添加边
            .addEdge("validate", "manager_approve")
            .addConditionalEdge("manager_approve", state ->
                state.status() == LeaveState.Status.MANAGER_APPROVED
                    ? "hr_confirm" : "reject")
            .addConditionalEdge("hr_confirm", state ->
                state.status() == LeaveState.Status.HR_APPROVED
                    ? "notify" : "reject")
            .addEdge("reject", "notify")

            // 设置入口和出口
            .entryNode("validate")
            .build();
    }
}

条件路由详解

conditional edge 根据当前状态或外部条件决定下一个节点。branch route 模式支持多分支。

java 
// 多分支条件路由
graph.addConditionalEdge("classify", state -> {
    return switch (state.leaveType()) {
        case "annual" -> "annual_check";     // 年假 → 余额检查
        case "sick" -> "sick_check";         // 病假 → 病历上传
        case "personal" -> "personal_check"; // 事假 → 理由审核
        default -> "reject";
    };
});
不推荐
  • 在 Agent Prompt 里写"如果你是审批通过，执行 A；否则执行 B"
  • 用 if-else 编码在 @Tool 中模拟分支
  • 一个节点做所有决策
推荐
  • 用 Graph 的条件边显式建模分支
  • 每个节点职责单一，路由逻辑清晰
  • 条件边可观测、可测试、可调试

Graph 与 ReactAgent 的协作

Graph 节点内部可以使用 ReactAgent 作为执行单元：

java 
@Component
public class AgentBasedNodes {

    private final ReactAgent queryAgent;
    private final ReactAgent approvalAgent;

    public AgentBasedNodes(
            @Qualifier("queryAgent") ReactAgent queryAgent,
            @Qualifier("approvalAgent") ReactAgent approvalAgent) {
        this.queryAgent = queryAgent;
        this.approvalAgent = approvalAgent;
    }

    @GraphNode(name = "query_employee_info")
    public LeaveState queryEmployee(LeaveState state) {
        // 用 ReactAgent 查询员工信息
        AgentResult result = queryAgent.execute(
            state.employeeId(),
            "查询员工 " + state.employeeId() + " 的请假配额信息"
        );
        return state.withQueryResult(result.output());
    }

    @GraphNode(name = "llm_approve")
    public LeaveState llmApprove(LeaveState state) {
        // 用 ReactAgent 生成审批决策
        AgentResult result = approvalAgent.execute(
            state.employeeId(),
            "审批 " + state.employeeId() + " 的" + state.leaveType() + "假申请"
        );
        return state.withManagerComment(result.output())
            .withStatus(LeaveState.Status.MANAGER_APPROVED);
    }
}

这种"外层 Graph + 内层 Agent"的组合模式，既保证了流程的确定性（Graph 定义分支），又保留了 Agent 的灵活性（节点内部可以推理和调用工具）。

常见问题（FAQ）

Graph 和 SequentialOrchestrator 有什么区别？
SequentialOrchestrator 是线性管线，只能顺序执行；Graph 支持条件分支、循环、并行子图。Graph 是更通用的状态机，SequentialOrchestrator 是它的特例。
Graph 状态可以持久化吗？
可以。通过 StatePersister 接口，将 GraphState 持久化到 Redis 或数据库。这对于长时间运行（如审批流程可能持续数天）的工作流至关重要。
一个 Graph 可以嵌套另一个 Graph 吗？
可以。子图（Subgraph）是一种节点类型，内部包含完整的 Graph。适用于复杂的子流程，比如"请假审批"内部嵌套"余额检查"子图。
Conditional Edge 的路由函数抛异常会怎样？
Graph 会进入"错误处理"模式。可在 Graph Builder 中配置 .errorHandler(ErrorStrategy.SKIP_NODE) 或 ErrorStrategy.TO_END 来定义失败行为。

本章结论

  • 状态机模型：Graph 用 Node + Edge + State 显式建模工作流。
  • 条件路由：Conditional Edge 实现通过/驳回等多分支逻辑。
  • 可视化：导出 Mermaid 或使用 SAA Admin Studio 可视化编辑。
  • 嵌套协作：Graph 节点内部可使用 ReactAgent，组合确定性和灵活性。
  • 错误处理：配置 ErrorStrategy 定义节点失败时的行为。
