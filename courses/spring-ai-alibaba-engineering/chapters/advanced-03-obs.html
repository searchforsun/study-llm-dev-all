为什么要学本章

Agent 是多步推理的"黑盒"——你看到输入和最终输出，中间发生了什么？工具调用是否成功？模型思考了哪些路径？没有可观测性的 Agent 服务，排查一个异常回答可能需要数小时。agent trace 和 prompt replay 是解决这一问题的关键。

结构化 Step Log

step log 记录 Agent 的每一步详细过程。

java 
@Component
public class AgentStepLogger {

    private static final Logger log = LoggerFactory.getLogger("AgentStep");

    /**
     * 记录 Agent 每一步的完整信息
     */
    public void logStep(StepRecord record) {
        // 通过 MDC 注入上下文
        MDC.put("request_id", record.requestId());
        MDC.put("step", String.valueOf(record.stepNumber()));
        MDC.put("agent_id", record.agentId());

        log.info("Agent Step: {} | Thought: {} | Tool: {} | ToolInput: {} | ToolOutput: {}",
            record.stepNumber(),
            truncate(record.thought(), 500),
            record.toolName(),
            truncate(record.toolInput(), 300),
            truncate(record.toolOutput(), 1000)
        );

        MDC.clear();
    }

    /**
     * 完整的 StepRecord 记录
     */
    public record StepRecord(
        String requestId,
        String agentId,
        int stepNumber,
        String thought,        // 模型思考内容
        String toolName,       // 调用的工具名（如有）
        String toolInput,      // 工具输入
        String toolOutput,     // 工具输出
        long durationMs,       // 本步耗时
        int tokensUsed,        // 本步 Token 消耗
        Instant timestamp
    ) {}

    private String truncate(String s, int max) {
        return s != null && s.length() > max
            ? s.substring(0, max) + "..."
            : s;
    }
}

// 在 Agent 执行中埋点
@Aspect
@Component
public class StepLoggingAspect {

    @Around("execution(* com.corpassist.agent.ReactAgent.execute(..))")
    public Object logAgentExecution(ProceedingJoinPoint pjp) throws Throwable {
        // 执行前记录
        AgentResult result = (AgentResult) pjp.proceed();
        // 执行后记录到 ES
        return result;
    }
}

Metrics 埋点

micrometer agent 指标暴露到 Prometheus + Grafana。

java 
@Component
public class AgentMetrics {

    private final MeterRegistry meterRegistry;

    public AgentMetrics(MeterRegistry meterRegistry) {
        this.meterRegistry = meterRegistry;
    }

    /**
     * 记录 Agent 执行指标
     */
    public void recordExecution(AgentResult result) {
        // 执行步数分布
        meterRegistry.summary("agent.steps.total")
            .record(result.steps());

        // Token 消耗
        meterRegistry.counter("agent.tokens.input",
            "model", result.model())
            .increment(result.inputTokens());
        meterRegistry.counter("agent.tokens.output",
            "model", result.model())
            .increment(result.outputTokens());

        // 工具调用计数
        result.toolCalls().forEach(tool ->
            meterRegistry.counter("agent.tool.calls",
                "tool", tool.name())
                .increment()
        );

        // 执行延迟
        meterRegistry.timer("agent.execution.duration")
            .record(result.executionTime(), TimeUnit.MILLISECONDS);

        // 成功率
        meterRegistry.counter("agent.execution.status",
            "status", result.isSuccess() ? "success" : "failure")
            .increment();
    }
}

Trace 链路追踪

otel trace 追踪 Agent 跨步骤、跨服务的调用链路。

java 
@Configuration
public class AgentTracingConfig {

    @Bean
    public Tracer agentTracer(OpenTelemetry openTelemetry) {
        return openTelemetry.getTracer("corpassist-agent");
    }
}

// 在 Agent 执行中加入 Span
@Service
public class TracedAgentService {

    private final Tracer tracer;
    private final ReactAgent agent;

    public TracedAgentService(Tracer tracer, ReactAgent agent) {
        this.tracer = tracer;
        this.agent = agent;
    }

    public Mono executeWithTrace(String userId, String message) {
        Span span = tracer.spanBuilder("agent.execute")
            .setAttribute("user.id", userId)
            .setAttribute("message.length", message.length())
            .startSpan();

        return Mono.fromFuture(agent.execute(userId, message))
            .doFinally(signalType -> {
                span.end();
            });
    }
}

Prompt 回放

prompt replay 是调试 Agent 最强大的功能：通过保存的 request_id 复现 Agent 的完整执行过程。

java 
@Service
public class PromptReplayService {

    private final ElasticsearchRestTemplate esTemplate;
    private final ChatModel chatModel;

    public PromptReplayService(ElasticsearchRestTemplate esTemplate, ChatModel chatModel) {
        this.esTemplate = esTemplate;
        this.chatModel = chatModel;
    }

    /**
     * 根据 request_id 回放 Agent 执行
     */
    public ReplayResult replay(String requestId) {
        // 1. 从 ES 获取完整轨迹
        List steps = esTemplate.search(
            Query.findByRequestId(requestId),
            StepRecord.class
        );

        // 2. 按步数还原每一步的 Prompt
        List reconstructedPrompts = steps.stream()
            .map(step -> """
                === Step %d ===
                Thought: %s
                Tool Call: %s(%s)
                Tool Result: %s
                """.formatted(
                    step.stepNumber(), step.thought(),
                    step.toolName(), step.toolInput(),
                    step.toolOutput()
                ))
            .toList();

        // 3. 用同一 Prompt 重新调用模型
        // （用于对比实际输出和预期输出）
        String fullPrompt = String.join("\n", reconstructedPrompts);
        Prompt prompt = new Prompt(new UserMessage(fullPrompt));
        String replayed = chatModel.call(prompt).getResult().getOutput().getContent();

        return new ReplayResult(requestId, reconstructedPrompts, replayed);
    }

    public record ReplayResult(
        String requestId,
        List steps,
        String replayedOutput
    ) {}
}

常见问题（FAQ）

结构化日志和普通日志有什么区别？
结构化日志以 JSON 格式输出，包含 request_id、step、agent_id 等字段，可直接导入 ES/Kibana 搜索。普通文本日志只能 grep，无法做结构化分析。
Agent Trace 和普通微服务 Trace 有什么不同？
普通 Trace 追踪"请求→服务 A→服务 B→响应"的路径。Agent Trace 额外追踪"Step 1 思考→Step 1 工具调用→Step 2 思考→..."的推理过程。Agent Trace 的 Span 数量 = 推理步数 × 2。
保存每条请求的完整轨迹会不会太占存储？
会。建议：生产环境按 1:10 采样（保存 10% 的轨迹），异常轨迹 100% 保存；7 天后归档到冷存储；只保存关键字段（thought/tool call），不保存原始图片输入。
回放时发现模型回答和当时不一样，正常吗？
正常。LLM 非确定性输出导致相同 Prompt 可能得到不同回答。回放重点是验证"逻辑路径是否正确"而非"每字必须一致"。如果回放结果与原始结果差异很大，说明 Prompt 设计不够稳定。

本章结论

  • 结构化日志：每步记录 request_id、thought、tool call、duration。
  • Metrics 埋点：Token 消耗、步数、延迟、成功率四类核心指标。
  • Trace 链路：OpenTelemetry 追踪 Agent 跨步骤的推理路径。
  • Prompt 回放：基于 request_id 复现 Agent 完整执行过程。
  • Debug Studio：Admin 中的可视化调试界面。
