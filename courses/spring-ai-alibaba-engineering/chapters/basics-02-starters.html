为什么要学本章

saa starter 是 SAA 的自动装配核心。一个依赖声明即可获得 ChatModel、EmbeddingModel、Tool 等完整 Bean。理解 Starter 的自动配置机制、BOM 版本管理以及多环境 Profile 隔离，是生产级 SAA 应用的基础。

学完你能
  • 使用 BOM 管理 SAA 依赖版本
  • 配置多环境 Profile 切换模型与 API Key
  • 理解 ChatModel Bean 的自动装配原理
  • 在 CorpAssist 项目中实现 dev/staging/prod 三环境隔离
本章先记住 3 件事
  • BOM 统一版本：用 spring-ai-alibaba-bom 管理依赖版本。
  • 自动配置即 Bean：引入 Starter 后 chat model bean 自动注入。
  • Profile 隔离：开发模型用 qwen-turbo 降本，生产用 qwen-max。

BOM 依赖管理

SAA 使用 dependency bom 统一管理所有模块版本。推荐在项目的 dependencyManagement 中声明 BOM。

xml 


    
        
            com.alibaba.cloud.ai
            spring-ai-alibaba-bom
            2026.0.0
            pom
            import
        
    



    
    
        com.alibaba.cloud.ai
        spring-ai-alibaba-starter-dashscope
    
    
    
        com.alibaba.cloud.ai
        spring-ai-alibaba-starter-graph

alibaba bom 同时管理了阿里云 SDK 的依赖版本，避免与 SAA 冲突。

不推荐
  • 每个依赖手动填写版本号
  • 同时引入多个不同版本的 SAA Starter
  • 忽略 BOM 直接使用最新版
推荐
  • 使用 spring-ai-alibaba-bom 统一版本
  • Starter 声明时不写版本（由 BOM 管理）
  • 定期更新 BOM 版本并回归测试

自动配置机制

SAA 的 auto config 遵循 Spring Boot Auto-Configuration 约定：通过 spring.factories 注册，依赖 Condition 注解按条件装配。

Starter 自动配置流程
flowchart LR
    Dep[声明 Starter 依赖] --> AutoConf[spring-ai-alibaba-starter-dashscope]
    AutoConf --> Condition{ConditionalOnClass\nDashScopeApi}
    Condition -->|满足| CreateBean[创建 ChatModel Bean]
    Condition -->|不满足| Skip[跳过，不报错]
    CreateBean --> Inject[注入到 Service]
    Inject --> CorpAssist[CorpAssist Agent\n使用 ChatModel]

    subgraph AutoConf[Starter 内部]
        DashScopeProperties[读取 application.yml\nspring.ai.dashscope.*]
        DashScopeChatModel[new DashScopeChatModel\n配置 Key/Model]
    end

Starter 启动时检查 classpath 中是否存在 DashScope SDK，满足条件才自动创建 Bean。

可配置属性一览

config properties 通过 @ConfigurationProperties(prefix = "spring.ai.dashscope") 绑定。

配置项类型默认值说明
spring.ai.dashscope.api-key String — 必填，DashScope API Key
spring.ai.dashscope.base-url String 官方 endpoint 自定义 API 网关地址
spring.ai.dashscope.chat.options.model String qwen-plus 聊天模型名称
spring.ai.dashscope.chat.options.temperature Double 0.8 生成随机性，Agent 推荐 0.7
spring.ai.dashscope.chat.options.max-tokens Integer 2048 最大输出 Token 数
spring.ai.dashscope.embedding.options.model String text-embedding-v3 嵌入模型名称
CorpAssist · 多环境配置隔离 CorpAssist S3 部署在 dev/staging/prod 三个环境。dev 使用 qwen-turbo 降低成本，staging 使用 qwen-plus 做灰度验证，prod 使用 qwen-max 保障质量。通过 Spring Profile 实现配置隔离。

多环境 Profile 配置

spring profile 是 Spring Boot 原生多环境方案。SAA 配置随 Profile 自动切换。

application-dev.yml（开发环境）

yaml 
# application-dev.yml
spring:
  ai:
    dashscope:
      api-key: ${DASHSCOPE_DEV_API_KEY}
      chat:
        options:
          model: qwen-turbo
          temperature: 0.9  # 开发时可略高

application-prod.yml（生产环境）

yaml 
# application-prod.yml
spring:
  ai:
    dashscope:
      api-key: ${DASHSCOPE_PROD_API_KEY}
      chat:
        options:
          model: qwen-max
          temperature: 0.7
      embedding:
        options:
          model: text-embedding-v3

多环境切换命令

bash 
# 开发启动
java -Dspring.profiles.active=dev -jar corpassist-agent.jar

# 生产启动（通过容器环境变量 SPRING_PROFILES_ACTIVE=prod）
export SPRING_PROFILES_ACTIVE=prod
java -jar corpassist-agent.jar
多环境管理要点
  • 每个 Profile 使用独立 API Key，生产 Key 有更高 QPS 配额
  • Dev 环境使用 multi env 的 qwen-turbo 降低开发成本 80%
  • 通过配置中心（Nacos/ACM）实现运行时动态切换，参见 A2A 与 Nacos

ChatModel Bean 使用实战

引入 Starter 后，chat model bean 自动注入。可直接注入 ChatModel 或自定义配置。

自动注入使用

java 
@Service
public class CorpAssistService {

    // 自动注入 Starter 创建的 ChatModel Bean
    private final ChatModel chatModel;
    private final EmbeddingModel embeddingModel;

    public CorpAssistService(ChatModel chatModel, EmbeddingModel embeddingModel) {
        this.chatModel = chatModel;
        this.embeddingModel = embeddingModel;
    }

    public String chat(String message) {
        Prompt prompt = new Prompt(new UserMessage(message));
        return chatModel.call(prompt).getResult().getOutput().getContent();
    }
}

自定义 ChatModel Bean

当需要多模型实例时，可手动创建：

java 
@Configuration
public class CorpAssistModelConfig {

    @Bean
    @Primary
    @ConditionalOnProperty(value = "corpassist.model.main", havingValue = "qwen-max")
    public ChatModel mainChatModel(DashScopeChatOptions options) {
        return new DashScopeChatModel(
            new DefaultDashScopeChatModel("qwen-max", options));
    }

    @Bean
    @Qualifier("turbo")
    public ChatModel turboChatModel() {
        return new DashScopeChatModel("qwen-turbo", null);
    }
}

常见问题（FAQ）

Starter 引入后 ChatModel Bean 没有创建，怎么办？
检查: (1) 是否在 dependencyManagement 中引入了 BOM；(2) spring.ai.dashscope.api-key 是否正确配置；(3) 启动日志是否有 DashScopeAutoConfiguration matched。通过 --debug 启动可看到 Condition 评估详情。
BOM 版本和 Spring Boot 版本必须匹配吗？
需要匹配。SAA 2026.0.0 要求 Spring Boot 3.4+，Spring Cloud 2024.0+。不匹配可能导致自动配置不生效。官方兼容性矩阵参见 SAA 官方文档。
同个项目需要同时用 qwen-plus 和 qwen-max，怎么配置？
使用 @Primary 标注主模型，@Qualifier 标注辅助模型。或使用 多 Agent 中不同 Agent 使用不同模型。
生产环境如何动态切换模型而不重启？
结合 Nacos 配置中心 + @RefreshScope 实现。当配置变更时，ChatModel Bean 自动重建。参见 Admin 与配置管理。

本章结论

  • BOM 统一版本：使用 spring-ai-alibaba-bom 管理依赖版本，避免版本冲突。
  • 自动配置按需装配：Starter 通过 Condition 注解按条件创建 Bean。
  • 多环境 Profile：dev/staging/prod 分别配置不同模型和 API Key。
  • Bean 自定义：通过 @Primary 组合 @Qualifier 实现多模型实例。
  • 生态集成：SAA Starter 与 Nacos、Sentinel、Actuator 无缝协作。

配置 CorpAssist 项目的多环境 Starter 依赖（约 15 分钟）。

操作步骤

  1. 创建一个新的 Spring Boot 项目，在 pom.xml 中添加 SAA BOM 和 Starter 依赖
  2. 编写 application-dev.yml / application-prod.yml 多环境配置
  3. 通过启动参数切换 Profile，验证模型是否发生变更

判断练习

  1. 项目引入 starter 后启动报错 No qualifying bean of type ChatModel，排障步骤是什么？

    参考答案

    三步排查: (1) 确认 pom.xml 是否引入了 starter；(2) 确认 application.yml 是否正确配置 api-key；(3) 开启 --debug 查看 DashScopeAutoConfiguration 是否 matched。
