常见面试题及回答

interview saa 整理了最常被问到的通义栈面试题：

Q1: "请设计一个企业级 Agent 系统"

text 
STARS 回答示例：

**S 场景**: 企业需要自动化处理员工的请假、报销、工单查询。日请求量 10 万+。

**T 技术选型**: Spring AI Alibaba + DashScope + Nacos。理由：
- 团队现有 Java + Spring Cloud 栈
- 需要 Nacos 服务治理
- 需要规范化工具集成（MCP）

**A 架构**:
- 路由 Agent (qwen-turbo) 做意图分类
- 每个场景独立 Graph 状态机
- InterruptNode 实现审批
- MCP 接入 ERP/OA 系统
- ChatMemory 管理上下文
- StepLogger + Micrometer 可观测

**R 容错**:
- Resilience4j 熔断保护 DashScope 调用
- 语义缓存（Redis）+ 多模型降级
- 工具白名单 + 审计日志

**S 评测**:
- 完成率 > 90% | 平均步数 < 8 | 准确率 > 95%

Q2: "你怎么保证 Agent 不会胡乱操作企业数据？"

回答要点：工具白名单（只允许读的不能写）+ 审批节点（写操作需人工确认）+ 审计日志（全链路可追溯）。详见 实战：CorpAssist Java Agent 的安全设计。

Q3: "对比 LangChain/LangGraph 和 Spring AI Alibaba 的优劣"

回答要点：概念同源但生态不同。Python 适合快速原型，Java 适合生产治理。详见 与 LangGraph 对照总结。

Q4: "Agent 上线后效果不好，你怎么排查？"

回答要点：从可观测性出发——查 Step Log 看哪一步偏离预期，查 Trace 看哪个工具调用异常，回放 Prompt 验证模型路径。详见 观测与调试。

常见坑与避坑

pitfall agent 总结了 Agent 系统设计中最常见的五个坑：

坑表现避坑方法
step budget pitfall Agent 无限循环、Token 暴增 设 maxSteps + maxExecutionTime
工具无鉴权 Agent 可直接调用删除 API 工具白名单 + 分级审批
上下文无限堆积 长对话后 LLM 质量急剧下降 滑动窗口 + 摘要压缩
单一模型 主模型限流后系统不可用 多模型降级链
无评测上线 上线后效果差，找不到优化方向 先建评测体系再上线
面试回答不推荐
  • "用过 LangChain 和 SAA"——只说用过不说对比
  • "Agent 就是不断调 API"——太肤浅
  • "我们直接用的开源项目"——没有深度
  • "没遇到过你说的坑"——没有工程经验
面试回答推荐
  • "SAA 的 Graph 和 LangGraph 的 StateGraph 概念对应..."——有深度对比
  • "Agent 的核心是 ReAct 循环和工具调用，但生产化需要..."——体系化理解
  • "我们遇到过一个坑，Agent 在第三步重复调用同一个工具..."——有实际经验
  • "评测是我们上线前的最后一道关..."——有工程规范

常见问题（FAQ）

Agent 面试题怎么准备？
三步：1) 掌握 STARS 框架（场景→选型→架构→容错→评测）；2) 准备一个真实项目经历（CorpAssist S3 就很好）；3) 能口述一个完整系统设计的 30 秒版本。建议提前演练三次。
面试官问"用过哪些 Agent 框架"怎么答？
不要只说名字。框架名 + 场景 + 对比：比如"用过 LangGraph 做原型验证、SAA 做生产上线。两者的核心概念其实是对应的——StateGraph 对应 SAA 的 Graph，只是 API 风格不同。"
没有实际 Agent 上线经验怎么办？
完成本课的 CorpAssist 实战项目（advanced-04-corpassist-agent）就有了。建议在简历中写"基于 Spring AI Alibaba 设计并实现企业办公 Agent 系统（CorpAssist S3）"。
面试频率最高的一个问题是什么？
"请设计一个 Agent 系统处理 X"（X 可以是工单、客服、审批等）。所有面试官关注的是：你是否有架构思维，还是只会调 API。用 STARS 框架回答可以系统展示你的设计能力。

本章结论

  • STARS 框架：Scene → Tech Stack → Architecture → Resilience → Scoring 逐层答题。
  • 常见坑：无步数预算、工具无鉴权、上下文无限堆积——面试中能识别避坑是关键。
  • 30 秒口述：准备好一个完整的 Agent 系统设计口述稿。
  • 全景展示：能画出通义栈四层架构（模型→框架→治理→观测）。
  • 实战经验：CorpAssist S3 项目是面试中最好的实例。
