为什么要学本章

面试官常问：「既然 Python LangGraph 生态更强，CorpAssist S3 为何用 Java？」答案不在语言优劣，而在治理、SLA 与双栈分工。langgraph mapping 把 Python StateGraph 与 SAA saa graph orchestration / react agent 逐项对齐，并与 agent-orchestration-java 的生产编排章衔接。

学完你能
  • 用六维表（编排、工具、模型、观测、治理、场景）对比 LangGraph 与 SAA
  • 说明 Python 实验栈与 Java 生产栈的边界与 OpenAPI 契约
  • 估算 DSL/Tool 从 LangGraph 迁到 SAA 的人日与风险点
  • 60 秒内答清「为什么不全用 Python」而不贬低任一堆栈
本章先记住 3 件事
  • 同构状态机：LangGraph checkpoint ≈ SAA Graph Redis checkpoint。
  • 实验在 Python，治理在 Java：tongyi dual stack 是分工不是二选一。
  • 话术讲 SLA：SSO、审计、Admin、NS2 HITL——Java 轨主场。

双栈边界

dual stack policy 规定：Python LangGraph 服务不得持有生产写 Token；Java SAA 为 S3 唯一写路径。Python 输出：DSL 版本、评测报告、Prompt diff。

LangGraph 能否直接部署为 CorpAssist 生产 Agent？

技术上可以，但违背架构：缺少 Spring Security SSO、统一 Advisor 链、Micrometer 与 Admin 治理。若短期 PoC，须标注 non-prod 且写操作 Mock。

选型话术

面试 60 秒结构：场景 → 约束 → 双栈分工 → 一条坑。避免贬低 Python 或夸大 Java。

开场（10s）

「CorpAssist S3 是多步办公 Agent，要接 SSO、审计与 HITL，在线生产在 Java SAA。」

双栈（25s）

「Python LangGraph 做 Prompt/Graph 实验与 golden；固化 DSL 后 Java 上线，DashScope 与 Tool OpenAPI 对齐。」

收尾（25s）

「选型看治理与 SLA 而非语言。我们踩过步数偏航坑，用 StepBudget + Trace 解决。」

刁钻问题话术要点
LangGraph 更新快，Java 跟得上吗？实验层跟 Python；生产跟 DSL 契约与 LTS BOM，不追 every release
为何不用 Dify 一把梭？NS2 审批与自定义 Tool 审计；Admin DSL + Git 更适合企业 CI
ReAct 还是 Graph？≤10 步动态用 ReactAgent；固定合规路径用 Graph + HITL
CorpAssist · 面试 defense 候选人画六维表并指出「观测、治理」两维 Java 领先；同时承认 Python 在批量仿真上的速度优势——体现 balanced 架构观。
和 advanced-01 面试章有何分工？
advanced-01 聚焦 S3 五连问与 STAR；本章聚焦 LangGraph↔SAA 对照与选型话术，可合并背诵。
LangGraph 的 checkpointer 与 SAA Redis checkpoint 字段如何对齐？
至少对齐：thread_id（= sessionId）、tenant_id、graph_version、last_node。迁移时写对照表，避免 Java 恢复 Python 实验状态导致 Tool 上下文错乱。

面试快问快答

LangGraph StateGraph 和 SAA Graph 是不是一回事？

答：概念同构：状态、节点、条件边、checkpoint。实现不同：LangGraph 在 Python runtime；SAA Graph 由 DSL + Spring 容器驱动，并集成 Admin、Advisor 与企业 Tool Gateway。

如果团队只会 Python，你会怎么推 CorpAssist 架构？

答：保留 Python 做实验与索引；引入 Java 薄层承载 SSO、写 Tool 与 SLA。用 OpenAPI 契约与 golden 轨迹降低协作摩擦，而不是强行全栈 Python 上线。
