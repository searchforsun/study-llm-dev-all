学完你能

  • 用 Micrometer + OpenTelemetry 导出 DashScope 调用 Span，并接入现有 APM
  • 设计 Prompt 回放存储：保留 messages 快照、模型参数与 Tool 结果，满足审计与复现

本章先记住 3 件事
  • 一条 trace 管到底：从 BFF 到 Tool 到 DashScope 同一 trace_id。
  • 日志给机器，回放给人：JSON 日志检索；回放 UI 看完整对话链。
  • 脱敏默认开：工号、金额、发票号 hash 后再存。

日志：结构化字段与级别

传统「打印 prompt 字符串」无法检索。S3 服务应输出 JSON 行日志，字段固定，便于 ELK / SLS 聚合：按 app=corpassist-s3、tool=roomBooking、outcome=error 过滤。

CorpAssist · S3 · 订会议室失败 运维在 SLS 搜 trace_id=7f3a...，看到顺序：intent=BOOK_ROOM → roomAvailability OK → roomBooking ERROR(OA 409 冲突) → LLM 仍回复「已预订」。据此修复：Tool ERROR 须约束模型不得编造成功。

java
@Slf4j
@Component
public class S3ObservabilityAdvisor implements CallAroundAdvisor {
  @Override
  public AdvisedResponse aroundCall(AdvisedRequest req, CallAroundAdvisorChain chain) {
    long start = System.currentTimeMillis();
    try {
      AdvisedResponse resp = chain.nextAroundCall(req);
      log.info("{}", Map.of(
          "trace_id", TraceContext.traceId(),
          "span", "llm.call",
          "model", req.adviseContext().getOrDefault("model", "unknown"),
          "latency_ms", System.currentTimeMillis() - start,
          "outcome", "ok"));
      return resp;
    } catch (Exception e) {
      log.warn("{}", Map.of("trace_id", TraceContext.traceId(), "outcome", "error",
          "error", e.getClass().getSimpleName()));
      throw e;
    }
  }
}
不推荐
  • INFO 全量打印 user prompt 明文
  • 每个 Tool 自定义 log 格式
  • 无 trace_id，网关与 Agent 日志对不上
推荐
  • MDC 注入 trace_id / session_id
  • 错误日志带 tool_name + error_code
  • 敏感字段走脱敏器再入 log

Trace：Span 模型与 DashScope 出口

distributed trace 把一次 S3 请求拆成 Span 树：HTTP in → intent LLM → tool.roomAvailability → tool.roomBooking → final LLM。Spring AI Alibaba 与 Micrometer Observation 集成后，可导出到 Jaeger / 阿里云 ARMS。

延迟P95 > 8s 持续 5m查 DashScope + Tool 慢 Span
错误率tool.error > 5%对接 OA/费控值班
成本tokens/req P99 飙升查是否陷入多轮 clarify 循环
先想 10 秒：采样 10% 时如何保证 badcase 必留 trace？

badcase 上报接口带 trace_id 强制 recording=true；或对 outcome=error 全量采样。Prompt 回放存储与 trace_id 关联即可补全未采样链路的上下文。

日志级别约定（S3）

级别场景示例
INFO正常 LLM/Tool 完成结构化 JSON 单行
WARN可恢复降级、Tool 4xxturbo 切换、OA 业务拒绝
ERROR未捕获异常、写 Tool 5xx带 trace_id，触发告警
DEBUG仅 stagingAdvisor 链顺序、cache miss 原因

生产禁止 DEBUG 打印完整 prompt；需要内容级排障走 Prompt 回放权限通道。

Prompt 回放：存储、检索与复现

prompt replay 在合规存储中保留每次 LLM 调用的 messages、generation 参数、Tool call/result，供研发在 Admin 调试台或内部 UI「一键复现」。这是 S3 面试高频：如何证明不是模型随机胡答，而是 Tool 返回了错误数据。

json
{
  "trace_id": "abc123",
  "session_id": "s3-emp-E1024",
  "recorded_at": "2026-06-03T10:15:00Z",
  "turns": [
    {
      "span": "llm.intent",
      "model": "qwen-plus",
      "messages": [{"role": "user", "content": "订明天下午三点小会议室"}],
      "completion": {"tool_calls": [{"name": "roomAvailability", "args": {"date": "2026-06-04"}}]}
    },
    {
      "span": "tool.roomAvailability",
      "result": {"rooms": [{"id": "R-301", "free": true}]}
    }
  ],
  "redaction": "employee_id hashed"
}
能力实现要点S3 价值
写入Advisor 异步写 OSS / DB；失败不阻塞主链高可用
检索trace_id / session_id / 时间范围工单式排障
复现用快照 messages 调 staging 模型验证 Prompt 修复
保留期90 天；审计类 1 年合规
CorpAssist · badcase 飞轮 客服标记「报销 Agent 答错税率」→ 系统带 trace_id 进入回放队列 → 研发在 staging 复现 → 修 Admin Prompt v1.2.1 → Eval 回归 → 关闭工单。与 NS3 成本/质量权衡叙事一致。
合规红线
  • 回放库与生产 DB 分库；研发只读 staging 复现权限。
  • 发票影像 URL 存签名短期链接，不进 messages 明文。
  • 导出 replay 包需审批水印。

与 Admin Studio 调试台联动

Prompt 回放 record 可 deep-link 到 Admin：携带 trace_id 打开对应会话，运营查看「用户原话 + Tool 结果 + 模型答复」时间线，研发在 staging 一键「用此快照重放」。缩短 badcase 从工单到 Prompt 发布的周期。

Micrometer 与 ARMS / Jaeger 如何选型？
已用阿里云栈选 ARMS OTLP 出口；多云或自建选 Jaeger/Tempo。关键是 trace_id 进 MDC，与 Spring AI observations 开关一致，避免双轨 instrumentation。

通义栈视角：观测与 Prompt 回放

S3 Agent 排障第一入口是 Trace + Prompt 回放，而非翻日志猜 prompt。Java Micrometer 与 Python OpenTelemetry 共用 W3C trace_id；Prompt 回放 JSON schema 双栈同构，便于 Python 实验后在 Java 生产复现。

不推荐
  • Java / Python 各一套 trace 字段，Grafana 无法关联
  • Prompt 回放存明文 PII，无脱敏与 TTL
  • 只在 error 时采样，步数偏航 badcase 抓不到
推荐
  • BFF 注入 traceparent，双栈 span 可串联
  • messages + Tool 快照脱敏存储，trace 驱动复现
  • 步数 / token / tool error 指标 + error 全采样

本章结论

  • 日志：JSON 固定字段 + MDC trace_id；Tool/LLM 分 span 记 outcome 与 tokens。
  • Trace：Micrometer + OTLP；P95/错误率/ token 告警；error 全采样。
  • Prompt 回放：messages + Tool 快照脱敏存储；trace 驱动 badcase 复现与 Prompt 迭代。

面试问答

用户说 Agent 乱答，你如何在一个小时内定位是模型还是 Tool 问题？

答：要 trace_id → SLS 查结构化日志 → Trace 看 tool.roomBooking Span 是否 error → Prompt 回放看 Tool result 是否已返回失败但 LLM 仍生成成功话术 → 若是后者修 System 约束；若是前者修 OA 集成。

Prompt 回放会不会泄露员工报销明细？

答：写入前脱敏工号/金额/发票号；影像用短期签名 URL；回放 UI 权限与审计日志；staging 复现不用生产 Key。
