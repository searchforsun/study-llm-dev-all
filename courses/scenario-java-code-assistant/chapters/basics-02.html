为什么学这章

代码助手的质量取决于上下文的质量——LLM 需要看到足够的代码才能生成准确的建议。但窗口有限（4K-128K），如何在有限的 Token 预算中选择最相关的文件、代码段和符号表，是代码助手的核心技术挑战。

学完后能做什么

计算和管理 Token 窗口预算；实现基于 LSP 符号表的文件选择策略；配置 Spring AI 的上下文管理；优化长上下文场景下的补全和问答质量。

本章要点速记

1) Token 预算分配：系统 Prompt 20% + 用户输入 10% + 代码上下文 60% + 历史 10%；2) 文件选择策略：最近打开的文件（MRU）+ LSP 符号引用文件（按引用频率排序）；3) 符号表包含类名/方法签名/变量类型，比全文检索更精确；4) Spring AI 的 ChatMemory + 自定义 DocumentRetriever 管理上下文；5) CorpAssist 的实验结论：包含 3-5 个相关文件 + 符号表是最优上下文组合，过多文件引入噪音降低准确率。

Token 窗口预算

每个代码助手的请求都要在有限的 Token 窗口内分配。合理分配各个部分的 Token 预算是保证生成质量的关键。

    "用户输入" : 800
    "当前文件" : 2400
    "相关文件(3个)" : 2400
    "符号表" : 400
    "对话历史" : 400
java
// Token 预算管理器
@Component
public class TokenBudgetManager {

    private static final int MAX_TOKENS = 8192;  // 8K 窗口

    // 各部分的预算百分比
    private static final Map<String, Double> BUDGET_RATIO = Map.of(
        "system", 0.20,     // 1600 Tokens
        "query", 0.10,      // 800 Tokens
        "currentFile", 0.30, // 2400 Tokens
        "relatedFiles", 0.30, // 2400 Tokens（最多 3 个文件）
        "symbolTable", 0.05,  // 400 Tokens
        "history", 0.05       // 400 Tokens
    );

    public TokenBudget allocate(String query, String currentFile,
                                  List<String> relatedFiles,
                                  Map<String, List<Symbol>> symbols) {
        TokenBudget budget = new TokenBudget();

        // System Prompt：固定内容，必须完整
        budget.setSystem(reserveTokens("system", "系统角色定义..."));

        // 用户输入：截断到预算
        budget.setQuery(truncateToBudget(query, "query"));

        // 当前文件：核心上下文，尽量完整
        budget.setCurrentFile(truncateToBudget(currentFile, "currentFile"));

        // 相关文件：按优先级从高到低截断
        int fileBudget = getTokenLimit("relatedFiles");
        for (String file : relatedFiles) {
            if (fileBudget <= 0) break;
            int fileTokens = estimateTokens(file);
            int allowed = Math.min(fileTokens, fileBudget / relatedFiles.size());
            budget.addRelatedFile(truncate(file, allowed));
            fileBudget -= allowed;
        }

        // 符号表：精简到仅包含引用的符号
        budget.setSymbolTable(compressSymbols(symbols, "symbolTable"));

        return budget;
    }

    public boolean isWithinBudget(TokenBudget budget) {
        return budget.totalTokens() <= MAX_TOKENS;
    }

    private String truncateToBudget(String content, String section) {
        int limit = getTokenLimit(section);
        if (estimateTokens(content) <= limit) return content;
        // 智能截断：保留开头和结尾
        int head = limit / 2;
        int tail = limit - head;
        return content.substring(0, charCount(head))
            + "\n// ... (截断 " + (estimateTokens(content) - limit) + " tokens) ...\n"
            + content.substring(content.length() - charCount(tail));
    }
}

文件选择策略

代码库可能包含数百个文件，但一次代码补全只需要 3-5 个最相关的文件。选择合适的文件是上下文工程的核心。

java
// 基于 MRU + LSP 符号引用的文件选择器
@Component
public class FileSelector {

    // 最近打开的文件（MRU）缓存
    private final Cache<String, LinkedHashSet<String>> mruCache;

    // LSP 客户端（用于获取引用信息）
    private final LspClient lspClient;

    public List<SelectedFile> selectFiles(String projectPath,
                                           String currentFile,
                                           int cursorLine) {
        // 1. 获取当前文件的符号定义和引用
        List<Symbol> currentSymbols = lspClient
            .getDocumentSymbols(projectPath, currentFile);

        // 2. 获取光标位置相关符号的引用文件
        Set<String> referencedFiles = new LinkedHashSet<>();
        for (Symbol symbol : currentSymbols) {
            if (symbol.getLine() <= cursorLine
                && symbol.getEndLine() >= cursorLine) {
                // 只取光标所在符号的引用
                List<Location> refs = lspClient
                    .findReferences(projectPath, currentFile, symbol.getName());
                refs.stream()
                    .map(Location::getFilePath)
                    .filter(f -> !f.equals(currentFile))
                    .forEach(referencedFiles::add);
            }
        }

        // 3. MRU 补充：如果引用文件不够，从最近打开的文件补充
        LinkedHashSet<String> mru = mruCache.get(projectPath,
            k -> new LinkedHashSet<>());
        if (referencedFiles.size() < 3) {
            for (String file : mru) {
                if (!file.equals(currentFile)) {
                    referencedFiles.add(file);
                    if (referencedFiles.size() >= 3) break;
                }
            }
        }

        // 4. 按引用频率排序
        return referencedFiles.stream()
            .map(f -> new SelectedFile(f, countReferences(f, currentSymbols)))
            .sorted((a, b) -> Integer.compare(b.referenceCount(),
                a.referenceCount()))
            .limit(3)
            .toList();
    }
}

符号表与 LSP

符号表是对代码结构的抽象表示，包含类、方法、变量、接口等定义的名称、类型、位置信息。LSP（Language Server Protocol）提供标准化的符号获取接口。

java
// LSP 符号管理
@Component
public class SymbolManager {

    private final LspClient lspClient;

    public SymbolTable buildSymbolTable(String projectPath,
                                         List<String> files) {
        SymbolTable table = new SymbolTable();
        for (String file : files) {
            List<Symbol> symbols = lspClient
                .getDocumentSymbols(projectPath, file);
            for (Symbol symbol : symbols) {
                table.addSymbol(file, symbol);
                // 递归获取子符号（方法的参数、类的字段）
                addChildren(table, file, symbol);
            }
        }
        return table;
    }

    // 将符号表压缩为 Token 友好的格式
    public String compressForPrompt(SymbolTable table) {
        StringBuilder sb = new StringBuilder();
        sb.append("符号表:\n");
        for (Map.Entry<String, List<Symbol>> entry
             : table.getSymbolsByFile().entrySet()) {
            sb.append("  ").append(entry.getKey()).append(":\n");
            for (Symbol s : entry.getValue()) {
                sb.append("    ")
                    .append(s.getKind())      // CLASS/METHOD/FIELD
                    .append(" ")
                    .append(s.getName())
                    .append(": ")
                    .append(s.getDetail())    // 返回类型/参数
                    .append("\n");
            }
        }
        return sb.toString();
    }
}

// 有符号表:

//    METHOD getUser(String): User, METHOD saveUser(User): void

Spring AI 上下文管理

Spring AI 通过 ChatMemory 和 DocumentRetriever 管理对话上下文和文档上下文。合理的配置可以在有限窗口内最大化信息密度。

java
// Spring AI 上下文管理配置
@Configuration
public class ContextManagementConfig {

    @Bean
    public ChatMemory codeChatMemory() {
        return ChatMemory.builder()
            .strategy(ChatMemory.Strategy.SLIDING_WINDOW)
            .windowSize(10)              // 保留最近 10 条消息
            .maxTokenUsage(2048)         // 历史消息最多 2048 Tokens
            .build();
    }

    @Bean
    public ChatClient codeChatClient(ChatClient.Builder builder,
                                      ChatMemory chatMemory,
                                      List<ToolCallback> tools) {
        return builder
            .defaultSystem("""
                你是 CorpAssist 代码助手。你可以访问以下信息：
                1. 当前文件内容
                2. 相关的 3-5 个文件
                3. 符号表（类/方法/变量定义）
                4. 对话历史（最近 10 条）

                请基于以上上下文回答代码问题。
                回答必须包含具体代码示例和行号引用。
                如果你不确定，请说明"我不确定，建议查阅官方文档"。
                """)
            .chatMemory(chatMemory)
            .tools(tools)
            .build();
    }

    // 自定义 DocumentRetriever 管理代码上下文
    @Bean
    public DocumentRetriever codeDocumentRetriever(
            VectorStore vectorStore,
            FileSelector fileSelector) {
        return new DocumentRetriever() {
            @Override
            public List<Document> retrieve(String query,
                                             String projectPath,
                                             String currentFile) {
                // 1. LSP 符号引用检索（优先）
                List<SelectedFile> refFiles = fileSelector
                    .selectFiles(projectPath, currentFile, 0);
                // 2. 向量检索补充
                SearchRequest request = SearchRequest.builder()
                    .query(query)
                    .topK(5)
                    .filterExpression("project == '" + projectPath + "'")
                    .build();
                List<Document> vectorDocs = vectorStore
                    .similaritySearch(request);
                // 3. 合并去重
                return mergeAndDedup(refFiles, vectorDocs);
            }
        };
    }
}

面试可能问

代码补全时如何选择"当前光标位置"的相关上下文？

三阶段策略：1) 语法层面——使用 LSP 获取光标所在符号的定义和引用，选择直接相关的文件（准确率 82%）；2) 语义层面——如果语法信息不够（如新写代码还没有符号引用），使用向量检索找到语义相似的文件（准确率 78%）；3) 回退层面——使用 MRU 最近打开文件（准确率 65%）。CorpAssist 按照语法→语义→回退的顺序尝试，88% 的请求在语法层面就能找到足够的上下文，不需要回退。关键优化：在 LSP 获取引用时，按引用频率排序只取前 3 个文件，避免引入不相关的引用。
