为什么学这章

代码问答的核心是 RAG——从代码库中检索相关代码段，然后让 LLM 基于检索结果回答。与通用的文本 RAG 不同，代码 RAG 需要理解代码结构（AST）、支持多语言、提供引用跳转。本章深入 Spring AI 的 DocumentReader 体系和代码 RAG 的 Java 实现。

学完后能做什么

实现 AST 切片策略，用 Spring AI DocumentReader 解析代码文件；构建仓库索引流水线；支持 Java/Python/TypeScript 等多语言检索；实现检索结果的引用跳转。

本章要点速记

1) AST 切片按函数/类/接口边界切分，比固定行数切分准确率高 30%；2) Spring AI DocumentReader 可扩展，通过自定义 Reader 支持多种代码文件；3) 仓库索引包含代码 Embedding 和元数据（文件路径、语言、符号名）；4) 引用跳转通过 LSP 获取符号位置信息，在检索结果中直接返回行号。

AST 切片与 Spring AI DocumentReader

代码文件的切片策略直接影响检索效果。基于 AST（抽象语法树）的切片按代码的逻辑边界（函数、类、接口）切分，每个切片是一个语义完整的代码单元。

// 自定义代码 DocumentReader
@Component
public class CodeDocumentReader implements DocumentReader {

    @Override
    public List<Document> read(String filePath) {
        String language = detectLanguage(filePath);
        String content = readFile(filePath);

        // 使用对应语言的 AST 解析器
        CodeParser parser = getParser(language);
        List<AstNode> nodes = parser.parse(content);

        // 按 AST 节点切片
        List<Document> documents = new ArrayList<>();
        for (AstNode node : nodes) {
            Document doc = new Document(
                node.getCode(),  // 切片内容
                Map.of(
                    "filePath", filePath,
                    "language", language,
                    "symbolName", node.getName(),
                    "symbolKind", node.getKind(),  // CLASS/METHOD/FIELD
                    "startLine", node.getStartLine(),
                    "endLine", node.getEndLine(),
                    "signature", node.getSignature(),
                    "project", extractProjectName(filePath)
                )
            );
            documents.add(doc);
        }
        return documents;
    }

仓库索引构建

代码索引流水线需要处理仓库的初始全量索引和日常增量更新。Spring Scheduling 实现定期索引更新。

java
// 代码仓库索引构建器
@Component
public class CodeIndexBuilder {

    private final GitService gitService;
    private final CodeDocumentReader documentReader;
    private final VectorStore vectorStore;
    private final EmbeddingModel embeddingModel;

    @Scheduled(cron = "0 0 2 * * ?")  // 每天凌晨 2 点全量索引
    public void fullIndex() {
        List<String> projects = gitService.listProjects();
        for (String project : projects) {
            try {
                indexProject(project);
            } catch (Exception e) {
                log.error("全量索引失败: {}", project, e);
            }
        }
    }

    @Scheduled(fixedRate = 300000)  // 每 5 分钟增量索引
    public void incrementalIndex() {
        List<GitChange> changes = gitService.getRecentChanges(5);
        for (GitChange change : changes) {
            if (change.getType() == GitChange.Type.DELETE) {
                // 从向量库删除
                vectorStore.delete(
                    filterExp -> filterExp.eq("filePath", change.getFilePath())
                );
            } else {
                // 更新或新增
                List<Document> docs = documentReader.read(change.getFilePath());
                // 先删除旧的再添加新的
                vectorStore.delete(
                    filterExp -> filterExp.eq("filePath", change.getFilePath())
                );
                vectorStore.add(docs);
            }
        }
    }

    private void indexProject(String project) {
        List<String> files = gitService.listFiles(project);
        for (String file : files) {
            List<Document> docs = documentReader.read(file);
            vectorStore.add(docs);
        }
        log.info("全量索引完成: {}, 文件数: {}", project, files.size());
    }
}

多语言支持与引用跳转

企业代码库通常包含多种语言。CorpAssist 的代码 RAG 支持 Java/Python/TypeScript/Go/SQL 等 8 种语言。

java
// 多语言解析器工厂
@Component
public class CodeParserFactory {

    private final Map<String, CodeParser> parsers = Map.of(
        "java", new JavaParser(),
        "py", new PythonParser(),
        "ts", new TypeScriptParser(),
        "js", new JavaScriptParser(),
        "go", new GoParser(),
        "sql", new SqlParser(),
        "kt", new KotlinParser(),
        "xml", new XmlParser()
    );

    public CodeParser getParser(String extension) {
        CodeParser parser = parsers.get(extension);
        if (parser == null) {
            return new GenericParser();  // 不支持的语言用通用文本解析
        }
        return parser;
    }

    public List<String> getSupportedExtensions() {
        return new ArrayList<>(parsers.keySet());
    }
}

// 检索结果包含引用跳转信息
@Data
@Builder
public class SearchResult {
    private String codeSnippet;
    private String filePath;
    private int startLine;
    private int endLine;
    private String symbolName;
    private String language;
    private double score;

    // 引用跳转 URL（用于 IDE 插件）
    public String getJumpUrl() {
        return String.format("file://%s#L%d-L%d", filePath, startLine, endLine);
    }

    // 代码引用格式
    public String formatCitation() {
        return String.format("【%s:%d-%d】\n```%s\n%s\n```",
            filePath, startLine, endLine, language, codeSnippet);
    }
}

CorpAssist 场景

开发者提问"UserService 的 getUser 方法是怎么实现的？" 代码 RAG 检索流程：1) 向量检索找到相似度最高的切片（预期是 UserService.java 的 getUser 方法切片）；2) 元数据过滤确认 language=java 且 project=user-service；3) 返回切片内容 + 引用跳转 file://UserService.java#L25-L45；4) LLM 基于切片内容生成回答。整个过程约 800ms。如果检索不到准确的切片（比如 getUser 方法太长被切成了多段），回退策略是从向量库检索文件的整体切片，然后使用正则提取目标方法。

本章结论

  • AST 混合切片（类+方法）检索准确率 88%，优于固定行数切片的 62%
  • Spring AI DocumentReader 可扩展，通过自定义 Reader 支持 8+ 种语言
  • 全量索引（每日凌晨）+ 增量索引（每 5 分钟）通过 Spring Scheduling 实现
  • 引用跳转通过 LSP 获取符号位置，在检索结果中直接提供 file:// URL
  • CorpAssist 检索 + 生成流程约 800ms，回退策略保障检索不到时的可用性

面试可能问

AST 切片和固定行数切片的效果差异有多大？为什么？

差异约 26%（88% vs 62%）。原因：固定行数切片会切断代码的逻辑边界。比如一个 80 行的 getUser 方法被 50 行限制切成两段，用户问"getUser 的逻辑"时只检索到上半段（变量声明部分），缺少下半段（核心逻辑部分），LLM 无法给出完整回答。AST 切片按方法边界切分，确保 getUser 作为一个整体进入检索，LLM 能看到完整的逻辑。CorpAssist 实践中还有"重叠切片"策略——每个切片包含前后 5 行相邻代码作为上下文，弥补 AST 边界处可能丢失的关联信息，准确率进一步提升到 91%。
