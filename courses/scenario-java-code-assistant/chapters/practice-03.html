约 30 分钟 · 阶段：Java 栈实战 · 场景：S4 代码助手

为什么学这章

代码补全的 Prompt 与对话模型完全不同，FIM（Fill-in-the-Middle）格式要求模型根据前后文填充中间代码。本章深入 FIM 格式、上下文精确拼装、风格约束、多文件补全策略。

学完后能做什么

构造 FIM 格式的补全 Prompt；精确选择光标前后的上下文；配置代码风格约束（缩进/命名/注释风格）；实现跨文件的多文件补全。

本章要点速记

1) FIM 格式：prefix（光标前代码） + suffix（光标后代码） + middle（LLM 填充）；2) 上下文选择规则：光标前 300 Tokens + 光标后 200 Tokens 是最优组合；3) 风格约束通过 infile 示例（同一文件已有的代码风格）隐式传递；4) 多文件补全需要额外注入当前打开的其他文件中相关符号的定义。

// FIM Prompt 构造器
@Component
public class FimPromptBuilder {

    private static final int PREFIX_TOKENS = 300;
    private static final int SUFFIX_TOKENS = 200;
    private static final int MAX_LINE_LENGTH = 120;

    public FimPrompt build(String fileContent, int cursorLine,
                            int cursorColumn, String language) {
        // 1. 按光标位置拆分 prefix 和 suffix
        String[] lines = fileContent.split("\n", -1);

        StringBuilder prefix = new StringBuilder();
        int prefixChars = 0;
        int targetChars = estimateChars(PREFIX_TOKENS);

        // 从光标前开始，向前取目标 Tokens 的内容
        for (int i = Math.max(0, cursorLine - 50); i < cursorLine; i++) {
            String line = lines[i] + "\n";
            prefixChars += line.length();
            if (prefixChars > targetChars) {
                prefix.append("// ... 上下文截断 ...\n");
                break;
            }
            prefix.append(line);
        }
        // 光标位置的前半行
        prefix.append(lines[cursorLine], 0, cursorColumn);

        // 取光标后的内容
        StringBuilder suffix = new StringBuilder();
        int suffixChars = 0;
        int targetSuffix = estimateChars(SUFFIX_TOKENS);

        // 光标位置的后半行
        if (cursorColumn < lines[cursorLine].length()) {
            suffix.append(lines[cursorLine].substring(cursorColumn)).append("\n");
            suffixChars += suffix.length();
        }
        for (int i = cursorLine + 1; i < Math.min(cursorLine + 30, lines.length); i++) {
            String line = lines[i] + "\n";
            suffixChars += line.length();
            if (suffixChars > targetSuffix) {
                suffix.append("// ... 上下文截断 ...\n");
                break;
            }
            suffix.append(line);
        }

        return new FimPrompt(prefix.toString(), suffix.toString());
    }

    public String formatForModel(FimPrompt fim) {
        // CodeQwen FIM 格式
        return "<fim_prefix>" + fim.prefix()
            + "<fim_suffix>" + fim.suffix()
            + "<fim_middle>";
    }
}

上下文拼装与风格约束

// 上下文拼装器（含风格约束）
@Component
public class ContextAssembler {

    public String assembleCompletionContext(String currentFile,
                                             int cursorLine,
                                             List<RelatedFile> relatedFiles,
                                             String language) {
        StringBuilder ctx = new StringBuilder();

        // 1. 风格参考（infile 示例）：当前文件的风格模式
        ctx.append("// 当前代码风格参考:\n");
        ctx.append(extractStylePattern(currentFile, language));
        ctx.append("\n");

        // 2. 相关符号定义（多文件支持）
        if (!relatedFiles.isEmpty()) {
            ctx.append("// 引用符号定义:\n");
            for (RelatedFile rf : relatedFiles) {
                ctx.append("//   ").append(rf.getFilePath())
                    .append(": ").append(rf.getSymbol()).append("\n");
            }
            ctx.append("\n");
        }

        // 3. FIM Prompt
        FimPrompt fim = new FimPromptBuilder()
            .build(currentFile, cursorLine, 0, language);
        ctx.append(fim.formatForModel(fim));

        return ctx.toString();
    }

    private String extractStylePattern(String code, String language) {
        // 检测代码的缩进风格（2空格/4空格/tab）
        // 命名风格（camelCase/snake_case）
        // 注释风格（// / /** / --）
        // 返回检测结果作为隐式风格约束
        int spaces = countLeadingSpaces(code);
        boolean usesTab = code.contains("\t");
        return String.format(
            "缩进: %s %d, 命名: camelCase",
            usesTab ? "tab" : "空格",
            usesTab ? 1 : spaces
        );
    }
}

// 多文件补全管理器
@Component
public class MultiFileCompletion {

    private final LspClient lspClient;

    public CompletionResult complete(String projectPath,
                                      String currentFile,
                                      int cursorLine,
                                      List<String> openFiles) {
        // 1. 获取光标位置符号
        Symbol cursorSymbol = lspClient
            .getSymbolAt(projectPath, currentFile, cursorLine);

        // 2. 从打开的文件中找到引用了该符号的文件
        List<RelatedFile> relatedFiles = new ArrayList<>();
        for (String openFile : openFiles) {
            if (openFile.equals(currentFile)) continue;
            List<Symbol> symbols = lspClient.getDocumentSymbols(projectPath, openFile);
            for (Symbol s : symbols) {
                if (s.getName().equals(cursorSymbol.getName())) {
                    relatedFiles.add(new RelatedFile(openFile, s.getName(),
                        s.getCode()));
                    break;
                }
            }
        }

        // 3. 构建多文件上下文 + FIM Prompt
        String context = new ContextAssembler()
            .assembleCompletionContext(currentFile, cursorLine,
                relatedFiles, detectLanguage(currentFile));

        // 4. 调用模型
        String completion = llmService.complete(context);

        return new CompletionResult(completion, relatedFiles);
    }
}

本章结论

  • FIM 格式（prefix+suffix+middle）是代码补全的标准 Prompt 格式，比对话格式产生纯代码而非自然语言
  • 上下文选择：前缀 300 Tokens + 后缀 200 Tokens 是最优组合
  • 风格约束通过 infile 示例隐式传递，比显式指令更准确
  • 多文件补全通过 LSP 符号引用注入相关文件中光标位置符号的定义
