为什么学这章

代码助手项目从开发到交付需要完整的交付物：索引方案文档、Demo 脚本、安全报告、上线清单。这是你向业务方展示项目价值的关键。

学完后能做什么

编写代码索引方案文档；准备 Demo 演示脚本；生成安全检测报告；制定上线检查清单。

本章要点速记

1) 索引方案文档包含：切片策略、Embedding 模型、向量库 Schema、更新频率；2) Demo 脚本覆盖 3 个场景（补全/问答/Review）+ 1 个异常场景（超时降级）；3) 安全报告包含：静态规则结果、AI 审查结果、修复建议；4) 上线清单 10 项：功能正确性 × 3 + 性能 × 2 + 安全 × 3 + 运维 × 2。

// Demo 脚本片段
@Component
public class DemoScript {

    public String[] scenarios() {
        return new String[]{
            // 场景 1: 代码补全
            "输入: 在 Java 文件中输入 'public String getUser'",
            "预期: 光标后出现补全建议，包含方法体",
            "验收: 补全延迟 < 500ms",

            // 场景 2: 代码问答
            "输入: 'UserService.getUser 返回什么类型?'",
            "预期: 回答包含类型信息 + 引用代码行",
            "验收: 回答延迟 < 5s，引用跳转可用",

            // 场景 3: Code Review
            "输入: 提交包含 SQL 查询的 PR",
            "预期: Review 评论标注 SQL 注入风险",
            "验收: 30s 内生成评论，准确率 > 70%",

            // 场景 4: 异常降级
            "输入: 断开网络后提问",
            "预期: 显示降级模式提示",
            "验收: 不崩溃，显示友好提示"
        };
    }
}

安全报告与上线清单

CorpAssist 交付

代码助手项目上线交付物完整清单：1) 索引方案文档（含切片策略和 Embedding 参数）；2) Demo 演示视频（4 个场景，共 8 分钟）；3) 安全检测报告（覆盖 12 个漏洞模式）；4) 上线检查清单（10 项，全部通过 ✓）；5) 运维手册（部署/扩容/告警）。业务方验收后签字，进入正式运营阶段。

本章结论

  • 索引方案文档包含 5 个关键配置项：切片/Embedding/向量库/调度/权限
  • Demo 脚本覆盖 4 个场景（3 正常 + 1 异常），每条含输入/预期/验收
  • 安全报告记录静态规则和 AI 审查的结果及修复建议
  • 上线清单 10 项覆盖功能/性能/安全/运维四个维度
