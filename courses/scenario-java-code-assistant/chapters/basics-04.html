静态规则（SpotBugs/Checkstyle）

静态规则工具扫描速度快、误报率低，是代码质量的第一道防线。

java
// SpotBugs + Checkstyle 集成
@Component
public class StaticCodeAnalyzer {

    private final SpotBugsEngine spotbugs;
    private final CheckstyleEngine checkstyle;

    public List<ReviewComment> analyze(String filePath, String code) {
        List<ReviewComment> comments = new ArrayList<>();

        // SpotBugs：检测 Bug 模式（NullPointer, 资源未关闭等）
        List<BugInstance> bugs = spotbugs.analyze(code);
        for (BugInstance bug : bugs) {
            comments.add(new ReviewComment(
                filePath, bug.getLine(), "BUG",
                bug.getPattern(), bug.getSeverity().toString()
            ));
        }

        // Checkstyle：检测代码风格和规范
        List<Violation> violations = checkstyle.analyze(code);
        for (Violation v : violations) {
            comments.add(new ReviewComment(
                filePath, v.getLine(), "STYLE",
                v.getMessage(), v.getSeverity().toString()
            ));
        }

        return comments;
    }
}

漏洞模式识别

LLM 可以识别静态工具难以检测的语义漏洞（如业务逻辑漏洞、敏感数据泄露）。使用 Few-shot Prompt 提供漏洞示例，让 LLM 进行模式匹配。

java
// AI 漏洞检测
@Component
public class AiVulnerabilityDetector {

    private final ChatClient chatClient;

    public List<Vulnerability> detectVulnerabilities(String code,
                                                       String language) {
        String response = chatClient.prompt()
            .user(u -> u.text("""
                请检查以下 {language} 代码中的安全漏洞。
                关注：SQL注入、XSS、硬编码密码、任意文件访问、权限绕过。

                漏洞示例（注入类）：
                "SELECT * FROM users WHERE id = " + userId  → SQL注入漏洞
                response.send(userInput)  → XSS漏洞
                password = "admin123"  → 硬编码密码

                代码：
                ```{language}
                {code}
                ```

                请按 JSON 格式输出：
                [{"type":"SQL_INJECTION","line":15,
                  "description":"存在SQL注入风险",
                  "severity":"HIGH","suggestion":"使用参数化查询"}]
                """)
                .param("language", language)
                .param("code", code)
            )
            .call()
            .content();

        return parseVulnerabilities(response);
    }
}

// 常见漏洞模式表
// | 模式 | 检测方法 | 严重级别 | 基准误报率 |
// | SQL注入 | 检测字符串拼接的 SQL 查询 | HIGH | 15% |
// | XSS | 检测未转义的用户输入输出 | HIGH | 10% |
// | 硬编码密码 | 检测字符串字面量中的密码关键字 | MEDIUM | 25% |
// | 路径遍历 | 检测未校验的文件路径参数 | HIGH | 20% |
// | 权限缺失 | 检测缺少 @PreAuthorize 的方法 | MEDIUM | 30% |

测试建议生成

基于代码上下文生成单元测试建议，包括测试框架选择、测试场景设计、边界值建议。

java
// 测试建议生成器
@Component
public class TestSuggestionGenerator {

    public TestSuggestions generate(String methodCode,
                                     String methodName,
                                     String className) {
        // 1. 解析方法的输入输出
        MethodSignature signature = parseSignature(methodCode);
        List<Parameter> params = signature.getParameters();

        // 2. 基于参数类型生成测试建议
        List<TestCase> testCases = new ArrayList<>();

        // 正常路径
        testCases.add(new TestCase(
            methodName + "_shouldSucceed_whenValidInput",
            "正常输入", buildValidParams(params)));

        // 空值路径
        for (Parameter p : params) {
            testCases.add(new TestCase(
                methodName + "_shouldHandle_when" + p.getName() + "IsNull",
                p.getName() + " 为空", Map.of(p.getName(), null)));
        }

        // 边界值
        for (Parameter p : params) {
            if ("int".equals(p.getType()) || "long".equals(p.getType())) {
                testCases.add(new TestCase(
                    methodName + "_shouldHandle_when" + p.getName() + "IsNegative",
                    p.getName() + " 为负数", Map.of(p.getName(), -1)));
                testCases.add(new TestCase(
                    methodName + "_shouldHandle_when" + p.getName() + "IsMax",
                    p.getName() + " 为最大值",
                    Map.of(p.getName(), Integer.MAX_VALUE)));
            }
        }

        return new TestSuggestions(testCases);
    }
}

CorpAssist 场景

开发者在 PR 中提交了一个新的 Controller 方法。CI 流水线自动触发代码质量检测：1) SpotBugs 检查到 unclosed PreparedStatement（严重：Bug）；2) Checkstyle 检查到方法名不符合驼峰命名（严重：Style）；3) AI 漏洞检测发现方法没有 @PreAuthorize 权限注解（严重：Security，置信度 75%）。三种检测合并后生成 Review 评论，标注每个问题的严重级别和修改建议。开发者根据评论修改后再次提交通过检测。整套流程 < 30 秒，每次 PR 平均发现 2.3 个问题。

  • 静态规则（SpotBugs/Checkstyle）准确率 95%+，作为代码质量第一道防线

  • 测试建议生成基于参数类型自动生成单元测试场景和边界值

  • CorpAssist CI 集成每次 PR 自动运行完整流水线，平均发现 2.3 个问题，耗时 < 30s
