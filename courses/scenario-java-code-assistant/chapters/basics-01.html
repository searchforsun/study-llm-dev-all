为什么学这章

代码助手是研发效能提升的关键工具，但正确衡量它的价值并不简单。本章从补全/问答/Review 三个场景出发，建立代码助手的效能指标体系，帮助你理解采纳率、节省时间、代码质量等核心指标，以及 CorpAssist 代码模块的技术边界。

学完后能做什么

区分代码补全、代码问答、Code Review 三种场景的技术差异；计算代码助手的采纳率和节省时间；解释 CorpAssist 代码模块的架构和边界；判断哪些研发场景适合用代码助手增强。

本章要点速记

1) 代码补全（Inline）用 FIM 格式，延迟 < 500ms；代码问答用 RAG 架构，延迟 < 5s；Code Review 用 Agent 多步推理，延迟 < 30s；2) 采纳率 = 用户接受建议数/总建议数，行业基准 20%-35%；3) CorpAssist 代码模块分为索引、检索、生成三层，各层独立部署可扩展；4) 技术边界：代码补全不适合跨文件上下文（单文件最优），代码问答适合库函数查询但复杂业务逻辑仍需人判断。

采纳率与节省时间

衡量代码助手 ROI 的核心指标是采纳率和节省时间。采纳率反映模型质量，节省时间反映业务价值。

java
// 采纳率追踪器
@Component
public class AdoptionTracker {

    private final MeterRegistry meterRegistry;
    private final Counter suggestionsCounter;
    private final Counter acceptedCounter;
    private final Timer acceptanceTimer;

    public AdoptionTracker(MeterRegistry meterRegistry) {
        this.meterRegistry = meterRegistry;
        this.suggestionsCounter = Counter.builder("code.suggestions.total")
            .register(meterRegistry);
        this.acceptedCounter = Counter.builder("code.suggestions.accepted")
            .register(meterRegistry);
        this.acceptanceTimer = Timer.builder("code.suggestions.acceptance.time")
            .publishPercentiles(0.5, 0.9, 0.99)
            .register(meterRegistry);
    }

    public void recordSuggestion(String userId, String language,
                                  boolean accepted, long latencyMs) {
        suggestionsCounter.increment();
        if (accepted) {
            acceptedCounter.increment();
        }
        // 按语言统计
        meterRegistry.counter("code.suggestions.by_language",
            "language", language, "accepted", String.valueOf(accepted))
            .increment();
        // 记录延迟
        acceptanceTimer.record(Duration.ofMillis(latencyMs));
    }

    public AdoptionReport getReport() {
        double total = suggestionsCounter.count();
        double accepted = acceptedCounter.count();
        double rate = total > 0 ? accepted / total * 100 : 0;
        return new AdoptionReport(total, accepted, rate,
            acceptanceTimer.getPercentile(0.5),
            acceptanceTimer.getPercentile(0.99));
    }

    // 使用示例
    // tracker.recordSuggestion("u123", "java", true, 320);

CorpAssist 场景

某 200 人研发团队接入 CorpAssist 代码助手后，进行为期 4 周的采纳率追踪。结果：日均建议 3,700 条，采纳率 28.6%（行业基准 20-35% 中上）；Java 开发者采纳率 28.5%，提示工程效果最好的 Python 团队 32.1%。按每条建议节省 15 秒计算，日均节省 3,700 × 28.6% × 15s ≈ 158 分钟 ≈ 2.6 小时。200 人团队整体研发效率提升约 2.8%。按平均年薪 40 万元计算，年化节省约 224 万元。采纳率提升 5% 意味着年化多节省 40 万元——这就是为什么 Prompt 优化能带来直接的财务回报。

CorpAssist 代码模块

CorpAssist 代码助手模块采用三层架构：索引层（代码仓库解析和 Embedding）、检索层（语义搜索和混合检索）、生成层（Prompt 拼装和 LLM 调用）。

java
// CorpAssist 代码模块配置
@Configuration
@EnableConfigurationProperties(CodeAssistantProperties.class)
public class CodeAssistantAutoConfiguration {

    @Bean
    public CodeIndexer codeIndexer(GitService gitService,
                                    DocumentParser documentParser,
                                    VectorStore vectorStore) {
        return new CodeIndexer(gitService, documentParser, vectorStore);
    }

    @Bean
    public CodeRetriever codeRetriever(VectorStore vectorStore,
                                        HybridSearchService hybridSearch) {
        return new CodeRetriever(vectorStore, hybridSearch);
    }

    @Bean
    public CodeGenerator codeGenerator(ChatClient.Builder builder,
                                        CodeRetriever retriever) {
        return new CodeGenerator(builder.build(), retriever);
    }
}

// 代码生成器
@Component
public class CodeGenerator {

    private final ChatClient chatClient;

    public String generate(String userQuery, String language,
                            List<CodeSnippet> context) {
        return chatClient.prompt()
            .user(u -> u.text("""
                用户问题：{query}
                编程语言：{language}
                相关代码上下文：
                {context}

                请根据上下文回答代码相关问题。
                回答必须包含具体代码示例和行号引用。
                """)
                .param("query", userQuery)
                .param("language", language)
                .param("context", formatContext(context))
            )
            .call()
            .content();
    }
}

Q: 什么时候应该用代码补全 vs 代码问答？

代码补全适合"我正在写这段代码，帮我续写"的场景——用户在编辑器中连续输入、需要即时建议。代码问答适合"这个函数怎么用"的场景——用户有明确的问题、需要从代码库中搜索答案。CorpAssist 的最佳实践是两者结合：编辑器内用补全（自动触发），侧边栏用问答（用户主动提问）。补全采纳率约 28%，问答用户满意度约 85%。

本章结论

  • 代码助手三大场景（补全/问答/Review）各有不同延迟要求和模型选择，需要分别设计
  • 采纳率 28.6%（Java）高于行业基准水平，每条建议节省约 15 秒
  • CorpAssist 代码模块三层架构（索引/检索/生成），各层独立部署
  • 技术边界决定了代码助手的合理使用范围：单文件补全最优，跨文件需预索引
  • 200 人团队接入代码助手后，年化节省约 224 万元，采纳率提升 5% 带来 40 万元额外收益

面试可能问

代码补全的采纳率 28.6%，和行业基准比怎么样？如何提升？

行业基准 20-35%，28.6% 属于中上水平。提升采纳率的方法：1) Prompt 优化——根据语言特性定制补全模板（Java 的 getter/setter 模式和 Python 的列表推导式完全不同）；2) 上下文窗口优化——实验发现当前文件 + 打开的其他文件的前 100 行是最佳上下文组合，更多文件反而引入噪音降低采纳率；3) 质量过滤——在后端增加一个快速校验（语法检查 + 长度过滤），低质量建议不展示给用户。CorpAssist 通过这些优化将 Java 采纳率从 22% 提升到 28.6%。
