本章要点速记

1) 采纳率追踪：按语言/场景/模型维度统计，结合 A/B 测试验证优化效果；2) 延迟对比：缓存命中时 15ms vs 未命中 320ms，小模型路由全局 P99 从 870ms 降至 420ms；3) 数据叙事公式：问题 + 行动 + 数据 + 业务价值；4) 面试话术：不说"做了优化"，说"将采纳率从 22% 提升到 28.6%，年节省 224 万元"。

采纳率追踪

java
// A/B 测试服务
@Component
public class AbTestService {

    private final Map<String, Double> experimentConfig = Map.of(
        "fim-context-300", 0.5,    // 50% 流量走新策略
        "model-route-v2", 0.3      // 30% 流量走新路由
    );

    public boolean isInExperiment(String userId, String experiment) {
        Double rate = experimentConfig.get(experiment);
        if (rate == null) return false;
        // 基于用户 ID 的哈希决定是否命中实验组
        int hash = Math.abs(userId.hashCode());
        return (hash % 100) / 100.0 < rate;
    }

    // A/B 测试报告
    public AbTestReport compare(String experiment, Duration duration) {
        // 对照组（未命中实验）
        List<Record> controlGroup = queryRecords(experiment, false, duration);
        // 实验组（命中实验）
        List<Record> treatmentGroup = queryRecords(experiment, true, duration);

        return new AbTestReport(
            experiment,
            calculateAdoptionRate(controlGroup),
            calculateAdoptionRate(treatmentGroup),
            calculateAvgLatency(controlGroup),
            calculateAvgLatency(treatmentGroup)
        );
    }
}

延迟对比与数据叙事

面试数据叙事

"我负责的代码助手项目将采纳率从 22% 提升到 28.6%（高于行业基准 20-35% 的中上水平）。核心做了三件事：1) 语义缓存命中 25% 的请求，延迟降至 15ms；2) 小模型路由将 45% 的简单请求分流到 qwen-turbo，全局 P99 从 870ms 降至 280ms；3) 延迟降低后采纳率自然提升——每降 100ms 采纳率升 1%。按 200 人团队计算，每条建议节省 15 秒、日均 3,700 条建议、采纳率 28.6%，年化节省约 224 万元。"

本章结论

  • 采纳率从 22% 提升到 28.6%，主要驱动力是延迟优化（每降 100ms 提 1%）
  • 延迟从 P99 870ms 降至 280ms（降幅 68%），三步优化各有贡献
  • A/B 测试验证每个优化措施的实际效果
  • 数据叙事框架：问题 + 行动 + 数据 + 业务价值
