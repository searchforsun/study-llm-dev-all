集成测试：Spring Boot 中的 RAG 评测

spring integration test 读 classpath JSONL 或 testcontainer 索引。与 Python 不同，Spring 测试更关注 Advisor 链组合的正确性而非 RAGAS 分数。

CorpAssist S1 · Spring 集成测试 Spring BFF 使用 QuestionAnswerAdvisor + RetrievalAugmentationAdvisor 串联。发版前需要验证：① Advisor 顺序是否正确 ② citation marker 格式 ③ retrievedDocuments 内容 ④ Python/Spring 在相同子集上指标 diff ≤ 0.02

代码：Spring Boot 评测基类

java 
// RagEvalIT.java — Spring 集成评测
@SpringBootTest
@ActiveProfiles("eval")
class RagEvalIT {

    @Autowired
    private ChatClient chatClient;

    @Autowired
    private VectorStore vectorStore;

    @Value("${eval.dataset.path:classpath:eval/corpassist-s1-v3.jsonl}")
    private String datasetPath;

    private List loadDataset() throws Exception {
        var resource = new ClassPathResource(datasetPath.replace("classpath:", ""));
        var lines = Files.readAllLines(resource.getFile().toPath());
        var mapper = new ObjectMapper();
        return lines.stream()
            .map(line -> {
                try { return mapper.readValue(line, EvalSample.class); }
                catch (Exception e) { throw new RuntimeException(e); }
            })
            .collect(Collectors.toList());
    }

    @Test
    void faithfulnessOnGoldenSubset() throws Exception {
        var samples = loadDataset().subList(0, 20);
        var scores = new ArrayList<Double>();

        for (var sample : samples) {
            var response = chatClient.prompt()
                .user(sample.getQuestion())
                .call()
                .content();
            // 简单验证：response 不为空
            assertThat(response).isNotEmpty();
        }
    }
}

Advisor 断言：验证检索与引用

advisor assert 验证 QuestionAnswerAdvisor 输出含 citation marker。这是 Spring AI 特有的评测方式。

代码：Advisor 链断言

java 
// AdvisorAssertTest.java — Advisor 断言
@SpringBootTest
class AdvisorAssertTest {

    @Autowired
    private ChatClient.Builder chatClientBuilder;

    @Test
    void shouldIncludeCitationMarkers() {
        var client = chatClientBuilder
            .defaultAdvisors(new QuestionAnswerAdvisor(vectorStore))
            .build();

        var response = client.prompt()
            .user("异地办公审批流程是什么？")
            .call()
            .content();

        // 断言包含 citation marker
        assertThat(response).contains("[doc");
        assertThat(response).contains("]");

        // 断言 retrievedDocuments 非空
        var advisors = client.getAdvisors();
        assertThat(advisors).hasSize(1);
    }

    @Test
    void shouldRefuseWhenNoContext() {
        var client = chatClientBuilder
            .defaultAdvisors(new QuestionAnswerAdvisor(vectorStore))
            .build();

        // 使用 out-of-domain 问题
        var response = client.prompt()
            .user("今天天气怎么样？")
            .call()
            .content();

        // 应拒答而非胡编
        assertThat(response).containsAnyOf("无法", "未找到", "没有相关信息");
    }
}

mock chatmodel：开发期可以用 MockChatModel 隔离 LLM 调用，但生产门禁必须用真实模型。

Spring AI Advisor 链与断言
flowchart LR
  USER[用户问题] --> QA[QuestionAnswerAdvisor]
  QA --> RET[检索 VectorStore]
  RET --> CONTEXT[组装上下文]
  CONTEXT --> LLM[LLM 生成]
  LLM --> CITE[citation 检查]
  CITE --> ASSERT{断言}
  ASSERT -->|含 [doc_id]| PASS[通过]
  ASSERT -->|无引用| FAIL[失败]

Spring AI 的 Advisor 链在生成前注入检索上下文。断言验证引用 marker 的存在和格式。

代码：Parity 检查 Java 端

java 
// ParityCheckIT.java — 双栈一致性检查
@SpringBootTest
class ParityCheckIT {

    @Autowired
    private EvalService evalService;

    @Test
    void parityWithPythonBaseline() throws Exception {
        // 读取 Python baseline
        var pythonReport = loadJson("classpath:eval/python_baseline.json");
        var springReport = evalService.runEval("eval/corpassist-s1-v3.jsonl", 50);

        var pythonFaith = pythonReport.get("aggregate").get("faithfulness").asDouble();
        var springFaith = springReport.getAggregate().getFaithfulness();

        double delta = Math.abs(pythonFaith - springFaith);
        assertThat(delta).isLessThanOrEqualTo(0.02);

        if (delta > 0.02) {
            // 生成 diff 报告
            exportParityReport(pythonReport, springReport);
        }
    }

    private JsonNode loadJson(String path) throws Exception {
        return new ObjectMapper().readTree(
            new ClassPathResource(path.replace("classpath:", "")).getFile()
        );
    }
}
不推荐
  • Mock VectorStore 返回固定检索文本
  • Python 和 Spring 各跑不同的数据集子集
  • 只比较平均分，不看逐题 diff
  • 不使用同一个索引版本
推荐
  • 真实向量库 + 同索引版本
  • 同一 JSONL hash 的子集
  • 逐题 diff，标记 delta > 0.02 的题
  • likely_cause 记录索引/Prompt/参数差异
Spring 分低一定是 bug 吗？

不一定。常见原因：① 索引同步延迟（Python 刚写完，Spring 读到旧数据）② Prompt advisor 顺序不同 ③ Tokenizer 差异（gpt-4o vs gpt-4o-mini）④ 检索参数不同（top_k、score_threshold）。逐题 diff 后定位。

报告输出：JUnit + JSON

junit report 与 json artifact 双输出，便于 CI 集成。Spring 侧生成与 Python 相同 schema 的 JSON 报告。

Spring 侧的 advisor chain 顺序会影响评测结果。CorpAssist 使用的顺序是：QuestionAnswerAdvisor → RetrievalAugmentationAdvisor → LoggingAdvisor。Advisor 顺序错误会导致 citation 格式异常。

java 
// EvalReportService.java — 评测报告生成
@Service
public class EvalReportService {

    public EvalReport generate(List<EvalResult> results, EvalThresholds thresholds) {
        var aggregate = new HashMap<String, Double>();
        aggregate.put("faithfulness", avg(results, EvalResult::getFaithfulness));
        aggregate.put("answer_relevancy", avg(results, EvalResult::getAnswerRelevancy));
        aggregate.put("context_recall", avg(results, EvalResult::getContextRecall));

        boolean passed = aggregate.entrySet().stream()
            .allMatch(e -> e.getValue() >= thresholds.getMin(e.getKey()));

        return EvalReport.builder()
            .runId("spring-eval-" + LocalDateTime.now())
            .gitSha(getGitSha())
            .datasetVersion(thresholds.getDatasetVersion())
            .aggregate(aggregate)
            .passed(passed)
            .sampleCount(results.size())
            .build();
    }

    private double avg(List<EvalResult> results,
                       ToDoubleFunction<EvalResult> extractor) {
        return results.stream().mapToDouble(extractor).average().orElse(0);
    }
}
常见误区
  • Spring 测试用内存 H2 向量库、Python 用 Milvus — 分数不可比。
  • 只对比平均分，不看同一题的双栈 diff 列表。
  • 延迟只测 LLM，不含检索与 Rerank。

本章结论

Spring 评测是 Python 评测的补充，确保生产 BFF 的 Advisor 链与 Python 索引链输出一致。

  • 集成测试：@SpringBootTest 加载同一 JSONL 子集，真实向量库。
  • Advisor 断言：验证 citation marker 格式、retrievedDocuments 非空、拒答正确。
  • 对比 Python 结果：逐题 diff，delta > 0.02 标记 likely_cause。
  • 双输出：junit.xml + json artifact，schema 与 Python 对齐。
