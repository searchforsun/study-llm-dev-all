为什么要学本章

没有固定评测集，每次改 Prompt 都像「凭感觉验收」。Golden Set 是 NS1 飞轮的燃料：坏例回流、发版回归都依赖同一套可 pin 的 JSONL。评测集的质量直接决定你对模型效果的判断可信度——数据污染、分布偏移都会让指标失灵。

本章先记住 4 件事
  • 版本 pin：dataset_version 与 git tag 对齐，禁止偷换评测集
  • gold 多样性：拒答、引用、数值计算题都要占比例
  • 分层采样：按意图/文档类型分层，避免只测简单 FAQ
  • 难例库：线上 badcase 必须可回流且带 trace

采样：从生产日志到评测样本

采样 决定评测集是否代表真实流量。CorpAssist S1 应从生产日志脱敏抽样，而非只用产品同事写的 20 条 demo 问句。

  • 随机抽样：适合建立 baseline，但易漏长尾
  • stratified sample：按 intent、doc_type、是否拒答分层
  • 主动学习：优先标注模型不确定或 disagreement 高的样本

标注规范：可操作的 rubric

annotation rubric 要可操作：什么算 faithful、什么算应拒答、引用指向哪个 chunk_id。标注规范的质量直接影响评测集的可信度。

标注 rubric 示例

markdown 
## 忠实度
- Pass：每个事实可在 contexts 找到原文支持
- Fail：出现库外政策名称、编造天数或虚构条款

## 拒答
- Pass：明确说明「制度库中未找到相关依据」
- Fail：无检索仍给出具体审批流程或数字

## 引用
- Pass：gold_answer 中标注了 chunk_id（如 [src:chunk-883]）
- Fail：引用不存在或内容与答案不匹配

## 合规
- Pass：措辞恰当，无「保证100%通过」等承诺
- Fail：出现绝对化承诺或误导性表述
不推荐
  • 「答案质量要好」无细则
  • 每标注员自定标准
  • gold 写完整段落复制粘贴
  • 无拒答标注指南
推荐
  • Likert + 通过/不通过二元门
  • 双盲标注 + 仲裁
  • gold 允许 paraphrase 但关键数字必须对
  • 拒答规范写 clear guideline

难例库：单独维护的关键子集

CorpAssist · 难例回流上次发版后线上出现「扫描件 PDF 表格审批天数错误」，已回流 12 条。难例库标记 source=prod_badcase，发版前 faithfulness 在此子集不得下降。

hard case 单独成库，回归时必须全量跑，不可与普通集混平均稀释。

难例回流是 NS1 飞轮的关键环节，确保每个线上 badcase 都转化为评测集的一部分。

代码：难例库标记与读取

python 
# hardcase_manager.py — 难例库管理
import json
from datetime import datetime
from typing import List, Dict

class HardCaseManager:
    """难例库管理器"""

    def __init__(self, path: str = "./data/hard_cases.jsonl"):
        self.path = path
        self.cases: List[Dict] = []

    def load(self):
        """加载难例库"""
        self.cases = []
        try:
            with open(self.path, "r", encoding="utf-8") as f:
                for line in f:
                    self.cases.append(json.loads(line))
        except FileNotFoundError:
            pass
        return self

    def add_case(self, question: str, contexts: List[str],
                 gold_answer: str, trace_id: str,
                 failure_type: str = "faithfulness"):
        """新增难例"""
        case = {
            "id": f"hc-{len(self.cases)+1:04d}",
            "question": question,
            "contexts": contexts,
            "gold_answer": gold_answer,
            "trace_id": trace_id,
            "failure_type": failure_type,
            "added_at": datetime.utcnow().isoformat(),
            "source": "prod_badcase",
        }
        self.cases.append(case)
        return case

    def save(self):
        """持久化"""
        with open(self.path, "w", encoding="utf-8") as f:
            for case in self.cases:
                f.write(json.dumps(case, ensure_ascii=False) + "\n")

    def run_gate(self, eval_results: List[Dict]) -> bool:
        """门禁：难例子集所有指标不得低于阈值"""
        failed = []
        for r in eval_results:
            if r["faithfulness"] < 0.85:
                failed.append(r)
        return len(failed) == 0, failed

评测集构建脚本

python 
# build_eval_dataset.py — 评测数据集构建
from dataclasses import dataclass, field
from typing import List, Optional
import json
import hashlib

@dataclass
class EvalSample:
    """评测样本"""
    question: str
    gold_answer: str
    contexts: List[str]
    metadata: Optional[dict] = None

@dataclass
class EvalDataset:
    """版本化的评测数据集"""
    name: str
    version: str
    samples: List[EvalSample] = field(default_factory=list)

    def add(self, sample: EvalSample):
        self.samples.append(sample)

    def compute_hash(self) -> str:
        """计算数据集 hash 用于 version pin"""
        content = json.dumps(
            [s.__dict__ for s in self.samples],
            sort_keys=True, ensure_ascii=False
        )
        return hashlib.sha256(content.encode()).hexdigest()[:12]

    def save_jsonl(self, path: str):
        with open(path, "w", encoding="utf-8") as f:
            for s in self.samples:
                f.write(json.dumps(s.__dict__, ensure_ascii=False) + "\n")

    def build_manifest(self) -> dict:
        return {
            "name": self.name,
            "version": self.version,
            "sha256": self.compute_hash(),
            "total": len(self.samples),
        }
能否用「更多简单题」把平均 faithfulness 刷高？

可以，但掩盖回归。应报告分层指标与难例子集，并在门禁中对 hard_case 单独设阈值。

本章结论

评测集是 NS1 飞轮的燃料。没有版本化的评测集，所有指标数字都不可信。

  • 采样：分层采样覆盖多意图、多难度；最低样本量 200 条起步。
  • 标注规范：可操作的 rubric + 双盲标注 + 仲裁流程，降低标注员漂移。
  • 难例库：线上 badcase 单独成库，回归全量跑，不与普通集混平均。
  • CorpAssist 评测集：版本 pin + sha256 + manifest，Python/Spring 共用同一 JSONL hash。

判断练习

  1. 产品说「我们多加 500 条简单 FAQ，指标肯定更漂亮」，你怎么回应？

    参考答案

    简单 FAQ 稀释难例权重，掩盖回归。应该保持分层比例固定，每次只更新版本号。新增样本必须标注分层并确保难例子集不低于最低样本量。

  2. 两个标注员在 50 条样本上只有 60% 一致，怎么办？

    参考答案

    先计算 Cohen's Kappa，若 <0.6 则暂停标注。回顾分歧样本，更新 annotation rubric，重新校准后再继续。分歧样本本身可以作为 hard case 入库。
