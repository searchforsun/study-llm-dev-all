为什么要学本章

知识库 re-embed、工单通知、评测跑批不能阻塞 SSE 主链路。message queue 提供削峰与解耦；但若无 dead letter 与 dlq replay，失败任务会静默丢失。

学完你能
  • 设计 index job 与 async notify 拓扑
  • 在 rabbitmq kafka 间选型（CorpAssist 默认 Rabbit 任务 + Kafka 日志可选）
  • 配置 job queue 消费幂等（衔接 basics-04）
本章先记住 3 件事
  • 主链路同步、重活异步：chat 同步；索引/报表异步。
  • 消息体带 tenantId + idempotencyKey。
  • DLQ 必须可观测、可 replay。

索引任务

文档上传 → BFF 发 IndexDocumentCommand → Python worker 分块 embed → 写向量库。进度经 SSE 或 WebSocket 通知管理端。

索引异步流
sequenceDiagram
  participant BFF
  participant Q as RabbitMQ
  participant W as Python Worker
  participant V as VectorStore
  BFF->>Q: publish index job
  Q->>W: consume
  W->>V: upsert vectors
  W->>BFF: callback status

通知

转人工、工单创建、配额预警走 notify.exchange，下游 SMS/邮件/IM 适配器消费。

死信

重试 3 次后进入 DLQ；运维控制台支持按 messageId replay（须再次 dedup）。

yaml 
spring:
  rabbitmq:
    listener:
      simple:
        retry:
          enabled: true
          max-attempts: 3
        default-requeue-rejected: false

本章结论

  • 索引任务：异步 worker + 状态回调；不阻塞 chat SSE。
  • 通知：独立 exchange，与索引队列隔离。
  • 死信：limited retry + DLQ + 可审计 replay。
