为什么要学本章

模型 Key、租户配额、路由规则不能硬编码在镜像里。nacos config 与 vault secret 分离；key rotation 失败会导致全站 401——密钥硬编码是本课生产坑。

学完你能
  • 划分 config center 与 secret mgmt 职责
  • 实现 dynamic config 热更新限流参数
  • 设计 api key vault 注入与 bootstrap config

Nacos

非敏感：模型路由表、quota tier、feature flag。dataId corpassist-gateway.yaml，@RefreshScope 刷新 RequestRateLimiter 参数。

Vault

DashScope/OpenAI Key 存 Vault path secret/corpassist/{tenant}/llm；应用启动或定时拉取，内存缓存 + 轮换双 Key 窗口。

yaml
spring:
  cloud:
    vault:
      uri: https://vault.corp.internal
      authentication: KUBERNETES

轮换

阶段Vault应用行为
双 Key 并存primary + secondary优先 primary，401 切 secondary
下线旧 Key仅 primary监控 401 归零后删除 secondary

本章结论

  • Nacos：非敏感动态配置；租户覆盖用独立 dataId。
  • Vault：API Key 与 DB 密码；禁止进 Git/镜像。
  • 轮换：双 Key 窗口 + 401 自动切换 + 审计日志。
