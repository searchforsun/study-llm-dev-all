章节测验

学完「Spring Cloud Gateway 与 LLM」后作答；可先「检查」再看「提示」或「答案」。路由 · 鉴权 · 限流

1. （单选）CorpAssist S2 中 /v1/stream/** 路由宜设 response-timeout: 0，主要原因是？

流式路由与 chat JSON 路由的超时策略不同，见本章「路由」与下一章 SSE。

答案：B — 流式连接长驻，固定读超时会断开正常 SSE；应配合不缓冲与空闲策略。

2. （单选）tenantId 应从哪里进入 BFF，用于 lookup_order？

鉴权一节强调：不可信任前端自报租户。

答案：C — Gateway 统一验签后注入 Header，BFF 与工具链只读该头。

3. （多选）Gateway RequestRateLimiter 单独难以完成、需 BFF 配合的限流目标包括？

决策表：QPS/连接在 Gateway；Token 与工具在 BFF。

答案：A、B — C 可由 Gateway 完成。Token 预算与 ERP 工具限流在 BFF/下游。

4. （填空）Spring Cloud Gateway 用 Route ________（英文）匹配 Path、Header 等条件，再经 Filter 转发。

术语 route-predicate；YAML 里写作 predicates。

答案：predicate（接受 predicates）

见本章判断练习与 FAQ「流式变批量」。

重连 ≠ 新 POST chat；幂等键用于写/提交。

答案：B — 对应服务端递增的 SSE id: 字段。

Python 生成器一节对比表。

答案：A、C、D — B 会阻塞并失去流式体验。

4. （填空）SSE 响应的 Content-Type 常为 text/________-stream。

Spring 常量 TEXT_EVENT_STREAM_VALUE。

答案：event-stream

NS2：结构化 dialog state，非 raw history。

粘性一节决策表。

与 max-ttl（绝对上限）区分。

答案：idle（接受 sliding）

答案：B、C — A 错误，重连不是新 POST。

Stripe 与 IETF 草案均使用该头名。
