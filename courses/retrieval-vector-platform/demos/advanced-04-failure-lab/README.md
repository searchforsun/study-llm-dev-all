# Demo：故障与降级实验

**章节**：[../../index.html#ch-advanced-04-failure]故障与降级

## 目标

实现四级降级路由，配置熔断器保护，编写故障演练场景。

## 实验内容

1. **降级路由实现**：用 Python 实现 DegradeRouter，根据健康检查结果自动切换 L0-L3 降级等级。
2. **Keyword Fallback**：当向量检索不可用时，自动降级到 ES BM25 搜索，返回关键词匹配结果。
3. **熔断器配置**：用 Resilience4j 在 Spring 中配置 Milvus 熔断器（50% 失败率、30 秒等待）。
4. **降级触发条件补充**：在 `degrade-config.yaml` 中补充一条你认为重要的降级触发条件（如「Milvus P95 > 1s 持续 5 分钟」）。
5. **故障演练**：模拟停止 Milvus 容器，验证自动降级到 BM25 的行为。

## 验收标准

- 降级路由在 Milvus 不可用时自动切换到 BM25。
- 熔断器在连续失败后打开，快速失败。
- 降级触发条件至少包含 3 条（含补充的）。
- 故障演练验证降级有效。

## 延伸

模拟双库（Milvus + ES）同时不可用，验证缓存兜底 L3 行为。
