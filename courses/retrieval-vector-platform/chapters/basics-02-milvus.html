为什么要学本章

Milvus 是 CorpAssist 的默认向量数据库。它的 collection schema、vector insert 和 vector search 接口是所有上层 RAG 系统的基础。部署不会？Python 索引写不进去？Spring 消费查不出来？本章解决这些问题。

学完你能

  • 设计 Collection Schema 并选择 index type
  • 用 PyMilvus 完成 bulk insert 和搜索
  • 用 MilvusClientV2（Spring）执行只读查询
本章先记住 3 件事
  • Schema 一旦创建不可改字段：向量维度、主键、metric type 在 drop 前不能变更。
  • Insert 后要 flush/索引才能搜：Milvus 是最终一致写入，索引是异步构建的。
  • Python 写索引，Java 读索引：CorpAssist 的 Python 索引工程负责写入，Spring 服务只读消费。

Milvus 部署模式与 Docker Compose 实战

Milvus 提供三种部署模式：Standalone（单机）、Cluster（分布式）和 Milvus Cloud（全托管）。CorpAssist 开发/测试环境使用 Standalone 模式，依赖 etcd（元数据）和 MinIO（向量存储）。

CorpAssist · Milvus 初始化部署 团队决定统一向量存储方案，选择 Milvus Standalone 作为开发环境。需要编写 docker-compose.yml 启动 etcd + MinIO + Milvus 三件套，并执行首次 Schema 创建。

先想 10 秒：etcd 挂了，Milvus 还能搜索吗？

不能。etcd 存储 Collection 元数据，查询节点需要读取这些信息。但 MinIO 中的向量数据不会丢失——恢复 etcd 后 Milvus 可重新加载。

Collection Schema 设计

collection schema 是 Milvus 的「表结构」。设计时需指定主键、向量字段（维度 + 度量类型）、标量字段（用于过滤）。

字段名类型主键用途
idInt64是自增主键
embeddingFloatVector[768]-文本 embedding 向量
chunk_textVarChar(65535)-原始文本
sourceVarChar(256)-文档来源
tenant_idInt64-多租户隔离
created_atInt64-时间戳过滤

Python：创建 Collection

python 
from pymilvus import connections, Collection, FieldSchema, CollectionSchema, DataType

connections.connect(host="localhost", port="19530")

fields = [
    FieldSchema(name="id", dtype=DataType.INT64, is_primary=True, auto_id=False),
    FieldSchema(name="embedding", dtype=DataType.FLOAT_VECTOR, dim=768),
    FieldSchema(name="chunk_text", dtype=DataType.VARCHAR, max_length=65535),
    FieldSchema(name="source", dtype=DataType.VARCHAR, max_length=256),
    FieldSchema(name="tenant_id", dtype=DataType.INT64),
    FieldSchema(name="created_at", dtype=DataType.INT64),
]
schema = CollectionSchema(fields, description="CorpAssist 知识库")
collection = Collection(name="corpassist_kb", schema=schema)

index_params = {
    "metric_type": "COSINE",
    "index_type": "HNSW",
    "params": {"M": 16, "efConstruction": 200},
}
collection.create_index("embedding", index_params)
print(f"Created '{collection.name}' with HNSW index")

Java：Spring 配置与验证

java 
import io.milvus.v2.client.MilvusClientV2;
import io.milvus.v2.client.ConnectConfig;

@Configuration
public class MilvusConfig {
    @Bean
    public MilvusClientV2 milvusClient() {
        MilvusClientV2 client = new MilvusClientV2(
            ConnectConfig.builder().uri("http://localhost:19530").build());
        boolean exists = client.hasCollection(
            io.milvus.v2.service.collection.request.HasCollectionReq.builder()
                .collectionName("corpassist_kb").build());
        if (!exists) {
            throw new IllegalStateException("Run Python index first!");
        }
        return client;
    }
}
不推荐
  • schema 中不加 tenant_id 过滤字段
  • embedding 维度选错，drop 重建
  • 文本字段用 VarChar(256) 被截断
推荐
  • 预留过滤字段（tenant_id, source）
  • 再三确认 dim 和 metric_type 后建索引
  • 长文本字段用 VarChar(65535)

数据插入与 Flush 语义

vector insert 是批量操作。数据先写入 delta buffer，然后异步 flush 到 MinIO。flush 完成后才可被搜索。

Python：Bulk Insert

python 
import numpy as np, time
collection = Collection("corpassist_kb")

entities = [
    list(range(100)),
    [np.random.rand(768).tolist() for _ in range(100)],
    [f"chunk_{i}" for i in range(100)],
    ["wiki"] * 100,
    [1] * 100,
    [int(time.time())] * 100,
]
result = collection.insert(entities)
collection.flush()
print(f"Inserted {result.insert_count} rows")

# 验证
q = collection.query(expr=f"id == {result.primary_keys[0]}",
    output_fields=["id", "chunk_text"])
print(f"Verified: {q[0]}")

Java：只读查询验证

java 
import io.milvus.v2.service.vector.request.QueryReq;
import io.milvus.v2.service.vector.response.QueryResp;

@Service
public class MilvusReadService {
    private final MilvusClientV2 client;

    public QueryResp queryById(long id) {
        return client.query(QueryReq.builder()
            .collectionName("corpassist_kb")
            .filter("id == " + id)
            .outputFields(List.of("id", "chunk_text", "source"))
            .build());
    }
}
写入延迟：Milvus 是最终一致写入。如果需要强一致读，在搜索请求中设置 guarantee_timestamp 参数。CorpAssist 索引工程 flush 后等待 2-3 秒确保数据可见。

向量搜索与参数配置

vector search 是核心操作。参数包括查询向量、Top-K、搜索参数（ef）、输出字段和过滤表达式。

Python：搜索

python 
collection.load()
query_vec = np.random.rand(768).tolist()

results = collection.search(
    data=[query_vec],
    anns_field="embedding",
    param={"metric_type": "COSINE", "params": {"ef": 64}},
    limit=5,
    output_fields=["id", "chunk_text", "source", "tenant_id"],
)
for hits in results:
    for hit in hits:
        print(f"id={hit.id} score={hit.score:.4f} source={hit.entity.get('source')}")
collection.release()

Java：Spring 搜索服务

java 
import io.milvus.v2.service.vector.request.SearchReq;
import io.milvus.v2.service.vector.request.data.FloatVec;

@Service
public class VectorSearchService {
    private final MilvusClientV2 client;

    public SearchResp search(float[] queryVector, int topK) {
        return client.search(SearchReq.builder()
            .collectionName("corpassist_kb")
            .data(List.of(new FloatVec(queryVector)))
            .annsField("embedding")
            .topK(topK)
            .outputFields(List.of("id", "chunk_text", "source"))
            .searchParams(Map.of("ef", 64, "metric_type", "COSINE"))
            .build());
    }
}

常见问题（FAQ）

insert 后搜不到新数据？
刚插入的数据在 delta buffer 中只对 query() 可见。调用 flush() 并等待索引构建完成（几秒到几十秒）后才可检索。
load() 和 release() 是什么？
load() 将数据加载到查询节点内存，搜索前必须调用。release() 释放内存。CorpAssist 生产环境保持常驻 loaded。
Python 和 Java SDK 怎么分工？
Python SDK（PyMilvus）功能完整，负责写入和 schema 管理。Java SDK（MilvusClientV2）只做只读搜索。CorpAssist 原则：Python 管写和 schema，Java 只管读。

本章结论

  • 部署选 Standalone：Docker Compose 起 etcd + MinIO + Milvus 三件套。
  • Schema 先行设计：预留过滤字段，确认维度和 metric type，建后不能改。
  • 写入注意 flush：Insert 后需 flush 才可搜索。
  • 搜索需 load：搜索前 load collection，用 ef 参数控召回。
  • Python 写，Java 读：双栈分工清晰，避免 SDK 冲突。
