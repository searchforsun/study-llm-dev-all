ANN 检索 top-50 中包含大量噪音——不相关文档混入结果。cross encoder 通过精细化比较大幅提升 top-5 的相关性。two stage 先宽召回再精排是生产级 RAG 的标配。

本章先记住 3 件事
  • Cross-encoder 精度高但慢：每对 query+doc 都要过模型，50 条候选中精排 top-5 耗时 200-500ms。
  • Bi-encoder（向量检索）是召回阶段：快速初筛候选；Cross-encoder 是精排阶段：慢但准。
  • Rerank Top-N 决定延迟：排 20 条 ≈ 80ms，排 100 条 ≈ 400ms。用精度曲线选最优值。

两阶段检索架构：Bi-encoder + Cross-encoder

two stage 检索将过程分为召回和精排。召回阶段用 ANN（bi-encoder）快速从百万级筛出 top-50；精排阶段用 cross encoder 逐对打分，选出 top-5。

Cross-encoder Rerank 实战

BGE Reranker 是 BAAI 开源的 cross-encoder 模型。它将 query 和每个 candidate 拼接后过 transformer，输出相关性分数。

Python：Rerank 集成

python 
from pymilvus import Collection
from sentence_transformers import CrossEncoder
import numpy as np

# 1. 加载 reranker
reranker = CrossEncoder("BAAI/bge-reranker-v2-m3", max_length=512)

# 2. ANN 召回 50 条
collection = Collection("corpassist_kb")
collection.load()
query_vec = np.random.rand(768).tolist()
ann_hits = collection.search(
    data=[query_vec],
    anns_field="embedding",
    param={"metric_type": "COSINE", "params": {"ef": 64}},
    limit=50,
    output_fields=["id", "chunk_text", "source"],
)[0]

# 3. Cross-encoder Rerank
query = "年假政策"
pairs = [(query, hit.entity.get("chunk_text")) for hit in ann_hits]
scores = reranker.predict(pairs)

# 4. 按精排分数排序取 top-5
ranked = sorted(zip(ann_hits, scores), key=lambda x: -x[1])
print("Reranked top-5:")
for hit, score in ranked[:5]:
    print(f"score={score:.4f} id={hit.id} source={hit.entity.get('source')}")
    print(f"  text={hit.entity.get('chunk_text')[:80]}...")

collection.release()

Java：Spring Rerank 服务

java 
import org.springframework.web.client.RestTemplate;
import java.util.*;

@Service
public class RerankService {
    // 假设 reranker 部署为 HTTP 服务
    private final RestTemplate rest = new RestTemplate();
    private final String rerankerUrl = "http://reranker-service:8000/rerank";

    public record RerankItem(String id, String text, double score) {}

    public List rerank(String query, List> candidates, int topK) {
        Map request = Map.of(
            "query", query,
            "documents", candidates.stream().map(c -> c.get("chunk_text")).toList(),
            "top_k", topK
        );
        // 调用 Python reranker 服务
        var response = rest.postForObject(rerankerUrl, request, List.class);
        // 解析并返回精排结果
        return (List) response;
    }
}
CorpAssist · Rerank 延迟预算 CorpAssist 的检索 SLO 是 500ms。ANN 召回 50 条需 50ms，Rerank 50 条需 300ms，总 350ms 在预算内。如果召回扩到 100 条，Rerank 涨到 600ms，超过 SLO。此时需要选择：降低召回数或升级 Rerank 硬件。

精度曲线与 Top-N 选择

rerank latency 与被精排的候选数成正比。precision curve 帮助找到最佳平衡点。

Python：画精度曲线

python 
import matplotlib.pyplot as plt
import numpy as np

def precision_curve(queries, candidate_sets, reranker, max_n=100, step=10):
    results = []
    for n in range(step, max_n + 1, step):
        precisions = []
        for query, candidates in zip(queries, candidate_sets):
            pairs = [(query, c["text"]) for c in candidates[:n]]
            scores = reranker.predict(pairs)
            ranked = sorted(zip(candidates[:n], scores), key=lambda x: -x[1])
            # top-5 精排中相关比例
            relevant = sum(1 for _, s in ranked[:5] if s > 0.5)
            precisions.append(relevant / 5)
        results.append({"rerank_n": n, "avg_precision@5": np.mean(precisions)})
    return results

# 假设已有数据
curves = precision_curve(queries_list, candidate_list, reranker)
# 输出表格：选 n=50 时精度~0.92，n=80 时~0.93，增益递减
不推荐
  • 全库 rerank（不先 ANN 召回）
  • 精排数拍脑袋定 20 条
  • 交叉编码器输出分数不验证
推荐
  • ANN 先召回 50-100，再精排
  • 用精度曲线选 rerank_n
  • 校对 cross-encoder 分数分布

常见问题（FAQ）

Rerank 和 ANN 第二次搜索有什么区别？
本质不同。ANN 是 bi-encoder：query 和 doc 各自独立编码为向量，用距离计算相似度。Cross-encoder 把 query+doc 拼接后过 transformer，能捕捉 query 和 doc 之间的深层交互。Cross-encoder 精度更高但无法用 ANN 索引。
Rerank 模型怎么部署？
推荐用 Triton Inference Server 或 FastAPI 封装。CorpAssist 将 bge-reranker-v2-m3 部署为独立 HTTP 服务，GPU 推理。batch size 可调整——batch=16 时 50 条只需 1-2 次推理。
Rerank TOP-5 和 ANN TOP-5 可能不同吗？
非常常见。ANN 的 bi-encoder 打分可能误判，cross-encoder 能纠正。CorpAssist 实践中约 30% 的最终 top-5 会被 rerank 重排序。

本章结论

  • 两阶段是标配：ANN 召回 + Cross-encoder 精排，精度和延迟兼顾。
  • Rerank 精度高于 ANN：cross-encoder 捕捉深层交互，约 30% 结果被重新排序。
  • 延迟与候选数线性相关：用精度曲线选最佳 rerank_n。
  • 部署为独立服务：Rerank 需要 GPU，独立部署不影响 ANN 召回。
  • Python 推理，Spring 调用：Rerank 模型在 Python 侧推理，Spring 侧 HTTP 调用。
