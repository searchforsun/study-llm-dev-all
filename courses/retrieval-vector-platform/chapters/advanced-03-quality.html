检索消融实验设计

ablation study 逐个移除或修改环节，观察对最终指标的影响。CorpAssist 的检索管线包含 5 个可消融变量。

消融实验：变量逐轮消融
flowchart TB

    BASELINE --> EXP2["Exp 2: ef=128"]

    BASELINE --> EXP4["Exp 4: no rewrite"]

    EXP1 --> REPORT["质量报告\n对比 5 组 exp"]

    EXP4 --> REPORT
    REPORT --> DECIDE["结论：选择 + 理由"]

每次只变一个变量，其余保持不变，结果归因清晰。

实验矩阵与假设验证

experiment matrix 系统性地组合变量。先用 hypothesis test 提出假设，再设计实验验证。

Python：实验矩阵运行器

python 
import itertools
import json

class RetrievalExperiment:
    def __init__(self, query_set, ground_truth):
        self.query_set = query_set
        self.ground_truth = ground_truth
        self.results = []

    def run(self, params: dict):
        """运行单组参数"""
        # 模拟检索并计算指标
        # 真实实现调用 Milvus/ES
        recall, precision, latency = self._execute(params)
        self.results.append({**params, "recall@10": recall, "p95_ms": latency})
        return self.results[-1]

    def grid_search(self, param_grid: dict):
        """遍历参数组合"""
        keys = param_grid.keys()
        for values in itertools.product(*param_grid.values()):
            params = dict(zip(keys, values))
            self.run(params)

    def report(self) -> str:
        """生成实验报告"""
        lines = ["## 检索质量实验报告", "", "| " + " | ".join(self.results[0].keys()) + " |"]
        lines.append("|" + "---|" * len(self.results[0].keys()))
        for r in self.results:
            row = " | ".join(f"{v:.4f}" if isinstance(v, float) else str(v) for v in r.values())
            lines.append(f"| {row} |")
        return "\n".join(lines)

# 使用
exp = RetrievalExperiment(query_set=range(100), ground_truth={})
exp.grid_search({"ef": [32, 64, 128], "rewrite": [True, False]})
print(exp.report())

Java：实验配置读取

java 
import com.fasterxml.jackson.databind.ObjectMapper;
import java.util.Map;

@Component
public class ExperimentConfigLoader {
    private final ObjectMapper mapper;

    public ExperimentConfigLoader() {
        this.mapper = new ObjectMapper();
    }

    public Map loadConfig(String path) throws IOException {
        // 读取实验 YAML/JSON 配置
        return mapper.readValue(new java.io.File(path), Map.class);
    }

    public static class ExperimentResult {
        public String name;
        public double recallAt10;
        public double p95Ms;
        public Map params;
    }
}

质量报告撰写

quality report 标准化记录实验设计和结果。每个报告包含：背景假设、实验矩阵、结果表格、结论和建议。

实验报告 YAML 模板

yaml 
experiment:
  name: "ef_search_ablation"
  date: "2026-05-31"
  dataset: "corpassist_1000_queries"

  hypothesis: "增大 ef_search 能提升 recall@10 ≥ 3%"

  baseline:
    ef: 64
    top_k: 50
    rewrite: true
    recall@10: 0.92
    p95_ms: 45

  experiments:
    - name: "ef_32"
      params: {ef: 32}
      recall@10: 0.88
      p95_ms: 22
    - name: "ef_128"
      params: {ef: 128}
      recall@10: 0.95
      p95_ms: 68
    - name: "ef_256"
      params: {ef: 256}
      recall@10: 0.97
      p95_ms: 135

  conclusion: >
    ef=128 提升 recall@10 从 0.92 到 0.95（+3%），
    p95 从 45ms 到 68ms（仍在 100ms SLO 内）。
    推荐生产使用 ef=128。

  recommendation: {ef: 128, top_k: 50, rewrite: true}
不推荐
  • 一次改多个变量，无法归因
  • 不记录环境和随机种子
  • 报告只有数字没有结论
  • 实验不设 baseline
推荐
  • 一次只变一个变量
  • 记录实验环境+种子
  • 结论解释数字含义
  • 始终与 baseline 对比

常见问题（FAQ）

消融实验和 A/B 测试有什么区别？
消融实验是离线评估：用固定数据集评估检索质量。A/B 测试是在线评估：真实用户流量分桶对比。消融实验快速验证假设，A/B 测试验证实际效果。两者互补。
评估集应该多大？
至少 100 条代表性查询。太少统计波动大，太多人工标注成本高。CorpAssist 从各租户按比例抽样 200-500 条。
实验发现 ef=256 提升很小但延迟翻倍，怎么决策？
看 SLO。如果 135ms 仍在 SLO 内且提升 2% recall 有价值，就选 ef=256。否则选 ef=128。关键是用业务场景决策。

本章结论

  • 消融实验科学归因：一次只变一个变量。
  • 假设先行再验证：先写清楚猜测再设计实验。
  • 实验矩阵系统化：Grid Search 覆盖参数组合。
  • 报告要标准化：背景、矩阵、结果、结论四部分。
  • 结论要有推荐：数字 + 业务解读 = 可执行。
