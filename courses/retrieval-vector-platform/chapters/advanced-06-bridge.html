Spring VectorStore 定义了一组标准的 DTO（Document、SearchResult）。Python 索引端需与这些 DTO 对齐。

Python 写入的字段必须和 Spring 读取的字段完全对齐，Integration Test 自动验证。

Java：Spring DTO

java 
import org.springframework.ai.document.Document;
import java.util.Map;

// Spring AI Document DTO
// 与 Python 写入的字段完全对齐
public class CorpAssistDocument {
    public static Document fromMilvus(Map fields) {
        String id = String.valueOf(fields.get("id"));
        String content = (String) fields.get("chunk_text");
        Map metadata = Map.of(
            "source", fields.getOrDefault("source", ""),
            "tenant_id", fields.getOrDefault("tenant_id", 0),
            "dept", fields.getOrDefault("dept", ""),
            "classification", fields.getOrDefault("classification", 0)
        );
        return new Document(id, content, metadata);
    }
}

Python：索引写入时字段对齐

python 
# Python 索引写入时必须和 Spring Contract 对齐
def insert_document(doc_id: str, text: str, source: str, tenant: int, dept: str):
    """写入文档，字段名与 Spring DTO 完全对齐"""
    collection = Collection("corpassist_kb")
    collection.insert([
        [hash(doc_id)],            # id → Document.id()
        [embedding],               # embedding 向量
        [text],                    # chunk_text → Document.content()
        [source],                  # source → metadata.source
        [tenant],                  # tenant_id → metadata.tenant_id
        [dept],                    # dept → metadata.dept
    ])
    collection.flush()

# 注意：千万不能把字段名写成 chunkText 或 content_text
# Spring 那边读的就是 chunk_text

Filter Parity：过滤表达式一致性

filter parity 确保 Python 和 Java 端使用相同的 filter 逻辑。Milvus filter 表达式在双栈中的写法差异：

Integration Test：验证一致性

java 
import org.junit.jupiter.api.Test;
import static org.junit.jupiter.api.Assertions.*;

@SpringBootTest
class ContractAlignmentTest {

    @Autowired private MilvusClientV2 client;

    @Test
    void testDocumentContract() {
        // 验证字段名和类型与 Python 端对齐
        var resp = client.describeCollection(
            io.milvus.v2.service.collection.request.DescribeCollectionReq.builder()
                .collectionName("corpassist_kb").build());

        var fields = resp.getCollectionSchema().getFieldSchemas();
        var fieldNames = fields.stream()
            .map(f -> f.getName() + ":" + f.getDataType())
            .toList();

        // 断言字段与 Python 写入一致
        assertTrue(fieldNames.contains("id:Int64"));
        assertTrue(fieldNames.contains("embedding:FloatVector"));
        assertTrue(fieldNames.contains("chunk_text:VarChar"));
        assertTrue(fieldNames.contains("source:VarChar"));
        assertTrue(fieldNames.contains("tenant_id:Int64"));
        assertTrue(fieldNames.contains("dept:VarChar"));
    }

    @Test
    void testSearchConsistency() {
        // 验证同一 query 在双栈中返回结果一致
        float[] queryVec = loadTestVector();
        var springResults = client.search(SearchReq.builder()
            .collectionName("corpassist_kb")
            .data(List.of(new FloatVec(queryVec)))
            .annsField("embedding")
            .topK(5)
            .searchParams(Map.of("ef", 64, "metric_type", "COSINE"))
            .expr("tenant_id == 1")
            .build());

        // Python 端跑同一查询，比对 top-5 id
        // 建议：跑完 Python 后断言 id 列表一致
        assertFalse(springResults.getSearchResults().get(0).isEmpty());
        // ...
    }

    private float[] loadTestVector() {
        return new float[768]; // 加载测试向量
    }
}

Python：Dual Read 对比脚本

python 
from pymilvus import Collection

def dual_read_compare(query_vec, top_k=5):
    """Python 端和 Spring 端搜索对比"""
    collection = Collection("corpassist_kb")
    collection.load()

    python_results = collection.search(
        data=[query_vec],
        anns_field="embedding",
        param={"metric_type": "COSINE", "params": {"ef": 64}},
        limit=top_k,
        output_fields=["id", "chunk_text", "source"],
    )

    python_ids = [h.id for h in python_results[0]]
    print(f"Python top-{top_k} ids: {python_ids}")

    # 注意：Spring 端应返回相同 id 列表（顺序可能不同）
    # 如果不同，说明 filter、metric_type 或 ef 不匹配
    # spring_ids = call_spring_search_api(query_vec)
    # assert set(python_ids) == set(spring_ids), "Dual read mismatch!"

    collection.release()
    return python_ids

联调清单与常见问题

dual read 对比是联调的核心手段。同一 query 在 Python 和 Spring 中各搜一次，对比 top-5 的 id 是否一致。

联调检查清单

  • [ ] Python 写入字段和 Spring DTO 字段一一对应
  • [ ] metric_type（COSINE / IP / L2）在双栈中一致
  • [ ] ef_search 参数值相同
  • [ ] Filter 表达式在双栈中写法一致且结果一致
  • [ ] 同一 query 的 top-5 id 在双栈中至少 4/5 一致
  • [ ] 时间戳或版本号对齐

常见问题（FAQ）

Python 和 Java 搜索同一 query 结果不同，最常见原因是什么？
最多的情况是 ef 参数不一致。Python 端 ef=128，Java 端 ef=64。其次是 filter 表达式写法差异——比如 String 引号用错。建议 Dual Read 作为 CI 的一部分持续验证。
字段名对齐后，Java 搜出来的 content 字段值是空的？
检查 output_fields 是否包含了 chunk_text。Java SDK 需要显式声明 output_fields，不声明默认只返回 id 和 score。Python 的 search() 也需要配 output_fields。
Spring AI VectorStore 的抽象层够用吗？还是需要直接调 MilvusClientV2？
简单 CRUD 用 Spring AI VectorStore 足够。但高级能力（自定义 filter、指定 ef、multi-vector search）需要直接用 MilvusClientV2。CorpAssist 实践：通用检索用 Spring AI，排障用原生 SDK。

本章结论

  • Contract 先行：Python 和 Java 的字段名、类型、语义提前对齐。
  • Filter Parity：filter 表达式在双栈中写法一致。
  • Integration Test 验证：字段结构和搜索结果都自动化验证。
  • Dual Read 排障：同一 query 双栈对比是联调核心手段。
  • Spring AI + 原生 SDK 结合：简单检索用 Spring AI，高级能力用原生 SDK。
