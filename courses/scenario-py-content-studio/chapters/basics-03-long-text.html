固定分块与语义分块

分块策略直接影响检索质量。固定分块简单高效，语义分块更精准但计算成本高。

策略方法适用场景召回率计算成本
固定分块按字符/token数切分长度统一的文档60-75%低
语义分块按段落/句子边界+Embedding营销文案、文章75-90%中
递归分块多级分割符递归切分层次结构文档70-85%中
LLM分块用LLM识别语义边界质量要求极高场景85-95%高

Python：固定分块 vs 语义分块

python
# chunk_strategies.py — 分块策略实现
from typing import List
import re

class FixedChunker:
    """固定大小分块"""
    def __init__(self, chunk_size: int = 512, overlap: int = 50):
        self.chunk_size = chunk_size
        self.overlap = overlap

    def chunk(self, text: str) -> List[str]:
        chunks, start = [], 0
        while start < len(text):
            end = start + self.chunk_size
            chunks.append(text[start:end])
            start += self.chunk_size - self.overlap
        return chunks

class SemanticChunker:
    """语义分块 — 按段落边界切分"""
    def __init__(self, max_size: int = 512, separator: str = r"\n\n"):
        self.max_size = max_size
        self.separator = separator

    def chunk(self, text: str) -> List[str]:
        paragraphs = re.split(self.separator, text)
        chunks, current = [], ""
        for para in paragraphs:
            if len(current) + len(para) < self.max_size:
                current += para + "\n\n"
            else:
                if current:
                    chunks.append(current.strip())
                current = para + "\n\n"
        if current:
            chunks.append(current.strip())
        return chunks

# 参数实验
def run_experiment(text: str, strategies: dict) -> dict:
    results = {}
    for name, chunker in strategies.items():
        chunks = chunker.chunk(text)
        results[name] = {
            "chunk_count": len(chunks),
            "avg_size": sum(len(c) for c in chunks) / len(chunks),
            "min_size": min(len(c) for c in chunks),
            "max_size": max(len(c) for c in chunks),
        }
    return results

Java：Spring AI分块配置

java
// ChunkConfig.java — Spring AI分块
import org.springframework.ai.document.Document;
import org.springframework.ai.transformer.splitter.TokenTextSplitter;
import java.util.List;

@Component
public class ChunkConfig {
    @Bean
    public TokenTextSplitter defaultSplitter() {
        return new TokenTextSplitter(512, 50, 5, 100, true);
    }

    public List splitWithStrategy(String text, String strategy) {
        return switch (strategy) {
            case "fixed" -> new TokenTextSplitter(256, 25, 3, 50, false).split(new Document(text));
            case "semantic" -> new TokenTextSplitter(512, 50, 5, 100, true).split(new Document(text));
            default -> throw new IllegalArgumentException("Unknown: " + strategy);
        };
    }
}

表格与代码块特殊分块

表格和代码块不能按普通文本切分，必须保留结构完整性。

Python：代码块分块

python
# special_chunker.py — 特殊内容分块
from typing import List
import re

class CodeBlockChunker:
    """代码块按函数边界分块"""
    def __init__(self, max_lines: int = 200):
        self.max_lines = max_lines

    def chunk_python(self, code: str) -> List[str]:
        func_pattern = r"(def |class |@)"
        chunks, current = [], []
        for line in code.split("\n"):
            if re.match(func_pattern, line) and current:
                chunks.append("\n".join(current))
                current = []
            current.append(line)
            if len(current) >= self.max_lines:
                chunks.append("\n".join(current))
                current = []
        if current:
            chunks.append("\n".join(current))
        return chunks

class TableChunker:
    """表格保持完整"""
    def __init__(self, max_rows: int = 30):
        self.max_rows = max_rows

    def chunk(self, table_text: str) -> List[str]:
        rows = table_text.split("\n")
        if len(rows) <= self.max_rows:
            return [table_text]
        chunks = []
        for i in range(0, len(rows), self.max_rows):
            chunks.append(rows[0] + "\n" + "\n".join(rows[i:i + self.max_rows]))
        return chunks

Python：网格搜索参数

python
# grid_search.py — 网格搜索
import numpy as np
from typing import List, Dict

def grid_search(text: str, questions: List[str]) -> List[Dict]:
    results = []
    for size in [256, 512, 768]:
        for overlap in [25, 50, 75]:
            chunks = FixedChunker(size, overlap).chunk(text)
            recall = np.mean([1.0 if any(kw in c for kw in q.split())
                              for q in questions
                              for c in [chunks]]
                             if chunks else [0.0])
            results.append({"size": size, "overlap": overlap,
                           "recall": recall, "chunks": len(chunks)})
    return sorted(results, key=lambda r: r["recall"], reverse=True)

best = grid_search("sample", ["Q1", "Q2"])[0]
print(f"Best: size={best['size']}, overlap={best['overlap']}, recall={best['recall']:.2f}")
不推荐
  • 表格从中切断，一半在chunk1一半在chunk2
  • 代码从函数中间切分，语法错误
  • 所有内容用同一分块参数
  • 不保留表头
推荐
  • 表格整块保留，超过30行再按行切
  • 代码按函数/类边界切分
  • 不同类型使用不同分块策略
  • 表头自动添加到每个分块

常见问题（FAQ）

chunk_size越大越好吗？
不是。越大每个chunk信息越多，但检索精度下降。建议512 tokens起步，根据场景调优。长文档推荐256-512，问答推荐128-256。
overlap设多少合适？
推荐chunk_size的10-20%。中文文本overlap=50左右通常效果不错。
混合类型文档怎么办？
用文档解析器识别段落类型，按类型选择策略。实现CompositeChunker内部路由。
