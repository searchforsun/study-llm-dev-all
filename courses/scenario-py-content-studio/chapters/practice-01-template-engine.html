Jinja2模板引擎与变量系统

Jinja2提供变量替换、条件判断和循环控制，是内容模板的理想选择。在内容生成场景中，变量分为两类：渲染变量（模板直接替换）和LLM变量（需调用LLM生成）。

Python：模板引擎完整实现

python
# template_engine.py — Jinja2模板引擎
from jinja2 import Environment, FileSystemLoader
from pydantic import BaseModel
from typing import List, Dict, Optional
from pathlib import Path

class ContentTemplate(BaseModel):
    name: str
    template_text: str
    variables: Dict[str, str]        # 变量名→类型
    llm_fill: List[str] = []         # 需LLM生成的变量
    description: str = ""

class TemplateEngine:
    def __init__(self, template_dir: str = "./templates"):
        self.env = Environment(loader=FileSystemLoader(template_dir))
        self.templates: Dict[str, ContentTemplate] = {}

    def register(self, template: ContentTemplate):
        path = Path(f"./templates/{template.name}.j2")
        path.parent.mkdir(parents=True, exist_ok=True)
        path.write_text(template.template_text)
        self.templates[template.name] = template

    def render(self, name: str, variables: dict) -> str:
        tpl = self.env.get_template(f"{name}.j2")
        return tpl.render(**variables)

    def list_templates(self) -> List[Dict]:
        return [{"name": t.name, "variables": t.variables,
                 "llm_fill": t.llm_fill, "desc": t.description}
                for t in self.templates.values()]

# 示例：商品描述模板
product_desc = ContentTemplate(

    variables={
        "product_name": "str", "tagline": "str", "description": "str",
        "price": "float", "stock": "int", "features": "list",

    },

模板引擎+LLM+MQ数据流
flowchart LR
    User[运营] --> Submit[提交批量任务]
    Submit --> MQ[RabbitMQ]
    MQ --> Worker[Worker消费]
    Worker --> TM[Jinja2渲染]
    TM --> LLM[LLM填充变量]
    LLM --> Result[写入结果]
    Result --> Done[完成事件]
    Done --> User

    Worker --> Redis[(Redis进度)]

批任务通过MQ异步处理，Worker负责模板渲染+LLM调用，进度写入Redis。

MQ批量生成任务

RabbitMQ承载批量任务。Producer将N条任务发布到队列，Consumer Worker消费并处理，支持并发控制和失败重试。

Python：MQ批量生成

python
# batch_generator.py — MQ批量生成
import asyncio, json, uuid
from dataclasses import dataclass, field
from typing import Optional

@dataclass
class BatchTask:
    task_id: str = field(default_factory=lambda: uuid.uuid4().hex[:12])
    template_name: str = ""
    variables: dict = field(default_factory=dict)
    status: str = "pending"   # pending/processing/done/failed
    result: Optional[str] = None
    error: Optional[str] = None

class BatchGenerator:
    def __init__(self, engine: TemplateEngine, llm_client, max_concurrency: int = 5):
        self.engine = engine
        self.llm = llm_client
        self.sem = asyncio.Semaphore(max_concurrency)

    async def process_task(self, task: BatchTask) -> BatchTask:
        async with self.sem:
            task.status = "processing"
            try:
                # 1. 渲染固定变量
                prompt = self.engine.render(task.template_name, task.variables)
                # 2. LLM填充动态内容
                response = await self.llm.generate(prompt)
                task.result = response
                task.status = "done"
            except Exception as e:
                task.status = "failed"
                task.error = str(e)
            return task

    async def process_batch(self, tasks: list[BatchTask]) -> list[BatchTask]:
        return await asyncio.gather(*[self.process_task(t) for t in tasks])

    def create_batch(self, template: str, variable_list: list[dict]) -> list[BatchTask]:
        return [BatchTask(template_name=template, variables=v) for v in variable_list]

# 使用示例
variable_list = [
    {"product_name": f"商品{i}", "price": 99.0 + i,
     "stock": 100, "features": ["功能A"], "call_to_action": "立即购买"}
    for i in range(100)
]
generator = BatchGenerator(engine, None)
tasks = generator.create_batch("product_desc", variable_list)
print(f"Created {len(tasks)} tasks")

Java：Spring Boot MQ配置

java
// RabbitMQConfig.java — 批量生成队列配置
@Configuration
public class RabbitMQConfig {
    public static final String BATCH_QUEUE = "content.batch.generate";
    public static final String BATCH_DLX = "content.batch.dlx";

    @Bean
    public Queue batchQueue() {
        return QueueBuilder.durable(BATCH_QUEUE)
            .withArgument("x-dead-letter-exchange", BATCH_DLX)
            .withArgument("x-dead-letter-routing-key", "batch.failed")
            .build();
    }

    @Bean
    public SimpleRabbitListenerContainerFactory batchFactory(
            ConnectionFactory connectionFactory) {
        SimpleRabbitListenerContainerFactory factory =
            new SimpleRabbitListenerContainerFactory();
        factory.setConnectionFactory(connectionFactory);
        factory.setConcurrentConsumers(5);
        factory.setMaxConcurrentConsumers(20);
        factory.setPrefetchCount(10);
        return factory;
    }
}

进度追踪与失败重试

批量生成必须提供进度反馈。通过Redis记录批任务状态，前端轮询或WebSocket获取进度。

Python：进度追踪

python
# progress_tracker.py — 进度追踪
import redis.asyncio as redis
import json

class ProgressTracker:
    def __init__(self, redis_url: str = "redis://localhost:6379/0"):
        self.redis = None
        self.redis_url = redis_url

    async def connect(self):
        self.redis = await redis.from_url(self.redis_url)

    async def init_batch(self, batch_id: str, total: int):
        await self.redis.hset(f"batch:{batch_id}", mapping={
            "total": total, "done": 0, "failed": 0, "status": "running"
        })

    async def mark_done(self, batch_id: str):
        await self.redis.hincrby(f"batch:{batch_id}", "done", 1)

    async def mark_failed(self, batch_id: str):
        await self.redis.hincrby(f"batch:{batch_id}", "failed", 1)

    async def get_progress(self, batch_id: str) -> dict:
        data = await self.redis.hgetall(f"batch:{batch_id}")
        if not data:
            return {"total": 0, "done": 0, "failed": 0, "percent": 0}
        total = int(data[b"total"])
        done = int(data.get(b"done", 0))
        failed = int(data.get(b"failed", 0))
        return {
            "total": total, "done": done, "failed": failed,
            "percent": round((done + failed) / total * 100, 1),
            "status": data.get(b"status", b"running").decode(),
        }

不推荐
  • 所有变量都写死在代码里，无法管理
  • 同步循环调用LLM生成N条（阻塞）
  • 无进度反馈，用户不知道处理到哪了
  • 失败直接跳过，无重试机制
推荐
  • Jinja2模板统一管理，变量分类清晰
  • MQ解耦+异步Worker并发处理
  • Redis记录进度，WebSocket推送
  • 失败任务自动重试3次+死信队列

常见问题（FAQ）

Jinja2的沙箱安全吗？用户输入会不会注入？
Jinja2默认模板沙箱是安全的，不支持任意Python代码执行。但变量值本身可能包含注入内容，建议对LLM生成的变量值做二次转义（HTML/JSON），防止跨站或格式破坏。
批量任务的并发数设多少合适？
取决于LLM API的QPS上限。qwen-plus建议并发5-10，qwen-turbo可到20-30。设太高会被限流429，设太低吞吐不够。建议用令牌桶做自适应限流。
LLM填充的变量质量不稳定怎么办？
1) 给LLM填充变量加few-shot示例；2) 对LLM输出用正则约束格式；3) 对关键变量（如价格）用固定变量而非LLM填充。

本章结论

  • 变量分类：固定变量模板渲染 + LLM变量动态填充，两类分治。
  • MQ批量解耦：RabbitMQ承载批任务，Worker并发消费，支持重试和死信。
  • 进度追踪：Redis记录批任务状态，事件驱动前端实时反馈。
  • 失败处理：自动重试3次 + 死信队列人工介入。
  • 质量管理：LLM填充变量加约束和few-shot，关键数据用固定变量。
