Python：FastAPI生成接口

python
# content_api.py — Python侧生成API
from fastapi import FastAPI, HTTPException
from fastapi.responses import StreamingResponse
from pydantic import BaseModel
from typing import Optional, AsyncGenerator
import json, asyncio, uuid, time

app = FastAPI(title="CorpAssist内容生成API")

class GenRequest(BaseModel):
    template_id: str
    variables: dict
    stream: bool = False
    temperature: float = 0.4
    max_tokens: int = 2048

class GenResponse(BaseModel):
    content_id: str
    text: str
    tokens_used: int
    latency_ms: int

@app.post("/api/v1/content/generate", response_model=GenResponse)
async def generate(req: GenRequest):
    """同步生成接口"""
    start = time.time()
    try:
        # 模拟LLM调用
        await asyncio.sleep(0.5)
        text = f"基于模板{req.template_id}生成的内容"
        return GenResponse(
            content_id=uuid.uuid4().hex[:12],
            text=text, tokens_used=150,
            latency_ms=int((time.time() - start) * 1000),
        )
    except Exception as e:
        raise HTTPException(status_code=500, detail=str(e))

@app.post("/api/v1/content/generate/stream")
async def generate_stream(req: GenRequest):
    """SSE流式生成接口"""
    async def event_stream() -> AsyncGenerator[str, None]:
        chunks = ["感谢您", "使用CorpAssist", "内容生成服务", "正在为您生成...", "完成"]
        for i, chunk in enumerate(chunks):
            await asyncio.sleep(0.3)
            yield f"data: {json.dumps({'token': chunk, 'index': i, 'done': i == len(chunks)-1})}\n\n"
    return StreamingResponse(event_stream(), media_type="text/event-stream")

@app.get("/api/v1/content/health")
async def health():
    return {"status": "ok", "service": "content-studio"}

Spring Boot编排层

Spring Boot通过Feign Client调用Python生成API，负责流程编排、权限校验和审计日志。

Java：Spring Feign客户端

java
// ContentStudioClient.java — Feign调用Python
@FeignClient(name = "content-studio", url = "${python.api.base:http://localhost:8000}")
public interface ContentStudioClient {

    @PostMapping("/api/v1/content/generate")
    GenResponse generate(@RequestBody GenRequest request);

    @PostMapping(value = "/api/v1/content/generate/stream",
                 consumes = "application/json")
    Flux generateStream(@RequestBody GenRequest request);
}

// GenResponse.java
@Data
public class GenResponse {
    private String contentId;
    private String text;
    private int tokensUsed;
    private int latencyMs;
}

// ContentController.java — 编排控制器
@RestController
@RequestMapping("/api/v2/content")
public class ContentController {
    private final ContentStudioClient pythonClient;
    private final AuditLogger auditLogger;

    @PostMapping("/generate")
    public ResponseEntity generate(@RequestBody GenRequest request) {
        long start = System.currentTimeMillis();
        GenResponse response = pythonClient.generate(request);
        auditLogger.log("content.generate", request.getTemplateId(),
                        System.currentTimeMillis() - start);
        return ResponseEntity.ok(response);
    }

    @PostMapping(value = "/generate/stream", produces = MediaType.TEXT_EVENT_STREAM_VALUE)
    public Flux generateStream(@RequestBody GenRequest request) {
        return pythonClient.generateStream(request)
            .timeout(Duration.ofSeconds(30))
            .onErrorResume(e -> Flux.just(new ChatChunk("服务超时，请稍后重试")));
    }
}

Python + Java：流式输出

流式输出采用SSE协议。Python侧将LLM生成的token逐块推送，Spring侧通过WebFlux的Flux接收并转发给前端。

SSE流式输出时序
sequenceDiagram
    participant F as 前端
    participant S as Spring
    participant P as Python
    participant L as LLM

    F->>S: POST /generate/stream
    S->>P: Feign POST /generate/stream
    P->>L: LLM调用
    L-->>P: Token逐块返回
    P-->>S: SSE data: token
    S-->>F: SSE data: token
    P-->>S: SSE data: [DONE]
    S-->>F: SSE data: [DONE]

SSE流式输出三跳传输：LLM→Python→Spring→前端。

异常处理与超时策略

双栈调用的异常处理是生产稳定性的关键。Python侧超时、LLM限流、Spring侧降级都需要覆盖。

异常场景Python处理Spring处理
LLM超时重试2次后返回504触发降级，返回缓存结果
LLM限流429指数退避重试返回「服务繁忙」提示
Python服务宕机-切换到备用Python实例
模板不存在返回404错误记录审计日志+返回给前端
内容审核拦截返回违规标签记录安全审计+通知运营

Python：异常处理中间件

python
# error_handler.py — 异常处理
from fastapi import Request
from fastapi.responses import JSONResponse
import traceback

@app.exception_handler(ValueError)
async def value_error_handler(request: Request, exc: ValueError):
    return JSONResponse(status_code=400, content={
        "error": "INVALID_REQUEST",
        "message": str(exc),
    })

@app.exception_handler(Exception)
async def general_error_handler(request: Request, exc: Exception):
    return JSONResponse(status_code=500, content={
        "error": "INTERNAL_ERROR",
        "message": "生成服务异常，请稍后重试",
        "request_id": request.headers.get("x-request-id", ""),
    })

Java：Spring降级配置

java
// ContentStudioFallback.java — 降级处理
@Component
public class ContentStudioFallback implements ContentStudioClient {
    @Override
    public GenResponse generate(GenRequest request) {
        // 返回降级提示
        GenResponse fallback = new GenResponse();
        fallback.setContentId("fallback");
        fallback.setText("内容生成服务暂时不可用，请稍后重试");
        return fallback;
    }

    @Override
    public Flux generateStream(GenRequest request) {
        return Flux.just(new ChatChunk("服务降级中，请稍后重试"));
    }
}
不推荐
  • Spring直接调用LLM SDK，不支持Python
  • 同步等待生成完成，阻塞Spring线程
  • Python和Spring无统一超时配置
  • 异常时直接返回500
推荐
  • Python专注LLM调用，Spring专注编排
  • SSE流式输出，前端实时展示
  • Feign超时30s+熔断降级
  • 友好错误提示+审计日志

常见问题（FAQ）

为什么不在Spring里直接调LLM？
Python的LLM生态（LangChain、Transformers）比Java成熟丰富。Jinja2、tiktoken、numpy等库在Python中调用更自然。双栈模式充分发挥各自优势：Python做AI，Java做工程。
SSE和WebSocket怎么选？
SSE（Server-Sent Events）是单向服务器推送，实现简单、原生HTTP支持、自动重连。WebSocket支持双向通信。内容生成场景只需要服务器推送到前端，SSE足够。SSE天然支持text/event-stream格式。
双栈部署需要哪些基础设施？
Python侧需要独立部署（Docker + FastAPI + uvicorn），Spring侧保持原有部署方式。中间加一层API网关统一入口。Python和Spring通过内部网络通信，建议走Kubernetes Service名而非IP。

本章结论

  • 分工明确：Python做LLM调用和模板渲染，Spring做编排和审计。
  • SSE流式：三跳传输（LLM→Python→Spring→前端），实时展示生成过程。
  • Feign调用：Spring通过Feign Client调用Python API，支持超时和降级。
  • 异常兜底：Python侧异常处理+Spring侧熔断降级，双保险。
  • 统一网关：API网关统一入口，Python和Spring对内网通信。
