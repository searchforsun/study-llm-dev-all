本章先记住3件事
  • 混合 > 单一：BM25补精确匹配，向量补语义相似，互补胜于单一。
  • Rerank必须做：Top-K粗排后Rerank精排，MRR可提升10-15%。
  • 权重靠实验：BM25/向量权重比（0.3/0.7或0.5/0.5）依赖数据集，必须验证。

BM25+向量混合检索

混合检索结合关键词的精确匹配和向量的语义相似，互补两类检索的缺陷。

混合检索管道
flowchart LR
    Query[用户查询] --> BM25[BM25关键词检索]
    Query --> Vector[向量语义检索]
    BM25 --> Fusion[分数融合]
    Vector --> Fusion
    Fusion --> Rerank[BGE-Reranker精排]
    Rerank --> TopK[Top-3结果]

查询同时走BM25和向量两条路径，分数融合后Rerank精排。

Python：混合检索实现

python
# hybrid_retriever.py — 混合检索
from rank_bm25 import BM25Okapi
from openai import OpenAI
import numpy as np
from typing import List, Dict

client = OpenAI(base_url="https://dashscope.aliyuncs.com/compatible-mode/v1")

class HybridRetriever:
    def __init__(self, bm25_weight: float = 0.3, vector_weight: float = 0.7):
        self.bm25_weight = bm25_weight
        self.vector_weight = vector_weight
        self.documents: List[str] = []
        self.bm25 = None

    def index(self, docs: List[str]):
        self.documents = docs
        tokenized = [doc.split() for doc in docs]
        self.bm25 = BM25Okapi(tokenized)

    def retrieve(self, query: str, top_k: int = 5) -> List[Dict]:
        # BM25分数
        tokenized_query = query.split()
        bm25_scores = self.bm25.get_scores(tokenized_query)
        bm25_norm = bm25_scores / (np.max(bm25_scores) + 1e-8)

        # 向量分数
        q_emb = client.embeddings.create(
            model="text-embedding-v3", input=query
        ).data[0].embedding
        vec_scores = np.array([
            np.dot(q_emb, self._get_emb(d))
            for d in self.documents
        ])
        vec_norm = vec_scores / (np.max(vec_scores) + 1e-8)

        # 融合
        combined = bm25_norm * self.bm25_weight + vec_norm * self.vector_weight
        top_indices = np.argsort(combined)[-top_k:][::-1]

        return [
            {"doc": self.documents[i], "score": float(combined[i]),
             "bm25": float(bm25_norm[i]), "vector": float(vec_norm[i])}
            for i in top_indices
        ]

    def _get_emb(self, text: str) -> np.ndarray:
        resp = client.embeddings.create(model="text-embedding-v3", input=text[:512])
        return np.array(resp.data[0].embedding)

@Service
public class HybridSearchService {
    private final VectorStore vectorStore;
    private final BM25Service bm25Service;

    public List hybridSearch(String query, double bm25Weight) {
        // 向量检索

            SearchRequest.query(query).withTopK(10));

        // BM25检索
        var bm25Results = bm25Service.search(query, 10);

        // 分数融合 (RRF)
        Map fused = new HashMap<>();
        for (int i = 0; i < vectorResults.size(); i++) {
            fused.merge(vectorResults.get(i).getId(),
                1.0 / (i + 60) * (1 - bm25Weight), Double::sum);
        }
        for (int i = 0; i < bm25Results.size(); i++) {
            fused.merge(bm25Results.get(i).getId(),
                1.0 / (i + 60) * bm25Weight, Double::sum);
        }
        return fused.entrySet().stream()
            .sorted(Map.Entry.comparingByValue().reversed())
            .limit(5)
            .map(e -> /* 查询文档详情 */)
            .collect(Collectors.toList());
    }
}
先想10秒：BM25权重0.3和0.7分别适合什么场景？

BM25权重0.3适合语义检索为主（如问答、文案相似匹配），0.7适合关键词精确匹配（如SKU查找、品牌名精确匹配）。推荐从0.5/0.5开始，用验证集调参。

BGE-Reranker精排

粗排后Top-K结果可能包含不相关文档。BGE-Reranker对每个候选计算相关性分数，重新排序。

Python：Reranker接入

python
# reranker.py — BGE-Reranker精排
from sentence_transformers import CrossEncoder
from typing import List, Dict

class BGEReranker:
    def __init__(self, model_name: str = "BAAI/bge-reranker-v2-m3"):
        self.model = CrossEncoder(model_name)

    def rerank(self, query: str, candidates: List[Dict], top_k: int = 3) -> List[Dict]:
        pairs = [(query, c["doc"][:512]) for c in candidates]
        scores = self.model.predict(pairs)
        for i, c in enumerate(candidates):
            c["rerank_score"] = float(scores[i])

        return sorted(candidates, key=lambda c: c["rerank_score"], reverse=True)[:top_k]

retriever = HybridRetriever()
retriever.index(["文档1内容...", "文档2内容..."])
candidates = retriever.retrieve("用户查询", top_k=10)

reranker = BGEReranker()
top3 = reranker.rerank("用户查询", candidates, top_k=3)
for t in top3:
    print(f"Rerank: {t['rerank_score']:.3f} | {t['doc'][:50]}")

失败模式与兜底

失败模式原因检测方法兜底策略
零结果素材库无相关内容检索结果数为0回退LLM直接生成+免责声明
低分结果查询太宽泛最高分<阈值触发关键词扩展/同义词
语义漂移向量检索主题偏离Rerank分数<0.3增加BM25权重
时效偏差检索到过期素材素材时间戳检查按时间倒序加权
不推荐
  • 只用向量检索，遗漏精确匹配
  • BM25和向量权重拍脑袋（如总设0.5/0.5）
  • Top-K固定取5，不管质量
  • 不处理零结果场景
推荐
  • BM25+向量混合+RRF融合
  • 权重通过网格搜索确定
  • 粗排10条→Rerank精排3条
  • 零结果/低分触发兜底策略

常见问题（FAQ）

BM25和向量权重怎么确定？
用标注数据集做网格搜索。取BM25 weight=[0.1,0.3,0.5,0.7,0.9]，在验证集上算MRR，取最高值。一般0.3/0.7（向量主导）或0.5/0.5（均衡）效果较好。
Reranker一定要用BGE吗？
BGE-reranker是开源选项之一。也可用Cohere Rerank、通义百炼的Rerank API。选型的核心是延迟：CrossEncoder比BiEncoder慢，但精度更高。生产建议用百炼Rerank API降低运维。
混合检索比纯向量慢多少？
BM25+向量两道检索约增加50-100ms延迟。如果在10K级别素材上，总检索延迟约200-300ms（含Rerank）。对于内容生成场景（LLM调用本身秒级），这个延迟可接受。

本章结论

  • BM25+向量互补：精确匹配+语义相似，优于任何单一检索方式。
  • 权重可调：BM25(0.3)/向量(0.7)为推荐起点，必须用数据验证。
  • BGE-Reranker精排：Top-10粗排后精排到Top-3，MRR可提升10-15%。
  • 兜底策略：零结果/低分/语义漂移/时效偏差四种失败模式需分别处理。
  • 评估指标：用召回率、MRR、Rerank后准确率评估检索质量。
