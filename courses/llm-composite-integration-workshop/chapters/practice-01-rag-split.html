RAG 分工设计：索引 vs 问答

rag split 将 RAG 系统拆分为两个独立子领域：索引（Indexing） 和 问答（Querying）。

维度索引（Python）问答（Spring / Python）
职责 文档解析、分块、向量化、写入 接收查询、检索、生成回答
调用频率 低频（每天几次～几天一次） 高频（每秒几十～几百次）
延迟要求 不敏感（分钟到小时级） 敏感（P99 < 5 秒）
向量库权限 读写 只读（vector readonly）
技术栈 Python + Celery Spring AI + 可选 Python

索引管线在 Python，问答编排在 Spring，向量库作为共享只读数据层。

Python 索引管线

python indexing 是 CorpAssist 的索引引擎，由 Celery Worker 异步执行。支持的文档类型包括 PDF、Word、Markdown、Confluence 导出等。

Python 索引管线完整实现

python
# Python 索引管线 - indexing_pipeline.py
from celery import Celery
from langchain_community.document_loaders import UnstructuredFileLoader
from langchain.text_splitter import RecursiveCharacterTextSplitter
from langchain_openai import OpenAIEmbeddings
from langchain_milvus import Milvus
import hashlib
from pathlib import Path

celery_app = Celery("corpassist_indexer",
    broker="redis://redis:6379/0",
    backend="redis://redis:6379/1")

@celery_app.task(bind=True, max_retries=3, default_retry_delay=60)
def index_document(self, file_path: str, tenant_id: str,
                  knowledge_base_id: str) -> dict:
    """索引单个文档"""
    try:
        # 1. 文档解析
        loader = UnstructuredFileLoader(file_path)
        documents = loader.load()

        # 2. 文档分块
        text_splitter = RecursiveCharacterTextSplitter(
            chunk_size=1000,
            chunk_overlap=200,
            separators=["\n\n", "\n", "。", "！", "？", " ", ""],
        )
        chunks = text_splitter.split_documents(documents)

        # 3. 添加元数据
        for chunk in chunks:
            chunk.metadata["tenant_id"] = tenant_id
            chunk.metadata["kb_id"] = knowledge_base_id
            chunk.metadata["chunk_hash"] = hashlib.md5(
                chunk.page_content.encode()).hexdigest()
            chunk.metadata["indexed_at"] = datetime.utcnow().isoformat()

        # 4. 写入向量库
        vector_store = Milvus(
            embedding_function=OpenAIEmbeddings(),
            collection_name=f"kb_{knowledge_base_id}",
            connection_args={"host": "milvus", "port": "19530"},
        )
        vector_store.add_documents(chunks)

        return {
            "status": "success",
            "file": file_path,
            "chunks": len(chunks),
            "tenant_id": tenant_id,
        }

    except Exception as e:
        self.retry(exc=e)

def index_batch(file_paths: list[str], tenant_id: str, kb_id: str):
    """批量提交索引任务"""
    tasks = []
    for fp in file_paths:
        task = index_document.delay(fp, tenant_id, kb_id)
        tasks.append(task.id)
    return tasks

索引版本化管理

index pipeline 支持索引版本化管理：每次全量重建时创建新 collection，切换流量到新版本，旧版本保留用于回滚。

python
# 索引版本管理
def create_index_version(kb_id: str, tenant_id: str) -> str:
    """创建新的索引版本"""
    version = f"kb_{kb_id}_v{int(time.time())}"
    # 创建新 collection
    vector_store = Milvus(
        embedding_function=OpenAIEmbeddings(),
        collection_name=version,
        connection_args={"host": "milvus", "port": "19530"},
    )
    # 记录版本元数据
    redis_client.hset(f"index_version:{kb_id}", "current", version)
    return version

def switch_index_version(kb_id: str, version: str):
    """切换索引版本"""
    redis_client.hset(f"index_version:{kb_id}", "current", version)
    redis_client.publish("index_switch", json.dumps({
        "kb_id": kb_id,
        "version": version,
        "switched_at": datetime.utcnow().isoformat(),
    }))

Spring 问答编排

spring orchestration 层使用 Spring AI 框架编排 RAG 问答流程。它负责：接收用户查询→检索相关文档→组装 LLM 上下文→生成回答。

Spring RAG 问答服务

java
// Spring RAG 问答编排服务
@Service
public class RagQueryService {

    private final VectorStore vectorStore;
    private final ChatClient chatClient;

    public RagQueryService(VectorStore vectorStore, ChatClient.Builder chatBuilder) {
        this.vectorStore = vectorStore;
        this.chatClient = chatBuilder
            .defaultSystem("""
                你是一个专业的企业知识助手。请基于提供的上下文回答用户问题。
                如果上下文不足以回答问题，请说明「无法从已有知识库中找到答案」。
                引用格式：[来源: 文档标题, 页码]
                """)
            .build();
    }

    public RagQueryResponse query(RagQueryRequest request, String tenantId) {
        // 1. 向量检索（按租户隔离）
        List<Document> documents = vectorStore.similaritySearch(
            SearchRequest.query(request.query())
                .withTopK(request.topK())
                .withFilterExpression("tenant_id == '" + tenantId + "'"));

        // 2. 组装 Prompt
        String context = documents.stream()
            .map(doc -> doc.getContent() + "\n[来源: " + doc.getMetadata().get("source") + "]")
            .collect(Collectors.joining("\n---\n"));

        // 3. LLM 生成
        String answer = chatClient.prompt()
            .user(u -> u.text("问题: {query}\n\n上下文:\n{context}")
                .param("query", request.query())
                .param("context", context))
            .call()
            .content();

        // 4. 返回结果
        List<SourceRef> sources = documents.stream()
            .map(doc -> new SourceRef(
                (String) doc.getMetadata().get("title"),
                (String) doc.getMetadata().get("source"),
                (Double) doc.getMetadata().get("distance")))
            .toList();

        return new RagQueryResponse(answer, sources);
    }
}

RAG 双栈联调完整流程：Python 做索引写入，BFF 做问答编排。

不推荐
  • Spring 直接调用 LLM 做索引嵌入
  • Python 既做索引也做问答，没有职责分离
  • 索引和问答共享同一个 Milvus collection
  • 索引完成没有通知，需要轮询状态
推荐
  • Python 专注索引写入，Spring 专注问答编排
  • 索引使用独立 collection 版本化管理
  • 索引完成后通过 Webhook 通知 BFF
  • 即使 Python 索引服务不可用，Spring 问答仍能使用已有索引

常见问题（FAQ）

为什么不在 Spring 中也实现索引功能？
索引需要大量 Python 生态库支持：Unstructured（文档解析）、Sentence-Transformers（本地嵌入）、PyMuPDF（PDF 处理）。Java 在这些领域的生态远不如 Python。此外，索引是低频率的批处理任务，与高频低延迟的问答服务混在一起会影响稳定性。
索引完成后如何通知 Spring BFF？
两种方式：1) Python 索引完成后通过 Webhook 回调 BFF 的 /api/v1/callbacks/index-complete 端点；2) 通过 MQ 发送 IndexCompleted 事件，BFF 监听消费。推荐 MQ 方式，解耦更强。详见 同步、异步与 MQ。
向量库中的旧索引如何处理？
使用索引版本化管理。每次全量重建创建新 collection（如 kb_123_v2），BFF 查询时从 Redis 获取当前活跃版本号。旧版本保留一段时间（如 7 天）用于回滚，之后自动清理。增量索引直接在原 collection 中追加，不需要版本切换。
Spring AI 的 VectorStore 支持 Milvus 的租户过滤吗？
支持。Spring AI 的 MilvusVectorStore 支持 FilterExpression，可以在检索时添加 tenant_id == 'xxx' 的表达式。CorpAssist 的每篇文档在索引时都写入了 tenant_id 元数据字段，检索时通过 filterExpression 参数实现租户隔离。

本章结论

  • 职责分离：Python 索引管线负责文档处理/向量化/写入，Spring 问答编排负责检索/LLM 生成
  • 只读查询：向量库对问答服务是只读的（vector readonly），避免写入和查询互相影响
  • 版本化管理：索引版本支持流量切换和回滚
  • 通知机制：索引完成后通过 Webhook/MQ 通知 BFF
