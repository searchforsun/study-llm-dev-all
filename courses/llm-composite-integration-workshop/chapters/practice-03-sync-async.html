异步长任务模式：202 Accepted + Job Status

long running 任务使用 202 Accepted + Job Status 模式。请求立即返回 job_id，客户端通过 job_id 轮询或等待回调。

异步长任务流程
sequenceDiagram
    participant C as 客户端
    participant BFF as Spring BFF
    participant DB as PostgreSQL
    participant MQ as RabbitMQ
    participant W as Python Worker

    C->>BFF: POST /api/v1/index/batch\n{files: [...]}
    BFF->>DB: INSERT job (status=pending)
    DB-->>BFF: job_id
    BFF->>MQ: Publish IndexJobCreated
    BFF-->>C: 202 Accepted\n{job_id, status_url}

    MQ->>W: Consume IndexJobCreated
    W->>W: 执行索引（分钟级）
    W->>DB: UPDATE job (status=completed)
    W->>MQ: Publish IndexJobCompleted

    MQ->>BFF: Consume IndexJobCompleted
    BFF->>BFF: 更新缓存/通知

    C->>BFF: GET {status_url}
    BFF-->>C: 200 {status: "completed", stats}

异步长任务模式：202 Accepted 立即返回，MQ 解耦执行过程，Job Status 查询结果。

Spring 异步任务 API

java
// Spring 异步任务 API
@RestController
@RequestMapping("/api/v1/index")
public class IndexController {

    private final JobRepository jobRepository;
    private final RabbitTemplate rabbitTemplate;

    @PostMapping("/batch")
    public ResponseEntity<JobResponse> submitBatch(
            @RequestBody @Valid BatchIndexRequest request,
            @RequestHeader("X-Tenant-Id") String tenantId) {

        // 1. 创建任务记录
        IndexJob job = IndexJob.builder()
            .id(UUID.randomUUID().toString())
            .tenantId(tenantId)
            .type("index_batch")
            .status(JobStatus.PENDING)
            .params(objectMapper.writeValueAsString(request))
            .createdAt(Instant.now())
            .build();
        jobRepository.save(job);

        // 2. 发送 MQ 消息
        rabbitTemplate.convertAndSend("corpassist.index",
            "job.created",
            new IndexJobEvent(job.getId(), tenantId, request));

        // 3. 返回 202 Accepted
        URI statusUrl = URI.create(
            "/api/v1/index/jobs/" + job.getId());
        return ResponseEntity.accepted()
            .location(statusUrl)
            .body(new JobResponse(job.getId(), "pending",
                statusUrl.toString()));
    }

    @GetMapping("/jobs/{jobId}")
    public ResponseEntity<IndexJob> getJobStatus(
            @PathVariable String jobId) {
        return jobRepository.findById(jobId)
            .map(ResponseEntity::ok)
            .orElse(ResponseEntity.notFound().build());
    }
}

Python 异步 Worker 消费

python
# Python Celery Worker - 消费索引任务
from celery import Celery
import httpx

celery = Celery("corpassist_index",
    broker="redis://redis:6379/0",
    backend="redis://redis:6379/1")

@celery.task(bind=True, max_retries=3, acks_late=True)
def process_index_job(self, job_id: str, tenant_id: str, files: list):
    """处理索引任务"""
    try:
        # 1. 更新任务状态为 processing
        update_job_status(job_id, "processing")

        # 2. 执行索引
        result = perform_indexing(files, tenant_id)

        # 3. 更新完成状态
        update_job_status(job_id, "completed", result=result)

        # 4. 通过 Webhook 回调 BFF
        callback_url = get_callback_url(job_id)
        if callback_url:
            httpx.post(callback_url, json={
                "job_id": job_id,
                "status": "completed",
                "result": result,
            })

    except Exception as exc:
        update_job_status(job_id, "failed", error=str(exc))
        self.retry(exc=exc)

def update_job_status(job_id: str, status: str, **extra):
    """通过 DB API 更新任务状态"""
    httpx.patch(
        f"http://spring-bff:8081/api/v1/index/jobs/{job_id}",
        json={"status": status, **extra},
        timeout=5,
    )

Spring 事件发布与消费

java
// Spring 异步事件监听
@Component
public class IndexJobEventListener {

    private final JobRepository jobRepository;

    @RabbitListener(queues = "corpassist.index.callback")
    public void handleIndexCompleted(IndexCompletedEvent event) {
        // 1. 更新数据库中任务状态
        jobRepository.findById(event.jobId()).ifPresent(job -> {
            job.setStatus(JobStatus.COMPLETED);
            job.setCompletedAt(Instant.now());
            job.setResult(event.result());
            jobRepository.save(job);
        });

        // 2. 触发后续操作（如更新索引版本）
        log.info("Index job completed: jobId={}, stats={}",
            event.jobId(), event.result());
    }
}

  • 长任务用同步 HTTP 等待数分钟

  • idempotency key 防重复提交
  • 提供任务状态查询和列表端点
  • MQ 消息使用 acks_late + retry

幂等设计

idempotency key 用于防止重复提交。客户端在请求中携带唯一的幂等键，服务端检测到重复键时返回已处理的结

果而不是重新执行。

Spring Idempotency 过滤器

java
// Spring 幂等过滤器
@Component
public class IdempotencyFilter implements WebFilter {

    private final StringRedisTemplate redis;

    @Override
    public Mono<Void> filter(ServerWebExchange exchange, WebFilterChain chain) {
        String idempotencyKey = exchange.getRequest()
            .getHeaders().getFirst("Idempotency-Key");

        if (idempotencyKey == null) {
            return chain.filter(exchange);
        }

        String cacheKey = "idempotency:" + idempotencyKey;

        return redis.opsForValue().get(cacheKey)
            .flatMap(cached -> {
                // 已处理过，返回上次结果
                ServerHttpResponse response = exchange.getResponse();
                response.setStatusCode(HttpStatus.OK);
                return response.writeWith(
                    Mono.just(response.bufferFactory()
                        .wrap(cached.getBytes(StandardCharsets.UTF_8))));
            })
            .switchIfEmpty(
                chain.filter(exchange).then(
                    // 缓存响应结果
                    Mono.defer(() -> cacheResponse(exchange, cacheKey))
                )
            );
    }
}

Python Idempotency 装饰器

python
# Python 幂等装饰器
from functools import wraps
from redis import Redis
import json

redis_client = Redis(host="redis", port=6379, db=0)

def idempotent(ttl_seconds: int = 86400):
    """幂等装饰器：Idempotency-Key 去重"""
    def decorator(func):
        @wraps(func)
        async def wrapper(request: Request, *args, **kwargs):
            idempotency_key = request.headers.get("Idempotency-Key")
            if not idempotency_key:
                return await func(request, *args, **kwargs)

            cache_key = f"idempotency:{idempotency_key}"
            cached = redis_client.get(cache_key)
            if cached:
                return JSONResponse(
                    content=json.loads(cached),
                    status_code=200,
                )

            response = await func(request, *args, **kwargs)
            if response.status_code == 200:
                redis_client.setex(
                    cache_key,
                    ttl_seconds,
                    json.dumps(response.body, ensure_ascii=False),
                )
            return response
        return wrapper
    return decorator

常见问题（FAQ）

202 Accepted 返回后，如果 Worker 挂了，任务会丢失吗？
不会。如果 Worker 在处理过程中崩溃，MQ 消息会重新入队（ack_late 模式）并重新投递到其他 Worker。如果 MQ 本身不可用，消息会持久化到磁盘，恢复后重新消费。如果任务已经提交到 Worker 但结果未持久化，BFF 的任务状态会保持 processing，可以设置超时检测并自动重置为 pending。
Idempotency-Key 存储多久？
CorpAssist 默认存储 24 小时（TTL=86400s），这与大多数业务场景的重复请求时间窗口一致。幂等键存储太久会占用 Redis 空间，太短则可能在 TTL 过期后用户重试导致重复。建议根据业务特性调整 TTL：索引任务 7 天，查询类任务 1 小时。
Python Worker 如何回传结果给 Spring BFF？
三种方式：1) MQ 回调：Python Worker 完成后通过 MQ 发送事件，Spring 监听消费；2) Webhook 回调：Python Worker 直接调用 BFF 的 /api/v1/index/callback；3) DB 共享：Worker 写入共享数据库，BFF 查询。推荐 MQ 方式，松耦合且支持重试。
同步和异步可以混用吗？
可以。CorpAssist 中 RAG 问答使用同步 SSE，索引任务使用异步 MQ。一个请求可以拆分为同步部分（立即返回初步结果）和异步部分（后台继续处理）。例如文档上传：同步 202 返回 job_id，异步索引完成后通知。关键的决策因素是「用户是否在等待该结果」。

本章结论

  • 模式选择：< 30s 用同步/SSE，> 30s 用异步 202 + Job Status
  • MQ 解耦：RabbitMQ 作为 Spring-Python 异步通道，事件驱动业务编排
  • 幂等：Idempotency-Key 防重复提交，Redis 缓存已处理结果
  • 状态管理：Job 状态机（pending→processing→completed/failed）支持追踪
