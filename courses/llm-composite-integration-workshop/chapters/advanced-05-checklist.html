常见问题（FAQ）

自检中有些能力我还不太熟，怎么办？
这是正常现象。自检的目的是找到差距而不是追求完美。对于不熟的能力：1) 回顾对应的课程章节；2) 完成对应的 Demo 练习；3) 在真实项目中实践。建议将自检结果保存，3 个月后再做一次对比进步。
我主要用 Python，Java 部分不熟怎么办？
双栈课程的目标不是让你成为 Java/Python 的专家，而是理解两边的协作方式。如果你主要是 Python 开发者，至少需要理解：1) Spring 治理面的职责和边界；2) 双栈之间的通信方式（OpenAPI/JWT/Header）；3) Docker Compose 整体架构。具体到 Java 代码实现，能读懂即可。
毕业后还有什么课程可以继续学习？
推荐进阶路径：1) 上下文工程（context-memory-engineering）—— 深入学习 LLM 上下文窗口管理；2) RAG 评估（llm-evaluation-quality）—— 学习评估 RAG 系统的准确性；3) LLM 应用后端（llm-application-backend）—— 深入学习分布式治理和微服务；4) A2A/Nacos 集成（ai-agent-protocol）—— 学习 Agent 间通信协议。
自检表对找工作有什么用？
自检表可以直接转化为简历技能列表和面试回答框架。例如：JD 要求「熟悉 RAG 系统」→ 你在自检中可以找到「LangChain RAG 实现、向量数据库集成、RAG 索引问答分离」对应的知识和经验。在面试中展示自检表，能系统性地证明你的双栈能力。
