留痕

审计事件应结构化存储，支持按 tenant、user、session、trace 检索。敏感字段脱敏规则与隐私与脱敏一致。

CorpAssist 审计事件流
sequenceDiagram
    participant U as 用户
    participant G as 网关
    participant A as Agent
    participant T as 工具/RAG
    participant L as 审计存储
    U->>G: 请求 + JWT

Python：审计写入

python 
from dataclasses import dataclass, asdict
import hashlib, json, logging

audit_log = logging.getLogger("corpassist.audit")

@dataclass
class AuditEvent:
    event_type: str
    trace_id: str
    tenant_id: str
    user_id: str
    payload: dict

    def emit(self):
        body = asdict(self)
        body["user_id_hash"] = hashlib.sha256(self.user_id.encode()).hexdigest()[:16]
        del body["user_id"]
        audit_log.info(json.dumps(body, ensure_ascii=False))

Java：Spring AI 审计 Advisor

java 
@Component
public class AuditTrailAdvisor implements CallAroundAdvisor {
    private final AuditEventPublisher publisher;

    @Override
    public AdvisedResponse aroundCall(AdvisedRequest req, CallAroundAdvisorChain chain) {
        long start = System.currentTimeMillis();
        AdvisedResponse resp = chain.nextAroundCall(req);
        publisher.publish(AuditEvent.builder()
            .traceId(MDC.get("traceId"))
            .tenantId(req.context().get("tenantId", String.class))
            .latencyMs(System.currentTimeMillis() - start)
            .refusalCode(req.context().get("refusalCode", String.class))
            .build());
        return resp;
    }
}

引用链

citation chain 将模型回答与检索片段绑定：用户看到「来源」链接，审计日志保存 doc_id、chunk_offset、similarity_score。

字段用途用户可见
doc_id知识库文档唯一标识是（文档标题）
chunk_id片段 ID否（内部）
score向量相似度可选（高级模式）
snippet_hash防篡改校验否
python 
def build_citation_chain(chunks: list[dict]) -> dict:
    citations = []
    for c in chunks:
        citations.append({
            "doc_id": c["doc_id"],
            "title": c["title"],
            "chunk_id": c["chunk_id"],
            "score": round(c["score"], 4),
            "url": f"/kb/docs/{c['doc_id']}#chunk-{c['chunk_id']}"
        })
    return {"citations": citations, "citation_count": len(citations)}

等保

mlps compliance（网络安全等级保护）要求日志留存、完整性保护与访问控制。LLM 审计事件应对照以下映射：

等保控制项CorpAssist 落地审计字段
安全审计（G3）全量 LLM 请求/响应留痕event_type, timestamp, user_id_hash
数据完整性（S3）审计日志 WORM 存储 + hashpayload_hash, storage_tier
个人信息保护PII 脱敏后入库redact_fields_applied
集中管控SIEM 接入 OpenTelemetrytrace_id, service.name
检查理解：审计日志能否记录完整用户手机号？

不能。与隐私与脱敏一致，日志仅存 hash 或掩码；完整 PII 仅在有合法事由的独立系统中短期留存并审批。

审计日志保留多久？
金融/政务常见 ≥6 个月在线、≥3 年归档；以企业合规为准。WORM 桶防止篡改。
OpenTelemetry 与 audit 事件关系？
trace/span 供性能排障；audit 事件供合规举证。同一 trace_id 关联，但 audit 字段更丰富（policy_id、refusal_code、citations）。

audit trail 抽样复核流程（等保测评常用）：

  1. 随机抽取 20 条 completion 事件，验证 trace_id 在网关/Agent/工具日志均可检索。
  2. 对有 citation 的回答，点开 doc_id 核对 snippet 与当时 KB 版本一致。
  3. grep 审计存储，确认无 11 位明文手机号。

本章结论

  • 留痕：结构化 audit 事件，trace_id 全链路关联，PII 脱敏。

  • 等保：审计字段映射等保控制项，支持 SIEM 与 WORM 存储。
