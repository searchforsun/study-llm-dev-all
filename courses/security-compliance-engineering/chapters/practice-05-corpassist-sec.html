验收

验收分三阶段：自动化脚本 → 双栈 spot check → 安全负责人签字。输出 acceptance-report.json 供行业合规差异章引用。每项 SEC 检查点对应可执行证据，避免「口头通过」。

不推荐
  • 仅勾选 Excel，无可复现脚本
  • 单栈通过即上线
  • 跳过红队 hotfix 直发生产
推荐
  • run_baseline_acceptance.py 产出 JSON 报告
  • Python/Java 各 spot check 3 条
  • 报告归档 WORM + 关联 release tag

Python：一键验收编排

python 
#!/usr/bin/env python3
"""CorpAssist 安全基线验收编排"""
import subprocess, json, sys
from pathlib import Path

CHECKS = [
    ("SEC-01", ["python", "verify_tool_authz.py"]),
    ("SEC-03", ["python", "verify_refusal.py"]),
    ("SEC-05", ["python", "verify_audit.py"]),
    ("SEC-08", ["python", "run_red_team.py", "--suite", "cases/l1-smoke.json"]),
]

def main():
    results = []
    for item_id, cmd in CHECKS:
        r = subprocess.run(cmd, capture_output=True, text=True)
        ok = r.returncode == 0 and "PASS" in r.stdout
        results.append({"id": item_id, "pass": ok, "output": r.stdout.strip()[:200]})
    report = {"baseline": "2026.06", "items": results, "all_pass": all(x["pass"] for x in results)}
    Path("acceptance-report.json").write_text(json.dumps(report, indent=2), encoding="utf-8")
    print("PASS: baseline accepted" if report["all_pass"] else "FAIL: see acceptance-report.json")
    sys.exit(0 if report["all_pass"] else 1)

if __name__ == "__main__":
    main()

基线版本 2026.06 与 audit trail 字段 baseline_version 对齐，便于事后证明「该次发布已通过哪版清单」。

动手练习

运行 CorpAssist 安全基线一键验收，补齐 acceptance-report.json 并勾选 SEC-01～08（约 35 分钟）。

操作步骤

  1. 打开基线清单表，确认 SEC-01～08 的「证据」列理解无误。
  2. 在 corpassist-security-baseline.yaml 增加 SEC-07 PII 检查项。
  3. 运行 python run_baseline_acceptance.py，收集各子脚本 PASS 输出。
  4. 若任一项 FAIL，根据关联章回修并重新验收直至 all_pass: true。
