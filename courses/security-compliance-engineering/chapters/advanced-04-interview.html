典型面试题 1：injection interview · 注入防护

题面：你的 LLM 应用怎么防止 prompt 注入攻击？

text 
CORAL 回答示例：

C (Context): 在 CorpAssist 中，用户通过客服输入框发送消息，
这些消息是「不可信输入」，可能包含注入载荷。

O (Objective): 目标是阻止用户输入覆盖系统指令、窃取上下文
或诱导模型执行未授权的行为。

R (Rationale): 没有单层防御能防住所有注入，需要纵深防御：
提示工程 + 输入检测 + 指令层级 + 输出审核 + 工具鉴权。

A (Action):
1. 指令层级：system prompt 声明规则优先级，用  标签
2. 输入检测：关键词 + Moderation API + Base64 解码
3. 输出审核：拦截包含敏感信息的回复
4. 工具鉴权：所有 tool call 经 OPA 策略引擎决策
5. 红队回归：90+ 攻击用例自动测试

L (Learning): 一开始只靠 system prompt，被绕过后增加了分层防御。
最大的教训是「模型越聪明越会执行恶意指令」——防御不能依赖模型自觉。

Java：安全面试代码题

题面：实现一个简单的输入安全检查器，检测注入关键词和 PII。

java 
import java.util.List;
import java.util.regex.Pattern;

public class SimpleInputSanitizer {
    private static final List BLOCKED_KEYWORDS = List.of(
        "ignore", "forget", "jailbreak", "DAN", "system prompt"
    );

    private static final List PII_PATTERNS = List.of(
        Pattern.compile("\\b1[3-9]\\d{9}\\b"),  // 手机号
        Pattern.compile("\\b\\d{18}\\b")          // 身份证
    );

    public static class SanitizeResult {
        public final boolean blocked;
        public final String reason;
        public final String sanitizedText;

        public SanitizeResult(boolean blocked, String reason, String text) {
            this.blocked = blocked;
            this.reason = reason;
            this.sanitizedText = text;
        }
    }

    public SanitizeResult check(String input) {
        // 关键词检查
        String lower = input.toLowerCase();
        for (String kw : BLOCKED_KEYWORDS) {
            if (lower.contains(kw)) {
                return new SanitizeResult(true, "关键字: " + kw, input);
            }
        }
        // PII 脱敏
        String sanitized = input;
        for (Pattern p : PII_PATTERNS) {
            sanitized = p.matcher(sanitized)
                .replaceAll(m -> m.group().substring(0, 3) + "****"
                    + m.group().substring(m.group().length() - 4));
        }
        boolean hasPII = !sanitized.equals(input);
        return new SanitizeResult(false,
            hasPII ? "PII 已脱敏" : "通过", sanitized);
    }
}

// 用法
var sanitizer = new SimpleInputSanitizer();
var result = sanitizer.check("我的手机号是13800138000");
// result.blocked = false
// result.sanitizedText = "我的手机号是138****8000"

Python：安全设计面试答案

python 
# 面试题：你怎么验证安全措施是有效的？
class SecurityValidationFramework:
    """
    安全验证框架 —— 三层验证确保安全措施有效。

    1. 单元验证: 每个安全组件有独立的测试用例
    2. 集成验证: 攻击用例端到端自动测试
    3. 红队验证: 人类安全专家模拟高级攻击

    答案示例：
    C: CorpAssist 上线前需要安全验证
    O: 确保所有已知攻击手法都被防御
    R: 自动化测试保证每次变更后安全水位不下降
    A: 90+ 攻击用例自动回归 + 季度红队演练
    L: 首次演练发现 3 个绕过，已加入用例库
    """

    @staticmethod
    def validate_security():
        """三步验证"""
        # Step 1: 单元测试 — 每个安全组件
        assert PIIDetector.redact("手机13800138000") == "手机138****8000"
        assert DelimiterGuard.wrap("") == "</system>"

        # Step 2: 集成测试 — 攻击用例
        runner = RedTeamRunner("http://localhost:8080", "key")
        cases = runner.load_corpus("attack-corpus.json")
        results = runner.run_all(cases)
        pass_rate = sum(1 for r in results if r.passed) / len(results)
        assert pass_rate > 0.95, f"Pass rate {pass_rate:.0%} < 95%"

        # Step 3: 审计验证
        assert AuditLogger.query_by_request("req-001") is not None

        return "All security validations passed"

安全面试用 CORAL/STAR 组织回答，展示纵深防御与验证闭环。

面试官问「你们遇到过什么安全事件」怎么答？
STAR 框架：S（Situation）：某次 CorpAssist 被红队发现可被注入。T（Task）：需要 48 小时内修复。A（Action）：增加了指令层级 + 输入检测 + OPA 策略 + 回归测试。R（Result）：修复后相同攻击无法绕过，24 小时完成。
被问到「你理解的 defense in depth 在 LLM 场景怎么体现」？
四层：1) Prompt 工程（指令层级、分隔符），2) 应用层检测（关键词、Moderation API），3) 运行时控制（工具鉴权、输出审核），4) 审计验证（日志、红队）。每层独立，任何一层被绕过还有下一层兜底。
LLM 安全工程师的面试题趋势？
2026 年趋势：1) Agent 安全（工具鉴权、MCP 安全），2) 合规落地（数据出境、行业基线），3) RAG 安全（间接注入、引用链），4) 策略即代码（OPA）。纯知识题减少，security QA 场景设计题增加。
面试官追问「你怎么证明方案有效」怎么答？
三层验证：单元测试（策略规则）、集成回归（攻击语料 pass rate >95%）、红队演练（季度）。CORAL 的 L（Lesson）段须给出可量化指标，而非「我们加强了 prompt」。
安全与产品排期冲突时怎么说？
STAR：说明风险（未鉴权工具可删库）、给出分阶段方案（先只读工具上线 + 48h 内补鉴权）、展示过往红队数据支撑优先级。避免只说「安全很重要」。
答辩常见扣分点
  • 只背 OWASP 编号 — 须结合 CorpAssist 数据流讲检测点与负责模块。
  • 无验证闭环 — 方案须交代回归语料、审计字段与 fail-closed 行为。
  • 忽略双栈 — Java/Python 任一侧未设防即完整攻击面。
  • 无量化结果 — 只说「做了红队」不说 pass rate，面试官会追问指标。
答辩前 15 分钟怎么自检？
过一遍 CORAL 三题：每题是否含 Context、可验证 Action、Lesson 指标；对照纵深防御表能否指到 CorpAssist 具体模块名。

本章结论

  • CORAL 框架：Context → Objective → Rationale → Action → Learning 适合安全设计题。
  • 纵深防御 4 层：Prompt → 应用 → 运行时 → 审计。
  • 9 要素安全架构：威胁建模 → 输入 → 输出 → 鉴权 → 数据 → 审计 → 策略 → 红队 → 合规。
  • LLM 安全独特挑战：间接注入、工具鉴权、RAG 溯源、政策对齐。
  • 验证闭环：单元测试 + 集成回归 + 红队演练三层验证。
