评估脚本：自动拉取子处理器

python 
# vendor_assessment.py — 对比子处理器清单
import json
from pathlib import Path

REQUIRED = {"region", "purpose", "data_categories", "dpa_covered"}

def score_vendor(name: str, subprocessors: list[dict]) -> dict:
    gaps = []
    for sp in subprocessors:
        missing = REQUIRED - set(sp.keys())
        if missing:
            gaps.append({"sub": sp.get("name"), "missing": sorted(missing)})
    return {
        "vendor": name,
        "score": max(0, 100 - 15 * len(gaps)),
        "gaps": gaps,
        "pass": len(gaps) == 0,
    }

report = score_vendor("cloud-a", json.loads(Path("cloud-a-subs.json").read_text()))
print(json.dumps(report, ensure_ascii=False, indent=2))

subprocessors 变更应触发增量评估；与 llm vendor risk 登记册联动（Owner、复审日、残余风险）。

合同

vendor dpa（数据处理协议）应覆盖：处理目的、数据类别、保留期、子处理器、跨境机制、审计权、删除与返还、安全事件通知。

跨境传输三路径

路径适用工程动作
境内处理 金融/政务默认 路由仅境内 endpoint；网络 ACL 阻断境外 DNS
scc mechanism 集团内跨境、已签署 SCC 的 SaaS 合同附录列明字段；传输加密 + 访问日志
安全评估 / 认证 关键信息基础设施、大规模个人信息出境 法务主导；工程提供 data mapping 与影响分析

cross border transfer 在 LLM 场景易被忽略的细节：

  • 支持工单系统可能在境外
  • 可观测性 SaaS（trace/log）默认美国区
  • 开源模型下载与权重更新 CDN
  • 员工境外远程运维

合同必备条款（工程可验收）

yaml 
# contract-checklist.yaml（摘录）
mandatory_clauses:
  - id: NO_TRAIN_DEFAULT
    text: "客户数据默认不用于模型训练；opt-in 需书面同意"
  - id: DELETE_ON_EXIT
    text: "合同终止 30 日内删除推理日志与缓存；提供删除证明"
  - id: SUBPROCESSOR_NOTICE
    text: "子处理器变更提前 30 日通知；客户可反对"
  - id: AUDIT_RIGHT
    text: "每年一次第三方审计报告或 SOC2 摘要"
  - id: EXIT_PORTABILITY
    text: "提供模型接口兼容期 ≥90 天；协助迁移嵌入维度文档"
exit_clause:
  rto_days: 14
  deliverables: [delete_certificate, last_30d_audit_sample, embedding_spec]

exit clause 应与技术 exit clause 演练一致：每季度做一次「切换备用模型」演练，验证 Prompt 模板、工具 schema、向量维度无需重写。

LLM 供应商常见问题清单（尽调访谈）

训练数据是否包含我方 Prompt？默认 opt-out 还是 opt-in？
要求书面答复并引用合同条款编号；工程侧验证 API 是否支持 X-No-Train: true 且默认开启。
日志保留多久？谁可以访问？是否用于产品改进？
保留期写入 DPA；访问地列入子处理器；改进用途需单独同意或匿名化。
模型权重或 API 下线时提前多久通知？
合同要求 ≥90 天兼容期；工程维护备用模型与 embedding 维度对照表。
是否支持客户自带密钥（BYOK）与境内 KMS？
金融客户常要求；政务客户常要求密钥不出境。无法支持则标注残余风险。

数据出境影响分析（DPIA 工程附件）

向法务提供一页纸摘要：

  • 数据类别：Prompt、Completion、Embedding、审核日志、工单附件。
  • 规模：日活用户、峰值 QPS、单条最大 token、留存周期。
  • 接收方：模型商、CDN、可观测性 SaaS、外包运维。
  • 必要性：是否可境内替代；替代成本与延迟影响。
  • 措施：加密、脱敏、访问控制、删除与审计。

退出演练交付物清单

  • 模型 API 兼容期证明与迁移窗口公告
  • 向量维度与索引格式说明（避免锁死供应商）
  • 删除证书 + 最后 30 日审计抽样
  • 子处理器终止服务书面确认
  • 备用模型切换 runbook 与 RTO 实测记录

每季度演练切换备用模型，记录 P95 延迟与回答质量差异；演练结论写入 llm vendor risk 登记册，作为下年度采购依据。

采购合同应明确：客户数据、模型输出、日志三类数据的删除时限与证明格式；工程验收以脚本抓取删除 API 回执为准。

子处理器变更通知流程

  1. 厂商提前 30 日邮件通知 + JSON 机器可读 diff。
  2. 法务评估出境与 DPA 覆盖；工程更新 data mapping 图。
  3. 客户反对权：触发备用 vendor 或停用相关功能。
  4. 更新 assessment-report.json 并重新跑 vendor_assessment.py。

投标技术卷附录建议顺序：数据流图 → 评估打分表 → DPA 条款对照 → 退出演练纪要 → 子处理器 JSON。法务审阅签字后再冻结版本号。

评估阶段角色分工

  • 架构：data mapping 与跨境边。
  • 安全：打分卡与红队问题清单回复。
  • 法务：DPA/SCC/安评路径。
  • 采购：退出条款与 SLA 违约金。

以上分工写入 RFP 应答表，避免评估期临时抓人。

本章结论

  • 评估：打分卡 + 数据流图 + 子处理器机器可读清单；变更触发复审。
  • 合同：DPA 覆盖训练/日志/审计/删除；出境走境内/SCC/安评；退出可演练。
