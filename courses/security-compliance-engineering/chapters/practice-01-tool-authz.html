本章先记住 3 件事
  • 动态注册：ChatClient 可见工具集 = f(用户角色, 会话意图)，非静态全量。
  • 二次鉴权：模型输出的 tool call 只是「意图」；ERP API 仍验 JWT + 业务规则。
  • 能力边界：capability control 含步数预算、单笔金额上限、人工确认。

白名单

CorpAssist · 客服 Agent 普通用户会话仅开放 query_order、faq_search；VIP 座席模式额外开放 apply_refund，且须二次 OTP。模型永远看不到 admin_delete_user。

Spring：按 SecurityContext 注册 Tool

java
List<ToolCallback> allowed = toolRegistry.forRole(
    SecurityContextHolder.getContext().getAuthentication().getAuthorities());
return chatClient.prompt(userMsg)
    .toolCallbacks(allowed)  // 非 defaultTools(ALL)
    .call();

Python：LangGraph tool 过滤

python
def bind_tools_for_session(tools: list, jwt_claims: dict) -> list:
    allow = ROLE_TOOL_MAP[jwt_claims["role"]]
    return [t for t in tools if t.name in allow]
先想 10 秒：prompt 里写「你没有 refund 工具」能否代替白名单？

不能。须从 API 层不注册 + 服务端拒绝未知 tool name 双保险。

OAuth

工具背后调用 ERP/CRM 时，使用用户委派 token 或 client-credentials + 用户上下文。Scope 须窄于应用默认权限。

yaml
# tool-oauth-scopes.yaml
tools:
  query_order:
    required_scopes: ["orders:read"]
  apply_refund:
    required_scopes: ["orders:read", "refunds:write"]
    extra_checks: ["amount_limit", "otp_if_over_500"]

Spring @Tool 方法内注入 OAuth2AuthorizedClient；Python 侧 httpx 携带 user token，401 时勿让模型重试换参数绕过。

沙箱

tool sandbox 用于代码执行、SQL、Shell 类工具：网络隔离、只读 FS、超时与输出长度上限。

高风险工具沙箱
flowchart LR
  Agent --> Gate[capability control]
  Gate -->|pass| Sandbox[gVisor/容器]
  Sandbox --> RO[只读 DB 副本]
  Sandbox -->|deny| Audit[audit trail]

MCP 服务器须 mcp auth：mutual TLS + 每 server 独立 allowlist，禁止 Agent 任意挂载社区 MCP。

本章结论

  • 白名单：按 role-tool-map 动态注册；prompt 不能替代 API 层拒绝。
  • OAuth：工具级 scope + 业务规则（金额/OTP）二次校验。
  • 沙箱：代码/SQL 类工具必须隔离；MCP 须认证与 allowlist。
