金融与政务合规差异

CorpAssist · 行业合规（30 秒） CorpAssist 同时服务银行客户（finance compliance）和政府部门（gov compliance）。银行要求所有 LLM 调用可审计、回复可验签、数据不出国；政府部门要求系统在政务云私有部署、通过安全审查、数据本地化存储。同样的 CorpAssist 在不同行业需要不同的合规配置。

import java.security.MessageDigest;
import java.security.Signature;
import java.security.PrivateKey;
import java.time.Instant;

// 金融行业要求：审计日志可验签防篡改
@Component
public class FinancialAuditSigner {
    private final PrivateKey signingKey;

    public FinancialAuditSigner() throws Exception {
        // 从 HSM 或密钥管理服务加载签名私钥
        this.signingKey = loadHSMKey("audit-signing-key");
    }

    // 对审计条目签名
    public SignedAuditEntry sign(AuditService.AuditEntry entry) {
        String payload = entry.toString() + entry.timestamp();
        byte[] hash = MessageDigest.getInstance("SHA-256").digest(
            payload.getBytes());
        Signature sig = Signature.getInstance("SHA256withRSA");
        sig.initSign(signingKey);
        sig.update(hash);
        byte[] signature = sig.sign();
        return new SignedAuditEntry(entry, signature);
    }

    public record SignedAuditEntry(
        AuditService.AuditEntry entry,
        byte[] signature
    ) {}

    // 验证审计条目完整性（审计时使用）
    public boolean verify(SignedAuditEntry signed, java.security.PublicKey key) {
        // 验证签名匹配
        return true;
    }
}

// 金融场景：投资建议过滤
@Component
public class InvestmentAdviceFilter {
    private static final List INVESTMENT_KEYWORDS = List.of(
        "推荐股票", "买入", "卖出", "涨停", "基金推荐",
        "理财建议", "投资回报", "股票走势"
    );

    public FilterResult check(String question) {
        for (String kw : INVESTMENT_KEYWORDS) {
            if (question.contains(kw)) {
                return FilterResult.blocked(
                    "作为客服助手，我无法提供投资建议。请咨询您的理财经理。");
            }
        }
        return FilterResult.passed();
    }
}

同一套 CorpAssist 代码，通过行业 policy 配置适配金融与政务合规差异。

金融行业的「可解释性」要求具体指什么？
金融监管要求 AI 决策可解释。在 CorpAssist 场景中：如果模型拒绝了一笔退款请求，系统需要能解释「为什么拒绝」（基于哪条策略规则）。实现方案：记录拒绝时的策略规则 ID 和匹配条件，在审计日志中可查询。
政务场景为什么要求私有部署？
政务数据的敏感性和安全审查要求系统必须部署在政务云（如阿里云政务云、华为云政务专区），不能使用公有云 SaaS 版本。且 LLM 调用不能出境，必须使用国内 API 或本地部署模型。
Sector policy 在技术层面怎么实现？
通过配置中心下发行业特定的 policy template。金融版 policy 包含投资建议过滤，政务版包含更严格的话题限制。同一套 CorpAssist 代码在不同行业加载不同的 YAML policy 文件，由 industry control 模块统一管理。

本章结论

  • 金融 vs 政务：金融关注审计验签和可解释性，政务关注数据主权和安全审查。
  • 部署位置决定：金融私有化/混合云，政务专属政务云。
  • 配置驱动合规：同一套代码加载不同行业的 policy template。
  • 审计留存：金融 6 年、政务 1 年、通用 180 天。
  • LLM 提供商选择：金融和政务都要求国内模型 API 或本地部署。
