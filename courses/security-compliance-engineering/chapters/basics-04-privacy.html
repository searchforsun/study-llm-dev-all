PII

常见 PII：手机号、身份证、邮箱、银行卡、住址、生物特征。脱敏方式包括掩码、哈希、令牌化与合成数据替换。

字段展示策略存储策略
手机138****8000哈希 + 盐
身份证110**********1234令牌化 vault
邮箱z***@corp.com加密列

import re
PHONE = re.compile(r"1[3-9]\d{9}")
def mask_phone(text): return PHONE.sub("[PHONE]", text)

识别 pii 应结合正则、NER 与业务词典；财务场景还需识别金额与合同编号。个保法下的最小必要原则要求：仅采集完成 CorpAssist 功能所必需的字段，并在隐私政策中明示用途。用户行使删除权（DSR）时，须同步擦除会话、向量 chunk 与可逆 vault 映射，并出具受理回执。

双栈共享脱敏配置（YAML 片段）

yaml
pii_rules:
  - id: phone_cn
    pattern: '1[3-9]\d{9}'
    mask: "[PHONE]"
  - id: id_card
    pattern: '\d{17}[\dXx]'
    mask: "[ID_CARD]"

日志

log sanitization 必须在写入前执行；禁止 DEBUG 打印完整 prompt。结构化日志使用占位符 [PHONE] 而非原文，并与 data masking 规则引擎共用配置源。

CorpAssist · 排障与隐私 研发要求「临时打开 DEBUG 看完整 prompt」排障。正确做法：用 trace_id 在脱敏审计库查摘要，或短时采样经审批的 shadow 环境，禁止生产明文日志。
日志脱敏流水线
flowchart LR
    E[事件] --> M[PII 规则引擎]
    M --> L[结构化日志]
    L --> S[SIEM/审计]

所有出口日志经同一脱敏引擎，避免各服务各写一套规则。

日志脱敏检查清单

  • DEBUG 是否全局禁用完整 prompt？
  • trace_id 是否贯穿网关到 Advisor？
  • SIEM 字段是否仅含占位符标签？
  • 日志保留是否满足等保最小周期？

不可以。应使用 hash/掩码；排障用 trace_id 关联，而非明文 PII。
embedding 是否算 PII？

存储

retention policy 规定会话、向量、审计表保留天数；tokenization vault 保存可逆令牌映射，与应用库分离。

长期记忆是否算存储？
是。跨会话记忆须 TTL、租户隔离，并纳入 DSR 删除范围。
脱敏后还能做个性化吗？
可以。用不可逆 hash 或令牌 ID 关联偏好，避免在向量中存明文 PII。

Java：输出侧 PII 掩码 Advisor

java
public AdvisedResponse aroundCall(AdvisedRequest req, CallAroundAdvisorChain chain) {
    AdvisedResponse resp = chain.nextAroundCall(req);
    String masked = piiRuleEngine.mask(resp.getContent());
    return resp.mutate().content(masked).build();
}

输出掩码与 C26 输出审核互补：前者去 PII，后者拦违法与泄密承诺。二者顺序建议：先生成 → PII 掩码 → 内容政策审核。

  1. 分块入库前运行 redact()，禁止明文身份证进入 embedding。
  2. 检索结果返回前再次掩码展示字段。
  3. 导出接口单独审批，与聊天掩码策略解耦。
